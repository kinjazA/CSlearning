上一节学了控制返回**什么数据**（响应模型），这一节学控制返回**什么状态码**。状态码就是 HTTP 响应里那个三位数的数字：200、201、404、500……

客户端拿到这个数字，第一时间就能知道请求是成功了、失败了、还是需要做其他处理。这一节内容很短，语法就一行代码，但背后的 HTTP 状态码体系值得理解透

---

## 1. 语法：一行搞定

```python
from fastapi import FastAPI

app = FastAPI()


@app.post("/items/", status_code=201)
def create_item(name: str):
    return {"name": name}
```

就是在装饰器里加一个 `status_code=201`，搞定。**注意**：`status_code` 是**装饰器的参数**，不是函数的参数。和 `response_model` 一样，写在 `@app.post(...)` 里面，不是写在 `def create_item(...)` 里面

```python
# ✅ 正确：写在装饰器里
@app.post("/items/", status_code=201)
def create_item(name: str):
    ...

# ❌ 错误：不是函数参数
@app.post("/items/")
def create_item(name: str, status_code: int = 201):   # 这是查询参数，不是状态码！
    ...
```

---

## 2. 不写 `status_code` 会怎样

如果你不写，FastAPI 默认返回 **200**（OK）

```python
@app.get("/items/")
def read_items():
    # 没写 status_code，默认返回 200
    return [{"name": "Foo"}]
```

大部分 GET 请求用默认的 200 就够了。**需要手动设 `status_code` 的主要场景是 POST（创建资源）**——按 HTTP 规范，创建资源应该返回 201，不是 200

## 3. HTTP 状态码速查

### 3.1 五大类

```
1xx（信息）    → 请求正在处理中，几乎不用
2xx（成功）    → 请求成功了
3xx（重定向）  → 要去别的地方拿数据
4xx（客户端错误）→ 你（客户端）搞错了
5xx（服务器错误）→ 我（服务器）出问题了
```

### 3.2 在 API 开发中真正会用到的状态码

**2xx 成功系列**：

```css
200 OK
    含义：请求成功，一切正常
    场景：GET 查询数据、PUT 更新数据
    是否有响应体：有

201 Created
    含义：资源创建成功
    场景：POST 创建新记录（用户注册、新建项目、提交投标）
    是否有响应体：有（通常返回创建好的资源）

204 No Content
    含义：操作成功，但没有内容返回
    场景：DELETE 删除资源（删完了就没东西可返回了）
    是否有响应体：没有！这个状态码不允许有响应体
```

**4xx 客户端错误系列**：

```css
400 Bad Request
    含义：请求有问题，但不是特定的错误类型
    场景：通用的客户端错误

401 Unauthorized
    含义：没有认证（没登录）
    场景：需要登录但没提供 Token

403 Forbidden
    含义：认证了但没权限
    场景：普通用户试图访问管理员接口

404 Not Found
    含义：请求的资源不存在
    场景：查询一个不存在的项目 ID

409 Conflict
    含义：和现有数据冲突
    场景：注册时用户名已存在

422 Unprocessable Entity
    含义：数据格式不对，验证失败
    场景：FastAPI 自动返回的——类型错误、约束不满足等
```

**5xx 服务器错误系列**：

```css
500 Internal Server Error
    含义：服务器内部出错了
    场景：你的代码有 bug、数据库连不上等（不需要你手动返回，出错时自动返回）
```

### 3.3 一张表记住

```css
操作           推荐状态码     含义
──────────────────────────────────────
查询（GET）      200         OK
创建（POST）     201         Created
更新（PUT）      200         OK
删除（DELETE）   204         No Content
数据验证失败     422         Unprocessable Entity（FastAPI 自动）
资源不存在       404         Not Found
没有登录        401         Unauthorized
没有权限        403         Forbidden
```

## 4. 用常量代替魔法数字

### 4.1 问题：201 是什么意思？

```python
@app.post("/items/", status_code=201)
```

三个月后回来看这行代码，201 是什么意思？得查一下。这就是"魔法数字"的问题——数字本身没有语义

### 4.2 用 `fastapi.status` 模块

```python
from fastapi import FastAPI, status

app = FastAPI()


@app.post("/items/", status_code=status.HTTP_201_CREATED)
def create_item(name: str):
    return {"name": name}
```

`status.HTTP_201_CREATED` 和 `201` 是完全一样的值，但代码可读性好得多——一眼就知道是"已创建"

### 4.3 编辑器自动补全

`status` 模块的最大好处是**编辑器自动补全**。输入 `status.HTTP_` 然后按 Tab，编辑器会列出所有可用的状态码：

```python
status.HTTP_200_OK
status.HTTP_201_CREATED
status.HTTP_204_NO_CONTENT
status.HTTP_400_BAD_REQUEST
status.HTTP_401_UNAUTHORIZED
status.HTTP_403_FORBIDDEN
status.HTTP_404_NOT_FOUND
status.HTTP_409_CONFLICT
status.HTTP_422_UNPROCESSABLE_ENTITY
status.HTTP_500_INTERNAL_SERVER_ERROR
# ... 还有很多
```

不需要记住数字，编辑器帮你选

### 4.4 也可以用 Python 标准库

```python
from http import HTTPStatus

@app.post("/items/", status_code=HTTPStatus.CREATED)
def create_item(name: str):
    return {"name": name}
```

`HTTPStatus.CREATED` 也是 201，来自 Python 标准库。两种方式都行，`fastapi.status` 更常用因为命名更直观（`HTTP_201_CREATED` 比 `HTTPStatus.CREATED` 多了数字，更好记）

## 5. 204 No Content 的特殊处理

### 5.1 问题

```python
@app.delete("/items/{item_id}", status_code=status.HTTP_204_NO_CONTENT)
def delete_item(item_id: int):
    # 删除操作...
    return {"message": "deleted"}   # ❌ 204 不允许有响应体！
```

HTTP 规范规定 204 响应**不能有响应体**。如果你返回了数据，客户端可能会报错或忽略

### 5.2 正确做法

```python
from fastapi import FastAPI, status

app = FastAPI()


@app.delete("/items/{item_id}", status_code=status.HTTP_204_NO_CONTENT)
def delete_item(item_id: int):
    # 执行删除操作...
    # 不返回任何内容（或者 return None）
    return
```

FastAPI 很聪明——当你设了 204，它会在 OpenAPI 文档里标注"此接口没有响应体"，让客户端开发者知道不需要解析响应

## 6. `status_code` 只是"默认"状态码

`status_code` 设置的是**正常情况下的默认状态码**。异常情况（比如资源不存在返回 404）不在这里处理——那是后面学的 `HTTPException` 的工作

```python
from fastapi import FastAPI, HTTPException, status

app = FastAPI()

items_db = {"foo": {"name": "Foo"}}


@app.get("/items/{item_id}", status_code=status.HTTP_200_OK)
def read_item(item_id: str):
    if item_id not in items_db:
        # 异常情况：返回 404
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"Item {item_id} not found"
        )
    # 正常情况：返回 200（装饰器里设的默认值）
    return items_db[item_id]
```

**分工**：

- 装饰器的 `status_code` → 正常成功时返回什么状态码
- `HTTPException` → 异常情况时返回什么状态码（后面会详细学）

## 7. 对应到你的投标项目

```python
from uuid import UUID, uuid4
from datetime import datetime
from decimal import Decimal

from fastapi import FastAPI, HTTPException, status
from pydantic import BaseModel, Field

app = FastAPI(title="智能投标决策系统")


# ==================== 模型 ====================

class ProjectBase(BaseModel):
    project_name: str = Field(min_length=1, max_length=200)
    location: str
    control_price: Decimal = Field(gt=0)

class ProjectCreate(ProjectBase):
    pass

class ProjectPublic(ProjectBase):
    id: UUID
    created_at: datetime


# ==================== 模拟数据库 ====================

fake_db: dict[UUID, dict] = {}


# ==================== 各种操作对应各种状态码 ====================

@app.post(
    "/api/projects/",
    status_code=status.HTTP_201_CREATED,    # 创建成功 → 201
    response_model=ProjectPublic,
)
def create_project(project: ProjectCreate):
    """
    创建项目 → 返回 201 Created

    201 告诉客户端："新资源创建好了，
    响应体里就是创建好的资源。"
    """
    project_id = uuid4()
    now = datetime.now()
    project_data = {
        **project.model_dump(),
        "id": project_id,
        "created_at": now,
    }
    fake_db[project_id] = project_data
    return project_data


@app.get(
    "/api/projects/{project_id}",
    status_code=status.HTTP_200_OK,         # 查询成功 → 200（默认值，可以省略）
    response_model=ProjectPublic,
)
def get_project(project_id: UUID):
    """
    查询项目 → 成功返回 200 OK
                不存在返回 404 Not Found
    """
    if project_id not in fake_db:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"项目 {project_id} 不存在"
        )
    return fake_db[project_id]


@app.put(
    "/api/projects/{project_id}",
    status_code=status.HTTP_200_OK,         # 更新成功 → 200
    response_model=ProjectPublic,
)
def update_project(project_id: UUID, project: ProjectCreate):
    """
    更新项目 → 成功返回 200 OK
    """
    if project_id not in fake_db:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"项目 {project_id} 不存在"
        )
    existing = fake_db[project_id]
    updated = {
        **existing,
        **project.model_dump(),
    }
    fake_db[project_id] = updated
    return updated


@app.delete(
    "/api/projects/{project_id}",
    status_code=status.HTTP_204_NO_CONTENT,  # 删除成功 → 204（无响应体）
)
def delete_project(project_id: UUID):
    """
    删除项目 → 成功返回 204 No Content

    204 表示"操作成功了，但没有内容返回"。
    函数不返回任何东西。
    """
    if project_id not in fake_db:
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"项目 {project_id} 不存在"
        )
    del fake_db[project_id]
    # 不 return 任何东西 → 204 No Content
```

这就是一个标准的 CRUD 接口的状态码设计：

```css
POST   创建 → 201 Created
GET    查询 → 200 OK
PUT    更新 → 200 OK
DELETE 删除 → 204 No Content
找不到  → 404 Not Found（用 HTTPException 抛出）
```

## 8. 本节核心知识点速查

```css
需求                      做法
──────────────────────────────────────────────────────────────────
设置默认状态码             @app.post("/items/", status_code=201)
用常量代替数字             status_code=status.HTTP_201_CREATED
GET 成功                  200 OK（默认，可以不写）
POST 创建成功             201 Created
DELETE 删除成功           204 No Content（不能有响应体）
资源不存在                404 Not Found（用 HTTPException）
验证失败                  422（FastAPI 自动返回，不用你管）
```

**本节最核心的两句话**：

1. 在装饰器里加 `status_code` 设置默认状态码，用 `fastapi.status` 模块的常量代替魔法数字，让代码自解释

2. `status_code` 只管"正常成功时返回什么码"。异常情况（404、401 等）交给 `HTTPException` 处理，后面会学
