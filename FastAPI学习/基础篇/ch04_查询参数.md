## 1. 什么是查询参数

### 1.1 先看一个真实的 URL

你在网上搜索的时候，注意过浏览器地址栏吗？

```css
https://www.google.com/search?q=fastapi&lang=zh&page=2
                               └──────────────────────┘
                               这里就是查询参数
```

`?` 后面的部分，就是查询参数。多个参数之间用 `&` 分隔。拆开来看：

```css
?q=fastapi  &  lang=zh  &  page=2
 ↑              ↑            ↑
 键=值          键=值         键=值

q    = "fastapi"    → 搜索关键词
lang = "zh"         → 语言
page = "2"          → 第几页
```

### 1.2 路径参数 vs 查询参数

```css
/projects/42?page=2&keyword=路面
 └────┬────┘ └────────┬────────┘
   路径参数         查询参数

路径参数：URL 路径的一部分，用来定位"哪个资源"
查询参数：附加的筛选/配置条件，用来控制"怎么获取"
```

## 2. FastAPI 怎么判断哪个是查询参数

### 2.1 核心规则：排除法

FastAPI 判断参数类型的逻辑非常简单：

```python
@app.get("/items/{item_id}")
def read_item(item_id: str, skip: int = 0, limit: int = 10):
    ...
```

FastAPI 看到函数有三个参数，逐个判断：

```
item_id → 名字出现在路径的 {item_id} 里 → 路径参数
skip    → 名字没出现在路径里             → 查询参数
limit   → 名字没出现在路径里             → 查询参数
```

**就这么简单：==在路径花括号里出现过的就是路径参数，没出现过的就是查询参数。==** 不需要做任何额外的标记配置

## 3. 查询参数的三种状态

一个查询参数到底是"必须传"还是"可以不传"，完全取决于你怎么定义它

### 3.1 有默认值 → 可选参数（不传就用默认值）

```python
@app.get("/items/")
def read_items(skip: int = 0, limit: int = 10):
    ...
```

`skip` 有默认值 `0`，`limit` 有默认值 `10`。用户可以不传，FastAPI 自动用默认值填充

```
GET /items/         → skip=0, limit=10  ✅ 合法
GET /items/?skip=5  → skip=5, limit=10  ✅ 合法
```

### 3.2 默认值为 None → 可选参数（不传就是 None）

```python
@app.get("/items/{item_id}")
def read_item(item_id: str, q: str | None = None):
    if q:
        return {"item_id": item_id, "q": q}
    return {"item_id": item_id}
```

```
GET /items/foo         → item_id="foo", q=None       ✅ 合法
GET /items/foo?q=搜索   → item_id="foo", q="搜索"    ✅ 合法
```

**`str | None = None` 拆解**：

```python
q: str | None = None
   ↑     ↑      ↑
   │     │      └── 默认值：如果用户不传，q 就是 None
   │     └──────── 允许的类型之一：None
   └────────────── 允许的类型之一：str

# 连起来读：q 可以是字符串，也可以是 None，默认是 None
```

> **Python 3.9 及以下**的写法：`q: Optional[str] = None`，效果完全一样

### 3.3 没有默认值 → 必填参数（不传就报错）

```python
@app.get("/items/{item_id}")
def read_item(item_id: str, needy: str):
    """
    needy 没有默认值 → 必须传，不传就报 422 错误
    """
    return {"item_id": item_id, "needy": needy}
```

```
GET /items/foo               → 422 错误！缺少必填参数 needy
GET /items/foo?needy=hello   → ✅ 合法，needy="hello"
```

报错信息长这样：

```json
{
    "detail": [
        {
            "type": "missing",
            "loc": ["query", "needy"],
            "msg": "Field required",
            "input": null
        }
    ]
}
```

- `"loc": ["query", "needy"]` → 错误在查询参数 `needy` 上
- `"msg": "Field required"` → 这个字段是必填的

### 3.4 三种状态对比总结

```python
@app.get("/items/{item_id}")
def read_item(
    item_id: str,                   # 路径参数（永远必填）
    needy: str,                     # 查询参数 - 必填（没有默认值）
    skip: int = 0,                  # 查询参数 - 可选（默认值 0）
    limit: int | None = None,       # 查询参数 - 可选（默认值 None）
):
    return {
        "item_id": item_id,
        "needy": needy,
        "skip": skip,
        "limit": limit
    }
```

**判断规则就一句话：==有默认值就是可选的，没默认值就是必填的==**

```css
写法                              是否必填      不传时的值
────────────────────────────────────────────────────────
needy: str                        必填          报 422 错误
skip: int = 0                     可选          0
limit: int | None = None          可选          None
```

**`= 0` 和 `= None` 的区别**：

- `= 0` 表示"有个合理的默认行为"——用户不传就从第 0 条开始，这是个有意义的默认值
- `= None` 表示"这个参数不存在"——用户不传就当它没有，代码里需要判断 `if limit is not None`

选哪个取决于你的业务逻辑

## 4. 布尔类型的查询参数

### 4.1 FastAPI 对 bool 类型特别宽容

```python
@app.get("/items/{item_id}")
def read_item(item_id: str, short: bool = False):
    """
    short 参数控制是否返回简短版本
    """
    item = {"item_id": item_id}
    if not short:
        # short 为 False 时，返回详细信息
        item["description"] = "这是一段很长的详细描述..."
    return item
```

以下这些 URL 中 `short` 的值**全部是 `True`**：

```
/items/foo?short=true
/items/foo?short=True
/items/foo?short=TRUE
/items/foo?short=1
/items/foo?short=yes
/items/foo?short=on
```

以下这些 `short` 的值**全部是 `False`**：

```
/items/foo?short=false
/items/foo?short=False
/items/foo?short=0
/items/foo?short=no
/items/foo?short=off
```

**为什么要支持这么多写法？**

因为不同系统、不同前端框架传布尔值的习惯不一样。有的传 `true/false`，有的传 `1/0`，有的 `yes/no`。FastAPI 全部兼容

### 4.2 什么时候用 bool 查询参数

典型场景就是"开关"类的控制：

```python
@app.get("/api/predict")
def predict(
    verbose: bool = False,     # 是否返回详细计算过程
    include_chart: bool = True # 是否包含图表数据
):
    ...
```

## 5. 路径参数和查询参数混合使用

### 5.1 FastAPI 怎么区分它们

再强调一遍这个规则：

```python
@app.get("/users/{user_id}/items/{item_id}")
def read_user_item(
    user_id: int,                      # → 路径参数（在路径的 {} 里）
    item_id: str,                      # → 路径参数（在路径的 {} 里）
    q: str | None = None,             # → 查询参数（不在路径里）
    short: bool = False,               # → 查询参数（不在路径里）
):
    item = {"item_id": item_id, "owner_id": user_id}
    if q:
        item["q"] = q
    if not short:
        item["description"] = "详细描述..."
    return item
```

**完整的请求 URL 长这样**：

```css
GET /users/5/items/abc?q=搜索&short=true
    └─┬──┘   └─┬─┘    └──────┬───────┘
   user_id   item_id      查询参数
    = 5      = "abc"    q="搜索", short=True
```

### 5.2 函数参数的顺序重要吗？

**不重要。** FastAPI 靠参数名匹配，不靠顺序：

```python
# 这两种写法效果完全一样

# 写法一：路径参数在前
def read_user_item(user_id: int, item_id: str, q: str | None = None):
    ...

# 写法二：查询参数在前（不推荐但合法）
def read_user_item(q: str | None = None, user_id: int, item_id: str):
    ...
```

但是**推荐把路径参数写前面，查询参数写后面**，这样代码读起来更清晰

### 5.3 但 Python 语法对顺序有要求

虽然 FastAPI 不在乎顺序，但 Python 本身有一个语法规则：**有默认值的参数必须写在没默认值的参数后面**

```python
# ❌ Python 语法错误（和 FastAPI 无关）
def read_item(skip: int = 0, item_id: str):
    ...
# SyntaxError: non-default argument follows default argument
# 意思是：没默认值的 item_id 不能写在有默认值的 skip 后面


# ✅ 正确
def read_item(item_id: str, skip: int = 0):
    ...
```

这不是 FastAPI 的规则，是 Python 语言本身的规则。记住就行：**必填参数写前面，有默认值的写后面**

## 6. 一个完整的例子：把所有情况串起来

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/api/bids/{bid_id}")
def get_bid_detail(
    # ---- 路径参数（必填，名字在路径的 {} 里） ----
    bid_id: int,

    # ---- 必填查询参数（没有默认值，不传就报错） ----
    project_name: str,

    # ---- 有默认值的查询参数（可选，不传用默认值） ----
    page: int = 1,
    page_size: int = 20,
    verbose: bool = False,

    # ---- 默认 None 的查询参数（可选，不传就是 None） ----
    keyword: str | None = None,
):
    """
    获取投标记录详情

    请求示例：
    GET /api/bids/42?project_name=路面改造&page=2&keyword=南昌&verbose=true

    解析结果：
    bid_id = 42              (路径参数，int)
    project_name = "路面改造" (必填查询参数)
    page = 2                 (有默认值，用户传了 2)
    page_size = 20           (有默认值，用户没传，用默认值 20)
    verbose = True           (有默认值，用户传了 true)
    keyword = "南昌"          (可选，用户传了)
    """
    result = {
        "bid_id": bid_id,
        "project_name": project_name,
        "page": page,
        "page_size": page_size,
    }

    if keyword:
        result["filter"] = f"按关键词 '{keyword}' 筛选"

    if verbose:
        result["detail"] = "包含完整的计算过程和中间数据..."

    return result
```

## 7. 实际开发中的常见模式

### 7.1 分页查询（最常用）

几乎每个列表接口都需要分页，这是固定套路：

```python
@app.get("/api/projects")
def list_projects(
    page: int = 1,           # 第几页，默认第 1 页
    page_size: int = 20,     # 每页多少条，默认 20 条
    sort_by: str = "created_at",  # 排序字段
):
    # skip 计算：第 1 页跳过 0 条，第 2 页跳过 20 条...
    skip = (page - 1) * page_size
    # 然后去数据库查询 skip 和 limit
    return {
        "page": page,
        "page_size": page_size,
        "skip": skip,
        "sort_by": sort_by
    }
```

### 7.2 搜索过滤（常用）

```python
@app.get("/api/bids")
def search_bids(
    keyword: str | None = None,      # 关键词搜索
    min_price: float | None = None,  # 最低价格
    max_price: float | None = None,  # 最高价格
    won: bool | None = None,         # 是否中标（None 表示不筛选）
    page: int = 1,
    page_size: int = 20,
):
    """
    GET /api/bids?keyword=路面&min_price=500&won=true&page=2

    所有筛选条件都是可选的，
    用户可以只用其中几个，不用的就不传
    """
    filters = {}
    if keyword:
        filters["keyword"] = keyword
    if min_price is not None:    # 注意：这里用 is not None 而不是 if min_price
        filters["min_price"] = min_price  # 因为 min_price=0 也是合法的值
    if max_price is not None:
        filters["max_price"] = max_price
    if won is not None:
        filters["won"] = won

    return {"filters": filters, "page": page, "page_size": page_size}
```

**注意 `if min_price is not None` 和 `if min_price` 的区别**：

```python
min_price = 0       # 合法值，表示"最低价格为 0"
min_price = None    # 表示"用户没传这个参数"

if min_price:       # 0 是 false，会被当成"没传" → 错误！
if min_price is not None:  # 只有 None 才被当成"没传" → 正确！
```

### 7.3 开关控制（常用）

```python
@app.get("/api/predictions/{pred_id}")
def get_prediction(
    pred_id: int,
    include_history: bool = False,    # 是否包含历史数据
    include_chart: bool = True,       # 是否包含图表数据
    format: str = "json",            # 返回格式：json / csv
):
    """
    GET /api/predictions/7?include_history=true&format=csv
    """
    ...
```

## 8. 容易混淆的点

### 8.1 查询参数和请求体的区别

这一节只讲了查询参数，后面还会学**请求体**（Request Body）。先预告一下它们的区别：

```css
查询参数：放在 URL 的 ? 后面，适合简单的筛选条件
  GET /items/?page=2&keyword=搜索

请求体：放在 HTTP 请求的 Body 里，适合复杂的结构化数据
  POST /predict
  Body: {"control_price": 1200, "competitors": 8, "history": [...]}
```

**什么时候用查询参数，什么时候用请求体？**

```css
查询参数适合：
  - 简单的键值对（数字、字符串、布尔值）
  - 筛选、分页、排序这类"控制类"参数
  - GET 请求（GET 请求一般不带请求体）

请求体适合：
  - 复杂的嵌套数据结构
  - 数据量比较大的情况
  - 创建/更新资源时提交的数据
  - POST / PUT / PATCH 请求
```

### 8.2 查询参数的值永远从 URL 来

不管你用什么工具发请求（浏览器、curl、Postman、前端代码），查询参数都是拼在 URL 后面的：

```bash
# curl
curl "http://127.0.0.1:8000/items/?skip=2&limit=5"

# Python requests
requests.get("http://127.0.0.1:8000/items/", params={"skip": 2, "limit": 5})
# requests 库的 params 参数会自动帮你拼到 URL 后面

# 浏览器
# 直接在地址栏输入完整 URL 就行
```

## 9. 本节核心知识点速查

```css
概念                     判断规则                              示例
──────────────────────────────────────────────────────────────────────────
路径参数               参数名在路径 {} 里                  /items/{item_id}
查询参数               参数名不在路径 {} 里                ?skip=0&limit=10
必填查询参数           没有默认值                          needy: str
可选查询参数           有默认值                            skip: int = 0
可选且可为 None        默认值为 None                       q: str | None = None
布尔查询参数           true/1/yes/on 都算 True            short: bool = False
```

**本节最核心的一句话**：函数参数有没有默认值，决定了它是必填还是可选。在路径花括号里出现过的是路径参数，没出现过的是查询参数
