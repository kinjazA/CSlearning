前面所有"接收数据"的例子，请求体都是 JSON 格式。但在现实世界中，还有一种非常常见的数据提交方式——**HTML 表单**



你在网页上填写的登录框、注册页、搜索栏，点击"提交"按钮后发送的数据，走的就不是 JSON，而是**表单编码**（`application/x-www-form-urlencoded`）



这一节讲的就是：**怎么在 FastAPI 里接收表单数据，而不是 JSON 数据。**核心就一点：**把 `Body()` 换成 `Form()`，其他一切都一样**

---

## 1. 先搞懂：表单数据和 JSON 数据到底有什么区别

### 1.1 同样的数据，两种不同的编码方式

假设用户要登录，提交用户名和密码。同样的数据，可以用两种方式发给服务器：

```css
方式一：JSON（你之前学的）
─────────────────────────────────────────
Content-Type: application/json

{"username": "alice", "password": "secret123"}


方式二：表单编码（这一节学的）
─────────────────────────────────────────
Content-Type: application/x-www-form-urlencoded

username=alice&password=secret123
```

**JSON** 用花括号和引号包裹，键值对用冒号分隔。**表单编码** 用 `key=value` 的格式，多个字段用 `&` 连接——和 URL 查询参数的格式一样

### 1.2 什么时候会收到表单数据

```css
收到 JSON 的场景：
  - 前端用 JavaScript（fetch / axios）发送请求
  - 移动端 App 调接口
  - 其他后端服务调你的 API

收到表单数据的场景：
  - HTML 的 <form> 标签直接提交（最传统的网页表单）
  - OAuth2 密码登录流程（规范要求用表单格式）
  - 某些第三方回调（支付回调、短信回调等）
```

### 1.3 为什么不能统一用 JSON

因为 HTML 的 `<form>` 标签原生只支持两种编码：

```html
<!-- 普通表单 → application/x-www-form-urlencoded -->
<form action="/login" method="post">
    <input name="username" value="alice">
    <input name="password" value="secret123">
    <button type="submit">登录</button>
</form>

<!-- 带文件上传的表单 → multipart/form-data -->
<form action="/upload" method="post" enctype="multipart/form-data">
    <input name="file" type="file">
    <button type="submit">上传</button>
</form>
```

HTML `<form>` 不支持 JSON。如果你想让不写 JavaScript 的纯 HTML 页面也能提交数据，就必须支持表单编码

## 3. 基本用法：用 `Form()` 接收表单字段

### 3.1 语法

```python
from typing import Annotated
from fastapi import FastAPI, Form

app = FastAPI()


@app.post("/login/")
def login(
    username: Annotated[str, Form()],
    password: Annotated[str, Form()],
):
    return {"username": username}
```

**就是把 `Body()` 换成了 `Form()`**，其他一模一样

### 3.2 和 Body、Query、Path、Cookie、Header 的对比

```python
from fastapi import Path, Query, Body, Cookie, Header, Form

# 从 URL 路径取      → Path()
item_id: Annotated[int, Path()]

# 从 URL 查询参数取  → Query()
page: Annotated[int, Query()] = 1

# 从 JSON 请求体取   → Body()
data: Annotated[str, Body()]

# 从 Cookie 取       → Cookie()
session_id: Annotated[str, Cookie()]

# 从 Header 取       → Header()
user_agent: Annotated[str, Header()]

# 从表单数据取       → Form()    ← 新学的
username: Annotated[str, Form()]
```

六种数据来源，六种标记方式，写法完全统一

### 3.3 为什么必须写 `Form()`

和 Cookie、Header 一样——不写的话 FastAPI 会把参数当成**查询参数**处理：

```python
# ❌ 不写 Form()，FastAPI 当它是查询参数
def login(username: str, password: str):
    # FastAPI 从 URL ?username=alice&password=secret 取值
    ...

# ✅ 写了 Form()，FastAPI 知道从表单数据取
def login(
    username: Annotated[str, Form()],
    password: Annotated[str, Form()],
):
    # FastAPI 从 POST 请求体的表单数据取值
    ...
```

### 3.4 Form() 也支持所有约束

```python
@app.post("/login/")
def login(
    username: Annotated[str, Form(
        min_length=3,
        max_length=50,
        description="用户名",
        examples=["alice"],
    )],
    password: Annotated[str, Form(
        min_length=8,
        description="密码，至少 8 位",
    )],
):
    return {"username": username}
```

`Form()` 继承自 `Body()`，所以 `Body()` 支持的所有参数（`min_length`、`max_length`、`gt`、`description`、`examples`、`alias` 等），`Form()` 全都支持

## 4. 关键限制：Form 和 JSON Body 不能混用

### 4.1 规则

**同一个接口里，要么全用 `Form()`，要么全用 `Body()`/Pydantic 模型，不能混着来**

```python
# ✅ 全是 Form
@app.post("/login/")
def login(
    username: Annotated[str, Form()],
    password: Annotated[str, Form()],
):
    ...

# ✅ 全是 JSON（Body / Pydantic 模型）
@app.post("/items/")
def create_item(item: Item):
    ...

# ❌ 混用 → 不行！
@app.post("/bad/")
def bad_endpoint(
    username: Annotated[str, Form()],    # 表单字段
    item: Item,                           # JSON 请求体
):
    ...
```

### 4.2 为什么不能混用

这不是 FastAPI 的限制，是 **HTTP 协议**的限制。一个 HTTP 请求只有一个 `Content-Type`：

```
Content-Type: application/x-www-form-urlencoded   → 整个请求体是表单格式
Content-Type: application/json                    → 整个请求体是 JSON 格式
```

不存在"一半是表单一半是 JSON"的请求。所以同一个接口的请求体要么全是表单，要么全是 JSON

### 4.3 但 Form 可以和 Path / Query / Cookie / Header 一起用

限制只针对**请求体**（Form 和 Body 不能混用）。其他来源的参数（Path、Query、Cookie、Header）和 Form 可以正常配合：

```python
# ✅ Form + Path + Query + Header → 没问题
@app.post("/projects/{project_id}/login")
def project_login(
    project_id: Annotated[int, Path()],               # 路径参数
    redirect: Annotated[str | None, Query()] = None,   # 查询参数
    user_agent: Annotated[str | None, Header()] = None, # Header
    username: Annotated[str, Form()],                   # 表单字段
    password: Annotated[str, Form()],                   # 表单字段
):
    ...
```

## 5. 表单模型：多个 Form 字段打包成 Pydantic 模型

### 5.1 问题

和 Query、Cookie、Header 一样——表单字段多了函数签名就很长：

```python
@app.post("/register/")
def register(
    username: Annotated[str, Form()],
    password: Annotated[str, Form()],
    email: Annotated[str, Form()],
    full_name: Annotated[str | None, Form()] = None,
    phone: Annotated[str | None, Form()] = None,
):
    ...
```

### 5.2 用模型打包

```python
from typing import Annotated
from fastapi import FastAPI, Form
from pydantic import BaseModel

app = FastAPI()


class LoginForm(BaseModel):
    username: str
    password: str


@app.post("/login/")
def login(data: Annotated[LoginForm, Form()]):
    """
    FastAPI 自动从表单数据里提取 username 和 password，
    填进 LoginForm 模型里。
    """
    return {"username": data.username}
```

**和 Query 模型、Cookie 模型、Header 模型完全一样的套路**：

```python
# 查询参数模型
params: Annotated[FilterParams, Query()]

# Cookie 模型
cookies: Annotated[MyCookies, Cookie()]

# Header 模型
headers: Annotated[CommonHeaders, Header()]

# 表单模型
form_data: Annotated[LoginForm, Form()]
```

### 5.3 在函数里怎么用

```python
@app.post("/login/")
def login(data: Annotated[LoginForm, Form()]):
    print(data.username)       # "alice"
    print(data.password)       # "secret123"
    print(data.model_dump())   # {"username": "alice", "password": "secret123"}
    return data
```

### 5.4 表单模型也能加约束

```python
from pydantic import BaseModel, Field, EmailStr


class RegisterForm(BaseModel):
    username: str = Field(min_length=3, max_length=50, description="用户名")
    password: str = Field(min_length=8, description="密码")
    email: EmailStr = Field(description="邮箱地址")
    full_name: str | None = Field(default=None, description="姓名")


@app.post("/register/")
def register(data: Annotated[RegisterForm, Form()]):
    return {"username": data.username, "email": data.email}
```

### 5.5 禁止多余的表单字段

```python
class LoginForm(BaseModel):
    model_config = {"extra": "forbid"}

    username: str
    password: str


@app.post("/login/")
def login(data: Annotated[LoginForm, Form()]):
    return data
```

如果客户端多传了一个 `extra` 字段：

```json
{
    "detail": [
        {
            "type": "extra_forbidden",
            "loc": ["body", "extra"],
            "msg": "Extra inputs are not permitted",
            "input": "some_value"
        }
    ]
}
```

## 6. 怎么测试表单接口

### 6.1 在 /docs 里测试

表单接口在 `/docs` 里能正常测试（不像 Cookie 那样受限）。打开 `/docs`，找到对应的接口，填入表单字段值，点击 "Try it out" → "Execute" 就行

### 6.2 用 curl 测试

```bash
# -d 参数发送表单数据
curl -X POST http://localhost:8000/login/ \
  -d "username=alice&password=secret123"

# 或者用 -F 参数（multipart/form-data 格式，也能用）
curl -X POST http://localhost:8000/login/ \
  -F "username=alice" \
  -F "password=secret123"
```

### 6.3 用 Python requests 测试

```python
import requests

# 注意：用 data= 而不是 json=
response = requests.post(
    "http://localhost:8000/login/",
    data={"username": "alice", "password": "secret123"},  # data= 发表单
)
print(response.json())
```

**关键区别**：

```python
# JSON 请求体 → 用 json= 参数
requests.post(url, json={"key": "value"})

# 表单数据 → 用 data= 参数
requests.post(url, data={"key": "value"})
```

### 6.4 HTML 表单直接提交

```html
<form action="http://localhost:8000/login/" method="post">
    <input name="username" placeholder="用户名">
    <input name="password" type="password" placeholder="密码">
    <button type="submit">登录</button>
</form>
```

注意 `<input>` 的 `name` 属性要和你 Python 代码里的字段名一致

## 7. Form vs Body 的选择

```css
什么时候用 Body()（JSON）：
  - 前端用 JavaScript（React/Vue/Angular）发请求
  - 移动端 App
  - 微服务之间通信
  - 数据结构复杂（嵌套对象、数组）
  - 现代 API 的默认选择

什么时候用 Form()：
  - HTML <form> 标签直接提交（传统网页）
  - OAuth2 密码登录（规范要求表单格式）
  - 第三方回调（支付回调等，对方指定表单格式）
  - 兼容旧系统

大部分情况：用 JSON（Body）
特定场景：用 Form
```

## 8. 对应到你的投标项目

```python
from typing import Annotated
from fastapi import FastAPI, Form, HTTPException, status
from pydantic import BaseModel, Field

app = FastAPI(title="智能投标决策系统")


# ==================== 登录接口（OAuth2 密码流要求用表单） ====================

class LoginForm(BaseModel):
    """登录表单"""
    username: str = Field(min_length=3, description="用户名")
    password: str = Field(min_length=8, description="密码")


# 模拟用户数据库
fake_users = {
    "admin": {"password": "admin12345", "role": "admin"},
    "editor": {"password": "editor123", "role": "editor"},
}


@app.post("/api/login/", status_code=status.HTTP_200_OK)
def login(form_data: Annotated[LoginForm, Form()]):
    """
    登录接口——使用表单提交

    为什么用 Form 而不是 JSON：
    1. OAuth2 密码流规范要求 username 和 password 用表单格式
    2. 传统的 HTML 登录页面直接提交表单
    3. 保持和标准认证流程的兼容性
    """
    user = fake_users.get(form_data.username)
    if not user or user["password"] != form_data.password:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="用户名或密码错误",
        )
    return {
        "message": "登录成功",
        "username": form_data.username,
        "role": user["role"],
    }


# ==================== 快速搜索（简单表单） ====================

@app.post("/api/search/")
def quick_search(
    keyword: Annotated[str, Form(
        min_length=1,
        max_length=100,
        description="搜索关键词",
    )],
    search_type: Annotated[str, Form(
        description="搜索类型：project / bid / company"
    )] = "project",
):
    """
    快速搜索——支持 HTML 搜索表单直接提交

    对应的 HTML：
    <form action="/api/search/" method="post">
        <input name="keyword" placeholder="搜索...">
        <select name="search_type">
            <option value="project">项目</option>
            <option value="bid">投标</option>
            <option value="company">企业</option>
        </select>
        <button type="submit">搜索</button>
    </form>
    """
    return {
        "keyword": keyword,
        "search_type": search_type,
        "results": [f"搜索 {search_type} 中包含 '{keyword}' 的结果..."],
    }
```

## 9. 六种数据来源的最终总结（完整版）

加上 Form 之后，FastAPI 的所有数据来源你都学完了：

```
数据来源           标记方式       自动推断？    Content-Type
──────────────────────────────────────────────────────────────────
URL 路径          Path()        ✅ 自动       —（在 URL 里）
URL 查询参数      Query()       ✅ 自动       —（在 URL 里）
JSON 请求体       Body()        ✅ 自动       application/json
表单请求体        Form()        ❌ 必须标记    application/x-www-form-urlencoded
HTTP Cookie       Cookie()      ❌ 必须标记    —（在 Cookie 头里）
HTTP Header       Header()      ❌ 必须标记    —（在请求头里）
```

**所有来源都支持**：

```python
# 约束参数
Annotated[str, Form(min_length=3, max_length=50)]

# 文档信息
Annotated[str, Form(description="用户名", examples=["alice"])]

# 打包成 Pydantic 模型
form_data: Annotated[LoginForm, Form()]
```

## 10. 本节核心知识点速查

```
概念                      做法
──────────────────────────────────────────────────────────────────
接收单个表单字段           Annotated[str, Form()]
表单字段加约束             Form(min_length=3, description="...")
多个表单字段打包成模型     Annotated[LoginForm, Form()]
禁止多余表单字段           model_config = {"extra": "forbid"}
安装依赖                   pip install python-multipart
Form 和 JSON Body 不能混用  ← HTTP 协议限制，不是 FastAPI 限制
Python requests 测试      requests.post(url, data={...})   # data= 不是 json=
```

**本节最核心的三句话**：

1. `Form()` 和 `Body()` 是一家人（`Form` 继承自 `Body`），语法完全一样，只是告诉 FastAPI "这是表单数据，不是 JSON"

2. 同一个接口里 Form 和 JSON Body 不能混用——一个 HTTP 请求只有一个 `Content-Type`，要么全是表单，要么全是 JSON

3. 大部分现代 API 用 JSON，只有登录（OAuth2 密码流）、HTML 表单直接提交、第三方回调等特定场景才用 Form
