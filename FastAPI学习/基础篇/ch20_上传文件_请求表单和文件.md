上一节学了接收表单文本数据（`Form()`），这一节学接收**文件**。文件上传和表单数据是近亲，它们都走 `multipart/form-data` 编码。区别只是一个传文本字段，一个传二进制文件。而且它们可以同时出现在一个请求里



这一节有两个核心选择要搞清楚：

- **`bytes` vs `UploadFile`**——小文件用 `bytes`（全部读进内存），大文件用 `UploadFile`（自动落盘）
- **File 和 Form 可以一起用，但不能和 JSON Body 一起用**

---

## 1. 文件上传的本质

### 1.1 三种 Content-Type 对比

到现在已经见过三种请求体编码了：

```css
application/json
  → JSON 格式，用 Body() 或 Pydantic 模型接收
  → {"name": "report.pdf", "size": 1024}

application/x-www-form-urlencoded
  → 普通表单，用 Form() 接收
  → username=alice&password=secret123

multipart/form-data
  → 带文件的表单，用 File() 接收
  → 包含文本字段和二进制文件数据，用分隔符隔开
```

**文件上传必须用 `multipart/form-data`**。这是唯一能在一个请求里同时传文本和二进制数据的编码方式

### 1.2 对应的 HTML 表单

```html
<!-- 普通表单（不能传文件） -->
<form action="/login" method="post">
    <input name="username" type="text">
    <button type="submit">提交</button>
</form>

<!-- 带文件的表单（必须加 enctype） -->
<form action="/upload" method="post" enctype="multipart/form-data">
    <input name="file" type="file">
    <button type="submit">上传</button>
</form>
```

关键区别：带文件的表单必须加 `enctype="multipart/form-data"`，否则浏览器不会发送文件内容

## 2. 两种接收方式：`bytes` vs `UploadFile`

### 2.1 方式一：`bytes`——全部读进内存

```python
from typing import Annotated
from fastapi import FastAPI, File

app = FastAPI()


@app.post("/files/")
def create_file(file: Annotated[bytes, File()]):
    """
    file 的类型是 bytes——文件的全部内容已经在内存里了。
    """
    return {"file_size": len(file)}
```

**工作原理**：FastAPI 把整个文件读进内存，变成一个 `bytes` 对象交给你

**适合**：小文件（几 KB 到几 MB），比如配置文件、小图标、CSV 数据

**不适合**：大文件（几十 MB 以上），因为会占满服务器内存。10 个用户同时上传 100MB 的文件 → 1GB 内存直接吃掉

### 2.2 方式二：`UploadFile`——智能缓存，大文件也能处理

```python
from fastapi import FastAPI, UploadFile

app = FastAPI()


@app.post("/uploadfile/")
def create_upload_file(file: UploadFile):
    """
    file 的类型是 UploadFile——一个类似文件的对象。
    文件内容不会一次性全部加载到内存里。
    """
    return {"filename": file.filename}
```

**工作原理**：FastAPI 使用 `SpooledTemporaryFile`，文件先存在内存里，超过一定大小（默认 1MB）后自动转存到磁盘临时文件。你拿到的是一个文件句柄，按需读取

**适合**：任何大小的文件，尤其是大文件（图片、视频、PDF、Excel 等）

### 2.3 `UploadFile` 比 `bytes` 多了什么

```python
@app.post("/uploadfile/")
async def create_upload_file(file: UploadFile):
    # 文件元信息
    print(file.filename)       # 原始文件名："报告.pdf"
    print(file.content_type)   # MIME 类型："application/pdf"
    print(file.size)           # 文件大小（字节数）

    # 读取文件内容
    contents = await file.read()     # 读取全部内容
    # 或者分块读取
    chunk = await file.read(1024)    # 读取 1024 字节

    # 重置读取位置（读完一遍后想再读一遍）
    await file.seek(0)

    # 写入（少见，但可以往临时文件里写东西）
    await file.write(b"some data")

    # 关闭文件
    await file.close()

    return {"filename": file.filename, "size": len(contents)}
```

### 2.4 一张表做对比

```
                bytes                          UploadFile
─────────────────────────────────────────────────────────────────
内存占用        全部加载到内存                   小文件在内存，大文件自动落盘
获取文件名      ❌ 拿不到                       ✅ file.filename
获取类型        ❌ 拿不到                       ✅ file.content_type
获取大小        len(file)                      file.size
读取方式        直接用（就是 bytes）             await file.read()
分块读取        ❌ 不行（已经全部在内存了）       ✅ await file.read(chunk_size)
需要 File()     ✅ 必须写 File()                ❌ 可以不写（但写了可以加元数据）
适合场景        小文件（< 几 MB）               任何大小的文件
```

**实际项目中 99% 的情况用 `UploadFile`**。`bytes` 只在你确定文件很小且需要简单处理时才用

### 2.5 注意 `UploadFile` 不需要写 `File()`

```python
# bytes 方式：必须写 File()
file: Annotated[bytes, File()]

# UploadFile 方式：可以不写 File()
file: UploadFile                           # ✅ 可以不写
file: Annotated[UploadFile, File()]        # ✅ 也可以写（用来加 description 等元数据）
```

FastAPI 看到类型是 `UploadFile` 就知道这是文件参数，不需要额外标记。但如果想加 `description` 等文档信息，就要用 `Annotated[UploadFile, File(description="...")]`

## 3. `async` 和 `sync` 的读取方式

### 3.1 在 `async def` 函数里：用 `await`

```python
@app.post("/upload/")
async def upload(file: UploadFile):
    contents = await file.read()       # ← 加 await
    await file.seek(0)                 # ← 加 await
    await file.close()                 # ← 加 await
    return {"size": len(contents)}
```

### 3.2 在普通 `def` 函数里：访问底层 `.file` 属性

```python
@app.post("/upload/")
def upload(file: UploadFile):
    contents = file.file.read()        # ← 用 file.file.read()，不加 await
    file.file.seek(0)
    file.file.close()
    return {"size": len(contents)}
```

`file.file` 是底层的 `SpooledTemporaryFile` 对象，它是同步的普通文件对象

**选哪个？** 大部分情况用 `async def` + `await` 就好。如果你的函数里没有其他 `await` 操作（没有异步数据库调用等），用普通 `def` 也行，FastAPI 会自动在线程池里运行它

## 5. 多文件上传

### 5.1 接收多个文件

```python
from fastapi import FastAPI, File, UploadFile

app = FastAPI()


@app.post("/uploadfiles/")
async def upload_files(files: list[UploadFile]):
    """
    客户端可以一次上传多个文件。
    files 是一个 UploadFile 列表。
    """
    return {"filenames": [f.filename for f in files]}
```

### 5.2 对应的 HTML 表单

```html
<!-- 注意 input 的 multiple 属性 -->
<form action="/uploadfiles/" enctype="multipart/form-data" method="post">
    <input name="files" type="file" multiple>
    <button type="submit">上传多个文件</button>
</form>
```

关键点：

- `<input>` 的 `name` 属性必须和 Python 参数名一致（`files`）
- 加了 `multiple` 属性后用户可以选择多个文件

### 5.3 bytes 列表也行

```python
@app.post("/files/")
async def create_files(files: Annotated[list[bytes], File()]):
    return {"file_sizes": [len(f) for f in files]}
```

但同样`list[bytes]` 会把所有文件读进内存，大文件会炸。用 `list[UploadFile]` 更安全

## 6. File + Form 混用：同时接收文件和表单文本字段

### 6.1 最常见的场景

用户上传文件的同时，还要填写一些文本信息。比如上传投标文件时要填项目名称：

```python
from typing import Annotated
from fastapi import FastAPI, File, Form, UploadFile

app = FastAPI()


@app.post("/submit/")
async def submit_bid(
    project_name: Annotated[str, Form()],         # 表单文本字段
    bid_price: Annotated[float, Form()],           # 表单文本字段
    file: Annotated[UploadFile, File()],           # 文件
):
    return {
        "project_name": project_name,
        "bid_price": bid_price,
        "filename": file.filename,
        "file_type": file.content_type,
    }
```

**File 和 Form 可以在同一个接口里混用**——它们都走 `multipart/form-data` 编码，天然兼容

### 6.2 对应的 HTML 表单

```html
<form action="/submit/" enctype="multipart/form-data" method="post">
    <input name="project_name" type="text" placeholder="项目名称">
    <input name="bid_price" type="number" placeholder="投标报价">
    <input name="file" type="file">
    <button type="submit">提交投标</button>
</form>
```

### 6.3 再次强调：File / Form 不能和 JSON Body 混用

```python
# ✅ File + Form → 都是 multipart/form-data，可以
@app.post("/ok/")
async def ok(
    name: Annotated[str, Form()],
    file: UploadFile,
):
    ...

# ❌ File + JSON Body → 不行
@app.post("/bad/")
async def bad(
    file: UploadFile,
    item: Item,            # Pydantic 模型默认是 JSON Body
):
    ...
```

**规则**：

```css
Form + File    → ✅ 都是 multipart/form-data
Form + Form    → ✅
File + File    → ✅
Form/File + Body（JSON） → ❌ HTTP 协议不允许
Form/File + Path/Query/Cookie/Header → ✅ 不影响，它们不在请求体里
```

**一个 HTTP 请求只有一个 body。** 而 `Body()`、`Form()`、`File()` 都是在描述：**“这个参数要从这一个请求体里，用哪种方式去解析”**。



所以“`Body` 和 `File` 不能共存在一个接口的参数里面”，**`Body` 想按 JSON 等普通 body 解析，`File` 想按 `multipart/form-data` 解析，它们要求的请求体格式冲突了**

## 7. 实际项目中文件上传的完整流程

上传文件后通常要保存到磁盘。这里展示一个完整的文件保存流程：

```python
import os
from uuid import uuid4
from typing import Annotated

from fastapi import FastAPI, File, UploadFile, HTTPException, status

app = FastAPI()

UPLOAD_DIR = "uploads"
os.makedirs(UPLOAD_DIR, exist_ok=True)


@app.post("/upload/", status_code=status.HTTP_201_CREATED)
async def upload_file(file: UploadFile):
    """上传文件并保存到服务器"""

    # 1. 检查文件类型
    allowed_types = {"application/pdf", "image/jpeg", "image/png"}
    if file.content_type not in allowed_types:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail=f"不支持的文件类型：{file.content_type}，"
                   f"仅支持 PDF、JPEG、PNG",
        )

    # 2. 检查文件大小（读取内容来获取大小）
    contents = await file.read()
    max_size = 10 * 1024 * 1024  # 10 MB
    if len(contents) > max_size:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail=f"文件太大：{len(contents)} 字节，最大允许 {max_size} 字节",
        )

    # 3. 生成唯一文件名（避免同名覆盖）
    ext = os.path.splitext(file.filename)[1]   # 获取扩展名，如 ".pdf"
    unique_name = f"{uuid4()}{ext}"             # 如 "a3b2c1d4-...-.pdf"
    save_path = os.path.join(UPLOAD_DIR, unique_name)

    # 4. 保存文件
    with open(save_path, "wb") as f:
        f.write(contents)

    return {
        "original_name": file.filename,
        "saved_name": unique_name,
        "content_type": file.content_type,
        "size": len(contents),
    }
```

### 大文件的分块保存（避免内存爆炸）

```python
@app.post("/upload-large/")
async def upload_large_file(file: UploadFile):
    """分块保存——适合大文件"""
    ext = os.path.splitext(file.filename)[1]
    unique_name = f"{uuid4()}{ext}"
    save_path = os.path.join(UPLOAD_DIR, unique_name)

    # 分块读取和写入，每次 1MB
    with open(save_path, "wb") as f:
        while True:
            chunk = await file.read(1024 * 1024)   # 每次读 1MB
            if not chunk:
                break
            f.write(chunk)

    return {"saved_name": unique_name}
```

**这个分块读取的模式非常重要**——上传 1GB 的视频文件也只占 1MB 内存。这里是指内存占用，每次读1MB，然后存进磁盘，直到chunk里没有内容，也就是读完了，那就break循环，即存完了

## 8. 对应到你的投标项目

```python
import os
from uuid import UUID, uuid4
from datetime import datetime
from decimal import Decimal
from typing import Annotated

from fastapi import FastAPI, File, Form, UploadFile, HTTPException, status
from pydantic import BaseModel, Field

app = FastAPI(title="智能投标决策系统")

UPLOAD_DIR = "uploads/bid_files"
os.makedirs(UPLOAD_DIR, exist_ok=True)


# ==================== 响应模型 ====================

class BidSubmissionResponse(BaseModel):
    """投标提交响应"""
    bid_id: UUID
    project_name: str
    bid_price: Decimal
    file_name: str
    file_type: str
    file_size: int
    submitted_at: datetime


# ==================== 投标提交接口（File + Form 混用） ====================

@app.post(
    "/api/bids/submit",
    status_code=status.HTTP_201_CREATED,
    response_model=BidSubmissionResponse,
)
async def submit_bid(
    # 表单文本字段
    project_name: Annotated[str, Form(
        min_length=1, max_length=200,
        description="项目名称",
    )],
    bid_price: Annotated[Decimal, Form(
        gt=0,
        description="投标报价，单位：万元",
    )],
    strategy: Annotated[str, Form(
        description="投标策略：conservative / moderate / aggressive",
    )] = "moderate",
    notes: Annotated[str | None, Form(
        description="投标备注（可选）",
    )] = None,

    # 投标文件
    bid_file: Annotated[UploadFile, File(
        description="投标文件（PDF 格式，最大 50MB）",
    )],
):
    """
    提交投标——同时上传文件和填写表单

    对应的 HTML 表单：
    <form action="/api/bids/submit" enctype="multipart/form-data" method="post">
        <input name="project_name" type="text" placeholder="项目名称">
        <input name="bid_price" type="number" step="0.01" placeholder="投标报价">
        <select name="strategy">
            <option value="conservative">保守</option>
            <option value="moderate">稳健</option>
            <option value="aggressive">激进</option>
        </select>
        <textarea name="notes" placeholder="备注"></textarea>
        <input name="bid_file" type="file" accept=".pdf">
        <button type="submit">提交投标</button>
    </form>
    """
    # 1. 验证文件类型
    if bid_file.content_type != "application/pdf":
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="投标文件必须是 PDF 格式",
        )

    # 2. 读取并验证大小
    contents = await bid_file.read()
    max_size = 50 * 1024 * 1024  # 50 MB
    if len(contents) > max_size:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="投标文件不能超过 50MB",
        )

    # 3. 保存文件
    bid_id = uuid4()
    ext = os.path.splitext(bid_file.filename)[1]
    saved_name = f"{bid_id}{ext}"
    save_path = os.path.join(UPLOAD_DIR, saved_name)

    with open(save_path, "wb") as f:
        f.write(contents)

    # 4. 返回结果
    return BidSubmissionResponse(
        bid_id=bid_id,
        project_name=project_name,
        bid_price=bid_price,
        file_name=bid_file.filename,
        file_type=bid_file.content_type,
        file_size=len(contents),
        submitted_at=datetime.now(),
    )


# ==================== 批量上传附件 ====================

@app.post("/api/bids/{bid_id}/attachments", status_code=status.HTTP_201_CREATED)
async def upload_attachments(
    bid_id: UUID,
    files: Annotated[list[UploadFile], File(
        description="附件（支持 PDF、JPG、PNG，最多 5 个文件）",
    )],
):
    """
    给已有的投标记录追加附件——多文件上传
    """
    if len(files) > 5:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="最多上传 5 个附件",
        )

    allowed_types = {"application/pdf", "image/jpeg", "image/png"}
    results = []

    for f in files:
        if f.content_type not in allowed_types:
            raise HTTPException(
                status_code=status.HTTP_400_BAD_REQUEST,
                detail=f"不支持的文件类型：{f.filename}（{f.content_type}）",
            )

        contents = await f.read()
        ext = os.path.splitext(f.filename)[1]
        saved_name = f"{uuid4()}{ext}"
        save_path = os.path.join(UPLOAD_DIR, saved_name)

        with open(save_path, "wb") as out:
            out.write(contents)

        results.append({
            "original_name": f.filename,
            "saved_name": saved_name,
            "size": len(contents),
            "content_type": f.content_type,
        })

    return {"bid_id": bid_id, "attachments": results}
```

## 9. 继承关系总览：File 是 Form 的子类

```css
Body()
  └── Form()         ← 接收表单文本字段
        └── File()   ← 接收上传文件
```

这意味着 `File()` 支持 `Form()` 和 `Body()` 的所有参数（`description`、`examples` 等）。同时也解释了为什么 File 和 Form 可以混用——它们本质上都是表单数据

## 10. 本节核心知识点速查

```
概念                      做法                                     说明
──────────────────────────────────────────────────────────────────────────────
小文件（全读到内存）       Annotated[bytes, File()]               拿到 bytes
大文件（智能缓存）         file: UploadFile                       拿到文件对象
获取文件名                 file.filename                          "报告.pdf"
获取类型                   file.content_type                      "application/pdf"
读取全部内容               await file.read()                      返回 bytes
分块读取                   await file.read(1024)                  每次读指定字节数
重置读取位置               await file.seek(0)                     读完想再读一遍
可选文件                   file: UploadFile | None = None         可以不传
多文件上传                 files: list[UploadFile]                收到列表
文件 + 表单文本            File() + Form() 混用                   同一个接口
加文档信息                 Annotated[UploadFile, File(description="...")] 
```

**本节最核心的三句话**：

1. 接收文件用 `UploadFile`（大小文件通吃），只有确定文件很小时才用 `bytes`。`UploadFile` 会自动在内存和磁盘之间切换，不怕大文件撑爆内存。

2. File 和 Form 可以在同一个接口混用（都是 `multipart/form-data`），但不能和 JSON Body 混用——这是 HTTP 协议的限制。

3. 实际项目中上传文件要做三件事：验证类型、验证大小、用唯一文件名保存。大文件用分块读取（`await file.read(chunk_size)`）避免内存爆炸
