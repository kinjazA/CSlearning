前面的模型都比较简单：几个字符串、几个数字。但真实业务的数据结构往往是"套娃"式的——一个项目里有多条投标记录，每条记录里又有多个标签。这一节讲的就是：**模型里面套模型、列表里面套模型、模型里面套列表套模型……任意深度嵌套。**预备知识 1.8 节已经初步介绍过嵌套模型，这一节会更深入，并且覆盖几种特殊场景

## 1. 列表字段：模型里放一个列表

### 1.1 不指定元素类型的列表（不推荐）

```python
class Item(BaseModel):
    name: str
    tags: list = []    # 只说了是列表，没说里面装什么
```

这样 `tags` 可以装任何东西：`[1, "abc", True, None, [1,2]]` 全部合法。FastAPI 没法校验列表里的元素

### 1.2 指定元素类型的列表（推荐）

```python
class Item(BaseModel):
    name: str
    tags: list[str] = []    # 字符串列表
```

`list[str]` 表示"一个列表，里面每个元素都必须是字符串"

```json
{"name": "Foo", "tags": ["rock", "metal"]}      // ✅ 全是字符串
{"name": "Foo", "tags": [1, 2, 3]}              // ❌ 元素不是字符串
{"name": "Foo", "tags": "不是列表"}               // ❌ 不是列表
{"name": "Foo", "tags": []}                      // ✅ 空列表合法
```

**永远指定列表里的元素类型。** `list` 不如 `list[str]`，因为后者能帮你校验每个元素

## 2. 集合字段：自动去重

### 2.1 list vs set

标签这种东西不应该有重复。Python 有一个专门处理不重复元素的类型：`set`

```python
class Item(BaseModel):
    name: str
    tags: set[str] = set()    # 字符串集合，自动去重
```

### 2.2 set 的自动去重效果

```json
// 请求（有重复）
{
    "name": "Foo",
    "tags": ["rock", "metal", "rock", "rock"]
}

// FastAPI 实际收到的（自动去重了）
{
    "name": "Foo",
    "tags": ["rock", "metal"]
}
```

用户传了重复的值，Pydantic 自动帮你去掉了。返回给客户端的 JSON 里也不会有重复

### 2.3 什么时候用 set，什么时候用 list

```css
list[str]  → 元素可以重复，顺序有意义
             例：历史报价列表 [980, 1100, 980] 三次报价都要保留

set[str]   → 元素不重复，顺序无意义
             例：项目标签 {"市政", "道路"} 不需要重复
```

## 3. 模型嵌套模型

### 3.1 定义子模型

```python
from pydantic import BaseModel


# 子模型：图片信息
class Image(BaseModel):
    url: str
    name: str


# 父模型：商品信息，包含一个 Image 字段
class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None
    tags: set[str] = set()
    image: Image | None = None      # 嵌套模型，可选
```

### 3.2 对应的 JSON 结构

```json
{
    "name": "Foo",
    "description": "一段描述",
    "price": 42.0,
    "tax": 3.2,
    "tags": ["rock", "metal"],
    "image": {
        "url": "http://example.com/photo.jpg",
        "name": "商品主图"
    }
}
```

`image` 字段不是一个简单的字符串或数字，它本身是一个 JSON 对象，有自己的 `url` 和 `name` 字段

### 3.3 FastAPI 会逐层校验

```json
// image 里缺了 url 字段
{
    "name": "Foo",
    "price": 42.0,
    "image": {
        "name": "只有名字没有链接"
    }
}
// → 422 错误："loc": ["body", "image", "url"], "msg": "Field required"
// 错误路径精确到 image 里面的 url 字段
```

不管嵌套多少层，每一层的每个字段都会被校验

## 4. 特殊类型：HttpUrl

### 4.1 问题：url 字段用 str 不够严格

```python
class Image(BaseModel):
    url: str       # 只要是字符串就行
    name: str
```

用户传 `"url": "不是网址"` 也能通过。但 URL 应该有特定格式

### 4.2 用 HttpUrl 替代 str

```python
from pydantic import BaseModel, HttpUrl

class Image(BaseModel):
    url: HttpUrl       # 必须是合法的 HTTP/HTTPS URL
    name: str
```

```json
{"url": "https://example.com/photo.jpg", "name": "照片"}   // ✅
{"url": "http://example.com/photo.jpg", "name": "照片"}    // ✅
{"url": "不是网址", "name": "照片"}                          // ❌ 422 错误
{"url": "ftp://example.com/file", "name": "文件"}           // ❌ 422 错误（不是 HTTP）
```

### 4.3 Pydantic 还有哪些特殊类型

`HttpUrl` 只是其中一个。Pydantic 提供了很多实用的内置类型：

```python
from pydantic import (
    HttpUrl,          # HTTP/HTTPS URL
    EmailStr,         # 合法的邮箱地址（需要安装 pydantic[email]）
    IPvAnyAddress,    # IP 地址（v4 或 v6）
    PositiveInt,      # 正整数（等同于 Field(gt=0) 但更语义化）
    PositiveFloat,    # 正浮点数
    NonNegativeInt,   # 非负整数（>= 0）
)
```

你不需要记住所有的，知道 Pydantic 有这些特殊类型，需要的时候查文档就行

## 5. 模型列表：列表里每个元素都是模型

### 5.1 一个字段是模型列表

```python
class Image(BaseModel):
    url: HttpUrl
    name: str


class Item(BaseModel):
    name: str
    price: float
    images: list[Image] | None = None    # 图片列表，每个元素是 Image 模型
```

```json
{
    "name": "Foo",
    "price": 42.0,
    "images": [
        {
            "url": "http://example.com/photo1.jpg",
            "name": "主图"
        },
        {
            "url": "http://example.com/photo2.jpg",
            "name": "细节图"
        }
    ]
}
```

`images` 是一个列表，列表里每个元素都必须符合 `Image` 模型的定义。FastAPI 会校验列表里的每一个元素

### 5.2 和简单列表的区别

```python
tags: list[str]           # 列表里是简单类型（字符串）
images: list[Image]       # 列表里是模型（每个元素是一个完整的对象）
```

## 6. 深度嵌套：模型套模型套模型

### 6.1 三层嵌套

```python
class Image(BaseModel):
    url: HttpUrl
    name: str


class Item(BaseModel):
    name: str
    price: float
    tags: set[str] = set()
    images: list[Image] | None = None    # Item 里嵌套 Image 列表


class Offer(BaseModel):
    name: str
    description: str | None = None
    price: float
    items: list[Item]                    # Offer 里嵌套 Item 列表
```

**嵌套关系**：

```
Offer
 ├── name: str
 ├── price: float
 └── items: list[Item]
      └── Item
           ├── name: str
           ├── price: float
           ├── tags: set[str]
           └── images: list[Image]
                └── Image
                     ├── url: HttpUrl
                     └── name: str
```

**对应的 JSON**：

```json
{
    "name": "促销活动",
    "description": "年终大促",
    "price": 999.0,
    "items": [
        {
            "name": "商品A",
            "price": 42.0,
            "tags": ["热销"],
            "images": [
                {"url": "http://example.com/a1.jpg", "name": "主图"},
                {"url": "http://example.com/a2.jpg", "name": "细节"}
            ]
        },
        {
            "name": "商品B",
            "price": 88.0,
            "tags": ["新品", "推荐"],
            "images": null
        }
    ]
}
```

FastAPI 会递归校验每一层的每一个字段。不管嵌套多深，错误信息都能精确告诉你问题出在哪一层的哪个字段

## 7. 纯列表作为请求体

### 7.1 场景：请求体的顶层不是对象，而是数组

有时候你需要一次性提交多条数据，JSON 的顶层直接就是一个列表：

```json
[
    {"url": "http://example.com/1.jpg", "name": "图片1"},
    {"url": "http://example.com/2.jpg", "name": "图片2"},
    {"url": "http://example.com/3.jpg", "name": "图片3"}
]
```

注意：最外层是 `[...]`（数组），不是 `{...}`（对象）

### 7.2 怎么接收

```python
@app.post("/images/multiple/")
def create_multiple_images(images: list[Image]):
    return images
```

直接把参数类型写成 `list[Image]`。FastAPI 会把整个请求体当作一个列表来解析，列表里每个元素都按 `Image` 模型校验

### 7.3 什么时候用

```css
普通场景（大多数情况）：
  请求体是一个 JSON 对象 → 用 Pydantic 模型

批量操作场景：
  请求体是一个 JSON 数组 → 用 list[模型]
  比如：批量导入数据、一次提交多条记录
```

## 8. 任意字典作为请求体

### 8.1 场景：你不知道键名是什么

有些情况下，数据的键名是不确定的，但你知道键和值的类型。

```python
@app.post("/index-weights/")
def create_index_weights(weights: dict[int, float]):
    return weights
```

### 8.2 对应的 JSON

```json
{
    "1": 0.5,
    "2": 0.3,
    "7": 0.2
}
```

**注意**：JSON 的键只能是字符串，所以实际传过来的是 `"1"`、`"2"`、`"7"`。但你声明了 `dict[int, float]`，Pydantic 会自动把字符串键 `"1"` 转换成整数 `1`

### 8.3 什么时候用

这种场景比较少见。典型的使用场景是：

```python
# 权重配置：不确定有多少个索引
weights: dict[int, float]
# {"0": 0.5, "1": 0.3, "2": 0.2}

# 动态配置：不确定有哪些配置项
config: dict[str, str]
# {"theme": "dark", "language": "zh", "timezone": "UTC+8"}

# 评分：不确定有哪些评委
scores: dict[str, float]
# {"评委A": 95.5, "评委B": 88.0, "评委C": 92.3}
```

大部分时候你应该用 Pydantic 模型（字段明确），只有在键名不确定时才用 `dict`

## 9. 对应到你的投标项目

```python
from fastapi import FastAPI, Path
from pydantic import BaseModel, Field, HttpUrl
from typing import Annotated

app = FastAPI(title="智能投标决策系统")


# ==================== 子模型：层层定义 ====================

class CompetitorInfo(BaseModel):
    """竞争对手信息"""
    company_name: str = Field(min_length=1, max_length=100)
    estimated_price: float | None = Field(
        default=None, gt=0,
        description="估计的对手报价"
    )
    strength: str = Field(
        default="unknown",
        description="对手实力评估：strong / medium / weak / unknown"
    )


class HistoricalBid(BaseModel):
    """历史投标记录"""
    project_name: str = Field(min_length=1, max_length=200)
    bid_price: float = Field(gt=0)
    won: bool
    tags: set[str] = set()                          # 标签，自动去重
    competitors: list[CompetitorInfo] = []           # 嵌套模型列表


class ProjectAttachment(BaseModel):
    """项目附件"""
    url: HttpUrl                                     # 必须是合法 URL
    filename: str = Field(min_length=1, max_length=200)
    file_type: str = Field(
        default="pdf",
        description="文件类型：pdf / docx / xlsx"
    )


# ==================== 主模型：组合所有子模型 ====================

class FullPredictRequest(BaseModel):
    """完整的预测请求——展示多层嵌套"""
    # 基本信息
    control_price: float = Field(gt=0, description="控制价，单位：万元")
    num_competitors: int = Field(ge=2, le=100)
    strategy: str = Field(default="moderate")

    # 嵌套单个模型（可选）
    # main_competitor: CompetitorInfo | None = None

    # 嵌套模型列表
    history: list[HistoricalBid] = Field(
        default=[],
        description="历史投标记录"
    )

    # 附件列表
    attachments: list[ProjectAttachment] = Field(
        default=[],
        description="项目相关附件"
    )

    # 自定义权重（键名不确定）
    factor_weights: dict[str, float] = Field(
        default={},
        description="各因素权重，如 {'price': 0.6, 'reputation': 0.4}"
    )


# ==================== 接口定义 ====================

@app.post("/api/projects/{project_id}/full-predict")
def full_predict(
    project_id: Annotated[int, Path(ge=1)],
    request: FullPredictRequest,
):
    """
    完整预测请求，JSON 结构示例：

    {
        "control_price": 1200.5,
        "num_competitors": 8,
        "strategy": "aggressive",
        "history": [
            {
                "project_name": "路面改造A",
                "bid_price": 980,
                "won": true,
                "tags": ["市政", "道路"],
                "competitors": [
                    {"company_name": "竞争者A", "estimated_price": 950}
                ]
            }
        ],
        "attachments": [
            {
                "url": "https://example.com/doc.pdf",
                "filename": "招标文件.pdf",
                "file_type": "pdf"
            }
        ],
        "factor_weights": {
            "price": 0.6,
            "reputation": 0.3,
            "experience": 0.1
        }
    }
    """
    return {
        "project_id": project_id,
        "history_count": len(request.history),
        "attachment_count": len(request.attachments),
        "factors": list(request.factor_weights.keys()),
        "recommended_price": request.control_price * 0.95,
    }


# 批量导入历史投标记录（纯列表请求体）
@app.post("/api/bids/batch-import")
def batch_import_bids(bids: list[HistoricalBid]):
    """
    JSON 请求体是一个数组：
    [
        {"project_name": "项目A", "bid_price": 980, "won": true, "tags": ["市政"]},
        {"project_name": "项目B", "bid_price": 1100, "won": false, "tags": ["道路"]},
        {"project_name": "项目C", "bid_price": 870, "won": true, "tags": ["桥梁"]}
    ]
    """
    return {
        "imported": len(bids),
        "total_won": sum(1 for b in bids if b.won),
    }
```

**嵌套关系图**：

```
FullPredictRequest
 ├── control_price: float
 ├── num_competitors: int
 ├── strategy: str
 ├── history: list[HistoricalBid]
 │    └── HistoricalBid
 │         ├── project_name: str
 │         ├── bid_price: float
 │         ├── won: bool
 │         ├── tags: set[str]              ← 自动去重
 │         └── competitors: list[CompetitorInfo]
 │              └── CompetitorInfo
 │                   ├── company_name: str
 │                   ├── estimated_price: float | None
 │                   └── strength: str
 ├── attachments: list[ProjectAttachment]
 │    └── ProjectAttachment
 │         ├── url: HttpUrl                ← 自动校验 URL 格式
 │         ├── filename: str
 │         └── file_type: str
 └── factor_weights: dict[str, float]      ← 动态键名
```

三层嵌套，每一层的每个字段都有类型校验。FastAPI 全部自动处理

## 10. 本节核心知识点速查

```css
数据结构                    写法                          校验效果
──────────────────────────────────────────────────────────────────
字符串列表               list[str]                   每个元素必须是字符串
去重集合                 set[str]                    自动去掉重复元素
嵌套单个模型             image: Image | None          字段本身是一个模型
嵌套模型列表             images: list[Image]          列表里每个元素是模型
深度嵌套                 模型套模型套模型              任意深度，逐层校验
纯列表请求体             list[Image] 作为参数类型      JSON 顶层是数组
任意字典                 dict[str, float]             键值对类型校验
合法 URL                 HttpUrl                      自动校验 URL 格式
```

**本节最核心的一句话**：Pydantic 模型可以任意嵌套——字段的类型可以是另一个模型、模型的列表、模型的集合，嵌套多少层都行。FastAPI 会递归校验每一层的每一个字段，错误信息精确到具体是哪一层的哪个字段出了问题
