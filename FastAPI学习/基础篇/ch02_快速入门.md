## 1. 先把代码跑起来，再回头拆解

### 1.1 完整代码（就这么几行）

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/")
async def root():
    return {"message": "Hello World"}
```

### 1.2 怎么运行

```bash
# 方式一：用 fastapi 命令（需要安装 fastapi-cli）
fastapi dev main.py

# 方式二：用 uvicorn（更通用，推荐初学用这个）  这个是在终端用
uvicorn main:app --reload
```

两种方式效果一样，都是启动一个本地开发服务器。区别是：

- `fastapi dev` 是 FastAPI 官方后来出的命令行工具，封装了 uvicorn，更简洁
- `uvicorn main:app --reload` 是底层命令，面试和实际项目里用得更多，==reload可以热加载，更改之后刷新就可以加载，不用重启项目==

### 1.3 启动后你能访问什么

```
http://127.0.0.1:8000        → 你的接口，返回 {"message": "Hello World"}
http://127.0.0.1:8000/docs   → 自动生成的交互式 API 文档（Swagger UI）
http://127.0.0.1:8000/redoc  → 另一种风格的 API 文档（ReDoc）
```

三个地址都打开看一眼，尤其是 `/docs`，后面会经常用它来测试接口

## 2. 逐行拆解：每一行到底在干什么

### 2.1 第一行：`from fastapi import FastAPI`

```python
from fastapi import FastAPI
```

**是什么？** 从 `fastapi` 这个包里导入 `FastAPI` 这个类

**为什么？** `FastAPI` 是整个框架的核心类。所有的接口、配置、中间件，都要挂在它的实例上面

**一个容易混淆的点：** `fastapi`（小写）是包名，`FastAPI`（大写）是类名。Python 里包名一般小写，类名一般大写开头，这是命名规范

### 2.2 第二行：`app = FastAPI()`

```python
app = FastAPI()
```

**是什么？** 创建一个 `FastAPI` 类的**实例**，赋值给变量 `app`

**为什么叫 `app`？** 这是约定俗成的命名。可以叫 `my_api`、`server` 或者任何名字。运行命令里要引用它：

```bash
uvicorn main:app --reload
#        ↑    ↑
#      文件名  变量名
# 意思是：去 main.py 文件里找 app 这个变量，它就是 FastAPI 应用
```

如果把变量改名叫 `my_api`，运行命令就要改成 `uvicorn main:my_api --reload`

**`FastAPI()` 可以传参数吗？** 可以。常用的参数有：

```python
app = FastAPI(
    title="智能投标决策系统",           # 文档页面上显示的标题
    description="基于博弈论的投标分析",  # 文档页面上的描述
    version="1.0.0"                     # API 版本号
)
```

这些参数不影响功能，但会让自动生成的文档更好看、更专业

### 2.3 第三行：`@app.get("/")`

```python
@app.get("/")
```

#### 2.3.1 `@` 是什么？—— 装饰器语法

`@` 是 Python 装饰器的语法糖。它的作用是把下面的函数"注册"到某个地方。在这里，`@app.get("/")` 做的事情是：**把下面的 `root` 函数注册到 `app` 上，告诉 FastAPI "当有人用 GET 方法访问 `/` 路径时，执行这个函数"**。等价于这样写（但没人这样写，太丑了）：

```python
async def root():
    return {"message": "Hello World"}

app.get("/")(root)  # 手动注册，效果和 @app.get("/") 一样
```

#### 2.3.2 `.get` 是什么？—— HTTP 方法

`app.get` 里的 `get` 对应 HTTP 的 GET 方法。FastAPI 为每种 HTTP 方法都提供了对应的装饰器：

```python
@app.get("/xxx")      # 处理 GET 请求     → 获取数据
@app.post("/xxx")     # 处理 POST 请求    → 提交/创建数据
@app.put("/xxx")      # 处理 PUT 请求     → 更新数据
@app.delete("/xxx")   # 处理 DELETE 请求  → 删除数据
```

**怎么选用哪个？** 看你这个接口是干什么的：

- 用户要"看"东西（获取列表、查详情）→ `@app.get`
- 用户要"交"东西（提交表单、发起计算）→ `@app.post`
- 用户要"改"东西 → `@app.put`
- 用户要"删"东西 → `@app.delete`

#### 2.3.3 `"/"` 是什么？—— 路径（路由）

`"/"` 就是 URL 中的路径部分。`/` 表示根路径，也就是 `http://127.0.0.1:8000/` 最后面那个斜杠。路径的设计直接决定了你的 API 长什么样：

```python
@app.get("/")                    # http://127.0.0.1:8000/
@app.get("/health")              # http://127.0.0.1:8000/health
@app.get("/api/projects")        # http://127.0.0.1:8000/api/projects
@app.get("/api/projects/42")     # http://127.0.0.1:8000/api/projects/42
```

#### 2.3.4 把三个概念合在一起理解

`@app.get("/")` 这一行同时做了三件事：

```
@app      → 注册到哪个应用上
.get      → 响应什么 HTTP 方法
("/")     → 监听什么 URL 路径
```

所以这行代码的完整含义是：**"在 app 这个应用上，注册一个路由：当客户端用 GET 方法请求 `/` 路径时，执行下面的函数"**

### 2.4 第四行：`async def root():`

```python
async def root():
```

#### 2.4.1 函数名 `root` 重要吗？

**不重要。** 可以叫任何名字：

```python
@app.get("/")
async def homepage():          # 叫 homepage 也行
    return {"message": "Hello World"}

@app.get("/")
async def 随便叫啥都可以():     # 甚至中文也不会报错（但别这样做）
    return {"message": "Hello World"}
```

FastAPI 不关心函数名叫什么。只关心装饰器里的路径和方法。函数名的作用是：让你自己和同事看得懂代码，以及在自动文档里显示

#### 2.4.2 `async def` 和 `def` 的区别

你可能注意到这里用的是 `async def` 而不是普通的 `def`。这涉及到异步编程。两种写法都能用，效果几乎一样：

```python
# 写法一：异步函数（官方示例用的）
@app.get("/")
async def root():
    return {"message": "Hello World"}

# 写法二：普通函数（完全可以，推荐初学用这个）
@app.get("/")
def root():
    return {"message": "Hello World"}
```

**什么时候用 `async def`？** 当你的函数里需要 `await` 某个异步操作（比如异步数据库查询）时，才必须用 `async def`。如果你的函数里没有 `await`，用 `def` 就行

#### 2.4.3 这个函数没有参数？

对，因为这个接口不需要接收任何数据。用户只是访问 `/` 看一下，不需要传任何参数。后面学到路径参数、查询参数、请求体的时候，函数就会有参数了：

```python
# 后面你会写的代码长这样
@app.get("/projects/{project_id}")
def get_project(project_id: int, verbose: bool = False):
    #              ↑ 路径参数             ↑ 查询参数
    ...

@app.post("/predict")
def predict(request: BidRequest):
    #          ↑ 请求体（Pydantic 模型）
    ...
```

### 2.5 第五行：`return {"message": "Hello World"}`

```python
return {"message": "Hello World"}
```

**是什么？** 返回一个 Python 字典

**然后呢？** FastAPI 自动把这个字典转换成 JSON ，设置好响应头（`Content-Type: application/json`），然后发回客户端。客户端收到的是：

```json
{"message": "Hello World"}
```

**不需要手动调用 `json.dumps()`，不需要手动设置响应头。** FastAPI 全部自动处理

**能返回什么类型？**

```python
# 字典 → 自动转 JSON 对象
return {"name": "张三", "age": 25}
# 客户端收到: {"name": "张三", "age": 25}

# 列表 → 自动转 JSON 数组
return [1, 2, 3]
# 客户端收到: [1, 2, 3]

# Pydantic 模型 → 自动转 JSON 对象
return BidResponse(price=1150.0, probability=0.68)
# 客户端收到: {"price": 1150.0, "probability": 0.68}

# 字符串 → 直接返回字符串
return "hello"
# 客户端收到: "hello"

# 数字 → 直接返回数字
return 42
# 客户端收到: 42
```

大部分时候会返回字典或 Pydantic 模型

## 3. 启动命令深入理解

### 3.1 `uvicorn main:app --reload` 每个部分什么意思

```bash
uvicorn main:app --reload
```

| 部分       | 含义             | 说明                                     |
| ---------- | ---------------- | ---------------------------------------- |
| `uvicorn`  | 运行 ASGI 服务器 | 它是实际监听端口、接收 HTTP 请求的程序   |
| `main`     | 文件名           | 指 `main.py`（不用写 `.py` 后缀）        |
| `:`        | 分隔符           | 分隔文件名和变量名                       |
| `app`      | 变量名           | 指 `main.py` 里的 `app = FastAPI()`      |
| `--reload` | 热重载           | 代码改了自动重启服务器，**只在开发时用** |

**常见问题**：

```bash
# 如果你的文件叫 server.py，变量叫 my_app
uvicorn server:my_app --reload

# 如果你的代码在子目录 app/main.py 里
uvicorn app.main:app --reload

# 指定端口（默认 8000）
uvicorn main:app --reload --port 3000

# 指定允许外部访问（默认只有本机能访问）
uvicorn main:app --reload --host 0.0.0.0
```

### 3.2 什么是 Uvicorn？为什么需要它？

FastAPI 本身**不能直接运行**。它只是一个框架，定义了"收到请求后该怎么处理"。但"监听端口、接收网络连接"这件事，它不干。Uvicorn 就是干这个的——它是一个 **ASGI 服务器**，负责：

```
网络层面的事情（Uvicorn 负责）：
  1. 监听端口（比如 8000）
  2. 接收客户端的 TCP 连接
  3. 解析 HTTP 请求报文
  4. 把解析好的请求交给 FastAPI
  5. 拿到 FastAPI 的返回值，包装成 HTTP 响应发回去

业务层面的事情（FastAPI 负责）：
  1. 根据路径找到对应的函数
  2. 校验请求数据
  3. 执行你写的业务逻辑
  4. 返回结果
```

打个比方：**Uvicorn 是前台接待，FastAPI 是后面干活的人。** 客人来了前台接待，问清楚要干什么，然后转交给对应的人处理，处理完前台再把结果交给客人

### 3.3 `--reload` 到底做了什么

```bash
# 加了 --reload
uvicorn main:app --reload
# Uvicorn 会监视你的代码文件，
# 只要你保存了任何改动，服务器自动重启，新代码立即生效。
# 开发的时候特别方便，不用手动停了再启动。

# 不加 --reload
uvicorn main:app
# 启动后代码就固定了，改了代码不会生效，
# 要手动 Ctrl+C 停掉再重新启动。
# 生产环境用这种方式，因为 --reload 有性能开销。
```

## 4. 自动文档是怎么回事

### 4.1 Swagger UI（`/docs`）

启动服务后访问 `http://127.0.0.1:8000/docs`，你会看到一个交互式网页。

**这个页面是怎么来的？**

1. FastAPI 扫描你代码里所有的 `@app.get`、`@app.post` 等装饰器
2. 读取每个接口的路径、方法、参数类型、返回值类型
3. 自动生成一份 OpenAPI 规范的 JSON 描述文件（在 `/openapi.json`）
4. Swagger UI 读取这个 JSON，渲染成交互式网页

**你一行文档代码都不用写**，全是 FastAPI 根据你的代码自动生成的

**能用它来干什么？**

- 直接在网页上填参数、点按钮测试接口（不用写 curl 命令）
- 查看每个接口接受什么参数、返回什么格式
- 分享给前端同事，他们看这个文档就知道怎么调你的接口

### 4.2 OpenAPI 和 JSON Schema 是什么

**OpenAPI** 是一个行业标准，规定了"怎么描述一个 API"。就像简历有固定格式一样，API 的描述也有固定格式。FastAPI 自动按照这个格式生成描述文件

**JSON Schema** 是另一个标准，规定了"怎么描述一个 JSON 数据的结构"。比如"这个字段是字符串、那个字段是数字、这个是必填的"

**只需要记住的就一点**：FastAPI 自动帮你生成这些东西。你只管写代码，文档自己就有了。当你定义了 Pydantic 模型，模型里的字段名、类型、是否必填，全部会反映在文档里

### 4.3 `/openapi.json` 是什么

访问 `http://127.0.0.1:8000/openapi.json`，你会看到一大坨 JSON。

这就是 FastAPI 自动生成的 API 描述文件。Swagger UI 和 ReDoc 都是读这个文件来渲染界面的。

你不需要手动编辑这个文件。但可以看一眼，了解它的结构：

```json
{
    "openapi": "3.1.0",
    "info": {
        "title": "FastAPI",
        "version": "0.1.0"
    },
    "paths": {
        "/": {
            "get": {
                "summary": "Root",
                "responses": {
                    "200": {
                        "description": "Successful Response"
                    }
                }
            }
        }
    }
}
```

能看出来吗？它把你代码里的路由信息（路径 `/`、方法 `get`、函数名 `Root`）全部提取出来，用标准格式描述了一遍。

## 5. 几个初学者常见的疑问

### 5.1 `127.0.0.1` 和 `localhost` 是什么？

```
127.0.0.1 = localhost = "你自己这台电脑"
```

当运行 `uvicorn main:app --reload` 时，服务器只在本地运行。别人的电脑访问不了。这是开发阶段正常的——你不会在开发的时候就让全世界访问你的代码。

`:8000` 是端口号。一台电脑可以同时跑很多服务，靠端口号区分。就像一栋楼有很多房间，端口号就是房号

### 5.2 为什么返回字典就行了？不用自己转 JSON？

FastAPI 在底层用了一个叫 `JSONResponse` 的东西。当你 `return {"message": "Hello World"}` 时，FastAPI 自动：

1. 调用 `json.dumps()` 把字典转成 JSON 字符串
2. 设置响应头 `Content-Type: application/json`
3. 设置状态码 200
4. 把 JSON 字符串作为响应体发回去

你直接返回字典，FastAPI 在背后做了这些。这就是框架的价值——**帮你做重复的脏活，让你专注于业务逻辑**

### 5.3 一个路径可以同时处理 GET 和 POST 吗？

可以，但要分开写两个函数：

```python
@app.get("/projects")
def list_projects():
    """GET /projects → 获取项目列表"""
    return [{"id": 1, "name": "项目A"}]


@app.post("/projects")
def create_project(data: dict):
    """POST /projects → 创建新项目"""
    return {"message": "创建成功", "data": data}
```

同一个路径 `/projects`，GET 和 POST 是两个不同的接口，执行不同的函数

### 5.4 路径有顺序要求吗？

==**有！** 固定路径要写在动态路径前面：==

```python
# ✅ 正确顺序
@app.get("/projects/latest")      # 固定路径，放前面
def get_latest():
    return {"name": "最新项目"}

@app.get("/projects/{project_id}") # 动态路径，放后面
def get_project(project_id: int):
    return {"id": project_id}


# ❌ 错误顺序（"latest" 会被当成 project_id）
@app.get("/projects/{project_id}") # 动态路径在前
def get_project(project_id: int):
    return {"id": project_id}

@app.get("/projects/latest")      # 永远不会被匹配到！
def get_latest():                  # 因为 "latest" 已经被上面的 {project_id} 捕获了
    return {"name": "最新项目"}
```

FastAPI 按照你代码中定义的顺序从上到下匹配。第一个匹配上的就会被执行
