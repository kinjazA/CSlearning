## 1. 一句话说清楚 Field 是什么

==和ch08的内容一致，都是讲Filed()==

前面学了 `Query()` 给查询参数加约束、`Path()` 给路径参数加约束、`Body()` 给请求体加约束。但有个问题：请求体的数据结构是用 Pydantic 模型定义的，模型**内部**的字段怎么加约束？答案就是 `Field()`——**用在模型内部，功能和 Query/Path/Body 一模一样**

```css
Query()  → 约束函数参数（查询参数）
Path()   → 约束函数参数（路径参数）
Body()   → 约束函数参数（请求体）
Field()  → 约束模型字段（Pydantic 模型内部）
```

它们四个是一家人，支持的参数完全一样（`gt`、`ge`、`max_length`、`title`、`description` 等）。唯一的区别是**用在哪里**

---

## 2. 导入的位置不同——容易踩的坑

```python
from fastapi import Query, Path, Body    # 从 fastapi 导入
from pydantic import Field               # 从 pydantic 导入！
```

**`Field` 是 Pydantic 的东西，不是 FastAPI 的。** 因为模型本身就是 Pydantic 定义的，模型字段的约束自然也用 Pydantic 的工具。这是初学者常犯的错

---

## 3. 完整示例

```python
from typing import Annotated
from fastapi import Body, FastAPI
from pydantic import BaseModel, Field

app = FastAPI()


class Item(BaseModel):
    name: str
    description: str | None = Field(
        default=None,
        title="物品描述",
        max_length=300
    )
    price: float = Field(
        gt=0,
        description="价格必须大于零"
    )
    tax: float | None = None


@app.put("/items/{item_id}")
def update_item(item_id: int, item: Annotated[Item, Body(embed=True)]):
    return {"item_id": item_id, "item": item}
```

### 3.1 逐字段拆解

```python
class Item(BaseModel):
    # ---- 字段一：name ----
    name: str
    # 没有 Field()，最简写法
    # 必填（没有默认值），类型是字符串，没有额外约束

    # ---- 字段二：description ----
    description: str | None = Field(
        default=None,          # 默认值 None → 可选字段
        title="物品描述",       # 文档标题
        max_length=300         # 最多 300 个字符
    )

    # ---- 字段三：price ----
    price: float = Field(
        gt=0,                  # 必须大于 0
        description="价格必须大于零"  # 文档描述
    )
    # 注意：没有 default 参数 → 必填字段
    # Field(gt=0) 不算默认值，它只是约束

    # ---- 字段四：tax ----
    tax: float | None = None
    # 没有 Field()，直接用 = None 设默认值
    # 可选，没有额外约束
```

### 3.2 Field() 的默认值写法

```python
# 写法一：default 参数
description: str | None = Field(default=None, max_length=300)

# 写法二：第一个位置参数（效果完全一样）
description: str | None = Field(None, max_length=300)

# 写法三：不用 Field，直接写默认值（没有额外约束时）
tax: float | None = None

# 必填字段：不传 default
price: float = Field(gt=0)   # 没有 default → 必填
```

### 3.3 不是每个字段都需要 Field()

如果一个字段不需要额外约束（不需要限制长度、范围等），也不需要加文档信息，直接写类型注解和默认值就够了：

```python
class Item(BaseModel):
    name: str                          # 不需要约束，不用 Field
    price: float = Field(gt=0)         # 需要约束（大于零），用 Field
    tax: float | None = None           # 不需要约束，不用 Field
```

**原则：Field() 只在你需要额外约束或文档信息时才用。** 不要为了"看起来统一"给每个字段都加 `Field()`

## 4. Field 支持的全部参数

因为 `Field` 和 `Query`/`Path`/`Body` 是一家人，参数完全一样：

```python
Field(
    # ---- 默认值 ----
    default=None,                    # 默认值（有默认值 = 可选）

    # ---- 数值约束 ----
    gt=0,                            # 大于
    ge=1,                            # 大于等于
    lt=100,                          # 小于
    le=99,                           # 小于等于

    # ---- 字符串约束 ----
    min_length=1,                    # 最短
    max_length=300,                  # 最长
    pattern="^[a-z]+$",              # 正则匹配

    # ---- 文档信息 ----
    title="字段标题",                 # 在文档中显示
    description="字段描述",           # 在文档中显示

    # ---- 其他 ----
    examples=["示例值1", "示例值2"],  # 文档中的示例
)
```

## 5. 它们四个到底是什么关系

官方文档提到 `Query`、`Path`、`Body` 和 `Field` 底层都继承自同一个体系。你不需要记住继承关系，只要记住这张图：

```css
你的数据从哪里来？            用什么加约束？         从哪导入？
─────────────────────────────────────────────────────────────
URL 路径（/items/{id}）      Path()               from fastapi
URL 查询参数（?key=val）     Query()              from fastapi
HTTP 请求体（JSON Body）     Body()               from fastapi
Pydantic 模型内部字段         Field()              from pydantic
```

**前三个是 FastAPI 的**（告诉框架"从哪里取数据"+ 加约束）。**最后一个是 Pydantic 的**（模型字段的约束是 Pydantic 自己管的）

## 6. 一个完整的对比：没有 Field vs 有 Field

### 6.1 没有 Field 的模型

```python
class BidRequest(BaseModel):
    control_price: float
    num_competitors: int
    strategy: str = "moderate"
    notes: str | None = None
```

FastAPI 只知道类型和是否必填。用户可以传：

```json
{
    "control_price": -100,           // 负数也行？
    "num_competitors": 0,            // 0 个竞争对手？
    "strategy": "",                  // 空字符串？
    "notes": "一段超级超级超级长的备注可以写一万个字......"  // 无限长？
}
```

类型是对的，但值不合理。你得在业务代码里手动检查

### 6.2 有 Field 的模型

```python
class BidRequest(BaseModel):
    control_price: float = Field(
        gt=0,
        description="项目控制价，单位：万元"
    )
    num_competitors: int = Field(
        ge=2, le=100,
        description="预估竞争对手数量"
    )
    strategy: str = Field(
        default="moderate",
        min_length=1, max_length=50,
        description="投标策略"
    )
    notes: str | None = Field(
        default=None,
        max_length=500,
        description="备注信息，最多 500 字"
    )
```

现在：

```json
{"control_price": -100, ...}       → 422 错误：必须大于 0
{"num_competitors": 0, ...}        → 422 错误：必须大于等于 2
{"strategy": "", ...}              → 422 错误：至少 1 个字符
{"notes": "一万个字..."}           → 422 错误：最多 500 字符
```

**约束在模型层面就拦住了，根本到不了业务代码**

## 7. 对应到你的投标项目

```python
from fastapi import FastAPI, Path, Body
from pydantic import BaseModel, Field
from typing import Annotated

app = FastAPI(title="智能投标决策系统")


# ==================== 模型定义（每个字段都有约束和文档说明） ====================

class HistoricalBid(BaseModel):
    """单条历史投标记录"""
    project_name: str = Field(
        min_length=1, max_length=200,
        description="历史项目名称"
    )
    bid_price: float = Field(
        gt=0,
        description="当时的投标报价，单位：万元"
    )
    won: bool = Field(
        description="是否中标"
    )


class PredictRequest(BaseModel):
    """投标预测请求"""
    control_price: float = Field(
        gt=0,
        description="本次项目的控制价，单位：万元",
        examples=[1200.5, 850.0]
    )
    num_competitors: int = Field(
        ge=2, le=100,
        description="预估竞争对手数量（不含自己）"
    )
    strategy: str = Field(
        default="moderate",
        min_length=1, max_length=50,
        description="投标策略：aggressive / moderate / conservative"
    )
    history: list[HistoricalBid] = Field(
        default=[],
        description="历史投标记录列表，用于辅助分析"
    )
    notes: str | None = Field(
        default=None,
        max_length=500,
        description="备注信息"
    )


class PredictResponse(BaseModel):
    """投标预测结果"""
    recommended_price: float = Field(
        description="推荐报价，单位：万元"
    )
    win_probability: float = Field(
        ge=0, le=1,
        description="预估中标概率"
    )
    price_range: dict = Field(
        description="推荐报价区间"
    )
    strategy: str = Field(
        description="使用的策略"
    )


# ==================== 接口定义 ====================

@app.post("/api/projects/{project_id}/predict", response_model=PredictResponse)
def predict_bid(
    project_id: Annotated[int, Path(ge=1, description="项目 ID")],
    request: PredictRequest,
):
    """
    提交投标参数，获取最优报价建议

    请求示例：
    {
        "control_price": 1200.5,
        "num_competitors": 8,
        "strategy": "aggressive",
        "history": [
            {"project_name": "路面改造A", "bid_price": 980, "won": true},
            {"project_name": "桥梁建设B", "bid_price": 1100, "won": false}
        ]
    }
    """
    recommended = request.control_price * 0.95

    return PredictResponse(
        recommended_price=recommended,
        win_probability=0.68,
        price_range={
            "low": request.control_price * 0.90,
            "high": request.control_price * 0.98
        },
        strategy=request.strategy,
    )
```

打开 `/docs` 页面，会看到每个字段旁边都有 `description` 的内容，还有约束信息（"大于 0"、"最多 500 字符"等）。不用写一行文档，全自动

## 8. 本节核心知识点速查

```css
谁              用在哪              从哪导入            功能
──────────────────────────────────────────────────────────────
Query()        函数参数（查询参数）  from fastapi       约束 + 标记来源
Path()         函数参数（路径参数）  from fastapi       约束 + 标记来源
Body()         函数参数（请求体）    from fastapi       约束 + 标记来源
Field()        Pydantic 模型字段    from pydantic      约束 + 文档信息
```

**本节最核心的一句话**：`Field()` 就是模型内部版本的 `Query()`/`Path()`/`Body()`。语法一样、参数一样、功能一样，只是用的地方不同——函数签名里用 Query/Path/Body，模型定义里用 Field。记住 Field 从 `pydantic` 导入，其他三个从 `fastapi` 导入
