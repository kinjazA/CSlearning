## 1. 为什么需要额外验证

### 1.1 光靠类型注解不够用

之前你学到 `q: str` 能保证参数是字符串。但"是字符串"这个条件太宽松了：

```python
@app.get("/items/")
def read_items(q: str | None = None):
    ...
```

用户传什么字符串都行：

```
?q=a                          ← 一个字符，太短了，搜什么？
?q=这是一段超级超级超级长的查询...  ← 几千个字符，可能搞崩你的数据库
?q=<script>alert('xss')</script>  ← 恶意输入
```

你需要告诉 FastAPI："这个字符串最少 3 个字符、最多 50 个字符"，甚至"必须符合某个格式"。这就是 `Query()` 干的事

## 2. Annotated + Query：给查询参数加规矩

### 2.1 核心语法

```python
from typing import Annotated
from fastapi import FastAPI, Query

app = FastAPI()


@app.get("/items/")
def read_items(q: Annotated[str | None, Query(max_length=50)] = None):
    results = {"items": [{"item_id": "Foo"}, {"item_id": "Bar"}]}
    if q:
        results.update({"q": q})
    return results
```

关键变化在参数的类型注解上。从简单的写法变成了带约束的写法：

```python
# 之前：只规定类型
q: str | None = None

# 现在：规定类型 + 额外约束
q: Annotated[str | None, Query(max_length=50)] = None
```

### 2.2 拆解这个语法——一层一层看

**第一层：`= None`**

```python
... = None
```

这是默认值。和之前一样，`= None` 表示这个参数是可选的，不传就是 `None`。**判断参数是否必填，永远看有没有默认值**

**第二层：`Annotated[..., ...]`**

```python
Annotated[str | None, Query(max_length=50)]
```

`Annotated` 是 Python 标准库 `typing` 里的东西。它的作用很简单：**给类型注解附加额外信息**

```python
Annotated[类型, 附加信息]
#          ↑        ↑
#     第一个参数   第二个参数
#     原来的类型   额外的约束/元数据
```

- 第一个参数 `str | None`：原来的类型注解，和之前一样
- 第二个参数 `Query(max_length=50)`：额外的约束条件

**第三层：`Query(max_length=50)`**

```python
Query(max_length=50)
```

`Query` 是从 `fastapi` 导入的。它告诉 FastAPI这是一个查询参数，并且有额外约束。`max_length=50` 就是约束条件，即最多 50 个字符

### 2.3 合在一起读

```python
q: Annotated[str | None, Query(max_length=50)] = None
```

翻译成人话：

> `q` 是一个查询参数，类型是字符串或 None，最大长度 50 个字符，默认值是 None

### 2.4 如果违反约束会怎样

```
GET /items/?q=这个字符串超过了五十个字符的限制这个字符串超过了五十个字符的限制这个字符串超过了五十个字符的限制
```

FastAPI 自动返回 422 错误：

```json
{
    "detail": [
        {
            "type": "string_too_long",
            "loc": ["query", "q"],
            "msg": "String should have at most 50 characters",
            "input": "这个字符串超过了..."
        }
    ]
}
```

## 3. 字符串能加哪些约束

### 3.1 最大长度 + 最小长度

```python
@app.get("/items/")
def read_items(
    q: Annotated[str | None, Query(min_length=3, max_length=50)] = None,
):
    ...
```

```
?q=ab         → 422 错误：至少 3 个字符
?q=abc        → ✅ 通过
?q=正常的搜索   → ✅ 通过
?q=超过50个字... → 422 错误：最多 50 个字符
```

### 3.2 正则表达式匹配

如果需要参数符合某种格式，可以用正则表达式：

```python
@app.get("/items/")
def read_items(
    q: Annotated[
        str | None,
        Query(min_length=3, max_length=50, pattern="^[a-zA-Z0-9_]+$")
    ] = None,
):
    ...
```

`pattern="^[a-zA-Z0-9_]+$"` 的意思是：只允许字母、数字和下划线

```
?q=hello_world  → ✅ 通过
?q=test123      → ✅ 通过
?q=有中文        → 422 错误：不匹配正则
?q=has space    → 422 错误：不匹配正则（有空格）
```

**正则表达式速查**（够用就行，不用深入学）：

```
^           → 字符串的开头
$           → 字符串的结尾
[a-z]       → 小写字母
[A-Z]       → 大写字母
[0-9]       → 数字
+           → 前面的东西出现一次或多次
*           → 前面的东西出现零次或多次
.           → 任意一个字符
\d          → 数字（等同于 [0-9]）
\w          → 字母数字下划线

常见组合：
^[a-zA-Z0-9_]+$     → 只允许字母数字下划线
^\d{4}-\d{2}-\d{2}$ → 日期格式 2024-01-15
^1[3-9]\d{9}$       → 中国大陆手机号
```

正则表达式本身是个大话题，不用现在全学会。知道 `pattern` 参数的存在，用到的时候查就行

## 4. 旧写法 vs 新写法

### 4.1 新写法（推荐，用 Annotated）

```python
from typing import Annotated
from fastapi import Query

# Annotated 写法：类型和约束放在 Annotated 里，默认值放在 = 后面
q: Annotated[str | None, Query(max_length=50)] = None
```

### 4.2 旧写法（0.95.0 之前，用 Query 作为默认值）

```python
from fastapi import Query

# 旧写法：Query() 直接作为默认值
q: str | None = Query(default=None, max_length=50)
```

### 4.3 两种写法的区别

功能上完全一样。但 **Annotated 写法更好**，原因是：

```python
# ---- 旧写法的问题 ----
q: str | None = Query(default=None, max_length=50)

# 默认值不是 None，而是 Query(...)
# 如果你在别的地方（非 FastAPI）调用这个函数，
# 不传 q 参数时，q 的值是一个 Query 对象，不是 None
# 这很反直觉，也容易出 bug


# ---- 新写法没有这个问题 ----
q: Annotated[str | None, Query(max_length=50)] = None

# 默认值就是 None，清清楚楚
# 不管在 FastAPI 里还是别的地方调用，不传 q 就是 None
```

### 4.4 一个绝对不能犯的错

用 Annotated 时，**不要在 Query() 里再写 default**：

```python
# ❌ 错误！两个地方都定义了默认值，FastAPI 不知道听哪个
q: Annotated[str, Query(default="rick")] = "morty"
# "rick" 还是 "morty"？矛盾了！

# ✅ 正确：默认值只在 = 后面写
q: Annotated[str, Query()] = "rick"
```

## 5. 必填 vs 可选——加了 Query 之后规则变了吗？

### 5.1 没变，还是看有没有默认值

```python
# 可选，默认 None
q: Annotated[str | None, Query(max_length=50)] = None

# 可选，默认 "fixedquery"
q: Annotated[str, Query(min_length=3)] = "fixedquery"

# 必填（没有默认值）
q: Annotated[str, Query(min_length=3)]
```

**规则从来没变过**：有默认值（包括 `None`）就是可选的，没默认值就是必填的。加了 `Query()` 只是附加了额外约束，不影响是否必填

### 5.2 一个特殊情况：必填但可以为 None

```python
# 必填，但值可以是 None
q: Annotated[str | None, Query(min_length=3)]
```

没有默认值 → 必填。类型是 `str | None` → 允许传 `None`。这意味着客户端**必须显式传这个参数**，但传的值可以是 `null`。这种场景很少见，知道有这回事就行

## 6. 查询参数接收多个值（列表）

### 6.1 同一个查询参数传多次

有时候需要一个参数接收多个值。比如按多个标签筛选：

```
GET /items/?tag=python&tag=fastapi&tag=docker
```

同一个参数名 `tag` 出现了三次，每次传不同的值

### 6.2 怎么接收

```python
@app.get("/items/")
def read_items(
    q: Annotated[list[str] | None, Query()] = None
):
    return {"q": q}
```

**关键点**：类型写成 `list[str]`，FastAPI 就知道这个参数可以接收多个值

```css
GET /items/?q=foo&q=bar
→ q = ["foo", "bar"]

GET /items/
→ q = None（没传，用默认值）
```

### 6.3 为什么必须显式写 Query()

```python
# ❌ 不写 Query()，FastAPI 会把 list[str] 当成请求体
q: list[str] | None = None

# ✅ 写了 Query()，FastAPI 才知道这是查询参数
q: Annotated[list[str] | None, Query()] = None
```

这是因为 FastAPI 的判断规则里，复杂类型（比如 `list`）默认会被当成请求体。加了 `Query()` 就是显式告诉 FastAPI："这是查询参数，不是请求体"

### 6.4 带默认值的列表

```python
@app.get("/items/")
def read_items(
    q: Annotated[list[str], Query()] = ["foo", "bar"]
):
    return {"q": q}
```

```
GET /items/          → q = ["foo", "bar"]（用默认值）
GET /items/?q=hello  → q = ["hello"]
```

## 7. 给参数加文档信息

### 7.1 title 和 description

这些不影响功能，但会让自动生成的文档更好看：

```python
@app.get("/items/")
def read_items(
    q: Annotated[
        str | None,
        Query(
            title="搜索关键词",
            description="用于在数据库中搜索匹配项目的关键词，支持模糊匹配",
            min_length=3,
            max_length=50,
        ),
    ] = None,
):
    ...
```

打开 `/docs`，你会在参数旁边看到 `title` 和 `description` 的内容

### 7.2 alias：参数别名

有时候 URL 里的参数名不是合法的 Python 变量名。比如前端要求参数名是 `item-query`（带横杠），但 Python 变量名不能有横杠

```python
@app.get("/items/")
def read_items(
    q: Annotated[str | None, Query(alias="item-query")] = None
):
    # URL 里用 item-query，代码里用 q
    return {"q": q}
```

```
GET /items/?item-query=hello
→ q = "hello"

# FastAPI 从 URL 里找 "item-query"（别名），
# 但在函数里你用变量名 q 来访问这个值
```

**什么时候用？**

- 前端约定的参数名有横杠或其他特殊字符
- 想让 URL 里的参数名更直观，但代码里用简短的变量名
- 和第三方系统对接，对方定好了参数名格式

### 7.3 deprecated：标记参数为"已弃用"

你想下线某个参数，但还有老客户端在用，不能直接删。可以标记为弃用：

```python
@app.get("/items/")
def read_items(
    q: Annotated[
        str | None,
        Query(
            deprecated=True,
            title="旧版搜索参数",
            description="请使用 keyword 参数替代",
        ),
    ] = None,
    keyword: Annotated[str | None, Query(min_length=3)] = None,
):
    # 新参数 keyword 和旧参数 q 都接受，优先用 keyword
    search_term = keyword or q
    ...
```

在 `/docs` 页面上，`q` 参数会被标记为 ~~deprecated~~，提醒使用者不要继续用

### 7.4 include_in_schema：从文档中隐藏参数

```python
@app.get("/items/")
def read_items(
    hidden_query: Annotated[str | None, Query(include_in_schema=False)] = None,
):
    ...
```

这个参数功能正常，但不会出现在 `/docs` 文档页面上。适合内部调试用的参数，不想让外部用户看到

## 8. 自定义验证器（AfterValidator）

### 8.1 什么时候用

`min_length`、`max_length`、`pattern` 这些内置约束能解决大部分场景。但有时候你的校验逻辑更复杂，比如：

- 参数值必须以 `isbn-` 或 `imdb-` 开头
- 参数值必须是某种业务编码格式
- 参数值必须通过某种自定义计算

这时候就需要自己写校验函数。

### 8.2 怎么用

```python
from typing import Annotated
from fastapi import FastAPI
from pydantic import AfterValidator

app = FastAPI()


# 第一步：定义校验函数
def check_valid_id(value: str) -> str:
    """
    校验函数的规则：
    1. 接收一个值作为参数（类型和参数声明一致）
    2. 如果值合法 → 返回这个值（或处理后的值）
    3. 如果值不合法 → raise ValueError("错误信息")
    """
    if not value.startswith(("isbn-", "imdb-")):
        raise ValueError('ID 必须以 "isbn-" 或 "imdb-" 开头')
    return value


# 第二步：用 AfterValidator 把校验函数挂上去
@app.get("/items/")
def read_items(
    id: Annotated[str | None, AfterValidator(check_valid_id)] = None,
):
    return {"id": id}
```

**运行效果**：

```
GET /items/?id=isbn-123456   → ✅ 通过，id = "isbn-123456"
GET /items/?id=imdb-tt037    → ✅ 通过，id = "imdb-tt037"
GET /items/?id=random-abc    → 422 错误：ID 必须以 "isbn-" 或 "imdb-" 开头
GET /items/                  → ✅ 通过，id = None（没传，用默认值）
```

### 8.3 AfterValidator vs BeforeValidator

```
AfterValidator  → 先做类型转换和内置校验，然后执行你的函数
                  你的函数拿到的是已经转好类型的值

BeforeValidator → 先执行你的函数，然后才做类型转换和内置校验
                  你的函数拿到的是原始的输入值
```

大部分情况用 `AfterValidator` 就行。

### 8.4 可以和 Query 一起用

```python
@app.get("/items/")
def read_items(
    id: Annotated[
        str | None,
        Query(min_length=5, max_length=50),  # 内置约束
        AfterValidator(check_valid_id),       # 自定义校验
    ] = None,
):
    ...
```

`Annotated` 里可以放多个约束，它们会按顺序执行：先检查长度，再执行你的自定义校验

## 9. 完整示例：把所有功能串在一起

```python
from typing import Annotated
from fastapi import FastAPI, Query
from pydantic import AfterValidator

app = FastAPI(title="智能投标决策系统")


def validate_keyword(value: str) -> str:
    """不允许搜索关键词包含 SQL 注入常见字符"""
    dangerous = ["'", '"', ";", "--", "/*"]
    for char in dangerous:
        if char in value:
            raise ValueError(f"搜索关键词不允许包含 '{char}'")
    return value


@app.get("/api/bids")
def search_bids(
    # 必填查询参数 + 长度约束 + 自定义校验
    keyword: Annotated[
        str,
        Query(
            min_length=2,
            max_length=100,
            title="搜索关键词",
            description="搜索投标记录的关键词，至少 2 个字符",
        ),
        AfterValidator(validate_keyword),
    ],

    # 可选查询参数 + 默认值
    page: Annotated[
        int,
        Query(title="页码", description="分页页码，从 1 开始"),
    ] = 1,

    page_size: Annotated[
        int,
        Query(title="每页条数", description="每页返回的记录数"),
    ] = 20,

    # 可选查询参数 + 列表类型
    tags: Annotated[
        list[str] | None,
        Query(title="标签筛选", description="按标签过滤，可多选"),
    ] = None,

    # 布尔开关
    include_analysis: Annotated[
        bool,
        Query(title="包含分析", description="是否在结果中包含分析数据"),
    ] = False,

    # 已弃用的旧参数
    q: Annotated[
        str | None,
        Query(deprecated=True, description="已弃用，请使用 keyword 参数"),
    ] = None,
):
    """
    搜索投标记录

    请求示例：
    GET /api/bids?keyword=路面改造&page=2&tags=市政&tags=道路&include_analysis=true
    """
    return {
        "keyword": keyword,
        "page": page,
        "page_size": page_size,
        "tags": tags,
        "include_analysis": include_analysis,
        "deprecated_q": q,
    }
```

## 10. 本节核心知识点速查

```css
功能                    写法                                    作用
──────────────────────────────────────────────────────────────────────────
最大长度              Query(max_length=50)                    字符串最多 50 字符
最小长度              Query(min_length=3)                     字符串至少 3 字符
正则匹配              Query(pattern="^[a-z]+$")               必须匹配正则
标题                  Query(title="搜索词")                    文档显示标题
描述                  Query(description="用于...")             文档显示描述
别名                  Query(alias="item-query")               URL 中用别名
弃用标记              Query(deprecated=True)                  文档标记为弃用
隐藏参数              Query(include_in_schema=False)          文档中不显示
接收列表              list[str], Query()                      ?q=a&q=b → ["a","b"]
自定义校验            AfterValidator(check_fn)                自定义校验逻辑
```

**Annotated 的基本结构**：

```python
参数名: Annotated[类型, Query(约束条件)] = 默认值
#         ↑         ↑       ↑                ↑
#       变量名     包装器   额外约束         是否必填
```

**本节最核心的一句话**：`Query()` 就是给查询参数加额外的规矩。类型注解管"是什么类型"，`Query()` 管"还要满足什么条件"。用 `Annotated` 把它们包在一起，默认值单独写在 `=` 后面
