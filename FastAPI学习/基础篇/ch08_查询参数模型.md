## 1. 问题：查询参数太多，函数签名爆炸

==我理解这一节就是把一些频繁复用的参数，可以写在一个类里，然后类里面可以用Field()来给参数加约束条件，然后接口绑定的函数直接调用即可==

==第二个点，为什么查询参数会有这样一个模型的写法，我理解是像路径参数，其实量不大，就一两个，但是查询参数可能一写就好几个，然后复用概率也大，所以干脆就单独拿出来做一个模型，然后之前查询参数使用Query()来添加约束，现在换成一个单独的模型了，改成用Fied()==

### 1.1 之前的写法

回顾一下前几节写的搜索接口：

```python
@app.get("/api/bids")
def search_bids(
    page: Annotated[int, Query(ge=1)] = 1,
    page_size: Annotated[int, Query(ge=1, le=100)] = 20,
    keyword: Annotated[str | None, Query(min_length=2, max_length=100)] = None,
    min_price: Annotated[float | None, Query(gt=0)] = None,
    max_price: Annotated[float | None, Query(gt=0)] = None,
    sort_by: Annotated[str, Query()] = "created_at",
    tags: Annotated[list[str] | None, Query()] = None,
):
    ...
```

七个查询参数，函数签名已经很长了。而且如果另一个接口 `/api/projects` 也需要分页和排序，这些参数就得重新写一遍

### 1.2 更好的方式：打包成模型

```python
class FilterParams(BaseModel):
    page: int = 1
    page_size: int = 20
    keyword: str | None = None
    sort_by: str = "created_at"


@app.get("/api/bids")
def search_bids(filters: Annotated[FilterParams, Query()]):
    ...

@app.get("/api/projects")
def search_projects(filters: Annotated[FilterParams, Query()]):
    ...
```

**一个模型定义，到处复用。** 函数签名从七行变成一行

## 2. 完整语法

### 2.1 核心代码

```python
from typing import Annotated, Literal
from fastapi import FastAPI, Query
from pydantic import BaseModel, Field

app = FastAPI()


class FilterParams(BaseModel):
    limit: int = Field(100, gt=0, le=100)
    offset: int = Field(0, ge=0)
    order_by: Literal["created_at", "updated_at"] = "created_at"
    tags: list[str] = []


@app.get("/items/")
def read_items(filter_query: Annotated[FilterParams, Query()]):
    return filter_query
```

这段代码里有三个新东西：`Field()`、`Literal`、以及 `Annotated[模型, Query()]` 的用法。逐个拆解

### 2.2 Field()——给模型字段加约束

之前用 `Query(ge=1, le=100)` 给查询参数加约束。现在参数放进了 Pydantic 模型里，约束也要跟着进去。**在模型内部，用 `Field()` 替代 `Query()`**

```python
from pydantic import Field

class FilterParams(BaseModel):
    limit: int = Field(100, gt=0, le=100)
    #                   ↑    ↑       ↑
    #                默认值  大于0  小于等于100
```

**`Field()` 和 `Query()` 的关系**：

```python
# 在函数参数里（不用模型时）→ 用 Query()
limit: Annotated[int, Query(gt=0, le=100)] = 100

# 在 Pydantic 模型里 → 用 Field()
class FilterParams(BaseModel):
    limit: int = Field(100, gt=0, le=100)

# 两者支持的约束参数完全一样：
# gt, ge, lt, le, min_length, max_length, pattern, title, description...
```

**为什么模型里用 Field 而非 Query？** 因为 `Query` 是告诉 FastAPI "从查询参数取数据"的一个标记，它属于 FastAPI。`Field` 是 Pydantic 自己的东西，用来约束模型字段

**Field() 的第一个参数是默认值**：

```python
limit: int = Field(100, gt=0, le=100)
#                   ↑
#            等价于 default=100

# 也可以显式写 default=
limit: int = Field(default=100, gt=0, le=100)

# 没有默认值 = 必填
name: str = Field(min_length=1)  # 必填，至少 1 个字符
```

### 2.3 Literal——限定只能是几个固定值

```python
from typing import Literal

class FilterParams(BaseModel):
    order_by: Literal["created_at", "updated_at"] = "created_at"
```

`Literal["created_at", "updated_at"]` 的意思是：**这个字段的值只能是 `"created_at"` 或 `"updated_at"`，传其他任何值都报错。**

```
?order_by=created_at   → ✅
?order_by=updated_at   → ✅
?order_by=name         → ❌ 422 错误
?order_by=price        → ❌ 422 错误
```

**Literal vs Enum 的区别**：

```python
# Literal：直接在类型注解里写死几个值，简单快捷
order_by: Literal["created_at", "updated_at"] = "created_at"

# Enum：定义一个枚举类，更正式，适合多处复用
class SortField(str, Enum):
    created_at = "created_at"
    updated_at = "updated_at"

order_by: SortField = SortField.created_at
```

==如果这几个固定值只在这一个地方用，`Literal` 更简洁。如果多个地方都要用同一组值，`Enum` 更好==

### 2.4 Annotated[模型, Query()]——告诉 FastAPI 从查询参数取

```python
@app.get("/items/")
def read_items(filter_query: Annotated[FilterParams, Query()]):
    return filter_query
```

**关键语法**：`Annotated[FilterParams, Query()]`

```python
filter_query: Annotated[FilterParams, Query()]
#               ↑            ↑           ↑
#            Annotated    模型类型     从查询参数取
```

**为什么要加 `Query()`？**回想第四节的判断规则：参数类型是 Pydantic 模型 → FastAPI 默认当成请求体。但这里想让它从查询参数取数据，所以必须显式用 `Query()` 标记

```python
# 不加 Query() → FastAPI 以为是请求体（需要客户端发 JSON Body）
filter_query: FilterParams

# 加了 Query() → FastAPI 知道从 URL 查询参数里取数据
filter_query: Annotated[FilterParams, Query()]
```

### 2.5 请求 URL 长什么样

模型里的每个字段对应一个查询参数，URL 的写法和之前完全一样：

```
GET /items/?limit=20&offset=10&order_by=updated_at&tags=python&tags=fastapi
```

FastAPI 自动把这些查询参数的值填进 `FilterParams` 模型里：

```python
filter_query.limit      # 20
filter_query.offset     # 10
filter_query.order_by   # "updated_at"
filter_query.tags       # ["python", "fastapi"]
```

不传的字段用默认值：

```css
GET /items/
→ limit=100, offset=0, order_by="created_at", tags=[]
```

## 3. 禁止多传参数（extra = "forbid"）

### 3.1 默认行为：多传的参数被忽略

正常情况下，用户在 URL 里多传了模型没定义的参数，FastAPI 不会报错，直接忽略：

```css
GET /items/?limit=10&color=red
```

`color` 不在 `FilterParams` 里，FastAPI 默默忽略它。大部分时候这没问题

### 3.2 严格模式：多传就报错

有些场景想严格控制，用户传了意料之外的参数说明可能拼错了参数名，你希望直接报错提醒，而不是默默忽略

```python
class FilterParams(BaseModel):
    # 加这一行配置，禁止任何模型里没定义的字段
    model_config = {"extra": "forbid"}

    limit: int = Field(100, gt=0, le=100)
    offset: int = Field(0, ge=0)
    order_by: Literal["created_at", "updated_at"] = "created_at"
    tags: list[str] = []
```

**`model_config = {"extra": "forbid"}` 是什么？**这是 Pydantic v2 的模型配置。`model_config` 是一个字典，用来控制模型的行为。`"extra": "forbid"` 表示"不允许传入模型里没定义的字段"

```css
GET /items/?limit=10&tool=plumbus

# 没有 forbid：tool 被忽略，正常返回
# 有了 forbid：422 错误
{
    "detail": [
        {
            "type": "extra_forbidden",
            "loc": ["query", "tool"],
            "msg": "Extra inputs are not permitted",
            "input": "plumbus"
        }
    ]
}
```

### 3.3 什么时候用 forbid

```css
用 forbid 的场景：
  - 对外的公开 API，想严格约束客户端行为
  - 防止用户拼错参数名不自知（比如传了 limt 想传 limit）

不用 forbid 的场景（大多数情况）：
  - 内部 API，灵活一点没关系
  - 可能会逐步添加新参数，不想每次加参数都让旧客户端报错
```

## 4. 在函数里怎么用

```python
@app.get("/items/")
def read_items(filter_query: Annotated[FilterParams, Query()]):
    # filter_query 是一个 FilterParams 对象
    # 直接用点号访问属性
    print(filter_query.limit)       # 100
    print(filter_query.offset)      # 0
    print(filter_query.order_by)    # "created_at"
    print(filter_query.tags)        # []

    # 转成字典
    print(filter_query.model_dump())
    # {"limit": 100, "offset": 0, "order_by": "created_at", "tags": []}

    return filter_query
```

和请求体里的 Pydantic 模型用法完全一样——都是对象，都用点号访问属性

## 5. 对应到你的投标项目

### 5.1 定义可复用的查询参数模型

```python
from typing import Annotated, Literal
from fastapi import FastAPI, Path, Query
from pydantic import BaseModel, Field

app = FastAPI(title="智能投标决策系统")


# ==================== 可复用的查询参数模型 ====================

class PaginationParams(BaseModel):
    """分页参数——几乎每个列表接口都需要"""
    page: int = Field(1, ge=1, description="页码，从 1 开始")
    page_size: int = Field(20, ge=1, le=100, description="每页条数，最多 100")


class BidSearchParams(BaseModel):
    """投标记录搜索参数"""
    model_config = {"extra": "forbid"}

    # 分页
    page: int = Field(1, ge=1)
    page_size: int = Field(20, ge=1, le=100)

    # 筛选
    keyword: str | None = Field(None, min_length=2, max_length=100)
    min_price: float | None = Field(None, gt=0)
    max_price: float | None = Field(None, gt=0)
    won: bool | None = None

    # 排序
    sort_by: Literal["bid_price", "created_at", "project_name"] = "created_at"
    sort_order: Literal["asc", "desc"] = "desc"


class SimulationParams(BaseModel):
    """仿真配置参数"""
    num_simulations: int = Field(10000, ge=100, le=1000000)
    confidence_level: float = Field(0.95, gt=0, le=1)
    random_seed: int | None = None
```

### 5.2 在接口里使用

```python
# 搜索投标记录——所有搜索/筛选/排序/分页参数打包在一起
@app.get("/api/bids")
def search_bids(params: Annotated[BidSearchParams, Query()]):
    """
    GET /api/bids?keyword=路面&min_price=500&sort_by=bid_price&sort_order=asc&page=2

    所有参数自动校验：
    - keyword 长度 2~100
    - min_price > 0
    - sort_by 只能是 bid_price / created_at / project_name
    - sort_order 只能是 asc / desc
    - page >= 1, page_size 在 1~100 之间
    - 多传了未定义的参数会报错（extra=forbid）
    """
    return {
        "filters": params.model_dump(),
        "results": ["...这里是查询结果..."],
    }


# 运行仿真——仿真配置参数打包
@app.post("/api/projects/{project_id}/simulate")
def run_simulation(
    project_id: Annotated[int, Path(ge=1)],
    sim_params: Annotated[SimulationParams, Query()],
):
    """
    POST /api/projects/42/simulate?num_simulations=50000&confidence_level=0.99

    路径参数 project_id 从 URL 路径取
    仿真参数 sim_params 从查询参数取
    """
    return {
        "project_id": project_id,
        "config": sim_params.model_dump(),
        "result": "仿真完成",
    }
```

### 5.3 复用的威力

如果你有多个接口需要分页，不用每次都写 `page` 和 `page_size`：

```python
# 接口 A：搜索投标记录
@app.get("/api/bids")
def search_bids(pagination: Annotated[PaginationParams, Query()]):
    ...

# 接口 B：搜索项目
@app.get("/api/projects")
def search_projects(pagination: Annotated[PaginationParams, Query()]):
    ...

# 接口 C：搜索用户
@app.get("/api/users")
def search_users(pagination: Annotated[PaginationParams, Query()]):
    ...

# 三个接口共用同一个分页模型
# 改一处（比如把 page_size 默认值从 20 改成 30），三个接口同时生效
```

## 6. Field vs Query vs Path——到底什么时候用哪个

到这一节为止，已经见过三个长得很像的东西：`Query()`、`Path()`、`Field()`。它们支持的约束参数几乎一模一样，很容易搞混。记住这个规则：

```css
位置                           用什么
────────────────────────────────────────────
函数参数 + 查询参数             Query()
函数参数 + 路径参数             Path()
Pydantic 模型里的字段           Field()
```

```python
from fastapi import Query, Path
from pydantic import Field, BaseModel

# Query → 函数参数，标记为查询参数
def foo(q: Annotated[str, Query(max_length=50)] = None): ...

# Path → 函数参数，标记为路径参数
def bar(id: Annotated[int, Path(ge=1)]): ...

# Field → 模型字段，约束字段值
class MyModel(BaseModel):
    name: str = Field(min_length=1, max_length=100)
    price: float = Field(gt=0)
```

**一句话**：在函数签名里用 `Query`/`Path`，在模型定义里用 `Field`。它们的约束参数（`gt`、`ge`、`max_length` 等）都一样

## 7. 本节核心知识点速查

```css
概念                            写法                                     作用
───────────────────────────────────────────────────────────────────────────────
查询参数模型         Annotated[MyModel, Query()]             把查询参数打包成模型
模型字段约束         Field(default, gt=0, le=100)            在模型内部加约束
固定取值             Literal["a", "b", "c"]                  限制只能是这几个值
禁止多传参数         model_config = {"extra": "forbid"}      多传未定义的参数就报错
```

**本节最核心的一句话**：查询参数多了就打包成 Pydantic 模型，用 `Field()` 加约束，用 `Annotated[模型, Query()]` 告诉 FastAPI 从查询参数取数据。一次定义，到处复用
