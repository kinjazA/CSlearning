## 1. 基本的路径参数

### 1.1 问题：为什么需要路径参数

假设有 100 个项目，要查每个项目的详情。不可能写 100 个接口：

```python
# ❌ 不可能这样写
@app.get("/projects/1")
def get_project_1(): ...

@app.get("/projects/2")
def get_project_2(): ...

@app.get("/projects/3")
def get_project_3(): ...
# ...写到天荒地老
```

需要的是：**路径里有一个"占位符"，用户访问的时候填上具体的值，FastAPI 把这个值传给函数**

### 1.2 语法：花括号 `{}` 标记动态部分

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/items/{item_id}")
def read_item(item_id):
    return {"item_id": item_id}
```

**拆解这段代码**：

```css
@app.get("/items/{item_id}")
                  ↑
            花括号包裹的部分就是路径参数
            名字叫 item_id（自己起的，叫啥都行）


def read_item(item_id):
                ↑
          ！！函数参数名必须和花括号里的名字一致！！
          FastAPI 会自动把 URL 里的值传进来
```

**运行效果**：

```css
用户访问 → /items/foo      → item_id 拿到 "foo"    → 返回 {"item_id": "foo"}
用户访问 → /items/42       → item_id 拿到 "42"     → 返回 {"item_id": "42"}
用户访问 → /items/hello    → item_id 拿到 "hello"  → 返回 {"item_id": "hello"}
```

### 1.3 注意：此时 item_id 是字符串

==上面代码里 `item_id` 没有加类型注解，所以 FastAPI 默认把它当**字符串**处理。==即使用户访问 `/items/42`，`item_id` 拿到的也是字符串 `"42"`，不是整数 `42`

```python
@app.get("/items/{item_id}")
def read_item(item_id):         # 没有类型注解
    print(type(item_id))        # <class 'str'>，永远是字符串
    return {"item_id": item_id}
```

这就引出了下一个问题：怎么让它自动变成整数？

## 2. 给路径参数加类型注解

### 2.1 加上 `: int`，三件事自动发生

```python
@app.get("/items/{item_id}")
def read_item(item_id: int):    # 加了 : int
    return {"item_id": item_id}
```

就多写了四个字符 `: int`，FastAPI 自动帮你做三件事：

#### 第一件：自动转换类型

```css
用户访问 /items/3

不加类型注解 → item_id 是字符串 "3"
加了 : int  → item_id 是整数 3
```

FastAPI 自动把 URL 里的字符串 `"3"` 转换成了 Python 的 `int` 类型 `3`。在函数里拿到的就是整数，可以直接做运算，不用自己 `int()` 转换

#### 第二件：自动校验数据

```css
用户访问 /items/foo

FastAPI 发现 "foo" 没法转成 int → 自动返回错误响应：

{
    "detail": [
        {
            "type": "int_parsing",
            "loc": ["path", "item_id"],
            "msg": "Input should be a valid integer, unable to parse string as an integer",
            "input": "foo"
        }
    ]
}
```

这个错误信息非常清晰：

- `"loc": ["path", "item_id"]` → 错误发生在路径参数 `item_id` 上
- `"msg": "Input should be a valid integer"` → 告诉用户应该传整数
- `"input": "foo"` → 你传了什么

**你一行校验代码都没写**，FastAPI 全帮你处理了。用户传了 `4.2`（浮点数）也会报同样的错，因为 `4.2` 不是整数

#### 第三件：自动更新文档

打开 `/docs` 页面，你会看到 `item_id` 参数被标注为 `integer` 类型。Swagger UI 还会提供输入框让你测试，输入非整数会直接提示错误

### 2.2 一行代码做了三件事——这就是 FastAPI 的核心思想

```python
def read_item(item_id: int):
#                       ↑
#              就这四个字符，同时实现了：
#              1. 类型转换（字符串 → 整数）
#              2. 数据校验（不是整数就报错）
#              3. 文档生成（文档上标注为 integer）
```

官方文档反复强调的"**声明一次，三处受益**"就是这个意思。在别的框架里（如 Flask）想实现同样的效果，至少得手动写好几行代码

### 2.3 常用的类型注解有哪些

```python
# 整数
@app.get("/projects/{project_id}")
def get_project(project_id: int): ...
# /projects/42 → project_id = 42（int 类型）
# /projects/abc → 422 错误


# 字符串（默认就是，但建议显式写出来）
@app.get("/users/{username}")
def get_user(username: str): ...
# /users/zhangsan → username = "zhangsan"


# 浮点数
@app.get("/scores/{score}")
def get_score(score: float): ...
# /scores/98.5 → score = 98.5（float 类型）
# /scores/abc → 422 错误


# 布尔值（FastAPI 对布尔值很宽容）
@app.get("/settings/{enabled}")
def get_setting(enabled: bool): ...
# /settings/true  → enabled = True
# /settings/1     → enabled = True
# /settings/yes   → enabled = True
# /settings/on    → enabled = True
# /settings/false → enabled = False
# /settings/0     → enabled = False
# /settings/no    → enabled = False
```

## 3. 路径顺序：一个容易踩的坑

### 3.1 问题场景

假设系统有个"获取当前用户信息"的接口 `/users/me`，同时还有个"根据用户 ID 获取用户信息"的接口 `/users/{user_id}`

```python
# ✅ 正确写法：固定路径在前，动态路径在后
@app.get("/users/me")
def read_user_me():
    return {"user_id": "当前登录的用户"}


@app.get("/users/{user_id}")
def read_user(user_id: str):
    return {"user_id": user_id}
```

### 3.2 为什么顺序重要

FastAPI 收到一个请求后，**从上到下**逐个匹配路径，**第一个匹配上的就执行，后面的不看了**。

```css
请求 /users/me 进来了：

第一个路由：/users/me → 精确匹配！执行 read_user_me()，结束。

请求 /users/42 进来了：

第一个路由：/users/me → 不匹配（42 ≠ me）
第二个路由：/users/{user_id} → 匹配！user_id = "42"，执行 read_user()
```

### 3.3 如果顺序反了会怎样

```python
# ❌ 错误写法：动态路径在前
@app.get("/users/{user_id}")    # 这个在前面
def read_user(user_id: str):
    return {"user_id": user_id}


@app.get("/users/me")           # 这个永远不会被执行
def read_user_me():
    return {"user_id": "当前登录的用户"}
```

```css
请求 /users/me 进来了：

第一个路由：/users/{user_id} → 匹配了！
  FastAPI 认为 user_id = "me"，执行 read_user()
  返回 {"user_id": "me"}

第二个路由：/users/me → 根本不会被检查到
```

`"me"` 被当成了 `user_id` 的值，`read_user_me` 函数永远不会被调用

### 3.4 记住这个原则

==固定路径写前面，动态路径写后面。具体的在前，通用的在后==

这个规则不光 FastAPI 是这样，几乎所有 Web 框架的路由匹配都是这个逻辑

### 3.5 同一路径同一方法不能重复定义

```python
# ❌ 两个完全一样的路由
@app.get("/users")
def read_users():
    return ["张三", "李四"]


@app.get("/users")           # 同样的路径和方法
def read_users_v2():
    return ["王五", "赵六"]
```

==只有第一个会被执行。第二个形同虚设，FastAPI 不会报错但永远走不到==

## 4. 预定义值：用枚举限制路径参数的取值

### 4.1 问题：如果路径参数只允许几个特定的值怎么办

假设你的投标系统支持三种策略：`aggressive`（激进）、`moderate`（稳健）、`conservative`（保守）。你希望用户只能从这三个里选，传其他的就报错。用 `str` 类型注解做不到这件事——`str` 只能保证是字符串，不能限制具体是哪几个字符串

### 4.2 什么是枚举（Enum）

枚举是 Python 内置的一种数据类型，用来定义**一组固定的常量**

```python
from enum import Enum


# 定义一个枚举类
class Color(str, Enum):
    red = "red"
    green = "green"
    blue = "blue"

# 使用
print(Color.red)         # Color.red
print(Color.red.value)   # "red"（拿到实际的字符串值）
print(Color.red.name)    # "red"（拿到属性名）
```

**为什么要继承 `str` 和 `Enum` 两个类？**

```python
class ModelName(str, Enum):
#               ↑     ↑
#             str    Enum
```

- 继承 `Enum`：让这个类成为枚举类，值是固定的那几个
- 继承 `str`：告诉 FastAPI 和文档系统"这些值都是字符串类型"

如果只继承 `Enum` 不继承 `str`，FastAPI 的自动文档不知道这些值是什么类型，渲染会有问题。**所以两个都要继承，这是固定写法，记住就行**

### 4.3 在 FastAPI 中使用枚举

```python
from enum import Enum
from fastapi import FastAPI


# ==================== 第一步：定义枚举类 ====================
class StrategyName(str, Enum):
    """投标策略：只允许这三个值"""
    aggressive = "aggressive"       # 激进
    moderate = "moderate"           # 稳健
    conservative = "conservative"   # 保守


app = FastAPI()


# ==================== 第二步：用枚举类作为类型注解 ====================
@app.get("/strategies/{strategy_name}")
def get_strategy(strategy_name: StrategyName):
    """
    strategy_name 的类型是 StrategyName（枚举类）
    FastAPI 会自动检查用户传的值是不是这三个中的一个
    """

    # ---- 比较枚举成员：用 is 或 == ----
    if strategy_name is StrategyName.aggressive:
        return {
            "strategy": strategy_name,
            "description": "高风险高收益，报价压到最低"
        }

    # ---- 拿到枚举的实际字符串值：用 .value ----
    if strategy_name.value == "conservative":
        return {
            "strategy": strategy_name,
            "description": "求稳为主，报价接近控制价"
        }

    # ---- 剩下的就是 moderate ----
    return {
        "strategy": strategy_name,
        "description": "平衡风险和收益"
    }
```

**运行效果**：

```css
GET /strategies/aggressive
→ {"strategy": "aggressive", "description": "高风险高收益，报价压到最低"}

GET /strategies/moderate
→ {"strategy": "moderate", "description": "平衡风险和收益"}

GET /strategies/conservative
→ {"strategy": "conservative", "description": "求稳为主，报价接近控制价"}

GET /strategies/random
→ 422 错误！"random" 不在枚举的可选值里
```

### 4.4 枚举在文档里的效果

打开 `/docs`，你会看到 `strategy_name` 参数变成了一个**下拉菜单**，只能从 `aggressive`、`moderate`、`conservative` 三个选项里选

### 4.5 枚举成员的两种比较方式

```python
# 方式一：用 is 比较枚举成员本身
if strategy_name is StrategyName.aggressive:
    ...

# 方式二：用 == 比较枚举成员的值（字符串）
if strategy_name.value == "aggressive":
    ...

# 两种都能用。推荐方式一，更 Pythonic，也更安全
# 因为方式二有可能你把字符串打错了（比如写成 "agressive" 少了个 g），
# Python 不会报错，但逻辑就错了
```

### 4.6 返回枚举成员时会怎样

可以直接把枚举成员塞进字典里返回，FastAPI 自动把它转成对应的字符串值：

```python
return {"strategy": strategy_name}

# strategy_name 是 StrategyName.aggressive（枚举成员）
# FastAPI 返回给客户端的 JSON 里是 "aggressive"（字符串）
# 转换是自动的，你不需要手动 .value
```

## 5. 包含路径的路径参数（特殊场景）

### 5.1 问题：路径参数里有斜杠 `/` 怎么办

正常情况下，`/` 是 URL 路径的分隔符。但有时候你的参数值本身就包含 `/`。比如想传一个文件路径 `home/johndoe/myfile.txt` 作为参数：

```
你想要的 URL：/files/home/johndoe/myfile.txt
```

但 FastAPI 会解析成四层路径 `/files/home/johndoe/myfile.txt`，而不把 `home/johndoe/myfile.txt` 当成一个整体参数

### 5.2 解决方案：`:path` 转换器

```python
@app.get("/files/{file_path:path}")
def read_file(file_path: str):
    return {"file_path": file_path}
```

==**关键语法**：`{file_path:path}`==

```
{file_path:path}
  ↑          ↑
 参数名    转换器类型（告诉 FastAPI：这个参数可以包含斜杠）
```

**运行效果**：

```
GET /files/home/johndoe/myfile.txt
→ file_path = "home/johndoe/myfile.txt"（整个路径作为一个参数值）

GET /files/data/2024/report.csv
→ file_path = "data/2024/report.csv"
```

### 5.3 这个功能用得多吗

不多。大部分 API 不会在路径参数里传文件路径。如果需要传文件路径，通常会放在查询参数或请求体里。但知道有这个东西就行，万一遇到了不会懵

## 6. 多个路径参数

一个路径里可以有多个动态部分：

```python
@app.get("/users/{user_id}/projects/{project_id}")
def get_user_project(user_id: int, project_id: int):
    """
    路径里有两个参数：user_id 和 project_id
    函数参数名必须和花括号里的名字一一对应
    """
    return {
        "user_id": user_id,
        "project_id": project_id
    }

# GET /users/5/projects/42
# → user_id = 5, project_id = 42
```

**命名规则**：花括号里的名字和函数参数名**必须一致**

```python
# ❌ 会报错：花括号里叫 user_id，函数参数叫 uid，名字不匹配
@app.get("/users/{user_id}")
def get_user(uid: int):
    ...

# ✅ 必须保持一致
@app.get("/users/{user_id}")
def get_user(user_id: int):
    ...
```

## 7. 本节核心知识点速查

```css
概念                     语法                              作用
──────────────────────────────────────────────────────────────────────
路径参数               /items/{item_id}                URL 里的动态变量
类型注解               item_id: int                    自动转换 + 校验 + 文档
路径顺序               固定路径写前面                    防止动态路径"吃掉"固定路径
枚举限制               class X(str, Enum)              限制参数只能是几个固定值
路径转换器             {file_path:path}                参数值可以包含斜杠 /
多路径参数             /users/{uid}/items/{iid}        一个路径里多个动态部分
```

**核心记忆点**：给路径参数加类型注解，一行代码同时实现类型转换、数据校验、文档生成。这是 FastAPI 的核心设计哲学——**声明一次，到处受益**
