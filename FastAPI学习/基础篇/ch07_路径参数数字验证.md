## 1. Path()：给路径参数加约束

### 1.1 和 Query() 完全对称

上一节`Query()` 给**查询参数**加约束。这一节`Path()` 给**路径参数**加约束。语法结构一模一样，只是把 `Query` 换成了 `Path`：

```python
from typing import Annotated
from fastapi import FastAPI, Path, Query

app = FastAPI()


@app.get("/items/{item_id}")
def read_items(
    item_id: Annotated[int, Path(title="要获取的项目 ID")],
    q: Annotated[str | None, Query(alias="item-query")] = None,
):
    results = {"item_id": item_id}
    if q:
        results.update({"q": q})
    return results
```

**对比一下两者的写法**：

```python
# 查询参数 + 约束
q: Annotated[str | None, Query(max_length=50)] = None

# 路径参数 + 约束
item_id: Annotated[int, Path(title="项目 ID")]
```

结构完全一致：`Annotated[类型, Query或Path(约束条件)]`

### 1.2 Path() 能加哪些信息

和 `Query()` 一样，`Path()` 支持这些参数：

```python
Path(
    title="项目 ID",              # 文档标题
    description="数据库中的主键",  # 文档描述
    ge=1,                         # 大于等于 1（数值约束，下面会详细讲）
    le=10000,                     # 小于等于 10000
    # deprecated=True,            # 标记为弃用（路径参数很少用这个）
)
```

### 1.3 路径参数永远是必填的

这是路径参数和查询参数的一个重要区别：

```python
# 查询参数可以是可选的
q: Annotated[str | None, Query()] = None       # ✅ 可选

# 路径参数永远是必填的，因为它是 URL 的一部分
item_id: Annotated[int, Path()]                 # 永远必填
```

想想就知道为什么——路径参数是 URL 路径的一部分。`/items/{item_id}` 里的 `{item_id}` 如果没有值，URL 就不完整，请求根本发不出去。所以就算你写了默认值或者 `None`，路径参数也还是必填的，默认值会被忽略：

```python
# 虽然写了默认值，但 item_id 仍然是必填的，这个默认值没有任何意义，不要这样写
item_id: Annotated[int, Path()] = 42    # 默认值被忽略，还是必填
```

## 2. 数值验证

### 2.1 为什么需要数值约束

上一节的字符串约束管的是"长度和格式"。数值类型没有长度的概念，它需要的是"范围"

### 2.2 四个约束参数

```css
参数     英文全称                     含义          数学符号
────────────────────────────────────────────────────────
gt      greater than                大于            >
ge      greater than or equal       大于等于         >=
lt      less than                   小于            <
le      less than or equal          小于等于         <=
```

**记忆方法**：就是英文首字母缩写。`g` = greater，`l` = less，`t` = than，`e` = equal

### 2.3 基本用法

```python
from typing import Annotated
from fastapi import FastAPI, Path

app = FastAPI()


# ==================== ge：大于等于 ====================
@app.get("/items/{item_id}")
def read_items(
    item_id: Annotated[int, Path(ge=1)],
    # item_id >= 1，也就是 1, 2, 3, ... 都行，0 和负数不行
):
    return {"item_id": item_id}

# /items/1    → ✅ item_id = 1
# /items/100  → ✅ item_id = 100
# /items/0    → ❌ 422 错误：值必须大于等于 1
# /items/-5   → ❌ 422 错误：值必须大于等于 1
```

```python
# ==================== gt + le：组合使用，限定范围 ====================
@app.get("/items/{item_id}")
def read_items(
    item_id: Annotated[int, Path(gt=0, le=1000)],
    # 0 < item_id <= 1000
    # gt=0 表示"大于 0"（不包含 0）
    # le=1000 表示"小于等于 1000"（包含 1000）
):
    return {"item_id": item_id}

# /items/0     → ❌ 必须大于 0（gt 不包含边界）
# /items/1     → ✅
# /items/1000  → ✅（le 包含边界）
# /items/1001  → ❌ 必须小于等于 1000
```

### 2.4 gt vs ge、lt vs le 的区别

```
ge=1  → 值 >= 1  → 1 可以 ✅，0 不行 ❌
gt=1  → 值 > 1   → 1 不行 ❌，2 可以 ✅

le=10 → 值 <= 10 → 10 可以 ✅，11 不行 ❌
lt=10 → 值 < 10  → 10 不行 ❌，9 可以 ✅
```

**什么时候用 gt，什么时候用 ge？**

```
ID 类的参数     → ge=1（ID 从 1 开始，1 本身是合法值）
"必须为正数"    → gt=0（大于 0，0 本身不算正数）
```

### 2.5 浮点数的约束

数值约束不只能用在整数上，浮点数也行：

```python
@app.get("/items/{item_id}")
def read_items(
    item_id: Annotated[int, Path(ge=0, le=1000)],
    q: str,
    size: Annotated[float, Query(gt=0, lt=10.5)],
    # size 是查询参数，浮点数类型
    # 0 < size < 10.5
):
    results = {"item_id": item_id, "size": size}
    return results
```

```
?size=0      → ❌ 必须大于 0
?size=0.1    → ✅
?size=5.5    → ✅
?size=10.5   → ❌ 必须小于 10.5（lt 不包含边界）
?size=10.49  → ✅
```

**浮点数用 gt/lt 比 ge/le 更常见**，因为浮点数的边界值往往不需要"刚好等于"。比如"大于 0"用 `gt=0` 比 `ge=0.0001` 更优雅

### 2.6 数值约束用在 Query() 和 Path() 上都行

这四个参数不是 `Path` 独有的。`Query`、`Path` 都能用：

```python
# 路径参数的数值约束
item_id: Annotated[int, Path(ge=1, le=10000)]

# 查询参数的数值约束
page: Annotated[int, Query(ge=1)] = 1
price: Annotated[float, Query(gt=0)] = 100.0
```

## 3. Query与Path

### 3.1 它们支持的参数完全一样

`Query()` 和 `Path()` 底层继承自同一个父类 `Param`，所以它们支持的参数是一样的：

```python
# 所有共享参数（Query 和 Path 都能用）：

# ---- 文档信息 ----
title="xxx"                  # 文档标题
description="xxx"            # 文档描述
deprecated=True              # 标记为弃用
include_in_schema=False      # 从文档中隐藏

# ---- 字符串约束 ----
min_length=3                 # 最小长度
max_length=50                # 最大长度
pattern="^[a-z]+$"           # 正则匹配

# ---- 数值约束 ----
gt=0                         # 大于
ge=1                         # 大于等于
lt=100                       # 小于
le=99                        # 小于等于

# ---- 别名 ----
alias="item-query"           # URL 中使用的名字
```

唯一的区别是使用场景：

```css
Query()  → 用在查询参数上
Path()   → 用在路径参数上
Body()   → 用在请求体上（后面会学到）
Header() → 用在请求头上（后面会学到）
Cookie() → 用在 Cookie 上（后面会学到）
```

### 3.2 记住：它们只是"标记"从哪里取数据 + 加约束

```python
@app.get("/items/{item_id}")
def read_items(
    item_id: Annotated[int, Path(ge=1)],           # 从路径取，要 >= 1
    q: Annotated[str | None, Query(max_length=50)] = None,  # 从查询参数取，最长 50
):
    ...
```

其实大部分情况下，不加 `Path()` 也完全能用——FastAPI 根据参数名是否出现在路径 `{}` 里就能自动判断。`Path()` 的主要作用是**加约束和加文档信息**

## 4. 参数顺序的问题（用 Annotated 就不用操心）

### 4.1 旧写法有个坑

如果不用 `Annotated`，用旧写法把 `Path()` 作为默认值，会遇到 Python 语法问题：

```python
# ❌ Python 语法错误！
# 有默认值的 item_id 写在了没默认值的 q 前面
@app.get("/items/{item_id}")
def read_items(item_id: int = Path(title="项目ID"), q: str):
    ...
# SyntaxError: non-default argument follows default argument
```

Python 规定：**有默认值的参数不能写在没默认值的参数前面**。旧写法的解决方案是用 `*` 技巧或调换参数顺序，挺麻烦的

### 4.2 用 Annotated 就没这个问题

```python
# ✅ Annotated 写法，参数顺序随便写
@app.get("/items/{item_id}")
def read_items(
    item_id: Annotated[int, Path(title="项目ID")],
    q: str,
):
    ...
```

因为 `Annotated` 把约束信息放在了类型注解里，而不是默认值里。函数参数的默认值该是什么就是什么，Python 不会报语法错误。**这也是为什么推荐用 Annotated 的又一个理由**

### 4.3 `*` 技巧（了解即可）

旧写法有个技巧：在参数列表第一个位置放一个 `*`，强制后面所有参数必须用关键字传参：

```python
# 旧写法的 * 技巧
@app.get("/items/{item_id}")
def read_items(*, item_id: int = Path(title="项目ID"), q: str):
    ...
```

`*` 之后的参数都变成了"关键字参数"，Python 不再检查默认值顺序，所以不会报语法错误。**但你用 Annotated 就完全不需要这个技巧，知道有这回事就行**

## 5. 本节核心知识点速查

```css
数值约束参数     含义              示例
──────────────────────────────────────────────────
gt=0           大于 0            0 不行，0.1 可以
ge=1           大于等于 1         1 可以，0 不行
lt=100         小于 100          100 不行，99 可以
le=100         小于等于 100       100 可以，101 不行

常见组合：
  ID 类          ge=1            （从 1 开始）
  正数           gt=0            （大于 0，不含 0）
  范围           ge=1, le=1000   （1 到 1000 之间）
  概率           gt=0, le=1      （0 到 1，不含 0）
  百分比         ge=0, le=100    （0 到 100）
  页码           ge=1            （从第 1 页开始）
  每页条数       ge=1, le=100    （最少 1 条，最多 100 条）
```

```css
Path vs Query 对比：

              Path()                    Query()
用在哪里      路径参数                   查询参数
是否必填      永远必填                   取决于有没有默认值
支持的约束    完全一样（字符串约束 + 数值约束 + 文档信息）
```

**本节最核心的一句话**：`Path()` 和 `Query()` 是一家人，语法一模一样，只是分别用在路径参数和查询参数上。数值约束就四个参数——`gt`、`ge`、`lt`、`le`，对应大于、大于等于、小于、小于等于，组合使用就能限定任意数值范围
