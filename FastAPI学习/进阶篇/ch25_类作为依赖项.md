上一节你学了依赖注入的基础：把共用逻辑提取成函数，用 `Depends()` 让 FastAPI 自动调用但那个版本有一个实际开发中很恼人的问题：**依赖项函数返回的是 `dict`，编辑器不知道里面有什么键**



你写 `commons["q"]` 的时候不会有自动补全，写错了键名（比如 `commons["qurey"]`）也不会有任何提示



这一节的核心就一件事：**用类（class）代替函数做依赖项，让返回值从 `dict` 变成有属性的对象，编辑器的自动补全和类型检查就全回来了**

---

## 1. 先看问题：函数依赖项返回 dict 的痛苦

### 1.1 上一节的代码

```python
async def common_parameters(q: str | None = None, skip: int = 0, limit: int = 100):
    return {"q": q, "skip": skip, "limit": limit}


@app.get("/items/")
async def read_items(commons: Annotated[dict, Depends(common_parameters)]):
    # commons 是一个 dict
    q = commons["q"]       # 编辑器不知道 "q" 是合法的键
    skip = commons["skip"] # 编辑器不知道 "skip" 的类型是 int
    return ...
```

### 1.2 问题在哪

```css
你写 commons["q"]     → 编辑器：不知道这个键存不存在，无法自动补全 ✗
你写 commons["qurey"] → 编辑器：不知道你拼错了，不会报错 ✗
你写 commons["skip"]  → 编辑器：不知道值的类型是 int，不会提示 int 的方法 ✗
```

`dict` 是一个"什么都能装"的容器，编辑器无法推断出里面有什么键、值是什么类型。你失去了所有的类型检查、自动补全、重构支持

### 1.3 如果返回的是对象就不一样了

```python
commons.q       → 编辑器知道 q 是 str | None，自动补全 ✓
commons.qurey   → 编辑器画红线：这个属性不存在 ✓
commons.skip    → 编辑器知道 skip 是 int，提示 int 的所有方法 ✓
```

所以目标很明确：**让依赖项返回一个有明确属性的对象，而不是 dict**

##2. 核心概念：Python 中的 "callable"（可调用对象）

### 2.1 什么是 callable

在 Python 中，任何可以用 `()` 调用的东西都叫 **callable**：

```python
# 函数是 callable
def my_func():
    ...
my_func()                    # ✓ 可以调用

# 类也是 callable
class MyClass:
    def __init__(self):
        ...
MyClass()                    # ✓ 可以调用（创建实例）

# lambda 是 callable
my_lambda = lambda x: x + 1
my_lambda(5)                 # ✓ 可以调用

# 实现了 __call__ 方法的对象也是 callable
class Processor:
    def __call__(self, data):
        ...
p = Processor()
p("hello")                   # ✓ 可以调用
```

### 2.2 FastAPI 的依赖项只要求是 callable

上一节说"依赖项是一个函数"——这其实不够精确。准确地说：

> **FastAPI 的依赖项可以是任何 callable**——函数、类、甚至实现了 `__call__` 的对象

FastAPI 拿到你传给 `Depends()` 的东西之后，会检查它的参数签名（函数看参数列表，类看 `__init__` 的参数列表），然后用和路径操作函数一样的方式去解析这些参数

### 2.3 为什么类可以做依赖项

```python
class Cat:
    def __init__(self, name: str):
        self.name = name

fluffy = Cat(name="Mr Fluffy")     # 调用 Cat → 创建实例
```

`Cat(name="Mr Fluffy")` 就是在"调用" Cat 类。所以：

```
函数依赖项：FastAPI 调用函数 → 得到函数的返回值 → 注入给你
类依赖项：  FastAPI 调用类   → 得到类的实例     → 注入给你
```

FastAPI 看 callable 参数的方式完全一样：

- 函数依赖项 → 看函数的参数列表
- 类依赖项 → 看 `__init__` 的参数列表

## 3. 从函数依赖项到类依赖项——逐步改造

### 3.1 原来的函数依赖项

```python
async def common_parameters(q: str | None = None, skip: int = 0, limit: int = 100):
    return {"q": q, "skip": skip, "limit": limit}
```

### 3.2 改成类依赖项

```python
class CommonQueryParams:
    def __init__(self, q: str | None = None, skip: int = 0, limit: int = 100):
        self.q = q
        self.skip = skip
        self.limit = limit
```

对比一下：

```css
函数版本：
  参数：q: str | None = None, skip: int = 0, limit: int = 100
  返回值：{"q": q, "skip": skip, "limit": limit}     ← dict，编辑器不认识

类版本：
  __init__ 参数：q: str | None = None, skip: int = 0, limit: int = 100
  "返回值"（实例）：self.q = q, self.skip = skip, self.limit = limit  ← 对象，编辑器认识
```

参数完全一样！只是"包装方式"不同——函数把值装进 dict，类把值装成属性

### 3.3 在路径操作函数中使用

```python
from typing import Annotated
from fastapi import Depends, FastAPI

app = FastAPI()

fake_items_db = [{"item_name": "Foo"}, {"item_name": "Bar"}, {"item_name": "Baz"}]


class CommonQueryParams:
    def __init__(self, q: str | None = None, skip: int = 0, limit: int = 100):
        self.q = q
        self.skip = skip
        self.limit = limit


@app.get("/items/")
async def read_items(commons: Annotated[CommonQueryParams, Depends(CommonQueryParams)]):
    response = {}
    if commons.q:                    # ← 编辑器知道 commons 有 .q 属性 ✓
        response.update({"q": commons.q})
    items = fake_items_db[commons.skip : commons.skip + commons.limit]
    #                     ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
    #                     编辑器知道 .skip 和 .limit 是 int ✓
    # 这里其实就是列表的切片，只不过切片参数是通过.类属性这样来访问得到
    
    response.update({"items": items})
    return response
```

### 3.4 执行过程（和函数版本几乎一样）

```css
客户端请求：GET /items/?q=hello&skip=1&limit=2

第一步：FastAPI 发现 Depends(CommonQueryParams)

第二步：FastAPI 看 CommonQueryParams.__init__ 的参数签名：
       q: str | None = None, skip: int = 0, limit: int = 100
       → 从查询参数取值

第三步：FastAPI 调用 CommonQueryParams(q="hello", skip=1, limit=2)
       → 创建一个实例，这个实例有三个属性：.q = "hello", .skip = 1, .limit = 2

第四步：把这个实例赋给 commons 参数

第五步：执行 read_items(commons=<CommonQueryParams 实例>)
```

和函数版本的区别只在第三步：函数版本调用函数得到 dict，类版本调用类得到实例。其他完全一样

## 4. Annotated 里的两个 CommonQueryParams 分别是什么

```python
commons: Annotated[CommonQueryParams, Depends(CommonQueryParams)]
#                  ^^^^^^^^^^^^^^^^^         ^^^^^^^^^^^^^^^^^
#                  第一个                      第二个
```

这是初学时最容易困惑的地方。两个 `CommonQueryParams` 的作用完全不同：

### 4.1 第二个（`Depends(CommonQueryParams)`）——给 FastAPI 看的

```python
Depends(CommonQueryParams)
```

这个是**告诉 FastAPI "请调用 CommonQueryParams 来获取这个参数的值"**。FastAPI 会：

1. 检查 `CommonQueryParams.__init__` 的参数签名
2. 从请求里提取对应的值
3. 调用 `CommonQueryParams(q=..., skip=..., limit=...)` 创建实例
4. 把实例赋给 `commons`

**这是 FastAPI 真正用来工作的部分**

### 4.2 第一个（`Annotated[CommonQueryParams, ...]`）——给编辑器看的

```python
Annotated[CommonQueryParams, ...]
```

这个是**告诉编辑器"commons 的类型是 CommonQueryParams"**。编辑器会：

1. 知道 `commons.q` 是 `str | None`
2. 知道 `commons.skip` 是 `int`
3. 提供自动补全
4. 在你打错属性名时画红线

**这是给开发体验用的，FastAPI 本身不用它来做数据解析**

### 4.3 验证：把第一个换成 Any 也能跑

```python
from typing import Annotated, Any
from fastapi import Depends

# 把类型注解改成 Any → FastAPI 照样工作
@app.get("/items/")
async def read_items(commons: Annotated[Any, Depends(CommonQueryParams)]):
    # FastAPI 依然会调用 CommonQueryParams、创建实例、注入给 commons
    # 但是！编辑器不知道 commons 是什么类型了
    # commons.q  → 编辑器不给自动补全了 ✗
    # commons.xyz → 编辑器不会报错了 ✗
    return {"q": commons.q}
```

功能上完全一样，但编辑器的支持全没了。所以**虽然第一个类型注解不影响运行，但强烈建议写上**

### 4.4 用一张图总结

```css
commons: Annotated[CommonQueryParams, Depends(CommonQueryParams)]
                   ─────────────────  ─────────────────────────
                          │                       │
                    给编辑器看的              给 FastAPI 看的
                   （类型提示）               （实际调用）
                          │                       │
                   提供自动补全            解析参数、创建实例
                   类型检查               注入返回值
                   重构支持               OpenAPI 文档
```

## 5. 快捷写法：`Depends()` 不传参数

### 5.1 重复的问题

```python
commons: Annotated[CommonQueryParams, Depends(CommonQueryParams)]
#                  ^^^^^^^^^^^^^^^^^         ^^^^^^^^^^^^^^^^^
#                  写了一遍                    又写了一遍
```

`CommonQueryParams` 出现了两次——第一个是类型注解，第二个是传给 `Depends()`。当两者相同时，可以简化

### 5.2 快捷写法

```python
# 完整写法
commons: Annotated[CommonQueryParams, Depends(CommonQueryParams)]

# 快捷写法——Depends() 不传参数
commons: Annotated[CommonQueryParams, Depends()]
```

当 `Depends()` 没有参数时，FastAPI 会看 `Annotated` 的第一个类型参数（`CommonQueryParams`），自动把它当作依赖项来调用

### 5.3 完整的快捷写法示例

```python
from typing import Annotated
from fastapi import Depends, FastAPI

app = FastAPI()

fake_items_db = [{"item_name": "Foo"}, {"item_name": "Bar"}, {"item_name": "Baz"}]


class CommonQueryParams:
    def __init__(self, q: str | None = None, skip: int = 0, limit: int = 100):
        self.q = q
        self.skip = skip
        self.limit = limit


@app.get("/items/")
async def read_items(commons: Annotated[CommonQueryParams, Depends()]):
    #                                                      ^^^^^^^^^
    #                                             不传参数！FastAPI 自动用第一个参数， CommonQueryParams
    response = {}
    if commons.q:
        response.update({"q": commons.q})
    items = fake_items_db[commons.skip : commons.skip + commons.limit]
    response.update({"items": items})
    return response
```

### 5.4 什么时候能用快捷写法

```css
能用快捷写法的场景（类型注解和依赖项是同一个类）：
  commons: Annotated[CommonQueryParams, Depends()]     ← ✓ Depends 自动用 CommonQueryParams

不能用快捷写法的场景（类型注解和依赖项不一样）：
  result: Annotated[dict, Depends(some_function)]       ← ✗ 必须传参数
  user: Annotated[UserPublic, Depends(get_current_user)] ← ✗ 类型和依赖项不同
```

**简单来说**：只有当依赖项是一个类，并且你的类型注解恰好就是这个类时，才能省略 `Depends()` 里的参数

### 5.5 觉得绕就别用

官方文档也说了：如果你觉得这个快捷写法让代码更难读了，就不用它。用完整写法 `Depends(CommonQueryParams)` 没有任何问题。这只是一个少写几个字的语法糖，不是必须的

## 6. 函数依赖项 vs 类依赖项——什么时候用哪个

### 6.1 对比

```css
                 函数依赖项                        类依赖项
────────────────────────────────────────────────────────────────────
返回值类型       dict（或任意对象）                  类的实例
编辑器支持       dict 时差，自定义类时好              好（有属性、有类型）
代码量           少（一个函数）                      多（要定义类 + __init__）
适合场景         简单的参数组装                      复杂的参数组合，需要类型安全
快捷写法         无                                 Depends() 可以不传参数
```

### 6.2 选择建议

```css
用函数依赖项：
  - 逻辑很简单，就是组装几个参数
  - 不需要编辑器的自动补全（参数不多、记得住）
  - 返回值要做进一步处理，不需要有属性

用类依赖项：
  - 参数多，需要编辑器帮忙记住有哪些属性
  - 多个接口都用到这组参数，需要类型安全
  - 想利用 Depends() 的快捷写法

实际项目中的常见做法：
  - 简单参数组装 → 函数
  - 分页、筛选、认证 → 类
  - 数据库连接 → 函数（通常配合 yield）
```

## 7. 类依赖项的进阶用法

### 7.1 给类加方法

类不只是存参数的容器，你可以给它加上有用的方法：

```python
class Pagination:
    def __init__(self, page: int = 1, page_size: int = 20):
        self.page = page
        self.page_size = page_size
        self.skip = (page - 1) * page_size

    def paginate(self, query_result: list) -> list:
        """对查询结果进行分页"""
        return query_result[self.skip : self.skip + self.page_size]

    def response(self, items: list, total: int) -> dict:
        """生成标准的分页响应"""
        return {
            "items": items,
            "page": self.page,
            "page_size": self.page_size,
            "total": total,
            "total_pages": (total + self.page_size - 1) // self.page_size,
        }


@app.get("/items/")
async def read_items(pagination: Annotated[Pagination, Depends()]):
    all_items = [...]               # 从数据库查所有数据
    items = pagination.paginate(all_items)     # ← 调用方法分页
    return pagination.response(items, len(all_items))  # ← 调用方法生成响应
```

函数依赖项做不到这个——dict 没有自定义方法

### 7.2 类依赖项可以有 Query、Path 等标注

类的 `__init__` 参数和路径操作函数的参数遵循完全一样的规则，所以你可以用 `Query()`、`Path()`、`Header()` 等标注：

```python
from fastapi import Query, Header

class CommonQueryParams:
    def __init__(
        self,
        q: Annotated[str | None, Query(min_length=3, description="搜索关键词")] = None,
        skip: Annotated[int, Query(ge=0, description="跳过条数")] = 0,
        limit: Annotated[int, Query(ge=1, le=100, description="每页条数")] = 20,
        user_agent: Annotated[str | None, Header()] = None,
    ):
        self.q = q
        self.skip = skip
        self.limit = limit
        self.user_agent = user_agent
```

`Query(min_length=3)` 的验证会自动生效，`description` 会出现在 `/docs` 页面——和直接写在路径操作函数里完全一样

## 8. 对应到你的投标项目

```python
from typing import Annotated

from fastapi import Depends, FastAPI, HTTPException, Query, status
from pydantic import BaseModel

app = FastAPI(title="智能投标决策系统")


# ==================== 类依赖项：分页参数 ====================

class Pagination:
    """
    通用分页参数。

    用类而不是函数，这样在路径操作函数里写 pagination.page
    编辑器就能自动补全出所有属性。
    """

    def __init__(
        self,
        page: Annotated[int, Query(ge=1, description="页码，从 1 开始")] = 1,
        page_size: Annotated[int, Query(ge=1, le=100, description="每页条数")] = 20,
    ):
        self.page = page
        self.page_size = page_size
        self.skip = (page - 1) * page_size

    def paginate(self, items: list) -> list:
        """对列表做分页截取"""
        return items[self.skip : self.skip + self.page_size]

    def build_response(self, items: list, total: int) -> dict:
        """生成标准分页响应"""
        return {
            "items": items,
            "page": self.page,
            "page_size": self.page_size,
            "total": total,
            "total_pages": -(-total // self.page_size),  # 向上取整
        }


# ==================== 类依赖项：项目筛选条件 ====================

class ProjectFilter:
    """
    项目筛选条件。

    和上一节用函数返回 dict 的版本对比：
    - filters["keyword"]   → 编辑器不知道这个键是否存在 ✗
    - filters.keyword      → 编辑器知道这个属性是 str | None ✓
    """

    def __init__(
        self,
        keyword: Annotated[str | None, Query(description="搜索关键词")] = None,
        project_type: Annotated[str | None, Query(description="市政/公路/房建")] = None,
        location: Annotated[str | None, Query(description="项目所在地")] = None,
        min_price: Annotated[float | None, Query(ge=0, description="最低控制价")] = None,
        max_price: Annotated[float | None, Query(description="最高控制价")] = None,
    ):
        self.keyword = keyword
        self.project_type = project_type
        self.location = location
        self.min_price = min_price
        self.max_price = max_price

    def has_filters(self) -> bool:
        """是否有任何筛选条件"""
        return any([
            self.keyword, self.project_type, self.location,
            self.min_price is not None, self.max_price is not None,
        ])

    def match(self, project: dict) -> bool:
        """检查一个项目是否匹配所有筛选条件"""
        if self.keyword and self.keyword not in project.get("project_name", ""):
            return False
        if self.project_type and project.get("project_type") != self.project_type:
            return False
        if self.location and self.location not in project.get("location", ""):
            return False
        price = project.get("control_price", 0)
        if self.min_price is not None and price < self.min_price:
            return False
        if self.max_price is not None and price > self.max_price:
            return False
        return True


# ==================== 类依赖项：模拟用户认证 ====================

class User(BaseModel):
    username: str
    role: str = "viewer"


class CurrentUser:
    """
    用户认证依赖项。

    注意：这个类的 __init__ 参数来自查询参数（简化版）。
    后面学安全章节时会从 Header 里取 Bearer Token。
    """

    def __init__(
        self,
        token: Annotated[str | None, Query(description="认证令牌")] = None,
    ):
        if not token:
            raise HTTPException(
                status_code=status.HTTP_401_UNAUTHORIZED,
                detail="请提供认证令牌",
            )
        fake_users = {
            "admin-token": User(username="admin", role="admin"),
            "editor-token": User(username="editor", role="editor"),
        }
        user = fake_users.get(token)
        if not user:
            raise HTTPException(
                status_code=status.HTTP_401_UNAUTHORIZED,
                detail="无效的认证令牌",
            )
        self.user = user
        self.username = user.username
        self.role = user.role

    def require_role(self, *roles: str):
        """检查用户是否有指定角色"""
        if self.role not in roles:
            raise HTTPException(
                status_code=status.HTTP_403_FORBIDDEN,
                detail=f"需要 {'/'.join(roles)} 角色",
            )

# ==================== 接口 ====================

@app.get("/api/projects/", tags=["项目管理"], summary="查询项目列表")
async def list_projects(
    pagination: Annotated[Pagination, Depends()],         # 快捷写法 ✓
    filters: Annotated[ProjectFilter, Depends()],         # 快捷写法 ✓
    auth: Annotated[CurrentUser, Depends()],               # 快捷写法 ✓
):
    """
    同时使用三个类依赖项。

    和上一节函数依赖项的对比：
    - 上一节：pagination["skip"]     → 编辑器不给自动补全
    - 这一节：pagination.skip        → 编辑器自动补全 + 类型检查
    - 上一节：filters["keyword"]     → 拼错键名不会报错
    - 这一节：filters.keyword        → 拼错属性名编辑器画红线

    请求示例：
    GET /api/projects/?page=1&page_size=2&keyword=南昌&project_type=市政&token=admin-token
    """
    # 过滤
    matched = [p for p in fake_projects if filters.match(p)]

    # 分页
    page_items = pagination.paginate(matched)

    # 构建响应
    return {
        "user": auth.username,
        **pagination.build_response(page_items, len(matched)),
    }


@app.get("/api/projects/{project_id}", tags=["项目管理"], summary="查看项目详情")
async def get_project(
    project_id: str,
    auth: Annotated[CurrentUser, Depends()],
):
    """只需要认证，不需要分页和筛选"""
    for p in fake_projects:
        if p["id"] == project_id:
            return {"user": auth.username, "project": p}
    raise HTTPException(status_code=404, detail="项目不存在")


@app.delete("/api/projects/{project_id}", tags=["项目管理"], summary="删除项目")
async def delete_project(
    project_id: str,
    auth: Annotated[CurrentUser, Depends()],
):
    """需要认证 + 管理员角色"""
    auth.require_role("admin")        # ← 调用类的方法来检查权限
    return {"detail": f"项目 {project_id} 已删除", "deleted_by": auth.username}
```

**对比上一节的函数依赖项版本**：

```css
上一节（函数依赖项，返回 dict）：
  pagination["skip"]            → 编辑器不知道有没有 "skip" 这个键
  filters["keyword"]            → 拼错了不报错
  不能调用方法                   → 分页和筛选逻辑只能写在路径操作函数里

这一节（类依赖项，返回对象）：
  pagination.skip               → 编辑器自动补全 ✓
  filters.keyword               → 拼错了画红线 ✓
  pagination.paginate(items)    → 可以调用方法 ✓
  auth.require_role("admin")   → 权限检查封装在类里 ✓
```

## 10. 本节核心知识点速查

```css
概念                                做法
──────────────────────────────────────────────────────────────────────────
用类做依赖项                        把参数放在 __init__ 里，用 self.xxx 存储
FastAPI 看什么                      __init__ 的参数签名（和路径操作函数一样的解析规则）
FastAPI 做什么                      调用类 → 创建实例 → 注入给你
Annotated 里的两个位置              第一个给编辑器（类型提示），第二个给 FastAPI（实际调用）
快捷写法                            Depends() 不传参数，FastAPI 自动用类型注解里的类
什么时候用快捷写法                   类型注解和依赖项是同一个类时
给 __init__ 加 Query/Path 等标注     完全支持，和路径操作函数里的写法一样
```

**本节最核心的三句话**：

1. **类可以做依赖项，因为 Python 的类是 callable**。调用一个类就是创建它的实例。FastAPI 看 `__init__` 的参数签名，用和路径操作函数一样的方式解析参数，创建实例后注入给你

2. **类依赖项比函数依赖项多一个好处：编辑器的完整支持**。函数返回 `dict` 时编辑器不知道里面有什么键；类返回实例时编辑器知道每个属性的名字和类型，能提供自动补全、类型检查、重构支持

3. **`Depends()` 不传参数是快捷写法**：当类型注解和依赖项是同一个类时，`Annotated[MyClass, Depends()]` 等价于 `Annotated[MyClass, Depends(MyClass)]`。这只是少写几个字的语法糖，觉得绕就用完整写法
