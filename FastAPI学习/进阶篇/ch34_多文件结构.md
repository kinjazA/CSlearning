# FastAPI 学习笔记（二十八）：大型应用——把一个文件拆成一个项目

> 到目前为止，所有代码都写在一个 `main.py` 里。几十行代码时没问题，但当你的投标系统有项目管理、用户认证、投标决策、历史记录……几十个接口时，一个文件就变成了几千行的"巨石"——找代码靠 Ctrl+F，改一处怕影响另一处
> 
> 
>
> 这一节的核心就是一个东西：**APIRouter**。它让你把路径操作拆到不同文件里，最后在 `main.py` 里拼起来。类比 Flask 的 Blueprint，Vue 的路由模块化，React 的路由分组
> 
> 
>
> **你的代码从"一个文件"进化成"一个工程"**

---

## 1. 为什么要拆文件

### 1.1 单文件的痛点

```python
# main.py —— 所有东西堆在一起
from fastapi import FastAPI

app = FastAPI()

# ===== 用户相关 =====
@app.get("/users/")
async def list_users(): ...

@app.get("/users/{user_id}")
async def get_user(): ...

@app.post("/users/")
async def create_user(): ...

# ===== 项目相关 =====
@app.get("/projects/")
async def list_projects(): ...

@app.post("/projects/")
async def create_project(): ...

# ===== 投标相关 =====
@app.get("/bids/")
async def list_bids(): ...

@app.post("/bids/")
async def create_bid(): ...

# ===== 管理后台 =====
@app.get("/admin/stats")
async def admin_stats(): ...

# …… 几十个接口全在这一个文件里
```

问题很明显：

```
1. 文件太长，找代码困难
2. 多人协作时 git 冲突不断（大家都在改 main.py）
3. 不同模块的依赖项、中间件、标签混在一起
4. 无法复用——"管理后台"想给另一个项目用，得从这个大文件里剪代码
```

### 1.2 拆文件后的样子

```css
app/
├── __init__.py          ← 空文件，让 app/ 成为 Python 包
├── main.py              ← 主入口：创建 FastAPI，拼装所有 Router
├── dependencies.py      ← 公共依赖项（认证、数据库会话等）
├── routers/
│   ├── __init__.py      ← 空文件，让 routers/ 成为子包
│   ├── users.py         ← 用户相关接口
│   ├── items.py         ← 物品相关接口
│   └── projects.py      ← 项目相关接口
└── internal/
    ├── __init__.py      ← 空文件
    └── admin.py         ← 管理后台接口（可能是别的团队提供的）
```

**main.py 变得极其简洁**，只负责创建 app 和拼装 Router，不包含任何具体的业务逻辑

## 2. APIRouter——"迷你 FastAPI"

### 2.1 APIRouter 和 FastAPI 的关系

```python
from fastapi import FastAPI
from fastapi import APIRouter

app = FastAPI()         # 主应用
router = APIRouter()    # "迷你应用"

# 两者的接口用法完全一样：
@app.get("/xxx")        # 在主应用上注册路径操作
@router.get("/xxx")     # 在 Router 上注册路径操作

# 最后把 Router 拼到主应用上：
app.include_router(router)
```

APIRouter 支持 FastAPI 的所有功能：路径参数、查询参数、请求体、依赖项、标签、响应模型……用法完全一样，只是最后要把 Router "插入"到主应用里.==我理解这里的意思是，如果用API路由来写的接口，那么就要接进主应用里，用@app写的，本来就在主应用里，应该是两者都可以共存==

### 2.2 最简单的 Router 示例

```python
# app/routers/users.py

from fastapi import APIRouter

router = APIRouter()


@router.get("/users/", tags=["users"])
async def read_users():
    return [{"username": "Rick"}, {"username": "Morty"}]


@router.get("/users/me", tags=["users"])
async def read_user_me():
    return {"username": "fakecurrentuser"}


@router.get("/users/{username}", tags=["users"])
async def read_user(username: str):
    return {"username": username}
```

注意：这个文件里没有 `FastAPI()`，没有 `app`。它只创建了一个 `APIRouter` 实例，然后在上面注册路径操作

### 2.3 在 main.py 里拼装

```python
# app/main.py

from fastapi import FastAPI
from .routers import users          # 导入 users 模块，这里是相对路径，一个点表示当前文件所在的python包

app = FastAPI()
app.include_router(users.router)    # 把 users文件里的 的 Router 拼进来

# 现在 /users/、/users/me、/users/{username} 就都注册到 app 上了
```

就这么简单。`include_router` 把 Router 里的所有路径操作"复制"到主应用里，效果和直接写在 main.py 一样

## 3. 用 prefix、tags、dependencies 消除重复

### 3.1 问题：每个路径操作都重复写同样的东西

```python
# 不用 prefix 时——每个路径操作都要写 /items
@router.get("/items/", tags=["items"])
@router.get("/items/{item_id}", tags=["items"])
@router.put("/items/{item_id}", tags=["items"])

# 每个操作都要加同一个依赖项
@router.get("/items/", dependencies=[Depends(get_token_header)])
@router.get("/items/{item_id}", dependencies=[Depends(get_token_header)])
@router.put("/items/{item_id}", dependencies=[Depends(get_token_header)])
```

重复了三次 `/items`、三次 `tags=["items"]`、三次 `dependencies=[Depends(...)]`

### 3.2 解决：在 APIRouter 上统一配置

```python
# app/routers/items.py

from fastapi import APIRouter, Depends, HTTPException
from ..dependencies import get_token_header    # 相对导入

router = APIRouter(
    prefix="/items",                            # 所有路径自动加 /items 前缀
    tags=["items"],                             # 所有路径操作自动加 items 标签
    dependencies=[Depends(get_token_header)],   # 所有路径操作自动执行这个依赖项
    responses={404: {"description": "Not found"}},  # 所有路径操作的公共响应文档
)

fake_items_db = {"plumbus": {"name": "Plumbus"}, "gun": {"name": "Portal Gun"}}


@router.get("/")                # 实际路径：/items/
async def read_items():
    return fake_items_db


@router.get("/{item_id}")      # 实际路径：/items/{item_id}
async def read_item(item_id: str):
    if item_id not in fake_items_db:
        raise HTTPException(status_code=404, detail="Item not found")
    return {"name": fake_items_db[item_id]["name"], "item_id": item_id}


@router.put(
    "/{item_id}",
    tags=["custom"],            # 追加标签 → 最终标签是 ["items", "custom"]
    responses={403: {"description": "Operation forbidden"}},  # 追加响应
)
async def update_item(item_id: str):
    if item_id != "plumbus":
        raise HTTPException(status_code=403, detail="You can only update the item: plumbus")
    return {"item_id": item_id, "name": "The great Plumbus"}
```

### 3.3 APIRouter 参数的完整说明

```css
参数                效果                                              合并规则
─────────────────────────────────────────────────────────────────────────────────
prefix              所有路径加前缀                                     直接拼接
                    prefix="/items" + @router.get("/") → /items/
                    prefix="/items" + @router.get("/{id}") → /items/{id}

tags                所有路径操作加标签                                  追加（不覆盖）
                    Router tags=["items"] + 某接口 tags=["custom"]
                    → 该接口最终 tags=["items", "custom"]

dependencies        所有路径操作加依赖项                                追加
                    Router + 接口自己的 + 参数里的 → 全部执行
                    执行顺序：Router 的 → 装饰器的 → 参数的

responses           所有路径操作加响应文档                              合并
                    Router 的 + 接口自己的 → 合在一起显示在文档里
```

### 3.4 关键细节：prefix 不能有末尾的 `/`

```python
# ✅ 正确
router = APIRouter(prefix="/items")

# ❌ 错误——会导致路径变成 /items//
router = APIRouter(prefix="/items/")
```

而路径操作的路径必须以 `/` 开头：

```python
@router.get("/")           # ✅ → /items/
@router.get("/{item_id}")  # ✅ → /items/{item_id}

@router.get("")            # ❌ 不要这样写
```

## 4. 公共依赖项放在 dependencies.py

### 4.1 为什么单独放

多个 Router 可能需要同一个依赖项（比如认证）。如果写在某个 Router 文件里，其他 Router 导入时就形成了模块间的耦合。放在独立的 `dependencies.py` 里，所有 Router 都从同一个地方导入

### 4.2 示例

```python
# app/dependencies.py

from typing import Annotated
from fastapi import Header, HTTPException


async def get_token_header(x_token: Annotated[str, Header()]):
    """验证请求头里的 X-Token"""
    if x_token != "fake-super-secret-token":
        raise HTTPException(status_code=400, detail="X-Token header invalid")


async def get_query_token(token: str):
    """验证查询参数里的 token"""
    if token != "jessica":
        raise HTTPException(status_code=400, detail="No Jessica token provided")
```

这两个依赖项在不同的地方使用：

```
get_query_token  → 全局依赖项（加在 FastAPI 上，所有接口都要验证）
get_token_header → 模块级依赖项（加在某些 Router 上，只有这些模块的接口要验证）
```

## 5. Python 相对导入——`.` 和 `..` 是什么意思

这是很多人卡住的地方。拆文件之后，模块之间要互相导入。理解相对导入的规则非常重要

### 5.1 文件结构和模块路径的对应

```css
文件系统路径                          Python 模块路径
──────────────────────────────────────────────────────
app/                                 app（包）
app/__init__.py                      app（包的标识文件）
app/main.py                         app.main
app/dependencies.py                  app.dependencies
app/routers/                         app.routers（子包）
app/routers/__init__.py              app.routers（子包的标识文件）
app/routers/items.py                 app.routers.items
app/routers/users.py                 app.routers.users
app/internal/                        app.internal（子包）
app/internal/admin.py                app.internal.admin
```

`__init__.py` 的作用就是告诉 Python "这个目录是一个包"。即使文件是空的也必须存在

### 5.2 相对导入的规则

```python
# 在 app/routers/items.py 里：

from .dependencies import get_token_header
# .  = 当前包（app/routers/）
# 找 app/routers/dependencies.py → 不存在！这是错误的

from ..dependencies import get_token_header
# .. = 父包（app/）
# 找 app/dependencies.py → 存在！这是正确的 ✓

from ...dependencies import get_token_header
# ... = 父包的父包（app/ 的上一层）
# 我们的项目没有这一层 → 报错！
```

记忆方法：

```css
.   = 当前目录（包）
..  = 上一级目录（父包）
... = 上两级目录（父包的父包）

每多一个点，就往上走一层。
```

### 5.3 相对导入 vs 绝对导入

```python
# 相对导入（推荐在包内部使用）
from .routers import items, users
from ..dependencies import get_token_header

# 绝对导入（也可以用，但要写完整路径）
from app.routers import items, users
from app.dependencies import get_token_header
```

两种都可以。相对导入更简洁，而且如果以后把 `app` 重命名为 `bidding_system`，相对导入不用改，绝对导入全部要改

### 5.4 避免名称冲突

```python
# ❌ 两个模块都有 router 变量，后面的覆盖前面的
from .routers.items import router
from .routers.users import router    # 覆盖了！

# ✅ 导入模块，通过模块名访问
from .routers import items, users

app.include_router(items.router)     # items.router
app.include_router(users.router)     # users.router
```

导入模块而不是直接导入变量，用模块名作为命名空间，避免变量名冲突

## 6. 主文件 main.py——把所有东西拼起来

### 6.1 完整的 main.py

```python
# app/main.py

from fastapi import Depends, FastAPI

from .dependencies import get_query_token, get_token_header
from .internal import admin
from .routers import items, users

app = FastAPI(dependencies=[Depends(get_query_token)])  # 全局依赖项


app.include_router(users.router)    # 直接包含，不加额外配置
app.include_router(items.router)    # items.py 里已经配了 prefix 和 tags

# admin 的 Router 没有配 prefix 和 tags（因为是共享模块，不能改）
# 在 include_router 时从外部加上
app.include_router(
    admin.router,
    prefix="/admin",
    tags=["admin"],
    dependencies=[Depends(get_token_header)],
    responses={418: {"description": "I'm a teapot"}},
)


@app.get("/")
async def root():
    return {"message": "Hello Bigger Applications!"}
```

### 6.2 三种不同的包含方式

```python
# 方式一：Router 自带配置，直接包含
# users.py 里的 @router.get("/users/") 保持原样
app.include_router(users.router)

# 方式二：Router 自带配置（prefix、tags、dependencies）
# items.py 里配了 prefix="/items"、tags=["items"]
app.include_router(items.router)

# 方式三：Router 没有配置，包含时从外部加
# admin.py 里的 Router 是"裸的"，没有 prefix/tags
# 因为它是共享模块，不能为某个项目硬编码配置
app.include_router(
    admin.router,
    prefix="/admin",         # 从外部加前缀
    tags=["admin"],          # 从外部加标签
    dependencies=[...],      # 从外部加依赖项
)
```

方式三非常关键——它让你在**不修改原始 Router 代码**的情况下，为它加上前缀、标签和依赖项。这意味着同一个 Router 可以在不同项目中复用，每个项目给它配不同的前缀和认证方式

### 6.3 全局依赖项

```python
app = FastAPI(dependencies=[Depends(get_query_token)])
```

这个依赖项会应用到**所有**路径操作——包括 main.py 里直接定义的，也包括通过 `include_router` 添加的

### 6.4 依赖项的执行顺序

```css
一个请求到达 /items/{item_id} 时，依赖项的执行顺序：

1. 全局依赖项（FastAPI 上的）
   → get_query_token（检查查询参数里的 token）

2. Router 级依赖项（APIRouter 上的）
   → get_token_header（检查请求头里的 X-Token）

3. 路径操作级依赖项（装饰器上的，如果有的话）

4. 参数级依赖项（函数参数里用 Depends 声明的）

任何一层验证失败，请求就被拦截，后面的不执行。
```

## 7. admin.py 的设计——共享模块的思路

### 7.1 代码

```python
# app/internal/admin.py

from fastapi import APIRouter

router = APIRouter()


@router.post("/")
async def update_admin():
    return {"message": "Admin getting schwifty"}
```

注意这个 Router **没有任何配置**——没有 prefix、没有 tags、没有 dependencies

### 7.2 为什么这样设计

```
这个 admin 模块是组织内多个项目共享的。

项目 A 可能想把它挂在 /admin，用 token 认证
项目 B 可能想把它挂在 /management，用 OAuth2 认证
项目 C 可能想把它挂在 /api/v1/admin，用 API Key 认证

如果在 admin.py 里硬编码了 prefix="/admin"，
那项目 B 和 C 就不得不改 admin.py 的代码——
但它是共享的，改了会影响项目 A。

所以 admin.py 保持"裸"的状态，
由每个项目在 include_router 时自己配置。
```

这是一个重要的设计原则：**可复用的 Router 不应该硬编码部署相关的配置**

## 8. include_router 时的参数合并规则

### 8.1 tags 和 responses 是追加的

```python
# Router 定义时：
router = APIRouter(tags=["items"], responses={404: {...}})

# 某个路径操作：
@router.put("/{id}", tags=["custom"], responses={403: {...}})

# 最终效果：
# tags = ["items", "custom"]        ← 合并
# responses = {404: {...}, 403: {...}}  ← 合并
```

### 8.2 include_router 的参数也是追加的

```python
# admin.py 里的 Router 没有配置

# main.py 里 include 时加了配置
app.include_router(
    admin.router,
    prefix="/admin",
    tags=["admin"],
    dependencies=[Depends(get_token_header)],
)

# 如果 admin.py 里某个路径操作自己也加了 tags=["special"]
# 最终 tags = ["admin", "special"]
```

### 8.3 prefix 是拼接的

```python
# Router 定义时：
router = APIRouter(prefix="/items")

# include 时又加了 prefix：
app.include_router(router, prefix="/api/v1")

# 最终路径：/api/v1/items/...
# prefix 是字符串拼接，不是覆盖
```

## 9. 同一个 Router 多次包含——API 版本控制

```python
# 同一个 Router 可以用不同的前缀包含两次
app.include_router(items.router, prefix="/api/v1")
app.include_router(items.router, prefix="/api/latest")

# 现在这两个路径都指向同一套代码：
# /api/v1/items/      → read_items()
# /api/latest/items/   → read_items()
```

这在 API 版本迁移时很有用——新版本 `/api/latest` 和旧版本 `/api/v1` 共用同一套代码，直到旧版本被废弃

## 10. Router 嵌套——Router 里包含 Router

```python
# 不只是 app 可以 include_router，Router 之间也可以嵌套

# 顶层 Router
api_router = APIRouter(prefix="/api")

# 子 Router
api_router.include_router(users.router)     # /api/users/...
api_router.include_router(items.router)     # /api/items/...

# 最后只需要把顶层 Router 包含到 app
app.include_router(api_router)
```

嵌套的效果就是 prefix 拼接。这种方式可以更灵活地组织路由层级

## 11. 最终的路由全景图

让我们把官方示例的所有路径操作列出来，看看最终效果：

```
路径                     方法     来源文件              tags          依赖项
────────────────────────────────────────────────────────────────────────────────────
/                        GET      main.py               -             get_query_token
/users/                  GET      routers/users.py      [users]       get_query_token
/users/me                GET      routers/users.py      [users]       get_query_token
/users/{username}        GET      routers/users.py      [users]       get_query_token
/items/                  GET      routers/items.py      [items]       get_query_token + get_token_header
/items/{item_id}         GET      routers/items.py      [items]       get_query_token + get_token_header
/items/{item_id}         PUT      routers/items.py      [items,custom] get_query_token + get_token_header
/admin/                  POST     internal/admin.py     [admin]       get_query_token + get_token_header
```

注意依赖项的叠加：

```
所有路径都有 get_query_token（全局依赖项）
/items/* 的路径还有 get_token_header（Router 级依赖项）
/admin/* 的路径也有 get_token_header（include_router 时加的）
```

### 12.2 各文件代码

**app/database.py——数据库连接**

```python
from sqlmodel import SQLModel, create_engine, Session

sqlite_url = "sqlite:///bidding.db"
engine = create_engine(sqlite_url, connect_args={"check_same_thread": False})


def create_db_and_tables():
    SQLModel.metadata.create_all(engine)


def get_session():
    with Session(engine) as session:
        yield session
```

**app/dependencies.py——公共依赖项**

```python
from typing import Annotated
from fastapi import Depends, Header, HTTPException
from sqlmodel import Session

from .database import get_session

# 数据库会话依赖项（几乎每个接口都要用）
SessionDep = Annotated[Session, Depends(get_session)]


# 认证依赖项（后面学安全认证时会替换成真正的 JWT 验证）
async def get_current_user(x_token: Annotated[str, Header()]):
    if x_token != "fake-super-secret-token":
        raise HTTPException(status_code=401, detail="认证失败")
    return {"username": "current_user"}
```

**app/models.py——所有模型**

```python
from sqlmodel import SQLModel, Field
from datetime import datetime


class ProjectBase(SQLModel):
    project_name: str = Field(min_length=1, max_length=200, index=True)
    location: str = Field(index=True)
    control_price: float = Field(gt=0)
    project_type: str = Field(index=True)


class Project(ProjectBase, table=True):
```

```python
from typing import Annotated
from fastapi import APIRouter, HTTPException, Query

from ..dependencies import SessionDep            # .. 回到 app/
from ..models import Project, ProjectCreate, ProjectPublic, ProjectUpdate
from sqlmodel import select

router = APIRouter(
    prefix="/projects",
    tags=["项目管理"],
    responses={404: {"description": "项目不存在"}},
)


@router.post("/", response_model=ProjectPublic, status_code=201)
def create_project(project: ProjectCreate, session: SessionDep):
    db_project = Project.model_validate(project)
    session.add(db_project)
    session.commit()
    session.refresh(db_project)
    return db_project


@router.get("/", response_model=list[ProjectPublic])
def list_projects(
    session: SessionDep,
    offset: int = 0,
    limit: Annotated[int, Query(le=100)] = 20,
):
    projects = session.exec(select(Project).offset(offset).limit(limit)).all()
    return projects


@router.get("/{project_id}", response_model=ProjectPublic)
def get_project(project_id: int, session: SessionDep):
    project = session.get(Project, project_id)
    if not project:
        raise HTTPException(status_code=404, detail="项目不存在")
    return project


@router.patch("/{project_id}", response_model=ProjectPublic)
def update_project(project_id: int, project: ProjectUpdate, session: SessionDep):
    db_project = session.get(Project, project_id)
    if not db_project:
        raise HTTPException(status_code=404, detail="项目不存在")
    project_data = project.model_dump(exclude_unset=True)
    if not project_data:
        raise HTTPException(status_code=400, detail="请至少提供一个要更新的字段")
    db_project.sqlmodel_update(project_data)
    session.add(db_project)
    session.commit()
    session.refresh(db_project)
    return db_project


@router.delete("/{project_id}")
def delete_project(project_id: int, session: SessionDep):
    project = session.get(Project, project_id)
    if not project:
        raise HTTPException(status_code=404, detail="项目不存在")
    session.delete(project)
    session.commit()
    return {"ok": True}
```

## 13. 本节核心知识点速查

```
概念                                  做法
──────────────────────────────────────────────────────────────────────────
创建 Router                           router = APIRouter()
加前缀                                APIRouter(prefix="/items")
加标签                                APIRouter(tags=["items"])
加依赖项                              APIRouter(dependencies=[Depends(...)])
包含 Router                           app.include_router(router)
包含时加配置                           app.include_router(router, prefix=..., tags=..., dependencies=...)
相对导入父包                           from ..dependencies import xxx
相对导入同包                           from .models import xxx
避免命名冲突                           导入模块而非变量：from .routers import items, users
Router 嵌套                           router.include_router(other_router)
同一 Router 多次包含                   app.include_router(router, prefix="/v1")
                                      app.include_router(router, prefix="/v2")
全局依赖项                             FastAPI(dependencies=[Depends(...)])
```

**本节最核心的三句话**：

1. **APIRouter 是"迷你 FastAPI"**——用法和 FastAPI 完全一样（`@router.get`、`@router.post`），但它不能独立运行，需要通过 `app.include_router()` 拼到主应用上。这让你可以把不同业务模块拆到不同文件里，各自独立开发

2. **prefix、tags、dependencies 可以在两个地方配置**——一是在 `APIRouter()` 创建时，二是在 `app.include_router()` 包含时。两处的配置会合并（不是覆盖）。共享模块建议不在 Router 里硬编码配置，让使用方在 `include_router` 时自行决定

3. **main.py 应该只做"拼装"**——创建 FastAPI 实例、注册中间件、包含各个 Router，不写具体的业务逻辑。业务逻辑放在各自的 Router 模块里，公共依赖项放在 `dependencies.py` 里，数据库配置放在 `database.py` 里。这样 main.py 保持极度简洁，整个项目结构一目了然
