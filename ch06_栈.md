# 第六章：栈（Stack）

> **核心一句话**：栈是一种 **后进先出（LIFO）** 的线性数据结构，就像叠盘子——最后放上去的盘子，最先被拿走。
>
## 6.1 栈的基础

### 6.1.1 栈的基本概念

#### 什么是栈？

**栈（Stack）** 是一种受限的线性表，只允许在**一端**（称为 **栈顶 top**）进行插入和删除操作

可以把栈想象成：

- 一摞叠起来的盘子：只能从最上面放，只能从最上面拿
- 浏览器的"后退"按钮：最近访问的页面最先被返回
- 撤销操作（Ctrl+Z）：最近的操作最先被撤销

#### 栈的核心特点：LIFO

Last In, First Out（后进先出）

```css
入栈顺序：A → B → C → D
出栈顺序：D → C → B → A    ← 最后进去的 D 最先出来
```

图示：

```
    ┌───┐
    │ D │  ← 栈顶（top）：最后进来，最先出去
    ├───┤
    │ C │
    ├───┤
    │ B │
    ├───┤
    │ A │  ← 栈底（bottom）：最先进来，最后出去
    └───┘
```

#### 关键术语

| 术语 | 含义 |
|------|------|
| **栈顶（top）** | 栈中允许操作的那一端，即最后一个入栈的元素所在位置 |
| **栈底（bottom）** | 栈的另一端，不允许直接操作 |
| **空栈** | 栈中没有任何元素 |
| **栈满** | 仅针对固定大小的数组实现，栈中元素达到上限 |

#### 栈的抽象数据类型（ADT）

> **补充知识：什么是 ADT？**
>
> ADT（Abstract Data Type，抽象数据类型）就是"只定义做什么，不管怎么做"。比如我们说"栈能 push、能 pop"，但不关心它底层用数组还是链表

栈的 ADT 定义：

```css
ADT Stack:
    数据：零个或多个元素的有序集合
    操作：
        push(x)    — 将元素 x 压入栈顶
        pop()      — 移除并返回栈顶元素
        peek()     — 返回栈顶元素（不移除）
        is_empty() — 判断栈是否为空
        size()     — 返回栈中元素个数
```

### 6.1.2 栈的常用操作详解

下面先用一个简单的表格概览，再逐一讲解：

| 操作 | 作用 | 时间复杂度 | Python list 对应 |
|------|------|-----------|-----------------|
| `push(x)` | 入栈：将 x 放到栈顶 | $O(1)$ | `list.append(x)` |
| `pop()` | 出栈：移除并返回栈顶元素 | $O(1)$ | `list.pop()` |
| `peek()` / `top()` | 查看栈顶元素（不移除） | $O(1)$     | `list[-1]` |
| `is_empty()` | 判断栈是否为空 | $O(1)$ | `len(list) == 0` |
| `size()` | 返回栈的元素个数 | $O(1)$ | `len(list)` |

#### 操作图解

```css
初始：空栈 []

push(10):  [10]          ← 10 入栈
push(20):  [10, 20]      ← 20 入栈
push(30):  [10, 20, 30]  ← 30 入栈
peek():    返回 30        ← 只看不拿
pop():     返回 30, 栈变为 [10, 20]  ← 拿走栈顶
pop():     返回 20, 栈变为 [10]
is_empty(): False
size():    1
pop():     返回 10, 栈变为 []
is_empty(): True
```

#### Python 语法复习：list 的相关操作

```python
# ===== list 基础操作 =====

my_list = []           # 创建空列表

# append()：在列表末尾添加元素（相当于 push）
my_list.append(10)     # [10]
my_list.append(20)     # [10, 20]
my_list.append(30)     # [10, 20, 30]

# pop()：移除并返回最后一个元素（相当于栈的 pop）
last = my_list.pop()   # last = 30, my_list = [10, 20]

# 负数索引：从后往前数
# my_list[-1] 表示最后一个元素（相当于 peek）
top = my_list[-1]      # top = 20，列表不变

# len()：获取列表长度
length = len(my_list)  # length = 2

# 布尔判断：空列表为 False，非空为 True
if not my_list:        # 如果列表为空
    print("空的")
else:
    print("不是空的")  # 会执行这一行
```

#### 异常处理

> **补充知识：什么是异常处理？**
>
> 在 Python 中，当程序遇到错误时会"抛出异常"。用 `try...except` 来捕获异常，避免程序崩溃。也可以用 `raise` 主动抛出异常来提示调用者

```python
# ===== 异常处理基础语法 =====

# 1. 捕获异常
try:
    result = 10 / 0           # 这行会出错（除以零）
except ZeroDivisionError:     # 捕获特定异常
    print("不能除以零！")

# 2. 主动抛出异常
def pop_from_stack(stack):
    if len(stack) == 0:
        # raise 用于主动抛出异常
        # IndexError 是 Python 内置的异常类型，表示索引越界
        raise IndexError("栈为空，无法 pop")
    return stack.pop()

# 3. 在栈中的应用
stack = []
try:
    pop_from_stack(stack)    # 空栈 pop，会抛出异常
except IndexError as e:      # as e 可以获取异常信息
    print(f"出错了：{e}")    # 输出：出错了：栈为空，无法 pop
```

## 6.2 栈的实现

### 6.2.1 基于数组的实现（顺序栈）

#### 原理

- 使用 Python 的 `list` 作为底层存储
- **栈顶在列表末尾**（这样 push 和 pop 都是 \(O(1)\)）
- Python 的 list 会自动扩容，不用手动管理大小

```css
list 内部：  [  A  |  B  |  C  |  D  |    |    ]
              0     1     2     3
                                ↑
                             栈顶（top）= 列表末尾
```

#### 完整实现

```python
class Stack:
    """
    基于 Python list 实现的栈（顺序栈）
    
    用法示例：
        s = Stack()
        s.push(10)
        s.push(20)
        print(s.peek())   # 20
        print(s.pop())    # 20
        print(s.size())   # 1
    """
    
    def __init__(self):
        """
        构造方法：初始化一个空栈
        
        补充知识：
        - __init__ 是 Python 类的构造方法，创建对象时自动调用
        - self 代表当前对象本身
        - self._items 是实例变量，用下划线开头表示"内部使用，外部别直接访问"
        """
        self._items = []    # 用一个空列表来存储栈中的元素
    
    def push(self, item):
        """
        入栈：将元素压入栈顶
        
        参数：
            item: 要入栈的元素（可以是任意类型）
        
        时间复杂度：O(1)（均摊，因为 list 偶尔需要扩容）
        """
        self._items.append(item)    # append 在列表末尾添加元素
    
    def pop(self):
        """
        出栈：移除并返回栈顶元素
        
        返回值：
            栈顶元素
        
        异常：
            IndexError: 如果栈为空
        
        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("pop from empty stack")  # 空栈不能 pop
        return self._items.pop()    # pop() 移除并返回列表最后一个元素
    
    def peek(self):
        """
        查看栈顶元素（不移除）
        
        返回值：
            栈顶元素
        
        异常：
            IndexError: 如果栈为空
        
        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("peek from empty stack")
        return self._items[-1]      # -1 索引获取列表最后一个元素
    
    def is_empty(self):
        """
        判断栈是否为空
        
        返回值：
            bool: True 表示空栈，False 表示非空
        
        时间复杂度：O(1)
        """
        return len(self._items) == 0
    
    def size(self):
        """
        获取栈的元素个数
        
        返回值：
            int: 栈中元素的数量
        
        时间复杂度：O(1)
        """
        return len(self._items)
    
    def __str__(self):
        """
        补充知识：
        - __str__ 是特殊方法，当你用 print() 打印对象时会自动调用
        - 返回一个字符串，表示对象的"可读形式"
        """
        return f"Stack(top -> {list(reversed(self._items))})"
    
    def __repr__(self):
        """
        补充知识：
        - __repr__ 是特殊方法，在交互式环境中直接输入变量名时显示
        - 通常返回"能重新创建这个对象"的字符串
        """
        return f"Stack({self._items})"


# ===== 使用示例 =====
if __name__ == "__main__":
    """
    补充知识：
    if __name__ == "__main__": 的意思是：
    "只有直接运行这个文件时才执行下面的代码，
     如果是被别的文件 import 进去的，就不执行。"
    """
    
    s = Stack()
    
    # 入栈
    print("--- 入栈操作 ---")
    s.push(10)
    print(f"push(10): {s}")    # Stack(top -> [10])
    s.push(20)
    print(f"push(20): {s}")    # Stack(top -> [20, 10])
    s.push(30)
    print(f"push(30): {s}")    # Stack(top -> [30, 20, 10])
    
    # 查看栈顶
    print(f"\npeek(): {s.peek()}")   # 30
    print(f"size(): {s.size()}")     # 3
    
    # 出栈
    print("\n--- 出栈操作 ---")
    print(f"pop(): {s.pop()}")       # 30
    print(f"pop(): {s.pop()}")       # 20
    print(f"当前栈: {s}")            # Stack(top -> [10])
    print(f"is_empty(): {s.is_empty()}")  # False
    
    print(f"pop(): {s.pop()}")       # 10
    print(f"is_empty(): {s.is_empty()}")  # True
    
    # 异常处理演示
    print("\n--- 异常处理演示 ---")
    try:
        s.pop()                      # 空栈 pop 会报错
    except IndexError as e:
        print(f"捕获异常：{e}")      # 捕获异常：pop from empty stack
```

**运行输出：**

```
--- 入栈操作 ---
push(10): Stack(top -> [10])
push(20): Stack(top -> [20, 10])
push(30): Stack(top -> [30, 20, 10])

peek(): 30
size(): 3

--- 出栈操作 ---
pop(): 30
pop(): 20
当前栈: Stack(top -> [10])
is_empty(): False
pop(): 10
is_empty(): True

--- 异常处理演示 ---
捕获异常：pop from empty stack
```

### 6.2.2 基于链表的实现（链式栈）

#### 原理

- 使用**单链表**作为底层存储
- **链表头节点作为栈顶**（头部插入/删除都是 \(O(1)\)）
- 不需要考虑容量限制，需要多少就分配多少

```css
push/pop 都在这一端操作
         ↓
栈顶 → [ D ] → [ C ] → [ B ] → [ A ] → None
        head                              栈底
```

#### Python 语法复习：类和链表节点

```python
# ===== 补充知识：链表节点 =====

class Node:
    """
    链表节点类
    
    每个节点包含两部分：
    1. data：存储的数据
    2. next：指向下一个节点的"指针"（在 Python 中就是引用）
    
    图示：
        ┌──────┬──────┐
        │ data │ next │ ──→ 指向下一个节点（或 None）
        └──────┴──────┘
    """
    def __init__(self, data):
        self.data = data    # 节点存储的数据
        self.next = None    # 指向下一个节点，初始为 None
```

#### 完整实现

```python
class Node:
    """链表节点"""
    def __init__(self, data):
        self.data = data    # 节点存储的数据
        self.next = None    # 指向下一个节点的引用


class LinkedStack:
    """
    基于单链表实现的栈（链式栈）
    
    核心思路：
    - 用链表的头节点（head）作为栈顶，这个头结点就是---→符号最左边的那个节点
    - push = 在头部插入新节点
    - pop  = 删除头部节点
    
    用法示例：
        s = LinkedStack()
        s.push(10)
        s.push(20)
        print(s.peek())   # 20
        print(s.pop())    # 20
    """
    
    def __init__(self):
        """初始化空栈"""
        self._top = None    # 栈顶指针，初始为 None（空栈）
        self._size = 0      # 栈的元素个数
    
    def push(self, item):
        """
        入栈：在链表头部插入新节点
        
        过程图解（push(30)）：
        
        之前：  top → [20] → [10] → None
        
        步骤1：创建新节点 [30]
        步骤2：新节点的 next 指向当前栈顶  [30] → [20] → [10] → None
        步骤3：更新 top 指向新节点   top → [30] → [20] → [10] → None
        
        时间复杂度：O(1)
        """
        new_node = Node(item)        # 步骤1：创建新节点
        new_node.next = self._top    # 步骤2：新节点指向原栈顶
        self._top = new_node         # 步骤3：更新栈顶为新节点
        self._size += 1              # 元素计数 +1
    
    def pop(self):
        """
        出栈：删除并返回链表头部节点的数据
        
        过程图解（pop()）：
        
        之前：  top → [30] → [20] → [10] → None
        
        步骤1：保存栈顶数据 data = 30
        步骤2：top 指向下一个节点   top → [20] → [10] → None
        
        （被删除的 [30] 节点没有引用了，Python 会自动回收内存）
        
        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("pop from empty stack")
        data = self._top.data        # 步骤1：保存栈顶数据
        self._top = self._top.next   # 步骤2：栈顶指向下一个节点
        self._size -= 1              # 元素计数 -1
        return data
    
    def peek(self):
        """
        查看栈顶元素（不移除）
        
        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("peek from empty stack")
        return self._top.data
    
    def is_empty(self):
        """
        判断栈是否为空
        
        时间复杂度：O(1)
        """
        return self._top is None
        # 补充知识：
        # "is" 用于判断两个变量是否是同一个对象
        # "==" 用于判断两个变量的值是否相等
        # 判断是否为 None 时，推荐用 "is None" 而不是 "== None"
    
    def size(self):
        """
        获取栈的元素个数
        
        时间复杂度：O(1)（因为我们用 _size 变量记录了）
        """
        return self._size
    
    def __str__(self):
        """打印栈的内容，从栈顶到栈底"""
        elements = []
        current = self._top          # 从栈顶开始遍历
        while current is not None:   # 当还没到链表末尾
            elements.append(str(current.data))
            current = current.next   # 移动到下一个节点
        return "LinkedStack(top -> [" + " → ".join(elements) + "])"


# ===== 使用示例 =====
if __name__ == "__main__":
    s = LinkedStack()
    
    print("--- 入栈操作 ---")
    for val in [10, 20, 30]:
        s.push(val)
        print(f"push({val}): {s}")
    
    print(f"\npeek(): {s.peek()}")
    print(f"size(): {s.size()}")
    
    print("\n--- 出栈操作 ---")
    while not s.is_empty():
        print(f"pop(): {s.pop()}, 剩余: {s}")
    
    print(f"\nis_empty(): {s.is_empty()}")
```

**运行输出：**

```
--- 入栈操作 ---
push(10): LinkedStack(top -> [10])
push(20): LinkedStack(top -> [20 → 10])
push(30): LinkedStack(top -> [30 → 20 → 10])

peek(): 30
size(): 3

--- 出栈操作 ---
pop(): 30, 剩余: LinkedStack(top -> [20 → 10])
pop(): 20, 剩余: LinkedStack(top -> [10])
pop(): 10, 剩余: LinkedStack(top -> [])

is_empty(): True
```

## 6.3 两种实现对比

### 6.3.1 时间复杂度对比

| 操作 | 数组实现（顺序栈） | 链表实现（链式栈） |
|------|-------------------|-------------------|
| `push` | $O(1)$均摊 | $O(1)$ |
| `pop       | $O(1)$             | $O(1)$ |
| `peek` | $O(1)$ | $O(1)$             |
| `is_empty` | $O(1)$ | $O(1)$ |
| `size` | $O(1)$ | $O(1)$ |

> **什么是"均摊 \(O(1)\)"？**
>
> Python 的 list 在 append 时，如果底层数组满了，会申请一块更大的内存（通常是原来的 1.125 倍），然后把所有元素复制过去。这次操作是 \(O(n)\)。但这种情况很少发生，平均到每次 append 操作上还是 \(O(1)\)，所以叫"均摊 \(O(1)\)"

### 6.3.2 空间与性能对比

| 对比维度 | 数组实现 | 链表实现 |
|---------|---------|---------|
| **内存分配** | 连续内存，缓存友好 | 分散内存，每个节点单独分配 |
| **额外空间** | 可能有未使用的预留空间 | 每个节点多一个 `next` 指针 |
| **扩容** | 偶尔需要扩容（自动处理） | 不需要，随用随分配 |
| **实现难度** | 简单 | 稍复杂（需要理解指针/引用） |
| **缓存性能** | 好（连续内存，CPU 缓存命中率高） | 差（内存分散） |

### 6.3.3 使用场景分析

```css
选择数组实现（推荐默认选择）：
├── 高频 push/pop 操作
├── 对性能要求高
├── 数据量大致可预估
└── Python 中直接用 list 即可

选择链表实现：
├── 需要频繁创建/销毁栈
├── 数据量完全不可预估
├── 学习链表和指针操作
└── 面试中要求手写链表实现
```

**实际建议**：在 Python 中，99% 的情况下直接用 `list` 就够了。链式栈主要用于面试和学习目的

## 6.4 栈的典型应用（重点）⭐

### 6.4.1 括号匹配问题

**问题描述**：给定一个只包含 `()`、`{}`、`[]` 的字符串，判断括号是否有效匹配。

**核心思路**：

1. 遇到**左括号**：入栈
2. 遇到**右括号**：检查栈顶是否是对应的左括号
   - 匹配 → 弹出栈顶，继续
   - 不匹配 → 无效
3. 遍历完成后，栈为空 → 有效

**过程图解**：

```css
输入: "{[()]}"

字符  操作          栈状态
 {    左括号，入栈   ['{']
 [    左括号，入栈   ['{', '[']
 (    左括号，入栈   ['{', '[', '(']
 )    右括号，匹配(  ['{', '[']        ← '(' 被弹出
 ]    右括号，匹配[  ['{']             ← '[' 被弹出
 }    右括号，匹配{  []                ← '{' 被弹出

结束，栈为空 → 有效 ✓
```

**完整实现**：

```python
def is_valid_parentheses(s: str) -> bool:
    """
    判断括号字符串是否有效
    
    参数：
        s: 只包含 '()', '{}', '[]' 的字符串
    
    返回值：
        bool: True 表示有效匹配，False 表示无效
    
    补充知识：
    - "s: str" 是类型注解，告诉阅读者 s 应该是字符串类型
    - "-> bool" 表示函数返回布尔值
    - 类型注解不会影响运行，只是提高可读性
    """
    
    stack = []    # 用列表模拟栈
    
    # 定义括号的映射关系：右括号 → 对应的左括号
    # 补充知识：这是一个字典（dict），用 {key: value} 创建
    mapping = {
        ')': '(',
        ']': '[',
        '}': '{'
    }
    
    for char in s:                      # 遍历字符串中的每个字符
        if char in '({[':               # 如果是左括号
            stack.append(char)          # 入栈
        elif char in ')}]':             # 如果是右括号
            # 两种失败情况：
            # 1. 栈为空（没有左括号可以匹配）
            # 2. 栈顶的左括号和当前右括号不匹配
            if not stack or stack[-1] != mapping[char]:
                return False
            stack.pop()                 # 匹配成功，弹出栈顶的左括号
    
    # 遍历完所有字符后，栈应该为空（所有左括号都找到了匹配）
    return len(stack) == 0


# ===== 测试 =====
test_cases = [
    ("()",       True),
    ("()[]{}",   True),
    ("(]",       False),
    ("([)]",     False),     # 交叉括号，无效
    ("{[()]}",   True),      # 嵌套括号，有效
    ("",         True),      # 空字符串，有效
    ("(((",      False),     # 只有左括号
    (")))",      False),     # 只有右括号
]

print("=== 括号匹配测试 ===")

    status = "✓" if result == expected else "✗"
    print(f"  {status} is_valid('{test_str}') = {result}")
```

### 6.4.2 表达式求值

#### 中缀表达式 → 后缀表达式（逆波兰表达式）

> **什么是中缀、后缀表达式？**
>
> - **中缀表达式**：运算符在操作数**中间**，就是我们平时写的方式，如 `3 + 4 * 2`
> - **后缀表达式**：运算符在操作数**后面**，也叫逆波兰表达式（RPN），如 `3 4 2 * +`
> - 后缀表达式的优点：**不需要括号，不需要考虑优先级**，用栈可以轻松计算

**中缀转后缀的算法（调度场算法 / Shunting Yard）**：

```
规则：
1. 遇到数字 → 直接输出
2. 遇到左括号 '(' → 入栈
3. 遇到右括号 ')' → 弹出并输出，直到遇到 '('
4. 遇到运算符：
   - 栈顶运算符优先级 >= 当前运算符 → 弹出并输出
   - 然后当前运算符入栈
5. 遍历结束 → 弹出所有剩余运算符
```

```python
def infix_to_postfix(expression: str) -> str:
    """
    将中缀表达式转换为后缀表达式（逆波兰表达式）
    
    参数：
        expression: 中缀表达式字符串，各元素用空格分隔
                    例如 "3 + 4 * 2"
    
    返回值：
        str: 后缀表达式，各元素用空格分隔
             例如 "3 4 2 * +"
    """
    
    # 运算符优先级定义
    precedence = {
        '+': 1,
        '-': 1,
        '*': 2,
        '/': 2
    }
    
    stack = []       # 运算符栈
    output = []      # 输出列表
    
    # split() 会将字符串按空格分割成列表
    # 例如 "3 + 4".split() → ['3', '+', '4']
    tokens = expression.split()
    
    for token in tokens:
        if token.isdigit():                    # 如果是数字
            output.append(token)               # 直接输出
        elif token == '(':                     # 如果是左括号
            stack.append(token)                # 入栈
        elif token == ')':                     # 如果是右括号
            # 弹出直到遇到左括号
            while stack and stack[-1] != '(':
                output.append(stack.pop())
            stack.pop()                        # 弹出左括号（不输出）
        elif token in precedence:              # 如果是运算符
            # 栈顶运算符优先级 >= 当前运算符时，弹出栈顶
            while (stack and 
                   stack[-1] != '(' and 
                   stack[-1] in precedence and
                   precedence[stack[-1]] >= precedence[token]):
                output.append(stack.pop())
            stack.append(token)                # 当前运算符入栈
    
    # 弹出栈中所有剩余运算符
    while stack:
        output.append(stack.pop())
    
    return ' '.join(output)    # 用空格连接成字符串


# ===== 测试 =====
print("=== 中缀转后缀 ===")
expressions = [
    "3 + 4 * 2",             # → 3 4 2 * +
    "( 3 + 4 ) * 2",         # → 3 4 + 2 *
    "1 + 2 * 3 - 4",         # → 1 2 3 * + 4 -
]
for expr in expressions:
    result = infix_to_postfix(expr)
    print(f"  中缀: {expr}")
    print(f"  后缀: {result}\n")
```

#### 后缀表达式求值

```python
def eval_postfix(expression: str) -> float:
    """
    计算后缀表达式（逆波兰表达式）的值
    
    算法：
    1. 遇到数字 → 入栈
    2. 遇到运算符 → 弹出两个数，计算结果，将结果入栈
    3. 最终栈中只剩一个数，就是结果
    
    参数：
        expression: 后缀表达式字符串，各元素用空格分隔
    
    返回值：
        float: 计算结果
    
    图解 "3 4 2 * +" 的求值过程：
    
    token  操作                栈状态
    3      数字，入栈          [3]
    4      数字，入栈          [3, 4]
    2      数字，入栈          [3, 4, 2]
    *      弹出 2 和 4，       [3, 8]        ← 4*2=8 入栈
    +      弹出 8 和 3，       [11]          ← 3+8=11 入栈
    
    结果：11
    """
    
    stack = []
    tokens = expression.split()
    
    for token in tokens:
        if token.lstrip('-').isdigit():       # 判断是否为数字（包括负数）
            # 补充知识：
            # lstrip('-') 去掉开头的负号再判断是否全是数字
            # 这样 "-5" 也能被识别为数字
            stack.append(float(token))
        else:
            # 注意：先弹出的是右操作数，后弹出的是左操作数
            # 这个顺序对减法和除法很重要！
            right = stack.pop()    # 右操作数
            left = stack.pop()     # 左操作数
            
            if token == '+':
                stack.append(left + right)
            elif token == '-':
                stack.append(left - right)
            elif token == '*':
                stack.append(left * right)
            elif token == '/':
                stack.append(left / right)
    
    return stack[0]    # 栈中最后一个元素就是结果


# ===== 测试 =====
print("=== 后缀表达式求值 ===")
test_cases = [
    ("3 4 2 * +",     11.0),      # 3 + 4*2 = 11
    ("3 4 + 2 *",     14.0),      # (3+4)*2 = 14
    ("5 1 2 + 4 * + 3 -", 14.0),  # 5 + (1+2)*4 - 3 = 14
]
for expr, expected in test_cases:
    result = eval_postfix(expr)
    status = "✓" if result == expected else "✗"
    print(f"  {status} eval('{expr}') = {result}")
```

### 6.4.3 函数调用栈（递归原理）

#### 什么是调用栈？

当程序执行函数调用时，系统会使用一个**调用栈（Call Stack）** 来管理函数的执行：

- 每次**调用**一个函数 → 将该函数的信息（参数、局部变量、返回地址）**压入**调用栈
- 函数**执行完毕** → 从调用栈**弹出**，返回到上一层函数

```python
def func_a():
    print("开始 a")
    func_b()           # 调用 b，b 的信息入栈
    print("结束 a")

def func_b():
    print("开始 b")
    func_c()           # 调用 c，c 的信息入栈
    print("结束 b")

def func_c():
    print("开始 c")
    print("结束 c")    # c 执行完，从栈弹出

func_a()

# 输出：
# 开始 a
# 开始 b
# 开始 c
# 结束 c  ← c 先结束（后进先出！）
# 结束 b
# 结束 a

# 调用栈变化过程：
#
# 调用 func_a()  →  [func_a]
# 调用 func_b()  →  [func_a, func_b]
# 调用 func_c()  →  [func_a, func_b, func_c]
# func_c 返回    →  [func_a, func_b]
# func_b 返回    →  [func_a]
# func_a 返回    →  []
```

#### 递归与栈的关系

**递归本质上就是在利用调用栈！** 每次递归调用都会在栈上压入一个新的"栈帧"。

```python
def factorial(n):
    """
    递归计算阶乘：n! = n × (n-1) × ... × 1
    
    调用栈示意（以 factorial(4) 为例）：
    
    factorial(4)  →  入栈
      factorial(3)  →  入栈
        factorial(2)  →  入栈
          factorial(1)  →  入栈，返回 1
        factorial(2) = 2 * 1 = 2  →  出栈
      factorial(3) = 3 * 2 = 6    →  出栈
    factorial(4) = 4 * 6 = 24     →  出栈
    """
    if n <= 1:          # 基准情况（Base Case）：递归的终止条件
        return 1
    return n * factorial(n - 1)    # 递归调用


def factorial_with_stack(n):
    """
    用显式栈模拟递归，帮助理解递归的本质
    
    这个函数和上面的递归版本做的事情完全一样，
    只是把"隐式调用栈"变成了"显式的栈"。
    """
    stack = []
    
    # 模拟递归调用：把每一层的 n 压入栈
    while n > 1:
        stack.append(n)
        n -= 1
    
    # 模拟递归返回：从栈中逐一弹出，累乘
    result = 1
    while stack:
        result *= stack.pop()
    
    return result


# ===== 测试 =====
print("=== 阶乘计算 ===")
for n in [1, 4, 5, 10]:
    r1 = factorial(n)
    r2 = factorial_with_stack(n)
    print(f"  {n}! = {r1}（递归）= {r2}（显式栈）")
```

> **⚠️ 递归深度限制**：Python 默认递归深度上限是 1000 层。
> 如果递归太深，会抛出 `RecursionError: maximum recursion depth exceeded`。
> 可以用 `import sys; sys.setrecursionlimit(10000)` 修改，但不建议设太大。

### 6.4.4 浏览器前进后退

**思路**：用**两个栈**分别管理"后退历史"和"前进历史"。

```
back_stack:    存放可以"后退"到的页面
forward_stack: 存放可以"前进"到的页面

规则：
1. 访问新页面 → 当前页面入 back_stack，清空 forward_stack
2. 后退（back） → 当前页面入 forward_stack，从 back_stack 弹出作为当前页面
3. 前进（forward） → 当前页面入 back_stack，从 forward_stack 弹出作为当前页面
```

```python
class BrowserHistory:
    """
    用两个栈模拟浏览器的前进/后退功能
    """
    
    def __init__(self, homepage: str):
        """
        初始化浏览器，设置主页
        
        参数：
            homepage: 主页 URL
        """
        self.current = homepage       # 当前页面
        self.back_stack = []          # 后退栈
        self.forward_stack = []       # 前进栈
    
    def visit(self, url: str):
        """
        访问新页面
        - 当前页面存入后退栈
        - 前进栈清空（因为进入了新的浏览路径）
        """
        self.back_stack.append(self.current)
        self.current = url
        self.forward_stack.clear()    # 清空前进历史
        # 补充知识：list.clear() 清空列表中的所有元素
        print(f"  访问: {url}")
    
    def back(self):
        """
        后退一步
        - 当前页面存入前进栈
        - 从后退栈弹出一个页面作为当前页面
        """
        if not self.back_stack:
            print("  无法后退，已是第一页")
            return
        self.forward_stack.append(self.current)
        self.current = self.back_stack.pop()
        print(f"  后退到: {self.current}")
    
    def forward(self):
        """
        前进一步
        - 当前页面存入后退栈
        - 从前进栈弹出一个页面作为当前页面
        """
        if not self.forward_stack:
            print("  无法前进，已是最新页")
            return
        self.back_stack.append(self.current)
        self.current = self.forward_stack.pop()
        print(f"  前进到: {self.current}")
    
    def status(self):
        """打印当前状态"""
        print(f"  当前页面: {self.current}")
        print(f"  后退栈:   {self.back_stack}")
        print(f"  前进栈:   {self.forward_stack}")


# ===== 模拟浏览过程 =====
print("=== 浏览器前进后退模拟 ===\n")
browser = BrowserHistory("google.com")

browser.visit("youtube.com")
browser.visit("github.com")
browser.visit("stackoverflow.com")
print()
browser.status()

print("\n--- 后退两次 ---")
browser.back()               # → github.com
browser.back()               # → youtube.com
print()
browser.status()

print("\n--- 前进一次 ---")
browser.forward()            # → github.com
print()
browser.status()

print("\n--- 访问新页面（前进栈被清空） ---")
browser.visit("python.org")  # 前进栈清空！
print()
browser.status()
```

### 6.4.5 深度优先搜索（DFS）

**深度优先搜索（Depth-First Search）** 是图/树的遍历算法。递归版的 DFS 隐式使用调用栈，我们也可以用显式栈来实现非递归版本。

```python
def dfs_iterative(graph: dict, start: str) -> list:
    """
    用栈实现深度优先搜索（非递归版）
    
    参数：
        graph: 图的邻接表表示（字典形式）
               例如 {'A': ['B', 'C'], 'B': ['D']} 表示 A 连接 B 和 C，B 连接 D
        start: 起始节点
    
    返回值：
        list: DFS 遍历顺序
    
    补充知识：
    - dict 是 Python 的字典类型，存储 key-value 对
    - set 是集合类型，元素不重复，查找速度快（O(1)）
    """
    
    visited = set()       # 记录已访问的节点（用集合，查找是 O(1)）
    result = []           # 记录遍历顺序
    stack = [start]       # 初始：起始节点入栈
    
    while stack:                          # 栈不为空就继续
        node = stack.pop()                # 弹出栈顶节点
        
        if node not in visited:           # 如果没访问过
            visited.add(node)             # 标记为已访问
            result.append(node)           # 记录遍历顺序
            
            # 将相邻节点入栈（逆序入栈，这样正序节点会先被弹出）
            # 补充知识：reversed() 返回反转后的迭代器
            for neighbor in reversed(graph.get(node, [])):
                # graph.get(node, []) 获取 node 的邻居列表
                # 如果 node 不在图中，返回空列表 []
                if neighbor not in visited:
                    stack.append(neighbor)
    
    return result


# ===== 示例 =====
# 构建一个图：
#     A
#    / \
#   B   C
#  / \   \
# D   E   F

graph = {
    'A': ['B', 'C'],
    'B': ['D', 'E'],
    'C': ['F'],
    'D': [],
    'E': [],
    'F': []
}

print("=== DFS 遍历（栈实现） ===")
print(f"  图结构: {graph}")
result = dfs_iterative(graph, 'A')
print(f"  DFS 顺序: {' → '.join(result)}")
# 输出：A → B → D → E → C → F
```

## 6.5 Python 中的"栈"实现方式

### 6.5.1 使用 list

最简单、最常用的方式：

```python
# 直接用 list 当栈
stack = []

stack.append(1)       # push: [1]
stack.append(2)       # push: [1, 2]
stack.append(3)       # push: [1, 2, 3]

top = stack[-1]       # peek: 3
val = stack.pop()     # pop:  val=3, stack=[1, 2]
is_empty = len(stack) == 0    # False
```

### 6.5.2 使用 collections.deque

```python
from collections import deque
# 补充知识：
# collections 是 Python 的标准库模块，提供了多种增强的数据结构
# deque 是 "double-ended queue"（双端队列）的缩写，读作 "deck"

stack = deque()

stack.append(1)       # push: deque([1])
stack.append(2)       # push: deque([1, 2])
stack.append(3)       # push: deque([1, 2, 3])

top = stack[-1]       # peek: 3
val = stack.pop()     # pop:  val=3, stack=deque([1, 2])
is_empty = len(stack) == 0    # False
```

### 两者对比

| 特性 | `list` | `collections.deque` |
|------|--------|---------------------|
| push（末尾添加） | `append()` — \(O(1)\) 均摊 | `append()` — \(O(1)\) |
| pop（末尾删除） | `pop()` — \(O(1)\) | `pop()` — \(O(1)\) |
| 头部操作 | \(O(n)\)（需要移动元素） | \(O(1)\)（双端队列优势） |
| 随机访问 | `list[i]` — \(O(1)\) | `deque[i]` — \(O(n)\) |
| 线程安全 | 否 | `append`/`pop` 是原子操作 |
| 建议 | 只当栈用时完全够用 | 同时需要栈+队列操作时更好 |

> **结论**：只当栈用时 `list` 就够了。如果你同时需要在两端操作（比如既要当栈又要当队列），用 `deque` 更高效

## 6.6 易错点总结

### 错误 1：空栈 pop

```python
# ❌ 错误：没检查就 pop
stack = []
val = stack.pop()    # IndexError: pop from empty list

# ✅ 正确：先判空
stack = []
if stack:            # 非空才 pop
    val = stack.pop()

# ✅ 或者用异常处理
try:
    val = stack.pop()
except IndexError:
    print("栈为空")
```

### 错误 2：peek 时搞混索引

```python
# ❌ 错误：用 stack[0] 当栈顶
stack = [1, 2, 3]
top = stack[0]       # 这是栈底！值是 1

# ✅ 正确：用 stack[-1] 获取栈顶
stack = [1, 2, 3]
top = stack[-1]      # 栈顶是 3
```

### 错误 3：括号匹配逻辑漏洞

```python
# ❌ 错误：忘记处理"右括号多于左括号"的情况
def bad_match(s):
    stack = []
    mapping = {')': '(', ']': '[', '}': '{'}
    for c in s:
        if c in '({[':
            stack.append(c)
        else:
            # 如果栈已经空了，stack[-1] 会报错！
            if stack[-1] == mapping[c]:    # ← 危险！
                stack.pop()
    return len(stack) == 0

# ✅ 正确：先检查栈是否为空
def good_match(s):
    stack = []
    mapping = {')': '(', ']': '[', '}': '{'}
    for c in s:
        if c in '({[':
            stack.append(c)
        else:
            if not stack or stack[-1] != mapping[c]:  # ← 先判空！
                return False
            stack.pop()
    return len(stack) == 0
```

### 错误 4：链表实现时指针错误

```python
# ❌ 错误：push 时忘记连接新节点和旧栈顶
def bad_push(self, item):
    new_node = Node(item)
    self._top = new_node     # 旧栈顶丢失了！链表断了！

# ✅ 正确：先连接，再更新栈顶
def good_push(self, item):
    new_node = Node(item)
    new_node.next = self._top    # 先让新节点指向旧栈顶
    self._top = new_node         # 再更新栈顶
```

### 错误 5：后缀表达式求值时操作数顺序

```python
# ❌ 错误：减法和除法时左右操作数搞反
# 后缀表达式 "5 3 -" 应该是 5 - 3 = 2

right = stack.pop()    # 先弹出的是右操作数（3）
left = stack.pop()     # 后弹出的是左操作数（5）

# ❌ 搞反了：
result = right - left   # 3 - 5 = -2（错误）

# ✅ 正确：
result = left - right   # 5 - 3 = 2（正确）
```

## 6.7 总结

### 核心要点速查表

```css
栈 = 受限的线性表，只能在一端操作
核心原则：LIFO（后进先出）

五大操作：
├── push(x)    → 入栈
├── pop()      → 出栈
├── peek()     → 看栈顶
├── is_empty() → 判空
└── size()     → 大小

两种实现：
├── 数组实现（顺序栈）→ Python list，推荐默认使用
└── 链表实现（链式栈）→ 单链表，面试常考

所有操作时间复杂度：O(1)

五大经典应用：
├── 括号匹配     → 左括号入栈，右括号匹配弹出
├── 表达式求值   → 中缀转后缀，后缀用栈计算
├── 函数调用栈   → 递归的本质就是栈
├── 浏览器前进后退 → 两个栈协同工作
└── DFS 深度优先搜索 → 用栈替代递归

Python 实现选择：
├── list        → 简单够用，99% 的场景
└── deque       → 需要双端操作时更优
```

1. **先掌握数组实现**：用 `list` 手写一个完整的 Stack 类
2. **再理解链表实现**：重点理解 push/pop 时指针的变化
3. **重点练习应用题**：括号匹配和表达式求值是面试高频题
4. **理解递归与栈的关系**：这对后续学习树、图的遍历至关重要
5. **刷题**：LeetCode 上搜索 "栈" 标签，从简单题开始
