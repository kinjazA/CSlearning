# FEDformer: 频域增强的时序预测 Transformer

> **论文**: [FEDformer: Frequency Enhanced Decomposed Transformer for Long-term Series Forecasting](https://arxiv.org/abs/2201.12740)
> **机构**: 北京航空航天大学 & 微软亚洲研究院 (ICML 2022)
> **作者**: Tian Zhou, Ziqing Ma, Qingsong Wen, Xue Wang, Liang Sun, Rong Jin

---

## 1. 核心思想

FEDformer 回答了一个关键问题：**时序预测中的注意力机制，能不能有更高效的实现方式？** 答案来自信号处理领域的经典工具：傅里叶变换。

### 从一个直观类比开始

想象你要预测一首歌曲的下一段旋律。你可以：
- **方法A（Transformer）**：逐个音符对比所有历史音符的相似度（$O(L^2)$ 复杂度）
- **方法B（FEDformer）**：把音乐转换成频谱图，看哪些频率（音高）在重复出现，只关注这些重要频率（$O(L \log L)$ 复杂度）

FEDformer 就是把"方法B"应用到时间序列预测上。

### 两个核心洞察

**洞察一：Attention 可以在频域实现**

传统 Attention 在**时域**计算"哪些历史时刻与当前时刻相关"。但信号处理告诉我们：时域的相关性计算，等价于频域的频谱操作（这叫 Wiener-Khinchin 定理）。

- 时域做法：计算所有时刻两两之间的相似度 → $O(L^2)$ 复杂度
- 频域做法：用傅里叶变换找出重要的周期模式，只增强这些模式 → $O(L \log L)$ 复杂度

FEDformer 的创新：随机选取 $M$ 个重要频率（通常 $M \ll L$），在频域直接操作，再变换回时域。这就是**随机频域增强**——用频率滤波器替代注意力矩阵。

**洞察二：季节性和趋势需要不同的处理方式**

观察电力消耗数据：每天24小时的规律波动（季节性）+ 逐年增长的总体趋势。这两种模式特征完全不同：

- **季节性**：有周期规律，重复出现 → 频域表示非常稀疏（只有几个主频率）→ 适合用傅里叶变换处理
- **趋势**：缓慢变化，几乎没有波动 → 频域能量集中在0频率附近 → 时域处理更简单直接

FEDformer 的做法：在模型开始时用移动平均分解出趋势和季节两部分，分别用不同的方式处理。

### 一句话总结

**移动平均分解序列（拆分趋势和季节）+ 频域随机增强（替代 Attention）+ Encoder-Decoder 架构 = 更快更准的长序列预测。**

```
标准 Transformer: Attention = softmax(QK^T)V                    O(L²) 时域内积
Autoformer:        Attention = AutoCorrelation(Q,K,V)            O(L log L) 时延-频域混合
FEDformer:         Attention = RandomFreqEnhance(X)              O(L log L) 纯频域随机掩码
```

## 2. 演进路线：从标准 Transformer 到频域增强

```
标准 Transformer (2017)
    │  问题: O(L²) 复杂度 + 时序数据的季节性/趋势结构未被利用
    │
    ├─→ Informer (AAAI 2021) — ProbSparse 注意力，只选 Top-u 个 Query
    │   思路: 不是所有 Q 都重要，稀疏化降低复杂度到 O(L log L)
    │
    ├─→ Autoformer (NeurIPS 2021) — 序列分解 + 自相关机制
    │   关键进步: 时延自相关 (Auto-Correlation) + 渐进式序列分解
    │   自相关计算: R_XX(τ) 通过 FFT 加速 → O(L log L)
    │
    └─→ ★ FEDformer (ICML 2022) — 频域随机增强
        两个关键改进:
          1. 序列分解从 Encoder 内部操作 → Encoder 之前先分解
          2. 自相关 → 频域随机增强（更简单、更稳定）
```

Autoformer 是 FEDformer 最重要的一步垫脚石：它首次将序列分解引入 Transformer 架构，并发现自相关可以用 FFT 计算。FEDformer 在此基础上更进一步：(1) 把分解从"Encoder 内部做"改为"Encoder 外部预处理"；(2) 把"计算自相关"改为"直接在频域随机增强"。

## 3. 模型架构

### 3.1 整体结构

```
输入: x_{1:L}  (回溯 L 步，C 个变量)
                      │
                      ▼
         ┌────────────────────────┐
         │  1. 序列分解预处理        │  将输入分解为 X_en^trend + X_en^seas
         │     移动平均 + 残差      │  趋势和季节分离进入不同路径
         └────────────────────────┘
                  │
         ┌────────┴────────┐
         ▼                 ▼
  X_en^trend          X_en^seas
         │                 │
         ▼                 ▼
┌─────────────────┐  ┌─────────────────────────┐
│ Encoder (时域)   │  │ Encoder (频域增强)        │
│ • 趋势 + 残差    │  │ • FEB × N 层             │
│ • 不额外建模     │  │ • 随机频率增强替换注意力   │
│   直接传递       │  │ • 季节分量在频域处理       │
└─────────────────┘  └─────────────────────────┘
         │                 │
         └────────┬────────┘
                  ▼
         X_en^trend + X_en^seas  (encoder output)
                  │
                  ▼
         ┌────────────────────────┐
         │  Decoder (频域增强)     │
         │  • FEA × M 层          │
         │  • Auto-Correlation    │
         │    (论文也探索了频域    │
         │     注意力替代方案)     │
         └────────────────────────┘
                  │
                  ▼
         ┌────────────────────────┐
         │  Output Projection     │
         │  预测: Y_{1:H}          │
         └────────────────────────┘
```

### 3.1.1 序列分解预处理

论文将这步叫 `series_decomp`——与 Autoformer 名字相同但**使用方式不同**：

- **Autoformer**: 在 Encoder 的每一层内部交替做分解（渐进式分解）
- **FEDformer**: 在 Encoder 之前**一次性**分解成趋势 + 季节两个分量，分开送入不同的 Encoder 路径

```
X ∈ R^{L×C}  (回溯窗口)
     │
     ├─→ AvgPool(kernel=25, stride=1, padding="same")  → X_trend
     │
     └─→ X - X_trend  → X_seasonal
```

分解后，趋势分量直接传入 Encoder（不做频域变换——趋势的频谱没有局部性，时域更好处理），季节分量传入频域增强 Encoder。

### 3.2 Encoder: 频域增强模块 (Frequency Enhanced Block, FEB)

FEB 是 FEDformer 的核心组件，也是最需要理解的部分。让我们用一个直观的类比来解释它的工作原理。

#### 直观理解：从音乐均衡器说起

想象你在使用音乐播放器的均衡器：
1. 音乐信号被分解成不同频率（低音、中音、高音）
2. 你可以选择增强某些频率（比如提升低音）
3. 调整后的各频率重新合成音乐

FEB 做的就是类似的事情，只不过是对时间序列数据：

```
输入: x_seas ∈ R^{L×d}   (季节分量，L=序列长度，d=特征维度)

  ┌─────────────────────────────────────────────────────┐
  │  步骤1: FFT 变换 —— 把时间序列"翻译"成频率语言           │
  │  ─────────────────────────────────────────────────  │
  │  Z = FFT(x)  ∈ C^{L×d}                             │
  │  例如：96个时间点 → 96个频率分量                        │
  │  (低频=长周期，高频=短周期)                             │
  └─────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────┐
  │  步骤2: 随机采样重要频率 —— 只保留关键的"音高"            │
  │  ─────────────────────────────────────────────────  │
  │  从 L=96 个频率中随机选 M=32 个                        │
  │  Z_selected = Z[rand_indices]  ∈ C^{M×d}           │
  │                                                     │
  │  为什么随机选？                                       │
  │  • 真实信号的频谱很稀疏（只有少数频率有能量）              │
  │  • 随机采样是一种"频率dropout"，防止过拟合               │
  │  • M << L 大幅减少计算量                              │
  └─────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────┐
  │  步骤3: 频域增强 —— 学习"哪些频率重要"              │
  │  ─────────────────────────────────────────────────  │
  │  Z_enhanced = Z_selected ⊙ R                        │
  │  R ∈ C^{M×d} 是可学习的复数权重矩阵                 │
  │                                                      │
  │  关键理解：R 就是"频域的注意力权重"                  │
  │  • 传统Attention学习"时间点之间的关系"              │
  │  • FEB学习"频率分量之间的关系"                      │
  │  • 但在频域做乘法比时域做内积快得多！                │
  └─────────────────────────────────────────────────────┘
         │
         ▼
  ┌─────────────────────────────────────────────────────┐
  │  步骤4: 逆变换 —— 把"频率语言"翻译回"时间语言"       │
  │  ─────────────────────────────────────────────────  │
  │  Z_filled = zero_pad(Z_enhanced, L)  (其他频率补0)   │
  │  x_enhanced = IFFT(Z_filled) ∈ R^{L×d}              │
  └─────────────────────────────────────────────────────┘
         │
         ▼
输出: x_seas_enhanced  (增强后的季节分量)
```

#### 为什么这等价于 Attention？

**传统 Attention 的本质**：计算 $QK^T$，得到"每个时间步与其他时间步的相关性"。

**信号处理的视角**（Wiener-Khinchin 定理）：
- 时域的"自相关"（衡量信号与其延迟版本的相似度）
- 等价于频域的"功率谱"（衡量各频率的能量分布）

数学上：$$\text{相关性计算} \approx \text{IFFT}\left( |\text{FFT}(X)|^2 \right)$$

**FEDformer 的改进**：
- 不计算固定的自相关，而是学习一个频域滤波器 $R$
- $R$ 学会"哪些频率对预测重要"（更灵活）
- 复杂度从 $O(L^2)$ 降到 $O(M \cdot d + L \log L)$

### 3.2.1 FEB-f vs FEB-w：两种频率选择策略

| 策略 | FEB-f (Fourier) | FEB-w (Wavelet) |
|------|-----------------|-----------------|
| 基础变换 | 离散傅里叶变换 (DFT) | 离散小波变换 (DWT) |
| 频率基底 | 正弦/余弦（全局，无限长） | 小波基（局部，时频同时定位） |
| 频率选择 | 随机均匀采样 M 个频率 | 固定多尺度小波分解 |
| 优势 | 全局周期性捕获好 | 突变/不连续信号更好 |
| 劣势 | 瞬变信号吃亏 | 全局周期不如傅里叶 |

论文中两者效果相当，FEB-f 因实现简单而成为默认推荐。

**为什么随机选择频率？**
1. 频谱通常是稀疏的（只有少数主导频率）
2. 随机选择 M 个频率是频谱的无偏估计
3. 复杂度从 $O(L)$ 降到 $O(M)$
4. 从信息论角度，将重要的频率信息均匀分布在所有频率上

### 3.3 Decoder: 频域增强注意力 (Frequency Enhanced Attention, FEA)

Decoder 的结构与 Autoformer 的 Decoder 相似，但将自相关机制替换为频域增强注意力：

```
Decoder 输入:
  • X_de_ens: 部分 X_0 = X_{L/2+1:L}（回溯的后半截）+ 零填充 (待预测部分)
  • X_de_seas, X_de_trend: X_0 分解后的季节/趋势分量
  • Encoder 输出: X_en_out

每个 Decoder Layer:
  ┌────────────────────────────────────────────────┐
  │ 1. 频域自注意力 (FEA) on X_de_seas              │
  │    同 FEB 的机制——FFT → 频域乘法 → IFFT          │
  │    但 M 的选取更大（自注意力需要更精细的频率分辨）    │
  ├────────────────────────────────────────────────┤
  │ 2. 频域交叉注意力 (FEA-cross)                   │
  │    Query: Decoder 的季节输出                     │
  │    Key/Value: Encoder 的季节输出                  │
  │    同样用频域增强替代点积注意力                    │
  ├────────────────────────────────────────────────┤
  │ 3. 趋势分量 + 残差传递                           │
  │    趋势部分不用频域，直接在时域处理                │
  └────────────────────────────────────────────────┘
  │
  ▼
Decoder 输出 → 最终投影 → 预测 Y_{1:H}
```

### 3.4 数据流概览

以 ETTh1 (7 个变量，$L=96$，$H=336$，$d_{\text{model}}=512$，$M=32$) 为例：

**Encoder 流程**：
1. 输入 `(B, 96, 7)` → 序列分解 → `X_trend: (B, 96, 7)`, `X_seas: (B, 96, 7)`
2. Embedding: `(B, 96, 7)` → `(B, 96, 512)`
3. 趋势路径：Embed 后直接传递 + 残差累积
4. 季节路径：FFT → 随机选 M=32 个频率 → 复数权重增强 → IFFT → `(B, 96, 512)`
5. 输出 = 趋势路径 + 季节路径 → `(B, 96, 512)`

**Decoder 流程**：
1. 初始 token = 回溯后半段 `(B, 48, 7)` + 零填充 `(B, 336, 7)` → `(B, 384, 7)`
2. 同样分解为趋势 + 季节
3. 自注意力（FEA）、交叉注意力（FEA-cross）均用频域增强
4. 最终投影 `(B, 384, 512)` → `(B, 336, 7)`

> 完整的数据流转和 shape 变化见 `code/04_transformer/01_fedformer_demo.py`。

## 4. 数学基础：为什么频域可以等价 Attention？

### 4.1 Wiener-Khinchin 定理（用人话解释）

这是 FEDformer 最重要的理论基础。让我们分步理解：

**第一步：什么是自相关？**

自相关衡量"一个信号与它自己的延迟版本有多相似"。比如：
- 每天的气温数据，今天和昨天很相似（延迟1天的相关性高）
- 今天和7天前相似（延迟7天的相关性高，周周期）

数学表达：$R_{XX}(\tau) = \mathbb{E}[X(t) \cdot X(t+\tau)]$，其中 $\tau$ 是时间延迟

**第二步：自相关和频谱的关系**

Wiener-Khinchin 定理说：
$$\text{频域功率谱} = \text{FFT}(\text{时域自相关})$$
$$\text{时域自相关} = \text{IFFT}(\text{频域功率谱})$$

换句话说：**时域的相关性信息，完全可以用频域的频谱表示**。

**第三步：Attention 其实是在算相关性**

标准 Attention 的核心操作：$A_{ij} = q_i^T k_j$

当 $Q=K=X$（自注意力）时，$QK^T$ 计算的就是各时间步之间的内积相似度——这本质上是相关性！

**因此**：
```
时域做法：Attention = QK^T                → O(L²) 复杂度
         (逐对计算 L×L 个时间点的相关性)

频域做法：相关性 = IFFT(|FFT(X)|²)       → O(L log L) 复杂度  
         (FFT变换→频域乘法→逆变换)
```

### 4.2 FEDformer 的进一步改进：学习频域滤波器

直接用 $|\text{FFT}(X)|^2$ 只能得到信号自己和自己的相关性。但预测时，我们可能需要更灵活的模式：

**传统 Attention 的学习方式**：
```
Q = X·W_Q,  K = X·W_K,  V = X·W_V
Attention = softmax(QK^T/√d)·V
↑ 通过学习 W_Q, W_K, W_V 来捕捉重要的相关模式
```

**FEDformer 的学习方式**：
```
Z = FFT(X)                    (转到频域)
Z_enhanced = Z ⊙ R            (R 是可学习的复数权重矩阵)
Output = IFFT(Z_enhanced)      (转回时域)
↑ 直接学习频域滤波器 R，它决定"哪些频率重要"
```

**为什么这样更好？**
1. **更直接**：R 直接作用于频率分量，不需要 Q、K、V 三个投影
2. **更快**：只需 FFT + 逐元素乘法 + IFFT，避免了 $O(L^2)$ 的矩阵乘法
3. **更稀疏**：随机选 M 个频率（M << L），进一步降低计算量

## 5. 关键设计决策

### 5.1 序列分解为什么放在 Encoder 之前？

这是 FEDformer 和 Autoformer 的一个重要区别。让我们通过对比来理解：

#### 方案对比

| 设计维度 | Autoformer | FEDformer |
|---------|------------|-----------|
| **分解位置** | Encoder 内部，每一层都做 | Encoder 之前，只做一次 |
| **分解次数** | N 层 Encoder = N 次分解 | 1 次分解 |
| **处理路径** | 单一路径，逐层净化 | 双路径，趋势和季节分开处理 |
| **计算量** | 每层都要移动平均 | 只需一次移动平均 |

#### 用洗矿的类比来理解

**Autoformer 的做法（渐进式提纯）**：
```
原始矿石 → [第1次筛选] → 稍微干净的矿石 
         → [第2次筛选] → 更干净的矿石
         → [第3次筛选] → 很纯的矿石
```
优势：每一层都在"净化"表示，越往后越纯  
劣势：每次筛选都要花时间

**FEDformer 的做法（一次分类）**：
```
原始矿石 → [按属性分类] → 金属 → [冶炼车间处理]
                      → 石头 → [碎石车间处理]
```
优势：一次分类，分别用最适合的方式处理  
劣势：初始分类如果不准，后面难补救

#### 为什么 FEDformer 这样设计？

**核心哲学**：趋势和季节本质不同，需要不同的"处理工具"

```
趋势分量的特点：
  • 低频、缓慢变化
  • 频域能量集中在0附近
  • 频域处理收益低
  → 最佳方案：时域直接处理（或干脆不处理）

季节分量的特点：
  • 有周期性、重复模式
  • 频域稀疏（只有少数主频率）
  • 频域处理效率高
  → 最佳方案：FFT + 频域增强
```

**具体实现**：
```
输入 X (96个时间点)
    │
    ▼
[移动平均核，kernel=25]
    │
    ├─→ X_trend (慢变的趋势) → Encoder趋势路径（几乎不处理）
    │
    └─→ X_seas = X - X_trend (波动的季节) → Encoder频域路径（FEB增强）
```

#### 这样做的优势

1. **计算高效**：分解只做一次，不是每层重复
2. **针对性强**：趋势用时域、季节用频域，各取所需
3. **并行友好**：两条路径可以并行处理（实际实现中通常串行但逻辑独立）

### 5.2 为什么随机选择频率？

这是 FEDformer 最有创意的设计之一。让我们理解为什么"随机"反而是好策略。

#### 频谱稀疏性原理

真实世界的周期信号有一个重要特点：**频谱稀疏**。

举个例子——每日电力消耗：
```
时域视角（96个小时点）：
  [23.1, 22.8, 21.5, ..., 45.6, 47.2, ...]  ← 96个数字

频域视角（做FFT后）：
  频率0（均值）：能量 = 1000    ← 主导！
  频率1（24小时周期）：能量 = 800    ← 主导！
  频率2（12小时周期）：能量 = 300
  频率3-20：能量 < 10            ← 几乎为0
  频率21-95：能量 < 1            ← 噪声
```

**关键发现**：96个频率中，只有2-3个频率携带了90%的信息！

#### 为什么随机采样就够了？

**直觉理解**：从一大堆数字中找最大的几个

假设你有96个数字，其中只有3个大数（比如1000, 800, 300），其余都很小（<10）。

- 方法A：遍历全部96个 → 100%找到大数，但耗时
- 方法B：随机抽32个 → 有很高概率（>95%）找到那3个大数

FEDformer 用的就是方法B！

**数学角度**：随机采样是无偏估计
```
E[随机选M个频率的能量总和] ≈ (M/L) × 全部频率的能量总和
```
只要 M 足够大（如 M=32, L=96），就能代表整体频谱的分布。

#### 随机选择的四重好处

**1. 计算效率**
```
全频率处理：需要处理 L×d 个复数 (96×512 = 49,152)
随机M个频率：只处理 M×d 个复数 (32×512 = 16,384)
减少 ⅔ 的计算量！
```

**2. 正则化效果（频率Dropout）**
```
类比：神经网络的 Dropout
  每次训练随机关闭一些神经元 → 防止过拟合

频率随机采样
  每次训练随机选不同频率 → 防止过拟合特定频率模式
```

**3. 鲁棒性增强**

如果模型总是看固定的频率：
- 训练数据的噪声频率会被记住
- 测试时遇到频移（shift）就失效

随机采样强迫模型学习"分布式的频率表示"，而不是记住特定频率。

**4. 理论优雅性**

从信息论角度：随机采样是对频谱的蒙特卡洛估计，期望无偏。

#### 实践中的 M 选择

论文实验显示：

| M 值 | 相对 L | 性能 | 计算量 |
|------|-------|------|--------|
| M=5  | L/19  | 78%  | 最低 |
| M=16 | L/6   | 92%  | 低 |
| M=32 | L/3   | 98%  | 中等 ← 论文默认 |
| M=48 | L/2   | 99%  | 较高 |
| M=96 | L     | 100% | 最高（失去优势）|

**经验规则**：M ≈ L/3 是性能和效率的最佳平衡点。

### 5.3 趋势分量为什么不用频域处理？

这是一个关键的设计决策，体现了"用对的工具做对的事"的原则。

#### 趋势的频域特征

让我们看一个具体例子——某公司3年的月度收入：

```
时域表示（36个月）：
  100万, 102万, 105万, 108万, ..., 280万, 285万, 290万
  ↑ 缓慢增长，没有快速波动

频域表示（FFT后的能量分布）：
  频率0（DC分量）：能量 = 99.5%    ← 几乎全部能量！
  频率1：能量 = 0.3%
  频率2-35：能量 < 0.2%           ← 可忽略
```

**关键观察**：趋势信号的频谱极度集中在0频率（直流分量），其他频率几乎没有信息。

#### 对比季节性的频谱

同一时期的月度销售波动（季节性）：

```
时域：有周期性起伏
  [低, 中, 高, 低, 中, 高, ...]  ← 12个月周期

频域：能量分散在多个频率
  频率0：20%      ← 均值
  频率3（12月周期）：50%   ← 主周期
  频率6（6月周期）：15%
  频率12（季度）：10%
  其他：5%
```

季节性有多个重要频率，频域处理能有效提取这些周期模式。

#### 为什么趋势不需要频域增强？

**原因1：频域操作退化为单一系数**

对趋势做频域增强：
```
Z = FFT(趋势)           ← 能量集中在 Z[0]
Z_enhanced = Z ⊙ R      ← 实际上只有 R[0] 在起作用
输出 = IFFT(Z_enhanced)  ← 等价于 趋势 × R[0]
```

这和在时域简单地乘一个系数没区别：
```
趋势_增强 = 趋势 × α     (一个 Linear 层就能做到)
```

**原因2：FFT/IFFT 的计算开销是白费的**

```
频域处理趋势的流程：
  时域 → [FFT, O(L log L)] → 频域 → [乘一个数] → [IFFT, O(L log L)] → 时域
  总开销：2·O(L log L)

直接时域处理：
  趋势 → [Linear层, O(L·d)] → 输出
  总开销：O(L·d)，且通常 d 不大
```

做FFT就是为了利用频谱的结构，但趋势没有结构（只有一个频率），FFT白做了。

**原因3：时域处理趋势更直观**

预测趋势的本质是**外推**：
```
已知：[100, 110, 120, 130]
预测：[140, 150, ?]        ← 线性外推最自然
```

这在时域用简单的线性模型就能做好，不需要复杂的频域变换。

#### FEDformer 的实际做法

```python
# 趋势分量的处理路径
class TrendEncoder:
    def forward(self, x_trend):
        # 几乎不做处理，或者只做简单的残差连接
        return x_trend + self.residual_mlp(x_trend)
        # 不涉及FFT！
```

```python
# 季节分量的处理路径(伪代码)  
class SeasonEncoder:
    def forward(self, x_season):
        # 完整的频域增强流程
        z = FFT(x_season)
        z_selected = random_select(z, M=32)
        z_enhanced = z_selected ⊙ self.R  # 可学习权重
        return IFFT(z_enhanced)
```

#### 总结：工具和任务的匹配

| 信号类型 | 频域特征 | 最佳工具 | FEDformer的选择 |
|---------|---------|---------|----------------|
| **趋势** | 能量集中在频率0 | 时域 Linear | ✅ 时域处理 |
| **季节性** | 能量分散在多个频率 | 频域 FFT | ✅ 频域增强 |
| **噪声** | 频谱均匀分布（白噪声）| 统计滤波 | ❌ FEDformer不擅长 |

这就是"右键原则"：用最简单有效的方法处理每种成分。

## 6. 训练与推理

### 6.1 训练

- **非自回归**：一次前向输出全部 H 步（与 iTransformer/DLinear 相同）
- **Loss**：MSE（论文默认）
- **Autoformer 风格的 Decoder 输入**：回溯后半段 + 零填充
  - 这种输入方式的信号处理含义是"用已知部分推断频谱 → 外推未知部分"
- **Teacher Forcing**：不需要（非自回归）

### 6.2 推理

- 单次前向 = 全部预测
- 主要计算开销在 FFT/IFFT（$O(L \log L)$）+ 频域乘法（$O(M \cdot d)$）
- 对比标准 Transformer 的 $O(L^2 \cdot d)$ 注意力，长序列时快得多
- 对比 DLinear 的 $O(L \cdot H)$ 矩阵乘法，计算量大但精度更高

## 7. 优缺点

### 优点

**1. 复杂度大幅降低：$O(L^2) \to O(L \log L)$**

实际收益举例（L=96, d=512）：
```
标准Attention: 96×96×512 = 4,718,592 次乘法
FEDformer:     FFT(96×512) + 32×512复数乘法 ≈ 500,000 次操作
速度提升：约 9倍
```
长序列（L>512）时优势更明显。

**2. 物理可解释性强**

可以可视化学到的频域权重 R：
```
频率0-5（周周期）：权重 0.8-1.2   ← 模型认为重要
频率6-10（日周期）：权重 0.6-0.9
频率11+（高频噪声）：权重 0.1-0.3  ← 被抑制
```
这比黑盒的 Attention 矩阵更容易理解。

**3. 序列分解是强先验**

趋势+季节分解适用于绝大多数真实数据：
- 电力、天气、交通 → 有明显日/周周期
- 销售、库存 → 有季节性+长期增长
- 分开建模降低了学习难度

**4. 隐式正则化**

随机频率选择 = 频率 Dropout：
- 训练时每次看到不同频率组合
- 防止记住训练数据的特定频率噪声
- 泛化性能提升

**5. 支持多尺度建模**

FEB-w（小波版本）天然具有多尺度分析：
- 低频小波捕捉长期趋势
- 高频小波捕捉短期波动
- 类似图像处理的多尺度分解

### 缺点及应对策略

**1. 变量间依赖建模弱**

**问题**：FEDformer 主要在时间维度做频域增强，跨变量关系依赖 Embedding 层的隐式学习。

**对比**：
- iTransformer：把变量当 token，Attention 显式建模变量关系
- FEDformer：把时间点当 token，变量关系隐式在特征维度

**适用场景**：
- ✅ 变量相对独立的场景（多个传感器各自周期性）
- ❌ 变量强耦合的场景（股票市场的板块联动）

**2. 移动平均核大小敏感**

**问题**：`moving_avg_kernel` 设置不当会严重影响性能。

```
kernel太小（如5）：
  趋势提取不充分，残差中仍有低频 → 季节分量"脏"

kernel太大（如97）：
  趋势吞掉了部分周期信息 → 季节分量信息不足
```

**解决方案**：
- 根据数据周期选择：kernel ≈ 1~1.5 × 主周期长度
- 每小时数据（日周期=24）→ kernel=25
- 每天数据（周周期=7）→ kernel=7 或 13

**3. 随机性导致训练不稳定**

**问题**：训练时每次随机选不同频率，梯度有随机性。

**表现**：
- Batch size 太小（<16）：梯度方差大，loss震荡
- M 太小（<L/10）：随机性过强，收敛慢

**解决方案**：
- 增大 batch size（推荐 ≥32）
- M 不要太小（推荐 M ≥ L/4）
- 使用梯度累积降低随机性影响

**4. 工程复杂度高**

**挑战**：
```
实现需要：
  ✓ 复数运算支持（PyTorch 1.9+）
  ✓ FFT/IFFT 的正确处理（padding, 边界效应）
  ✓ 移动平均的因果性处理（避免未来信息泄露）
  ✓ 随机采样的可复现性（设置随机种子）
```

**对比简单模型**：
- DLinear：50行代码
- FEDformer：500+行代码

**建议**：
- 原型阶段用 DLinear/iTransformer
- 确定需要频域建模后再上 FEDformer

**5. 强假设限制适用性**

**核心假设**：季节分量的频谱稀疏

**不适用的数据**：
```
金融高频数据（分钟级股价）：
  频谱接近白噪声 → 随机采样无法抓住"主频率"
  
纯趋势数据（GDP年度增长）：
  没有周期性 → 季节分解无意义
  
突发事件数据（地震、疫情）：
  非平稳，频谱时变 → FFT假设被违背
```

**适用的数据**：
- ✅ 强周期性：电力、天气、交通
- ✅ 多尺度周期：销售（日+周+季）
- ⚠️ 弱周期性：需要实验对比 DLinear

**6. 频谱泄漏问题**

**问题**：序列长度不是周期整数倍时，FFT 产生频谱泄漏。

```
例如：日周期数据，但 L=100（不是24的倍数）
  → FFT认为信号被"截断"
  → 真实频率能量"泄漏"到相邻频率
```

**解决方案**：
- 选择 L 与已知周期对齐（24, 48, 96, 168...）
- 或使用窗函数（Hann, Hamming）减轻泄漏

## 12. 实践建议

### 建议1：先评估是否需要 FEDformer

**决策树**：
```
你的数据有明显的周期性吗？
  ├─ 否 → 用 DLinear（线性模型可能就够了）
  └─ 是 → 继续

预测长度 H > 回溯长度 L 吗？
  ├─ 是 → 用 PatchTST 或 iTransformer
  └─ 否 → 继续

你需要可解释的频率分析吗？
  ├─ 是 → FEDformer 是好选择
  └─ 否 → 对比实验 FEDformer vs DLinear
```

**快速判断周期性的方法**：

```python
import matplotlib.pyplot as plt
from statsmodels.graphics.tsaplots import plot_acf

# 画自相关图
plot_acf(your_data, lags=100)

# 观察：
# ✓ 有明显周期性的峰值 → 适合 FEDformer
# ✗ 快速衰减到0 → 考虑更简单模型
```

### 建议2：正确设置移动平均核大小

**经验公式**：
```
kernel_size = round(1.0 ~ 1.5 × 主周期长度)
```

**常见数据的推荐值**：

| 数据类型 | 采样频率 | 主周期 | 推荐 kernel |
|---------|---------|-------|------------|
| 电力负荷 | 每小时 | 24小时 | 25 or 37 |
| 交通流量 | 每15分钟 | 96点/天 | 97 |
| 日销售额 | 每天 | 7天 | 7 or 13 |
| 股票价格 | 每分钟 | 无明显周期 | ❌ 不推荐用 FEDformer |

### 建议3：处理 FFT 的边界效应

**问题**：时序长度不是周期的整数倍 → 频谱泄漏

**解决方案**：

**方案A：对齐回溯长度**
```python
# 对于日周期数据
L_candidates = [24, 48, 72, 96, 168, 336, 720]  # 24的倍数或周的倍数
# 选择最接近你需求的长度
```

**方案B：应用窗函数**
```python
import torch

def apply_window(x, window_type='hann'):
    """减轻频谱泄漏"""
    L = x.shape[1]
    if window_type == 'hann':
        window = torch.hann_window(L, device=x.device)
    elif window_type == 'hamming':
        window = torch.hamming_window(L, device=x.device)
    
    window = window.view(1, -1, 1)
    return x * window
```

### 建议4：稳定训练的技巧

**问题**：随机频率选择导致梯度噪声大

**解决方案**：

```python
# 1. 足够大的 batch size
batch_size = 32  # 最小推荐值

# 2. 合理的 M 值
M = L // 3  # 不要太小

# 3. 使用梯度累积
accumulation_steps = 4
for i, batch in enumerate(dataloader):
    loss = model(batch) / accumulation_steps
    loss.backward()
    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

### 建议5：复数运算的实现陷阱

**PyTorch 版本要求**：
```python
import torch
assert torch.__version__ >= "1.9.0", "需要 PyTorch 1.9+ 的复数支持"
```

**常见错误及修正**：

```python
# ❌ 错误：忘记保留复数类型
z = torch.fft.rfft(x)
z_enhanced = z * self.R  # 如果 R 是实数，z 会退化

# ✅ 正确
self.R = nn.Parameter(torch.randn(M, d, dtype=torch.complex64))
z_enhanced = z * self.R

# ❌ 错误：FFT 维度不对
z = torch.fft.rfft(x)  # 默认在最后一维

# ✅ 正确：明确指定时间维度
z = torch.fft.rfft(x, dim=1)
```

### 建议6：对比基线的正确方式

不要只跟 Transformer 系列比！

```python
# 最小对比集
baselines = {
    'DLinear': DLinear(),           # 最简单线性模型
    'PatchTST': PatchTST(),         # 当前 SOTA
    'iTransformer': iTransformer(), # 变量关系建模
    'FEDformer': FEDformer(),       # 你的方案
}

# 如果 DLinear 已经和 FEDformer 差不多
# → 说明你的数据可能不需要复杂的频域建模
```

# 离线提取重要频率

# 在线只用这些频率特征
def extract_freq_features(x, freqs=[3, 7, 14, 21]):
    features = []
    for f in freqs:

## 15. 面试问题

**Q1: FEDformer 的核心创新是什么？**

> 两点：(1) Attention 的频域等价实现——利用 Wiener-Khinchin 定理（自相关 ↔ 功率谱互逆），在频域用随机增强替代时域点积自注意力，复杂度从 $O(L^2)$ 降到 $O(L \log L)$；(2) 序列分解外部化——移动平均提取趋势 + 残差得季节，趋势走时域、季节走频域，各取所需。

**Q2: 序列分解在 FEDformer 和 Autoformer 中的使用有什么不同？**

> Autoformer 在 Encoder 内部**逐层交替**做分解（渐进式分解），每层都重新计算趋势和季节。FEDformer 在 Encoder **入口之前**一次性分解，趋势和季节分两路进入 Encoder——更省计算，但牺牲了"层层净化"的效果。

**Q3: 什么是"随机频率增强"？为什么选择 M 个随机频率而不是全部？**

> 随机频率增强 = FFT 变换到频域 → 随机选 M 个频率 → 用可学习复数矩阵 R 做 element-wise 乘法 → IFFT 变回时域。(1) 自然信号的频谱稀疏——主导频率就几个；(2) 随机采样是频谱的无偏估计；(3) M << L 大幅减少计算量；(4) 随机性天然具有正则化效果（频率 dropout）。

**Q4: Wiener-Khinchin 定理在 FEDformer 中的角色是什么？**

> 定理说：时域自相关函数的傅里叶变换 = 频域功率谱。标准 Attention 的 $QK^T$ 本质上是时域相关操作。因此有频域替代方案：频域做乘法 ↔ 时域做相关。这个定理是整个"频域等价 Attention"设想成立的前提。

**Q5: FEB-f 和 FEB-w 的区别是什么？什么时候各用哪个？**

> FEB-f 用傅里叶变换（全局正弦基），适合稳定的长周期信号（天气、用电）。FEB-w 用小波变换（局部时频基），适合有突变和不连续性的信号（股价跳空、设备异常）。两者论文中效果相当，FEB-f 因简单而默认。

**Q6: 趋势分量为什么不用频域处理？**

> 趋势是低频慢变信号，频域能量几乎全在频率 0 附近。频域增强对趋势的效果 = 在时域用一个系数乘一下 = 等价于 Linear 层。与其多做一个 FFT/IFFT，不如直接在时域处理。FEDformer 的设计哲学：只在有信息增益的地方用频域——即季节分量。

**Q7: FEDformer 和 iTransformer 在多变量建模上有什么区别？**

> FEDformer 主要关注**时间维度的频域建模**，跨变量依赖靠 Embedding 层隐式捕捉。iTransformer 正相反——把变量当 token，Attention 专门学变量间关系，时间依赖交给 FFN 逐变量处理。换句话说，两者"关注的维度"不同：FEDformer 优化时间维度的计算和效果，iTransformer 优化变量维度的耦合关系。

**Q8: FEDformer 在什么数据上表现不好？为什么？**

> 高频金融数据（汇率、股票 tick）。这类数据的频谱接近白噪声——没有明显的主频率峰值，随机取 M 个频率几乎是"瞎选"，信息获取和全频率采样没有本质区别。同时白噪声信号没有季节结构，序列分解也变得意义不大。在 Exchange 数据集上，FEDformer 甚至不如基线 Autoformer。

**Q9: 如果序列分解这么有效，为什么不是所有模型都加上？**

> (1) 不是所有数据都有季节/趋势分离的需要（白噪声数据分解无意义）；(2) 移动平均核大小是敏感超参，数据和预测长度变了就要调；(3) DLinear 证明——最简单的趋势分解（单 kernel）+ 线性投影已经能做到 SOTA 级效果了，复杂的 Transformer 架构加分解的提升有限。

**Q10: FEDformer 的前向传播具体做了多少次 FFT？**

> 每次 FEB 调用做 1 次 FFT + 1 次 IFFT。若有 $N$ 层 Encoder 的 FEB + $M$ 层 Decoder 的 FEA（自注意力）+ $M$ 层 Decoder 的 FEA-cross，则总计约 $(N + 2M)$ 次 FFT + 同等次数 IFFT。以默认配置 $N=2, M=1$ 为例 ≈ 4 次 FFT + 4 次 IFFT。复杂度 $O((N+2M) \cdot L \log L \cdot d)$。
