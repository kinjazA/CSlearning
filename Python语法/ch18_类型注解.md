## 08 类型注解

### 08.1 变量的类型注解

> * 允许开发者为函数参数、函数返回值以及变量**指定预期的数据类型**。类型注解不改变程序的运行行为，它们主要是为了提高代码的可读性和可维护性，同时也便于静态分析工具、IDE、linters等识别代码的类型信息，从而提供更好的代码补全、类型检查和错误提示等功能。

```python
# 基础数据类型注释
var_1: int = 10
var_2: str = "greaf"
var_3: bool = True
# 类对象类型注释
class Student:
    pass
stu1: Student = Student()   # 表示stu1这个对象是Student类型
# 基础容器类型注释
list1: list = [1342,512,654] 
tuple1: tuple = (4,534,76435,376)
dict1: dict = {"wearf":243}
# 容器类型详细注解
list1: list[int] = [1342,512,654] 
tuple1: tuple[int,str,bool] = (4,"frqew",True)
dict1: dict[str,int] = {"wearf":243}
```

> * 除了使用 `变量:类型`这种语法之外，也可以在注释中进行类型注解`# type:类型`，而且感觉这样更好，使得代码本身不至于过于冗长，更加易读
> * 一般来说，很清晰看出类型的，不写。对于一些复杂的，为了代码的可读性更强，可以写下注解
> * 注解只是提示性的，写错了也不会报错

```python
import random
var1 = random.randint(1, 10)   # type:int
```

### 08.2 函数的类型注解

> * 形参注解

```python
# 语法格式
def func1(形参名：类型,形参名:类型,……):

def add(x:int,y:int):
    return x + y
```

> * 返回值注解（一个横杆，一个大于号）

```python
语法格式
def func1(形参名：类型,形参名:类型,……) -> 返回值类型：

def add(x:int,y:int) -> int :
    return x + y
```

### 08.3 Union联合类型注解

> * 使用Union类型可以定义联合类型注解
> * 表示一个变量可以是多种类型中的任意一种，可以使用 `Union` 类型来自 `typing` 模块。这允许你在类型注解中指定多个可能的类型。

```python
# 对变量进行Union注解
from typing import Union
list1: list[Union[str,int]] = [12,43,"erg"]
# 对函数进行Union注解
def func1(data:Union[str,int]) -> Union[str,int]:
    pass
```
