# 第十章 01 对象

## 01 初识对象

> * ### 类（Class）
>   
>   类是一种自定义的、用户定义的数据结构，定义了一组属性（变量）和方法（函数）的集合，用于描述一类对象的共同特性和行为。简单来说，类就是一个蓝图或模板，它描述了一类事物的抽象特点和行为
> * ### 对象（Object）
>
>   对象是类的实例，是根据类的定义创建出来的具体实体。每个对象都拥有类中定义的属性（数据）和可以执行的方法（功能）。当你根据类创建一个对象时，这个**对象就继承了类的所有属性和方法**，但每个对象可以有不同的属性值，表现出不同的状态。
> * ### 类与对象的关系
>
>   类与对象之间的关系就像是模具与铸件的关系。**类是制造对象的模具，规定了对象应该具有的属性和行为**；而**对象则是根据类这个模具制造出来的具体产品**，每个对象都是类的一个实例，拥有自己的身份和状态。

## 02 类的定义和使用

> * 类名称要大写字母开头（规范要求）
> * class是关键字，表示要定义类了
> * **类的属性**即定义在类中的变量（成员变量）
> * **类的行为**即定义在类中的函数（成员方法）

```python
# 类的定义语法
class 类名称:
	类属性
    
    类行为
    
# 创建类对象的语法
对象 = 类名称()
```

> 成员方法的定义语法
>
> * **self关键字在定义类中是不可缺少的**，用来表示类对象自身
> * 当使用类对象调用方法时，**self会自动被python传入，意思是传参时，不用特意加上，可以忽略**
> * 在方法内部，想要访问类的成员变量，必须使用self（**self.成员变量名**）

```python
# 成员方法的定义语法
def 方法名(self, 形参1, 形参2, ……):
    方法体
```

```python
class Student:
    name = None

    def say_hi(self):   # 成员方法参数self不可或缺
        print(f"大家好，我是{self.name}")   # 在成员方法中访问变量，要用self.变量名

    def say_hi2(self, msg):   # 打招呼方式升级，额外增加一段话msg
        print(f"大家好，我是{self.name},{msg}")   # msg不是成员变量，是外部传进去的，不用self去访问
        
st1 = Student()   # 创建一个类Student的对象：st1        
st1.name = "周杰伦"     # 给对象st1的成员变量name赋值
st1.say_hi()   # self参数不用传参     
st1.say_hi2("我喜欢吃香蕉")
        
st2 = Student()   # 创建一个类Student的对象：st2        
st2.name = "张丽英"     
st2.say_hi()          
st2.say_hi2("我喜欢吃苹果")      
```

## 03 构造方法`__init__()`

> 属性（成员变量）的赋值
>
> * `__init__()`方法可以实现在创建类对象时，自动执行，将传入参数自动传递给`__init__()`方法使用

```python
# 使用构造方法对成员变量进行赋值
class Student:
    name = None
    age = None
    tel = None

    def __init__(self,name, age, tel):
        self.name = name   # 类里访问成员变量，借助 self.
        self.age = age
        self.tel = tel
st1 = Student("lisa",21,123242345)   # __init__()方法会自动执行，创建一个对象，传入属性值
print(st1.name)
```

## 04 魔术方法

> * python类内置方法又称为魔术方法，`__init__()`方法便是其一

### 04.1 字符串方法`__str__()`

> * 主要作用是为对象提供一个人性化的字符串表示形式，使得对象在需要被转换为字符串时（比如打印或使用`str()`函数时），能够返回一个易于理解的、对用户友好的描述信息
> * 当定义了一个类，如果实现了`__str__`方法，那么当尝试以字符串形式表示该类的实例时（例如使用`print()`函数或`str()`函数），Python就会调用这个`__str__`方法来生成返回的字符串。这对于调试和日志记录非常有用，因为它可以让对象的状态更加直观

```python
class Student:
    def __init__(self,name,age):
        self.name = name
        self.age = age
stu = Student("周杰伦",31)
print(stu)
print(str(stu))   # 以上两句print结果一样，都是stu的内存地址

class Student:
    def __init__(self,name,age):
        self.name = name
        self.age = age
    def __str__(self):
        return f"Student类对象，name:{self.name},age:{self.age}"
stu = Student("周杰伦",31)
print(stu)
print(str(stu))   # 以上两句print结果一样
```

### 04.2 比较方法`__lt__()`

> * 用于实现对象之间的“小于”比较操作。这个方法中的"lt"代表"less than"
> * 当对两个对象使用 `<` 操作符时，Python会自动调用`__lt__`方法来判断。此方法应接受一个参数（即要比较的另一个对象），并返回一个布尔值：如果第一个对象小于第二个对象，则返回`True`；否则返回`False`
> * 如果是要判断小于等于（大于等于），则需要使用`__le__()`

```python
class MyClass:
    def __init__(self, value):
        self.value = value
    
    def __lt__(self, other):
        return self.value < other.value

a = MyClass(5)
b = MyClass(10)

print(a < b)  # 输出: True
print(b < a)  # 输出: False
```

### 04.3 等于比较方法`__eq__()`

> * 若不实现`__eq__()`方法，对象之间的比较是在比内存地址，结果一定是False
> * 实现了`__eq__()`方法，则可以按照自己想法来决定2个对象相等的比较条件

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

p1 = Point(1, 2)
p2 = Point(1, 2)
print(p1 == p2)  # 输出: True
```

## 05 封装

> * 面向对象编程的简单理解就是基于模板（类）去创建实体（对象），使用对象完成功能开发
> * 面向对象包含三大特点：封装、继承、多态
> * 封装（Encapsulation）是一种将数据（属性）和操作这些数据的方法（行为）捆绑在一起，并对外界隐藏其内部实现细节的设计原则。其核心目标是提高代码的安全性、模块化和可维护性
````python
class BankAccount:
    def __init__(self, balance=0):
        self.__balance = balance  # 私有属性，外部无法直接访问

    def deposit(self, amount):
        self.__balance += amount

    def withdraw(self, amount):
        if amount <= self.__balance:
            self.__balance -= amount
        else:
            print("Insufficient funds")

    def get_balance(self):  # 提供一个方法来安全地获取余额
        return self.__balance
````

> * 对于不公开的隐藏属性和行为，类中提供了**私有成员**的形式来支持，有私有成员变量、私有成员方法
> * 私有成员变量：变量名以__开头（2个下划线）
> * 私有成员方法：方法名以__开头（2个下划线）
> * **私有方法无法直接被类对象使用，类中可以使用，私有变量无法赋值，也无法获取值**

```python
class Phone:
    __current_voltage = None

    def __keep_single_score(self):
        print("CPU单核运行")
phone = Phone()
print(phone.__current_voltage)   # 会报错
```

```python
class Phone:
    __is_5g_enabled = False
    
    def __check_5g(self):
        if self.__is_5g_enabled == True:   # 类内调用变量借助 self.
            print("5g开启")
        else:
            print("5g关闭，使用4g网络")

    def call_by_5g(self):
        self.__check_5g()   # 类内的方法调用也要借助 slef.
        print("正在通话中")
phone = Phone()   # 创建对象
phone.call_by_5g()   # 调用公开方法
```

## 06 继承

> * 继承（Inheritance）是一种机制，允许我们定义一个类（称为子类或派生类），继承另一个类（称为父类或基类）的属性和方法。可以复用现有类的代码，减少重复，并且可以在此基础上增加新的功能或修改已有功能，以满足新的需求，从而实现了代码的可扩展性和可重用性
>
> * 子类在继承父类时，
>
>   ​	单继承：`class 子类名称（父类名称）:`
>
>   ​	多继承：`class 子类名称(父类1，父类2，……)：`
>
>  * 在多继承里，如果两个父类有同样名称的成员变量或方法，那么子类在调用时，按照先到先得的原则，也就是使用先继承的。 `class MyPhone(Phone, NFCReader, RemoteControl):`父类Phone和NFCReader都有一个同名变量（方法），那么子类会继承Phone的，先到先得（从左往右）
>
>  * `pass关键字`：是一个占位语句，表示无内容，空的意思

```python
# 单继承，即一个子类继承了一个父类
class Phone:
    IMEI = None
    producer = "HM"

    def call_by_4g(self):
        print("4g通话")

class Phone2024(Phone):
    face_id = "1001"

    def call_by_5g(self):
        print("5g通话")

phone =Phone2024()
phone.call_by_4g()   # 这个方法就是从父类继承过来的
print(phone.producer)   # 这个成员变量也是从父类继承过来的

# 多继承，即一个子类从多个父类得到继承
class Phone:
    IMEI = None
    producer = "HM"

    def call_by_4g(self):
        print("4g通话")

class NFCReader:
    nfc_type = "第五代"
    producer = "GL"

    def read_card(self):
        print("读卡中")

    def write_card(self):
        print("写卡中")

class RemoteControl:
    rc_type = "红外遥控"

    def control(self):
        print("红外遥控开启")

class MyPhone(Phone, NFCReader, RemoteControl):
    # 因为没有新功能，这个类没有新的成员变量和方法，但这里也不能空着，所以用pass关键字
    pass   # 表示这里是空的，使语法不报错
phone = MyPhone()
phone.call_by_4g() 
```

## 07 复写

> * 子类在继承了父类的成员属性和成员方法之后，如果对其不满意，那么可以进行复写，即在子类中重新定义同名的属性或方法即可

```python
class Phone:
    IMEI = None
    producer = "ihgji"
    
    def call_by_5g(self):
        print("5g通话中")

class MyPhone(Phone):
    producer = "6y4wu"   # 直接会覆盖父类中的
    
    def call_by_5g(self):
        print("5g通话准备完毕")
        print("5g 通话中")   # 同样覆盖
phone = MyPhone()
```

> * 复写后如果还要使用父类的属性和方法，可以通过以下两种方式
> * `父类名.成员变量          父类名.成员方法(self)`
> * `super().成员变量        super().成员方法()`
> * 个人感觉`super()`方式比较适用单继承。当子类为多继承时，不太方便，感觉用方式1不容易搞错

```python
class Phone:
    IMEI = None
    producer = "ihgji"
    
    def call_by_5g(self):
        print("5g通话中")

class MyPhone(Phone):
    producer = "6y4wu"   
    
    def call_by_5g(self):
        print(f"父类的厂商为：{Phone.producer}")   # 调用父类属性的方式1
        print(f"父类的厂商为：{super().producer}")   # 调用父类属性的方式2
        Phone.call_by_5g(self)   # 调用父类方法的方式1
        super().call_by_5g()   # 调用父类方法的方式2
phone = MyPhone()
phone.call_by_5g()
```

## 08 多态

> * 多态指的是多种状态，即完成某个行为时，使用不同的对象会得到不同的状态
> * 多态在Python中的实现主要依赖于鸭子类型（Duck Typing）的概念，即“如果它走起来像鸭子，叫起来也像鸭子，那它就是鸭子”。这意味着在Python中，一个对象是否支持某个操作不是由其继承自特定的类来决定的，而是看它是否实现了所需的方法或属性。因此，只要对象实现了相应的方法，就可以被看作是该类型的对象，而无需关心其具体类型
> * 抽象类就是包含抽象方法的类，抽象方法就是相当于只提出一个概念，不涉及具体细节(pass)。多用于顶层设计，以便子类做具体实现

```python
# 抽象基类（模拟接口）
class Animal:
    def speak(self):   # 这是个抽象方法，具体靠子类来完成，就是顶层设计
        pass

# 实现抽象基类
class Dog(Animal):
    def speak(self):
        return "Woof!"

class Cat(Animal):
    def speak(self):
        return "Meow!"

def animal_sound(animal: Animal):
    """多态性体现：无论传入什么类型的动物，只要它实现了speak方法即可"""
    print(animal.speak())

# 创建不同类型的对象
dog = Dog()
cat = Cat()

# 调用同一个函数，但表现不同的行为
animal_sound(dog)  # 输出: Woof!
animal_sound(cat)  # 输出: Meow!
```
