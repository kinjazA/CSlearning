## 01 异常以及捕获

> * 当python解释器遇到错误无法执行下去时，会出现一些关于错误的提示，这就是异常，也是常说的bug
> * 对可能出现的bug进行**提前准备及处理**，称之为异常处理（捕获异常）

```python
# 基本捕获异常的语法：
try:
    # 可能发生报错的代码段
except:
    # 如果出现异常（这里暗含try里所有异常都会触发此处），执行的代码
else:
    # 这里写如果没有出现上述异常（这里是全部异常）则执行的代码
finally:
    # 无论是否有异常都要执行的代码
# else和finally都是可选的
    
# 捕获指定的异常
try:
    # 可能出现异常的代码
except NameError as e：   # 这里是指定NameError这一错误
	# 如果try里面的代码发生了NameError错误，则执行这里的代码
    
# 捕获多个异常
try:
    # 可能出现异常的代码
except(NameError,ZeroDivisionError) as e:   
    # 如果try里发生了上面两种异常，则执行该处代码
    
# 捕获所有异常
try:
	1 / 0   # 或其他可能出错的代码
except Exception as e:
    print("出现了异常")   # 该方法不利于定位错误具体类型
```

## 02 异常的传递

> * 下面的例子可以发现，func01中发生异常，而我们的异常捕获模块中写的是main函数中的func02。因为func02又调用了func01，所以异常捕获模块虽然不在func01，但也能捕获到该异常，这就是异常的传递
> * 意味着我们不需要知道确切的代码问题出现在哪，**只要存在调用关系，可以直接对最高层级进行捕获**

```python
def func01():
    num = 1 / 0   # 除0异常
def func02():
    func01()
def main():
    try:
        func02()
    except Exception as e:   # 接收全部异常
        print(e)
main()
```

## 03 python模块

> * 模块（Module）是一个.py结尾的文件，里面定义了函数，类，变量等等，可以导入模块到当前代码中进行使用
> * 模块里写的方法都可以以，模块.方法() 这样的形式进行调用（个人理解）

```python
# 导入方式
import 模块名
from 模块名 import 类、变量、方法等
from 模块名 import *   # 这个 * 表示导入模块内所有的方法
import 模块名 as 别名
from 模块名 import 功能名 as 别名
```

```python
# 通过import导入模块 这里是导入了整个模块
import time # time是python内置的模块
print("早上好，加载中……")
time.sleep(6)   # 通过 . 就可以使用模块内全部的功能（函数、类、变量等）
print("加载成功")

# 通过from导入    就不是把整个模块都导入进来，只是导入具体的功能
# 导入time模块里的sleep方法
from time import sleep
print("早上好，加载中……")
sleep(6)   # 因为导入的是具体的sleep()方法，所以不用再 . 调用了
print("加载成功")

# 使用 * 导入time全部功能
from time import *   # 这里可以看做就是一个个导入了全部功能
print("早上好，加载中……")
sleep(6)   # 所以可以这样写
print("加载成功")

# 使用as个特定功能取个别名
# 因为有些方法名字很长，每次使用很麻烦，所以可以自定义一个简短的名字
import time as t
from time import sleep as sl
```

## 04 自定义模块

> * 模块和文件要放在同一个层级之内
> * 当调入的多个模块里面的方法同名时，后面写的会覆盖前面的，也就是调出来的是后面的
```python
# my_module.py
def func1(a, b):
	print(a + b)

# test.py
import my_module as mod
sum = mod.func1(32,6534)
print(sum)
# 或者
# from my_module import func1
# func1(32,6534)

# 调入多个模块且方法重名
from module1 import func1
from modeul2 import func1
# 此时func1会识别为第二条调入，覆盖了第一条的
```

## 05 \__main__变量

> * 在编写完一个模块之后，我们会在最后尝试运行模块，以检查是否达到效果。这样的话，当我们调用该模块时，便会重复执行。因此有以下解决方法
> * 当我们在模块页面运行模块时，python会将内置变量`__name__`识别为`__main__`,而如果是在其他代码中调用该模块，则不会识别成`__main__`，因而可以通过一个if判断来解决上述问题

```python
# module1.py
def test(a, b):
    print(a + b)
# 只在当前文件中会调用该函数，才会执行if里的。当在其他代码中调用时，不会执行if里的
if __name__ == "__main__":
    test(1,2)   # 只有在当前模块页执行代码时才会进行函数测试
```

## 06 \__all__变量

> * 若模块中有`__all__`变量，当使用from xxx import * 导入时，，只能导入这个列表中的元素

```python
__all__ =["test1","few",……]   # 这是个列表变量
```

## 07 python包

> * python包就是一个文件夹，里面包含了一个`__init__.py`文件，这个文件表明这是一个包而不是其他普通的文件夹，同时里面有许多的模块文件
> * pycharm可以直接new一个package，里面附带了`__init.py`文件，vscode的话，搜一下，繁琐一点点
> * 导入包时，可以在`__init__.py`文件中写`__all__ = [ ]`列为表内写可以导入的模块，这种是对于import * 的写法的
