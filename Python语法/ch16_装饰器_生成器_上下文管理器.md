## 一、装饰器（Decorator）

### 1.1 前置知识：函数是"一等公民"

在 Python 中，函数和整数、字符串一样，是一个**对象**。这意味着函数可以：

- 赋值给变量
- 作为参数传递给另一个函数
- 作为另一个函数的返回值

如果你不理解这一点，装饰器就无从谈起。我们先看几个例子：

```python
# ========== 示例1：函数赋值给变量 ==========
def greet(name):
    return f"你好，{name}！"

# 把函数 greet 赋值给变量 say_hello（注意：没有加括号，不是调用）
say_hello = greet

# 通过新的变量名调用，效果完全一样
print(say_hello("张三"))  # 输出：你好，张三！
print(greet("张三"))      # 输出：你好，张三！

# 它们指向的是同一个函数对象
print(say_hello is greet)  # 输出：True


# ========== 示例2：函数作为参数传递 ==========
def execute(func, value):
    """接收一个函数和一个值，调用该函数并返回结果"""
    return func(value)

def double(x):
    return x * 2

def square(x):
    return x ** 2

# 把 double 函数本身作为参数传进去
print(execute(double, 5))  # 输出：10
# 把 square 函数本身作为参数传进去
print(execute(square, 5))  # 输出：25


# ========== 示例3：函数作为返回值 ==========
def make_greeter(greeting):
    """根据不同的问候语，动态创建并返回一个新函数"""
    def greeter(name):
        return f"{greeting}，{name}！"
    return greeter  # 返回的是函数本身，不是调用结果

hello_greeter = make_greeter("你好")   # 得到一个新函数
hi_greeter = make_greeter("Hi")        # 得到另一个新函数

print(hello_greeter("李四"))  # 输出：你好，李四！
print(hi_greeter("王五"))     # 输出：Hi，王五！
```

> **核心认知**：在 Python 中，函数名（不带括号）就是一个指向函数对象的变量。加了括号 `func()` 才是调用。理解这一点，你就理解了装饰器的基础。

### 1.2 函数装饰器——最基本的形态

#### 大白话解释

想象你买了一个手机壳（装饰器），套在手机（原函数）上。手机的功能没变，但是多了保护、好看等"额外功能"。装饰器就是这样——**它在不修改原函数代码的情况下，给函数增加额外的功能**。

#### 严谨定义

装饰器（Decorator）是一个**接收函数作为参数，并返回一个新函数**的高阶函数。通过 `@` 语法糖，可以在函数定义时自动应用装饰器。

#### 装饰器的运行原理

```python
# ========== 第一步：不用 @ 语法糖，手动理解装饰器的本质 ==========

def my_decorator(func):
    """
    这就是一个装饰器函数。
    参数 func：被装饰的原始函数
    返回值：一个新的 wrapper 函数（包裹了原函数）
    """
    def wrapper(*args, **kwargs):
        # ------- 原函数执行之前的额外操作 -------
        print(f"[日志] 即将调用函数：{func.__name__}")
        
        # ------- 调用原函数，拿到返回值 -------
        result = func(*args, **kwargs)
        
        # ------- 原函数执行之后的额外操作 -------
        print(f"[日志] 函数 {func.__name__} 调用完毕")
        
        # ------- 把原函数的返回值原样返回 -------
        return result
    
    return wrapper  # 注意：返回的是 wrapper 函数本身，不是 wrapper()


def add(a, b):
    """计算两数之和"""
    return a + b


# 手动应用装饰器（不用 @ 语法糖的写法）
# 本质：把 add 传给 my_decorator，得到一个新函数，再赋值回 add
add = my_decorator(add)

# 此时调用 add，实际上调用的是 wrapper
print(add(3, 5))
# 输出：
# [日志] 即将调用函数：add
# 8
# [日志] 函数 add 调用完毕
```

```python
# ========== 第二步：使用 @ 语法糖——等价于上面的手动写法 ==========

def my_decorator(func):
    def wrapper(*args, **kwargs):
        print(f"[日志] 即将调用函数：{func.__name__}")
        result = func(*args, **kwargs)
        print(f"[日志] 函数 {func.__name__} 调用完毕")
        return result
    return wrapper


@my_decorator  # 等价于：add = my_decorator(add)
def add(a, b):
    """计算两数之和"""
    return a + b


# 直接调用，效果和手动写法完全一样
print(add(3, 5))
# 输出：
# [日志] 即将调用函数：add
# 8
# [日志] 函数 add 调用完毕
```

#### 完整实战示例：计时装饰器

在实际开发中，经常需要测量函数的执行时间，这是装饰器最经典的应用之一：

```python
import time

def timer(func):
    """
    计时装饰器：测量被装饰函数的执行时间
    """
    def wrapper(*args, **kwargs):
        start_time = time.time()          # 记录开始时间
        result = func(*args, **kwargs)    # 执行原函数
        end_time = time.time()            # 记录结束时间
        elapsed = end_time - start_time   # 计算耗时
        print(f"[计时] {func.__name__} 耗时 {elapsed:.4f} 秒")
        return result
    return wrapper


@timer
def slow_function():
    """模拟一个耗时操作"""
    time.sleep(1.5)  # 休眠 1.5 秒
    return "操作完成"


result = slow_function()
print(result)
# 输出：
# [计时] slow_function 耗时 1.5012 秒
# 操作完成
```

#### `*args` 和 `**kwargs` 的作用

你可能注意到 `wrapper(*args, **kwargs)` 这个写法。它的作用是**让装饰器能适配任何参数签名的函数**：

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        # *args   接收所有位置参数，打包成元组 (1, 2, 3)
        # **kwargs 接收所有关键字参数，打包成字典 {"name": "张三"}
        print(f"位置参数: {args}")
        print(f"关键字参数: {kwargs}")
        return func(*args, **kwargs)  # 原样传递给原函数
    return wrapper


@my_decorator
def example(a, b, name="匿名"):
    return f"{name}: {a + b}"


print(example(1, 2, name="张三"))
# 输出：
# 位置参数: (1, 2)
# 关键字参数: {'name': '张三'}
# 张三: 3
```

### 1.3 类装饰器——用类来实现装饰器

#### 大白话解释

前面我们用函数来写装饰器。但有时候装饰器需要**维护状态**（比如记录函数被调用了多少次），用类来写更方便，因为类天然有实例属性可以存储状态。

#### 严谨定义

类装饰器是一个**实现了 `__call__` 方法的类**。当用 `@` 语法应用时，类的实例替代了原函数，而每次"调用函数"时，实际上是在调用实例的 `__call__` 方法。

#### 原理拆解

```python
class MyDecorator:
    def __init__(self, func):
        """
        当 @MyDecorator 应用到函数上时，Python 会执行：
        func = MyDecorator(func)
        也就是说，func 变成了 MyDecorator 的一个实例
        """
        self.func = func  # 保存原函数的引用

    def __call__(self, *args, **kwargs):
        """
        当调用 func() 时，因为 func 现在是 MyDecorator 的实例，
        所以会自动调用 __call__ 方法
        """
        print(f"[装饰器] 函数 {self.func.__name__} 被调用了")
        return self.func(*args, **kwargs)


# 使用方式和函数装饰器完全一样
@MyDecorator  # 等价于：greet = MyDecorator(greet)
def greet(name):
    return f"你好，{name}！"


print(greet("张三"))
# 输出：
# [装饰器] 函数 greet 被调用了
# 你好，张三！
```

#### 完整实战示例：调用计数器

```python
class CallCounter:
    """
    类装饰器：记录被装饰函数被调用的次数
    这是类装饰器的典型用途——需要在多次调用之间维护状态
    """
    def __init__(self, func):
        self.func = func
        self.count = 0  # 初始化调用计数为 0

    def __call__(self, *args, **kwargs):
        self.count += 1  # 每次调用时计数加 1
        print(f"[计数器] {self.func.__name__} 已被调用 {self.count} 次")
        return self.func(*args, **kwargs)

    def reset(self):
        """提供一个重置计数的方法"""
        self.count = 0
        print(f"[计数器] {self.func.__name__} 的调用计数已重置")


@CallCounter
def say_hello(name):
    return f"Hello, {name}!"


# 多次调用，观察计数变化
print(say_hello("Alice"))    # 第 1 次
print(say_hello("Bob"))      # 第 2 次
print(say_hello("Charlie"))  # 第 3 次

# 输出：
# [计数器] say_hello 已被调用 1 次
# Hello, Alice!
# [计数器] say_hello 已被调用 2 次
# Hello, Bob!
# [计数器] say_hello 已被调用 3 次
# Hello, Charlie!

# 因为 say_hello 现在是 CallCounter 的实例，可以调用额外方法
say_hello.reset()
# 输出：[计数器] say_hello 的调用计数已重置

print(say_hello("Dave"))
# 输出：
# [计数器] say_hello 已被调用 1 次
# Hello, Dave!
```

> **类装饰器 vs 函数装饰器如何选择？**
> - 如果装饰器是无状态的（纯粹添加功能）→ 用函数装饰器更简洁
> - 如果装饰器需要维护状态（计数、缓存等）→ 用类装饰器更清晰
> - 如果装饰器需要提供额外方法（如 reset）→ 用类装饰器

### 1.4 带参装饰器——装饰器本身也能接收参数

#### 大白话解释

前面的装饰器都是"固定功能"的：打日志就只能打日志，计时就只能计时。但如果我想让装饰器的行为可以配置呢？比如：日志的级别可以选 `INFO` 或 `DEBUG`？这就需要**带参数的装饰器**。

核心思路：**在装饰器外面再包一层函数**，这层函数负责接收参数，然后返回真正的装饰器。

#### 严谨定义

带参装饰器是一个**返回装饰器的函数**（工厂函数）。它接收配置参数，返回一个标准装饰器（接收函数、返回新函数）。因此，带参装饰器实际上有**三层嵌套**。

#### 三层结构详解

```python
def repeat(n):
    """
    第一层：装饰器工厂函数
    作用：接收参数 n（重复次数），返回真正的装饰器
    
    当写 @repeat(3) 时，Python 先执行 repeat(3)，得到中间那层 decorator，
    然后再把 decorator 应用到函数上，即 func = decorator(func)
    """
    def decorator(func):
        """
        第二层：真正的装饰器
        作用：接收被装饰的函数 func，返回包装后的新函数 wrapper
        """
        def wrapper(*args, **kwargs):
            """
            第三层：替代原函数的新函数
            作用：在这里实现具体的装饰逻辑
            """
            results = []
            for i in range(n):  # n 来自最外层的参数
                result = func(*args, **kwargs)
                results.append(result)
                print(f"  第 {i + 1}/{n} 次执行，结果：{result}")
            return results  # 返回所有执行结果的列表
        return wrapper
    return decorator


# 使用带参装饰器
@repeat(3)  # Python 执行流程：repeat(3) → 返回 decorator → decorator(greet) → 返回 wrapper
def greet(name):
    return f"你好，{name}！"


# 调用
result = greet("张三")
# 输出：
#   第 1/3 次执行，结果：你好，张三！
#   第 2/3 次执行，结果：你好，张三！
#   第 3/3 次执行，结果：你好，张三！

print(f"所有结果：{result}")
# 输出：所有结果：['你好，张三！', '你好，张三！', '你好，张三！']
```

#### 完整实战示例：可配置的日志装饰器

```python
import time

def log(level="INFO", show_args=True, show_time=True):
    """
    带参装饰器工厂：可配置日志级别、是否显示参数、是否显示耗时
    
    使用方式：
        @log()                              → 全部默认值
        @log(level="DEBUG")                 → 只改日志级别
        @log(level="WARN", show_args=False) → 改级别 + 隐藏参数
    """
    def decorator(func):
        def wrapper(*args, **kwargs):
            # 构建日志前缀
            prefix = f"[{level}] {func.__name__}"
            
            # 是否显示参数信息
            if show_args:
                args_str = ", ".join(
                    [repr(a) for a in args] +
                    [f"{k}={v!r}" for k, v in kwargs.items()]
                )
                print(f"{prefix} 被调用，参数：({args_str})")
            else:
                print(f"{prefix} 被调用")
            
            # 执行原函数并可选地计时
            start = time.time()
            result = func(*args, **kwargs)
            elapsed = time.time() - start
            
            # 是否显示耗时
            if show_time:
                print(f"{prefix} 执行完毕，耗时 {elapsed:.4f} 秒，返回值：{result!r}")
            else:
                print(f"{prefix} 执行完毕，返回值：{result!r}")
            
            return result
        return wrapper
    return decorator


# ===== 使用示例 =====

@log(level="DEBUG", show_args=True, show_time=True)
def add(a, b):
    """加法函数"""
    return a + b


@log(level="WARN", show_args=False, show_time=False)
def risky_operation():
    """模拟风险操作"""
    return "操作成功"


print(add(10, 20))
# 输出：
# [DEBUG] add 被调用，参数：(10, 20)
# [DEBUG] add 执行完毕，耗时 0.0000 秒，返回值：30
# 30

print()

print(risky_operation())
# 输出：
# [WARN] risky_operation 被调用
# [WARN] risky_operation 执行完毕，返回值：'操作成功'
# 操作成功
```

### 1.5 functools.wraps——装饰器的"善后工作"

#### 大白话解释

装饰器有一个"副作用"——装饰后的函数，名字和文档都变成了 `wrapper` 的，原函数的信息丢失了。`functools.wraps` 就是来解决这个问题的：它把原函数的名称、文档字符串等元信息"复制"到 wrapper 上。

#### 问题演示

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper


@my_decorator
def add(a, b):
    """计算两数之和"""
    return a + b


# 查看函数的元信息——全部变成了 wrapper 的！
print(add.__name__)    # 输出：wrapper（期望：add）
print(add.__doc__)     # 输出：None（期望：计算两数之和）
```

这在调试、日志、自动生成 API 文档等场景中会导致严重问题。

#### 使用 functools.wraps 解决

```python
import functools

def my_decorator(func):
    @functools.wraps(func)  # 关键：加上这一行
    def wrapper(*args, **kwargs):
        """我是 wrapper 的文档，但会被 @wraps 覆盖"""
        return func(*args, **kwargs)
    return wrapper


@my_decorator
def add(a, b):
    """计算两数之和"""
    return a + b


# 现在元信息正确了！
print(add.__name__)    # 输出：add ✓
print(add.__doc__)     # 输出：计算两数之和 ✓

# 额外功能：可以通过 __wrapped__ 访问原始函数
print(add.__wrapped__)        # 输出：<function add at 0x...>
print(add.__wrapped__(1, 2))  # 输出：3（直接调用原函数，跳过装饰器）
```

> **实战规则**：只要你写装饰器，就应该**无条件加上 `@functools.wraps(func)`**。这是 Python 社区的最佳实践，也是面试常考点。

### 1.6 装饰器实战汇总

下面给出一个综合示例，把前面学到的所有知识点融合在一起：

```python
import functools
import time

# ========== 综合实战：带参数的计时 + 日志装饰器 ==========

def monitor(level="INFO", slow_threshold=1.0):
    """
    装饰器工厂：监控函数执行
    
    参数:
        level: 日志级别
        slow_threshold: 慢查询阈值（秒），超过则警告
    """
    def decorator(func):
        @functools.wraps(func)  # 保留原函数元信息
        def wrapper(*args, **kwargs):
            # 记录调用信息
            args_repr = [repr(a) for a in args]
            kwargs_repr = [f"{k}={v!r}" for k, v in kwargs.items()]
            signature = ", ".join(args_repr + kwargs_repr)
            
            print(f"[{level}] 调用 {func.__name__}({signature})")
            
            # 计时执行
            start = time.time()
            try:
                result = func(*args, **kwargs)
                elapsed = time.time() - start
                
                # 判断是否为慢函数
                if elapsed > slow_threshold:
                    print(f"[WARN] ⚠ {func.__name__} 执行耗时 {elapsed:.4f} 秒，超过阈值 {slow_threshold} 秒！")
                else:
                    print(f"[{level}] {func.__name__} 执行成功，耗时 {elapsed:.4f} 秒")
                
                return result
            except Exception as e:
                elapsed = time.time() - start
                print(f"[ERROR] {func.__name__} 执行异常：{type(e).__name__}: {e}，耗时 {elapsed:.4f} 秒")
                raise  # 重新抛出异常，不吞掉
        
        return wrapper
    return decorator


# ===== 使用示例 =====

@monitor(level="DEBUG", slow_threshold=0.5)
def fetch_data(url):
    """模拟从 URL 获取数据"""
    time.sleep(0.3)  # 模拟网络请求
    return {"status": 200, "data": "模拟数据"}


@monitor(level="INFO", slow_threshold=0.1)
def slow_query(sql):
    """模拟一个慢查询"""
    time.sleep(0.5)  # 模拟慢查询
    return [{"id": 1}, {"id": 2}]


# 验证 functools.wraps 是否生效
print(f"函数名: {fetch_data.__name__}")  # 输出：fetch_data ✓
print(f"文档: {fetch_data.__doc__}")      # 输出：模拟从 URL 获取数据 ✓
print()

# 调用测试
fetch_data("https://api.example.com/data")
print()
slow_query("SELECT * FROM big_table")
```

## 二、生成器（Generator）

### 2.1 什么是生成器？为什么需要它？

#### 大白话解释

假设你要读一本 10 万页的书。常规做法（列表）是先把 10 万页全部复印一遍放桌上，然后一页页看。生成器的做法是：你翻一页，它现场给你"生成"一页，看完这页再生成下一页。这样你的桌子上始终只有一页纸——**省内存**。

#### 严谨定义

生成器（Generator）是一种特殊的迭代器。它通过 `yield` 关键字来"暂停"函数执行并产出一个值，下次迭代时从暂停的地方继续执行。生成器是**惰性求值**的——值只在需要时才计算。

#### 列表 vs 生成器的内存对比

```python
import sys

# 方式一：列表——一次性把所有元素全部创建在内存中
numbers_list = [i for i in range(1_000_000)]
print(f"列表占用内存: {sys.getsizeof(numbers_list):,} 字节")  # 约 8,000,056 字节（约 8MB）

# 方式二：生成器——不会一次性创建所有元素
numbers_gen = (i for i in range(1_000_000))
print(f"生成器占用内存: {sys.getsizeof(numbers_gen):,} 字节")  # 约 200 字节！

# 但两者在功能上都能遍历：
# for num in numbers_list: ...
# for num in numbers_gen: ...
```

> 100 万个元素，列表要 8MB，生成器只要 200 字节——差了 **4 万倍**。这就是生成器的价值。

### 2.2 yield 关键字详解

#### yield 的核心机制

`yield` 做了两件事：
1. **产出一个值**（类似 `return`，但不终止函数）
2. **暂停函数执行**，保留当前的所有局部变量和执行位置

下次调用 `next()` 时，从上次暂停的位置继续执行。

```python
def simple_generator():
    """一个最简单的生成器函数"""
    print("--- 开始执行 ---")
    yield 1                          # 第一次暂停，产出 1
    print("--- 第一次恢复后继续 ---")
    yield 2                          # 第二次暂停，产出 2
    print("--- 第二次恢复后继续 ---")
    yield 3                          # 第三次暂停，产出 3
    print("--- 函数即将结束 ---")
    # 函数结束，自动抛出 StopIteration


# 创建生成器对象（注意：此时函数体还没有执行！）
gen = simple_generator()
print(type(gen))  # <class 'generator'>

# 手动逐步调用 next() 来驱动生成器
print(next(gen))
# 输出：
# --- 开始执行 ---
# 1

print(next(gen))
# 输出：
# --- 第一次恢复后继续 ---
# 2

print(next(gen))
# 输出：
# --- 第二次恢复后继续 ---
# 3

# 再调用一次，函数体执行完毕，抛出 StopIteration
try:
    next(gen)
except StopIteration:
    print("生成器已耗尽！")
# 输出：
# --- 函数即将结束 ---
# 生成器已耗尽！
```

#### 用 for 循环遍历生成器

在实际开发中，我们很少手动调用 `next()`，而是直接用 `for` 循环。`for` 循环内部会自动处理 `StopIteration`：

```python
def countdown(n):
    """倒计时生成器"""
    print(f"倒计时开始！从 {n} 开始")
    while n > 0:
        yield n  # 每次循环暂停一次，产出当前的 n
        n -= 1   # 恢复执行后，n 减 1
    print("发射！🚀")


# for 循环自动调用 next()，遇到 StopIteration 自动停止
for count in countdown(5):
    print(f"  {count}...")

# 输出：
# 倒计时开始！从 5 开始
#   5...
#   4...
#   3...
#   2...
#   1...
# 发射！🚀
```

#### 实战示例：逐行读取大文件

这是生成器在实际项目中最常见的用途之一：

```python
def read_large_file(file_path):
```

### 2.3 send()——向生成器"发送"数据

#### 大白话解释

`next()` 只能从生成器往外"拿"数据（单向的）。`send()` 可以在恢复生成器的同时，往生成器里面"塞"一个数据（双向的）。`yield` 表达式的值就是 `send()` 传进来的那个数据。

#### 详细机制

```python
def accumulator():
    """
    累加器生成器：展示 send() 的工作机制
    """
    total = 0
    while True:
        # yield total  → 暂停，产出当前的 total
        # received =   → 恢复时，send() 传入的值赋给 received
        received = yield total
        
        if received is None:
            # 如果用 next() 恢复，send 的值为 None
            print("  [收到 None，跳过累加]")
        else:
            total += received
            print(f"  [收到 {received}，累计 = {total}]")


# 创建并启动生成器
gen = accumulator()

# 第一步：必须先用 next() 或 send(None) "启动"生成器
# 让它运行到第一个 yield 处暂停
initial = next(gen)
print(f"初始值: {initial}")  # 输出：初始值: 0

# 第二步：用 send() 发送数据并获取下一个产出值
result = gen.send(10)
print(f"产出值: {result}")
# 输出：
#   [收到 10，累计 = 10]
#   产出值: 10

result = gen.send(20)
print(f"产出值: {result}")
# 输出：
#   [收到 20，累计 = 30]
#   产出值: 30

result = gen.send(5)
print(f"产出值: {result}")
# 输出：
#   [收到 5，累计 = 35]
#   产出值: 35
```

> **注意**：生成器第一次必须用 `next(gen)` 或 `gen.send(None)` 启动。因为第一次执行时，还没有运行到 `yield` 处，没有地方"接收"你 send 的值。

### 2.4 throw() 和 close()——控制生成器的生命周期

#### throw()——在生成器内部"抛出"异常

```python
def careful_generator():
    """展示 throw() 的用法"""
    try:
        while True:
            try:
                value = yield "等待中..."
                print(f"  收到: {value}")
            except ValueError as e:
                # 捕获通过 throw() 注入的异常
                print(f"  捕获到 ValueError: {e}")
    except GeneratorExit:
        # 当 close() 被调用时触发
        print("  生成器收到关闭信号，正在清理资源...")


gen = careful_generator()
print(next(gen))                        # 输出：等待中...

gen.send("正常数据")                     # 输出：收到: 正常数据
next(gen)                               # 输出：等待中...

# 向生成器内部抛出一个异常
gen.throw(ValueError, "数据格式错误")    # 输出：捕获到 ValueError: 数据格式错误
```

#### close()——优雅地关闭生成器

```python
def resource_generator():
    """展示 close() 如何触发清理工作"""
    print("打开资源（如数据库连接）")
    try:
        while True:
            yield "数据"
    except GeneratorExit:
        # close() 会在 yield 处注入 GeneratorExit 异常
        print("关闭资源（如释放数据库连接）")


gen = resource_generator()
print(next(gen))  # 输出：打开资源 → 数据
print(next(gen))  # 输出：数据

# 手动关闭生成器
gen.close()       # 输出：关闭资源（如释放数据库连接）

# 关闭后再调用 next() 会抛出 StopIteration
try:
    next(gen)
except StopIteration:
    print("生成器已关闭")
```

### 2.5 生成器表达式

#### 大白话解释

列表推导式用 `[]`，生成器表达式用 `()`。前者立即计算并生成列表，后者延迟计算、逐个产出。

```python
# 列表推导式：立即计算，全部存入内存
squares_list = [x ** 2 for x in range(10)]
print(squares_list)       # [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]
print(type(squares_list)) # <class 'list'>

# 生成器表达式：延迟计算，需要时才算
squares_gen = (x ** 2 for x in range(10))
print(squares_gen)        # <generator object <genexpr> at 0x...>
print(type(squares_gen))  # <class 'generator'>

# 逐个取出
print(next(squares_gen))  # 0
print(next(squares_gen))  # 1
print(next(squares_gen))  # 4

# 也可以用 for 循环遍历剩余的
for val in squares_gen:
    print(val, end=" ")   # 9 16 25 36 49 64 81
print()
```

#### 常见用法：配合内置函数

```python
# 求 1 到 100 万的平方和
# 方式一（列表推导式）：先创建 100 万个元素的列表，再求和——浪费内存
total_list = sum([x ** 2 for x in range(1_000_000)])

# 方式二（生成器表达式）：边算边加，内存中始终只有一个值——推荐
total_gen = sum(x ** 2 for x in range(1_000_000))

# 两者结果相同，但方式二内存效率远高于方式一
print(total_gen)  # 333332833333500000

# 其他配合内置函数的例子
numbers = [3, -1, 4, -1, 5, -9, 2, 6]
has_negative = any(x < 0 for x in numbers)        # True，存在负数
all_positive = all(x > 0 for x in numbers)         # False，不全是正数
max_abs = max(abs(x) for x in numbers)             # 9，绝对值最大的
joined = ", ".join(str(x) for x in numbers if x > 0)  # "3, 4, 5, 2, 6"
```

### 2.6 惰性求值（Lazy Evaluation）

#### 大白话解释

"惰性求值"就是"不到万不得已不干活"。对比两种策略：

- **急切求值**（Eager）：先把所有结果算好存起来，等你来拿。
- **惰性求值**（Lazy）：你每来要一个结果，我现场给你算一个。

生成器就是惰性求值的典型实现。

#### 实战示例：管道式数据处理

惰性求值最强大的地方在于可以把多个处理步骤像管道一样连接起来，每个元素流过整个管道而不是一步一步地处理整个数据集：

```python
def read_lines(data):
    """第一步：模拟逐行读取（实际项目中可以是文件/网络流）"""
    for line in data:
        yield line


def filter_non_empty(lines):
    """第二步：过滤掉空行"""
    for line in lines:
        stripped = line.strip()
        if stripped:
            yield stripped


def parse_to_int(lines):
    """第三步：尝试将每行转为整数，跳过无法转换的"""
    for line in lines:
        try:
            yield int(line)
        except ValueError:
            pass  # 跳过非数字行


def filter_positive(numbers):
    """第四步：只保留正数"""
    for num in numbers:
        if num > 0:
            yield num


# 原始数据
raw_data = [
    "42",
    "",
    "  -7  ",
    "hello",
    "  100  ",
    "",
    "  0  ",
    "88",
    "world",
    "-3",
    "  55  ",
]

# 构建生成器管道——注意：此时什么都还没有执行！
step1 = read_lines(raw_data)
step2 = filter_non_empty(step1)
step3 = parse_to_int(step2)
step4 = filter_positive(step3)

# 直到 for 循环（或 list()、next()）驱动时才开始执行
# 每个元素从头到尾流过整个管道，一个一个地处理
result = list(step4)
print(result)  # [42, 100, 88, 55]

# 即使原始数据有几十亿行，内存中也始终只有"当前正在处理的那一个元素"
```

> **惰性求值的三大好处**：
> 1. **省内存**：不需要存储中间结果
> 2. **可处理无限序列**：比如无限产生素数
> 3. **可短路**：如果只需要前 10 个结果，后面的根本不会被计算

## 三、上下文管理器（Context Manager）

### 3.1 从 with 语句说起

#### 大白话解释

你用完图书馆的书要还，用完水龙头要关，用完文件要关闭。但人总会忘，尤其是代码出了异常的时候。上下文管理器就是一个"自动管家"——**不管发生什么，它保证资源最终会被清理**。

#### 严谨定义

上下文管理器是一个实现了 `__enter__()` 和 `__exit__()` 方法的对象。与 `with` 语句配合使用时，`__enter__()` 在进入 `with` 块时执行（获取资源），`__exit__()` 在离开 `with` 块时执行（释放资源），即使发生异常也不例外。

#### 为什么需要上下文管理器？

```python
# ===== 不用 with 的写法——容易出错 =====

f = open("data.txt", "w")
try:
    f.write("Hello, World!")
    # 如果这里抛出异常...
    # result = 1 / 0  # 取消注释试试
finally:
    f.close()  # 必须手动关闭，还得写 try/finally 来保证

# ===== 用 with 的写法——简洁且安全 =====

with open("data.txt", "w") as f:
    f.write("Hello, World!")
    # 无论是否发生异常，离开 with 块时文件会自动关闭
# 到这里，文件已经被自动关闭了
```

---

### 3.2 \_\_enter\_\_ 和 \_\_exit\_\_——手动实现上下文管理器

#### 详细机制

```python
class ManagedResource:
    """
    自定义上下文管理器示例
    
    with ManagedResource() as resource:
        ...
    
    执行流程：
    1. 创建 ManagedResource 实例
    2. 调用实例的 __enter__()，返回值赋给 as 后的变量
    3. 执行 with 块中的代码
    4. 无论是否异常，最终调用实例的 __exit__()
    """
    
    def __init__(self, name):
        self.name = name
        print(f"[{self.name}] __init__: 对象被创建")

    def __enter__(self):
        """
        进入 with 块时被调用
        返回值会赋给 as 后面的变量
        通常在这里完成资源获取（打开文件、连接数据库等）
        """
        print(f"[{self.name}] __enter__: 获取资源")
        return self  # 返回自身（也可以返回其他对象）

    def __exit__(self, exc_type, exc_val, exc_tb):
        """
        离开 with 块时被调用（无论正常退出还是异常退出）
        
        参数：
            exc_type: 异常类型（没有异常则为 None）
            exc_val:  异常值（没有异常则为 None）
            exc_tb:   异常追踪信息（没有异常则为 None）
        
        返回值：
            True  → 表示异常已处理，不再向外传播（吞掉异常）
            False → 表示异常未处理，继续向外传播（默认行为）
        """
        if exc_type is not None:
            print(f"[{self.name}] __exit__: 捕获到异常 {exc_type.__name__}: {exc_val}")
        print(f"[{self.name}] __exit__: 释放资源")
        return False  # 不吞掉异常，让异常正常传播

    def do_work(self):
        print(f"[{self.name}] 正在使用资源...")


# ===== 正常使用 =====
print("=== 正常情况 ===")
with ManagedResource("资源A") as res:
    res.do_work()

# 输出：
# [资源A] __init__: 对象被创建
# [资源A] __enter__: 获取资源
# [资源A] 正在使用资源...
# [资源A] __exit__: 释放资源

print()

# ===== 发生异常时 =====
print("=== 异常情况 ===")
try:
    with ManagedResource("资源B") as res:
        res.do_work()
        raise ValueError("模拟一个错误")  # 抛出异常
except ValueError:
    print("异常被外层捕获")

# 输出：
# [资源B] __init__: 对象被创建
# [资源B] __enter__: 获取资源
# [资源B] 正在使用资源...
# [资源B] __exit__: 捕获到异常 ValueError: 模拟一个错误
# [资源B] __exit__: 释放资源
# 异常被外层捕获
```

#### 实战示例：数据库连接管理器

```python
import sqlite3

class DatabaseConnection:
    """
    数据库连接上下文管理器
    自动管理连接的打开、提交和关闭
    """
    def __init__(self, db_path):
        self.db_path = db_path
        self.connection = None
        self.cursor = None

    def __enter__(self):
        """打开连接并返回游标"""
        self.connection = sqlite3.connect(self.db_path)
        self.cursor = self.connection.cursor()
        print(f"数据库连接已打开: {self.db_path}")
        return self.cursor  # 返回游标给 as 变量

    def __exit__(self, exc_type, exc_val, exc_tb):
        """根据是否有异常决定提交还是回滚，最后关闭连接"""
        if exc_type is not None:
            # 发生异常 → 回滚事务
            self.connection.rollback()
            print(f"事务已回滚（原因: {exc_type.__name__}: {exc_val}）")
        else:
            # 没有异常 → 提交事务
            self.connection.commit()
            print("事务已提交")
        
        # 无论如何都关闭连接
        self.connection.close()
        print("数据库连接已关闭")
        return False  # 不吞掉异常


# 使用示例
# with DatabaseConnection("my_app.db") as cursor:
#     cursor.execute("CREATE TABLE IF NOT EXISTS users (id INTEGER, name TEXT)")
#     cursor.execute("INSERT INTO users VALUES (1, '张三')")
# 离开 with 块后，事务自动提交，连接自动关闭
```

### 3.3 contextlib.contextmanager——用生成器简化写法

#### 大白话解释

每次都写一个类、定义 `__enter__` 和 `__exit__` 方法，有点繁琐。Python 的 `contextlib` 模块提供了一个装饰器 `@contextmanager`，让你用**一个生成器函数**就能实现上下文管理器。

#### 核心规则

用 `@contextmanager` 装饰的生成器函数里，**`yield` 之前的代码相当于 `__enter__`**，**`yield` 之后的代码相当于 `__exit__`**，**`yield` 的值就是 `as` 后面的变量**。

```python
from contextlib import contextmanager

@contextmanager
def managed_resource(name):
    """
    用生成器实现的上下文管理器
    
    yield 之前 = __enter__（获取资源）
    yield 的值 = as 后面的变量
    yield 之后 = __exit__（释放资源）
    """
    # ----- 相当于 __enter__ -----
    print(f"[{name}] 获取资源")
    resource = {"name": name, "status": "active"}
    
    try:
        yield resource  # 把 resource 交给 with 块使用
        # ----- with 块中的代码在这里执行 -----
    except Exception as e:
        print(f"[{name}] 捕获到异常: {e}")
        raise  # 重新抛出（如果想吞掉异常，就不 raise）
    finally:
        # ----- 相当于 __exit__（无论是否异常都执行） -----
        resource["status"] = "closed"
        print(f"[{name}] 资源已释放")


# 使用方式和之前完全一样
with managed_resource("我的资源") as res:
    print(f"使用资源: {res}")
    # 正常使用...

# 输出：
# [我的资源] 获取资源
# 使用资源: {'name': '我的资源', 'status': 'active'}
# [我的资源] 资源已释放
```

## 四、可变与不可变、深浅拷贝

### 4.1 可变对象 vs 不可变对象

#### 大白话解释

- **不可变对象**：一旦创建就不能修改。好比刻在石头上的文字——想改只能换一块新石头。
- **可变对象**：创建后可以随意修改。好比写在白板上的文字——可以擦了重写。

#### 分类表

| 不可变（Immutable） | 可变（Mutable） |
|:---:|:---:|
| `int`, `float`, `bool` | `list` |
| `str` | `dict` |
| `tuple` | `set` |
| `frozenset` | `bytearray` |
| `bytes` | 自定义类的实例（默认） |

#### 代码验证

```python
# ========== 不可变对象示例 ==========

# 整数是不可变的
a = 10
print(f"a = {a}, id(a) = {id(a)}")

a = a + 5  # 这不是"修改" a，而是创建了新整数 15，让 a 指向它
print(f"a = {a}, id(a) = {id(a)}")  # id 变了！说明是新对象

# 字符串是不可变的
s = "hello"
print(f"s = {s}, id(s) = {id(s)}")

s = s + " world"  # 创建了新字符串 "hello world"
print(f"s = {s}, id(s) = {id(s)}")  # id 变了！

# 尝试直接修改字符串会报错
# s[0] = "H"  # TypeError: 'str' object does not support item assignment

# 元组是不可变的
t = (1, 2, 3)
# t[0] = 10  # TypeError: 'tuple' object does not support item assignment


# ========== 可变对象示例 ==========

# 列表是可变的
lst = [1, 2, 3]
print(f"lst = {lst}, id(lst) = {id(lst)}")

lst.append(4)  # 直接在原对象上修改
print(f"lst = {lst}, id(lst) = {id(lst)}")  # id 不变！还是同一个对象

lst[0] = 100  # 直接修改元素
print(f"lst = {lst}, id(lst) = {id(lst)}")  # id 不变！

# 字典是可变的
d = {"name": "张三"}
print(f"d = {d}, id(d) = {id(d)}")

d["age"] = 25  # 直接添加键值对
print(f"d = {d}, id(d) = {id(d)}")  # id 不变！
```

#### 可变对象在函数传参中的"坑"

```python
def add_item(item, target_list=[]):
    """
    ⚠ 经典陷阱：默认参数使用可变对象
    
    默认参数只在函数定义时创建一次！
    后续每次调用共享同一个列表对象，导致"串数据"。
    """
    target_list.append(item)
    return target_list


# 第一次调用
print(add_item("a"))  # ['a'] — 看起来正常
# 第二次调用
print(add_item("b"))  # ['a', 'b'] — 出问题了！"b" 和之前的 "a" 混在一起了
# 第三次调用
print(add_item("c"))  # ['a', 'b', 'c'] — 越来越多！


# ===== 正确写法 =====
def add_item_safe(item, target_list=None):
    """用 None 作为默认值，在函数体内创建新列表"""
    if target_list is None:
        target_list = []  # 每次调用都创建新列表
    target_list.append(item)
    return target_list


print(add_item_safe("a"))  # ['a']
print(add_item_safe("b"))  # ['b'] ✓ 互不影响
print(add_item_safe("c"))  # ['c'] ✓
```

### 4.2 赋值、浅拷贝、深拷贝——三者的本质区别

#### 一句话总结

| 操作 | 效果 |
|:---|:---|
| **赋值**（`=`） | 不拷贝，只是给同一个对象起了一个新名字 |
| **浅拷贝**（`copy.copy()`） | 创建新的外层容器，但内部元素仍然共享 |
| **深拷贝**（`copy.deepcopy()`） | 创建完全独立的副本，内外全部递归复制 |

#### 直观图解

```python
import copy

original = [[1, 2, 3], [4, 5, 6]]

# 赋值：assigned 和 original 是同一个对象
assigned = original

# 浅拷贝：外层是新列表，但内部的 [1,2,3] 和 [4,5,6] 还是原来的
shallow = copy.copy(original)

# 深拷贝：完全独立的全新对象，内外都是新的
deep = copy.deepcopy(original)

# 验证
print(f"assigned is original:          {assigned is original}")           # True
print(f"shallow is original:           {shallow is original}")            # False
print(f"shallow[0] is original[0]:     {shallow[0] is original[0]}")     # True ← 内层共享！
print(f"deep is original:              {deep is original}")               # False
print(f"deep[0] is original[0]:        {deep[0] is original[0]}")        # False ← 内层也独立！
```

### 4.3 copy.copy() 浅拷贝详解

```python
import copy

# ========== 浅拷贝的"共享内层"问题 ==========

original = {
    "name": "张三",
    "scores": [90, 85, 92],          # 内层是可变对象（列表）
    "info": {"city": "北京", "age": 25}  # 内层是可变对象（字典）
}

shallow = copy.copy(original)

# 修改浅拷贝的顶层——不影响原对象
shallow["name"] = "李四"
print(f"original['name']: {original['name']}")  # 张三 ✓ 不受影响
print(f"shallow['name']:  {shallow['name']}")   # 李四

# 修改浅拷贝的内层——原对象也被影响了！
shallow["scores"].append(78)
print(f"original['scores']: {original['scores']}")  # [90, 85, 92, 78] ← 被影响了！
print(f"shallow['scores']:  {shallow['scores']}")   # [90, 85, 92, 78]

shallow["info"]["city"] = "上海"
print(f"original['info']: {original['info']}")  # {'city': '上海', 'age': 25} ← 被影响了！
print(f"shallow['info']:  {shallow['info']}")   # {'city': '上海', 'age': 25}
```

#### 常见的浅拷贝方式

```python
import copy

# 以下方式全部是浅拷贝，效果等同于 copy.copy()

# 列表的浅拷贝
lst = [1, [2, 3], 4]
a = lst.copy()        # 方法一：list.copy()
b = lst[:]            # 方法二：切片
c = list(lst)         # 方法三：构造函数
d = copy.copy(lst)    # 方法四：copy 模块

# 字典的浅拷贝
dic = {"a": 1, "b": [2, 3]}
e = dic.copy()        # 方法一：dict.copy()
f = dict(dic)         # 方法二：构造函数
g = {**dic}           # 方法三：字典解包
h = copy.copy(dic)    # 方法四：copy 模块

# 集合的浅拷贝
s = {1, 2, 3}
i = s.copy()          # 方法一：set.copy()
j = set(s)            # 方法二：构造函数
```

### 4.4 copy.deepcopy() 深拷贝详解

```python
import copy

# ========== 深拷贝：完全独立的副本 ==========

original = {
    "name": "张三",
    "scores": [90, 85, 92],
    "info": {"city": "北京", "age": 25}
}

deep = copy.deepcopy(original)

# 修改深拷贝的内层——原对象完全不受影响
deep["scores"].append(78)
deep["info"]["city"] = "上海"

print(f"original['scores']: {original['scores']}")  # [90, 85, 92] ✓ 不受影响
print(f"deep['scores']:     {deep['scores']}")      # [90, 85, 92, 78]

print(f"original['info']:   {original['info']}")    # {'city': '北京', 'age': 25} ✓ 不受影响
print(f"deep['info']:       {deep['info']}")        # {'city': '上海', 'age': 25}
```

#### 深拷贝能处理循环引用

```python
import copy

# 构造一个循环引用
lst = [1, 2]
lst.append(lst)  # lst 的第三个元素指向 lst 自身！

print(lst)  # [1, 2, [...]]  ← Python 用 [...] 表示循环引用

# copy.deepcopy 能正确处理循环引用
deep_lst = copy.deepcopy(lst)
print(deep_lst)              # [1, 2, [...]]
print(deep_lst[2] is deep_lst)  # True ← 循环引用在副本中也正确重建了
print(deep_lst is lst)          # False ← 但确实是不同的对象
```

### 4.5 实战中的常见坑

```python
import copy

# ===== 坑 1：不可变容器中包含可变元素 =====

t = ([1, 2], [3, 4])  # 元组是不可变的，但内部的列表是可变的！

# 你不能替换元组的元素
# t[0] = [10, 20]  # TypeError

# 但你能修改元组中列表的内容！
t[0].append(999)
print(t)  # ([1, 2, 999], [3, 4]) ← 元组的"不可变"只保护了外层结构


# ===== 坑 2：多个变量引用同一个可变对象 =====

a = [1, 2, 3]
b = a          # b 和 a 指向同一个列表
b.append(4)
print(a)       # [1, 2, 3, 4] ← a 也被改了！

# 如果想让 b 独立，必须拷贝
b = a.copy()   # 浅拷贝（如果列表中有嵌套可变对象，用 deepcopy）
b.append(5)
print(a)       # [1, 2, 3, 4] ← a 不受影响 ✓
print(b)       # [1, 2, 3, 4, 5]


# ===== 坑 3：列表乘法创建重复引用 =====

# 错误方式
matrix = [[0] * 3] * 3  # 看起来是 3x3 矩阵
matrix[0][0] = 1
print(matrix)
# [[1, 0, 0], [1, 0, 0], [1, 0, 0]] ← 三行全变了！
# 因为 * 3 创建的是三个指向同一个列表的引用

# 正确方式
matrix = [[0] * 3 for _ in range(3)]  # 用列表推导式创建独立的行
matrix[0][0] = 1
print(matrix)
# [[1, 0, 0], [0, 0, 0], [0, 0, 0]] ← 只有第一行变了 ✓
```
