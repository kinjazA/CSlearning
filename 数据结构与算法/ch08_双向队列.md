# 第八章：双向队列（Deque）——完全学习笔记

> **核心一句话**：双向队列（Deque）是一种**两端都能进出**的线性数据结构，它同时具备栈和队列的能力——既能后进先出，也能先进先出

---

## 8.1 双向队列基础与常用操作

### 8.1.1 双向队列的基本概念

#### 什么是双向队列？

**双向队列（Deque，全称 Double-Ended Queue）** 是一种两端都允许插入和删除的线性数据结构。读作 "deck"（像扑克牌的"一副"）。

你可以把双向队列想象成：

- 一条**两端都有门**的走廊：人可以从前门进、前门出，也可以从后门进、后门出
- 一个**两头都开口**的管道：东西可以从任意一端塞入或取出

#### 栈、队列、双向队列的关系

```css
栈（Stack）：
    只有一端能操作（后进先出 LIFO）
    ┌───┐
    │   │ ↕ 只能从这端进出
    │   │
    │   │
    └───┘

队列（Queue）：
    一端进，另一端出（先进先出 FIFO）
    入口 →  ┌───────────┐  → 出口
            └───────────┘

双向队列（Deque）：★
    两端都能进出（既能 LIFO，也能 FIFO）
    ↔ 入/出  ┌───────────┐  入/出 ↔
             └───────────┘
    前端(front)            后端(rear)
```

> **关键理解**：双向队列是**栈和队列的超集**——
> - 如果只用一端操作，它就是栈
> - 如果从一端进、另一端出，它就是队列
> - 它比栈和队列都更灵活

#### 关键术语

| 术语 | 含义 |
|------|------|
| **前端（front/head）** | 双向队列的左端 |
| **后端（rear/tail）** | 双向队列的右端 |
| **push_front** | 从前端插入 |
| **push_rear** | 从后端插入 |
| **pop_front** | 从前端删除 |
| **pop_rear** | 从后端删除 |

#### Deque 能模拟栈和队列

```python
# === Deque 当「栈」用 ===
# 只操作后端（或只操作前端）
deque.push_rear(x)    # 入栈
deque.pop_rear()      # 出栈  → 后进先出（LIFO）

# === Deque 当「队列」用 ===
# 后端进，前端出
deque.push_rear(x)    # 入队
deque.pop_front()     # 出队  → 先进先出（FIFO）

# === Deque 的独特能力 ===
# 前端也能插入，后端也能删除——这是栈和队列都做不到的
deque.push_front(x)   # 从前端插入（插队！）
deque.pop_rear()      # 从后端删除
```

### 8.1.2 双向队列的常用操作详解

| 操作 | 作用 | 时间复杂度 | Python deque 对应 |
|------|------|-----------|-------------------|
| `push_front(x)` | 从前端插入 x | $O(1)$     | `deque.appendleft(x)` |
| `push_rear(x)` | 从后端插入 x | $O(1)$ | `deque.append(x)` |
| `pop_front()` | 从前端移除并返回 | $O(1)$ | `deque.popleft()` |
| `pop_rear()` | 从后端移除并返回 | $O(1)$ | `deque.pop()` |
| `peek_front()` | 查看前端元素（不移除） | $O(1)$ | `deque[0]` |
| `peek_rear()` | 查看后端元素（不移除） | $O(1)$ | `deque[-1]` |
| `is_empty()` | 判断是否为空 | $O(1)$ | `len(deque) == 0` |
| `size()` | 获取元素个数 | $O(1)$ | `len(deque)` |

#### 操作图解

```css
初始：空双向队列 []

push_rear(10):    [10]              ← 从后端插入
push_rear(20):    [10, 20]          ← 从后端插入
push_front(5):    [5, 10, 20]       ← 从前端插入（5 插到最前面）
push_rear(30):    [5, 10, 20, 30]   ← 从后端插入

peek_front():     返回 5             ← 看前端
peek_rear():      返回 30            ← 看后端

pop_front():      返回 5,  变为 [10, 20, 30]   ← 前端移除
pop_rear():       返回 30, 变为 [10, 20]        ← 后端移除

size():           2
is_empty():       False
```

#### 与栈、队列操作的对比

| 操作含义 | 栈 | 队列 | 双向队列 |
|---------|---|------|---------|
| 添加元素 | `push` (一端) | `enqueue` (后端) | `push_front` + `push_rear` (两端) |
| 移除元素 | `pop` (同一端) | `dequeue` (前端) | `pop_front` + `pop_rear` (两端) |
| 查看元素 | `peek` (一端) | `peek` (前端) | `peek_front` + `peek_rear` (两端) |
| 操作端数 | 1 端 | 2 端（一进一出） | 2 端（两端都能进出） |

## 8.2 双向队列实现

### 8.2.1 基于双向链表的实现

#### 为什么用双向链表而不是单链表？

回忆一下单链表的结构：

```css
单链表（只有 next 指针）：
  [A] → [B] → [C] → [D] → None

问题：从后端删除时，需要找到 D 的前一个节点 C，
      但单链表只能从头遍历到 C，时间复杂度 O(n)
```

**双向链表**每个节点同时有 `prev`（前驱）和 `next`（后继）指针，可以双向遍历：

```css
双向链表：
  None ← [A] ⇄ [B] ⇄ [C] ⇄ [D] → None
          ↑                     ↑
        head                  tail

优势：从后端删除 D 时，直接通过 D.prev 找到 C，O(1)
```

#### 双向链表节点

```python
class DNode:
    """
    双向链表节点

    与单链表节点的区别：多了一个 prev 指针

    图示：
        ┌──────┬──────┬──────┐
        │ prev │ data │ next │
        └──────┴──────┴──────┘
           ↑               ↓
        前一个节点      后一个节点

    单链表节点（对比）：
        ┌──────┬──────┐
        │ data │ next │      ← 只能往后走
        └──────┴──────┘
    """

    def __init__(self, data: object) -> None:
        self.data: object = data
        self.prev: DNode | None = None   # 指向前一个节点
        self.next: DNode | None = None   # 指向后一个节点
        # 补充知识：
        # "DNode | None" 是 Python 3.10+ 的联合类型写法
        # 意思是：这个变量的类型可以是 DNode，也可以是 None
```

#### 完整实现

```python
class DNode:
    """双向链表节点"""

    def __init__(self, data: object) -> None:
        self.data: object = data
        self.prev: DNode | None = None
        self.next: DNode | None = None


class LinkedDeque:
    """
    基于双向链表实现的双向队列

    内部结构（使用哨兵节点简化边界处理）：

    sentinel_head ⇄ [A] ⇄ [B] ⇄ [C] ⇄ sentinel_tail
         ↑                                    ↑
       哨兵头                                哨兵尾
    （不存真实数据）                      （不存真实数据）

    补充知识：什么是哨兵节点（Sentinel / Dummy Node）？
    哨兵节点是一种编程技巧：在链表的头和尾各放一个"假节点"，
    它们不存储实际数据，只是为了让代码更简洁。
    有了哨兵节点后，插入和删除操作就不需要单独处理"链表为空"
    "只有一个节点"等边界情况了，因为哨兵节点永远存在。

    为什么用哨兵节点？
    不用哨兵：push_front 要判断链表是否为空，为空时要同时更新 head 和 tail
    用哨兵：  push_front 永远是在 sentinel_head 后面插入，逻辑统一

    用法示例：
        dq = LinkedDeque()
        dq.push_rear(10)
        dq.push_front(5)
        print(dq.peek_front())   # 5
        print(dq.pop_rear())     # 10
    """

    def __init__(self) -> None:
        """
        初始化空双向队列

        初始状态：
        sentinel_head ⇄ sentinel_tail
        （两个哨兵互相连接，中间没有真实节点）
        """
        self._sentinel_head: DNode = DNode(None)   # 哨兵头节点
        self._sentinel_tail: DNode = DNode(None)    # 哨兵尾节点

        # 哨兵头尾互相连接
        self._sentinel_head.next = self._sentinel_tail
        self._sentinel_tail.prev = self._sentinel_head

        self._size: int = 0

    def push_front(self, item: object) -> None:
        """
        从前端插入元素

        过程图解（push_front(X)，当前有 A、B）：

        之前：
        SH ⇄ [A] ⇄ [B] ⇄ ST
        （SH = sentinel_head, ST = sentinel_tail）

        步骤：在 SH 和 A 之间插入 X

        之后：
        SH ⇄ [X] ⇄ [A] ⇄ [B] ⇄ ST

        时间复杂度：O(1)
        """
        self._insert_between(item, self._sentinel_head, self._sentinel_head.next)

    def push_rear(self, item: object) -> None:
        """
        从后端插入元素

        过程图解（push_rear(X)，当前有 A、B）：

        之前：
        SH ⇄ [A] ⇄ [B] ⇄ ST

        步骤：在 B 和 ST 之间插入 X

        之后：
        SH ⇄ [A] ⇄ [B] ⇄ [X] ⇄ ST

        时间复杂度：O(1)
        """
        self._insert_between(item, self._sentinel_tail.prev, self._sentinel_tail)

    def pop_front(self) -> object:
        """
        从前端移除并返回元素

        过程图解（pop_front()，当前有 A、B、C）：

        之前：
        SH ⇄ [A] ⇄ [B] ⇄ [C] ⇄ ST

        步骤：删除 SH 后面的第一个真实节点 A

        之后：
        SH ⇄ [B] ⇄ [C] ⇄ ST

        返回 A 的数据

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("pop_front from empty deque")
        return self._remove_node(self._sentinel_head.next)

    def pop_rear(self) -> object:
        """
        从后端移除并返回元素

        过程图解（pop_rear()，当前有 A、B、C）：

        之前：
        SH ⇄ [A] ⇄ [B] ⇄ [C] ⇄ ST

        步骤：删除 ST 前面的最后一个真实节点 C

        之后：
        SH ⇄ [A] ⇄ [B] ⇄ ST

        返回 C 的数据

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("pop_rear from empty deque")
        return self._remove_node(self._sentinel_tail.prev)

    def peek_front(self) -> object:
        """
        查看前端元素（不移除）

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("peek_front from empty deque")
        return self._sentinel_head.next.data

    def peek_rear(self) -> object:
        """
        查看后端元素（不移除）

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("peek_rear from empty deque")
        return self._sentinel_tail.prev.data

    def is_empty(self) -> bool:
        """判断双向队列是否为空"""
        return self._size == 0

    def size(self) -> int:
        """获取元素个数"""
        return self._size

    # ========== 内部辅助方法 ==========

    def _insert_between(self, item: object,
                        predecessor: DNode,
                        successor: DNode) -> None:
        """
        在两个节点之间插入新节点（内部方法）

        参数：
            item:        要插入的数据
            predecessor: 前驱节点（新节点插在它后面）
            successor:   后继节点（新节点插在它前面）

        过程图解：

        之前：  ... ⇄ [predecessor] ⇄ [successor] ⇄ ...

        步骤1：创建新节点 [new_node]
        步骤2：new_node.prev = predecessor
        步骤3：new_node.next = successor
        步骤4：predecessor.next = new_node
        步骤5：successor.prev = new_node

        之后：  ... ⇄ [predecessor] ⇄ [new_node] ⇄ [successor] ⇄ ...

        补充知识：
        方法名以下划线 _ 开头表示"内部方法，不建议外部直接调用"。
        这是 Python 的命名约定，不是强制限制。
        """
        new_node: DNode = DNode(item)

        # 设置新节点的指针
        new_node.prev = predecessor
        new_node.next = successor

        # 更新前驱和后继节点的指针
        predecessor.next = new_node
        successor.prev = new_node

        self._size += 1

    def _remove_node(self, node: DNode) -> object:
        """
        删除指定节点（内部方法）

        过程图解（删除 node B）：

        之前：  ... ⇄ [A] ⇄ [B] ⇄ [C] ⇄ ...

        步骤1：A.next = C    （跳过 B）
        步骤2：C.prev = A    （跳过 B）

        之后：  ... ⇄ [A] ⇄ [C] ⇄ ...

        被删除的 [B] 不再被任何节点引用，Python 会自动回收内存
        """
        predecessor: DNode = node.prev
        successor: DNode = node.next

        predecessor.next = successor
        successor.prev = predecessor

        self._size -= 1

        # 清除被删除节点的指针（帮助垃圾回收，可选但推荐）
        node.prev = None
        node.next = None

        return node.data

    def __str__(self) -> str:
        """打印双向队列内容，从前端到后端"""
        elements: list[str] = []
        current: DNode | None = self._sentinel_head.next
        while current is not self._sentinel_tail:
            elements.append(str(current.data))
            current = current.next
        return "Deque(front [" + " ⇄ ".join(elements) + "] rear)"


# ===== 使用示例 =====
if __name__ == "__main__":
    dq: LinkedDeque = LinkedDeque()

    print("=== 基于双向链表的双向队列 ===\n")

    # 各种入队操作
    print("--- 入队操作 ---")
    dq.push_rear(10)
    print(f"  push_rear(10):  {dq}")
    dq.push_rear(20)
    print(f"  push_rear(20):  {dq}")
    dq.push_front(5)
    print(f"  push_front(5):  {dq}")
    dq.push_front(1)
    print(f"  push_front(1):  {dq}")
    dq.push_rear(30)
    print(f"  push_rear(30):  {dq}")

    # 查看两端
    print(f"\n  peek_front(): {dq.peek_front()}")   # 1
    print(f"  peek_rear():  {dq.peek_rear()}")      # 30
    print(f"  size():       {dq.size()}")            # 5

    # 各种出队操作
    print("\n--- 出队操作 ---")
    print(f"  pop_front(): {dq.pop_front()}, 剩余: {dq}")   # 1
    print(f"  pop_rear():  {dq.pop_rear()},  剩余: {dq}")   # 30
    print(f"  pop_front(): {dq.pop_front()}, 剩余: {dq}")   # 5

    # 当栈使用
    print("\n--- 当栈使用（只操作后端）---")
    dq2: LinkedDeque = LinkedDeque()
    dq2.push_rear(100)      # push
    dq2.push_rear(200)      # push
    dq2.push_rear(300)      # push
    print(f"  push 100, 200, 300: {dq2}")
    print(f"  pop_rear(): {dq2.pop_rear()}")     # 300（后进先出）
    print(f"  pop_rear(): {dq2.pop_rear()}")     # 200
    print(f"  当栈用，LIFO！")

    # 当队列使用
    print("\n--- 当队列使用（后端进，前端出）---")
    dq3: LinkedDeque = LinkedDeque()
    dq3.push_rear(100)      # enqueue
    dq3.push_rear(200)      # enqueue
    dq3.push_rear(300)      # enqueue
    print(f"  enqueue 100, 200, 300: {dq3}")
    print(f"  pop_front(): {dq3.pop_front()}")   # 100（先进先出）
    print(f"  pop_front(): {dq3.pop_front()}")   # 200
    print(f"  当队列用，FIFO！")
```

### 8.2.2 基于环形数组的实现

#### 原理

和队列的环形数组实现类似，但**两端都能移动**：

- `front` 指向队列的前端元素
- 入队后端 → 和普通队列一样，在 `(front + size) % capacity` 处放置元素
- 入队前端 → `front` 需要**往前移动一格**（向左），用取模绕回
- 出队前端 → `front` 往后移动一格（向右）
- 出队后端 → 直接移除最后一个元素

#### 关键公式：前端向左移动

```python
# 往右移动（和队列一样）：
next_right = (index + 1) % capacity

# 往左移动（双向队列新增的）：★
next_left = (index - 1 + capacity) % capacity

# 为什么要 "+ capacity"？
# 因为 Python 的 % 对负数的处理在不同语言中不同
# 加上 capacity 确保结果永远是非负数
# 例如 capacity=5：
#   (0 - 1 + 5) % 5 = 4 % 5 = 4   ← 从索引 0 往左绕到索引 4
#   (3 - 1 + 5) % 5 = 7 % 5 = 2   ← 正常情况
```

#### 环形数组双向操作图解

```css
容量 capacity = 6，初始 front = 0，size = 0

第1步 push_rear(A)：
  rear = (0 + 0) % 6 = 0
  索引：  0     1     2     3     4     5
  数组：[ A  |  _  |  _  |  _  |  _  |  _  ]
         ↑
       front=0, size=1

第2步 push_rear(B)：
  rear = (0 + 1) % 6 = 1
  数组：[ A  |  B  |  _  |  _  |  _  |  _  ]
         ↑
       front=0, size=2

第3步 push_front(X)：★ 前端插入！
  新 front = (0 - 1 + 6) % 6 = 5    ← 绕到末尾！
  数组：[ A  |  B  |  _  |  _  |  _  |  X  ]
                                        ↑
                                     front=5, size=3
  逻辑顺序（前端到后端）：X → A → B

第4步 push_front(Y)：
  新 front = (5 - 1 + 6) % 6 = 4
  数组：[ A  |  B  |  _  |  _  |  Y  |  X  ]
                                  ↑
                               front=4, size=4
  逻辑顺序：Y → X → A → B

第5步 pop_rear()：后端移除，返回 B
  数组：[ A  |  _  |  _  |  _  |  Y  |  X  ]
                                  ↑
                               front=4, size=3
  逻辑顺序：Y → X → A

第6步 pop_front()：前端移除，返回 Y
  新 front = (4 + 1) % 6 = 5
  数组：[ A  |  _  |  _  |  _  |  _  |  X  ]
                                        ↑
                                     front=5, size=2
  逻辑顺序：X → A
```

#### 完整实现

```python
class ArrayDeque:
    """
    基于环形数组实现的双向队列

    核心思路：
    - 用固定大小的数组（list）作为底层存储
    - _front 指向前端元素的索引
    - _size  记录当前元素个数
    - 后端位置通过 (_front + _size) % _capacity 计算
    - 前端插入时 front 往左移（取模绕回）
    - 所有操作均为 O(1)

    用法示例：
        dq = ArrayDeque(8)
        dq.push_rear(10)
        dq.push_front(5)
        print(dq.peek_front())   # 5
        print(dq.pop_rear())     # 10
    """

    def __init__(self, capacity: int = 10) -> None:
        """
        初始化双向队列

        参数：
            capacity: 数组容量（默认 10）
        """
        self._capacity: int = capacity
        self._items: list[object | None] = [None] * capacity
        self._front: int = 0      # 注意这里没有哨兵头节点，这个是直接指向第一个值节点的
        self._size: int = 0

    def push_front(self, item: object) -> None:
        """
        从前端插入元素

        步骤：
        1. front 向左移动一格（取模绕回）
        2. 在新的 front 位置放置元素

        时间复杂度：O(1)
        """
        if self._size == self._capacity:
            raise OverflowError("push_front to full deque")

        # front 向左移一格
        self._front = (self._front - 1 + self._capacity) % self._capacity
        self._items[self._front] = item
        self._size += 1

    def push_rear(self, item: object) -> None:
        """
        从后端插入元素

        步骤：
        1. 计算后端位置 = (front + size) % capacity
        2. 在该位置放置元素

        时间复杂度：O(1)
        """
        if self._size == self._capacity:
            raise OverflowError("push_rear to full deque")

        rear_index: int = (self._front + self._size) % self._capacity
        self._items[rear_index] = item
        self._size += 1

    def pop_front(self) -> object:
        """
        从前端移除并返回元素

        步骤：
        1. 取出 front 位置的元素
        2. front 向右移动一格

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("pop_front from empty deque")

        data: object = self._items[self._front]
        self._items[self._front] = None              # 帮助垃圾回收
        self._front = (self._front + 1) % self._capacity   # front 右移
        self._size -= 1
        return data

    def pop_rear(self) -> object:
        """
        从后端移除并返回元素

        步骤：
        1. 计算最后一个元素的索引 = (front + size - 1) % capacity
        2. 取出该位置的元素

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("pop_rear from empty deque")

        # 最后一个元素的索引
        rear_index: int = (self._front + self._size - 1) % self._capacity
        data: object = self._items[rear_index]
        self._items[rear_index] = None
        self._size -= 1
        return data

    def peek_front(self) -> object:
        """查看前端元素"""
        if self.is_empty():
            raise IndexError("peek_front from empty deque")
        return self._items[self._front]

    def peek_rear(self) -> object:
        """查看后端元素"""
        if self.is_empty():
            raise IndexError("peek_rear from empty deque")
        rear_index: int = (self._front + self._size - 1) % self._capacity
        return self._items[rear_index]

    def is_empty(self) -> bool:
        """判断是否为空"""
        return self._size == 0

    def is_full(self) -> bool:
        """判断是否已满"""
        return self._size == self._capacity

    def size(self) -> int:
        """获取元素个数"""
        return self._size

    def __str__(self) -> str:
        """打印双向队列内容，从前端到后端"""
        if self.is_empty():
            return "ArrayDeque(front [] rear)"

        elements: list[str] = []
        for i in range(self._size):
            index: int = (self._front + i) % self._capacity
            elements.append(str(self._items[index]))
        return "ArrayDeque(front [" + " ⇄ ".join(elements) + "] rear)"

    def debug_array(self) -> str:
        """
        调试用：显示底层数组的真实状态

        这个方法可以帮助你理解环形数组内部的"绕回"现象
        """
        arr_str: str = ", ".join(
            str(x) if x is not None else "_" for x in self._items
        )
        # 补充知识：这是一个生成器表达式
        # 等价于：
        # parts = []
        # for x in self._items:
        #     if x is not None:
        #         parts.append(str(x))
        #     else:
        #         parts.append("_")
        # arr_str = ", ".join(parts)

        return f"  底层数组: [{arr_str}], front={self._front}, size={self._size}"


# ===== 使用示例 =====
```

### 8.2.3 两种实现对比

| 对比维度 | 双向链表实现 | 环形数组实现 |
|---------|------------|-------------|
| **push_front** | $O(1)$ | $O(1)$ |
| **push_rear**  | $O(1)$                                 | $O(1)$ |
| **pop_front** | $O(1)$ | $O(1)$ |
| **pop_rear** | $O(1)$ ← 双向链表优势 | $O(1)$ |
| **空间** | 每个节点 2 个指针（prev + next）开销大 | 可能有空闲槽位 |
| **缓存性能** | 差（内存分散） | 好（连续内存） |
| **容量** | 无限制 | 固定大小 |
| **实现复杂度** | 哨兵节点简化了边界处理 | 取模运算需要仔细 |

> **为什么 pop_rear 特别提到双向链表？**
>
> 用**单链表**实现 deque 时，pop_rear 需要 \(O(n)\)（要遍历到倒数第二个节点）。**双向链表**有 prev 指针，可以直接定位倒数第二个节点，所以 pop_rear 也是 $O(1)$。这就是为什么实现双向队列要用双向链表而不是单链表

### 8.2.4 Python `collections.deque` 详解

在实际开发中，直接使用标准库的 `collections.deque`：

```python
from collections import deque

# ===== 创建 =====
dq: deque[int] = deque()               # 空双向队列
dq2: deque[int] = deque([1, 2, 3])     # 用列表初始化
dq3: deque[int] = deque(maxlen=5)      # 限制最大长度

# ===== 两端操作 =====
dq = deque([10, 20, 30])

# 后端操作
dq.append(40)         # 后端添加   → deque([10, 20, 30, 40])
dq.pop()              # 后端移除   → deque([10, 20, 30])，返回 40

# 前端操作
dq.appendleft(5)      # 前端添加   → deque([5, 10, 20, 30])
dq.popleft()          # 前端移除   → deque([10, 20, 30])，返回 5

# ===== 查看两端 =====
front: int = dq[0]    # 前端元素：10
rear: int = dq[-1]    # 后端元素：30

# ===== 其他实用方法 =====

# rotate(n)：旋转，将后端 n 个元素移到前端
dq = deque([1, 2, 3, 4, 5])
dq.rotate(2)          # deque([4, 5, 1, 2, 3])  ← 后面 2 个移到前面
dq.rotate(-2)         # deque([1, 2, 3, 4, 5])  ← 前面 2 个移到后面
# 补充知识：
# rotate(n>0)：右旋，后端的 n 个元素搬到前端
# rotate(n<0)：左旋，前端的 n 个元素搬到后端
# 就像旋转木马一样

# extend / extendleft：批量添加
dq = deque([1, 2, 3])
dq.extend([4, 5])          # 后端批量添加 → deque([1, 2, 3, 4, 5])
dq.extendleft([0, -1])     # 前端批量添加 → deque([-1, 0, 1, 2, 3, 4, 5])
# 注意：extendleft 会逐个 appendleft，所以顺序是反的！
# extendleft([0, -1]) 等价于先 appendleft(0) 再 appendleft(-1)

# count / remove / reverse
dq = deque([1, 2, 3, 2, 1])
print(dq.count(2))         # 2（出现了 2 次）
dq.remove(3)               # 移除第一个值为 3 的元素 → deque([1, 2, 2, 1])
dq.reverse()               # 原地反转 → deque([1, 2, 2, 1])

# ===== maxlen 自动丢弃特性 =====
# 当 deque 达到 maxlen 时，从一端添加会自动从另一端丢弃
buffer: deque[str] = deque(maxlen=3)
buffer.append("a")     # deque(['a'])
buffer.append("b")     # deque(['a', 'b'])
buffer.append("c")     # deque(['a', 'b', 'c'])  ← 满了
buffer.append("d")     # deque(['b', 'c', 'd'])  ← 'a' 被自动丢弃！

# 这个特性非常适合实现"最近 N 条记录"的缓存

# ===== 线程安全 =====
# deque 的 append、appendleft、pop、popleft 都是原子操作
# 在多线程环境中可以安全使用（不需要加锁）
# 补充知识：
# "原子操作" 意味着操作要么完全执行，要么完全不执行，
# 不会被其他线程中断导致数据损坏
```

#### Python deque 操作方法总览

| 方法 | 作用 | 时间复杂度 |
|------|------|-----------|
| `append(x)` | 后端添加 | $O(1)$ |
| `appendleft(x)` | 前端添加 | $O(1)$ |
| `pop()` | 后端移除 | $O(1)$ |
| `popleft()` | 前端移除 | $O(1)$ |
| `extend(iterable)` | 后端批量添加 | $O(k)$ |
| `extendleft(iterable)` | 前端批量添加（注意顺序反转） | $O(k)$          |
| `rotate(n)` | 旋转 | $O(n)$          |
| `count(x)` | 计数 | $O(n)$ |
| `remove(x)` | 移除第一个匹配 | $O(n)$ |
| `reverse()` | 原地反转 | $O(n)$ |
| `dq[i]` | 随机访问（下标） | $O(n)$ ← 注意！ |
| `len(dq)` | 长度 | $O(1)$ |

> **⚠️ 重要注意**：`deque[i]` 随机访问是 $O(n)$，不像 list 是 $O(1)$
> 如果需要频繁按下标访问，用 `list` 更合适。`deque` 的优势在于**两端操作都是 $O(1)$

## 8.3 双向队列应用

### 8.3.1 滑动窗口最大值（单调双向队列）⭐

这是双向队列最经典的应用！（LeetCode 239）

**问题**：给定数组和窗口大小 k，返回每个窗口中的最大值。

**为什么要用双向队列而不是普通队列？**

因为我们需要：

- **前端出队**：移除滑出窗口的元素 → `popleft()`
- **后端出队**：维护单调性时，移除比新元素小的元素 → `pop()`
- **前端查看**：获取当前最大值 → `dq[0]`

普通队列只能前端出队，而这里还需要后端出队，所以必须用双向队列。

```python
from collections import deque


def max_sliding_window(nums: list[int], k: int) -> list[int]:
    """
    滑动窗口最大值——单调双向队列

    参数：
        nums: 整数数组
        k:    窗口大小

    返回值：
        list[int]: 每个窗口的最大值

    核心思路：
    维护一个单调递减的双向队列（从前到后递减）：
    - 队头 = 当前窗口的最大值的索引
    - 新元素进来时，从队尾踢掉所有比它小的（因为它们不可能再成为最大值）
    - 队头元素滑出窗口时，从队头移除

    为什么存索引不存值？
    因为需要判断元素是否滑出窗口（只有索引能做这个判断）

    图解（nums = [1, 3, -1, -3, 5, 3, 6, 7], k = 3）：

    i=0, num=1:  队列=[0]           队列值=[1]
    i=1, num=3:  1<3，踢掉1
                 队列=[1]           队列值=[3]
    i=2, num=-1: -1<3，直接入队
                 队列=[1,2]         队列值=[3,-1]
                 窗口[1,3,-1] → 最大值=3 ✓

    i=3, num=-3: -3<-1，直接入队
                 队列=[1,2,3]       队列值=[3,-1,-3]
                 窗口[3,-1,-3] → 最大值=3 ✓

    i=4, num=5:  队头1仍在窗口内(4-3+1=2, 1<2)→踢掉队头
                 然后 -3<5，踢掉；-1<5，踢掉；3<5，踢掉
                 队列=[4]           队列值=[5]
                 窗口[-1,-3,5] → 最大值=5 ✓
    ...
    """
    if not nums or k == 0:
        return []

    dq: deque[int] = deque()     # 存储索引的单调递减双向队列
    result: list[int] = []

    for i in range(len(nums)):
        # 步骤1：队头滑出窗口 → 从前端移除（popleft）
        if dq and dq[0] < i - k + 1:
            dq.popleft()                 # ← 双向队列前端操作

        # 步骤2：维护单调递减 → 从后端移除所有比当前值小的（pop）
        while dq and nums[dq[-1]] < nums[i]:
            dq.pop()                     # ← 双向队列后端操作

        # 步骤3：当前索引从后端入队
        dq.append(i)                     # ← 双向队列后端操作

        # 步骤4：窗口形成后，队头就是最大值
        if i >= k - 1:
            result.append(nums[dq[0]])   # ← 双向队列前端查看

    return result


# ===== 测试 =====
print("=== 滑动窗口最大值（双向队列）===\n")

nums: list[int] = [1, 3, -1, -3, 5, 3, 6, 7]
k: int = 3

print(f"  数组: {nums}")
print(f"  窗口大小: {k}")
print(f"  结果: {max_sliding_window(nums, k)}")
# 输出: [3, 3, 5, 5, 6, 7]

# 详细过程
print("\n  --- 详细过程 ---")
dq: deque[int] = deque()
for i in range(len(nums)):
    if dq and dq[0] < i - k + 1:
        removed: int = dq.popleft()
        print(f"  i={i}: 队头索引{removed}滑出窗口，popleft")

    kicked: list[int] = []
    while dq and nums[dq[-1]] < nums[i]:
        kicked.append(nums[dq.pop()])
    if kicked:
        print(f"  i={i}: 从后端踢掉 {kicked}（它们比 {nums[i]} 小）")

    dq.append(i)

    dq_vals: list[int] = [nums[j] for j in dq]
    if i >= k - 1:
        w_start: int = i - k + 1
        window: list[int] = nums[w_start:i + 1]
        print(f"  i={i}: 窗口{window}, 队列值{dq_vals}, 最大值={nums[dq[0]]}")
    else:
        print(f"  i={i}: 窗口未满, 队列值{dq_vals}")
```

### 8.3.2 回文检测

**问题**：判断一个字符串是否是回文（正读反读都一样）。

**为什么用双向队列？** 可以同时从两端取字符进行比较，非常直观。

```python
from collections import deque


def is_palindrome(s: str) -> bool:
    """
    用双向队列判断字符串是否是回文

    思路：
    1. 将字符串的每个字符放入双向队列
    2. 同时从前端和后端取出字符进行比较
    3. 如果所有对应字符都相同，就是回文

    图解（s = "racecar"）：

    初始队列：[r, a, c, e, c, a, r]

    第1轮：pop_front()='r', pop_rear()='r'  → 相同 ✓
           队列: [a, c, e, c, a]
    第2轮：pop_front()='a', pop_rear()='a'  → 相同 ✓
           队列: [c, e, c]
    第3轮：pop_front()='c', pop_rear()='c'  → 相同 ✓
           队列: [e]
    剩余 ≤ 1 个字符 → 回文 ✓

    参数：
        s: 输入字符串

    返回值：
        bool: True 表示是回文
    """

    # 只保留字母和数字，统一转小写
    cleaned: str = ''.join(ch.lower() for ch in s if ch.isalnum())
    # 补充知识：
    # ch.isalnum() 判断字符是否是字母或数字（过滤掉空格、标点等）
    # ch.lower()   将字符转为小写
    # ''.join(...)  将生成器中的字符拼接成字符串
    # 例如 "A man, a plan" → "amanplan"

    dq: deque[str] = deque(cleaned)

    while len(dq) > 1:
        front_char: str = dq.popleft()    # 从前端取
        rear_char: str = dq.pop()         # 从后端取
        if front_char != rear_char:
            return False

    return True


# ===== 测试 =====
print("=== 回文检测（双向队列）===\n")

test_cases: list[tuple[str, bool]] = [
    ("racecar", True),
    ("hello", False),
    ("A man, a plan, a canal: Panama", True),    # 忽略标点和空格
    ("Was it a car or a cat I saw?", True),
    ("", True),                                   # 空字符串是回文
    ("a", True),                                  # 单字符是回文
    ("ab", False),
]

for text, expected in test_cases:
    result: bool = is_palindrome(text)
    status: str = "✓" if result == expected else "✗"
    display: str = text if len(text) <= 35 else text[:32] + "..."
    print(f"  {status} is_palindrome(\"{display}\") = {result}")
```

### 8.3.3 撤销/重做系统

**思路**：用双向队列存储操作历史，支持撤销（从后端弹出）和限制历史长度（满时从前端丢弃最旧的）。

```python
from collections import deque


class UndoRedoManager:
    """
    撤销/重做管理器

    结构：
    - undo_stack (deque): 存储已执行的操作（用 deque 限制历史长度）
    - redo_stack (list):  存储被撤销的操作

    为什么 undo 用 deque 而不用 list？
    因为我们想限制历史记录数量。当历史满了，最旧的操作（前端）
    要被自动丢弃。用 deque(maxlen=N) 就能自动实现。
    """

    def __init__(self, max_history: int = 50) -> None:
        """
        参数：
            max_history: 最大历史记录数
        """
        self._undo_stack: deque[str] = deque(maxlen=max_history)
        self._redo_stack: list[str] = []
        self._max_history: int = max_history

    def execute(self, action: str) -> None:
        """
        执行一个新操作

        新操作会清空 redo 栈（因为进入了新的操作分支）
        """
        self._undo_stack.append(action)
        self._redo_stack.clear()
        print(f"  [执行] {action}")

    def undo(self) -> str | None:
        """
        撤销最近的操作

        返回值：
            被撤销的操作名，如果没有可撤销的操作则返回 None
        """
        if not self._undo_stack:
            print("  [撤销] 无操作可撤销")
            return None

        action: str = self._undo_stack.pop()         # 从后端弹出（最近的操作）
        self._redo_stack.append(action)               # 存入重做栈
        print(f"  [撤销] {action}")
        return action

    def redo(self) -> str | None:
        """
        重做最近撤销的操作

        返回值：
            被重做的操作名，如果没有可重做的操作则返回 None
        """
        if not self._redo_stack:
            print("  [重做] 无操作可重做")
            return None

        action: str = self._redo_stack.pop()
        self._undo_stack.append(action)
        print(f"  [重做] {action}")
        return action

    def history(self) -> list[str]:
        """查看操作历史"""
        return list(self._undo_stack)


# ===== 模拟文本编辑器 =====
print("=== 撤销/重做系统 ===\n")

editor: UndoRedoManager = UndoRedoManager(max_history=5)

# 执行一系列操作
editor.execute("输入 'Hello'")
editor.execute("输入 ' World'")
editor.execute("加粗文字")
editor.execute("修改字体大小")

print(f"\n  历史记录: {editor.history()}")

# 撤销两次
print()
editor.undo()    # 撤销"修改字体大小"
editor.undo()    # 撤销"加粗文字"
print(f"\n  历史记录: {editor.history()}")

# 重做一次
print()
editor.redo()    # 重做"加粗文字"
print(f"\n  历史记录: {editor.history()}")

# 执行新操作（redo 栈被清空）
print()
editor.execute("插入图片")
editor.redo()    # 无法重做（已清空）
print(f"\n  历史记录: {editor.history()}")

# 演示 maxlen 自动丢弃
print("\n--- 演示历史上限（maxlen=5）---")
editor.execute("操作A")
editor.execute("操作B")
editor.execute("操作C")
print(f"  历史记录: {editor.history()}")
# 最旧的操作会被自动丢弃，只保留最近 5 条
```

### 8.3.4 工作窃取调度器（Work Stealing）

**场景**：在多线程/多核编程中，每个线程有自己的任务双向队列。
线程从自己队列的**前端**取任务执行，空闲线程从其他线程队列的**后端**"偷"任务。

```python
from collections import deque


class Worker:
    """
    工作者（模拟一个线程/处理器）

    每个 Worker 有自己的任务双向队列：
    - 自己取任务：从前端取（pop_front）→ 像队列一样
    - 被别人偷任务：从后端取（pop_rear）→ 减少冲突

    为什么两端取而不是都从一端？
    因为在多线程环境中，如果大家都操作同一端，就需要加锁等待。
    自己从前端取、别人从后端偷，两端分开，减少竞争。
    """

    def __init__(self, name: str) -> None:
        self.name: str = name
        self.tasks: deque[str] = deque()

    def add_task(self, task: str) -> None:
        """添加任务到后端"""
        self.tasks.append(task)

    def run_task(self) -> str | None:
        """从前端取一个任务执行"""
        if self.tasks:
            task: str = self.tasks.popleft()       # 自己从前端取
            print(f"    {self.name} 执行: {task}")
            return task
        return None

    def steal_task(self) -> str | None:
        """被别人从后端偷一个任务"""
        if self.tasks:
            task: str = self.tasks.pop()            # 别人从后端偷
            return task
        return None

    def is_idle(self) -> bool:
        """判断是否空闲"""
        return len(self.tasks) == 0


def simulate_work_stealing(workers: list[Worker]) -> None:
    """模拟工作窃取调度"""

    print("  [调度开始]")
    round_num: int = 1

    # 持续调度直到所有任务完成
    while any(not w.is_idle() for w in workers):
        # 补充知识：
        # any() 函数：如果可迭代对象中有任何一个为 True，返回 True
        # any([False, False, True]) → True
        # any([False, False, False]) → False
        # 这里检查是否有任何 worker 还有未完成的任务

        print(f"\n  --- 第{round_num}轮 ---")

        for worker in workers:
            if not worker.is_idle():
                # 有任务就执行
                worker.run_task()
            else:
                # 没任务就尝试偷
                stolen: bool = False
                for other in workers:
                    if other is not worker and len(other.tasks) > 1:
                        # 只在别人有多个任务时才偷（避免偷光）
                        task: str | None = other.steal_task()
                        if task:
                            print(f"    {worker.name} 从 {other.name} 偷到: {task}")
                            worker.tasks.appendleft(task)  # 偷来的任务放前端
                            worker.run_task()
                            stolen = True
                            break
                if not stolen:
                    print(f"    {worker.name} 空闲（无任务可偷）")

        round_num += 1

    print("\n  [所有任务完成]")


# ===== 模拟 =====
print("=== 工作窃取调度器 ===\n")

# 创建 3 个 Worker
w1: Worker = Worker("Worker-1")
w2: Worker = Worker("Worker-2")
w3: Worker = Worker("Worker-3")

# 分配任务（不均匀！Worker-1 任务最多）
for i in range(5):
    w1.add_task(f"任务-A{i + 1}")
w2.add_task("任务-B1")
# Worker-3 没有任务

print(f"  初始任务分配:")
print(f"    Worker-1: {list(w1.tasks)}")
print(f"    Worker-2: {list(w2.tasks)}")
print(f"    Worker-3: {list(w3.tasks)}")

simulate_work_stealing([w1, w2, w3])
```

### 8.3.5 最近 N 条消息缓存

利用 `deque(maxlen=N)` 自动维护固定大小的缓存。

```python
from collections import deque
from datetime import datetime


class MessageCache:
    """
    最近 N 条消息缓存

    利用 deque 的 maxlen 特性：
    当消息数超过上限时，最旧的消息（前端）会被自动丢弃

    这比手动检查长度然后删除要简洁得多：
    ❌ if len(messages) >= max_size: messages.pop(0)  # 手动管理
    ✅ deque(maxlen=max_size)                         # 自动管理
    """

    def __init__(self, max_messages: int = 100) -> None:
        self._messages: deque[dict[str, str]] = deque(maxlen=max_messages)
        # 补充知识：
        # dict[str, str] 表示键和值都是字符串的字典
        # deque(maxlen=100) 表示最多存 100 个元素

    def add_message(self, sender: str, content: str) -> None:
        """添加一条消息"""
        message: dict[str, str] = {
            "sender": sender,
            "content": content,
            "time": datetime.now().strftime("%H:%M:%S")
            # 补充知识：
            # datetime.now() 获取当前时间
            # strftime("%H:%M:%S") 格式化为 "时:分:秒" 字符串
        }
        self._messages.append(message)

    def get_recent(self, n: int = 5) -> list[dict[str, str]]:
        """获取最近 n 条消息"""
        # 补充知识：
        # list(self._messages) 将 deque 转为 list
        # [-n:] 获取最后 n 个元素（切片语法）
        return list(self._messages)[-n:]

    def get_oldest(self) -> dict[str, str] | None:
        """获取最旧的消息（前端）"""
        if self._messages:
            return self._messages[0]       # 前端 = 最旧
        return None

    def get_newest(self) -> dict[str, str] | None:
        """获取最新的消息（后端）"""
        if self._messages:
            return self._messages[-1]      # 后端 = 最新
        return None

    def total(self) -> int:
        """当前缓存的消息总数"""
        return len(self._messages)


# ===== 模拟 =====
print("=== 消息缓存（maxlen 自动丢弃）===\n")

cache: MessageCache = MessageCache(max_messages=5)    # 只保留最近 5 条

# 发送 8 条消息（超过上限，最旧的会被丢弃）
messages: list[tuple[str, str]] = [
    ("小明", "大家好"),
    ("小红", "你好呀"),
    ("小华", "今天天气真好"),
    ("小李", "一起吃饭吧"),
    ("小张", "好的没问题"),       # 第 5 条，缓存满了
    ("小明", "去哪里吃？"),       # 第 6 条，"大家好"被丢弃
    ("小红", "校门口那家如何"),   # 第 7 条，"你好呀"被丢弃
    ("小华", "可以可以"),         # 第 8 条，"今天天气真好"被丢弃
]

for sender, content in messages:
    cache.add_message(sender, content)
    print(f"  [{sender}]: {content}  (缓存数: {cache.total()})")

print(f"\n  最近 3 条消息:")
for msg in cache.get_recent(3):
    print(f"    [{msg['time']}] {msg['sender']}: {msg['content']}")

print(f"\n  最旧的消息: {cache.get_oldest()['content']}")
print(f"  最新的消息: {cache.get_newest()['content']}")
```

### 8.4.1 设计循环双端队列（LeetCode 641）

**问题**：设计一个固定大小的循环双端队列。

```python
class MyCircularDeque:
    """
    LeetCode 641 - 设计循环双端队列

    用环形数组实现，和我们在 8.2.2 节的实现一致
    """

    def __init__(self, k: int) -> None:
        """
        初始化，容量为 k
        """
        self._capacity: int = k
        self._items: list[int] = [0] * k
        self._front: int = 0
        self._size: int = 0

    def insertFront(self, value: int) -> bool:
        """从前端插入，成功返回 True"""
        if self.isFull():
            return False
        self._front = (self._front - 1 + self._capacity) % self._capacity
        self._items[self._front] = value
        self._size += 1
        return True

    def insertLast(self, value: int) -> bool:
        """从后端插入"""
        if self.isFull():
            return False
        rear: int = (self._front + self._size) % self._capacity
        self._items[rear] = value
        self._size += 1
        return True

    def deleteFront(self) -> bool:
        """从前端删除"""
        if self.isEmpty():
            return False
        self._front = (self._front + 1) % self._capacity
        self._size -= 1
        return True

    def deleteLast(self) -> bool:
        """从后端删除"""
        if self.isEmpty():
            return False
        self._size -= 1
        return True

    def getFront(self) -> int:
        """获取前端元素"""
        if self.isEmpty():
            return -1
        return self._items[self._front]

    def getRear(self) -> int:
        """获取后端元素"""
        if self.isEmpty():
            return -1
        rear: int = (self._front + self._size - 1) % self._capacity
        return self._items[rear]

    def isEmpty(self) -> bool:
        return self._size == 0

    def isFull(self) -> bool:
        return self._size == self._capacity


# ===== 测试 =====

print(f"  insertLast(2):  {dq.insertLast(2)}")    # True
print(f"  insertFront(3): {dq.insertFront(3)}")   # True
print(f"  insertFront(4): {dq.insertFront(4)}")   # False（满了）
print(f"  getRear():      {dq.getRear()}")         # 2
print(f"  isFull():       {dq.isFull()}")          # True
print(f"  deleteLast():   {dq.deleteLast()}")      # True
print(f"  insertFront(4): {dq.insertFront(4)}")   # True
print(f"  getFront():     {dq.getFront()}")        # 4
```

### 8.4.2 滑动窗口中位数（进阶思路）

**问题**（LeetCode 480）：给定数组和窗口大小 k，返回每个窗口的中位数。

这里给出思路框架，帮你理解双向队列在高级问题中的作用：

```python
# 思路说明（伪代码）
#
# 滑动窗口中位数比最大值难，因为中位数需要"有序"信息
#
# 方法1：排序的双向队列（适合小窗口）
# - 维护一个有序的 deque
# - 中位数直接取中间元素
# - 插入时用二分查找找位置：O(log k)
# - 删除滑出元素：O(k)
#
# 方法2：两个堆（面试更常考）
# - 大顶堆存较小的一半
# - 小顶堆存较大的一半
# - 中位数从两个堆顶取
#
# 这里展示方法1的简化版，帮你理解思路

from collections import deque
import bisect
# 补充知识：
# bisect 是 Python 标准库中的二分查找模块
# bisect.insort(list, item) 将 item 插入有序列表的正确位置


def median_sliding_window_simple(nums: list[int], k: int) -> list[float]:
    """
    滑动窗口中位数（简化版，使用排序列表）

    参数：
        nums: 整数数组
        k:    窗口大小

    返回值：
        list[float]: 每个窗口的中位数
    """
    window: list[int] = sorted(nums[:k])    # 初始窗口，排序
    result: list[float] = []

    # 计算中位数的辅助函数
    def get_median(sorted_window: list[int], size: int) -> float:
        """
        获取排序数组的中位数

        补充知识：
        - 如果元素个数是奇数，中位数是中间那个
        - 如果元素个数是偶数，中位数是中间两个的平均值
        """
        if size % 2 == 1:                               # 奇数个
            return float(sorted_window[size // 2])
            # 补充知识：// 是整除运算符
            # 5 // 2 = 2, 7 // 2 = 3
        else:                                            # 偶数个
            mid: int = size // 2
            return (sorted_window[mid - 1] + sorted_window[mid]) / 2.0

    result.append(get_median(window, k))

    for i in range(k, len(nums)):
        # 移除滑出窗口的元素
        old_val: int = nums[i - k]
        window.remove(old_val)       # O(k)

        # 插入新元素（保持有序）
        bisect.insort(window, nums[i])    # O(k)

        result.append(get_median(window, k))

    return result


# ===== 测试 =====
print("=== 滑动窗口中位数 ===\n")
nums: list[int] = [1, 3, -1, -3, 5, 3, 6, 7]
k: int = 3
result: list[float] = median_sliding_window_simple(nums, k)

print(f"  数组: {nums}")
print(f"  窗口大小: {k}")
print(f"  中位数序列: {result}")
# 输出: [1.0, -1.0, -1.0, 3.0, 5.0, 6.0]

# 验证过程
for i in range(len(nums) - k + 1):
    window: list[int] = sorted(nums[i:i + k])
    print(f"  窗口 {nums[i:i+k]} → 排序 {window} → 中位数 {result[i]}")
```

## 8.5 易错点总结

### 错误 1：混淆操作方向

```python
from collections import deque

dq: deque[int] = deque([1, 2, 3])

# ❌ 想从前端添加，却用了 append（后端添加）
dq.append(0)          # [1, 2, 3, 0]  ← 0 跑到后面去了！

# ✅ 前端添加要用 appendleft
dq.appendleft(0)      # [0, 1, 2, 3]  ← 0 在最前面

# ❌ 想从后端移除，却用了 popleft（前端移除）
dq.popleft()          # 移除了前端的元素！

# ✅ 后端移除要用 pop
dq.pop()              # 移除后端元素
```

**记忆口诀**：

```
append     = 后端添加（默认方向）
appendleft = 前端添加（左 = 前）
pop        = 后端移除（默认方向）
popleft    = 前端移除（左 = 前）
```

### 错误 2：extendleft 的顺序"反了"

```python
from collections import deque

dq: deque[int] = deque([3, 4, 5])

# ❌ 以为 extendleft([1, 2]) 会变成 [1, 2, 3, 4, 5]
dq.extendleft([1, 2])
print(dq)    # deque([2, 1, 3, 4, 5])  ← 顺序反了！

# 原因：extendleft 是逐个 appendleft
# 先 appendleft(1) → [1, 3, 4, 5]
# 再 appendleft(2) → [2, 1, 3, 4, 5]

# ✅ 如果想保持顺序，先 reverse 再 extendleft
dq2: deque[int] = deque([3, 4, 5])
dq2.extendleft(reversed([1, 2]))    # → deque([1, 2, 3, 4, 5])
```

### 错误 3：环形数组 front 向左移动忘记加 capacity

```python
capacity: int = 6

# ❌ 错误：直接 -1，可能得到负数
front: int = 0
front = (front - 1) % capacity       # Python 中 -1 % 6 = 5，碰巧对了
# 但在 C/Java 中 -1 % 6 = -1，会出 bug！

# ✅ 正确：加上 capacity 确保非负（跨语言通用写法）
front = (front - 1 + capacity) % capacity    # (0-1+6) % 6 = 5 ✓
```

### 错误 4：双向链表删除时指针遗漏

```python
# ❌ 错误：只更新了一个方向的指针
def bad_remove(node: DNode) -> None:
    node.prev.next = node.next     # 前驱跳过 node → ✓
    # 忘记更新后继的 prev！
    # node.next.prev 还指向被删除的 node → 💥

# ✅ 正确：两个方向的指针都要更新
def good_remove(node: DNode) -> None:
    node.prev.next = node.next     # 前驱的 next 跳过 node
    node.next.prev = node.prev     # 后继的 prev 跳过 node
```

### 错误 5：deque 随机访问的性能陷阱

```python
from collections import deque

dq: deque[int] = deque(range(100000))

# ❌ 频繁用下标访问 → O(n) 每次，总共 O(n²)
total: int = 0
for i in range(len(dq)):
    total += dq[i]         # 每次 O(n)！非常慢

# ✅ 如果需要遍历，直接用 for-in → O(n) 总共
total = 0
for val in dq:             # 迭代是 O(1) 每步
    total += val

# ✅ 如果需要频繁随机访问，用 list 而不是 deque
my_list: list[int] = list(range(100000))
val: int = my_list[50000]   # O(1)
```

### 错误 6：空 deque 操作

```python
from collections import deque

dq: deque = deque()

# ❌ 空 deque 的各种操作都会报错
# dq.pop()         → IndexError
# dq.popleft()     → IndexError
# dq[0]            → IndexError
# dq[-1]           → IndexError

# ✅ 先检查再操作
if dq:                        # 非空才操作
    val = dq.popleft()

# ✅ 或者用 try-except
try:
    val = dq.popleft()
except IndexError:
    print("双向队列为空")
```

## 8.6 总结

### 核心要点速查表

```
双向队列（Deque）= 两端都能进出的线性表
全称：Double-Ended Queue，读作 "deck"

它是栈和队列的超集：
├── 只操作一端  → 当栈用（LIFO）
├── 一端进另一端出 → 当队列用（FIFO）
└── 两端都操作  → 双向队列独有能力

八大操作（全部 O(1)）：
├── push_front(x)  → 前端插入    deque.appendleft(x)
├── push_rear(x)   → 后端插入    deque.append(x)
├── pop_front()    → 前端移除    deque.popleft()
├── pop_rear()     → 后端移除    deque.pop()
├── peek_front()   → 查看前端    deque[0]
├── peek_rear()    → 查看后端    deque[-1]
├── is_empty()     → 判空        len(deque) == 0
└── size()         → 大小        len(deque)

两种实现：
├── 双向链表：每个节点有 prev + next，哨兵节点简化边界
└── 环形数组：左移公式 (i-1+cap)%cap，右移 (i+1)%cap

Python 推荐用法：
    from collections import deque
    dq = deque()            # 无限大小
    dq = deque(maxlen=100)  # 限制大小，自动丢弃最旧

经典应用：
├── 滑动窗口最大值   → 单调递减双向队列，O(n)
├── 回文检测         → 两端同时取字符比较
├── 撤销/重做系统    → maxlen 限制历史 + 后端操作
├── 工作窃取调度     → 自己从前端取，别人从后端偷
└── 消息缓存         → maxlen 自动管理固定大小缓存

⚠️ 注意 deque[i] 随机访问是 O(n)，不是 O(1)！
   频繁下标访问请用 list。
```

### 学习建议

1. **先理解"两端操作"的核心概念**：对比栈（一端）、队列（两端但各有分工）、双向队列（两端完全自由）
2. **双向链表实现重点理解哨兵节点**：它让代码变得非常统一优雅
3. **环形数组实现重点理解左移公式**：`(i - 1 + capacity) % capacity`
4. **滑动窗口最大值是必考题**：理解为什么要用双向队列（因为需要两端都能出队）
5. **实际开发直接用 `collections.deque`**：它的 `maxlen` 特性非常实用
6. **刷题推荐**：LeetCode 239（滑动窗口最大值）、641（设计循环双端队列）

### 栈、队列、双向队列知识图谱

```
线性数据结构（受限操作）
│
├── 栈（Stack）
│   ├── 特点：LIFO（后进先出）
│   ├── 操作端：1 端
│   └── 经典应用：括号匹配、DFS、调用栈
│
├── 队列（Queue）
│   ├── 特点：FIFO（先进先出）
│   ├── 操作端：2 端（一进一出）
│   └── 经典应用：BFS、层序遍历、任务调度
│
└── 双向队列（Deque）★ 本章
    ├── 特点：两端都能进出
    ├── 操作端：2 端（两端都能进出）
    ├── 是栈和队列的超集
    └── 经典应用：滑动窗口、回文检测、工作窃取
```
