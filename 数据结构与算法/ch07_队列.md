> **核心一句话**：队列是一种 **先进先出（FIFO）** 的线性数据结构，就像排队买奶茶——先排队的人先买到

## 7.1 队列基础与常用操作

### 7.1.1 队列的基本概念

#### 什么是队列？

**队列（Queue）** 是一种受限的线性表，只允许在**一端插入**（称为 **队尾 rear/tail**），在**另一端删除**（称为 **队头 front/head**）

你可以把队列想象成：

- 奶茶店排队：先来的人先点单，后来的人排在后面
- 打印机任务：先提交的文档先打印
- 客服电话排队：先打进来的电话先被接听

#### 队列的核心特点：FIFO

**FIFO = First In, First Out（先进先出）**

```css
入队顺序：A → B → C → D
出队顺序：A → B → C → D    ← 最先进去的 A 最先出来
```

图示：

```css
入队（enqueue）        出队（dequeue）
    ↓                      ↓
  ┌───┬───┬───┬───┐
  │ D │ C │ B │ A │  →  A 先出去
  └───┴───┴───┴───┘
  队尾              队头
 (rear)           (front)
```

#### 队列 vs 栈：一张图看清区别

```css
栈（Stack）—— LIFO（后进先出）：
    只有一个口 ↕
    ┌───┐
    │ C │  ← 最后进，最先出
    │ B │
    │ A │  ← 最先进，最后出
    └───┘

队列（Queue）—— FIFO（先进先出）：
    入口 →  ┌───┬───┬───┐  → 出口
            │ C │ B │ A │
            └───┴───┴───┘
    队尾                队头
    最后进              最先出
```

#### 关键术语

| 术语 | 含义 |
|------|------|
| **队头（front/head）** | 队列中允许**删除**的一端，即最先入队的元素所在位置 |
| **队尾（rear/tail）** | 队列中允许**插入**的一端，即最后入队的元素所在位置 |
| **入队（enqueue）** | 在队尾插入一个元素 |
| **出队（dequeue）** | 从队头移除一个元素 |
| **空队列** | 队列中没有任何元素 |

#### 队列的抽象数据类型（ADT）

> **复习**：ADT（Abstract Data Type）= 只定义"做什么"，不管"怎么做"。

```css
ADT Queue:
    数据：零个或多个元素的有序集合
    操作：
        enqueue(x)  — 将元素 x 添加到队尾
        dequeue()   — 移除并返回队头元素
        peek()      — 返回队头元素（不移除）
        is_empty()  — 判断队列是否为空
        size()      — 返回队列中元素个数
```

### 7.1.2 队列的常用操作详解

| 操作 | 作用 | 时间复杂度（目标） |
|------|------|-------------------|
| `enqueue(x)` | 入队：将 x 添加到队尾 | $O(1)$ |
| `dequeue()` | 出队：移除并返回队头元素 | $O(1)$ |
| `peek()` / `front()` | 查看队头元素（不移除） | $O(1)$ |
| `is_empty()` | 判断队列是否为空 | $O(1)$ |
| `size()` | 返回队列的元素个数 | $O(1)$ |

#### 操作图解

```css
初始：空队列 []

enqueue(10):  [10]              ← 10 从队尾入队
enqueue(20):  [10, 20]          ← 20 从队尾入队
enqueue(30):  [10, 20, 30]      ← 30 从队尾入队
peek():       返回 10            ← 查看队头（不移除）
dequeue():    返回 10, 队列变为 [20, 30]  ← 队头出队
dequeue():    返回 20, 队列变为 [30]
is_empty():   False
size():       1
dequeue():    返回 30, 队列变为 []
is_empty():   True
```

## 7.2 队列实现

### 7.2.1 基于链表的实现

#### 原理

- 使用**单链表**作为底层存储
- **链表头节点**作为队头（方便 $O(1)$ 删除）
- **链表尾节点**作为队尾（用一个 `_rear` 指针记录，方便 $O(1)$插入）
- 入队 = 在链表尾部追加节点
- 出队 = 删除链表头部节点

```css
  出队在这端                     入队在这端
    ↓                               ↓
  front                            rear
    ↓                               ↓
  [ A ] → [ B ] → [ C ] → [ D ] → None
  队头                               队尾
这里我一直搞混，队列里的队头，对应的是链表里的头节点，队列的队尾，对应的是链表里的尾节点
```

#### 为什么需要一个 `_rear` 指针？

如果只有 `_front` 指针，每次入队时需要从头遍历到尾部才能添加新节点，时间复杂度是$ O(n)$，有了 `_rear` 指针，直接在尾部追加即可，时间复杂度降为 $ O(n)$

#### 完整实现

```python
class Node:
    """
    链表节点类

    属性：
        data: 节点存储的数据
        next: 指向下一个节点的引用（默认 None）

    图示：
        ┌──────┬──────┐
        │ data │ next │ ──→ 下一个节点（或 None）
        └──────┴──────┘
    """

    def __init__(self, data: object) -> None:
        self.data: object = data     # 节点存储的数据
        self.next: Node | None = None  # 指向下一个节点
        # 补充知识：
        # "Node | None" 是类型注解，表示 next 的类型是 Node 或者 None
        # 在旧版本中写作 Optional[Node]（需要 from typing import Optional）


class LinkedQueue:
    """
    基于单链表实现的队列

    核心思路：
    - _front 指向队头节点（出队端）
    - _rear  指向队尾节点（入队端）
    - enqueue = 在尾部追加新节点
    - dequeue = 删除头部节点

    用法示例：
        q = LinkedQueue()
        q.enqueue(10)
        q.enqueue(20)
        print(q.peek())      # 10
        print(q.dequeue())   # 10
    """

    def __init__(self) -> None:
        """初始化空队列"""
        self._front: Node | None = None   # 队头指针
        self._rear: Node | None = None    # 队尾指针
        self._size: int = 0               # 队列长度

    def enqueue(self, item: object) -> None:
        """
        入队：在队尾添加新元素

        过程图解（enqueue(30)，当前队列有 A → B）：

        之前：  front → [A] → [B] → None
                                ↑
                              rear

        步骤1：创建新节点 [30]
        步骤2：让当前 rear 的 next 指向新节点
                front → [A] → [B] → [30] → None
        步骤3：更新 rear 指向新节点
                front → [A] → [B] → [30] → None
                                       ↑
                                     rear

        时间复杂度：O(1)
        """
        new_node: Node = Node(item)       # 步骤1：创建新节点

        if self.is_empty():
            # 特殊情况：队列为空时，front 和 rear 都指向新节点
            self._front = new_node
            self._rear = new_node
        else:
            # 正常情况：在尾部追加
            self._rear.next = new_node    # 步骤2：旧队尾指向新节点
            self._rear = new_node         # 步骤3：更新队尾指针

        self._size += 1

    def dequeue(self) -> object:
        """
        出队：移除并返回队头元素

        过程图解（dequeue()，当前队列 A → B → C）：

        之前：  front → [A] → [B] → [C] → None
                                       ↑
                                     rear

        步骤1：保存队头数据 data = A
        步骤2：front 指向下一个节点
                        front → [B] → [C] → None
                                        ↑
                                      rear
        （被删除的 [A] 节点没有引用了，Python 自动回收内存）

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("dequeue from empty queue")

        data: object = self._front.data      # 步骤1：保存队头数据
        self._front = self._front.next       # 步骤2：队头指向下一个节点
        self._size -= 1

        if self.is_empty():
            # 如果队列变空了，rear 也要设为 None
            self._rear = None

        return data

    def peek(self) -> object:
        """
        查看队头元素（不移除）

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("peek from empty queue")
        return self._front.data

    def is_empty(self) -> bool:
        """
        判断队列是否为空

        时间复杂度：O(1)
        """
        return self._size == 0

    def size(self) -> int:
        """
        获取队列中元素个数

        时间复杂度：O(1)
        """
        return self._size

    def __str__(self) -> str:
        """
        打印队列内容，从队头到队尾

        补充知识：
        - __str__ 是特殊方法（也叫魔术方法 / dunder method）
        - 当你用 print() 打印对象时会自动调用
        - 返回一个方便人类阅读的字符串
        """
        elements: list[str] = []
        current: Node | None = self._front
        while current is not None:
            elements.append(str(current.data))
            current = current.next
        return "Queue(front → [" + " → ".join(elements) + "] ← rear)"


# ===== 使用示例 =====
if __name__ == "__main__":
    q: LinkedQueue = LinkedQueue()

    print("=== 基于链表的队列 ===\n")

    # 入队
    print("--- 入队操作 ---")
    for val in [10, 20, 30, 40]:
        q.enqueue(val)
        print(f"  enqueue({val}): {q}")

    # 查看队头
    print(f"\n  peek(): {q.peek()}")       # 10（最先入队的）
    print(f"  size(): {q.size()}")         # 4

    # 出队
    print("\n--- 出队操作 ---")
    while not q.is_empty():
        val = q.dequeue()
        print(f"  dequeue(): {val}, 剩余: {q}")

    print(f"\n  is_empty(): {q.is_empty()}")  # True

    # 异常处理演示
    print("\n--- 异常处理演示 ---")
    try:
        q.dequeue()
    except IndexError as e:
        print(f"  捕获异常：{e}")
```

**运行输出：**

```css
=== 基于链表的队列 ===

--- 入队操作 ---
  enqueue(10): Queue(front → [10] ← rear)
  enqueue(20): Queue(front → [10 → 20] ← rear)
  enqueue(30): Queue(front → [10 → 20 → 30] ← rear)
  enqueue(40): Queue(front → [10 → 20 → 30 → 40] ← rear)

  peek(): 10
  size(): 4

--- 出队操作 ---
  dequeue(): 10, 剩余: Queue(front → [20 → 30 → 40] ← rear)
  dequeue(): 20, 剩余: Queue(front → [30 → 40] ← rear)
  dequeue(): 30, 剩余: Queue(front → [40] ← rear)
  dequeue(): 40, 剩余: Queue(front → [] ← rear)

  is_empty(): True

--- 异常处理演示 ---
  捕获异常：dequeue from empty queue
```

### 7.2.2 基于数组的实现

#### 问题：朴素数组实现的缺陷

如果直接用 Python `list` 来实现队列，最自然的想法是：

```python
# ❌ 朴素实现（有性能问题）
queue = [10, 20, 30, 40]

# 入队：在末尾添加 → O(1)，没问题
queue.append(50)

# 出队：从头部删除 → O(n)！问题在这里！
queue.pop(0)   # 删除第一个元素后，后面所有元素都要前移
```

为什么 `pop(0)` 是$ O(n)$？因为 list 的底层是连续数组，删除头部后要把剩余元素全部向前移动一位：

```css
删除前：  [10, 20, 30, 40, 50]
           ↑ 删除这个

删除后：  [20, 30, 40, 50]
           ←  ←  ←  ←        全部前移，O(n)！
```

#### 解决方案：环形数组（Circular Array）

**核心思想**：不真正删除元素，而是移动 `front` 指针。当指针走到数组末尾时，"绕回"到数组开头，形成一个逻辑上的"环"

==这里我理解环形并不是物理结构上的连接，不是像环形链表那样。而是通过两个指针，当到了之前的最大size时，再绕回到size0==

```css
物理上是一条线：
  [ _, _, C, D, E, _, _, _ ]
          ↑        ↑
        front    rear

逻辑上是一个环：
         ┌───┐
       ┌─│ E │←─┐
       │ └───┘   │
     ┌───┐     ┌───┐
     │ _ │     │ D │
     └───┘     └───┘
     ┌───┐     ┌───┐
     │ _ │     │ C │ ← front
     └───┘     └───┘
       │ ┌───┐   │
       └─│ _ │──┘
         └───┘
```

#### 关键公式：取模运算

```python
# 取模运算（%）实现"绕回"
# 补充知识：% 是取余运算符
# 7 % 5 = 2    （7 除以 5，余数是 2）
# 10 % 5 = 0   （10 除以 5，余数是 0，正好绕回起点）

# 环形数组中的"下一个位置"：
next_index = (current_index + 1) % capacity

# 例如，容量为 5 的环形数组：
# 当前位置 0 → 下一个位置 (0+1) % 5 = 1
# 当前位置 1 → 下一个位置 (1+1) % 5 = 2
# 当前位置 3 → 下一个位置 (3+1) % 5 = 4
# 当前位置 4 → 下一个位置 (4+1) % 5 = 0  ← 绕回到开头！
```

#### 完整实现

```python
class ArrayQueue:
    """
    基于环形数组实现的队列

    核心思路：
    - 用固定大小的数组（list）作为底层存储
    - _front 指向队头元素的索引
    - _size  记录当前元素个数（用 size 来判断空/满，比用 rear 更简洁）
    - 队尾位置通过 (_front + _size) % _capacity 计算得出
    - 入队/出队都是 O(1)，不需要移动元素

    图示（capacity=6, front=2, size=3）：

    索引：    0     1     2     3     4     5
    数组：  [  _  |  _  |  A  |  B  |  C  |  _  ]
                         ↑                 ↑
                       front              rear（计算得出）
                      (索引2)           (2+3)%6 = 5，但最后一个元素在索引4

    用法示例：
        q = ArrayQueue(5)
        q.enqueue(10)
        q.enqueue(20)
        print(q.peek())      # 10
        print(q.dequeue())   # 10
    """

    def __init__(self, capacity: int = 10) -> None:
        """
        初始化队列

        参数：
            capacity: 队列容量（默认 10）

        补充知识：
        - "capacity: int = 10" 中的 "= 10" 是默认参数值
        - 调用时可以写 ArrayQueue() 使用默认值 10
        - 也可以写 ArrayQueue(20) 指定容量为 20
        """
        self._capacity: int = capacity                    # 数组容量
        self._items: list[object | None] = [None] * capacity  # 初始化数组
        # 补充知识：[None] * 5 → [None, None, None, None, None]
        self._front: int = 0                              # 队头索引
        self._size: int = 0                               # 当前元素个数

    def enqueue(self, item: object) -> None:
        """
        入队：在队尾添加元素

        过程图解（capacity=5, front=1, size=3, 入队元素=X）：

        之前：
        索引：  0     1     2     3     4
        数组：[  _  |  A  |  B  |  C  |  _  ]
                      ↑
                   front=1, size=3
                   队尾位置 = (1+3)%5 = 4

        之后（X 放到索引 4）：
        索引：  0     1     2     3     4
        数组：[  _  |  A  |  B  |  C  |  X  ]
                      ↑
                   front=1, size=4

        时间复杂度：O(1)
        """
        if self._size == self._capacity:
            raise OverflowError("enqueue to full queue")
            # 补充知识：OverflowError 表示超出容量限制

        # 计算队尾位置（利用取模实现环形）
        rear_index: int = (self._front + self._size) % self._capacity
        self._items[rear_index] = item  # 这个item就是入队元素
        self._size += 1

    def dequeue(self) -> object:
        """
        出队：移除并返回队头元素

        过程图解（capacity=5, front=1, size=3）：

        之前：
        索引：  0     1     2     3     4
        数组：[  _  |  A  |  B  |  C  |  _  ]
                      ↑
                   front=1, size=3

        之后（逻辑上移除 A，front 后移）：
        索引：  0     1     2     3     4
        数组：[  _  |  _  |  B  |  C  |  _  ]
                            ↑
                         front=2, size=2

        注意：我们并不真正"删除"索引 1 的数据（设为 None 是为了帮助垃圾回收），
        只是把 front 向后移动了一位。这就是为什么时间复杂度是 O(1)！

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("dequeue from empty queue")

        data: object = self._items[self._front]     # 取出队头元素
        self._items[self._front] = None              # 帮助垃圾回收（可选）
        self._front = (self._front + 1) % self._capacity  # front 后移（环形）
        self._size -= 1
        return data

    def peek(self) -> object:
        """
        查看队头元素（不移除）

        时间复杂度：O(1)
        """
        if self.is_empty():
            raise IndexError("peek from empty queue")
        return self._items[self._front]

    def is_empty(self) -> bool:
        """
        判断队列是否为空

        时间复杂度：O(1)
        """
        return self._size == 0

    def is_full(self) -> bool:
        """
        判断队列是否已满（数组实现特有的操作）

        时间复杂度：O(1)
        """
        return self._size == self._capacity

    def size(self) -> int:
        """
        获取队列中元素个数

        时间复杂度：O(1)
        """
        return self._size

    def __str__(self) -> str:
        """打印队列内容，从队头到队尾"""
        if self.is_empty():
            return "ArrayQueue(front → [] ← rear)"

        elements: list[str] = []
        for i in range(self._size):
            index: int = (self._front + i) % self._capacity
            elements.append(str(self._items[index]))
        return "ArrayQueue(front → [" + " → ".join(elements) + "] ← rear)"


# ===== 使用示例 =====
if __name__ == "__main__":
    q: ArrayQueue = ArrayQueue(5)    # 容量为 5

    print("=== 基于环形数组的队列 ===\n")

    # 入队
    print("--- 入队操作 ---")
    for val in [10, 20, 30, 40]:
        q.enqueue(val)
        print(f"  enqueue({val}): {q}")

    # 查看状态
    print(f"\n  peek(): {q.peek()}")
    print(f"  size(): {q.size()}")
    print(f"  is_full(): {q.is_full()}")     # False（容量5，当前4个）

    # 出队两个
    print("\n--- 出队操作 ---")
    print(f"  dequeue(): {q.dequeue()}")    # 10
    print(f"  dequeue(): {q.dequeue()}")    # 20
    print(f"  当前队列: {q}")               # [30 → 40]

    # 继续入队（验证环形特性）
    print("\n--- 验证环形特性：继续入队 ---")
    for val in [50, 60, 70]:
        q.enqueue(val)
        print(f"  enqueue({val}): {q}")

    print(f"\n  is_full(): {q.is_full()}")  # True

    # 满队列入队测试
    print("\n--- 满队列入队测试 ---")
    try:
        q.enqueue(80)
    except OverflowError as e:
        print(f"  捕获异常：{e}")

    # 全部出队
    print("\n--- 全部出队 ---")
    while not q.is_empty():
        print(f"  dequeue(): {q.dequeue()}")
```

```css
  is_full(): True

--- 满队列入队测试 ---
  捕获异常：enqueue to full queue

--- 全部出队 ---
  dequeue(): 30
  dequeue(): 40
  dequeue(): 50
  dequeue(): 60
  dequeue(): 70
```

#### 环形数组的"绕回"过程详解

这是环形数组最核心也最容易困惑的部分，我们用一个完整的例子走一遍：

```css
容量 capacity = 4

操作序列：enqueue(A), enqueue(B), enqueue(C), dequeue(), dequeue(), enqueue(D), enqueue(E)

第1步 enqueue(A)：
  索引：  0     1     2     3
  数组：[ A  |  _  |  _  |  _  ]
         ↑
       front=0, size=1
       rear = (0+1)%4 = 1（下一个入队位置）

第2步 enqueue(B)：
  数组：[ A  |  B  |  _  |  _  ]
         ↑
       front=0, size=2

第3步 enqueue(C)：
  数组：[ A  |  B  |  C  |  _  ]
         ↑
       front=0, size=3

第4步 dequeue() → 返回 A：
  数组：[ _  |  B  |  C  |  _  ]
               ↑
             front=1, size=2

第5步 dequeue() → 返回 B：
  数组：[ _  |  _  |  C  |  _  ]
                     ↑
                   front=2, size=1

第6步 enqueue(D)：
  rear = (2+1)%4 = 3
  数组：[ _  |  _  |  C  |  D  ]
                     ↑
                   front=2, size=2

第7步 enqueue(E)：★ 关键步骤！
  rear = (2+2)%4 = 0  ← 绕回到索引 0！
  数组：[ E  |  _  |  C  |  D  ]
         ↑           ↑
       rear绕回    front=2, size=3

  逻辑顺序（队头到队尾）：C → D → E
  物理存储：索引 2 → 3 → 0（绕了一圈！）
```

### 7.2.3 两种实现对比

| 对比维度 | 链表实现 | 数组（环形）实现 |
|---------|---------|-----------------|
| **入队** | $O(1)$ | $O(1)$ |
| **出队** | $O(1)$ | $O(1)$ |
| **空间** | 动态分配，无浪费 | 固定大小，可能浪费 |
| **额外开销** | 每个节点有 next 指针 | 可能有空闲槽位 |
| **缓存性能** | 差（内存分散） | 好（连续内存） |
| **容量限制** | 无（动态增长） | 有（固定大小） |
| **实现难度** | 注意指针操作 | 注意取模和边界 |
| **适用场景** | 不确定大小 | 已知大小上限 |

> **实际建议**：
> - 在 Python 中做题或写项目时，直接用 `collections.deque`

### 7.2.4 Python 中使用队列的推荐方式

在实际 Python 开发中，我们通常不自己造轮子，而是用标准库：

```python
from collections import deque

# ===== 用 deque 作为队列 =====
queue: deque = deque()

# 入队（从右端添加）
queue.append(10)        # deque([10])
queue.append(20)        # deque([10, 20])
queue.append(30)        # deque([10, 20, 30])

# 出队（从左端删除）—— O(1)！
val: int = queue.popleft()    # val = 10, deque([20, 30])
# 补充知识：
# popleft() 是 deque 特有的方法，从左端弹出元素，时间复杂度 O(1)
# 普通 list 的 pop(0) 也能从头部删除，但时间复杂度是 O(n)

# 查看队头
front: int = queue[0]         # 20

# 判空和长度
print(len(queue))             # 2
print(len(queue) == 0)        # False

# ===== 为什么用 deque 而不是 list？ =====
# list.pop(0) → O(n)，需要移动所有元素
# deque.popleft() → O(1)，双端队列专门优化了头部操作

# ===== 设置最大长度（可选）=====
# 补充知识：maxlen 参数可以限制 deque 的最大长度
# 超过长度时，会自动从另一端丢弃元素
bounded_queue: deque = deque(maxlen=3)
bounded_queue.append(1)    # deque([1])
bounded_queue.append(2)    # deque([1, 2])
bounded_queue.append(3)    # deque([1, 2, 3])
bounded_queue.append(4)    # deque([2, 3, 4]) ← 1 被自动丢弃！
```

## 7.3 队列典型应用

### 7.3.1 广度优先搜索（BFS）⭐

**广度优先搜索（Breadth-First Search）** 是图/树的遍历算法，**逐层**访问节点。BFS 的核心数据结构就是**队列**。

> **对比 DFS**：
> - DFS（深度优先）用**栈**——一条路走到黑
> - BFS（广度优先）用**队列**——一层一层往外扩散

#### 算法思路

```
1. 起始节点入队，标记为已访问
2. 当队列不为空时：
   a. 出队一个节点，处理它
   b. 将它所有未访问的邻居入队，标记为已访问
3. 重复直到队列为空
```

#### 图示

```
     A
    / \
   B   C
  / \   \
 D   E   F

BFS 遍历过程（起点 A）：

步骤  出队   队列状态（从左到右=从队头到队尾）  已访问
 1     -     [A]                               {A}
 2     A     [B, C]                            {A, B, C}
 3     B     [C, D, E]                         {A, B, C, D, E}
 4     C     [D, E, F]                         {A, B, C, D, E, F}
 5     D     [E, F]                            同上
 6     E     [F]                               同上
 7     F     []                                同上

结果：A → B → C → D → E → F（逐层遍历！）
```

#### 完整实现

```python
from collections import deque


def bfs(graph: dict[str, list[str]], start: str) -> list[str]:
    """
    用队列实现广度优先搜索

    参数：
        graph: 图的邻接表表示
               例如 {'A': ['B', 'C']} 表示 A 与 B、C 相连
        start: 起始节点

    返回值：
        list[str]: BFS 遍历顺序

    补充知识：
    - dict[str, list[str]] 是类型注解，表示：
      "键是字符串，值是字符串列表" 的字典
    - 这是 Python 3.9+ 的写法
    """

    visited: set[str] = set()       # 记录已访问的节点
    # 补充知识：
    # set() 创建一个空集合
    # 集合的特点：元素不重复，查找速度 O(1)
    # set.add(x) 添加元素
    # x in set   判断元素是否在集合中

    result: list[str] = []          # 记录遍历顺序
    queue: deque[str] = deque()     # BFS 用队列

    # 起始节点入队
    queue.append(start)
    visited.add(start)

    while queue:                    # 队列不为空就继续
        node: str = queue.popleft()   # 出队队头节点
        result.append(node)           # 记录遍历顺序

        # 遍历当前节点的所有邻居
        for neighbor in graph.get(node, []):
            # graph.get(node, []) 获取 node 的邻居列表
            # 如果 node 不在图中，返回空列表 []（避免 KeyError）
            if neighbor not in visited:
                visited.add(neighbor)      # 标记为已访问
                queue.append(neighbor)     # 邻居入队

    return result


# ===== 示例 =====
# 构建一个图：
#     A
#    / \
#   B   C
#  / \   \
# D   E   F

graph: dict[str, list[str]] = {
    'A': ['B', 'C'],
    'B': ['D', 'E'],
    'C': ['F'],
    'D': [],
    'E': [],
    'F': []
}

print("=== BFS 广度优先搜索 ===")
print(f"  图结构: {graph}")
order: list[str] = bfs(graph, 'A')
print(f"  BFS 顺序: {' → '.join(order)}")
# 输出：A → B → C → D → E → F
```

#### 对比 DFS 和 BFS

```python
from collections import deque


def bfs(graph: dict[str, list[str]], start: str) -> list[str]:
    """BFS：用队列"""
    visited: set[str] = set()
    result: list[str] = []
    queue: deque[str] = deque([start])    # 初始化时直接传入起始节点
    visited.add(start)

    while queue:
        node: str = queue.popleft()       # 队列：从头部取 → 先进先出
        result.append(node)
        for neighbor in graph.get(node, []):
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return result


def dfs(graph: dict[str, list[str]], start: str) -> list[str]:
    """DFS：用栈"""
    visited: set[str] = set()
    result: list[str] = []
    stack: list[str] = [start]            # 用 list 当栈

    while stack:
        node: str = stack.pop()           # 栈：从尾部取 → 后进先出
        if node not in visited:
            visited.add(node)
            result.append(node)
            for neighbor in reversed(graph.get(node, [])):
                if neighbor not in visited:
                    stack.append(neighbor)
    return result


graph: dict[str, list[str]] = {
    'A': ['B', 'C'],
    'B': ['D', 'E'],
    'C': ['F'],
    'D': [], 'E': [], 'F': []
}

print("=== BFS vs DFS ===")
print(f"  BFS: {' → '.join(bfs(graph, 'A'))}")  # A → B → C → D → E → F（逐层）
print(f"  DFS: {' → '.join(dfs(graph, 'A'))}")  # A → B → D → E → C → F（一路深入）
```

### 7.3.2 二叉树的层序遍历 ⭐

层序遍历是 BFS 在二叉树中的典型应用，也是面试高频题（LeetCode 102）。

```python
from collections import deque


class TreeNode:
    """
    二叉树节点

    属性：
        val:   节点值
        left:  左子节点
        right: 右子节点

    图示：
          val
         /   \
       left  right
    """

    def __init__(self, val: int = 0,
                 left: 'TreeNode | None' = None,
                 right: 'TreeNode | None' = None) -> None:
        self.val: int = val
        self.left: TreeNode | None = left
        self.right: TreeNode | None = right
        # 补充知识：
        # 'TreeNode | None' 加了引号是因为类还没定义完，
        # 这叫"前向引用"，Python 需要用字符串来延迟解析类型


def level_order(root: TreeNode | None) -> list[list[int]]:
    """
    二叉树的层序遍历

    参数：
        root: 二叉树根节点

    返回值：
        list[list[int]]: 每一层的节点值列表
        例如 [[1], [2, 3], [4, 5, 6]] 表示三层

    思路：
    - 用队列做 BFS
    - 关键技巧：在每层开始前，记录当前队列长度 = 当前层的节点数
    - 然后恰好处理这么多个节点，就完成了一层

    图示：
            1
           / \
          2   3
         / \   \
        4   5   6

    过程：
    第1层：队列 [1]      → 处理 1 个节点 → 结果 [1]
    第2层：队列 [2, 3]   → 处理 2 个节点 → 结果 [2, 3]
    第3层：队列 [4, 5, 6] → 处理 3 个节点 → 结果 [4, 5, 6]
    """

    if root is None:
        return []

    result: list[list[int]] = []
    queue: deque[TreeNode] = deque([root])

    while queue:
        level_size: int = len(queue)     # 当前层有多少个节点
        current_level: list[int] = []    # 存储当前层的值

        for _ in range(level_size):      # 恰好处理这一层的所有节点
            # 补充知识：
            # for _ in range(n) 中的 _ 是一个约定俗成的变量名，
            # 表示"我不需要用到这个循环变量"

            node: TreeNode = queue.popleft()     # 出队
            current_level.append(node.val)       # 记录当前层的值

            # 将子节点入队（下一层的节点）
            if node.left is not None:
                queue.append(node.left)
            if node.right is not None:
                queue.append(node.right)

        result.append(current_level)     # 当前层处理完毕，加入结果

    return result


# ===== 构建测试用的二叉树 =====
#         1
#        / \
#       2   3
#      / \   \
#     4   5   6

root: TreeNode = TreeNode(1)
root.left = TreeNode(2)
root.right = TreeNode(3)
root.left.left = TreeNode(4)
root.left.right = TreeNode(5)
root.right.right = TreeNode(6)

print("=== 二叉树层序遍历 ===")
levels: list[list[int]] = level_order(root)
for i, level in enumerate(levels):
    # 补充知识：
    # enumerate() 同时获取索引和值
    # enumerate(['a', 'b', 'c']) → (0,'a'), (1,'b'), (2,'c')
    print(f"  第 {i + 1} 层: {level}")

# 输出：
# 第 1 层: [1]
# 第 2 层: [2, 3]
# 第 3 层: [4, 5, 6]
```

### 7.3.3 任务调度 / 打印队列模拟

在操作系统中，任务按提交顺序排队执行，这就是队列的典型应用。

```python
from collections import deque
import time


class Task:
    """
    任务类

    属性：
        name:     任务名称
        duration: 执行时间（秒）
    """

    def __init__(self, name: str, duration: int) -> None:
        self.name: str = name
        self.duration: int = duration

    def __str__(self) -> str:
        return f"Task({self.name}, {self.duration}s)"


class TaskScheduler:
    """
    简单的任务调度器（FIFO 顺序执行）

    模拟：任务按提交顺序排队，依次执行
    """

    def __init__(self) -> None:
        self._queue: deque[Task] = deque()

    def submit(self, task: Task) -> None:
        """提交任务到队列"""
        self._queue.append(task)
        print(f"  [提交] {task} 已加入队列，当前排队数: {len(self._queue)}")

    def run_all(self) -> None:
        """按顺序执行所有任务"""
        print("\n  [开始执行]")
        step: int = 1

        while self._queue:
            task: Task = self._queue.popleft()    # FIFO：先提交的先执行
            print(f"  {step}. 正在执行: {task.name}（需要 {task.duration}s）")
            # 实际项目中这里会真正执行任务
            # time.sleep(task.duration)  # 模拟执行时间（这里注释掉以加快演示）
            print(f"     ✓ {task.name} 完成")
            step += 1

        print("  [全部完成]")

    def pending_count(self) -> int:
        """查看等待中的任务数"""
        return len(self._queue)


# ===== 模拟任务调度 =====
print("=== 任务调度器模拟 ===\n")

scheduler.run_all()
```

### 7.3.4 滑动窗口最大值（进阶）⭐

**问题**（LeetCode 239）：给定一个数组和窗口大小 k，返回每个窗口中的最大值。

```
输入：nums = [1, 3, -1, -3, 5, 3, 6, 7], k = 3
窗口滑动过程：
  [1  3  -1] -3  5  3  6  7    → 最大值 3
   1 [3  -1  -3] 5  3  6  7    → 最大值 3
   1  3 [-1  -3  5] 3  6  7    → 最大值 5
   1  3  -1 [-3  5  3] 6  7    → 最大值 5
   1  3  -1  -3 [5  3  6] 7    → 最大值 6
   1  3  -1  -3  5 [3  6  7]   → 最大值 7

输出：[3, 3, 5, 5, 6, 7]
```

**思路**：使用**单调递减队列**（队列中的元素从队头到队尾递减），队头始终是当前窗口的最大值。

```python
from collections import deque


def max_sliding_window(nums: list[int], k: int) -> list[int]:
    """
    滑动窗口最大值

    参数：
        nums: 整数数组
        k:    窗口大小

    返回值：
        list[int]: 每个窗口的最大值

    核心思路（单调递减队列）：
    - 队列中存储的是元素的 **索引**（不是值）
    - 维护队列中的元素从队头到队尾 **单调递减**
    - 队头始终是当前窗口中最大值的索引

    为什么要存索引而不是值？
    因为我们需要判断队头元素是否已经滑出窗口，
    只有存索引才能做这个判断。
    """

    if not nums or k == 0:
        return []

    dq: deque[int] = deque()     # 存储索引的单调递减队列
    result: list[int] = []

    for i in range(len(nums)):
        # 步骤1：移除已经滑出窗口的队头元素
        # 如果队头的索引已经不在窗口 [i-k+1, i] 范围内，就移除
        if dq and dq[0] < i - k + 1:
            dq.popleft()

        # 步骤2：维护单调递减性
        # 从队尾开始，移除所有比当前元素小的（因为它们不可能再成为最大值了）
        while dq and nums[dq[-1]] < nums[i]:
            dq.pop()       # 注意：这里是 pop() 从队尾移除

        # 步骤3：当前索引入队
        dq.append(i)

        # 步骤4：当窗口形成后（i >= k-1），记录结果
        # 队头就是当前窗口的最大值
        if i >= k - 1:
            result.append(nums[dq[0]])

    return result


# ===== 测试 =====
print("=== 滑动窗口最大值 ===\n")
nums: list[int] = [1, 3, -1, -3, 5, 3, 6, 7]
k: int = 3

print(f"  数组: {nums}")
print(f"  窗口大小: {k}")
print(f"  结果: {max_sliding_window(nums, k)}")
# 输出: [3, 3, 5, 5, 6, 7]

# 详细过程追踪
print("\n  --- 详细过程 ---")
dq: deque[int] = deque()
for i in range(len(nums)):
    if dq and dq[0] < i - k + 1:
        dq.popleft()
    while dq and nums[dq[-1]] < nums[i]:
        dq.pop()
    dq.append(i)

    # 打印当前状态
    window_start: int = max(0, i - k + 1)
    window: list[int] = nums[window_start:i + 1]
    dq_values: list[int] = [nums[j] for j in dq]
    # 补充知识：[nums[j] for j in dq] 是列表推导式
    # 等价于：
    # dq_values = []
    # for j in dq:
    #     dq_values.append(nums[j])

    if i >= k - 1:
        print(f"  i={i}, 窗口{window}, 队列值{dq_values}, 最大值={nums[dq[0]]}")
    else:
        print(f"  i={i}, 窗口{window}（未满）, 队列值{dq_values}")
```

### 7.3.5 热土豆游戏（约瑟夫问题简化版）

**问题**：一群人围成一圈，从某人开始传递一个"热土豆"，数到第 n 个人时，该人出局。重复直到只剩一人。

```python
from collections import deque


def hot_potato(names: list[str], num: int) -> str:
    """
    热土豆游戏（队列模拟循环过程）

    参数：
        names: 参与者名单
        num:   每轮传递次数（数到 num 的人出局）

    返回值：
        str: 最后幸存者的名字

    核心思路：
    - 用队列模拟"围成一圈"
    - 每次从队头取出一个人，放到队尾（模拟传递）
    - 传递 num 次后，队头的人出局（dequeue 不再放回）
    - 重复直到只剩一人

    图示（names=['A','B','C','D','E'], num=3）：

    初始队列：[A, B, C, D, E]

    第1轮（传3次）：
      传1: A 到队尾 → [B, C, D, E, A]
      传2: B 到队尾 → [C, D, E, A, B]
      传3: C 出局！ → [D, E, A, B]

    第2轮（传3次）：
      传1: D 到队尾 → [E, A, B, D]
      传2: E 到队尾 → [A, B, D, E]
      传3: A 出局！ → [B, D, E]
      ...以此类推
    """

    queue: deque[str] = deque(names)
    # 补充知识：deque(names) 用列表初始化一个 deque

    print(f"  初始: {list(queue)}")

    round_num: int = 1
    while len(queue) > 1:
        # 传递 num-1 次（把人从队头移到队尾）
        for _ in range(num - 1):
            queue.append(queue.popleft())     # 队头 → 队尾

        # 第 num 个人出局
        eliminated: str = queue.popleft()
        print(f"  第{round_num}轮: {eliminated} 出局！剩余: {list(queue)}")
        round_num += 1

    survivor: str = queue[0]
    print(f"  最终幸存者: {survivor}")
    return survivor


# ===== 测试 =====

players: list[str] = ["小明", "小红", "小华", "小李", "小张"]
hot_potato(players, 3)
```

### 7.4.1 用队列实现栈（LeetCode 225）

**思路**：用一个队列，每次 push 新元素后，把前面的元素全部出队再入队，让新元素跑到队头。

```python
from collections import deque


class MyStack:
    """
    用队列实现栈

    核心思路：
    - 只用一个队列
    - 每次 push 新元素后，把队列里"之前的元素"全部重新排到新元素后面
    - 这样新元素就在队头了（实现了后进先出）

    图示（push(3)，队列中已有 [1, 2]）：

    步骤1：3 入队         → [1, 2, 3]
    步骤2：把前面的元素依次出队再入队（重复 size-1 次）
           1 出队再入队   → [2, 3, 1]
           2 出队再入队   → [3, 1, 2]

    现在队头是 3，实现了 LIFO！
    """

    def __init__(self) -> None:
        self._queue: deque[int] = deque()

    def push(self, x: int) -> None:
        """入栈"""
        self._queue.append(x)
        # 把之前的所有元素重新排到后面
        for _ in range(len(self._queue) - 1):
            self._queue.append(self._queue.popleft())

    def pop(self) -> int:
        """出栈"""
        return self._queue.popleft()

    def top(self) -> int:
        """查看栈顶"""
        return self._queue[0]

    def empty(self) -> bool:
        """判空"""
        return len(self._queue) == 0


# ===== 测试 =====
print("=== 用队列实现栈 ===")
s: MyStack = MyStack()
s.push(1)
s.push(2)
s.push(3)
print(f"  top():  {s.top()}")     # 3（后进先出）
print(f"  pop():  {s.pop()}")     # 3
print(f"  pop():  {s.pop()}")     # 2
print(f"  top():  {s.top()}")     # 1
print(f"  empty(): {s.empty()}")  # False
```

### 7.4.2 最近的请求次数（LeetCode 933）

**问题**：实现一个 `RecentCounter` 类，统计最近 3000 毫秒内的请求次数。

```python
from collections import deque


class RecentCounter:
    """
    统计最近 3000 毫秒内的请求次数

    思路：
    - 用队列保存所有请求的时间戳
    - 每次新请求来时，把 3000ms 之前的旧请求全部出队
    - 队列中剩余的元素数量就是答案

    为什么用队列？
    - 请求按时间顺序到达（先到的时间戳小）
    - 过期的请求一定在队头（先进先出）
    - 完美符合 FIFO 特性
    """

    def __init__(self) -> None:
        self._queue: deque[int] = deque()

    def ping(self, t: int) -> int:
        """
        记录一次请求，返回最近 3000ms 内的请求数

        参数：
            t: 当前请求的时间戳（毫秒），保证递增

        返回值：
            int: 时间范围 [t-3000, t] 内的请求总数
        """
        self._queue.append(t)              # 新请求入队

        # 移除 3000ms 之前的过期请求
        while self._queue[0] < t - 3000:
            self._queue.popleft()

        return len(self._queue)


# ===== 测试 =====
print("=== 最近请求计数器 ===")
counter: RecentCounter = RecentCounter()

requests: list[int] = [1, 100, 3001, 3002, 7000]
for t in requests:
    count: int = counter.ping(t)
    print(f"  ping({t}): 最近3000ms内有 {count} 个请求")

# 输出：
# ping(1):    最近3000ms内有 1 个请求   → [1] 范围 [-2999, 1]
# ping(100):  最近3000ms内有 2 个请求   → [1, 100]
# ping(3001): 最近3000ms内有 3 个请求   → [1, 100, 3001]
# ping(3002): 最近3000ms内有 3 个请求   → [100, 3001, 3002]  ← 1 被移除
# ping(7000): 最近3000ms内有 1 个请求   → [7000]  ← 前面全部过期
```

### 7.4.3 完全平方数（LeetCode 279）—— BFS 经典应用

**问题**：给一个正整数 n，找到最少几个完全平方数之和等于 n。

```python
from collections import deque


def num_squares(n: int) -> int:
    """
    找到最少的完全平方数之和等于 n

    例如：
    n = 12 → 4 + 4 + 4 = 12 → 答案是 3
    n = 13 → 4 + 9 = 13     → 答案是 2

    思路（BFS）：
    - 把问题看成"从 n 走到 0"的最短路径
    - 每次可以减去一个完全平方数（1, 4, 9, 16, ...）
    - BFS 天然找最短路径（因为逐层搜索）

    图示（n=12）：
    第0层:  12
    第1层:  12-1=11, 12-4=8, 12-9=3
    第2层:  8-4=4, 3-1=2, ...
    第3层:  4-4=0 ✓ 找到了！→ 答案是 3 步
    """

    # 预计算所有不超过 n 的完全平方数
    squares: list[int] = []
    i: int = 1
    while i * i <= n:
        squares.append(i * i)
        i += 1
    # 例如 n=12 → squares = [1, 4, 9]

    # BFS
    queue: deque[tuple[int, int]] = deque()
    # 补充知识：
    # tuple 是元组，不可变的序列，用 () 创建
    # (remaining, steps) 表示"还剩多少"和"用了几步"
    # deque[tuple[int, int]] 表示队列中每个元素是一个 (int, int) 元组

    queue.append((n, 0))          # 初始状态：(剩余值, 步数)
    visited: set[int] = {n}       # 记录已访问的数值，避免重复计算
    # 补充知识：{n} 创建一个包含 n 的集合

    while queue:
        remaining, steps = queue.popleft()
        # 补充知识：这叫"元组解包"
        # 等价于：
        # pair = queue.popleft()
        # remaining = pair[0]
        # steps = pair[1]

        for sq in squares:
            next_val: int = remaining - sq

            if next_val == 0:
                return steps + 1       # 找到了！

            if next_val > 0 and next_val not in visited:
                visited.add(next_val)
                queue.append((next_val, steps + 1))

    return n    # 最坏情况：全用 1（不会真的走到这里）


# ===== 测试 =====
print("=== 完全平方数（BFS） ===")
test_cases: list[tuple[int, int]] = [
    (12, 3),    # 4+4+4
    (13, 2),    # 4+9
    (1, 1),     # 1
    (7, 4),     # 1+1+1+4
]
for n, expected in test_cases:
    result: int = num_squares(n)
    status: str = "✓" if result == expected else "✗"
    print(f"  {status} numSquares({n}) = {result}")
```

## 7.5 易错点总结

### 错误 1：用 list.pop(0) 当队列出队

```python
# ❌ 错误：pop(0) 是 O(n)，性能很差
queue: list[int] = [1, 2, 3, 4, 5]
val: int = queue.pop(0)    # O(n)！所有后面的元素要前移

# ✅ 正确：用 deque 的 popleft()
from collections import deque
queue: deque[int] = deque([1, 2, 3, 4, 5])
val: int = queue.popleft()   # O(1)！
```

### 错误 2：链表队列出队后忘记处理 rear 指针

```python
# ❌ 错误：最后一个元素出队后，rear 还指向被删除的节点
def bad_dequeue(self) -> object:
    data = self._front.data
    self._front = self._front.next
    self._size -= 1
    return data    # 如果队列变空了，self._rear 还指向旧节点！

# ✅ 正确：队列变空时，同步更新 rear
def good_dequeue(self) -> object:
    data = self._front.data
    self._front = self._front.next
    self._size -= 1
    if self.is_empty():          # ← 别忘了这步！
        self._rear = None
    return data
```

### 错误 3：环形数组计算索引不取模

```python
capacity: int = 5

# ❌ 错误：不取模，索引可能越界
rear_index: int = front + size    # 可能大于 capacity-1！

# ✅ 正确：取模实现环形
rear_index: int = (front + size) % capacity
```

### 错误 4：BFS 中入队时忘记标记 visited

```python
# ❌ 错误：出队时才标记 visited → 同一个节点可能被多次入队
while queue:
    node = queue.popleft()
    if node not in visited:     # 出队时才检查
        visited.add(node)
        for neighbor in graph[node]:
            queue.append(neighbor)    # 可能重复入队！

# ✅ 正确：入队时就标记 visited → 避免重复入队
while queue:
    node = queue.popleft()
    for neighbor in graph[node]:
        if neighbor not in visited:   # 入队前检查
            visited.add(neighbor)     # 入队时就标记
            queue.append(neighbor)
```

### 错误 5：空队列 dequeue

```python
from collections import deque

# ❌ 错误：没检查就 popleft
queue: deque = deque()
val = queue.popleft()    # IndexError: pop from an empty deque

# ✅ 正确：先判空
queue: deque = deque()
if queue:                # 非空才出队
    val = queue.popleft()

# ✅ 或者用异常处理
try:
    val = queue.popleft()
except IndexError:
    print("队列为空")
```

### 错误 6：混淆栈和队列的操作方向

```python
from collections import deque

# 栈（LIFO）：同一端进出
stack: list[int] = []
stack.append(1)       # 从右端进
stack.pop()           # 从右端出 ← 同一端！

# 队列（FIFO）：一端进，另一端出
queue: deque[int] = deque()
queue.append(1)       # 从右端进
queue.popleft()       # 从左端出 ← 不同端！

# ❌ 常见混淆：队列用 pop() 而不是 popleft()
# queue.pop()    ← 这是从右端出，变成了栈的行为！
# queue.popleft() ← 这才是队列的正确出队操作
```

## 7.6 总结

### 核心要点速查表

```css
队列 = 受限的线性表，一端进一端出
核心原则：FIFO（先进先出）

五大操作：
├── enqueue(x) → 入队（队尾添加）
├── dequeue()  → 出队（队头移除）
├── peek()     → 查看队头
├── is_empty() → 判空
└── size()     → 大小

两种实现：
├── 链表实现：front 和 rear 两个指针，动态大小
└── 环形数组实现：取模运算实现环形，固定大小

所有操作时间复杂度：O(1)

Python 推荐用法：
    from collections import deque
    queue = deque()
    queue.append(x)       # 入队
    queue.popleft()       # 出队 ← 别用 pop(0)！

五大经典应用：
├── BFS 广度优先搜索 → 逐层遍历图/树
├── 二叉树层序遍历   → BFS 的树上应用
├── 任务调度         → 先到先服务
├── 滑动窗口         → 单调队列优化
└── 热土豆 / 约瑟夫  → 队列模拟循环

队列 vs 栈 速记：
├── 栈 = 后进先出（LIFO）= 叠盘子
└── 队列 = 先进先出（FIFO）= 排队买奶茶
```

### 学习建议

1. **先理解 FIFO 概念**：和栈的 LIFO 对比记忆
2. **链表实现重点理解双指针**：front 和 rear 的配合，特别是队列变空时的处理
3. **环形数组重点理解取模**：`(front + size) % capacity` 这个公式要烂熟于心
4. **BFS 是最重要的应用**：这是后续学习图算法、最短路径的基础
5. **实际写代码时直接用 `deque`**：不用自己造轮子
6. **刷题推荐**：LeetCode 上搜索"队列"和"BFS"标签，从简单题开始
