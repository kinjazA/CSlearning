## 一、核心思想：Divide / Conquer / Combine 三段式拆解

分治的本质是**将一个规模为 (n) 的难题，拆解为若干规模更小、结构相同的子问题，独立求解后再合并**。三个步骤缺一不可：

| 阶段        | 名称 | 含义                 | 关键问题                     |
| ----------- | ---- | -------------------- | ---------------------------- |
| **Divide**  | 分   | 把原问题切分为子问题 | 切几刀？怎么切？             |
| **Conquer** | 治   | 递归求解子问题       | 何时停？基线条件 (Base Case) |
| **Combine** | 合   | 由子解构造原问题的解 | 合并的代价是多少？           |

> ⚠️ **新手最常忽略 Combine**：很多人把分治写成"递归 + 切片"就完了，但**真正决定算法效率的，是 Combine 步骤的复杂度**。归并排序的 (O(n \log n)) 是因为合并代价为 (O(n))，快排的最坏 (O(n^2)) 也是因为分区不平衡导致 Combine 退化。

### 1.1 主定理 (Master Theorem) 速估

对于形如：

[T(n) = aT(n/b) + f(n)]

的递推（(a \ge 1, b > 1)），令 (c_{\text{crit}} = \log_b a)，则：

- **Case 1**：(f(n) = O(n^{c_{\text{crit}} - \epsilon})) → (T(n) = \Theta(n^{c_{\text{crit}}})) （子问题主导）
- **Case 2**：(f(n) = \Theta(n^{c_{\text{crit}}})) → (T(n) = \Theta(n^{c_{\text{crit}}} \log n)) （平衡）
- **Case 3**：(f(n) = \Omega(n^{c_{\text{crit}} + \epsilon})) → (T(n) = \Theta(f(n))) （合并主导）

### 1.2 经典递推一览（背下来！）

| 算法               | 递推式                    | 主定理 Case | 复杂度                                 |
| ------------------ | ------------------------- | ----------- | -------------------------------------- |
| 二分查找           | (T(n) = T(n/2) + O(1))    | Case 2      | (O(\log n))                            |
| 归并排序           | (T(n) = 2T(n/2) + O(n))   | Case 2      | (O(n \log n))                          |
| 快速排序（平均）   | (T(n) = 2T(n/2) + O(n))   | Case 2      | (O(n \log n))                          |
| 快速选择（平均）   | (T(n) = T(n/2) + O(n))    | Case 3      | (O(n))                                 |
| Karatsuba 大数乘法 | (T(n) = 3T(n/2) + O(n))   | Case 1      | (O(n^{\log_2 3}) \approx O(n^{1.585})) |
| Strassen 矩阵乘    | (T(n) = 7T(n/2) + O(n^2)) | Case 1      | (O(n^{\log_2 7}) \approx O(n^{2.807})) |

## 二、经典基础

### 2.1 汉诺塔 (Tower of Hanoi)

> **问题**：将 (n) 个圆盘从 A 柱借助 B 柱全部移动到 C 柱，每次只能移动一个，且大盘不能压在小盘上。

**分治思维**：要把 (n) 个盘从 A 移到 C，等价于：

1. **Divide**：把上面 (n-1) 个盘从 A 借助 C 移到 B；
2. **Conquer**：把最底下第 (n) 个盘直接从 A 移到 C；
3. **Combine**：把 B 上的 (n-1) 个盘借助 A 移到 C。

```python
from typing import List, Tuple


def hanoi(n: int, source: str = "A", auxiliary: str = "B", target: str = "C") -> List[Tuple[str, str]]:
    """汉诺塔分治求解。

    Args:
        n: 圆盘数量，必须 >= 1。
        source:    起始柱标识。
        auxiliary: 辅助柱标识。
        target:    目标柱标识。

    Returns:
        移动步骤序列，每个元素形如 (from_peg, to_peg)。
    """
    moves: List[Tuple[str, str]] = []

    def _solve(k: int, src: str, aux: str, tgt: str) -> None:
        # Base Case：只剩一个盘，直接搬走
        if k == 1:
            moves.append((src, tgt))
            return

        # Divide：把上面 k-1 个盘从 src 借助 tgt 搬到 aux
        _solve(k - 1, src, tgt, aux)

        # Conquer：把第 k 个（最大）盘直接从 src 搬到 tgt
        moves.append((src, tgt))

        # Combine：把 aux 上的 k-1 个盘借助 src 搬到 tgt
        _solve(k - 1, aux, src, tgt)

    _solve(n, source, auxiliary, target)
    return moves


if __name__ == "__main__":
    for step, (frm, to) in enumerate(hanoi(3), start=1):
        print(f"Step {step:>2}: {frm} -> {to}")
```

**复杂度分析**：

- **Time Complexity：(O(2^n))**。递推 (T(n) = 2T(n-1) + 1)，解得 (2^n - 1) 次移动，这是信息论下限，无法优化。
- **Space Complexity：(O(n))**。递归栈深度为 (n)；`moves` 列表占用 (O(2^n)) 但属于输出本身。

### 2.2 由前序 + 中序序列重建二叉树（LeetCode 105）

**核心观察**：

- **前序**：`[根, [左子树前序], [右子树前序]]`
- **中序**：`[[左子树中序], 根, [右子树中序]]`

只要在中序中定位到根，就能切出左右子树的边界。

```
preorder = [3, 9, 20, 15, 7]
inorder  = [9, 3, 15, 20, 7]

根 = 3，在 inorder 中下标 = 1
左子树 inorder: [9]            -> 1 个节点
右子树 inorder: [15, 20, 7]    -> 3 个节点

         3
        / \
       9   20
           / \
          15  7
from typing import Dict, List, Optional


class TreeNode:
    """二叉树节点。"""

    def __init__(
        self,
        val: int = 0,
        left: Optional["TreeNode"] = None,
        right: Optional["TreeNode"] = None,
    ) -> None:
        self.val = val
        self.left = left
        self.right = right


def build_tree(preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
    """根据前序与中序遍历序列重建二叉树。

    Args:
        preorder: 前序遍历结果，所有元素互不相同。
        inorder:  中序遍历结果，所有元素互不相同。

    Returns:
        重建后的二叉树根节点，若输入为空则返回 None。
    """
    # 工程优化：用哈希表把 inorder 查找从 O(n) 降到 O(1)
    inorder_idx: Dict[int, int] = {val: idx for idx, val in enumerate(inorder)}

    def _build(pre_l: int, pre_r: int, in_l: int, in_r: int) -> Optional[TreeNode]:
        # Base Case：区间为空，没有节点
        if pre_l > pre_r:
            return None

        # Divide：前序首元素就是当前子树的根
        root_val = preorder[pre_l]
        root_pos = inorder_idx[root_val]
        left_size = root_pos - in_l  # 左子树节点数

        # Conquer：递归构建左右子树
        left_subtree = _build(
            pre_l + 1, pre_l + left_size, in_l, root_pos - 1
        )
        right_subtree = _build(
            pre_l + left_size + 1, pre_r, root_pos + 1, in_r
        )

        # Combine：以根节点连接左右子树
        return TreeNode(root_val, left_subtree, right_subtree)

    return _build(0, len(preorder) - 1, 0, len(inorder) - 1)
```

**复杂度分析**：

- **Time Complexity：(O(n))**。每个节点只被处理一次，哈希表查找 (O(1))。
- **Space Complexity：(O(n))**。哈希表 (O(n)) + 递归栈最坏 (O(n))（链式树）。

> 💡 **工程提醒**：千万不要用 `inorder.index(root_val)` 在循环中查找，那会让总复杂度退化为 (O(n^2))。**预处理哈希表是面试加分项**。

## 三、工业级核心

### 3.1 归并排序 (Merge Sort)

归并排序是**外部排序 (External Sort) 的鼻祖**：当数据量超过内存（如 100 GB 日志排序，机器只有 16 GB 内存）时，可以：

1. 把大文件切成 (k) 个能放进内存的小块；
2. 每块在内存中排好序后落盘；
3. 用 (k) 路归并把所有有序小块合成一个大有序文件。

这正是 **Hadoop / Spark 的 Shuffle Sort 阶段**所做的事情。

#### 3.1.1 标准归并排序（双指针版，工业推荐）

> 这里**不使用** `arr[:mid]` 切片，原因详见第六章。

```python
from typing import List


def merge_sort(arr: List[int]) -> List[int]:
    """归并排序入口，返回新的有序列表（不修改原列表）。"""
    data = arr[:]  # 拷贝一次，保护输入
    _merge_sort(data, 0, len(data) - 1)
    return data


def _merge_sort(arr: List[int], left: int, right: int) -> None:
    """对 arr[left..right] 闭区间原地归并排序。"""
    # Base Case：区间长度 <= 1
    if left >= right:
        return

    mid = (left + right) // 2

    # Divide：左右两半各自排序
    _merge_sort(arr, left, mid)
    _merge_sort(arr, mid + 1, right)

    # Combine：合并两个有序半段
    _merge(arr, left, mid, right)


def _merge(arr: List[int], left: int, mid: int, right: int) -> None:
    """合并 arr[left..mid] 与 arr[mid+1..right] 两个有序段。"""
    temp: List[int] = arr[left : right + 1]  # 仅在合并时一次性拷贝
    i = 0                # 左半段在 temp 中的起点
    j = mid - left + 1   # 右半段在 temp 中的起点
    left_end = mid - left
    right_end = right - left
    k = left             # 写回 arr 的位置

    while i <= left_end and j <= right_end:
        if temp[i] <= temp[j]:   # 注意 <= 保证稳定排序
            arr[k] = temp[i]
            i += 1
        else:
            arr[k] = temp[j]
            j += 1
        k += 1

    # 处理剩余尾巴（最多有一个非空）
    while i <= left_end:
        arr[k] = temp[i]
        i += 1
        k += 1
    while j <= right_end:
        arr[k] = temp[j]
        j += 1
        k += 1
```

**Combine 步骤图解**：

```
合并  L=[2, 5, 8]   R=[1, 6, 9]
       i↑                j↑

step1: 比较 2 vs 1  →  写 1   结果: [1]
step2: 比较 2 vs 6  →  写 2   结果: [1, 2]
step3: 比较 5 vs 6  →  写 5   结果: [1, 2, 5]
step4: 比较 8 vs 6  →  写 6   结果: [1, 2, 5, 6]
step5: 比较 8 vs 9  →  写 8   结果: [1, 2, 5, 6, 8]
step6: L 空, 拷贝 R 剩余     结果: [1, 2, 5, 6, 8, 9]
```

**复杂度分析**：

- **Time Complexity：(O(n \log n))**，最好/最坏/平均均如此（对输入分布无敏感）。
- **Space Complexity：(O(n))**。`temp` 缓冲区 + 递归栈 (O(\log n))，主导项是 (O(n))。

#### 3.1.2 进阶应用：归并排序求逆序对数（LeetCode 剑指 Offer 51）

**逆序对**：(i < j) 且 (arr[i] > arr[j]) 的 ((i, j)) 对数。暴力 (O(n^2))，归并思想 (O(n \log n))。

**关键点**：在 `merge` 时，若左半段元素 `arr[i] > arr[j]`，则左半段从 `i` 到 `mid` 的**所有**元素都比 `arr[j]` 大（因为左半段已有序），一次性贡献 `mid - i + 1` 个逆序对。

```python
from typing import List


def count_inversions(nums: List[int]) -> int:
    """返回数组中逆序对的总数。

    Args:
        nums: 整数数组（不会被修改）。

    Returns:
        逆序对个数。
    """
    arr = nums[:]  # 不破坏入参

    def _solve(left: int, right: int) -> int:
        if left >= right:
            return 0

        mid = (left + right) // 2

        # Divide + Conquer：左右两半各自的逆序对数
        count = _solve(left, mid) + _solve(mid + 1, right)

        # Combine：跨越左右两半的逆序对在合并时统计
        temp: List[int] = []
        i, j = left, mid + 1
        while i <= mid and j <= right:
            if arr[i] <= arr[j]:
                temp.append(arr[i])
                i += 1
            else:
                # 关键：arr[i..mid] 全部 > arr[j]
                count += mid - i + 1
                temp.append(arr[j])
                j += 1
        # 拷贝剩余
        temp.extend(arr[i : mid + 1])
        temp.extend(arr[j : right + 1])
        arr[left : right + 1] = temp

        return count

    return _solve(0, len(arr) - 1)
```

**复杂度分析**：

- **Time Complexity：(O(n \log n))**，与归并排序同阶。
- **Space Complexity：(O(n))**，临时数组 + 递归栈。

### 3.2 快速排序 (Quick Sort)

快排在工业界的真正地位：**Python 的 `list.sort()` 是 Timsort（归并变种），但 C++ STL 的 `std::sort` 主体是 IntroSort（快排 + 堆排兜底）**。掌握快排的工程化写法是面试硬通货。

#### 3.2.1 原地分区 + 随机化 Pivot

**为什么必须随机化？**
若每次取 `arr[right]` 作 pivot，遇到**已排序**或**逆序**输入时，分区极不平衡（一边 0 个，一边 (n-1) 个），递归退化为 (O(n^2)) 且栈深 (n)，**直接 RecursionError**。随机化把"最坏输入"变成"最坏概率"。

```python
import random
from typing import List


def quick_sort(nums: List[int]) -> None:
    """随机化原地快速排序，直接修改入参。"""
    _quick_sort(nums, 0, len(nums) - 1)


def _quick_sort(arr: List[int], left: int, right: int) -> None:
    """对 arr[left..right] 闭区间排序。"""
    # Base Case
    if left >= right:
        return

    # Divide：分区，pivot 落到正确的最终位置
    pivot_idx = _partition(arr, left, right)

    # Conquer：递归两侧（pivot 已就位，不再参与）
    _quick_sort(arr, left, pivot_idx - 1)
    _quick_sort(arr, pivot_idx + 1, right)

    # Combine：原地分区天然就是合并，无需额外操作


def _partition(arr: List[int], left: int, right: int) -> int:
    """Lomuto 分区方案 + 随机化 pivot。

    Returns:
        pivot 在分区后所处的最终下标。
    """
    # 随机化：把随机位置元素换到末尾作 pivot
    rand = random.randint(left, right)
    arr[rand], arr[right] = arr[right], arr[rand]
    pivot = arr[right]

    # i 指向 "<=pivot 区域" 的最后一个位置
    i = left - 1
    for j in range(left, right):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]

    # 把 pivot 换到正确位置 i+1
    arr[i + 1], arr[right] = arr[right], arr[i + 1]
    return i + 1
```

**Lomuto 分区图解**（pivot = 4）：

```
初始:  [3, 7, 1, 8, 2, 5, 4]    i=-1, pivot=4
             ^
j=0: 3<=4   i=0, swap(0,0)  [3, 7, 1, 8, 2, 5, 4]
j=1: 7>4    跳过
j=2: 1<=4   i=1, swap(1,2)  [3, 1, 7, 8, 2, 5, 4]
j=3: 8>4    跳过
j=4: 2<=4   i=2, swap(2,4)  [3, 1, 2, 8, 7, 5, 4]
j=5: 5>4    跳过
最后: swap(i+1, right) = swap(3, 6)
                            [3, 1, 2, 4, 7, 5, 8]
                                     ^
                                  pivot 就位
```

**复杂度分析**：

- **Time Complexity**：平均 (O(n \log n))，最坏 (O(n^2))（极端不平衡，随机化下概率极低）。
- **Space Complexity**：平均 (O(\log n)) 递归栈，最坏 (O(n))。**原地排序**，无额外数据空间。

> 💡 **进阶**：处理大量重复元素时，应改用**三路快排 (Dutch National Flag)** ：把数组分成 `<pivot | =pivot | >pivot` 三段，重复元素只参与一次递归，对 `[1,1,1,1,...,1]` 这类输入从 (O(n^2)) 优化到 (O(n))。

## 四、高频场景：快速选择 (Quick Select) 解决 Top-K

**问题**：在长度 (n) 的无序数组中，找出第 (k) 大的元素（或前 (k) 大）。

| 解法              | 时间            | 空间        | 适用场景                |
| ----------------- | --------------- | ----------- | ----------------------- |
| 全排序后取        | (O(n \log n))   | (O(1))      | 数据量小、需要全序      |
| 大小为 k 的小顶堆 | (O(n \log k))   | (O(k))      | **流式数据 / 海量数据** |
| Quick Select      | **平均 (O(n))** | (O(\log n)) | **静态数组、内存够用**  |

### 4.1 核心洞察

快排每次分区后 pivot 一定在最终位置 `p`：

- 若 `p == target`：找到了；
- 若 `p < target`：答案在右半，**只递归右半**；
- 若 `p > target`：答案在左半。

**只走一边**让递推从 (T(n) = 2T(n/2) + n) 变成 (T(n) = T(n/2) + n)，由主定理 Case 3 得 (O(n))。

```python
import random
from typing import List


def find_kth_largest(nums: List[int], k: int) -> int:
    """返回数组中第 k 大的元素（k 从 1 开始计数）。

    Args:
        nums: 整数数组，长度至少为 k。
        k:    1 <= k <= len(nums)。

    Returns:
        第 k 大的整数。
    """
    arr = nums[:]                       # 不破坏入参
    target = len(arr) - k               # 排序后该元素的下标

    def _partition(left: int, right: int) -> int:
        rand = random.randint(left, right)
        arr[rand], arr[right] = arr[right], arr[rand]
        pivot = arr[right]
        i = left - 1
        for j in range(left, right):
            if arr[j] <= pivot:
                i += 1
                arr[i], arr[j] = arr[j], arr[i]
        arr[i + 1], arr[right] = arr[right], arr[i + 1]
        return i + 1

    def _select(left: int, right: int) -> int:
        # Base Case：区间退化为单点
        if left == right:
            return arr[left]

        # Divide：分区
        p = _partition(left, right)

        # Conquer：只递归包含目标的一侧（关键优化！）
        if p == target:
            return arr[p]
        elif p < target:
            return _select(p + 1, right)
        else:
            return _select(left, p - 1)

        # Combine：选择型问题不需要合并，直接返回

    return _select(0, len(arr) - 1)
```

**复杂度分析**：

- **Time Complexity**：平均 (O(n))，最坏 (O(n^2))（随机化后概率可忽略）。
- **Space Complexity**：平均 (O(\log n)) 递归栈。

> 🏭 **工业落地**：Top-K 高频热搜、推荐系统的候选筛选、监控系统的异常 TopN，**数据全在内存且只需一次性查询时**用 Quick Select；**数据是流式或不断更新时**用堆。两者不是替代关系。

## 五、最大子数组和（分治解法 · LeetCode 53）

> **问题**：给定整数数组，找出和最大的连续子数组及其和。

经典 DP（Kadane 算法）是 (O(n))，但**分治解法是面试高频考点**，因为它能让你深刻理解"跨界合并"的思想，且能扩展到**线段树**等高级结构。

### 5.1 分治三段式

最大子数组只可能出现在三个位置：

```
   [.................|..................]
       左半段内       中点    右半段内
                     ↑
              或 跨越中点的子数组
```

- **Divide**：从中点切开；
- **Conquer**：递归求左半最大、右半最大；
- **Combine**：求**必须跨越中点**的最大子数组和（从中点向左/右各扫一遍前缀和），三者取最大。

```python
from typing import List


def max_subarray(nums: List[int]) -> int:
    """分治法求最大子数组和。

    Args:
        nums: 非空整数数组（允许全负）。

    Returns:
        最大连续子数组之和。
    """
    if not nums:
        raise ValueError("nums must be non-empty")

    def _solve(left: int, right: int) -> int:
        # Base Case：单元素
        if left == right:
            return nums[left]

        mid = (left + right) // 2

        # Divide + Conquer
        left_max = _solve(left, mid)
        right_max = _solve(mid + 1, right)

        # Combine：必须包含 mid 与 mid+1 的最大跨界和
        cross_max = _max_crossing(left, mid, right)

        return max(left_max, right_max, cross_max)

    def _max_crossing(left: int, mid: int, right: int) -> int:
        """求一个必须横跨 mid 和 mid+1 的子数组最大和。"""
        # 从 mid 向左扫，求最大左侧前缀和
        left_best = nums[mid]
        cur = 0
        for i in range(mid, left - 1, -1):
            cur += nums[i]
            if cur > left_best:
                left_best = cur

        # 从 mid+1 向右扫，求最大右侧前缀和
        right_best = nums[mid + 1]
        cur = 0
        for i in range(mid + 1, right + 1):
            cur += nums[i]
            if cur > right_best:
                right_best = cur

        return left_best + right_best

    return _solve(0, len(nums) - 1)
```

**Combine 步骤图解**（`nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]`）：

```
切分:    [-2, 1, -3, 4 | -1, 2, 1, -5, 4]
                       mid

左扫:    起点 4，向左累加
         4, 4+(-3)=1, 1+1=2, 2+(-2)=0   →  left_best  = 4
右扫:    起点 -1，向右累加
         -1, -1+2=1, 1+1=2, 2+(-5)=-3, -3+4=1   →  right_best = 2

跨界最大 = 4 + 2 = 6
（即子数组 [4, -1, 2, 1] = 6）
```

**复杂度分析**：

- **Time Complexity：(O(n \log n))**。递推 (T(n) = 2T(n/2) + O(n))，主定理 Case 2。
- **Space Complexity：(O(\log n))**，仅递归栈。

> 💡 **对比 DP**：Kadane 算法是 (O(n)) 时间 (O(1)) 空间，分治版更慢。**但分治的思路可推广到二维矩阵最大子矩阵和、区间合并查询等场景**，思想价值远大于此题本身。

## 六、【重点】Python 分治工程防坑指南

### 6.1 坑一：递归深度限制 `RecursionError`

Python 默认递归深度上限是 **1000**（CPython 实现细节，可通过 `sys.getrecursionlimit()` 查看）。

```python
import sys

print(sys.getrecursionlimit())  # 通常输出 1000
```

**触发场景**：

- 数组超过 1000 个元素的快排/归并，**最坏情况栈深 (O(n))**；
- 链式结构（极度不平衡的 BST）的递归遍历；
- 不带随机化的快排遇到已排序数组。

**应对方案（按推荐度排序）**：

```python
import sys

# 方案 1：调高上限（治标）
sys.setrecursionlimit(100_000)
```

> ⚠️ **`setrecursionlimit` 不是银弹**：
>
> - 调得过高可能撞上**操作系统线程栈大小**（Linux 默认 8 MB），导致 **段错误 (Segmentation Fault)** 而非 Python 异常，**直接进程崩溃**；
> - 推荐配合 `threading.stack_size()` 一起调，或者干脆**改写成迭代**。

**方案 2（推荐）**：把递归改写成**显式栈迭代**：

```python
from typing import List


def quick_sort_iterative(nums: List[int]) -> None:
    """迭代版快排，彻底避开递归深度问题。"""
    stack: List[tuple[int, int]] = [(0, len(nums) - 1)]
    while stack:
        left, right = stack.pop()
        if left >= right:
            continue
        # ... 此处复用前面的 _partition
        # p = _partition(nums, left, right)
        # stack.append((left, p - 1))
        # stack.append((p + 1, right))
```

**方案 3**：保证算法本身的"好"——**随机化 pivot 让最坏情况几乎不发生**。

### 6.2 坑二：递归中**绝对不要用 `arr[:mid]` 切片传参**

这是初学者最常犯的"看起来很优雅、实则灾难"的错误。

#### 6.2.1 内存开销分析

Python 的列表切片 `arr[:mid]` 是 (O(mid)) 时间 + (O(mid)) 空间的**深拷贝**（拷贝引用，但创建新 list 对象）。

考察归并排序两种写法：

| 写法                                    | 单层切片代价 | 总额外空间    |
| --------------------------------------- | ------------ | ------------- |
| **切片版**：`merge_sort(arr[:mid])`     | (O(n))       | (O(n \log n)) |
| **双指针版**：`merge_sort(arr, l, mid)` | (O(1))       | (O(n))        |

> **(O(n \log n)) vs (O(n)) 的内存差距**：对 1 GB 数据，切片版要吃掉 ~30 GB 临时内存，**直接 OOM**；双指针版稳稳的 1 GB。

#### 6.2.2 性能直观对比

```python
import time
from typing import List


def merge_sort_slice(arr: List[int]) -> List[int]:
    """❌ 反面教材：切片版"""
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort_slice(arr[:mid])    # 拷贝 mid 个元素
    right = merge_sort_slice(arr[mid:])   # 拷贝 n-mid 个元素
    # 合并...
    merged: List[int] = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            merged.append(left[i]); i += 1
        else:
            merged.append(right[j]); j += 1
    merged.extend(left[i:]); merged.extend(right[j:])
    return merged


# ✅ 正确：见 3.1.1 节双指针版 _merge_sort
```

简单基准（(n = 100{,}000) 随机整数，仅供示意）：

```
切片版:   ~0.85 s, 峰值内存约 4× 原数组
双指针版: ~0.32 s, 峰值内存约 2× 原数组
```

#### 6.2.3 工程铁律

> **铁律**：**所有需要"对子区间递归"的分治算法，一律传 `(arr, left, right)` 三元组，不传切片**。

适用范围：归并排序、快速排序、快速选择、二分查找、最大子数组分治、归并求逆序对、线段树构建……

唯一例外：**只读、不递归、且性能不敏感**的场景（如打印调试），切片可读性更好。

## 七、总结：分治算法工程心法

1. **先想 Combine 的代价**——它决定整体复杂度，比 Divide 重要 10 倍；
2. **能原地就原地**——双指针 `(l, r)` 永远优于切片 `arr[:mid]`；
3. **永远随机化**——任何依赖输入分布的算法都要随机化兜底；
4. **递归是把双刃剑**——大规模数据先评估栈深，必要时改迭代；
5. **基线条件 (Base Case) 写在最上面**——比较顺序、等于号、空区间一次性想清楚；
6. **复杂度用主定理快速心算**——不要每次都重新展开递推。
