## 一、树的存储结构：孩子兄弟表示法

### 1.1 理论要点

孩子兄弟表示法（又称**二叉链表表示法**）是将一般树映射为二叉树的核心存储结构。其核心思想：

- 每个节点包含两个指针：**第一个孩子**（`first_child`）和**下一个兄弟**（`next_sibling`）
- 将"父 → 多子"关系拆解为"父 → 长子 → 长子的兄弟链"

> **本质**：把一棵多叉树用二叉链表来存储——左指针指向第一个孩子，右指针指向右兄弟

### 1.2 图解逻辑

以一棵普通树为例：

```css
原始树:                       孩子兄弟表示法 (二叉树视角):
        A                              A
      / | \                           /
     B  C  D                         B
    / \    |                        / \
   E   F   G                      E   C
                                    \   \
                                     F   D
                                        /
                                       G
```

**转换规则**：
- A 的第一个孩子是 B → `A.first_child = B`
- B 的下一个兄弟是 C → `B.next_sibling = C`
- C 的下一个兄弟是 D → `C.next_sibling = D`
- B 的第一个孩子是 E → `B.first_child = E`
- E 的下一个兄弟是 F → `E.next_sibling = F`
- D 的第一个孩子是 G → `D.first_child = G`

### 1.3 Python 代码实现

```python
from __future__ import annotations
from typing import Optional


class CSNode:
    """孩子兄弟表示法的树节点。

    Attributes:
        data: 节点存储的数据。
        first_child: 指向第一个孩子节点。
        next_sibling: 指向下一个兄弟节点。
    """

    def __init__(
        self,
        data: str,
        first_child: Optional[CSNode] = None,
        next_sibling: Optional[CSNode] = None,
    ) -> None:
        self.data = data
        self.first_child = first_child
        self.next_sibling = next_sibling

    def __repr__(self) -> str:
        return f"CSNode({self.data!r})"


def build_tree_from_dict(
    data: str,
    children_map: dict[str, list[str]],
) -> CSNode:
    """根据邻接表（父 → 孩子列表）构建孩子兄弟表示法的树。

    Args:
        data: 根节点的值。
        children_map: 键为父节点值，值为其孩子节点值的有序列表。

    Returns:
        以孩子兄弟表示法存储的树的根节点。
    """
    node = CSNode(data)
    children = children_map.get(data, [])
    if not children:
        return node

    # 递归构建第一个孩子
    node.first_child = build_tree_from_dict(children[0], children_map)

    # 通过兄弟链串联其余孩子
    current = node.first_child
    for child_data in children[1:]:
        current.next_sibling = build_tree_from_dict(child_data, children_map)
        current = current.next_sibling

    return node


def print_cs_tree(node: Optional[CSNode], indent: int = 0) -> None:
    """以缩进方式打印孩子兄弟树（还原为多叉树视角）。"""
    if node is None:
        return
    print("  " * indent + node.data)
    # 遍历所有孩子：先访问 first_child，再沿 next_sibling 链遍历
    child = node.first_child
    while child:
        print_cs_tree(child, indent + 1)
        child = child.next_sibling


# ── 测试用例 ──
if __name__ == "__main__":
    children_map: dict[str, list[str]] = {
        "A": ["B", "C", "D"],
        "B": ["E", "F"],
        "D": ["G"],
    }
    root = build_tree_from_dict("A", children_map)
    print("=== 孩子兄弟表示法还原的多叉树 ===")
    print_cs_tree(root)
```

### 1.4 复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 构建树 | $O(n)$ | 每个节点恰好创建一次 |
| 查找第 k 个孩子 | $O(k)$ | 需沿兄弟链顺序查找 |
| 空间复杂度 | $O(n)$ | 每个节点两个指针，共 \(2n\) 个指针域 |

### 1.5 实际应用

- **文件系统目录树**：目录的子文件 / 子目录天然适合孩子兄弟链
- **XML/HTML DOM 解析**：兄弟节点构成同级链，子节点构成孩子链
- **编译器语法树**：将多叉语法树转换为二叉结构以简化遍历

## 二、转换逻辑

### 2.1 树 → 二叉树

#### 理论要点

树转换为二叉树的规则极为简单——**孩子兄弟表示法本身就是一棵二叉树**：

1. 树中每个节点的**第一个孩子** → 二叉树的**左子节点**
2. 树中每个节点的**下一个兄弟** → 二叉树的**右子节点**

> **关键性质**：转换后的二叉树，根节点一定**没有右子树**（根节点没有兄弟）

#### 图解

```
    A                    A
  / | \                 /
 B  C  D      →       B
/ \    |              / \
E  F   G             E   C
                      \   \
                       F   D
                          /
                         G
```

#### Python 代码实现

```python
from __future__ import annotations
from typing import Optional


class BinaryTreeNode:
    """标准二叉树节点。"""

    def __init__(
        self,
        data: str,
        left: Optional[BinaryTreeNode] = None,
        right: Optional[BinaryTreeNode] = None,
    ) -> None:
        self.data = data
        self.left = left
        self.right = right

    def __repr__(self) -> str:
        return f"BTNode({self.data!r})"


def general_tree_to_binary(node: Optional[CSNode]) -> Optional[BinaryTreeNode]:
    """将孩子兄弟表示法的树转换为二叉树。

    映射规则:
        CSNode.first_child  → BinaryTreeNode.left
        CSNode.next_sibling → BinaryTreeNode.right

    Args:
        node: 孩子兄弟表示法的树节点。

    Returns:
        对应的二叉树节点，若输入为 None 则返回 None。
    """
    if node is None:
        return None

    bt_node = BinaryTreeNode(node.data)
    bt_node.left = general_tree_to_binary(node.first_child)
    bt_node.right = general_tree_to_binary(node.next_sibling)
    return bt_node


def inorder_traversal(node: Optional[BinaryTreeNode]) -> list[str]:
    """二叉树中序遍历。"""
    if node is None:
        return []
    return (
        inorder_traversal(node.left)
        + [node.data]
        + inorder_traversal(node.right)
    )


def preorder_traversal(node: Optional[BinaryTreeNode]) -> list[str]:
    """二叉树先序遍历。"""
    if node is None:
        return []
    return (
        [node.data]
        + preorder_traversal(node.left)
        + preorder_traversal(node.right)
    )


# ── 测试用例 ──
if __name__ == "__main__":
    # 沿用第一节的 build_tree_from_dict
    children_map = {"A": ["B", "C", "D"], "B": ["E", "F"], "D": ["G"]}
    cs_root = build_tree_from_dict("A", children_map)
    bt_root = general_tree_to_binary(cs_root)

    print("先序遍历:", preorder_traversal(bt_root))
    # ['A', 'B', 'E', 'F', 'C', 'D', 'G']
    print("中序遍历:", inorder_traversal(bt_root))
    # ['E', 'F', 'B', 'C', 'G', 'D', 'A']
```

### 2.2 森林 → 二叉树

#### 理论要点

森林是 \(m\) 棵互不相交的树的集合$ \{T_1, T_2, \ldots, T_m\}$。转换步骤：

1. 将每棵树 \($T_i\)$ 各自转换为二叉树 \($B_i$\)（使用 2.1 的方法）
2. 由于每棵二叉树 \($B_i$\) 的根节点**右子树为空**，将 \($B_2$\) 作为 \($B_1$\) 根的右子树，\($B_3$\) 作为 \($B_2$\) 根的右子树，依此类推

> **本质**：森林中各棵树的根互为"兄弟"

#### 图解

```css
森林:
  T1:  A       T2:  H       T3:  K
      / \           |
     B   C          I
     |
     D

步骤1 — 各自转为二叉树:
  B1:  A       B2:  H       B3:  K
      /             /
     B             I
    / \
   D   C

步骤2 — 根节点通过右链串联:
         A
        / \
       B   H
      / \ /
     D  C I
            \
             K
```

#### Python 代码实现

```python
def forest_to_binary(
    roots: list[CSNode],
) -> Optional[BinaryTreeNode]:
    """将森林（多棵孩子兄弟表示法的树）转换为单棵二叉树。

    Args:
        roots: 森林中每棵树的根节点列表。

    Returns:
        合并后的二叉树根节点。
    """
    if not roots:
        return None

    # 先将每棵树转换为二叉树
    binary_trees: list[BinaryTreeNode] = []
    for r in roots:
        bt = general_tree_to_binary(r)
        if bt is not None:
            binary_trees.append(bt)

    if not binary_trees:
        return None

    # 将各二叉树的根通过右指针串联
    for i in range(len(binary_trees) - 1):
        binary_trees[i].right = binary_trees[i + 1]

    return binary_trees[0]


# ── 测试用例 ──
if __name__ == "__main__":
    # 构建森林中的三棵树
    t1 = build_tree_from_dict("A", {"A": ["B", "C"], "B": ["D"]})
    t2 = build_tree_from_dict("H", {"H": ["I"]})
    t3 = build_tree_from_dict("K", {})

    bt = forest_to_binary([t1, t2, t3])
    print("森林转二叉树 — 先序:", preorder_traversal(bt))
    # ['A', 'B', 'D', 'C', 'H', 'I', 'K']
    print("森林转二叉树 — 中序:", inorder_traversal(bt))
    # ['D', 'B', 'C', 'A', 'I', 'H', 'K']
```

### 2.3 遍历对照表

#### 理论要点

树 / 森林的遍历与其对应二叉树的遍历之间存在严格的对应关系：

| 树的遍历 | 森林的遍历 | 对应二叉树的遍历 |
|---------|-----------|----------------|
| 先根遍历 | 先序遍历 | **先序遍历** |
| 后根遍历 | 中序遍历 | **中序遍历** |

**为什么后根 = 中序？** 以孩子兄弟二叉树的视角分析：
- 树的后根遍历：先递归遍历所有子树，最后访问根
- 映射到二叉树后，"所有子树"变成了左子树及其右链，"最后访问根"恰好是中序遍历中"左 → 根 → 右"的"根"位置

#### Python 代码——验证遍历对照

```python
def tree_preorder(node: Optional[CSNode]) -> list[str]:
    """树的先根遍历（孩子兄弟表示法）。"""
    if node is None:
        return []
    result = [node.data]
    child = node.first_child
    while child:
        result.extend(tree_preorder(child))
        child = child.next_sibling
    return result


def tree_postorder(node: Optional[CSNode]) -> list[str]:
    """树的后根遍历（孩子兄弟表示法）。"""
    if node is None:
        return []
    result: list[str] = []
    child = node.first_child
    while child:
        result.extend(tree_postorder(child))
        child = child.next_sibling
    result.append(node.data)
    return result


# ── 测试用例 ──
if __name__ == "__main__":
    children_map = {"A": ["B", "C", "D"], "B": ["E", "F"], "D": ["G"]}
    cs_root = build_tree_from_dict("A", children_map)
    bt_root = general_tree_to_binary(cs_root)

    print("树的先根遍历:", tree_preorder(cs_root))
    print("二叉树先序:  ", preorder_traversal(bt_root))
    # 两者相同: ['A', 'B', 'E', 'F', 'C', 'D', 'G']

    print("树的后根遍历:", tree_postorder(cs_root))
    print("二叉树中序:  ", inorder_traversal(bt_root))
    # 两者相同: ['E', 'F', 'B', 'C', 'G', 'D', 'A']
```

#### 复杂度分析

所有遍历的时间复杂度均为 \(O(n)\)，空间复杂度为 \(O(h)\)（\(h\) 为树高，即递归栈深度）。

#### 实际应用

- **数据库查询优化**：将多叉查询计划树转为二叉结构后，中序遍历自然产出 SQL 的执行顺序。
- **表达式求值**：算术表达式树的后根遍历直接生成后缀表达式（逆波兰式）。

## 三、哈夫曼树（Huffman Tree）

### 3.1 WPL（带权路径长度）

#### 定义

设一棵二叉树有 \(n\) 个叶子节点，第 \(i\) 个叶子的权值为 \(w_i\)，从根到该叶子的路径长度为 \(l_i\)，则该树的**带权路径长度**为：

$$
\text{WPL} = \sum_{i=1}^{n} w_i \cdot l_i
$$

> **哈夫曼树**就是 WPL 最小的二叉树（最优二叉树）

#### 示例

对于权值集合 \(\{2, 3, 7, 9\}\)：

```css
方案A (非最优):          方案B (哈夫曼树):
       *                       *
      / \                     / \
     *   9                   *   *
    / \                     / \ / \
   *   7                  2  3 7  9
  / \
 2   3

WPL_A = 2×3+3×3+7×2+9×1     WPL_B = 2×2+3×2+7×2+9×2
      = 6+9+14+9 = 38              = 4+6+14+18 = 42

再看真正的哈夫曼构造:
         *[21]
        /     \
      *[12]    9
      /   \
    *[5]   7
    / \
   2   3

WPL_H = 2×3+3×3+7×2+9×1
      = 6+9+14+9 = 38   ← 这才是最小WPL（非满二叉树）
```

### 3.2 哈夫曼树的构建

#### 理论要点（贪心算法）

1. 将所有权值作为只含根节点的独立树，放入**最小优先队列**（最小堆）
2. 每次取出权值最小的两棵树，合并为一棵新树（新根权值 = 两子权值之和）
3. 将新树放回优先队列
4. 重复步骤 2-3，直到队列中只剩一棵树，即为哈夫曼树

> 使用 `heapq` 模块实现最小堆，使每次取最小值的操作为 $O(\log n)$

#### 图解构建过程

以权值 \(\{2, 3, 7, 9\}\) 为例：

```css
初始堆:  [2, 3, 7, 9]

第1轮: 取出 2, 3 → 合并为 5
堆:  [5, 7, 9]
     5
    / \
   2   3

第2轮: 取出 5, 7 → 合并为 12
堆:  [9, 12]
      12
     /  \
    5    7
   / \
  2   3

第3轮: 取出 9, 12 → 合并为 21
堆:  [21]  ← 完成
       21
      /  \
     9    12
         / \
        5   7
       / \
      2   3
```

#### Python 代码实现

```python
from __future__ import annotations

import heapq
from typing import Optional


class HuffmanNode:
    """哈夫曼树节点。

    Attributes:
        weight: 节点权值。
        char: 叶子节点对应的字符（内部节点为 None）。
        left: 左子节点。
        right: 右子节点。
    """

    def __init__(
        self,
        weight: int,
        char: Optional[str] = None,
        left: Optional[HuffmanNode] = None,
        right: Optional[HuffmanNode] = None,
    ) -> None:
        self.weight = weight
        self.char = char
        self.left = left
        self.right = right

    # 为 heapq 提供比较方法（仅按 weight 排序）
    def __lt__(self, other: HuffmanNode) -> bool:
        return self.weight < other.weight

    def __repr__(self) -> str:
        tag = self.char if self.char else "*"
        return f"HNode({tag}, w={self.weight})"


def build_huffman_tree(freq_map: dict[str, int]) -> Optional[HuffmanNode]:
    """使用最小堆构建哈夫曼树。

    Args:
        freq_map: 字符 → 频率（权值）的映射。

    Returns:
        哈夫曼树的根节点。若输入为空则返回 None。

    Raises:
        ValueError: 若频率映射为空。
    """
    if not freq_map:
        raise ValueError("频率映射不能为空")

    # 初始化最小堆：每个字符生成一个叶子节点
    heap: list[HuffmanNode] = [
        HuffmanNode(weight=w, char=c) for c, w in freq_map.items()
    ]
    heapq.heapify(heap)  # O(n) 建堆

    # 反复合并最小的两棵树
    while len(heap) > 1:
        left = heapq.heappop(heap)   # 取最小
        right = heapq.heappop(heap)  # 取次小
        merged = HuffmanNode(
            weight=left.weight + right.weight,
            left=left,
            right=right,
        )
        heapq.heappush(heap, merged)

    return heap[0]


def calculate_wpl(
    node: Optional[HuffmanNode],
    depth: int = 0,
) -> int:
    """计算哈夫曼树的带权路径长度 (WPL)。

    Args:
        node: 当前节点。
        depth: 当前节点的深度。

    Returns:
        以该节点为根的子树的 WPL。
    """
    if node is None:
        return 0
    # 叶子节点：贡献 = 权值 × 深度
    if node.left is None and node.right is None:
        return node.weight * depth
    return (
        calculate_wpl(node.left, depth + 1)
        + calculate_wpl(node.right, depth + 1)
    )


# ── 测试用例 ──
if __name__ == "__main__":
    freq = {"a": 2, "b": 3, "c": 7, "d": 9}
    root = build_huffman_tree(freq)
    print(f"哈夫曼树根节点: {root}")
    print(f"WPL = {calculate_wpl(root)}")
    # WPL = 2*3 + 3*3 + 7*2 + 9*1 = 38（可能因堆排列顺序不同而结构不同，但 WPL 相同）
```

### 3.3 哈夫曼编码（Huffman Coding）

#### 理论要点

哈夫曼编码是一种**前缀编码**（任何字符的编码都不是其他字符编码的前缀），其特点：

- 高频字符 → 短编码
- 低频字符 → 长编码
- 从根到叶子节点的路径决定编码：**左走记 `0`，右走记 `1`**

> **信息论意义**：哈夫曼编码是在已知字符频率时，平均码长最短的前缀编码方案

#### Python 代码实现

```python
def generate_huffman_codes(
    root: Optional[HuffmanNode],
) -> dict[str, str]:
    """遍历哈夫曼树，为每个叶子节点生成编码。

    约定: 左子树路径为 '0'，右子树路径为 '1'。

    Args:
        root: 哈夫曼树根节点。

    Returns:
        字符 → 哈夫曼编码的映射字典。
    """
    codes: dict[str, str] = {}

    def _dfs(node: Optional[HuffmanNode], current_code: str) -> None:
        if node is None:
            return
        # 叶子节点 → 记录编码
        if node.char is not None:
            codes[node.char] = current_code if current_code else "0"
            return
        _dfs(node.left, current_code + "0")
        _dfs(node.right, current_code + "1")

    _dfs(root, "")
    return codes


def huffman_encode(text: str, codes: dict[str, str]) -> str:
    """使用哈夫曼编码表对文本进行编码。

    Args:
        text: 原始文本。
        codes: 字符 → 编码映射。

    Returns:
        编码后的二进制字符串。
    """
    return "".join(codes[ch] for ch in text)


def huffman_decode(
    encoded: str,
    root: Optional[HuffmanNode],
) -> str:
    """使用哈夫曼树对二进制字符串进行解码。

    Args:
        encoded: 编码后的二进制字符串。
        root: 哈夫曼树根节点。

    Returns:
        解码后的原始文本。
    """
    if root is None:
        return ""

    result: list[str] = []
    current = root

    for bit in encoded:
        if bit == "0":
            current = current.left  # type: ignore[assignment]
        else:
            current = current.right  # type: ignore[assignment]

        # 到达叶子节点 → 解码出一个字符
        if current.left is None and current.right is None:
            result.append(current.char or "")
            current = root  # 回到根节点继续解码

    return "".join(result)


# ── 完整测试用例 ──
if __name__ == "__main__":
    # 1. 统计频率
    text = "aabbbcccccccdddddddd"
    freq_map: dict[str, int] = {}
    for ch in text:
        freq_map[ch] = freq_map.get(ch, 0) + 1
    print(f"字符频率: {freq_map}")
    # {'a': 2, 'b': 3, 'c': 7, 'd': 8}

    # 2. 构建哈夫曼树
    root = build_huffman_tree(freq_map)
    print(f"WPL = {calculate_wpl(root)}")

    # 3. 生成编码
    codes = generate_huffman_codes(root)
    print(f"哈夫曼编码表: {codes}")
    # 可能的输出（编码可能因堆顺序不同而不同，但码长一致）:
    # {'d': '0', 'c': '10', 'a': '110', 'b': '111'}

    # 4. 编码
    encoded = huffman_encode(text, codes)
    print(f"编码结果: {encoded}")
    print(f"原始文本长度: {len(text)} 字符")
    print(f"编码后长度:   {len(encoded)} 比特")

    # 5. 解码验证
    decoded = huffman_decode(encoded, root)
    print(f"解码结果: {decoded}")
    assert decoded == text, "解码失败！"
    print("✓ 编解码验证通过")

    # 6. 压缩率
    original_bits = len(text) * 8  # ASCII 每字符 8 位
    ratio = len(encoded) / original_bits * 100
    print(f"压缩率: {ratio:.1f}% (原 {original_bits} 位 → 现 {len(encoded)} 位)")
```

### 3.4 复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 构建哈夫曼树 | $O(n \log n)$ | \(n\) 个字符，每次堆操作 \(O(\log n)\)，共 \(n-1\) 次合并 |
| 生成编码表 | $O(n)$ | DFS 遍历所有叶子节点 |
| 编码文本 | $O(m)$ | \(m\) 为文本长度，查表 \(O(1)\) |
| 解码文本 | $O(m)$ | \(m\) 为编码串长度，逐位沿树走 |
| 空间复杂度 | $O(n)$ | 哈夫曼树共 \(2n - 1\) 个节点 |

### 3.5 实际应用

- **文件压缩**：ZIP、GZIP 格式内部使用哈夫曼编码（DEFLATE 算法的核心组件）
- **图像压缩**：JPEG 格式在量化后使用哈夫曼编码进行无损压缩
- **网络传输**：HTTP/2 的 HPACK 头部压缩使用静态哈夫曼表
- **通信系统**：在字符频率已知的场景中，哈夫曼编码可达到接近信息熵的平均码长

| 主题 | 核心思想 | 关键操作复杂度 |
|------|---------|---------------|
| 孩子兄弟表示法 | 左指针 = 长子，右指针 = 兄弟 | 构建 $O(n)$，查第 k 子 $O(k)$ |

| 哈夫曼树 | 贪心：每次合并最小两棵 | 构建 $O(n \log n)$ |
| 哈夫曼编码 | 前缀码，高频短编码 | 编解码 $O(m)$ |
