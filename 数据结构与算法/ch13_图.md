## 1. 图的基本概念与分类

### 1.1 图的数学定义

图（Graph）是一种非线性数据结构，形式化定义为一个二元组：

$$
G = (V, E)
$$
其中：

- $V$（Vertices）是**顶点的有限集合**，$V = \{v_1, v_2, \dots, v_n\}$
- $E$（Edges）是**边的有限集合**，每条边是顶点的二元组 $e = (u, v)$，其中 $u, v \in V$

若边带有权重，则进一步定义权重函数 $w: E \to \mathbb{R}$

### 1.2 顶点与边

| 术语 | 含义 |
|:----:|:----:|
| 顶点（Vertex / Node） | 图中的基本单元，代表一个对象（如城市、用户） |
| 边（Edge / Link） | 两个顶点之间的关系（如道路、好友关系） |
| 度（Degree） | 无向图中与一个顶点相连的边数 |
| 入度 / 出度 | 有向图中指向顶点 / 从顶点出发的边数 |
| 邻接（Adjacent） | 两个顶点之间存在一条边 |
| 路径（Path） | 由边依次连接的顶点序列 |
| 环（Cycle） | 起点和终点相同的路径 |

### 1.3 有向图 vs 无向图

- **无向图（Undirected Graph）**：边没有方向，$(u, v)$等价于 $(v, u)$。例：微信好友关系

   **有向图（Directed Graph / Digraph）**：边有方向，$(u, v)$表示从 $u$到 $v$ 的一条弧。例：微博关注关系、网页超链接

### 1.4 加权图 vs 非加权图

- **加权图（Weighted Graph）**：每条边带一个权值 $w(u, v)$。例：地图的距离、时间、费用
- **非加权图**：边无权值（或默认权值为 1）

### 1.5 特殊图

| 类型 | 说明 |
|:----:|:----:|
| 简单图 | 无自环、无重复边的图 |
| 多重图 | 允许两顶点之间有多条边 |
| 连通图 | 无向图中任意两顶点之间都存在路径 |
| 强连通图 | 有向图中任意两顶点互相可达 |
| 完全图 | 任意两顶点之间都有边，边数为 $\binom{n}{2}$ 或 $n(n-1)$ |
| 二分图 | 顶点可分成两组，边只在组间存在 |
| 树 | **连通且无环**的无向图，边数恰好为 \(n-1\) |
| DAG | 有向无环图，拓扑排序的前提 |

> **树 vs 图的关系**：树是图的一个特例。所有树都是图，但图不一定是树

### 1.6 稀疏图 vs 稠密图

- **稀疏图（Sparse Graph）**：$|E| \ll |V|^2$。推荐使用**邻接表**
- **稠密图（Dense Graph）**：$|E|$ 接近 $|V|^2$。可使用**邻接矩阵**

### 1.7 实际应用场景

| 场景 | 建模方式 |
|:----:|:--------:|
| 社交网络 | 用户为顶点，好友/关注关系为边 |
| 地图导航 | 路口为顶点，道路为边（加权） |
| 依赖关系 | 模块/任务为顶点，依赖为有向边（DAG） |
| 网页爬虫 | 网页为顶点，超链接为有向边 |
| 推荐系统 | 用户-商品二分图 |
| 编译器 | 函数调用关系图、数据流图 |

### 本节小结 + 面试高频点

- ✅ 必须掌握图的数学定义$ G = (V, E)$
- ✅ 能区分有向/无向、加权/非加权、稠密/稀疏图
- 🔥 面试高频问：**什么时候用邻接表，什么时候用邻接矩阵？**（答：看稀疏还是稠密）
- 🔥 树 vs 图：树是连通无环的特殊图

## 2. 图的两种主要表示方式

### 2.1 邻接矩阵（Adjacency Matrix）

#### 原理

使用一个 $n \times n$的二维矩阵$A$，其中：

$$
A_{ij} = 
\begin{cases}
1 \quad (\text{或权重 } w) & \text{若 } (v_i, v_j) \in E \\
0 \quad (\text{或 } \infty) & \text{否则}
\end{cases}
$$

#### 优缺点

| 维度 | 说明 |
|:----:|:----:|
| 优点 | 查询两顶点是否相邻 $O(1)$；实现简单；适合稠密图 |
| 缺点 | 空间复杂度 $O(V^2)$，稀疏图浪费严重；遍历邻居需要 $O(V)$ |

#### Python 实现

```python
from __future__ import annotations
from typing import Optional


class GraphAdjMatrix:
    """基于邻接矩阵的无向图实现。

    用 list[list[int]] 存储邻接矩阵。0 表示无边，1 表示有边。
    可自行扩展为加权图（将 1 替换为权重，0 替换为 float('inf')）。
    """

    def __init__(self, vertices: list[int], edges: list[list[int]]) -> None:
        # self.vertices 存储所有顶点值；顶点在矩阵中的索引与列表位置一一对应
        # 注意事项：若要支持删除顶点，必须同步更新矩阵中的行和列
        self.vertices: list[int] = list(vertices)
        n: int = len(self.vertices)
        # 用列表推导式初始化 n x n 的全零矩阵
        # 时间复杂度：O(n^2)；空间复杂度：O(n^2)
        self.adj_mat: list[list[int]] = [[0] * n for _ in range(n)]

        # 批量添加边
        # 解包（unpacking）：当写 for a, b in some_list 时，Python 会要求每个元素正好有 2 个值，否则会报错。自动解包，取的就是列表的第一个元素，而第一个元素又是一个列表，表示一条边[a,b]
        for u, v in edges:
            self.add_edge(u, v)

    def size(self) -> int:
        """返回顶点数量，时间复杂度 O(1)。"""
        return len(self.vertices)

    def add_vertex(self, val: int) -> None:
        """添加顶点：在 vertices 末尾追加，同时给矩阵加一行一列。

        时间复杂度：O(n)，因为每行要追加一个 0。
        """
        n: int = self.size()
        self.vertices.append(val)
        # 为已存在的每一行末尾追加一个 0
        for row in self.adj_mat:
            row.append(0)
        # 新增一行（长度 n+1），对应新顶点的出边
        self.adj_mat.append([0] * (n + 1))

    def remove_vertex(self, index: int) -> None:
        """按索引删除顶点，并同步删除对应的行和列。

        时间复杂度：O(n^2)，因为要从每一行里 pop 一个元素。
        """
        if index < 0 or index >= self.size():
            raise IndexError(f"顶点索引 {index} 越界")
        # pop 顶点
        self.vertices.pop(index)
        # pop 对应行
        self.adj_mat.pop(index)
        # 对剩余每一行，删除第 index 列
        for row in self.adj_mat:
            row.pop(index)

    def add_edge(self, i: int, j: int) -> None:
        """在索引为 i、j 的两顶点之间添加无向边。时间复杂度 O(1)。"""
        if i < 0 or j < 0 or i >= self.size() or j >= self.size() or i == j:
            raise IndexError(f"非法边 ({i}, {j})")
        # 无向图：两个方向都要置 1
        self.adj_mat[i][j] = 1
        self.adj_mat[j][i] = 1

    def remove_edge(self, i: int, j: int) -> None:
        """删除一条无向边。时间复杂度 O(1)。"""
        if i < 0 or j < 0 or i >= self.size() or j >= self.size() or i == j:
            raise IndexError(f"非法边 ({i}, {j})")
        self.adj_mat[i][j] = 0
        self.adj_mat[j][i] = 0

    def print(self) -> None:
        """打印邻接矩阵，方便调试。"""
        print("顶点列表:", self.vertices)
        print("邻接矩阵:")
        for row in self.adj_mat:
            print(row)


if __name__ == "__main__":
    # 简单测试
    g = GraphAdjMatrix(vertices=[1, 2, 3, 4], edges=[[0, 1], [1, 2], [2, 3]])
    g.print()
    g.add_vertex(5)
    g.add_edge(3, 4)  # 4 的索引是 4（第五个顶点）
    g.print()
```

> **numpy 对比**：若需要大规模稠密图运算，可用 `np.zeros((n, n), dtype=np.int8)` 替代二维列表，内存更紧凑，矩阵运算更快；但对动态增删顶点不友好

#### 🐍 Python 语法小回顾

- **二维列表初始化**：`[[0] * n for _ in range(n)]` 正确；**不要**写成 `[[0] * n] * n`，那样会共用同一个子列表引用
- **列表索引与切片**：`row[index]` 为$O(1)$；`list.pop(index)` 会让后续元素前移，为$ O(n)$
- **异常抛出**：`raise IndexError("...")` 用于传递清晰的错误信息

### 2.2 邻接表（Adjacency List）——推荐方式

#### 原理

对每个顶点维护一个它所连接的邻居列表：

$$
\text{Adj}[v] = { u \mid (v, u) \in E }
$$

#### 优缺点

| 维度 | 说明 |
|:----:|:----:|
| 优点 | 空间复杂度 $O(V + E)$，稀疏图极省空间；遍历邻居高效 |
| 缺点 | 判断两顶点是否相邻需要 $O(\deg(v))$；实现稍复杂 |

#### 面向对象设计：`Vertex` 类 + `GraphAdjList` 类

```python
from __future__ import annotations
from typing import Optional


class Vertex:
    """顶点类：包装顶点的值。

    为什么用类而不是直接用整数/字符串？
    1. 避免值相同但语义不同的顶点冲突（通过对象 id 区分）。
    2. 未来可扩展属性，例如颜色（用于二分图）、距离（用于 Dijkstra）。
    """

    def __init__(self, val: int) -> None:
        self.val: int = val

    def __repr__(self) -> str:
        # 方便打印调试，遵循 "开发者可读" 原则
        return f"V({self.val})"


class GraphAdjList:
    """基于邻接表的无向图实现。

    核心数据结构：
        self.adj_list: dict[Vertex, list[Vertex]]   顶点：与该顶点相连的点的列表
        键为顶点对象，值为其邻居列表。
    """

    def __init__(self, edges: list[list[Vertex]]) -> None:
        # 使用 dict 做邻接表，查找 O(1)
        self.adj_list: dict[Vertex, list[Vertex]] = {}
        # 逐条加入边；add_edge 内部会自动调用 add_vertex 保证顶点已存在
        for edge in edges:
            self.add_vertex(edge[0])
            self.add_vertex(edge[1])
            self.add_edge(edge[0], edge[1])

    # ---------- 基本查询 ----------
    def size(self) -> int:
        """返回顶点数，时间复杂度 O(1)。"""
        return len(self.adj_list)

    # ---------- 顶点操作 ----------
    def add_vertex(self, vet: Vertex) -> None:
        """添加顶点。若已存在则跳过。

        时间复杂度：O(1)（字典键存在性判断 + 赋值均摊 O(1)）。
        """
        if vet in self.adj_list:
            return
        # 新顶点的邻居列表初始化为空
        self.adj_list[vet] = []

    def remove_vertex(self, vet: Vertex) -> None:
        """删除顶点及其所有相连的边。

        时间复杂度：O(V + E)，因为要扫描所有邻居列表清理对该顶点的引用。
        """
        if vet not in self.adj_list:
            raise ValueError(f"顶点 {vet} 不存在")
        # 1. 先从字典中移除该顶点
        self.adj_list.pop(vet)
        # 2. 再遍历其他顶点的邻居列表，移除所有指向 vet 的引用
        #    使用列表推导式重新赋值，比 list.remove 更清晰
        for neighbors in self.adj_list.values():
            if vet in neighbors:
                neighbors.remove(vet)

    # ---------- 边操作 ----------
    def add_edge(self, vet1: Vertex, vet2: Vertex) -> None:
        """添加无向边。两端必须已存在，且不允许自环。

        时间复杂度：O(1)。
        """
        if vet1 not in self.adj_list or vet2 not in self.adj_list:
            raise ValueError("边的端点必须先加入图")
        if vet1 is vet2:
            raise ValueError("不允许自环")
        # 无向图：双向追加
        self.adj_list[vet1].append(vet2)
        self.adj_list[vet2].append(vet1)

    def remove_edge(self, vet1: Vertex, vet2: Vertex) -> None:
        """删除无向边。若边不存在则抛异常。

        时间复杂度：O(deg(vet1) + deg(vet2))。
        """
        if vet1 not in self.adj_list or vet2 not in self.adj_list:
            raise ValueError("边的端点不存在")
        if vet1 is vet2:
            raise ValueError("不允许自环")
        # list.remove 在第一次匹配到时删除，O(n)
        self.adj_list[vet1].remove(vet2)
        self.adj_list[vet2].remove(vet1)

    # ---------- 辅助方法 ----------
    def print(self) -> None:
        """打印邻接表。"""
        print("邻接表:")
        for vet, neighbors in self.adj_list.items():
            print(f"  {vet}: {neighbors}")


if __name__ == "__main__":
    v1, v2, v3, v4, v5 = Vertex(1), Vertex(2), Vertex(3), Vertex(4), Vertex(5)
    g = GraphAdjList(edges=[[v1, v2], [v2, v3], [v3, v4], [v4, v5], [v1, v5]])
    g.print()
    g.add_vertex(Vertex(6))
    g.remove_edge(v1, v5)
    g.print()
```

#### 🐍 Python 语法小回顾

- **泛型字典类型**：`dict[Vertex, list[Vertex]]`（Python 3.9+）。早期版本需写 `Dict[Vertex, List[Vertex]]` 并 `from typing import Dict, List`
- **`dict.pop(key)`**：返回并删除键，若不存在则抛 `KeyError`
- **`list.remove(x)`**：删除第一次出现的 \(x\)，$O(n)$；找不到会抛 `ValueError`
- **列表推导式**：`[x for x in lst if cond]`，用于生成新列表，语义比 `filter` 更直观
- **`is` vs `==`**：前者比较对象身份，后者比较值。`vet1 is vet2` 用来严格判断是否同一个对象

### 本节小结 + 面试高频点

- ✅ 邻接矩阵：空间 $O(V^2)$，查询 $O(1)$，适合稠密图
- ✅ 邻接表：空间 $O(V+E)$，遍历邻居高效，适合稀疏图与大多数面试题
- 🔥 面试高频：**手撕邻接表**、**对比两种表示方法**
- 🔥 注意无向图添加边要**双向**写入，这是常见 bug 来源

## 3. 图的基本操作与维护

### 3.1 关键注意事项

1. **无向图的对称性**：添加/删除边时必须同时更新 `adj[u]` 和 `adj[v]`
2. **删除顶点必须清理所有指向它的边**，否则会出现"悬空引用"
3. **避免自环与重复边**：简单图必须在添加边时做校验
4. **Python 字典的键要求可哈希**：`Vertex` 类默认继承 `object` 的 `__hash__`，按对象 id 哈希，天然唯一

### 3.2 异常处理规范

```python
# 典型模式：先校验，后操作
if vet not in self.adj_list:
    raise ValueError(f"顶点 {vet} 不存在")
```

使用专门的异常类型（`IndexError`、`ValueError`、`KeyError`）能让调用方精确 catch

### 3.3 时间复杂度对照表

| 操作 | 邻接矩阵 | 邻接表 |
|:----:|:--------:|:------:|
| 添加顶点 | $O(V)$ | $O(1)$ |
| 删除顶点 | $O(V^2)$ | $O(V + E)$ |
| 添加边 | $O(1)$ | $O(1)$ |
| 删除边 | $O(1)$ | $O(\deg(v))$ |
| 查询边是否存在 | $O(1)$ | $O(\deg(v))$ |
| 遍历某顶点所有邻居 | $O(V)$ | $O(\deg(v))$ |
| 空间复杂度 | $O(V^2)$ | $O(V + E)$ |

### 本节小结 + 面试高频点

- 🔥 **删除顶点**的复杂度差异是常考点：矩阵要 resize 所有行列，邻接表只需清理相关边
- 🔥 实际工程中，90% 的场景用邻接表

## 4. 图的遍历算法

图的遍历是从某个起点出发，访问所有可达顶点，每个顶点最多访问一次

### 4.1 广度优先遍历（BFS）

#### 原理

- 从起点出发，按**层级**逐层扩散
- 访问完第 \(k\) 层的所有顶点后，再访问第 \(k+1\) 层
- 借助**队列**（FIFO）实现
- 天然适合**求最短路径**（非加权图）、**最短步数**问题

#### 复杂度

- 时间：$O(V + E)$
- 空间：$O(V)$（访问集合 + 队列）

#### Python 实现

```python
from __future__ import annotations
from collections import deque
from typing import Optional


def bfs(graph: GraphAdjList, start: Vertex) -> list[Vertex]:
    """广度优先遍历，返回访问顺序。

    参数:
        graph: 邻接表表示的图
        start: 起始顶点

    返回:
        按 BFS 顺序访问的顶点列表
    """
    # visited 集合避免重复访问；set 的查找为 O(1) 均摊
    visited: set[Vertex] = {start}
    # deque 是双端队列，popleft 为 O(1)，是 BFS 的标准选择
    # 切勿使用 list.pop(0)，那是 O(n)
    queue: deque[Vertex] = deque([start])
    order: list[Vertex] = []

    while queue:
        vet = queue.popleft()           # 出队一个顶点
        order.append(vet)               # 记录访问顺序
        # 遍历所有邻居：未访问则入队
        for neighbor in graph.adj_list[vet]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

    return order


if __name__ == "__main__":
    v1, v2, v3, v4, v5 = Vertex(1), Vertex(2), Vertex(3), Vertex(4), Vertex(5)
    g = GraphAdjList(edges=[[v1, v2], [v2, v3], [v3, v4], [v4, v5], [v1, v5]])
    print("BFS 顺序:", bfs(g, v1))
```

#### 🐍 Python 语法小回顾

- **`collections.deque`**：双端队列，`append`/`appendleft`/`pop`/`popleft` 均为 $O(1)$。BFS 必备
- **`set[Vertex]`**：泛型集合，$O(1)$平均查找/插入
- **`while queue:`**：利用容器的"真值"特性，空集合/空队列自动为 `False`

### 4.2 深度优先遍历（DFS）

#### 原理

- 从起点出发，**沿一条路径尽可能深**地访问，直到无路可走再回溯
- 可用**递归**（隐式栈）或**显式栈**实现
- 是**回溯算法**、**拓扑排序**、**环检测**的基础

#### 复杂度

- 时间：$O(V + E)$
- 空间：$O(V)$（递归栈深度最坏为 $V$）

#### 递归版实现

```python
from __future__ import annotations


def _dfs(graph: GraphAdjList, visited: set[Vertex], order: list[Vertex], vet: Vertex) -> None:
    """DFS 递归辅助函数。

    参数:
        graph: 图
        visited: 已访问集合（引用传递，递归间共享状态）
        order: 访问顺序列表（引用传递）
        vet: 当前访问的顶点
    """
    visited.add(vet)            # 标记当前顶点已访问
    order.append(vet)           # 先序记录
    # 对每个未访问的邻居递归下去
    for neighbor in graph.adj_list[vet]:
        if neighbor not in visited:
            _dfs(graph, visited, order, neighbor)


def dfs(graph: GraphAdjList, start: Vertex) -> list[Vertex]:
    """DFS 主函数，返回访问顺序。"""
    visited: set[Vertex] = set()
    order: list[Vertex] = []
    _dfs(graph, visited, order, start)
    return order
```

#### 迭代版实现（使用显式栈）

```python
def dfs_iterative(graph: GraphAdjList, start: Vertex) -> list[Vertex]:
    """DFS 迭代版：用显式栈模拟递归。

    优点：避免 Python 递归深度限制（默认 1000），适合大图。
    注意：访问顺序与递归版可能略有不同，取决于邻居压栈顺序。
    """
    visited: set[Vertex] = set()
    order: list[Vertex] = []
    stack: list[Vertex] = [start]

    while stack:
        vet = stack.pop()       # 栈顶弹出，O(1)
        if vet in visited:
            continue
        visited.add(vet)
        order.append(vet)
        # 为了模拟递归的顺序，可倒序压栈
        for neighbor in reversed(graph.adj_list[vet]):
            if neighbor not in visited:
                stack.append(neighbor)

    return order
```

#### 🐍 Python 语法小回顾

- **递归函数命名**：常用前缀下划线 `_dfs` 表示"内部辅助"，不建议外部直接调用
- **可变对象作参数**：`visited`、`order` 是按引用传递的，递归中修改会反映到外层
- **`sys.setrecursionlimit(10000)`**：当递归深度超过默认 1000 时可调整，但更推荐改写成迭代版

### 4.3 BFS vs DFS 对比

| 维度 | BFS | DFS |
|:----:|:---:|:---:|
| 数据结构 | 队列（`deque`） | 栈（显式 list 或递归） |
| 访问顺序 | 按层级 | 沿路径深入 |
| 空间复杂度 | $O(V)$，队列最多存一层 | $O(V)$，最坏等于树高 |
| 典型应用 | 最短路径（无权）、连通性、层序遍历 | 环检测、拓扑排序、回溯、连通分量 |
| 路径特性 | 保证步数最少 | 不保证最短路径 |
| 实现偏好 | 迭代更自然 | 递归更简洁 |

### 本节小结 + 面试高频点

- 🔥 **BFS 用 deque**，切忌 `list.pop(0)`
- 🔥 **DFS 递归要注意 `visited` 标记时机**：通常在"进入函数时"标记，避免重复压栈
- 🔥 **"最短步数"→ BFS**，**"所有路径" / "是否有环"→ DFS**
- 🔥 LeetCode 200（岛屿数量）、LeetCode 127（单词接龙）、LeetCode 207（课程表）皆为高频

## 5. 图的高级算法与经典问题

### 5.1 环检测（Cycle Detection）

#### 5.1.1 有向图环检测（三色标记法）

核心思想：每个顶点有三种状态

- `WHITE`：未访问
- `GRAY`：访问中（在当前 DFS 路径上）
- `BLACK`：已完成

若 DFS 过程中遇到 `GRAY` 顶点，说明存在**回边** → 有环。

```python
from __future__ import annotations


WHITE, GRAY, BLACK = 0, 1, 2


def has_cycle_directed(graph: dict[int, list[int]]) -> bool:
    """检测有向图是否存在环，使用三色标记法。

    参数:
        graph: 邻接表，键为顶点 id，值为邻居 id 列表

    返回:
        True 表示有环，False 表示无环（DAG）
    """
    color: dict[int, int] = {u: WHITE for u in graph}

    def dfs(u: int) -> bool:
        color[u] = GRAY                     # 进入：标记为灰
        for v in graph[u]:
            if color[v] == GRAY:            # 回边 → 有环
                return True
            if color[v] == WHITE and dfs(v):
                return True
        color[u] = BLACK                    # 完成：标记为黑
        return False

    # 图可能不连通，需对每个未访问顶点发起 DFS
    for u in graph:
        if color[u] == WHITE and dfs(u):
            return True
    return False
```

#### 5.1.2 无向图环检测（DFS 父节点法）

```python
def has_cycle_undirected(graph: dict[int, list[int]]) -> bool:
    """无向图环检测：DFS 时记录父节点，遇到已访问但非父节点即有环。"""
    visited: set[int] = set()

    def dfs(u: int, parent: int) -> bool:
        visited.add(u)
        for v in graph[u]:
            if v not in visited:
                if dfs(v, u):
                    return True
            elif v != parent:           # 访问过且非父节点 → 有环
                return True
        return False

    for u in graph:
        if u not in visited:
            if dfs(u, -1):
                return True
    return False
```

> 也可用**并查集**（Union-Find）检测无向图环：合并前若两端点已同根，则有环。

### 5.2 连通分量与连通性判断

```python
def count_connected_components(graph: dict[int, list[int]]) -> int:
    """统计无向图连通分量数量。

    思路：对每个未访问顶点发起 BFS/DFS，每次发起计数 +1。
    时间复杂度：O(V + E)
    """
    visited: set[int] = set()
    count: int = 0

    def dfs(u: int) -> None:
        visited.add(u)
        for v in graph[u]:
            if v not in visited:
                dfs(v)

    for u in graph:
        if u not in visited:
            dfs(u)
            count += 1

    return count
```

**应用**：LeetCode 547（省份数量）、LeetCode 200（岛屿数量）。

### 5.3 拓扑排序（Topological Sort）

#### 场景

- 课程安排（先修课 → 后续课）
- 编译依赖（.h → .cpp → .o）
- 任务调度（DAG 任务流）

前提：**有向无环图（DAG）**。

#### Kahn 算法（BFS 版）

```python
from collections import deque


def topological_sort_bfs(graph: dict[int, list[int]]) -> list[int]:
    """Kahn 算法：基于入度的 BFS 拓扑排序。

    步骤:
        1. 计算每个顶点的入度
        2. 入度为 0 的顶点入队
        3. 出队一个顶点 u，将其加入结果；u 的每个邻居入度 -1，若变为 0 则入队
        4. 若最终结果包含所有顶点，则返回；否则说明有环
    时间复杂度：O(V + E)
    """
    in_degree: dict[int, int] = {u: 0 for u in graph}
    for u in graph:
        for v in graph[u]:
            in_degree[v] = in_degree.get(v, 0) + 1

    # 初始入度为 0 的顶点入队
    queue: deque[int] = deque([u for u in in_degree if in_degree[u] == 0])
    order: list[int] = []

    while queue:
        u = queue.popleft()
        order.append(u)
        for v in graph[u]:
            in_degree[v] -= 1
            if in_degree[v] == 0:
                queue.append(v)

    if len(order) != len(in_degree):
        raise ValueError("图中存在环，无法拓扑排序")
    return order
```

#### DFS 版

```python
def topological_sort_dfs(graph: dict[int, list[int]]) -> list[int]:
    """DFS 拓扑排序：后序压栈，最终反转。

    关键：在 DFS 完成（回溯前）才将顶点压入结果栈。
    """
    visited: set[int] = set()
    on_path: set[int] = set()   # 检测环
    order: list[int] = []

    def dfs(u: int) -> None:
        if u in on_path:
            raise ValueError("图中存在环")
        if u in visited:
            return
        on_path.add(u)
        for v in graph[u]:
            dfs(v)
        on_path.remove(u)
        visited.add(u)
        order.append(u)          # 后序加入

    for u in graph:
        if u not in visited:
            dfs(u)

    return order[::-1]            # 反转得到拓扑序
```

**应用**：LeetCode 207（课程表）、LeetCode 210（课程表 II）。

### 5.4 最短路径

#### 5.4.1 Dijkstra 算法（非负权）

适用条件：**所有边权非负**。
时间复杂度：使用二叉堆 \(O((V + E) \log V)\)。

```python
import heapq
import math
from typing import Optional


def dijkstra(graph: dict[int, list[tuple[int, float]]], start: int) -> dict[int, float]:
    """Dijkstra 最短路径算法。

    参数:
        graph: 加权邻接表，键为顶点，值为 [(邻居, 权重), ...]
        start: 起点

    返回:
        从 start 到每个顶点的最短距离字典
    """
    # 初始化所有距离为无穷大
    dist: dict[int, float] = {u: math.inf for u in graph}
    dist[start] = 0.0
    # 优先队列（最小堆），元素为 (当前距离, 顶点)
    pq: list[tuple[float, int]] = [(0.0, start)]

    while pq:
        d, u = heapq.heappop(pq)
        # 关键优化：若堆顶距离已过期，直接跳过
        # 这是因为同一个顶点可能因多次松弛而被多次入堆
        if d > dist[u]:
            continue
        for v, w in graph[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))

    return dist
```

> **为什么 Dijkstra 要求非负权？** 因为一旦一个顶点"最短距离"被确定（从堆中弹出），就不再更新；负权可能使之后出现更短路径，破坏贪心性。

#### 5.4.2 Bellman-Ford 算法（支持负权）

时间复杂度 \(O(V \cdot E)\)。

```python
def bellman_ford(
    vertices: list[int],
    edges: list[tuple[int, int, float]],
    start: int,
) -> dict[int, float]:
    """Bellman-Ford 最短路径，可处理负权边并检测负权环。

    参数:
        vertices: 顶点列表
        edges: [(u, v, w), ...] 有向边及权重
        start: 起点
    """
    dist: dict[int, float] = {v: math.inf for v in vertices}
    dist[start] = 0.0

    # V-1 轮松弛：每轮对所有边尝试更新
    for _ in range(len(vertices) - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                updated = True
        if not updated:
            break  # 提前终止优化

    # 第 V 轮若仍能松弛，说明存在负权环
    for u, v, w in edges:
        if dist[u] + w < dist[v]:
            raise ValueError("图中存在从起点可达的负权环")

    return dist
```

**应用**：LeetCode 743（网络延迟时间）、LeetCode 787（K 站中转内最便宜的航班）。

### 5.5 最小生成树（MST）

给定连通无向加权图，MST 是连接所有顶点且边权和最小的无环子图（即树），包含 \(V - 1\) 条边。

#### 5.5.1 Prim 算法（基于顶点）

思路：从任意顶点出发，每次选择"离当前树最近的外部顶点"加入。

```python
import heapq


def prim_mst(graph: dict[int, list[tuple[int, float]]]) -> float:
    """Prim 算法求 MST 的总权重。

    时间复杂度：O((V + E) log V)
    """
    if not graph:
        return 0.0
    start = next(iter(graph))
    visited: set[int] = {start}
    # 堆中存所有从 visited 出发的边 (权重, 目标顶点)
    pq: list[tuple[float, int]] = [(w, v) for v, w in graph[start]]
    heapq.heapify(pq)
    total: float = 0.0

    while pq and len(visited) < len(graph):
        w, v = heapq.heappop(pq)
        if v in visited:
            continue
        visited.add(v)
        total += w
        for nxt, nw in graph[v]:
            if nxt not in visited:
                heapq.heappush(pq, (nw, nxt))

    return total
```

#### 5.5.2 Kruskal 算法（基于边 + 并查集）

思路：将所有边按权重升序排序，依次选边，若不形成环则加入。用**并查集**检测环。

```python
class UnionFind:
    """并查集：路径压缩 + 按秩合并，近似 O(α(n))。"""

    def __init__(self, n: int) -> None:
        self.parent: list[int] = list(range(n))
        self.rank: list[int] = [0] * n

    def find(self, x: int) -> int:
        # 递归路径压缩
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x: int, y: int) -> bool:
        """合并两集合；若已同根返回 False。"""
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
        self.parent[ry] = rx
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1
        return True


def kruskal_mst(n: int, edges: list[tuple[int, int, float]]) -> float:
    """Kruskal 算法求 MST 总权重。

    参数:
        n: 顶点数量（顶点编号 0..n-1）
        edges: [(u, v, w), ...]
    时间复杂度：O(E log E)（主要来自排序）
    """
    uf = UnionFind(n)
    total: float = 0.0
    count: int = 0
    # sorted 返回新列表，按 w 升序
    for u, v, w in sorted(edges, key=lambda e: e[2]):
        if uf.union(u, v):
            total += w
            count += 1
            if count == n - 1:
                break
    if count < n - 1:
        raise ValueError("图不连通，无法构造生成树")
    return total
```

#### 🐍 Python 语法小回顾

- **`heapq`**：最小堆，`heappush` / `heappop` 均为 \(O(\log n)\)。若要最大堆，存 `-value`。
- **`sorted(iterable, key=func)`**：稳定排序，\(O(n \log n)\)。
- **`@staticmethod`**：声明不需要 `self` 的方法，如工具函数。
- **元组解包**：`for u, v, w in edges` 直接解包三元组，清晰高效。

- 🔥 **环检测**：有向图三色法 / 无向图父节点法 / 并查集。
- 🔥 **拓扑排序**：BFS（Kahn）最常考，记住"入度 0 入队"流程。

def bfs_shortest(graph, start, target):
    queue = deque([(start, 0)])
    visited = {start}
    while queue:
        node, step = queue.popleft()
        if node == target:
            return step

                visited.add(nxt)

## 7. 性能对比与最佳实践

### 7.1 邻接矩阵 vs 邻接表

| 场景 | 推荐 |
|------|------|
| 图非常稠密（\(E \approx V^2\)） | 邻接矩阵（缓存友好） |
| 图稀疏（大多数面试与实际场景） | 邻接表 |
| 需要频繁查询"(u,v) 是否有边" | 邻接矩阵 \(O(1)\) |
| 需要频繁遍历"u 的所有邻居" | 邻接表 \(O(\deg(u))\) |
| 需要矩阵运算（如幂，闭包） | 邻接矩阵 + numpy |

### 7.2 Python 实现优化技巧

1. **队列用 `collections.deque`**，永远不要用 `list.pop(0)`。
2. **集合查找比列表快**：判重用 `set`，不用 `list`。
3. **避免重复入堆**：Dijkstra 用 `if d > dist[u]: continue` 惰性删除。
4. **大图递归改迭代**：防栈溢出。可配合 `sys.setrecursionlimit()`。
5. **`defaultdict` 简化建图**：

   ```python
   from collections import defaultdict
   graph = defaultdict(list)
   for u, v in edges:
       graph[u].append(v)
       graph[v].append(u)
   ```

6. **启发式搜索**：A\* 用估价函数剪枝；用 `heapq` 维护 `(f, g, node)`
7. **考虑 networkx**：工程里大规模分析直接用成熟库，别重造轮子
8. **numpy / scipy.sparse**：处理超大稠密或稀疏图的矩阵运算

### 本节小结 + 面试高频点

- 🔥 **能用 `set` 别用 `list` 判重**；能用 `deque` 别用 `list` 当队列
- 🔥 面试若出大数据，提到**邻接表 + `defaultdict`** 基本稳
