# 第五章 链表（Linked List）

## 写在前面：为什么需要链表？

上一节学了数组，它的**核心痛点**是：

- 插入/删除元素需要移动大量元素 → O(n)
- 需要一整块连续的内存空间
- 扩容代价高，需要复制所有元素

**链表**就是为了解决这些问题而诞生的数据结构

### 一句话定义

> **链表（Linked List）是一种线性数据结构，由一系列"节点"（Node）组成，每个节点包含数据和指向下一个节点的引用（指针），节点在内存中可以不连续存放**

### 从内存角度理解链表

数组像一排连号的储物柜，链表则更像**寻宝游戏**——每个宝箱里有一件宝物（数据）和一张纸条（指针），纸条上写着下一个宝箱的位置

```css
数组（连续存储）:
地址:  1000  1004  1008  1012  1016
      ┌─────┬─────┬─────┬─────┬─────┐
      │ 10  │ 20  │ 30  │ 40  │ 50  │
      └─────┴─────┴─────┴─────┴─────┘

链表（离散存储）:
      ┌────────────┐    ┌────────────┐    ┌────────────┐
      │ 数据: 10   │    │ 数据: 20   │    │ 数据: 30   │
      │ 下一个: ───┼──→ │ 下一个: ───┼──→ │ 下一个: None│
      └────────────┘    └────────────┘    └────────────┘
       地址: 1000        地址: 3048        地址: 2016
       ↑                                  
      head(头指针)                        这是"尾节点"，指向 None
```

> 💡 **关键区别：** 数组的元素必须连续存放，而链表的节点可以散落在内存的任意位置——它们靠"指针"串联在一起

## 核心概念：节点（Node）

节点是链表的基本组成单元，每个节点包含两部分：

```css
┌─────────────────────────┐
│  数据域（data/val）       │ ← 存储实际数据
│  指针域（next）           │ ← 存储下一个节点的地址/引用
└─────────────────────────┘
```

### 在 Python 中定义节点

```python
class ListNode:
    """
    链表节点类。
    
    属性:
        val  : 节点存储的数据
        next : 指向下一个节点的引用（指针），默认为 None 表示没有后继
    """
    def __init__(self, val=0, next=None):
        self.val = val      # 数据域：存储数据
        self.next = next    # 指针域：指向下一个节点
```

```python
# 感觉这样写比较好
class ListNode:
    """链表节点类"""
    def __init__(self, val: int):
        self.val: int = val               # 节点值
        self.next: ListNode | None = None # 指向下一节点的引用，写明next要么是节点要么是None
```

> ⚠️ **Python 语法补充：`class` 和 `__init__`**
>
> ```python
> class 类名:
>  def __init__(self, 参数1, 参数2, ...):
>      self.属性1 = 参数1
>      self.属性2 = 参数2
> ```
>
> - `class` 关键字用来定义一个"类"，类是对象的模板/蓝图
> - `__init__` 是"构造方法"，在创建对象时自动调用
> - `self` 代表对象自身（类似其他语言的 `this`），必须作为第一个参数
> - `self.val = val` 意思是"把传入的 val 值保存为这个对象的属性"
>
> ```python
> # 创建节点对象
> node = ListNode(10)       # 调用 __init__，val=10, next=None
> print(node.val)           # 输出: 10
> print(node.next)          # 输出: None
> ```

### 手动创建一个简单链表

```python
# 创建三个独立的节点
node1 = ListNode(10)
node2 = ListNode(20)
node3 = ListNode(30)

# 此时三个节点互不关联，就像三个孤立的宝箱
# node1.next = None
# node2.next = None
# node3.next = None

# 把它们串联起来：node1 → node2 → node3
node1.next = node2    # node1 的 next 指向 node2
node2.next = node3    # node2 的 next 指向 node3
# node3.next 保持 None，表示链表结束

# 现在 node1 就是"头节点"（head），通过它可以访问整个链表
head = node1  # 这里只是相当于多了个变量，并没有改变链表实际结构
```

```
head
 ↓
┌──────────┐     ┌──────────┐     ┌──────────┐
│ val: 10  │     │ val: 20  │     │ val: 30  │
│ next: ───┼────→│ next: ───┼────→│ next:None│
└──────────┘     └──────────┘     └──────────┘
```

## 4.2.1 链表常用操作

### 1. 初始化链表

#### 方式一：逐个创建节点并串联

```python
class ListNode:
    """链表节点类"""
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


# 逐个创建并手动链接
n1 = ListNode(10)
n2 = ListNode(20)
n3 = ListNode(30)
n4 = ListNode(40)

n1.next = n2
n2.next = n3
n3.next = n4

head = n1   # 头指针指向第一个节点
```

#### 方式二：利用构造函数链式创建（更简洁）

```python
# 利用 __init__ 的 next 参数，从后往前构建
# 先创建最后一个节点，再往前串
head = ListNode(10, ListNode(20, ListNode(30, ListNode(40))))  # 套娃写法

# 等价于：
# n4 = ListNode(40)          → 40 → None
# n3 = ListNode(30, n4)      → 30 → 40 → None
# n2 = ListNode(20, n3)      → 20 → 30 → 40 → None
# n1 = ListNode(10, n2)      → 10 → 20 → 30 → 40 → None
# head = n1
```

#### 方式三：从列表（数组）构建链表（最实用）

在刷题和实际使用中，我们经常需要从一个普通列表快速构建链表：

```python
def build_linked_list(values: list) -> ListNode:
    """
    从 Python 列表构建链表。
    
    参数:
        values : 包含数据的列表，如 [10, 20, 30]
    返回:
        链表的头节点；如果列表为空，返回 None
    """
    if not values:          # 空列表
        return None
    
    # 创建头节点
    head = ListNode(values[0]) #values是列表，列表第一个元素取做头结点
    
    # current 指针用于追踪当前正在操作的节点
    current = head
    
    # 从第二个元素开始，逐个创建节点并链接
    for val in values[1:]:               # values[1:] 是切片，取第 2 个到最后
        current.next = ListNode(val)     # 创建新节点，挂在 current 后面
        current = current.next           # current 前进到新节点
    
    return head


# 测试
head = build_linked_list([10, 20, 30, 40, 50])
```

#### 辅助函数：打印链表（后续所有示例都会用到）

```python
def print_linked_list(head: ListNode) -> None:
    """
    打印链表的所有节点，格式: 10 → 20 → 30 → None
    
    参数:
        head : 链表的头节点
    """
    current = head                      # 从头节点开始
    while current is not None:          # 只要当前节点不为空
        print(current.val, end=" → ")   # 打印当前节点的值
        current = current.next          # 移动到下一个节点
    print("None")                       # 链表结尾


# 测试
head = build_linked_list([10, 20, 30, 40, 50])
print_linked_list(head)
# 输出: 10 → 20 → 30 → 40 → 50 → None
```

> ⚠️ **Python 语法补充：`is` 和 `==` 的区别**
>
> - ==`is` 判断两个变量是否指向**同一个对象**（比较内存地址）==
> - ==`==` 判断两个变量的**值是否相等**==
> - 判断是否为 `None` 时，Python 惯例用 `is None` / `is not None`
>
> ```python
> a = [1, 2, 3]
> b = [1, 2, 3]
> print(a == b)    # True —— 值相等
> print(a is b)    # False —— 不是同一个对象
>
> c = None
> print(c is None)     # True ✅ 推荐写法
> print(c == None)     # True，但不推荐
> ```

### 2. 插入节点

链表的插入操作是它的**核心优势**——只需要修改指针，不需要移动任何元素

#### 2.1 头插法（在链表头部插入）

头插法是最简单的插入方式：**新节点始终插在链表头部**，成为新的头节点。

```css
插入前:  head → [20] → [30] → [40] → None

要插入值为 10 的新节点:

第一步：创建新节点，让它的 next 指向原来的 head
         new_node → [10] → [20] → [30] → [40] → None
                    ↑
第二步：更新 head，让 head 指向新节点
         head → [10] → [20] → [30] → [40] → None
```

```python
def insert_at_head(head: ListNode, val) -> ListNode:
    """
    头插法：在链表头部插入新节点。
    
    参数:
        head : 原链表的头节点
        val  : 新节点的值
    返回:
        新的头节点（即刚插入的节点）
    """
    new_node = ListNode(val)     # 第一步：创建新节点
    new_node.next = head         # 第二步：新节点的 next 指向原头节点
    return new_node              # 第三步：返回新节点作为新的头节点


# 测试
head = build_linked_list([20, 30, 40])
print("插入前: ", end="")
print_linked_list(head)          # 20 → 30 → 40 → None

head = insert_at_head(head, 10)  # 在头部插入 10
print("插入后: ", end="")
print_linked_list(head)          # 10 → 20 → 30 → 40 → None

# 连续头插法建表
head = None
for val in [50, 40, 30, 20, 10]:     # 注意顺序是反的！
    head = insert_at_head(head, val)
print_linked_list(head)               # 10 → 20 → 30 → 40 → 50 → None
```

> 💡 **头插法的特点：** 插入顺序和最终链表顺序**相反**！先插入的在后面，后插入的在前面。时间复杂度 **O(1)**

#### 用头插法从列表构建链表

```python
def build_by_head_insert(values: list) -> ListNode:
    """
    用头插法从列表构建链表。
    注意：最终链表的顺序与列表顺序相反！
    
    参数:
        values : 数据列表
    返回:
        链表头节点
    """
    head = None
    for val in values:
        new_node = ListNode(val)
        new_node.next = head
        head = new_node
    return head


# 测试
head = build_by_head_insert([10, 20, 30, 40, 50])
print_linked_list(head)
# 输出: 50 → 40 → 30 → 20 → 10 → None  ← 顺序反了！

# 如果想让最终顺序和列表一致，可以先反转列表
head = build_by_head_insert([10, 20, 30, 40, 50][::-1])
print_linked_list(head)
# 输出: 10 → 20 → 30 → 40 → 50 → None  ← 正序！
```

#### 2.2 尾插法（在链表尾部插入）

尾插法：**新节点始终插在链表末尾**。最终链表的顺序与插入顺序**一致**

```css
插入前:  head → [10] → [20] → [30] → None

要在尾部插入值为 40 的新节点:

第一步：遍历到最后一个节点（val=30 的节点）
第二步：让最后节点的 next 指向新节点

         head → [10] → [20] → [30] → [40] → None
                                       ↑ 新插入的
```

```python
def insert_at_tail(head: ListNode, val) -> ListNode:
    """
    尾插法：在链表尾部插入新节点。
    
    参数:
        head : 链表头节点
        val  : 新节点的值
    返回:
        头节点（不变，除非原链表为空）
    """
    new_node = ListNode(val)
    
    # 特殊情况：空链表，新节点就是头节点
    if head is None:
        return new_node
    
    # 遍历到最后一个节点
    current = head
    while current.next is not None:    # 注意：是 current.next，不是 current
        current = current.next
    
    # 此时 current 指向最后一个节点
    current.next = new_node            # 把新节点挂在末尾
    return head


# 测试
head = build_linked_list([10, 20, 30])
print("插入前: ", end="")
print_linked_list(head)              # 10 → 20 → 30 → None

head = insert_at_tail(head, 40)
print("插入后: ", end="")
print_linked_list(head)              # 10 → 20 → 30 → 40 → None
```

#### 用尾插法从列表构建链表

```python
def build_by_tail_insert(values: list) -> ListNode:
    """
    用尾插法从列表构建链表。
    最终链表的顺序与列表顺序一致。
    
    参数:
        values : 数据列表
    返回:
        链表头节点
    """
    if not values:
        return None
    
    head = ListNode(values[0])     # 创建头节点
    tail = head                    # tail 指针始终指向最后一个节点
    
    for val in values[1:]:
        tail.next = ListNode(val)  # 在 tail 后面创建新节点
        tail = tail.next           # tail 前进到新的最后节点
    
    return head


# 测试
head = build_by_tail_insert([10, 20, 30, 40, 50])
print_linked_list(head)
# 输出: 10 → 20 → 30 → 40 → 50 → None  ← 顺序一致！
```

> 💡 **尾插法的特点：** 插入顺序和最终链表顺序**一致**。但每次插入需要遍历到尾部 O(n)——除非你维护一个 `tail` 尾指针，这样就是 O(1)

#### 头插法 vs 尾插法 对比

| 特性 | 头插法 | 尾插法 |
|------|--------|--------|
| 插入位置 | 链表头部 | 链表尾部 |
| 最终顺序 | 与插入顺序**相反** | 与插入顺序**一致** |
| 时间复杂度 | **O(1)** | O(n)，用尾指针可优化为 O(1) |
| 是否需要遍历 | 不需要 | 需要（无尾指针时） |
| 常见场景 | 栈的实现、逆序建表 | 队列的实现、正序建表 |

#### 2.3 在指定位置插入

==就是比方说，要在i处插入，那就把i指向的next赋给新节点，然后再把i指向next改成新节点，先夺过来，然后再改，顺序不能反==

```css
在节点 A 后面插入新节点 new:

插入前:  ... → [A] → [B] → ...

第一步：new.next = A.next     （新节点指向 B），这里是因为B的位置存在A.next
第二步：A.next = new          （A 指向新节点）,这里再把A的下一个节点位置设为new

插入后:  ... → [A] → [new] → [B] → ...

⚠️ 顺序不能反！如果先执行第二步，A.next 就不再指向 B 了，B 就"丢失"了！
```

```python
def insert_after(node: ListNode, val) -> None:
    """
    在指定节点 node 后面插入新节点。
    
    参数:
        node : 在此节点后面插入
        val  : 新节点的值
    """
    if node is None:
        print("错误：不能在空节点后面插入")
        return
    
    new_node = ListNode(val)
    new_node.next = node.next    # 第一步：新节点指向原后继（必须先做！）
    node.next = new_node         # 第二步：前驱节点指向新节点


# 测试
head = build_linked_list([10, 20, 40, 50])
print("插入前: ", end="")
print_linked_list(head)          # 10 → 20 → 40 → 50 → None

# 在值为 20 的节点后面插入 30
# 先找到值为 20 的节点
current = head
while current and current.val != 20:
    current = current.next

insert_after(current, 30)
print("插入后: ", end="")
print_linked_list(head)          # 10 → 20 → 30 → 40 → 50 → None
```

#### 2.4 按下标位置插入

```python
def insert_at_index(head: ListNode, index: int, val) -> ListNode:
    """
    在链表的第 index 个位置插入新节点（下标从 0 开始）。
    
    例如 index=0 表示在头部插入，index=2 表示插在原第 2 个节点的位置。
    
    参数:
        head  : 链表头节点
        index : 插入位置（0-based）
        val   : 新节点的值
    返回:
        新的头节点
    """
    # 特殊情况：在头部插入
    if index == 0:
        new_node = ListNode(val)
        new_node.next = head
        return new_node
    
    # 找到第 index-1 个节点（即插入位置的前一个节点）
    current = head
    for i in range(index - 1):
        if current is None:           # 下标越界
            print(f"错误：下标 {index} 越界")
            return head
        current = current.next
    
    if current is None:
        print(f"错误：下标 {index} 越界")
        return head
    
    # 在 current 后面插入新节点
    new_node = ListNode(val)
    new_node.next = current.next
    current.next = new_node
    return head


# 测试
head = build_linked_list([10, 20, 40, 50])
print("插入前: ", end="")
print_linked_list(head)            # 10 → 20 → 40 → 50 → None

head = insert_at_index(head, 2, 30)
print("插入后: ", end="")
print_linked_list(head)            # 10 → 20 → 30 → 40 → 50 → None
```

#### 插入操作时间复杂度

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 头插法 | **O(1)** | 直接在头部操作 |
| 尾插法（无尾指针） | **O(n)** | 需要遍历到尾部 |
| 尾插法（有尾指针） | **O(1)** | 直接在尾部操作 |
| 在已知节点后插入 | **O(1)** | 只需修改两个指针 |
| 按下标插入 | **O(n)** | 需要先遍历找到位置 |

### 3. 删除节点

#### 3.1 删除头节点

```css
删除前:  head → [10] → [20] → [30] → None
删除后:  head → [20] → [30] → None

只需要让 head 指向第二个节点即可：
head = head.next
```

```python
def delete_head(head: ListNode) -> ListNode:
    """
    删除链表的头节点。
    
    参数:
        head : 链表头节点
    返回:
        新的头节点
    """
    if head is None:
        print("错误：链表为空，无法删除")
        return None
    
    new_head = head.next     # 新的头节点是原来的第二个节点
    head.next = None         # 断开旧头节点的连接（帮助垃圾回收）
    return new_head


# 测试
head = build_linked_list([10, 20, 30, 40])
print("删除前: ", end="")
print_linked_list(head)       # 10 → 20 → 30 → 40 → None

head = delete_head(head)
print("删除后: ", end="")
print_linked_list(head)       # 20 → 30 → 40 → None
```

> ⚠️ **Python 语法补充：垃圾回收（Garbage Collection）**
>
> Python 使用引用计数和垃圾回收机制来自动管理内存。当一个对象不再被任何变量引用时，Python 会自动释放它占用的内存。所以 `head.next = None` 这一步在 Python 中不是必须的，但它是一个好习惯，表明你理解了指针的断开操作。在 C/C++ 中，你必须手动释放内存（`free()`），否则会造成内存泄漏

#### 3.2 删除指定节点的后继

```css
删除 A 后面的节点 B:

删除前:  ... → [A] → [B] → [C] → ...
删除后:  ... → [A] → [C] → ...

操作：A.next = A.next.next    即 A 直接指向 C，跳过 B
```

```python
def delete_after(node: ListNode) -> None:
    """
    删除指定节点 node 后面的节点。
    
    参数:
        node : 要删除其后继的节点
    """
    if node is None or node.next is None:
        print("错误：没有可删除的后继节点")
        return
    
    # 保存要删除的节点（可选，用于打印或返回被删数据）
    removed = node.next
    
    # 跳过被删节点，直接指向被删节点的后继
    node.next = removed.next
    
    # 断开被删节点的连接（好习惯）
    removed.next = None
    
    print(f"已删除节点: {removed.val}")


# 测试
head = build_linked_list([10, 20, 30, 40, 50])
print("删除前: ", end="")
print_linked_list(head)        # 10 → 20 → 30 → 40 → 50 → None

# 删除第二个节点（val=20）后面的节点（val=30）
# 先找到 val=20 的节点
node = head
while node and node.val != 20:
    node = node.next

delete_after(node)              # 已删除节点: 30
print("删除后: ", end="")
print_linked_list(head)        # 10 → 20 → 40 → 50 → None
```

#### 3.3 按值删除

```python
def delete_by_value(head: ListNode, target) -> ListNode:
    """
    删除链表中第一个值为 target 的节点。
    
    参数:
        head   : 链表头节点
        target : 要删除的值
    返回:
        新的头节点
    """
    # 特殊情况 1：空链表
    if head is None:
        print("链表为空")
        return None
    
    # 特殊情况 2：要删除的是头节点
    if head.val == target:
        return head.next          # 直接返回第二个节点作为新头
    
    # 一般情况：遍历找到目标节点的前驱
    current = head
    while current.next is not None:
        if current.next.val == target:    # 找到了！current 是目标的前驱
            current.next = current.next.next   # 跳过目标节点
            return head
        current = current.next
    
    print(f"未找到值为 {target} 的节点")
    return head


# 测试

head = delete_by_value(head, 30)    # 删除中间节点
print_linked_list(head)              # 10 → 20 → 40 → 50 → None

head = delete_by_value(head, 10)    # 删除头节点
print_linked_list(head)              # 20 → 40 → 50 → None

head = delete_by_value(head, 50)    # 删除尾节点
print_linked_list(head)              # 20 → 40 → None
```

#### 3.4 按下标删除

```python
def delete_at_index(head: ListNode, index: int) -> ListNode:
    """
    删除链表中第 index 个节点（下标从 0 开始）。
    
    参数:
        head  : 链表头节点
        index : 要删除的位置
    返回:
        新的头节点
    """
    if head is None:
        print("链表为空")
        return None
    
    # 删除头节点（index == 0）
    if index == 0:
        return head.next
    
    # 找到第 index-1 个节点（目标节点的前驱）
    current = head
    for i in range(index - 1):
        if current.next is None:
            print(f"下标 {index} 越界")
            return head
        current = current.next
    
    # current 现在是目标节点的前驱
    if current.next is None:
        print(f"下标 {index} 越界")
        return head
    
    current.next = current.next.next   # 跳过目标节点
    return head


# 测试
head = build_linked_list([10, 20, 30, 40, 50])
head = delete_at_index(head, 2)     # 删除下标 2 的节点（值为 30）
print_linked_list(head)              # 10 → 20 → 40 → 50 → None
```

#### 删除操作时间复杂度

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 删除头节点 | **O(1)** | 直接修改 head |
| 删除已知节点的后继 | **O(1)** | 直接修改指针 |
| 按值删除 | **O(n)** | 需要遍历查找 |
| 按下标删除 | **O(n)** | 需要遍历到指定位置 |

> 💡 **关键理解：** 链表的删除操作本身是 O(1)（只需要改指针），但**找到要删除的位置**需要 O(n)。这和数组正好相反——数组找位置是 O(1)，但删除操作本身需要移动元素 O(n)

### 4. 访问节点

链表**不支持随机访问**。要访问第 i 个节点，必须从头开始逐个遍历。

```python
def access_by_index(head: ListNode, index: int):
    """
    访问链表中第 index 个节点的值（下标从 0 开始）。
    
    参数:
        head  : 链表头节点
        index : 要访问的下标
    返回:
        对应节点的值；如果越界返回 None
    """
    current = head
    count = 0
    
    while current is not None:
        if count == index:
            return current.val     # 找到了，返回值
        current = current.next     # 前进到下一个节点
        count += 1
    
    # 遍历完了还没找到，说明越界
    print(f"下标 {index} 越界")
    return None


# 测试
head = build_linked_list([10, 20, 30, 40, 50])

print(access_by_index(head, 0))    # 输出: 10
print(access_by_index(head, 2))    # 输出: 30
print(access_by_index(head, 4))    # 输出: 50
print(access_by_index(head, 5))    # 输出: 下标 5 越界 \n None
```

### 5. 查找节点

```python
def find_node(head: ListNode, target) -> int:
```

## 重要概念：有头节点 vs 无头节点链表

### 无头节点（无空头）链表

前面所有代码示例都是**无头节点链表**：`head` 直接指向第一个**有效数据节点**。

```
head → [10] → [20] → [30] → None
       ↑ 第一个节点就存储有效数据
```

**问题：** 在无头节点链表中，头部的操作需要**特殊处理**

```python
# 例如删除操作：头节点和其他节点的删除逻辑不同
def delete_value_no_dummy(head, target):
    # 😰 必须特判：如果要删的是头节点
    if head and head.val == target:
        return head.next
    
    # 一般情况
    curr = head
    while curr and curr.next:
        if curr.next.val == target:
            curr.next = curr.next.next
            return head
        curr = curr.next
    return head
```

### 有头节点（有空头 / 哨兵节点）链表

在链表最前面加一个**不存储有效数据**的"虚拟头节点"（也叫**哨兵节点 / dummy node**），它的 `val` 通常为 0 或 None，纯粹是为了简化操作

```css
dummy → [10] → [20] → [30] → None
  ↑       ↑
哨兵节点  第一个有效数据节点
(val无意义)

dummy.next 才是链表的第一个有效节点
```

**优势：** 有了哨兵节点后，所有节点（包括第一个有效节点）都有"前驱节点"，**插入和删除的逻辑完全统一**，不再需要特判头节点

```python
def delete_value_with_dummy(head, target):
    """
    使用哨兵节点（dummy）删除链表中值为 target 的节点。
    所有位置的删除逻辑完全一致，不需要特判！
    """
    # 创建哨兵节点，指向原链表的头节点
    dummy = ListNode(0)
    dummy.next = head
    
    # 从哨兵节点开始遍历
    curr = dummy
    while curr.next is not None:
        if curr.next.val == target:
            curr.next = curr.next.next     # 统一的删除逻辑
            # 如果只删第一个就 break，删除所有就继续循环
            break
        else:
            curr = curr.next
    
    return dummy.next    # 返回哨兵节点的 next（真正的头节点）
```

#### 完整对比示例

```python
# ============ 无哨兵节点版本 ============
def insert_at_index_v1(head: ListNode, index: int, val) -> ListNode:
    """无哨兵节点：在第 index 个位置插入"""
    # 😰 特判：在头部插入
    if index == 0:
        new_node = ListNode(val)
        new_node.next = head
        return new_node              # 头节点变了，要返回新的头
    
    # 找到第 index-1 个节点
    curr = head
    for i in range(index - 1):
        if curr is None:
            return head
        curr = curr.next
    
    if curr is None:
        return head
    
    new_node = ListNode(val)
    new_node.next = curr.next
    curr.next = new_node
    return head                      # 头节点没变


# ============ 有哨兵节点版本 ============
def insert_at_index_v2(head: ListNode, index: int, val) -> ListNode:
    """有哨兵节点：在第 index 个位置插入——逻辑统一，无需特判！"""
    dummy = ListNode(0)              # 创建哨兵节点
    dummy.next = head                # 哨兵指向原头节点
    
    # 找到第 index-1 个节点（从 dummy 开始数）
    # 对于 index=0 的情况，prev 就是 dummy 本身——完美统一！
    prev = dummy
    for i in range(index):
        if prev.next is None:
            return dummy.next
        prev = prev.next
    
    # 统一的插入逻辑，不管是头部还是中间还是尾部
    new_node = ListNode(val)
    new_node.next = prev.next
    prev.next = new_node
    
    return dummy.next                # 返回真正的头节点


# 测试
head = build_linked_list([20, 30, 40])

# 在头部插入 10
head = insert_at_index_v2(head, 0, 10)
print_linked_list(head)    # 10 → 20 → 30 → 40 → None

# 在中间插入 25
head = insert_at_index_v2(head, 2, 25)
print_linked_list(head)    # 10 → 20 → 25 → 30 → 40 → None
```

### 有头节点 vs 无头节点 总结

| 维度 | 无头节点 | 有头节点（哨兵） |
|------|---------|----------------|
| 结构 | head 直接指向第一个数据节点 | head/dummy 指向一个空节点，空节点再指向第一个数据节点 |
| 头部操作 | 需要**特殊处理**（因为头节点没有前驱） | **无需特判**，所有节点都有前驱 |
| 代码简洁性 | 较多分支判断 | 逻辑统一，代码更简洁 |
| 空间 | 不浪费 | 多一个节点的空间（可以忽略不计） |
| 使用场景 | 简单链表、教学演示 | **刷题时强烈推荐！** 减少 bug |
| 返回值 | 返回 head | 返回 dummy.next |

> 💡 **刷题建议：** 在 LeetCode 等平台刷链表题时，养成使用哨兵节点（dummy node）的习惯，可以大大减少边界条件的处理！

## 4.2.2 数组 vs. 链表

### 底层结构对比

```css
数组（Array）—— 连续存储
┌─────┬─────┬─────┬─────┬─────┐
│  10 │  20 │  30 │  40 │  50 │
└─────┴─────┴─────┴─────┴─────┘
  [0]   [1]   [2]   [3]   [4]
  ↑ 通过下标直接计算地址

链表（Linked List）—— 离散存储
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 10 | ───┼──→ │ 20 | ───┼──→ │ 30 | ───┼──→ │ 40 |None│
└─────────┘    └─────────┘    └─────────┘    └─────────┘
 地址:1000      地址:3048      地址:2016      地址:5072
  ↑ 只能从头开始，沿指针逐个查找
```

### 全面对比表

| 维度 | 数组 | 链表 |
|------|------|------|
| **存储方式** | 连续内存空间 | 离散内存空间，靠指针连接 |
| **访问速度** | O(1) 随机访问 | O(n) 顺序访问 |
| **头部插入** | O(n) 移动所有元素 | **O(1)** 修改指针 |
| **中间插入** | O(n) 移动后续元素 | O(n) 查找 + O(1) 插入 |
| **尾部插入** | O(1) 摊还 | O(n) 无尾指针 / O(1) 有尾指针 |
| **头部删除** | O(n) 移动所有元素 | **O(1)** 修改指针 |
| **中间删除** | O(n) 移动后续元素 | O(n) 查找 + O(1) 删除 |
| **查找** | O(n) 线性 / O(log n) 二分 | O(n) 只能线性 |
| **内存利用** | 紧凑，无额外开销 | 每个节点额外存一个指针 |
| **内存分配** | 需要连续大块空间 | 可以零散分配 |
| **缓存性能** | 优秀（连续访问，缓存命中率高） | 较差（节点分散，缓存命中率低） |
| **扩容** | 需要分配新空间并复制 O(n) | 天然支持动态扩展，无需复制 |
| **大小** | 固定或需要预分配 | 天然动态，用多少分配多少 |

### 什么时候用数组？什么时候用链表？

```css
选择数组的场景:
├── 需要频繁随机访问（按下标取值）
├── 数据量已知或变化不大
├── 对内存使用要求紧凑
└── 需要利用缓存性能（如大量遍历计算）

选择链表的场景:
├── 需要频繁在头部或中间插入/删除
├── 数据量不确定，需要动态增长
├── 不需要随机访问，主要是顺序遍历
└── 内存碎片化严重，难以分配大块连续空间
```

## 4.2.3 常见链表类型

### 1. 单向链表（Singly Linked List）

前面所有代码都是单向链表，每个节点只有一个 `next` 指针，只能从头往尾单向遍历

```css
head → [10] → [20] → [30] → None
       只能往右走 →→→
```

**局限：** 无法从某个节点回退到前一个节点，如果要访问前驱，必须从头重新遍历

---

### 2. 双向链表（Doubly Linked List）

每个节点有**两个指针**：`next` 指向后继，`prev` 指向前驱。可以双向遍历。

```css
None ← [10] ⇄ [20] ⇄ [30] ⇄ [40] → None
        ↑
       head
```

```python
class DoublyListNode:
    """
    双向链表节点类。
    
    属性:
        val  : 数据域
        prev : 指向前一个节点（前驱）
        next : 指向下一个节点（后继）
    """
    def __init__(self, val=0, prev=None, next=None):
        self.val = val
        self.prev = prev    # 前驱指针
        self.next = next    # 后继指针
```

#### 构建双向链表

```python
def build_doubly_linked_list(values: list) -> DoublyListNode:
    """
    从列表构建双向链表。
    
    参数:
        values : 数据列表
    返回:
        双向链表的头节点
    """
    if not values:
        return None
    
    head = DoublyListNode(values[0])
    current = head
    
    for val in values[1:]:
        new_node = DoublyListNode(val)
        new_node.prev = current          # 新节点的前驱指向 current
        current.next = new_node          # current 的后继指向新节点
        current = new_node               # current 前进到新节点
    
    return head


def print_doubly_list(head: DoublyListNode) -> None:
    """正向打印双向链表"""
    current = head
    while current is not None:
        print(current.val, end=" ⇄ ")
        current = current.next
    print("None")


def print_doubly_list_reverse(head: DoublyListNode) -> None:
    """反向打印双向链表（先走到尾部，再往回打印）"""
    if head is None:
        print("None")
        return
    
    # 先走到尾节点
    current = head
    while current.next is not None:
        current = current.next
    
    # 从尾部往回遍历
    while current is not None:
        print(current.val, end=" ⇄ ")
        current = current.prev
    print("None")


# 测试
head = build_doubly_linked_list([10, 20, 30, 40, 50])
print("正向: ", end="")
print_doubly_list(head)
# 输出: 正向: 10 ⇄ 20 ⇄ 30 ⇄ 40 ⇄ 50 ⇄ None

print("反向: ", end="")
print_doubly_list_reverse(head)
# 输出: 反向: 50 ⇄ 40 ⇄ 30 ⇄ 20 ⇄ 10 ⇄ None
```

#### 双向链表的插入

```css
在节点 A 和 C 之间插入 new:

插入前:   ... ⇄ [A] ⇄ [C] ⇄ ...

需要修改 4 个指针:
① new.next = A.next      (new 指向 C)
② new.prev = A           (new 指向 A)
③ A.next.prev = new      (C 的 prev 指向 new)
④ A.next = new           (A 的 next 指向 new)

插入后:   ... ⇄ [A] ⇄ [new] ⇄ [C] ⇄ ...

⚠️ 注意顺序：第 ③ 步必须在第 ④ 步之前，因为第 ④ 步会改变 A.next
```

```python
def doubly_insert_after(node: DoublyListNode, val) -> DoublyListNode:
    """
    在双向链表的 node 后面插入新节点。
    
    参数:
        node : 在此节点后面插入
        val  : 新节点的值
    返回:
        新插入的节点
    """
    new_node = DoublyListNode(val)
    
    new_node.next = node.next       # ① 新节点的 next 指向原后继
    new_node.prev = node            # ② 新节点的 prev 指向 node
    
    if node.next is not None:       # ③ 如果有原后继，更新它的 prev
        node.next.prev = new_node
    
    node.next = new_node            # ④ node 的 next 指向新节点
    
    return new_node


# 测试
head = build_doubly_linked_list([10, 20, 40, 50])
print("插入前: ", end="")
print_doubly_list(head)       # 10 ⇄ 20 ⇄ 40 ⇄ 50 ⇄ None

# 在 20 后面插入 30
node = head.next              # 找到 val=20 的节点
doubly_insert_after(node, 30)
print("插入后: ", end="")
print_doubly_list(head)       # 10 ⇄ 20 ⇄ 30 ⇄ 40 ⇄ 50 ⇄ None

# 验证反向遍历
print("反向: ", end="")
print_doubly_list_reverse(head)
# 输出: 反向: 50 ⇄ 40 ⇄ 30 ⇄ 20 ⇄ 10 ⇄ None
```

#### 双向链表的删除

```css
删除节点 B:

删除前:   ... ⇄ [A] ⇄ [B] ⇄ [C] ⇄ ...

只需修改 2 个指针（比插入简单！因为我们已经知道 B 的前驱和后继）:
① A.next = B.next    (A 指向 C)
② C.prev = B.prev    (C 指向 A)

删除后:   ... ⇄ [A] ⇄ [C] ⇄ ...
```

```python
def doubly_delete_node(head: DoublyListNode, node: DoublyListNode) -> DoublyListNode:
    """
    删除双向链表中的指定节点。
    双向链表的优势：已知节点可以 O(1) 删除，不需要知道前驱！
    
    参数:
        head : 链表头节点
        node : 要删除的节点
    返回:
        新的头节点
    """
    # 如果要删除的是头节点
    if node.prev is None:
        head = node.next
        if head is not None:
            head.prev = None
        return head
    
    # 一般情况
    node.prev.next = node.next           # 前驱的 next 跳过 node
    
    if node.next is not None:            # 如果 node 不是尾节点
        node.next.prev = node.prev       # 后继的 prev 跳过 node
    
    # 断开 node 的连接
    node.prev = None
    node.next = None
    
    return head


# 测试
head = build_doubly_linked_list([10, 20, 30, 40, 50])

# 删除 val=30 的节点
target = head.next.next         # 找到 val=30 的节点
head = doubly_delete_node(head, target)
print_doubly_list(head)         # 10 ⇄ 20 ⇄ 40 ⇄ 50 ⇄ None
```

> 💡 **双向链表删除的优势：** 在单向链表中，删除一个节点必须知道它的**前驱**——这需要 O(n) 遍历。而双向链表每个节点都有 `prev` 指针，所以**已知节点时可以 O(1) 直接删除**

#### 单向 vs 双向链表 对比

| 维度 | 单向链表 | 双向链表 |
|------|---------|---------|
| 每个节点的指针数 | 1 个（next） | 2 个（prev + next） |
| 遍历方向 | 只能从头到尾 → | 双向都可以 ⇄ |
| 内存开销 | 较小 | 较大（多一个 prev 指针） |
| 删除已知节点 | O(n)（需要找前驱） | **O(1)**（直接用 prev） |
| 插入已知位置前方 | O(n)（需要找前驱） | **O(1)**（直接用 prev） |
| 代码复杂度 | 简单 | 稍复杂（需要维护两个指针） |
| 典型应用 | 简单列表、哈希表链地址法 | LRU 缓存、浏览器前进后退、文本编辑器 |

### 3. 循环链表（Circular Linked List）

尾节点的 `next` 不再指向 `None`，而是指向头节点，形成一个"环"。

```css
单向循环链表:

head → [10] → [20] → [30] → [40] ─┐
        ↑                           │
        └───────────────────────────┘

双向循环链表:

   ┌──────────────────────────────────────────┐
   ↓                                          │
  [10] ⇄ [20] ⇄ [30] ⇄ [40] ─────────────────┘
   ↑                           ↓
   └───────────────────────────┘
```

#### 构建单向循环链表

```python
def build_circular_linked_list(values: list) -> ListNode:
    """
    从列表构建单向循环链表。
    
    参数:
        values : 数据列表
    返回:
        循环链表的头节点
    """
    if not values:
        return None
    
    head = ListNode(values[0])
    current = head
    
    for val in values[1:]:
        current.next = ListNode(val)
        current = current.next
    
    # 关键一步：尾节点的 next 指向头节点，形成环！
    current.next = head
    
    return head


def print_circular_list(head: ListNode) -> None:
    """
    打印循环链表（需要检测回到头节点时停止，否则会无限循环！）
    """
    if head is None:
        print("空链表")
        return
    
    current = head
    while True:
        print(current.val, end=" → ")
        current = current.next
        if current is head:       # 又回到了头节点，结束！
            break
    print(f"(回到 {head.val})")


# 测试
head = build_circular_linked_list([10, 20, 30, 40])
print_circular_list(head)
# 输出: 10 → 20 → 30 → 40 → (回到 10)
```

#### 循环链表的遍历要点

```python
# ❌ 错误写法：这会导致无限循环！
# current = head
# while current is not None:      # 永远不会是 None！
#     print(current.val)
#     current = current.next

# ✅ 正确写法：使用 do-while 的等效写法

        break
# 输出: 10 20 30 40

# 输出: 10 20 30 40
```

#### 循环链表的应用：约瑟夫环问题

约瑟夫问题是循环链表的经典应用：n 个人围成一圈，从第一个人开始报数，每报到第 m 个人就出列，然后下一个人重新从 1 开始报数，直到所有人出列

```python
def josephus(n: int, m: int) -> list:
    """
    约瑟夫环问题：n 个人围成一圈，每报到第 m 个人出列。
    
    参数:
        n : 总人数
        m : 每隔 m 个人出列
    返回:
        出列顺序的列表
    """
    # 构建 1~n 的循环链表
    head = ListNode(1)
    current = head
    for i in range(2, n + 1):
        current.next = ListNode(i)
        current = current.next
    current.next = head              # 形成环
    
    result = []                      # 记录出列顺序
    prev = current                   # prev 始终指向 current 的前一个节点
    current = head
    
    while current.next is not current:   # 只剩一个人时停止
        # 报数：数 m-1 次（因为 current 本身算第 1 个）
        for _ in range(m - 1):
            prev = current
            current = current.next
        
        # current 出列
        result.append(current.val)
        prev.next = current.next     # 从环中删除 current
        current = prev.next          # 下一个人从被删者的下一位开始
    
    result.append(current.val)       # 最后一个人
    return result


# 测试：7 个人围成一圈，每报到第 3 个人出列
print(josephus(7, 3))
# 输出: [3, 6, 2, 7, 5, 1, 4]
```

| 类型 | 特点 | 尾节点的 next | 遍历方向 | 典型应用 |
|------|------|--------------|---------|---------|
| 单向链表 | 每个节点有一个 next 指针 | None | 单向 → | 最基础的链表 |
| 双向链表 | 每个节点有 prev + next | None | 双向 ⇄ | LRU 缓存、文本编辑器 |

### 1. 实现栈（Stack）—— 用头插法 + 删头

```python
class LinkedStack:
    """
    用链表实现的栈（Stack）。
    利用头插法实现入栈，删除头节点实现出栈。
    所有操作均为 O(1)。
    """
    def __init__(self):
        self._head = None      # 头节点，栈顶
        self._size = 0         # 栈中元素个数
    
    def push(self, val) -> None:
        """入栈：在头部插入新节点（头插法）"""
        new_node = ListNode(val)
        new_node.next = self._head
        self._head = new_node
        self._size += 1
    
    def pop(self):
        """出栈：删除头节点并返回其值"""
        if self.is_empty():
            raise IndexError("栈为空，无法出栈")
        val = self._head.val
        self._head = self._head.next
        self._size -= 1
        return val
    
    def peek(self):
        """查看栈顶元素（不删除）"""
        if self.is_empty():
            raise IndexError("栈为空")
        return self._head.val
    
    def is_empty(self) -> bool:
        """判断栈是否为空"""
        return self._size == 0
    
    def size(self) -> int:
        """返回栈中元素个数"""
        return self._size
    
    def __str__(self) -> str:
        """打印栈的内容"""
        elements = []
        current = self._head
        while current:
            elements.append(str(current.val))
            current = current.next
        return "栈顶 → " + " → ".join(elements) + " → 栈底"


# 测试
stack = LinkedStack()
stack.push(10)
stack.push(20)
stack.push(30)
print(stack)            # 栈顶 → 30 → 20 → 10 → 栈底

print(stack.pop())      # 30
print(stack.peek())     # 20
print(stack.size())     # 2
print(stack)            # 栈顶 → 20 → 10 → 栈底
```

### 2. 实现队列（Queue）—— 用尾插法 + 删头

```python
class LinkedQueue:
    """
    用链表实现的队列（Queue）。
    尾插法实现入队，删除头节点实现出队。
    维护 tail 指针，使入队也是 O(1)。
    """
    def __init__(self):
        self._head = None      # 队头（出队端）
        self._tail = None      # 队尾（入队端）
        self._size = 0
    
    def enqueue(self, val) -> None:
        """入队：在尾部插入"""
        new_node = ListNode(val)
        if self.is_empty():
            self._head = new_node
            self._tail = new_node
        else:
            self._tail.next = new_node
            self._tail = new_node
        self._size += 1
    
    def dequeue(self):
        """出队：删除头节点并返回其值"""
        if self.is_empty():
            raise IndexError("队列为空，无法出队")
        val = self._head.val
        self._head = self._head.next
        if self._head is None:       # 队列变空了，tail 也要置空
            self._tail = None
        self._size -= 1
        return val
    
    def front(self):
        """查看队头元素"""
        if self.is_empty():
            raise IndexError("队列为空")
        return self._head.val
    
    def is_empty(self) -> bool:
        return self._size == 0
    
    def size(self) -> int:
        return self._size
    
    def __str__(self) -> str:
        elements = []
        current = self._head
        while current:
            elements.append(str(current.val))
            current = current.next
        return "队头 → " + " → ".join(elements) + " → 队尾"


# 测试
queue = LinkedQueue()
queue.enqueue("顾客A")
queue.enqueue("顾客B")
queue.enqueue("顾客C")
print(queue)                 # 队头 → 顾客A → 顾客B → 顾客C → 队尾

print(queue.dequeue())       # 顾客A（最先来的最先出）
print(queue.front())         # 顾客B
print(queue)                 # 队头 → 顾客B → 顾客C → 队尾
```

### 3. 浏览器前进后退（双向链表应用）

```python
class BrowserHistory:
    """
    用双向链表模拟浏览器的前进/后退功能。
    
    原理: 每个节点存一个网页 URL。
    current 指针指向当前正在浏览的页面。
    后退 = current 往 prev 方向移动。
    前进 = current 往 next 方向移动。
    访问新页面 = 在 current 后面插入新节点，并清除后续历史。
    """
    def __init__(self, homepage: str):
        self._current = DoublyListNode(homepage)
    
    def visit(self, url: str) -> None:
        """访问新页面"""
        new_page = DoublyListNode(url)
        new_page.prev = self._current
        self._current.next = new_page    # 清除了之前的前进历史
        self._current = new_page
        print(f"访问: {url}")
    
    def back(self) -> str:
        """后退"""
        if self._current.prev is None:
            print("已经是最早的页面了")
            return self._current.val
        self._current = self._current.prev
        print(f"后退到: {self._current.val}")
        return self._current.val
    
    def forward(self) -> str:
        """前进"""
        if self._current.next is None:
            print("已经是最新的页面了")
            return self._current.val
        self._current = self._current.next
        print(f"前进到: {self._current.val}")
        return self._current.val
    
    def current_page(self) -> str:
        return self._current.val


# 测试
browser = BrowserHistory("google.com")
browser.visit("youtube.com")     # 访问: youtube.com
browser.visit("github.com")     # 访问: github.com
browser.back()                   # 后退到: youtube.com
browser.back()                   # 后退到: google.com
browser.forward()                # 前进到: youtube.com
browser.visit("python.org")     # 访问: python.org（github.com 的历史被清除）
browser.forward()                # 已经是最新的页面了
```

### 4. 音乐播放器循环播放（循环链表应用）

```python
class MusicPlayer:
    """
    用循环链表模拟音乐播放器的循环播放功能。
    """
    def __init__(self):
        self._current = None       # 当前播放的歌曲节点
        self._size = 0
    
    def add_song(self, name: str) -> None:
        """添加歌曲到播放列表"""
        new_song = ListNode(name)
        if self._current is None:
            # 第一首歌，自己指向自己
            new_song.next = new_song
            self._current = new_song
        else:
            # 插在 current 后面
            new_song.next = self._current.next
            self._current.next = new_song
        self._size += 1
    
    def next_song(self) -> str:
        """播放下一首（循环）"""
        if self._current is None:
            return "播放列表为空"
        self._current = self._current.next
        return f"正在播放: {self._current.val}"
    
    def current_song(self) -> str:
        if self._current is None:
            return "播放列表为空"
        return f"当前: {self._current.val}"
    
    def play_n_songs(self, n: int) -> None:
        """连续播放 n 首歌（演示循环效果）"""
        if self._current is None:
            print("播放列表为空")
            return
        for _ in range(n):
            print(self.next_song())
```

### 5. LRU 缓存（双向链表 + 哈希表，面试高频题）

LRU（Least Recently Used，最近最少使用）缓存是双向链表最经典的应用。原理：用双向链表维护访问顺序，最近访问的放在头部，最久没访问的在尾部；缓存满时，淘汰尾部的元素

```python
class LRUNode:
    """LRU 缓存节点"""
    def __init__(self, key=0, val=0):
        self.key = key
        self.val = val
        self.prev = None
        self.next = None


class LRUCache:
    """
    LRU 缓存：双向链表 + 字典（哈希表）实现。
    
    - 双向链表：维护访问顺序（头部=最近使用，尾部=最久未使用）
    - 字典：实现 O(1) 查找节点
    
    所有操作均为 O(1)。
    """
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}                      # key → LRUNode 的映射
        
        # 创建哨兵节点（简化边界处理）
        self.head = LRUNode()               # 虚拟头节点
        self.tail = LRUNode()               # 虚拟尾节点
        self.head.next = self.tail
        self.tail.prev = self.head
    
    def _remove(self, node: LRUNode) -> None:
        """从双向链表中删除指定节点"""
        node.prev.next = node.next
        node.next.prev = node.prev
    
    def _add_to_front(self, node: LRUNode) -> None:
        """将节点添加到双向链表头部（表示最近使用）"""
        node.next = self.head.next
        node.prev = self.head
        self.head.next.prev = node
        self.head.next = node
    
    def get(self, key: int) -> int:
        """获取值：如果存在，将其移到头部并返回值；否则返回 -1"""
        if key in self.cache:
            node = self.cache[key]
            self._remove(node)             # 从当前位置删除
            self._add_to_front(node)       # 移到头部（标记为最近使用）
            return node.val
        return -1
    
    def put(self, key: int, val: int) -> None:
        """存入键值对：如果已存在则更新；如果容量满则淘汰最久未使用的"""
        if key in self.cache:
            node = self.cache[key]
            node.val = val                 # 更新值
            self._remove(node)
            self._add_to_front(node)       # 移到头部
        else:
            if len(self.cache) >= self.capacity:
                # 淘汰尾部节点（最久未使用）
                lru_node = self.tail.prev
                self._remove(lru_node)
                del self.cache[lru_node.key]
            
            new_node = LRUNode(key, val)
            self.cache[key] = new_node
            self._add_to_front(new_node)   # 放到头部


# 测试
cache = LRUCache(3)          # 容量为 3
cache.put(1, "A")
cache.put(2, "B")
cache.put(3, "C")            # 缓存: C ⇄ B ⇄ A

print(cache.get(1))          # 输出: A（1 被访问，移到头部）
                             # 缓存: A ⇄ C ⇄ B

cache.put(4, "D")            # 容量满，淘汰最久未使用的 B
                             # 缓存: D ⇄ A ⇄ C

print(cache.get(2))          # 输出: -1（B 已被淘汰）
print(cache.get(3))          # 输出: C
```

## 综合对比：链表操作的时间复杂度速查表

| 操作 | 单向链表 | 双向链表 | 说明 |
|------|---------|---------|------|
| 访问第 i 个节点 | O(n) | O(n) | 必须从头遍历 |
| 头部插入 | **O(1)** | **O(1)** | 链表的核心优势 |
| 尾部插入（有尾指针） | **O(1)** | **O(1)** | 需要维护尾指针 |
| 尾部插入（无尾指针） | O(n) | O(n) | 需要遍历到尾部 |
| 已知位置插入 | **O(1)** | **O(1)** | 只需改指针 |
| 头部删除 | **O(1)** | **O(1)** | 直接改 head |
| 已知节点删除 | O(n) | **O(1)** | 单向需找前驱，双向直接用 prev |
| 查找 | O(n) | O(n) | 只能线性查找 |
| 获取长度 | O(n) | O(n) | 需遍历（除非维护 size 变量） |
