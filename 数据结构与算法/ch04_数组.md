## 写在前面：什么是数组？

在正式讲操作之前，我们必须先搞清楚数组**到底是什么**

### 一句话定义

> **数组（Array）是一段连续的内存空间，用来存储相同类型的数据元素，通过下标（索引）来直接访问每一个元素**

### 从内存角度理解数组

计算机的内存可以想象成一排**编了号的储物柜**，每个柜子都有一个唯一的地址

```css
内存地址:  1000   1004   1008   1012   1016   1020   1024   ...
         ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┐
         │  10  │  20  │  30  │  40  │  50  │      │      │
         └──────┴──────┴──────┴──────┴──────┴──────┴──────┘
下标:       [0]    [1]    [2]    [3]    [4]
```

**关键特征：**

- **连续存储**：数组中的元素在内存中是紧挨着放的，中间没有空隙
- **固定大小**：在传统语言（C/C++/Java）中，数组一旦创建，长度就不能改变
- **相同类型**：每个元素的类型是一样的，占用相同大小的内存空间（比如 int 占 4 字节）

### 为什么下标从 0 开始？

这不是随意规定的，而是和**寻址公式**有关：

```css
元素地址 = 数组首地址 + 下标 × 每个元素的大小
```

比如数组首地址是 `1000`，每个 int 占 `4` 字节：

- `arr[0]` 的地址 = 1000 + 0 × 4 = **1000** ✅
- `arr[1]` 的地址 = 1000 + 1 × 4 = **1004** ✅
- `arr[3]` 的地址 = 1000 + 3 × 4 = **1012** ✅

如果下标从 1 开始，每次寻址都要多做一次减法 `(下标-1) × 大小`，浪费计算资源

### Python 中的"数组"——`list`

**重要提醒：** Python 中没有传统意义上的数组，取而代之的是更强大的 `list`（列表）

| 特性 | 传统数组（C/Java） | Python list |
|------|-------------------|-------------|
| 长度 | 固定，创建后不可变 | **动态**，可随时增删 |
| 元素类型 | 必须相同 | **可以不同**（但不推荐混用） |
| 内存分配 | 一次性连续分配 | 底层使用**动态数组**策略 |
| 下标起始 | 0 | 0 |

Python 的 `list` 底层实现本质上就是一个**动态数组**——它在内部维护一个 C 语言数组，当空间不够时自动扩容。所以用 `list` 来学习数组的操作是完全合适的

> 💡 本节中，统一使用 Python 的 `list` 来演示数组操作，但讲解时会同时说明底层原理

## 4.1.1 数组常用操作

### 1. 初始化数组

"初始化"就是创建数组并赋予初始值的过程

#### 方式一：直接赋值创建（最常用）

```python
# 创建一个包含 5 个整数的数组
arr = [10, 20, 30, 40, 50]
print(arr)          # 输出: [10, 20, 30, 40, 50]
print(type(arr))    # 输出: <class 'list'>
```

#### 方式二：创建指定长度的空数组

在很多算法题中，我们需要先创建一个固定大小、全部填充默认值的数组

```python
# 创建长度为 5 的数组，所有元素初始化为 0
arr = [0] * 5
print(arr)  # 输出: [0, 0, 0, 0, 0]

# 创建长度为 3 的数组，所有元素初始化为 None（表示"空"）
arr = [None] * 3
print(arr)  # 输出: [None, None, None]
```

> ⚠️ **Python 语法补充：`[值] * n`**
>
> 这是 Python 列表的"重复"操作符。`[0] * 5` 等价于 `[0, 0, 0, 0, 0]`
> 但对于可变对象（如列表嵌套），千万不要这样用！
>
> ```python
> # ❌ 错误示范：创建二维数组
> wrong = [[0] * 3] * 3
> wrong[0][0] = 1
> print(wrong)  # 输出: [[1, 0, 0], [1, 0, 0], [1, 0, 0]]  三行都被改了！
>
> # ✅ 正确做法：用列表推导式
> correct = [[0] * 3 for _ in range(3)]
> correct[0][0] = 1
> print(correct)  # 输出: [[1, 0, 0], [0, 0, 0], [0, 0, 0]]  只改了第一行
> ```
>
> 原因：`[[0]*3] * 3` 创建的三行其实是**同一个列表对象的三个引用**，改一个全都会变

#### 方式三：列表推导式创建

```python
# 创建 [0, 1, 4, 9, 16]，即前 5 个整数的平方
arr = [i ** 2 for i in range(5)]
print(arr)  # 输出: [0, 1, 4, 9, 16]

# range(5) 生成 0, 1, 2, 3, 4
# i ** 2 是 Python 中的幂运算，等价于 i 的平方
```

> ⚠️ **Python 语法补充：`range()` 函数**
>
> `range()` 是 Python 中生成整数序列的内置函数，左闭右开，非常常用：
>
> ```python
> range(5)        # 生成 0, 1, 2, 3, 4（从 0 开始，不包含 5）
> range(2, 7)     # 生成 2, 3, 4, 5, 6（从 2 开始，不包含 7）
> range(0, 10, 2) # 生成 0, 2, 4, 6, 8（从 0 开始，步长为 2，不包含 10）
> # 还可以第三个参数写负数，比如-1，表示逆着生成，就是倒序递减
> ```

#### 方式四：使用 `array` 模块创建真正的数组（了解即可）

```python
import array

# 'i' 表示元素类型为 int（有符号整数）
arr = array.array('i', [10, 20, 30, 40, 50])
print(arr)  # 输出: array('i', [10, 20, 30, 40, 50])

# 这是真正的"类型固定"数组，不能混入其他类型
# arr.append("hello")  # ❌ 会报错 TypeError
```

> 实际开发和刷题中，99% 的情况用 `list` 就够了，`array` 模块了解即可

### 2. 访问元素

数组最大的优势就是**随机访问**——通过下标，可以在$O(1)$时间内直接拿到任意位置的元素

#### 基本访问

```python
arr = [10, 20, 30, 40, 50]

# 通过下标访问（下标从 0 开始）
print(arr[0])   # 输出: 10（第 1 个元素）
print(arr[2])   # 输出: 30（第 3 个元素）
print(arr[4])   # 输出: 50（第 5 个元素，也是最后一个）
```

#### 负数下标（Python 特有）

```python
arr = [10, 20, 30, 40, 50]

# Python 支持负数下标，从末尾倒着数
print(arr[-1])  # 输出: 50（最后一个元素）
print(arr[-2])  # 输出: 40（倒数第二个元素）
print(arr[-5])  # 输出: 10（倒数第五个，即第一个元素）
```

对应关系图示：

```css
正向下标:    0     1     2     3     4
           ┌─────┬─────┬─────┬─────┬─────┐
           │ 10  │ 20  │ 30  │ 40  │ 50  │
           └─────┴─────┴─────┴─────┴─────┘
负向下标:   -5    -4    -3    -2    -1
```

#### 切片访问（Python 特有，非常重要）

```python
arr = [10, 20, 30, 40, 50]

# 语法：arr[起始:结束)  ——包含起始，不包含结束，左闭右开
print(arr[1:3])    # 输出: [20, 30]     （下标 1 到 2）
print(arr[:3])     # 输出: [10, 20, 30] （从头到下标 2）
print(arr[2:])     # 输出: [30, 40, 50] （从下标 2 到末尾）
print(arr[:])      # 输出: [10, 20, 30, 40, 50]（完整拷贝）

# 带步长的切片：arr[起始:结束:步长]
print(arr[::2])    # 输出: [10, 30, 50] （每隔一个取一个）
print(arr[::-1])   # 输出: [50, 40, 30, 20, 10]（反转数组！）
```

#### 修改元素

```python
arr = [10, 20, 30, 40, 50]
arr[2] = 99        # 将下标 2 的元素改为 99
print(arr)         # 输出: [10, 20, 99, 40, 50]
```

#### 越界访问

```python
arr = [10, 20, 30, 40, 50]

# 访问不存在的下标会报错
# print(arr[5])    # ❌ IndexError: list index out of range
# print(arr[-6])   # ❌ IndexError: list index out of range

# 安全访问的写法：先检查长度
index = 5
if 0 <= index < len(arr):
    print(arr[index])
else:
    print(f"下标 {index} 越界！数组长度为 {len(arr)}")
    # 输出: 下标 5 越界！数组长度为 5
```

> ⚠️ **Python 语法补充：`len()` 函数**
>
> `len(arr)` 返回列表的长度（元素个数），时间复杂度为 O(1)

#### 时间复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 按下标访问 `arr[i]` | **$O(1)$** | 通过寻址公式直接计算，与数组大小无关 |
| 修改元素 `arr[i] = x` | **$O(1)$** | 同上 |

这就是数组的**核心优势**：**常数时间的随机访问**

==所以从这里也可以看出为什么数组的元素类型要一致，这样才方便通过寻址公式来计算，如果类型不一致，寻址公式就失效了==

### 3. 插入元素

插入是数组中**代价较高**的操作，因为要保持元素的连续性

#### 在末尾插入：`append()`

```python
arr = [10, 20, 30]

# append() 在数组末尾添加一个元素
arr.append(40)
print(arr)  # 输出: [10, 20, 30, 40]

arr.append(50)
print(arr)  # 输出: [10, 20, 30, 40, 50]
```

**原理分析：** 末尾插入不需要移动任何已有元素，所以是$O(1)$操作

#### 在指定位置插入：`insert()`

```python
arr = [10, 20, 30, 40, 50]

# insert(位置, 值) —— 在指定下标处插入元素
arr.insert(2, 99)
print(arr)  # 输出: [10, 20, 99, 30, 40, 50]
```

**底层发生了什么？** 在下标 2 处插入 99，下标 2 及之后的所有元素都要**往后移动一位**：

```css
插入前:  [10, 20, 30, 40, 50]
                  ↑ 要在这里插入 99

第一步：把后面的元素逐个后移（从后往前移，避免覆盖）
         [10, 20, 30, 40, 50,  _ ]   ← 先预留空间
         [10, 20,  _, 30, 40, 50]    ← 后面元素集体右移

第二步：在空出的位置放入新元素
         [10, 20, 99, 30, 40, 50]    ← 插入完成
```

#### 用纯代码模拟插入过程（帮助理解底层原理）

```python
def my_insert(arr: list, index: int, value) -> None:
    """
    手动模拟数组插入操作（不使用内置 insert 方法）。
    
    参数:
        arr   : 目标数组（会被原地修改）
        index : 插入位置的下标
        value : 要插入的值
    """
    # 第一步：在末尾添加一个占位元素，扩展数组长度
    arr.append(None)
    
    # 第二步：从最后一个有效元素开始，逐个往后搬移
    # 注意要从后往前移动，否则会覆盖还没搬的元素
    i = len(arr) - 1          # i 指向最后一个位置（刚刚 append 出来的）
    while i > index:
        arr[i] = arr[i - 1]   # 把前一个元素复制到当前位置
        i -= 1
    
    # 第三步：在目标位置放入新值
    arr[index] = value


# 测试
arr = [10, 20, 30, 40, 50]
my_insert(arr, 2, 99)
print(arr)  # 输出: [10, 20, 99, 30, 40, 50]
```

#### 时间复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 末尾插入 `append()` | **O(1)** | 不需要移动元素 |
| 头部插入 `insert(0, x)` | **O(n)** | 所有 n 个元素都要后移 |
| 中间插入 `insert(i, x)` | **O(n)** | 平均需要移动 n/2 个元素 |

### 4. 删除元素

删除和插入是"反操作"，同样需要移动元素来填补空隙

#### 按值删除：`remove()`

```python
arr = [10, 20, 30, 40, 50]

# remove(值) —— 删除第一个匹配的元素
arr.remove(30)
print(arr)  # 输出: [10, 20, 40, 50]

# 如果有多个相同值，只删除第一个
arr2 = [10, 20, 30, 20, 50]
arr2.remove(20)
print(arr2)  # 输出: [10, 30, 20, 50]（只删了第一个 20）

# 删除不存在的值会报错
# arr.remove(999)  # ❌ ValueError: list.remove(x): x not in list
```

#### 按下标删除：`pop()`

```python
arr = [10, 20, 30, 40, 50]

# pop(下标) —— 删除指定下标的元素，并返回该元素的值
removed = arr.pop(2)
print(removed)  # 输出: 30（被删除的元素）
print(arr)      # 输出: [10, 20, 40, 50]

# pop() 不传参数时，默认删除最后一个元素
last = arr.pop()
print(last)     # 输出: 50
print(arr)      # 输出: [10, 20, 40]
```

#### 使用 `del` 关键字删除

```python
arr = [10, 20, 30, 40, 50]

# del 可以删除指定下标的元素
del arr[1]
print(arr)  # 输出: [10, 30, 40, 50]

# del 还可以删除一个切片（批量删除）
arr2 = [10, 20, 30, 40, 50]
del arr2[1:3]       # 删除下标 1 和 2 的元素
print(arr2)         # 输出: [10, 40, 50]
```

> ⚠️ **`del` 和 `pop()` 的区别：**
> - `pop()` 会**返回**被删除的元素，`del` 不会
> - 需要用到被删除的值时用 `pop()`，否则用 `del`

#### 底层原理图示

```
删除 arr[2]（即元素 30）：

删除前:  [10, 20, 30, 40, 50]
                  ↑ 要删除这个

第一步：从被删位置开始，后面的元素逐个前移
         [10, 20, 40, 50, 50]   ← 40 前移到下标 2
         [10, 20, 40, 50, 50]   ← 50 前移到下标 3
                                    （最后一个 50 是残留值）

第二步：缩减数组长度，丢弃末尾的残留值
         [10, 20, 40, 50]       ← 删除完成
```

#### 用纯代码模拟删除过程

```python
def my_remove(arr: list, index: int):
    """
    手动模拟数组删除操作（不使用内置 pop/remove 方法）。
    
    参数:
        arr   : 目标数组（会被原地修改）
        index : 要删除的元素的下标
    返回:
        被删除的元素值
    """
    # 越界检查
    if index < 0 or index >= len(arr):
        raise IndexError(f"下标 {index} 越界，数组长度为 {len(arr)}")
    
    # 保存要删除的值（用于返回）
    removed_value = arr[index]
    
    # 从被删位置开始，把后面的元素逐个前移
    i = index
    while i < len(arr) - 1:
        arr[i] = arr[i + 1]   # 把后一个元素复制到当前位置
        i += 1
    
    # 删除末尾的多余元素
    arr.pop()  # 这里只是缩减长度，O(1) 操作
    
    return removed_value


# 测试
arr = [10, 20, 30, 40, 50]
removed = my_remove(arr, 2)
print(f"删除了: {removed}")   # 输出: 删除了: 30
print(f"数组: {arr}")         # 输出: 数组: [10, 20, 40, 50]
```

#### 时间复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 删除末尾 `pop()` | **O(1)** | 不需要移动元素 |
| 删除头部 `pop(0)` | **O(n)** | 所有元素都要前移 |
| 按值删除 `remove(x)` | **O(n)** | 先查找 O(n) + 移动 O(n) |
| 按下标删除 `pop(i)` | **O(n)** | 平均需要移动 n/2 个元素 |

### 5. 遍历数组

"遍历"就是**逐个访问**数组中的每一个元素。这是最基本也最常用的操作

#### 方式一：for 循环直接遍历元素

```python
arr = [10, 20, 30, 40, 50]

# 最简洁的写法，直接拿到每个元素的值
for element in arr:
    print(element, end=" ")
# 输出: 10 20 30 40 50
```

> ⚠️ **Python 语法补充：`end=" "`**
>
> `print()` 默认每次输出后换行。`end=" "` 改为输出后跟一个空格，不换行

#### 方式二：通过下标遍历

```python
arr = [10, 20, 30, 40, 50]

# 当你需要同时用到下标和值时，用 range + len
for i in range(len(arr)):
    print(f"arr[{i}] = {arr[i]}")

# 输出:
# arr[0] = 10
# arr[1] = 20
# arr[2] = 30
# arr[3] = 40
# arr[4] = 50
```

> ⚠️ **Python 语法补充：f-string 格式化**
>
> `f"..."` 是 Python 3.6+ 引入的格式化字符串。在花括号 `{}` 内可以直接写变量或表达式：
>
> ```python
> name = "小明"
> age = 20
> print(f"{name}今年{age}岁")  # 输出: 小明今年20岁
> ```

#### 方式三：`enumerate()` 同时获取下标和值（推荐）

```python
arr = [10, 20, 30, 40, 50]

# enumerate() 同时返回下标和值，是 Python 最地道的写法
for index, value in enumerate(arr):
    print(f"下标 {index} -> 值 {value}")

# 输出:
# 下标 0 -> 值 10
# 下标 1 -> 值 20
# 下标 2 -> 值 30
# 下标 3 -> 值 40
# 下标 4 -> 值 50
```

> 💡 **推荐程度排序：**
> - 只需要值 → `for x in arr`
> - 需要下标和值 → `for i, x in enumerate(arr)`（最推荐）
> - 需要下标和值 → `for i in range(len(arr))`（可以但不够 Pythonic）

#### 方式四：while 循环遍历

```python
arr = [10, 20, 30, 40, 50]

i = 0                       # 初始化下标
while i < len(arr):         # 循环条件：下标 < 数组长度
    print(arr[i], end=" ")
    i += 1                  # 下标递增（不要忘记！否则死循环）
# 输出: 10 20 30 40 50
```

#### 遍历的时间复杂度

所有遍历方式的时间复杂度都是 **O(n)**，因为每个元素恰好被访问一次

### 6. 查找元素

#### 方式一：`in` 关键字（判断是否存在）

```python
arr = [10, 20, 30, 40, 50]

# in 返回布尔值，判断元素是否在数组中
print(30 in arr)    # 输出: True
print(99 in arr)    # 输出: False

# 常用于 if 判断
if 30 in arr:
    print("找到了 30")
```

#### 方式二：`index()` 方法（查找下标）

```python
arr = [10, 20, 30, 40, 50]

# index(值) 返回该值第一次出现的下标
pos = arr.index(30)
print(pos)  # 输出: 2

# 如果不存在会报错
# arr.index(999)  # ❌ ValueError: 999 is not in list

# 安全写法：先判断再查找
target = 999
if target in arr:
    pos = arr.index(target)
    print(f"找到了，下标是 {pos}")
else:
    print(f"{target} 不在数组中")
# 输出: 999 不在数组中
```

#### 方式三：手动实现线性查找（理解原理）

```python
def linear_search(arr: list, target) -> int:
    """
    线性查找：从头到尾逐个比较，找到目标值返回其下标。
    
    参数:
        arr    : 要查找的数组
        target : 目标值
    返回:
        找到返回下标，没找到返回 -1
    """
    for i in range(len(arr)):
        if arr[i] == target:    # 找到了
            return i            # 直接返回下标，结束函数
    return -1                   # 遍历完都没找到，返回 -1


# 测试
arr = [10, 20, 30, 40, 50]
print(linear_search(arr, 30))   # 输出: 2
print(linear_search(arr, 99))   # 输出: -1
```

#### `count()` 方法（统计出现次数）

```python
arr = [10, 20, 30, 20, 50, 20]

# count(值) 返回该值出现的次数
print(arr.count(20))    # 输出: 3
print(arr.count(99))    # 输出: 0
```

#### 时间复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| `in` 判断 | **O(n)** | 最坏情况需要遍历整个数组 |
| `index()` 查找 | **O(n)** | 最坏情况需要遍历整个数组 |
| `count()` 统计 | **O(n)** | 必须遍历整个数组 |
| 线性查找 | **O(n)** | 同上 |

> 💡 在**无序数组**中查找，只能逐个比较，所以都是 $O(n)$。如果数组**有序**，可以用**二分查找**优化到 $O(log n)$，后续章节会学到

### 7. 扩容数组

#### 为什么需要扩容？

传统数组的长度在创建时就固定了。如果空间用完但还要插入新元素，就必须"扩容"——创建一个更大的新数组，把旧数据搬过去

#### 手动模拟扩容过程

```python
def expand_array(old_arr: list, expand_size: int) -> list:
    """
    模拟数组扩容：创建更大的新数组，将旧数组元素复制过去。
    
    参数:
        old_arr     : 原数组
        expand_size : 扩展的长度（新数组 = 旧长度 + 扩展长度）
    返回:
        扩容后的新数组
    """
    # 第一步：创建一个更大的新数组，用 0 填充
    new_length = len(old_arr) + expand_size
    new_arr = [0] * new_length
    
    # 第二步：把旧数组的元素逐个复制到新数组
    for i in range(len(old_arr)):
        new_arr[i] = old_arr[i]
    
    # 第三步：返回新数组（旧数组将被垃圾回收）
    return new_arr


# 测试
arr = [10, 20, 30, 40, 50]
print(f"扩容前: {arr}, 长度: {len(arr)}")
# 输出: 扩容前: [10, 20, 30, 40, 50], 长度: 5

arr = expand_array(arr, 5)
print(f"扩容后: {arr}, 长度: {len(arr)}")
# 输出: 扩容后: [10, 20, 30, 40, 50, 0, 0, 0, 0, 0], 长度: 10
```

#### 扩容的底层原理图示

```css
旧数组（长度 5）:
内存地址:  1000   1004   1008   1012   1016
         ┌──────┬──────┬──────┬──────┬──────┐
         │  10  │  20  │  30  │  40  │  50  │
         └──────┴──────┴──────┴──────┴──────┘

扩容后新数组（长度 10），分配了一段新的连续内存:
内存地址:  2000   2004   2008   2012   2016   2020   2024   2028   2032   2036
         ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
         │  10  │  20  │  30  │  40  │  50  │   0  │   0  │   0  │   0  │   0  │
         └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
          ↑ 复制自旧数组                       ↑ 新增的空间

旧数组占用的内存（1000-1016）被释放回收。
```

#### Python list 的自动扩容机制

Python 的 `list` 底层已经实现了**自动扩容**，完全不用手动管理。它的策略非常聪明：

```python
import sys

arr = []
print(f"{'元素个数':<10} {'底层容量(字节)':<15}")
print("-" * 30)

for i in range(20):
    # sys.getsizeof() 返回对象占用的字节数
    print(f"{len(arr):<10} {sys.getsizeof(arr):<15}")
    arr.append(i)
```

运行结果（大致）：

```css
元素个数    底层容量(字节)
------------------------------
0          56             ← 空列表本身的开销
1          88             ← 扩容！分配了 4 个位置
2          88
3          88
4          88             ← 4 个位置用完了
5          120            ← 再次扩容！
...
```

> 💡 **Python list 的扩容策略：**
>
> 当容量不够时，Python 不是每次只多分配 1 个位置，而是按照约 **1.125 倍**的比例扩容（具体公式因版本而异）。这样做的好处是：虽然单次扩容的复制代价是 O(n)，但**摊还**到每次 `append` 上，平均时间复杂度仍然是 **O(1)**。这就是所谓的**摊还分析（Amortized Analysis）**

#### 扩容的时间复杂度

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 单次扩容 | **O(n)** | 需要复制 n 个元素到新数组 |
| `append()` 摊还 | **O(1)** | 扩容不是每次都发生，平均下来是 O(1) |

## 4.1.2 数组的优点与局限性

### 优点

#### 1. 随机访问效率极高——O(1)

这是数组**最核心**的优势

```python
arr = list(range(1000000))  # 创建一百万个元素的数组

# 无论访问第 1 个还是第 999999 个，速度完全一样
import time

start = time.time()
_ = arr[0]               # 访问第一个
time1 = time.time() - start

start = time.time()
_ = arr[999999]           # 访问最后一个
time2 = time.time() - start

# 两次耗时几乎相同，都是纳秒级别
```

**为什么？** 因为有**寻址公式**：`地址 = 首地址 + 下标 × 元素大小`，一步到位，不需要逐个查找

#### 2. 内存空间利用率高

数组只存储数据本身，不需要额外的指针或元数据（对比链表，每个节点还要额外存一个指针）

```css
数组:  [数据][数据][数据][数据]        ← 100% 都是有效数据

链表:  [数据|指针] → [数据|指针] → ... ← 每个元素额外占用一个指针的空间
```

#### 3. 缓存友好（Cache-Friendly）

这是一个**非常重要**但容易被忽略的优势。现代 CPU 从内存读取数据时，不是一个字节一个字节读，而是一次性读取一个"缓存行"（通常 64 字节）

因为数组的元素在内存中是连续存放的，读取一个元素时，**相邻的元素很可能已经一起被加载到 CPU 缓存中了**，下次访问时直接从缓存取，速度极快

```css
CPU 缓存行加载示意（假设缓存行 64 字节）:

数组:  [A][B][C][D][E][F][G][H]...
       ├────── 一次加载 ──────┤
       访问 A 时，B~H 也被加载到缓存，后续访问命中率极高

链表:  [A]──→[C]──→[X]──→[F]...    节点可能散落在内存各处
       访问 A 后，缓存中不一定有 C，大量"缓存未命中"
```

### 局限性

#### 1. 插入和删除效率低——O(n)

前面已经详细讲过，在数组中间插入或删除元素，需要移动大量元素

```css
最坏情况：在头部插入/删除

插入: 所有 n 个元素都要后移 → O(n)
删除: 所有 n-1 个元素都要前移 → O(n)
```

#### 2. 长度固定，扩容代价高

传统数组长度不可变。即使是 Python list 有自动扩容，每次扩容都需要 O(n) 的复制操作，而且会浪费一部分预留空间

```python
# 假设你需要一个恰好存 1000 个元素的数组
# 但底层可能分配了 1024 个位置的空间
# 那 24 个位置就是被浪费的
```

#### 3. 内存要求连续

数组需要一整块连续的内存空间。如果内存碎片化严重，即使**总的可用内存足够**，也可能**找不到足够大的连续空间**来分配数组

```
内存情况（X = 已占用，O = 空闲）:

[XXXOOXXXOOXXXOO]   ← 有 6 个空闲位置
                    但最长的连续空间只有 2
                    无法创建长度为 3 的数组！
```

### 优缺点总结表

| 维度 | 优点 | 局限性 |
|:----:|:----:|:------:|
| 访问 | O(1) 随机访问 | — |
| 插入/删除 | 末尾操作 O(1) | 中间/头部 O(n)，需移动元素 |
| 内存 | 空间利用率高，缓存友好 | 需要连续空间，可能浪费预留空间 |
| 大小 | — | 固定大小，扩容代价 O(n) |
| 查找 | 有序时可二分 O(log n) | 无序时只能线性 O(n) |

## 4.1.3 数组典型应用

### 1. 存储和访问有序数据

数组是存储有序数据最自然的选择

```python
# 应用场景：学生成绩管理
scores = [92, 85, 78, 96, 88]

# 计算平均分
average = sum(scores) / len(scores)
print(f"平均分: {average}")  # 输出: 平均分: 87.8

print(f"升序: {scores_sorted}")         # 输出: 升序: [78, 85, 88, 92, 96]

scores_sorted_desc = sorted(scores, reverse=True)  # 降序

```

### 2. 用作栈（Stack）——后进先出

数组的末尾操作（`append` + `pop`）正好满足栈的特性

```python
# 用数组模拟栈（Stack）
# 栈的特点：后进先出（LIFO, Last In First Out）

stack = []                  # 空栈

# 入栈（push）—— 放在末尾
stack.append("第一本书")     # 先放第一本
stack.append("第二本书")     # 再放第二本
stack.append("第三本书")     # 最后放第三本
print(f"当前栈: {stack}")
# 输出: 当前栈: ['第一本书', '第二本书', '第三本书']

# 出栈（pop）—— 从末尾取
top = stack.pop()           # 取出最上面的（最后放的）
print(f"取出: {top}")       # 输出: 取出: 第三本书
print(f"当前栈: {stack}")
# 输出: 当前栈: ['第一本书', '第二本书']

# 查看栈顶（peek）—— 不取出，只看
print(f"栈顶: {stack[-1]}")  # 输出: 栈顶: 第二本书

# 判断栈是否为空
print(f"栈是否为空: {len(stack) == 0}")  # 输出: False
```

### 3. 用作队列（Queue）——先进先出

> ⚠️ **注意：** 虽然可以用 `list` 模拟队列，但因为头部删除是 O(n)，性能不好。实际开发中应该用 `collections.deque`。这里仅作原理演示

```python
from collections import deque

# 用 deque 实现队列（Queue）
# 队列的特点：先进先出（FIFO, First In First Out）
# 就像排队买票：先来的先买

queue = deque()              # 空队列

# 入队（enqueue）—— 从队尾进
queue.append("顾客A")
queue.append("顾客B")
queue.append("顾客C")
print(f"当前队列: {list(queue)}")
# 输出: 当前队列: ['顾客A', '顾客B', '顾客C']

# 出队（dequeue）—— 从队头出
first = queue.popleft()      # O(1) 操作！比 list.pop(0) 快得多
print(f"服务: {first}")      # 输出: 服务: 顾客A
print(f"当前队列: {list(queue)}")
# 输出: 当前队列: ['顾客B', '顾客C']
```

### 5. 哈希表的底层实现

哈希表（后面会详细学）的底层就是一个数组。通过"哈希函数"把键（key）映射到数组下标，实现 O(1) 的查找

```python
# 简化版演示（仅展示原理，不是完整实现）
class SimpleHashTable:
    """
    极简哈希表：演示数组作为哈希表底层存储的原理。
    """
    def __init__(self, size: int = 10):
        # 底层就是一个数组！
        self.table = [None] * size   # 创建固定大小的数组
        self.size = size
    
    def _hash(self, key: str) -> int:
        """哈希函数：把字符串 key 映射为数组下标"""
        # 把 key 中每个字符的 ASCII 值加起来，然后对数组长度取余
        # ord() 函数返回字符的 ASCII 码值
        return sum(ord(c) for c in key) % self.size
    
    def put(self, key: str, value):
        """存入键值对"""
        index = self._hash(key)      # 计算下标
        self.table[index] = value     # 存到数组对应位置
    
    def get(self, key: str):
        """根据 key 取值"""
        index = self._hash(key)      # 计算下标
        return self.table[index]      # 从数组对应位置取出


# 使用演示
ht = SimpleHashTable()
ht.put("name", "小明")
ht.put("age", 20)
print(ht.get("name"))   # 输出: 小明
print(ht.get("age"))    # 输出: 20
```

### 6. 排序与查找算法的基础

几乎所有经典排序算法（冒泡、选择、插入、快排、归并等）和查找算法（二分查找）都基于数组

```python
# 二分查找示例（要求数组有序）
def binary_search(arr: list, target: int) -> int:
    """
    二分查找：在有序数组中快速找到目标值。
    
    原理：每次取中间值与目标比较，将搜索范围缩小一半。
    时间复杂度: O(log n)，远快于线性查找的 O(n)。
    
    参数:
        arr    : 升序排列的数组
        target : 要查找的目标值
    返回:
        找到返回下标，没找到返回 -1
    """
    left = 0                    # 搜索范围的左边界
    right = len(arr) - 1        # 搜索范围的右边界
    
    while left <= right:
        mid = (left + right) // 2   # 取中间位置（// 是整除）
        
        if arr[mid] == target:
            return mid              # 找到了，返回下标
        elif arr[mid] < target:
            left = mid + 1          # 目标在右半边，缩小左边界
        else:
            right = mid - 1         # 目标在左半边，缩小右边界
    
    return -1                       # 没找到


# 测试
sorted_arr = [2, 5, 8, 12, 16, 23, 38, 56, 72, 91]

print(binary_search(sorted_arr, 23))   # 输出: 5（下标 5）
print(binary_search(sorted_arr, 100))  # 输出: -1（不存在）
```

## 综合对比：数组操作的时间复杂度速查表

| 操作 | 时间复杂度 | 备注 |
|------|-----------|------|
| 访问 `arr[i]` | O(1) | 数组的核心优势 |

| 末尾删除 `pop()` | O(1) | 无需移动元素 |
| 头部删除 `pop(0)` | O(n) | 需移动所有元素 |
| 中间删除 `pop(i)` | O(n) | 平均移动 n/2 个元素 |
| 线性查找 `x in arr` | O(n) | 逐个比较 |
| 二分查找 | O(log n) | 前提：数组有序 |
| 遍历 | O(n) | 每个元素访问一次 |
| 扩容 | O(n) | 需复制所有元素 |
