# 4.3 IP-Adapter：用图像做提示词

## 文字的边界

"请生成一张梵高《星月夜》风格的画。"

这个 prompt 对于 Stable Diffusion 来说，它理解的"梵高风格"是训练数据中所有和 "Van Gogh"、"Starry Night" 相关的图像-文本对在 CLIP 嵌入空间中的平均方向。这个方向确实包含了梵高风格的一些特征——漩涡状的笔触、强烈的黄蓝对比——但你无法用它精确控制笔触的弯曲程度、色彩的饱和偏向、或者构图的节奏感。用文字描述一种视觉风格，就像用文字描述一首歌的旋律——你最多只能说"这是一首悲伤的歌"或"这是一首快歌"，但要精确传达旋律本身，你需要直接播放它。

IP-Adapter（Image Prompt Adapter）于 2023 年下半年提出，它解决的问题就是：**能不能把一张图像直接作为提示词，而不需要先用文字描述它的视觉内容？**

答案是能。给 IP-Adapter 一张参考图像，它会提取这张图像的风格和内容特征，然后将这些特征注入到 Stable Diffusion 的去噪过程中。最终生成的图像会融合参考图像的视觉特质和文本提示词的语义引导——比如参考图提供梵高式的笔触和色调，文本提供"一只猫坐在窗台上"的构图。

## 图像特征从哪里来

IP-Adapter 使用 CLIP 的图像编码器来提取参考图像的特征。这很自然——CLIP 的双编码器（文本 + 图像）本来就是训练来把图文映射到同一个语义空间中的。文本嵌入可以注入 U-Net 做条件生成，图像嵌入理论上有同样的资格。

但在直接将 CLIP 图像嵌入注入 U-Net 时遇到一个问题。文本嵌入是 $L \times d$ 的序列（77 个 token，每个 768 维）——它是一个序列，每个 token 位置携带了不同的语义信息。交叉注意力机制天然适合处理这种序列化的条件：不同的空间位置可以关注不同的 token（"cat" 关注猫的区域，"sofa" 关注沙发的区域）。

图像嵌入则不同。CLIP 图像编码器输出的是一个**整体表示**——它用一个向量（或少数几个向量）总结了整张图像的内容。"这张图像的整体风格和内容"被压缩进这个单一的全局表示中。如果你直接把它当作一种 token 序列给交叉注意力用，效果很差——因为交叉注意力期望从不同语义单元（"猫"区别于"沙发"）中做细粒度的信息检索，而全局图像嵌入没有这种单元级的区分。

IP-Adapter 的第二个洞察是：**不要把图像特征塞进文本交叉注意力的管道里**。文本交叉注意力的 K 和 V 来自文本嵌入序列，这 77 个 token 各自有着清晰的语义分工。图像嵌入作为一个整体的"风格指纹"介入生成的最佳方式，是在文本交叉注意力之外独立地发挥作用。

## 解耦的交叉注意力

IP-Adapter 的具体做法是在 U-Net 的每个交叉注意力层旁边，**新增一个独立的交叉注意力模块**，专门用于图像特征。

标准文本交叉注意力保持不变：$$h_{\text{spatial}} \leftarrow h_{\text{spatial}} + \text{CrossAttn}_{\text{text}}(h_{\text{spatial}}, \tau_\theta(y))$$

IP-Adapter 在旁边新加一个图像交叉注意力：$$h_{\text{spatial}} \leftarrow h_{\text{spatial}} + \text{CrossAttn}_{\text{image}}(h_{\text{spatial}}, c_{\text{img}})$$

两者的输出直接相加。图像特征 $c_{\text{img}}$ 来自 CLIP 图像编码器的输出，经过一个可训练的投影网络（小型 MLP + 层归一化）映射到和文本嵌入相同的维度空间。这个投影网络是 IP-Adapter 主要训练的部分——它负责将 CLIP 图像编码器的"整体表示"翻译为 U-Net 交叉注意力能消化的"特征序列"。

为什么叫"解耦"？因为文本和图像的交叉注意力用了**不同的 K 和 V 投影矩阵**。如果共用同一套 K 和 V 矩阵，文本 token 的 77 个向量和图像的整体表示会在同一个线性空间中互相干扰——网络不知道哪部分激活是来自文本语义的、哪部分是图像风格的。解耦后，文本交叉注意力继续做它擅长的事（把文本 token 和空间位置对齐），图像交叉注意力做它擅长的事（把整张图像的全局视觉特质扩散到整个画面），两者通过输出的加法统一融合。

## 训练：从噪声图像对中学习

IP-Adapter 的训练目标很简单：给定一张参考图像和一个文本描述，生成一张同时匹配参考图像的风格/内容和文本描述的语义的图像。训练损失就是标准的去噪损失。

训练数据需要是一个三元组：(参考图, 目标图, 文本描述)。参考图提供视觉特征引导，目标图是去噪训练的 ground truth，文本描述提供语义引导。一个典型的训练样本：参考图是"梵高风格的麦田"，文本是"a starry night sky"，目标图是"一幅梵高风格的星夜画"。模型学会：去噪时要让生成的图像看起来像梵高画的（参考图提供的信息），同时画的内容是星空（文本提供的信息）。

在训练过程中，CLIP 图像编码器是冻结的——它只负责提取参考图的特征，不参与梯度更新。U-Net 的主体也是冻结或仅微调很小的一部分。主要训练的参数是图像投影网络和新增的图像交叉注意力层中的 K 和 V 投影矩阵。这种设计让 IP-Adapter 可以在一个已有的 SD checkpoint 之上以相对小的代价训练出来——原论文报告只需要约 1000 万对训练数据，在几天的 GPU 训练后就能获得泛化良好的图像提示能力。

## 变体：FaceID 和 Plus

IP-Adapter 在发布后衍生出了几个重要的变体。

**IP-Adapter FaceID** 专注于人脸一致性。标准的 IP-Adapter 用 CLIP 图像嵌入作为参考特征，但 CLIP 嵌入在保持精确的身份信息方面并不够强——它更擅长捕捉"一个人脸"和"一只猫"之间的语义差异，而不是"这张脸"和"那张脸"之间的细微几何差异。FaceID 变体引入了一个额外的**人脸识别模型**（通常是 ArcFace 或类似模型），用其人脸嵌入作为补充特征，和 CLIP 嵌入一起注入 U-Net。在生成特定人物时，FaceID 能提供比标准 IP-Adapter 更高的面部保真度。

**IP-Adapter Plus** 是一个工程优化版本，主要是改进了图像投影网络的设计，使其能从一个参考图中提取更丰富的多层次特征——不仅用 CLIP 图像的全局嵌入，还提取 CLIP 视觉 Transformer 中间层的 patch-level 特征（在 ViT 中，最后一层之前的隐藏层保留了更多局部空间信息），并将这些多层次特征一起注入。提升的效果是对参考图的构图结构的尊重程度更高——不仅学到风格，也学到了一些构图上的偏好。

## 和 ControlNet、img2img 的区别

IP-Adapter、ControlNet 和经典的 img2img 都能"用图像影响生成"，但三者的控制维度不同。

**img2img** 是最原始也是最直接的方式——它把参考图通过 VAE 编码为潜变量，加上一定量的噪声，然后从这个带噪潜变量出发做去噪。它强约束了输出和参考图之间的像素级相似性。噪声加得越少，输出越像参考图（在极限情况下等于去噪强度为 0，几乎就是原图的微调版）。问题是你不能同时保留参考图的构图形态和大幅度改变它的内容——你只能做局部的风格迁移或细节修改。

**ControlNet** 控制的是**空间结构**——边缘、深度、骨架。它不在乎参考图的颜色、纹理、光照——它只提取参考图中的空间结构信息，然后让文本去填充这些结构之内的语义内容。ControlNet 无法做"把参考图的风格迁移到新构图上"——因为风格不是空间结构。

**IP-Adapter** 控制的是**风格和全局视觉特征**。它不在乎参考图的精确空间结构——它只提取"这张图整体看起来像什么"的信息，然后将这个信息作为全局视觉 bias 注入生成过程。输出的构图由文本 prompt 主导，但视觉基调由参考图主导。

三者可以组合使用。最经典的多控制管道：IP-Adapter 提供风格和色调基准 + ControlNet 提供姿态和构图控制 + 文本 prompt 提供语义内容 + img2img 提供起始布局。各自取一个维度，互不冲突。

## 实际使用中的细节

IP-Adapter 在推理时有一个**图像引导权重**的超参数，类似于 CFG 的 guidance scale。权重越高，图像参考的影响越强；权重为 0 时退化为标准文本生成。一般推荐的起步值是 0.7 到 1.0。

多张参考图也可以一起使用。IP-Adapter 可以同时接受多个参考图，分别提取 CLIP 嵌入，然后在图像交叉注意力中对它们的特征取平均或加权混合。这对于平衡多种风格特征非常有用。

和文本一样，IP-Adapter 的图像引导在生成早期（$t$ 大时）影响最大——图像的整体风格和色调在去噪的前 30% 步中就已经确立，后续步骤主要是在此基础上做优化和细节打磨。很多 IP-Adapter 用户在实践中选择在去噪的最后一段降低图像引导权重，让文本有更大的自由度来精修细节。
