# 3.2 数据集与标注

## 大就是好吗

Stable Diffusion 的预训练数据量级是数十亿级的图像-文本对。SD 1.x 使用 LAION-2B 的一个精筛子集（约 6 亿对），SD 2.x 进一步扩大了筛选范围，SDXL 和 SD 3 等后续模型使用的数据集更大且经过了更精细的人工审核。这个量级的数据不是任何一个团队能够手工标注的——每一对图像和文本都是从互联网上自动爬取、自动配对的。自动配对意味着噪声：alt-text 可能是无意义的文件名，图像和描述可能完全无关。模型如何处理这种噪声，训练者如何在噪声和规模之间权衡，是整个数据集环节的核心问题。

LAION 的构建流程完全是自动化的：从 Common Crawl 网页快照中提取 `<img>` 标签和 `alt` 属性，用 CLIP 计算图文相似度并过滤低分对，再叠加美学评分、NSFW 过滤和水印检测等自动指标。规模巨大（LAION-5B 包含 58 亿对），但也内建了系统偏差——CLIP 过滤优先保留 CLIP 容易理解的图像，训练出的模型又进一步放大了这种倾向。

图像预处理方面，SDXL 引入的**宽高比分桶**（aspect ratio bucketing）是关键的改进：将图像按宽高比分入不同桶（1:1、4:3、16:9 等），缩放而非裁剪到目标分辨率，让模型学会在不同宽高比下生成合理构图。这个技术也被 LoRA 训练工具（如 Kohya sd-scripts）普遍采用。

以上是预训练的故事。但对于本笔记的读者来说，更实际的场景不是从头训练一个 SD，而是在预训练模型上做**微调**——尤其是用 LoRA 做个性化训练。下面我们把重点放在这里。

## 微调数据集的核心原则

LoRA 的训练数据量通常只有 5 到 50 张图像。在这个量级下，每一张图的影响力都被放大——预训练时一张坏图被数十亿张好图平均掉，微调时一张坏图可能直接污染一个概念。

有两条贯穿整个数据集准备过程的铁律：

**第一条：数据质量比数量重要十倍。** 10 张光线一致、构图清晰、主体突出、背景干净的高质量图像，远胜于 50 张模糊、光照混乱、角度雷同、背景杂乱的图像。一张欠曝的照片会让 LoRA 学会把"欠曝"和你的概念绑定；一张背景中出现同一个红色水杯的照片会让模型以为这个水杯是你概念的一部分。量少而精是 LoRA 数据集的第一准则。

**第二条：多样性是你的防过拟合武器。** 如果 20 张训练图全是在同一个房间、同一种灯光下、同一个角度拍的，LoRA 会学到"这个房间的光线和背景 = 这个概念"。推理时换个背景，概念就变形了。好的数据集需要覆盖：不同的背景、不同的光照条件（自然光 / 室内光，不是同一种灯）、不同的角度（正面、侧面、45 度、远距离、近距离）、不同的服装（如果是人物）、不同的姿态和表情（如果是人物或动物）。多样性的目的是迫使模型把"概念本身"和"拍摄条件"分离开——当概念在多种条件下都出现，而拍摄条件每次都不同，梯度下降会发现唯一的一致信号就是概念本身。

但多样性也有边界。所有训练图像必须在视觉上明确地属于同一个概念。如果你的"概念"是一只猫，训练集里混入了一张邻居家的猫（花色相似但不完全相同），LoRA 会学到两只猫的加权平均，最终生成的不像任何一只。**多样性是在同一概念内的变化，不是概念的模糊化。**

## 不同类型概念的图像要求

不同概念对训练图像的需求差异很大。

**人脸**是要求最高的概念类型。LoRA 需要捕捉五官的精确几何关系和皮肤纹理特征，这要求：至少 10 到 20 张高质量图像；必须覆盖多角度（正面、左右侧面、微微仰视/俯视），否则 LoRA 只能生成训练集中的那几个特定角度；表情需要有多样性（微笑、严肃、自然），但不能有极端的夸张表情（会被学到变形）；背景必须多样化且干净——如果 5 张照片都在同一面白墙前，这面白墙就会变成概念的一部分。眼镜、帽子等配饰要特别注意：如果所有训练图都戴眼镜，眼镜就变成不可拆卸的了；如果有的戴有的不戴，需要在标注中写明，让模型把眼镜视为可选项。

**人物全身 / 服装**类概念对姿态和背景的多样性要求更高。如果所有照片都是站立姿势，LoRA 生成"running"时姿态会不自然。最好包含站、坐、走、转身等不同动作，以及全身、半身、特写等不同景别。服装的褶皱和面料纹理在单一光线下往往呈现特定的高光和阴影模式，不同光照条件可以帮助模型把这些纹理特征和光照解耦。

**物体**类概念（如一个手办、一个杯子、一只宠物）需要覆盖不同角度来捕捉三维形状。同一个物体的正面和侧面在像素层面完全不同，模型需要从多角度的训练数据中推导出物体的三维一致性。背景多样性格外重要——手办放在白色桌面上拍的 10 张图会让模型认为"白色桌面"是手办概念的一部分，推理时无论什么 prompt 都会带白色桌面背景。

**画风 / 风格**概念的数据集构建逻辑完全不同。这里需要的不是同一主体的多角度呈现，而是同一种风格在不同主体上的体现。训练 20 张"梵高风格"应该包含风景、人像、静物等多种题材——因为在风格学习中，多样性不再是防范手段，而是教学核心。你希望模型学到的是笔触、色调、构图偏好这些跨主体的共同特征，而非任何一个具体主体。反过来，如果所有 20 张都是麦田风景，模型会把"麦田"当成风格特征的一部分。

**姿势 / 动作**（如特定的舞蹈动作、武术姿势）需要同一姿势在不同人物、不同服装、不同背景下的呈现，否则模型会把"穿红色衣服的那个人"和"这个姿势"绑定。

## 图像质量的具体标准

以下标准对大多数 LoRA 训练都适用：

**分辨率**：图像的短边不应低于 768 像素（SD 1.5）或 1024 像素（SDXL）。低分辨率图像在编码到潜空间时会丢失细节，这些丢失的细节无法在训练中被恢复。但也不需要过度追求超高分辨率——SD 1.5 的 VAE 在 512 分辨率上训练，超过 1024 的细节会被下采样破坏，反而引入 aliasing 伪影。

**清晰度**：每张图的主体应该对焦清晰，运动模糊和景深模糊只在有意为之且标注中说明了的时候才是可接受的。

**压缩伪影**：JPEG 压缩产生的高频块效应会被 VAE 编码进潜空间，然后被 LoRA 学到。优先使用 PNG 或高质量 JPEG（quality > 90），避免从社交媒体多次转发后下载的图片。

**水印和文字**：图像中的水印、字幕、UI 元素会被模型当作视觉特征学习。如果一张训练图中右下角有半透明的 logo，生成结果可能也会出现类似的 logo 痕迹。务必裁剪或修复这些元素。

**一致性检查**：训练前应该删除构图重复的图像（两张几乎一样但曝光不同的照片只保留更好的一张）、确认训练集里没有意外混入的无关图像（批量下载时经常发生）、确认所有图像确实属于同一个概念。

## 标注：LoRA 质量控制的最大杠杆

标注是 LoRA 训练中最容易被低估的环节。训练图像是输入，标注是教师——它告诉模型"这些视觉特征对应什么语义标签"。标注错了、漏了、多了，模型要么学偏，要么学不会。

### 两种标注流派

LoRA 社区中并存两种标注策略，各有拥趸和适用场景。

**自然语言标注**（也称 "captioning style"）使用完整的自然语言句子描述每张图像，如 "a photo of zwx woman wearing a red dress, standing in a garden, soft sunlight, shallow depth of field"。这种标注直接利用了 CLIP 文本编码器在预训练中学会的语言理解能力——CLIP 天生就理解 "red dress" vs "blue dress"、"garden" vs "office" 的视觉差异。它的优势是和预训练模型的文本空间高度对齐，推理时可以自由地用自然语言组合各种属性。缺点是自动标注模型（如 BLIP）产生的描述可能包含错误或遗漏，而这些错误会原封不动地传递到 LoRA 中。

**标签标注**（也称 "tagging style" 或 "Danbooru style"）使用逗号分隔的标签列表，如 "zwx, 1girl, red dress, standing, garden, soft lighting, bokeh"。这种标注源自动漫图像社区（Danbooru 的文化），经 WD14 tagger 的广泛使用而成为主流实践之一。它的优势是简洁——每个概念是独立的标签，推理时增删一个标签只影响一个维度，组合性强。劣势是 CLIP 文本编码器并没有在标签列表上大规模预训练过（它主要见过自然语言句子），标签格式和 CLIP 的训练分布有 gap——虽然实践证明这个 gap 在很多场景下并不致命，尤其是在动漫/二次元风格的微调上表现得很好。

**哪种更好？** 没有标准答案，取决于你的概念类型和偏好。人脸/人物类通常推荐自然语言标注——人脸细节需要丰富的语义上下文来精确解耦。物体/风格类两种均可。动漫/二次元类标签标注有压倒性的社区实践优势（WD14 的生态）。许多有经验的训练者采用混合策略：核心触发词用标签式的一个 token（如 "zwx"），其他属性用自然语言描述。关键是选定一种后保持**一致性**——不要在数据集中混用两种标注风格，这会让模型在两种语义模式之间困惑。

### 触发词的设计

触发词（trigger word）是 LoRA 数据集中最重要的一个标签——它是你在推理时用来激活概念的"钥匙"。

一个好的触发词应该满足：在 CLIP 的 tokenizer 词表中**不存在或极其罕见**，以确保它的嵌入向量在训练前几乎是空白的。不能是一个有意义的英文单词——"rose" 作为触发词会让 LoRA 和模型已有的"玫瑰花"语义纠缠在一起，训练效果打折且推理不稳定。也不能是太短的 token——单个字母或数字的组合在 CLIP tokenizer 中可能被切分为不直观的子词。

社区实践中，常见做法是用 2 到 4 个字母的无意义组合，如 "zwx"、"sks"、"htn"，或者是概念名的变体，如你叫 Bob 可以发明 "b0bzz" 作为触发词。关键是在整个数据集中**统一使用同一个触发词**，并且推理时只用这一个词来激活概念。

触发词在每张训练图像的标注中都必须出现——这是所有训练样本之间唯一的语义不变项。如果有些图写了触发词有些没写，模型无法判断触发词到底在指什么。

### 写了什么，模型就学什么；没写的，也可能被学到

这是 LoRA 标注中最微妙也最重要的认知。

**写了的东西变成可控条件。** 如果你在所有标注中都写了 "zwx woman wearing glasses"，模型学会 "glasses" 是可选项——推理时写 "glasses" 就戴眼镜，不写就不戴。**但前提是训练集中有的图戴眼镜、有的图不戴，且标注据实反映了这种变化。** 如果你每张训练图里人物都戴眼镜，而且每张标注都写了 "wearing glasses"，那么写不写 glasses 在推理时都会戴眼镜——因为模型没见过不戴眼镜的版本，没有学习到眼镜的"开/关"这个维度的分离表征。

**没写的东西会成为不可控的默认特征。** 如果训练图中都有一个特定的棕色皮沙发作为背景，但标注中从未提及这个沙发，模型会把这个沙发当作"zwx woman"概念的一个隐式部分。推理时即使你在 prompt 中写 "standing in a forest"，她身后仍然可能浮现棕色沙发的纹理——因为模型学到的概念分布中，“棕沙发背景”被编码为一个 latent prior。

**写出来但标注错误的东西直接毒化概念。** 一张图中人物穿的是深蓝色牛仔裤，但 BLIP 自动标注为 "black pants"。这个错误如果没被人检查修正，LoRA 会学到"深蓝色牛仔裤的颜色叫做 black"。在以后的生成中，"black pants" 可能出现深蓝色的偏移，"blue jeans" 则可能变淡或偏色。

这三种情况的处理策略：

- **你希望在推理时能控制的属性**（服装颜色、发型、表情、背景类型、光照条件）→ 在标注中写明，并确保训练集中有这个属性的不同取值。
- **你希望始终和概念绑定的属性**（身体特征、脸型、不可改变的身份标记）→ 不要在标注中写，让它隐式地成为触发词的一部分。
- **你完全不想要的背景杂物**（路人、logo、脏乱的桌面）→ 在准备数据时就**物理删除**（裁剪、修复、换图），不要指望通过"不写"就能让模型忽略——"不写"意味着它变成隐式默认值。

### 自动标注与人工修正

对于 20 到 50 张图的 LoRA 训练集，强烈建议走一遍"自动标注 + 人工复核修正"的流程。

先用 BLIP、WD14 或 JoyCaption 等工具自动生成初始标注。BLIP 系列（BLIP-2、BLIP-3）擅长自然语言描述，WD14 擅长标签列表，JoyCaption 是较新的专门为扩散模型微调优化的标注模型，生成的描述质量普遍更好。

然后人工逐张检查：自动标注的颜色是否准确（BLIP 经常把蓝色识别为紫色、把灰色识别为棕色）；物体的空间关系是否正确（"cat on the table" vs "cat next to the table"）；是否有自动标注没提到但你希望可控的属性（如你想控制发型，但 BLIP 没写，就自己加上）。检查标注时问自己一个问题："如果我只读这段标注，我能多准确地还原出这张图的关键视觉特征？"如果不能，标注就需要补充。

对于 200 张以上的大规模 LoRA 训练集，人工逐张检查不现实。此时的标准做法是信任自动标注的一致性，但至少抽查 20% 的标注确认质量，并确保触发词在使用中保持一致。

### 负面样本和正则化图像

一个经常被问到的问题是：LoRA 训练需要像 Dreambooth 那样的类别先验图像（regularization images）吗？

通用共识是：**LoRA 通常不需要。** LoRA 只更新 U-Net 中极小比例的参数，不足以引发全量微调级别的 Language Drift。但有一个边界情况值得注意：如果你训练的概念和预训练模型中已存在的某个高频概念高度重合（比如训练一个特定的人脸，而基础模型中已经有海量"人脸"的强先验），加入少量正则化图像或对某些已有概念的衰减训练可能会改善结果。在 Kohya sd-scripts 中，可以通过 `--reg_data_dir` 指定正则化图像目录，但绝大多数 LoRA 训练者从不使用这个功能，且结果完全 OK。

## 数据集的组织

在磁盘上，一个 LoRA 训练集的标准目录结构如下：

```
dataset/
  ├── 1_zwx woman standing in garden.jpg
  ├── 1_zwx woman sitting on sofa.jpg
  ├── 1_zwx woman close-up portrait.jpg
  ├── ...
  └── 1_zwx woman walking on beach.jpg
```

每张图像文件对应一个同名的 `.txt` 标注文件：

```
1_zwx woman standing in garden.txt      →  "zwx woman, standing, garden, natural light"
1_zwx woman sitting on sofa.txt         →  "zwx woman, sitting, indoor, sofa, warm lighting"
1_zwx woman close-up portrait.txt       →  "zwx woman, close-up, portrait, soft focus"
1_zwx woman walking on beach.txt        →  "zwx woman, walking, beach, sunset, ocean background"
```

关键的约定：Kohya sd-scripts 将文件名开头的数字理解为**重复次数**（repeats）。数字 `1_` 表示这张图在每个 epoch 中被使用 1 次。如果你有一张角度独特性特别高的图想让模型多看看，可以设置 `5_` 让它在每个 epoch 重复 5 次。对于小数据集（< 20 张），通常全部 `1_` 就够了，数据增强会自动提供多样性。

图像和标注放在同一个目录下，训练脚本按文件名匹配。这是社区最通用的组织方式，在 Kohya sd-scripts、OneTrainer、sd-scripts 的各种 GUI wrapper 中都能直接使用。

## 常见错误清单

以下是社区中反复出现的 LoRA 训练数据错误：

**数据集太"干净"。** 20 张图全在同一个房间、同一天拍、同一个角度、同样的表情。模型产生了强烈的过拟合，推理时任何偏离训练分布的条件都会导致质量崩溃。

**标注泄漏。** 标注中写了一个属性但训练集中这个属性从不变化（如所有图标注了 "smiling" 但所有图都在微笑）。推理时不写 "smiling" 也会微笑，写了 "angry" 还是在微笑——因为模型没学会这个属性是个可调的自由度。

**触发词不一致。** 数据集 20 张图用了 3 种不同的触发词，或者有的写了触发词有的没写。训练结束后触发词不能稳定激活概念。

**训练数据中包含生成图像的"污染"。** 用 Stable Diffusion 生成了一些参考图然后混入训练集，或者从 CivitAI 下载了别人的 AI 图来做训练数据。AI 生成图像中蕴含的 VAE 和 U-Net 的特定伪影会被 LoRA 二次学习并放大，产生类似"复印机的复印"的退化效应。

**背景里的小物件。** 训练过程中没注意到的背景细节——墙上的一幅画、桌上的一个杯子、窗外的树——都可能被 LoRA 当作概念特征内化。检查训练图时问自己："如果生成的图像中出现了这个细节，我会觉得它是 bug 还是 feature？"如果是 bug，就该在数据准备阶段处理掉。

**概念定义太宽。** "我想训一个艺术风格 LoRA"，然后放了 50 张风格各异的画进去。LoRA 无法从中提取统一模式，训练出的效果是模糊的"平均画风"而非任何明确的风格。风格型 LoRA 需要数据集中存在肉眼可见的一致视觉模式。

**概念定义太窄。** 反过来，"我想训一个特定人物的 LoRA"，但只放了她 3 张 45 度角微笑的户外照片。推理时不论 input 是什么，输出永远是一个 45 度角微笑的户外人像。

下一节，我们将进入 Dreambooth 的技术细节——它和 LoRA 走的是截然不同的微调路线。
