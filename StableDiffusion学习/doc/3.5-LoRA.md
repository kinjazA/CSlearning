# 3.5 LoRA：低秩的魔力

## 一个经验观察

2018 年，几位研究者在分析大规模神经网络时注意到了一个现象：过参数化的模型虽然权重矩阵看起来是满秩的，但它们在微调时实际发生的变化，往往集中在矩阵的一个很小的**低秩子空间**中。也就是说，全量微调产生的权重变化矩阵 $\Delta W$——虽然它和 $W$ 本身尺寸相同，可以是一个几万乘几万的庞大矩阵——但其"有效秩"通常很低，可以用两个小得多的矩阵的乘积来近似。

这个观察最初是在 NLP 领域的大语言模型微调中被 Edward Hu 等人系统化地提出的，论文名为 *LoRA: Low-Rank Adaptation of Large Language Models*（2021）。这个思想后来被 Simo Ryu 等人迁移到了 Stable Diffusion 的 U-Net 中，迅速成为扩散模型社区最重要的微调方法。

为什么低秩假设在扩散模型中同样成立？直觉是这样的：预训练的 U-Net 已经学会了从噪声中重建图像的通用能力——它知道物体的形状、纹理、光照在潜空间中应该怎么分布。当你微调它来学一个新概念（比如你家猫的样子、梵高的画风），你不是在从零教它理解视觉世界，而是在已知的概念基座上叠加一层薄薄的"个性化偏差"。这层偏差的复杂性远低于基础能力——它只需要编码"这只特定猫的毛色花纹和别的猫有什么不同"，而不是"猫是什么"——所以它可以被压缩到一个低维子空间中。

## 权重分解：一大变两小

LoRA 的核心操作非常简洁。假设 U-Net 中有一个线性层（注意力层中的 Q、K、V、O 投影矩阵，或者卷积层展开后的等效矩阵），其权重矩阵为 $W \in \mathbb{R}^{d_{\text{out}} \times d_{\text{in}}}$。在微调过程中，这个矩阵接收到的更新是一个同样尺寸的矩阵 $\Delta W$。LoRA 不做全量更新，而是将 $\Delta W$ 分解为两个小矩阵的乘积：$$\Delta W = B A$$

其中 $B \in \mathbb{R}^{d_{\text{out}} \times r}$，$A \in \mathbb{R}^{r \times d_{\text{in}}}$，而 $r$ 是**秩**（rank），$r \ll \min(d_{\text{in}}, d_{\text{out}})$。对于 SD 1.x 的 U-Net，$d_{\text{in}}$ 和 $d_{\text{out}}$ 通常在 320 到 1280 之间，而 $r$ 通常取 4 到 128。当 $r=8$ 时，$A$ 和 $B$ 的总参数量仅是原始矩阵 $W$ 的 $8 \times (d_{\text{in}} + d_{\text{out}}) / (d_{\text{in}} \times d_{\text{out}})$——当维度较大时，这个比例可以低至千分之一以下。

前向传播时，原始层的计算变为：$$h_{\text{out}} = W h_{\text{in}} + \Delta W h_{\text{in}} = W h_{\text{in}} + B A h_{\text{in}}$$

第一项 $W h_{\text{in}}$ 使用冻结的预训练权重计算（不产生梯度）。第二项 $B A h_{\text{in}}$ 携带 LoRA 的可训练参数。推理时，可以将 $\Delta W = BA$ 加到 $W$ 上合并为一个矩阵，无额外推理开销——这是 LoRA 相比其他 adapter 方法的一个重要优势。

用更形象的语言来理解这个分解：$A$ 矩阵将输入向量从 $d_{\text{in}}$ 维压缩到 $r$ 维（相当于在问"输入的哪几个方向对于新概念最重要？"），$B$ 矩阵将 $r$ 维的压缩表示再映射回 $d_{\text{out}}$ 维（相当于"这几个重要方向应该对应输出空间的哪些变化？"）。$r$ 就是压缩瓶颈的宽度——它决定了这个适配器有多少"自由度"来表达新概念和旧概念之间的差异。

## 哪些层需要 LoRA

在 Stable Diffusion 的 U-Net 中，LoRA 通常施加在交叉注意力和自注意力的 Q（Query）、K（Key）、V（Value）、O（Output）四个投影矩阵上。这是社区经过大量实验后收敛的最佳实践。

为什么是注意力层而不是卷积层或 FFN 层？交叉注意力是文本信号进入 U-Net 的唯一通道——一个 LoRA 施加在这里，可以直接影响"文本的哪些语义对应图像的哪些特征"，这是控制生成内容最直接的杠杆。自注意力负责图像不同空间位置之间的信息交互——LoRA 施加在这里可以影响构图的全局布局、物体的相对位置和形状一致性。

更重要的是，注意力层的 Q、K、V、O 矩阵天然适合低秩分解。这些矩阵在预训练模型中往往具有较高的内在秩，但在微调时发生的变化——如前所述——通常集中在低秩子空间中。相比之下，卷积层的核本身参数量小（典型的 3×3 卷积核），低秩分解的收益不明显，而且对低秩假设的成立性也缺乏验证。

实践中，一个标准的 SD 1.5 LoRA 作用于 U-Net 所有层的 Q、K、V、O 矩阵，总计约 50 到 100 个线性层。当 $r=8$ 时，总可训练参数通常在 10M 到 20M 之间，仅占 U-Net 总参数（860M）的约 1% 到 2%。文件大小在 10MB 到 200MB 之间（取决于 rank 和是否使用 fp16 存储），相对于 Dreambooth 的 4GB 到 5GB，优势明显。

## 训练一个 LoRA：从数据到参数

LoRA 的训练过程在宏观上和 Dreambooth 一致——同样是标准的扩散损失，同样是冻结 VAE 和 CLIP 文本编码器。但一个关键的简化是，LoRA 通常可以**不加先验保留损失**。

为什么 LoRA 不需要？Dreambooth 需要先验保留损失，是因为它在全量更新 U-Net，模型容易发生 Language Drift——学的概念会侵占"猫"这个词原本的语义空间。LoRA 只更新了 U-Net 中极少数参数，外加一个标识触发词，它的修改被局限在极小的参数空间里，不足以显著漂移 "cat"、"dog" 这类通用词的语义。而且 LoRA 通常和带触发词的描述搭配使用（如 "a photo of mycat, sitting"），推理时如果不写触发词，LoRA 大概率不会激活，进一步降低了副作用。但这并不意味着 LoRA 的训练可以随便来。这里有三个关键的超参数直接决定微调质量：

**秩 $r$** 控制模型的表达能力上限。$r$ 太小，LoRA 学不全概念的所有细节——比如人脸微表情或复杂的织物纹理可能需要更多的自由度来捕捉。$r$ 太大，可训练参数变多，小数据集下容易过拟合，且模型文件增大。对于简单概念（一种颜色风格、一种构图偏好），低于 $r=8$ 可能就够了。对于复杂主体（特定人脸 + 多种服装 + 多种姿态），$r=32$ 到 128 更合适。

**网络 alpha（$\alpha$）** 出现在 LoRA 前向公式的缩放因子中：$h_{\text{out}} = W h_{\text{in}} + \frac{\alpha}{r} \cdot B A h_{\text{in}}$。

初看这个公式，很可能会产生困惑：`r=8, α=8` 时 `α/r = 1`，乘个 1 有什么意义？为什么不直接写成 $W h_{\text{in}} + B A h_{\text{in}}$？

答案藏在 **$r$ 和 $\alpha$ 解耦了模型表达能力和有效学习率**。$B$ 和 $A$ 在初始化时通常使用 Kaiming 初始化——$A$ 用标准正态分布，$B$ 用全零矩阵——这意味着训练开始时 $\Delta W = 0$，无论 $r$ 取多少初始状态完全一致。但当 $r$ 改变时，参与更新的参数量变了，同样的学习率会产生不同大小的有效步长：$r$ 翻倍意味着有更多的参数在推动 $\Delta W$，梯度的总贡献会相应增大。如果不在前面放一个缩放因子和 $r$ 耦合，你每换一个 $r$ 都要重新调学习率。

把 $\alpha$ 设为和 $r$ 相等的值（$\alpha/r = 1$），是将 $r=8$ 的动力学作为**默认基准线**。当你想要增强 LoRA 的影响时，不需要重调学习率，直接把 $\alpha$ 设为 $r$ 的两倍（$\alpha/r = 2$）——相当于把整个 $\Delta W$ 的幅度放大两倍。当你想要减弱时，把 $\alpha$ 设为 $r$ 的一半（$\alpha/r = 0.5$）。$\alpha$ 和 $r$ 分离之后，$r$ 只管"给 LoRA 多少自由度"，$\alpha$ 只管"这些自由度的影响力有多大"，两者独立可调。

社区经验法则 "$\alpha = r$" 并非"乘了等于没乘所以无所谓"，而是一个精心设计的**校准基点**——让 $r=8$ 时的有效更新幅度等于直接被实验验证为合理的默认值，然后你用 $\alpha$ 在这个基点上放大或缩小。

**学习率** 是 LoRA 训练中最容易踩坑的超参数。U-Net 的注意力层在不同深度对语义的响应不同——浅层（高分辨率层如 64×64）处理纹理和边缘，深层（低分辨率如 8×8 瓶颈层）处理全局构图和语义。因此一个越来越普遍的做法是**分层学习率**：给深层（瓶颈层和 16×16 层）更高的学习率（因为这里有更多的语义信息，且较少有预训练的强约束），给浅层（64×64 和 32×32 层）较低的学习率（防止纹理层面过拟合训练集的少数几张图）。更精细的策略还包括对 Q、K、V 矩阵和 O 矩阵使用不同的学习率——Q 和 K 影响注意力图中的匹配模式，V 和 O 影响注意力的输出表示，两者的学习动态不同。

## LoRA 的变体景观

LoRA 的流行催生了一系列变体，各有微调。

**LoCon**（LoRA for Convolution）将低秩分解推广到卷积层。它不直接分解 $k \times k$ 卷积核（太低效），而是将卷积视为 $k^2 \times c_{\text{in}} \times c_{\text{out}}$ 的线性变换，用两个比这个小得多的卷积来近似其更新。LoCon 在处理某些需要精细纹理控制的任务上有优势——因为纹理信息主要集中在 U-Net 的浅层（高分辨率层），而这些层的卷积操作比重较大。

**LoHa**（Low-rank Hadamard）在联邦学习社区中起源，使用 Hadamard 积来做分解，在某些场景下比标准 LoRA 训练更稳定。

**IA3** 甚至更极端——它不给每个权重矩阵加独立的适配器，而是在整个网络中只插入可学习的缩放向量（每个中间特征维度一个）。它的可训练参数量比 LoRA 还少几十倍，但表达能力也相应更低。

**QLoRA** 结合了 LoRA 和 4-bit 量化。先对预训练权重做 4-bit NormalFloat 量化（减少显存占用），然后在量化后的模型上训练 LoRA 权重——训练时的梯度反传需要短暂地反量化权重到 fp16 再计算。这个技术主要面向在消费级 GPU（如 RTX 3090 / 4090 的 24GB VRAM）上微调超大规模模型。对于 SD 1.5 的 U-Net（860M 参数），一张 24GB 的卡用标准 LoRA 已经可以轻松训练，QLoRA 的额外收益不大。但对于 SDXL（U-Net 约 2.6B 参数）或更大模型，QLoRA 是能否在自己电脑上跑的临界点。

## 实践中训练一个好 LoRA

社区在经历了大量 trial and error 后，总结出了一些提高 LoRA 质量的实践知识。

**数据第一**。这一点在 3.2 节已经强调过，但值得在此重申：一个 LoRA 的质量，80% 取决于训练数据的质量。10 张光线一致、构图清晰、主体突出的高质量图像，远胜于 50 张模糊、光照混乱、角度重复杂乱的图像。数据集中应该有意覆盖多样化的背景、角度和光照条件，防止 LoRA 学到"砖墙背景 = 我的猫"这样的伪关联。

**标签决定学什么**。LoRA 训练中，你标注了什么，模型就会学什么。如果你在每张训练图像的描述中都写了 "mycat, a cat sitting, indoor, soft lighting, orange tabby, brick wall background"，模型会学会将所有这些东西和触发词 "mycat" 关联起来——包括砖墙背景。如果推理时不想要砖墙背景，你需要在训练标注中**把不想学的东西写出来**让它成为可控的条件，或者**完全不写**让它隐式地消失。策略一是写全然后推理时通过负面 prompt 去掉（更灵活但更难训练），策略二是只写你希望和触发词绑定的内容（如 "mycat, orange tabby cat"），让背景、光照等信息作为"无标注噪声"被模型平均掉。两种策略各有适用场景，取决于你希望的推理灵活度。

**步数控制**。LoRA 的训练步数通常比 Dreambooth 多，因为参数更新受限，收敛慢。对于 20 张左右的数据集，通常需要 1000 到 3000 步总的优化。过少的步数会导致概念学不全（underfitting）——触发词只能部分激活目标特征。过多的步数会导致过拟合（overfitting）——触发词不仅激活了目标特征，还在生成的图像中印上了训练图像的固定构图和背景（所谓"图像记忆"或 "memorization"）。中间步骤的检查点保存是重要的实践——训练过程中每 500 步或 1000 步保存一次，推理测试，选择最优。

**正则化少即是多**。与 Dreambooth 不同，LoRA 训练中额外加正则化通常会降低效果。权重衰减（weight decay）、dropout、强数据增强等标准深度学习的正则化手段，在 LoRA 微调中往往弊大于利——因为低秩分解本身已经内置了强正则化（参数空间被限制在低秩流形上），再加额外正则会限制模型捕捉细节的能力。唯一常用的"正则化"是对训练图像的少量随机裁剪和翻转（数据增强），目的是增加数据多样性而非限制模型复杂性。

## LoRA 的组合与操作

LoRA 的一个独特优势是**可叠加**。你可以训练一个"梵高画风"的 LoRA、一个"赛博朋克城市"的 LoRA、一个"特定人物"的 LoRA，在推理时同时加载三者，调节各自的权重来控制混合比例。在 Automatic1111 和 ComfyUI 中，LoRA 的权重是一个标量系数——1.0 表示完整强度，0.5 表示半强度，-1.0 表示反方向（减去该概念的特征）。

LoRA 权重之间不是严格线性可加的——同时加载多个 LoRA 时，它们对同一层权重矩阵的更新会叠加。如果两个 LoRA 修改了同一个注意力层中 V 矩阵的同一行方向，它们的组合效果可能出现非线性的交互（"冲突"）。但在实践中，只要每个 LoRA 的 rank 不大且权重不过高，这种叠加通常是平滑且可控的。

还有一个鲜为人知但非常有用的操作：**通过加权求和对已训练的 LoRA 做概念混合**。不需要重新训练，直接把两个 LoRA 的权重做逐参数的插值——$W_{\text{merged}} = \lambda W_{\text{LoRA1}} + (1 - \lambda) W_{\text{LoRA2}}$——就能产生两者的概念混合。例如，将"素描画风"的 LoRA 和"水彩画风"的 LoRA 做 0.5:0.5 的加权平均，可以得到中间画风的输出。这在概念空间中的语义探索非常有用。

## LoRA 的深层意义

如果把视线从技术上抬起来，LoRA 的成功不仅仅在于它是一个"更省显存的微调方法"。它的深层意义是降低了扩散模型个性化的门槛，从而催生了整个社区生态。

一个 Dreambooth checkpoint 是 4GB，而一个 LoRA 是 10MB。4GB 意味着你需要专门的模型托管网站（CivitAI 上绝大多数是 LoRA 而非 Dreambooth 文件），意味着下载需要等几分钟而非几秒，意味着你无法在一张消费级 GPU 上同时加载两个概念。10MB 则把模型分享变得像分享图片一样便捷，让"训练自己的概念 → 分享到社区 → 别人下载使用"的循环加速了几个数量级。

CivitAI 上数十万的 LoRA 文件、各种概念和风格的百花齐放、以及整个 Stable Diffusion 生态的繁荣——LoRA 是这一切的底层基础设施。
