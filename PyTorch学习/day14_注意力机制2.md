# 01 多潜变量注意力机制（Multi-Latent Attention，MLA）

是 DeepSeek-V2/V3 中提出的一种全新的注意力机制。它与 MQA/GQA 的"减少 KV 头数"思路截然不同，走了一条"**低秩压缩 KV Cache**"的路线，在 KV Cache 压缩率上甚至超越了 MQA，同时模型质量还能超过 MHA



MLA 通过**低秩联合压缩** $K$和 $V$到一个低维潜在向量，实现了比 GQA 更激进的 KV Cache 压缩。与 GQA 减少 KV 头数的思路不同，MLA 从维度角度入手：不再分别缓存完整的 K 和 V，而是缓存一个更小的**联合压缩表示** ($c_t \in \mathbb{R}^{d_c}$)，其中 \($d_c \ll d_{\text{model}}$)。解码时从 ($c_t$) 恢复出 K 和 V



这里涉及到压缩后处理旋转位置编码的问题，这个笔记专注于注意力机制及其变形。因此，等后续位置编码学习后再来补充

# 02 滑动窗口注意力机制

## 02.1 理论直觉

核心还是解决普通注意力机制里内存占用过大的问题。思想就是，不让每个位置去看全序列，而是只看它附近的一个局部窗口。这里暗含了一个token最相关的，就在它自身附近的一个局部范围



比方说窗口半径是$w$，那么第$i$个token的注意力范围就限定在$[i-w,i+w]$，然后对于第$i+1$个以及之后的更多，窗口就会沿着序列滑过去



这里其实还有一个意思，就是$w$肯定是一个远比$n$（全部token数）要小的值，这样才能节省

## 02.2 手撕代码

```python
import torch
import torch.nn.functional as F

def sliding_window_attention_naive(Q, K, V, window_size, causal=False):
    """
    Q, K, V: [B, H, N, D]
    window_size: int, 表示单侧窗口半径 w
    causal: 是否使用因果掩码
    """
    B, H, N, D = Q.shape
    output = torch.zeros_like(Q)

    scale = D ** -0.5

    for i in range(N):
        if causal:
            left = max(0, i - window_size)
            right = i
        else:
            left = max(0, i - window_size)
            right = min(N - 1, i + window_size)

        # 取当前位置 query: [B, H, 1, D]
        q_i = Q[:, :, i:i+1, :]

        # 取局部窗口 key/value
        k_local = K[:, :, left:right+1, :]   # [B, H, W, D]
        v_local = V[:, :, left:right+1, :]   # [B, H, W, D]

        # attention score: [B, H, 1, W]
        scores = torch.matmul(q_i, k_local.transpose(-1, -2)) * scale

        # softmax over local window
        attn = F.softmax(scores, dim=-1)

        # 加权求和: [B, H, 1, D]
        out_i = torch.matmul(attn, v_local)

        output[:, :, i:i+1, :] = out_i

    return output
```
