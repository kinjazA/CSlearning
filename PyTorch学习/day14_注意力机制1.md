# 01 背景

## 01.1 Seq2Seq 的瓶颈

在经典的编码器—解码器（Encoder-Decoder）架构中，编码器将整个输入序列压缩为一个**固定长度的上下文向量** \(c\)，解码器再从 \(c\) 生成输出：

$$
c = \text{Encoder}(x_1, x_2, \dots, x_T)
$$
**核心问题**：当输入序列很长时，单一向量 \(c\) 无法承载所有信息，导致长距离依赖丢失，这被称为**信息瓶颈（Information Bottleneck）**

## 01.2 注意力的直觉

注意力机制的核心思想：**解码器在每一步生成时，不再只看一个固定向量，而是动态地"关注"编码器的不同位置，按相关性加权汇聚信息**。这类似于人类阅读时，眼睛会反复扫视与当前理解最相关的部分

# 02基础注意力机制

## 02.1 注意力通用框架

不管是加性还是乘性，注意力机制都分三步。想象在图书馆找资料。脑子里有一个**问题**（Query），书架上每本书都有一个**标签**（Key）和**内容**（Value）。要做的事情是：

1. **比较**：拿问题和每本书的标签对比，看哪些书和问题最相关 → 得到一个"相关性分数"
2. **归一化**：把这些分数转化为"注意力权重"（加起来等于 1，像概率一样）
3. **汇聚**：按照权重，把各本书的内容加权混合，得到最终需要的信息==(这里我理解就是说我的问题可能和很多本书都有关，然后我根据这个问题和标签计算得到的相关性矩阵来选择性的加权书的内容，即我要的答案)==

用数学语言写出来：

**第一步：算分数**
$$
e_{t,i}=score(st,hi)
$$
这里 $s_t$是解码器在第$ t$步的隐状态（"问题"），$h_i$是编码器第$i$个位置的隐状态（第$i$本书的"标签"）。`score` 函数怎么定义，就是加性和乘性的区别所在

**第二步：归一化**
$$
\alpha_{t,i}=\frac{\exp(e_{t,i})}{\sum_{j=1}^T\exp(e_{t,j})}
$$
就是对所有位置做一个 softmax，把分数变成 0 到 1 之间、且总和为 1 的权重

**第三步：加权求和**
$$
c_t = \sum_{i=1}^{T} \alpha_{t,i} \cdot h_i
$$


权重大的位置贡献多，权重小的位置贡献少，最终得到一个"精选摘要" $c_t$。现在问题归结为：**score 函数怎么设计？** 这就引出了两种经典方案

## 02.2 加性注意力

加性注意力的思路是：不直接比较$s_t$ 和$h_i$，而是**把它们都投影到一个共同的空间里，拼在一起，让一个小型神经网络来判断它们有多相关**。就好比不直接比较两个东西，而是请一个"评委"来打分

### 1 数学推导

**第一步：分别投影**

把解码器状态$s_t$和编码器状态$h_i$分别通过一个线性变换，映射到同一个维度$d_a$：
$$
W_s s_t \in \mathbb{R}^{d_a}, \quad W_h h_i \in \mathbb{R}^{d_a}
$$
**第二步：相加 + 非线性激活**

把两个投影结果加起来，过一个 tanh：
$$
u_{t,i} = \tanh(W_s s_t + W_h h_i)
$$
为什么要加起来而不是拼接？因为加法在这里等价于先拼接再用特定结构的权重矩阵做线性变换，但参数更少。为什么要 tanh？因为需要非线性。如果没有 tanh，两次线性变换叠加还是线性变换，表达能力就退化了

**第三步：打分**

用一个可学习向量 $v \in \mathbb{R}^{d_a}$把 $d_a$维的向量压缩成一个标量分数：
$$
e_{t,i} = v^{\top} \tanh(W_s s_t + W_h h_i)
$$
可以把这整个过程看成一个**单隐层的小 MLP**：输入是$s_t$ 和$h_i$，隐层维度是$d_a$，激活函数是`tanh`，输出是一个标量

**完整流程串起来**：
$$
e_{t,i} = v^{\top} \tanh(W_s s_t + W_h h_i)
$$

$$
\alpha_{t,i} = \text{softmax}i(e_{t,i}) = \frac{\exp(e_{t,i})}{\sum_j \exp(e_{t,j})}
$$

$$
c_t = \sum_i \alpha_{t,i} \, h_i
$$

### 2 可学习参数清单

| 参数  | 形状                     | 作用                 |
| ----- | ------------------------ | -------------------- |
| $W_s$ | $(d_a, d_s)$      | 投影解码器状态       |
| $W_h$ | $(d_a, d_h)$ | 投影编码器状态       |
| $v$   | $(d_a,)$        | 将隐层输出压缩为标量 |

其中$d_s$是解码器隐状态维度，$d_h$是编码器隐状态维度，$d_a$是注意力隐层维度（超参数）一个重要的优点：**$d_s $和$d_h$可以不同**，因为它们各自被投影到了$d_a$维

### 3 代码

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class BahdanauAttention(nn.Module):
    def __init__(self, encoder_dim: int, decoder_dim: int, attn_dim: int):
        """
        encoder_dim: 编码器隐状态维度 d_h
        decoder_dim: 解码器隐状态维度 d_s
        attn_dim:    注意力隐层维度 d_a（超参数，通常取和隐状态差不多大）
        """
        super().__init__()
        # W_h: 把编码器状态从 d_h 维投影到 d_a 维
        self.W_h = nn.Linear(encoder_dim, attn_dim, bias=False)
        # W_s: 把解码器状态从 d_s 维投影到 d_a 维
        self.W_s = nn.Linear(decoder_dim, attn_dim, bias=False)
        # v: 把 d_a 维的隐层输出压缩成 1 个标量
        self.v = nn.Linear(attn_dim, 1, bias=False)

    def forward(self, encoder_outputs, decoder_hidden):
        """
        encoder_outputs: (batch, src_len, encoder_dim)
            编码器每个时间步的隐状态，一共 src_len 个
        decoder_hidden:  (batch, decoder_dim)
            解码器当前时间步的隐状态（一个向量）
        """
        # === 第一步：分别投影 ===
        # encoder 端：对每个位置都投影
        # (batch, src_len, encoder_dim) → (batch, src_len, attn_dim)
        proj_encoder = self.W_h(encoder_outputs)

        # decoder 端：只有一个向量，需要扩展维度以便和 encoder 相加
        # (batch, decoder_dim) → (batch, 1, decoder_dim) → (batch, 1, attn_dim)
        proj_decoder = self.W_s(decoder_hidden.unsqueeze(1))
        # unsqueeze(1) 是为了广播：(batch, 1, attn_dim) + (batch, src_len, attn_dim)
        # 会自动广播成 (batch, src_len, attn_dim)

        # === 第二步：相加 + tanh + 打分 ===
        # (batch, src_len, attn_dim)
        energy = torch.tanh(proj_encoder + proj_decoder)

        # (batch, src_len, attn_dim) → (batch, src_len, 1) → (batch, src_len)
        scores = self.v(energy).squeeze(-1)

        # === 第三步：softmax 归一化 ===
        # (batch, src_len)，每个位置得到一个 0~1 的权重，总和为 1
        attn_weights = F.softmax(scores, dim=-1)

        # === 第四步：加权求和 ===
        # attn_weights: (batch, src_len) → (batch, 1, src_len)
        # encoder_outputs: (batch, src_len, encoder_dim)
        # bmm 结果: (batch, 1, encoder_dim) → squeeze → (batch, encoder_dim)
        context = torch.bmm(attn_weights.unsqueeze(1), encoder_outputs).squeeze(1)

        return context, attn_weights
```

## 02.3 乘性注意力

乘性注意力的思路更直接：**直接用点积来衡量两个向量的相似度**。点积的几何含义是什么？两向量的点积 $a⋅b=∥a∥∥b∥cos⁡θ$。方向越一致（$\theta$越小），点积越大。所以点积天然就能度量"两个向量有多相似"

### 1 三种方式

| 名称    | 公式                                    | 特点                 |
| ------- | --------------------------------------- | -------------------- |
| Dot     | $_{t,i} = s_t^{\top} h_i$             | 最简单，要求维度相同 |
| General | $_{t,i} = s_t^{\top} W h_i$           | 引入可学习矩阵       |
| Concat  | $_{t,i} = v^{\top} \tanh(W[s_t; h_i])$| 等价于 Bahdanau      |

```python
class LuongAttention(nn.Module):
    def __init__(self, method: str, hidden_dim: int):
        """
        method:     'dot', 'general', 或 'concat'
        hidden_dim: 隐状态维度（dot 模式要求 encoder 和 decoder 维度相同）
        """
        super().__init__()
        self.method = method
        
        if method == "dot":
        # 什么都不用创建，点积不需要参数
        	pass  

        elif method == "general":
            # W 矩阵：(hidden_dim, hidden_dim)
            self.W = nn.Linear(hidden_dim, hidden_dim, bias=False)

        elif method == "concat":
            # 拼接后投影：(2*hidden_dim) → hidden_dim
            self.W = nn.Linear(hidden_dim * 2, hidden_dim, bias=False)
            # 压缩到标量
            self.v = nn.Linear(hidden_dim, 1, bias=False)

    def forward(self, decoder_hidden, encoder_outputs):
        """
        decoder_hidden:  (batch, hidden_dim)     解码器当前隐状态
        encoder_outputs: (batch, src_len, hidden_dim) 编码器所有隐状态
        """
        if self.method == "dot":
            # 最简单：直接点积
            # decoder_hidden: (batch, hidden_dim) → (batch, hidden_dim, 1)
            # encoder_outputs: (batch, src_len, hidden_dim)
            # bmm 结果: (batch, src_len, 1) → (batch, src_len)
            scores = torch.bmm(
                encoder_outputs, decoder_hidden.unsqueeze(2)
            ).squeeze(2)

        elif self.method == "general":
            # 先对 encoder 输出做线性变换，再点积
            # self.W(encoder_outputs): (batch, src_len, hidden_dim)
            # 再和 decoder_hidden 点积
            scores = torch.bmm(
                self.W(encoder_outputs), decoder_hidden.unsqueeze(2)
            ).squeeze(2)

        elif self.method == "concat":
            # 把 decoder_hidden 扩展到和 encoder_outputs 同样长
            src_len = encoder_outputs.size(1)
            # (batch, hidden_dim) → (batch, src_len, hidden_dim)
            dec_expanded = decoder_hidden.unsqueeze(1).expand(-1, src_len, -1)

            # 拼接: (batch, src_len, 2*hidden_dim)
            concatenated = torch.cat([dec_expanded, encoder_outputs], dim=2)

            # 过 MLP: tanh + 线性
            scores = self.v(torch.tanh(self.W(concatenated))).squeeze(2)

        # softmax 归一化
        attn_weights = F.softmax(scores, dim=-1)

        # 加权求和
        context = torch.bmm(attn_weights.unsqueeze(1), encoder_outputs).squeeze(1)

        return context, attn_weights
```

`torch.bmm` 是批量矩阵乘法。这里做的事情就是：对于 batch 中的每个样本，用权重向量（1×src_len）乘以编码器输出矩阵（src_len×encoder_dim），得到加权和（1×encoder_dim）

### 2 维度问题

这里有个重要的细节。当维度$ d $很大时，点积的数值也会很大。假设$ s $和 $h $的每个分量都是均值 0、方差 1 的随机变量，那么：
$$
s^{\top} h = \sum_{i=1}^{d} s_i h_i
$$
每一项$s_i h_i$ 的方差是 1（两个独立的均值 0 方差 1 的随机变量之积，方差 = 1×1 = 1），$d $项相加，方差变成$d$。当 $d=512$ 时，点积的标准差大约是$\sqrt{512} \approx 22.6$，这意味着分数可能在 -50 到 +50 之间波动。这么大的数值送进 softmax，会导致输出几乎是 one-hot 的（一个接近 1，其余接近 0），梯度接近于零。这就是后来 Transformer 中引入**缩放因子** $\sqrt{d_k}$的原因：
$$
e_{t,i} = \frac{s_t^{\top} h_i}{\sqrt{d}}
$$
除以 $\sqrt{d}$后方差回到 1，softmax 的输入在合理范围内，梯度能正常流动。**加性注意力为什么没有这个问题？** 因为 tanh 的输出范围被限制在 (−1,1) 之间，天然地控制了数值范围，不需要额外缩放

# 03 自注意力机制

**自注意力**的核心变化：Query、Key、Value **全部来自同一个序列**，用于建模序列内部元素之间的依赖关系

## 03.1 为什么需要 Q、K、V 三个

既然都来自同一个序列，为什么不直接用原始向量做点积？为什么要搞三个投影矩阵？

### 1 不做投影会怎样？

假设输入序列是$x_1, x_2, \dots, x_n$，直接用原始向量做点积：

$$
e_{ij} = x_i^\top x_j
$$
这里有一个致命问题：**点积是对称的**，即$x_i^\top x_j = x_j^\top x_i$。这意味着"A 关注 B 的程度"和"B 关注 A 的程度"永远相同。但在语言中，关系通常是不对称的。"她"强烈地关注"小红"（因为要知道"她"指代谁），但"小红"不一定需要关注"她"

### 2 K、V 的角色分工

通过三个不同的投影矩阵，每个词同时拥有三种身份：

- **Query（$q_i=x_iW^Q$）**："我在找什么信息？"—— 这是这个词作为**提问者**的表示
- **Key（$k_i = x_i W^K$）**："我能提供什么信息？"—— 这是这个词作为**被查找者**的标签
- **Value（$v_i = x_i W^V$）**："如果你选中了我，我实际传递什么内容？"—— 这是这个词作为**信息提供者**的内容

一个类比：图书馆查资料

-  Query 是脑子里的问题
- 每本书的 Key 是封面和标题（用来判断相不相关）
- 每本书的 Value 是书的正文内容（实际要读的东西）

Key 和 Value 虽然来自同一本书，但扮演不同角色。标题和正文不是一回事。而且因为$W^Q \neq W^K$，==同一个 $x_i$ 被投影到了不同的空间，**打破了对称性**，模型可以学到不对称的关注模式==

## 03.2 数学推导

### 第一步：线性投影

给定输入矩阵 $X \in \mathbb{R}^{n \times d_{\text{model}}}$（$n$ 个 token，每个$d_{\text{model}}$维，就是词向量的元素个数），通过三个权重矩阵得到 Q、K、V：
$$
Q = X W^Q \in \mathbb{R}^{n \times d_k}
$$

$$
K = X W^K \in \mathbb{R}^{n \times d_k}
$$

$$
V = X W^V \in \mathbb{R}^{n \times d_v}
$$

其中$W^Q \in \mathbb{R}^{d_{\text{model}} \times d_k}$，$W^K \in \mathbb{R}^{d_{\text{model}} \times d_k}$，$W^V \in \mathbb{R}^{d_{\text{model}} \times d_v}$。注意 Q 和 K 的最后一个维度必须相同（都是$d_k$），因为它们要做点积。V 的维度$d_v$可以不同（但实践中通常也取$d_k$）

### 第二步：计算注意力分数矩阵

每个 Query 和每个 Key 做点积：
$$
Q K^\top \in \mathbb{R}^{n \times n}
$$
这个矩阵的第$(i,j)$个元素就是$q_i^\top k_j$，表示位置$ i $对位置 $j $的"原始关注程度"

==以文本翻译为例，这里矩阵Q的每一行都表示一个词，列数就是单个词向量的元素个数，这个值一般是可以自定的。然后K转置后，每一列表示一个词，行则是单个词向量的元素，所以Q和K转置的矩阵乘法，第一行表示词1对词1，词1对词2，词1对词3……的一个关注度，第二行就是词2对词1，词2对词2，词2对词3……的关注度，后面的行……==

假设序列长度 n=3：
$$
QK^\top=\begin{pmatrix}q_1^\top k_1&q_1^\top k_2&q_1^\top k_3\\q_2^\top k_1&q_2^\top k_2&q_2^\top k_3\\q_3^\top k_1&q_3^\top k_2&q_3^\top k_3\end{pmatrix}
$$
第$ i $行就是位置$ i $对所有位置的关注分数

### 第三步：缩放

$$
\frac{QK^\top}{\sqrt{d_k}}
$$

为什么要除以 $\sqrt{d_k}$？这个上节在对比加性和乘性注意力时讲过

### 第四步：Softmax 归一化

$$
A = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right) \in \mathbb{R}^{n \times n}
$$

softmax 是**逐行**做的。第$ i $行经过 softmax 后，所有元素在 0 到 1 之间，且求和为 1。这一行就是位置$ i $对所有位置的**注意力权重分布**==也就是每一行都表示其他词对第i个词的重要程度==

### 第五步：加权求和

$$
\text{Output} = A \, V \in \mathbb{R}^{n \times d_v}
$$



矩阵$ A $乘以$ V$，效果是：对于输出的第$ i$ 行，它是所有位置的 Value 向量按照第$ i $行权重加权求和的结果：
$$
\text{output}i = \sum{j=1}^{n} \alpha_{ij} \, v_j
$$
==因为矩阵V的每一行都是表示一个词向量，最终输出的结果，第一行表示词1在不同特征上融合了上下文其他词重要程度的表示，第二行表示词2在不同特征上融合了上下文其他词重要程度的表示……因为矩阵A就是一个注意力矩阵，每一行都表示一个词对其他词的重要程度，再乘上矩阵V，就是按照重要程度的不同词的内容的加权求和==

### 完整公式

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right) V
$$
一行公式，五步操作，这就是整个自注意力的全部

## 03.3 代码

```python
class SelfAttention(nn.Module):
    """单头自注意力层"""

    def __init__(self, d_model: int, d_k: int, d_v: int):
        """
        d_model: 输入向量的维度（比如 512）
        d_k:     Q 和 K 投影后的维度（比如 64）
        d_v:     V 投影后的维度（比如 64）
        """
        super().__init__()
        # 三个投影矩阵：把 d_model 维的输入分别投影到不同的空间
        self.W_Q = nn.Linear(d_model, d_k, bias=False)
        self.W_K = nn.Linear(d_model, d_k, bias=False)
        self.W_V = nn.Linear(d_model, d_v, bias=False)

    def forward(
        self,
        x: torch.Tensor,                  # (batch, seq_len, d_model)
        mask: torch.Tensor | None = None,
    ) -> tuple[torch.Tensor, torch.Tensor]:
        """
        x 就是输入序列。注意：Q、K、V 全部从同一个 x 计算得来
        这就是"自"注意力的含义
        """
        # 投影：同一个 x，通过不同的矩阵，得到不同角色的表示
        Q = self.W_Q(x)  # (batch, seq_len, d_k)  —— "我在找什么"
        K = self.W_K(x)  # (batch, seq_len, d_k)  —— "我能提供什么标签"
        V = self.W_V(x)  # (batch, seq_len, d_v)  —— "我实际的内容"

        # 调用核心函数
        output, attn_weights = scaled_dot_product_attention(Q, K, V, mask)

        return output, attn_weights
```

# 04 多头注意力机制

## 04.1 拼接方式

单头注意力只能捕获一种"关注模式"。多头注意力让模型同时从**不同的表示子空间**中学习信息：

多头注意力的**概念定义**是这样的：有$ h $个头，每个头$ i $拥有**自己独立的**三组投影矩阵：
$$
W_i^Q \in \mathbb{R}^{d{\text{model}} \times d_k}, \quad W_i^K \in \mathbb{R}^{d{\text{model}} \times d_k}, \quad W_i^V \in \mathbb{R}^{d_{\text{model}} \times d_v}
$$


每个头的计算过程：
$$
Q_i = X W_i^Q \in \mathbb{R}^{n \times d_k}
$$

$$
K_i = X W_i^K \in \mathbb{R}^{n \times d_k}
$$

$$
V_i = X W_i^V \in \mathbb{R}^{n \times d_v}
$$

注意这里**每个头的投影输入都是完整的$ X$，也就是完整的 $d_{\text{model}}$维**。$W_i^Q$ 的左边维度是 $d_{\text{model}}$，不是 $d_k$。所以每个头投影时，能看到输入的全部$d_{\text{model}}$个维度。投影的**输出**维度是 $d_k$（论文中取$d_k = d_{\text{model}} / h$），这是一个被压缩过的表示。但这个压缩是通过**可学习的线性变换**完成的，输入的全部信息都参与了



然后把$ h $个头的输出沿最后一维拼接，再过一个输出投影：

$$
\text{MultiHead}(X) = \text{Concat}(\text{head}_1, \dots, \text{head}_h) \,W^O
$$
其中 $\text{Concat}$的结果是$\mathbb{R}^{n \times (h \cdot d_v)}$，$W^O \in \mathbb{R}^{(h \cdot d_v) \times d_{\text{model}}}$。当$d_k = d_v = d_{\text{model}} / h$ 时，$h \cdot d_v = d_{\text{model}}$，输出维度回到$d_{\text{model}}$

==这里很好理解，单头注意力最后输出的矩阵，每一行都表示一个词在不同特征上融合了上下文其他词的影响下的表示，所以多头注意力最后拼接应该是横着拼接，也就相当于是在扩充每一个词的列数，丰富特征数==

## 04.2 代码

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math


class MultiHeadAttention(nn.Module):
    def __init__(self, d_model: int, n_heads: int):
        super().__init__()
        assert d_model % n_heads == 0

        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads

        # 这三个矩阵的输入维度都是 d_model
        # 它们在概念上等价于 h 个独立的 (d_model, d_k) 矩阵横向拼接
        # 但写成一个 (d_model, d_model) 矩阵可以用一次矩阵乘法搞定
        self.W_Q = nn.Linear(d_model, d_model, bias=False)
        self.W_K = nn.Linear(d_model, d_model, bias=False)
        self.W_V = nn.Linear(d_model, d_model, bias=False)

        # 输出投影：把拼接后的多头输出再混合一次
        self.W_O = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x: torch.Tensor, mask: torch.Tensor | None = None):
        """
        x: (batch, seq_len, d_model) —— 原始输入，每个 token 是完整的 d_model 维
        """
        B, T, _ = x.shape

        # ====== 投影：输入是完整的 d_model 维，输出也是 d_model 维 ======
        # 这一步每个输出维度都依赖于输入的全部 d_model 个维度
        Q = self.W_Q(x)  # (B, T, d_model)
        K = self.W_K(x)  # (B, T, d_model)
        V = self.W_V(x)  # (B, T, d_model)

        # ====== reshape：在投影后的空间中，把 d_model 维解释为 (h, d_k) ======
        # 这一步不做任何计算，只是重新解释维度布局
        # 被拆分的是投影后的结果，不是原始输入
        Q = Q.view(B, T, self.n_heads, self.d_k).transpose(1, 2)  # (B, h, T, d_k)
        K = K.view(B, T, self.n_heads, self.d_k).transpose(1, 2)  # (B, h, T, d_k)
        V = V.view(B, T, self.n_heads, self.d_k).transpose(1, 2)  # (B, h, T, d_k)

        # ====== 缩放点积注意力：h 个头并行计算 ======
        # matmul 把 (B, h) 视为批次维度，只对最后两维做矩阵乘法
        # (B, h, T, d_k) @ (B, h, d_k, T) = (B, h, T, T)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)

        if mask is not None:
            scores = scores.masked_fill(mask == 0, float("-inf"))

        attn_weights = F.softmax(scores, dim=-1)  # (B, h, T, T)

        # (B, h, T, T) @ (B, h, T, d_k) = (B, h, T, d_k)
        attn_output = torch.matmul(attn_weights, V)

        # ====== 拼接：把 h 个头的 d_k 维输出拼回 d_model 维 ======
        # transpose + view 等价于 concat，但零开销
        attn_output = attn_output.transpose(1, 2).contiguous().view(B, T, self.d_model)
        # (B, h, T, d_k) → (B, T, h, d_k) → (B, T, d_model)

        # ====== 输出投影：让不同头的信息互相融合 ======
        output = self.W_O(attn_output)  # (B, T, d_model)

        return output
```

# 05 多查询注意力机制（Multi-Query Attention，MQA）

## 05.1 多头注意力（Multi-Head Attention, MHA）的痛点

**标准多头注意力（MHA）的直觉**
 想象一个有 8 个调查员（Head）的团队在破案。在 MHA 中，每个调查员都有自己独立的问题清单（Query），以及自己独立的一套完整档案（Keys）和线索库（Values）。这使得每个人都能从完全不同的角度查案，效果很好，但代价是他们需要极大的办公空间（内存）来存放各自的档案



**多查询注意力（MQA）的直觉**
 在 MQA 中，依然保留 8 个调查员，他们依然有各自独立的问题清单（Queries）。但是，**所有人共享同一套档案（Key）和线索库（Value）**。这大幅减少了占用的办公空间。虽然视角少了一点，但事实证明，只要问题（Query）不同，共享底层资料库对最终破案（生成文本）的质量影响微乎其微，但效率却有了质的飞跃



在自回归生成（Autoregressive Decoding）阶段，模型每次只生成一个 Token。为了避免重复计算之前的 Token，会把之前所有 Token 的 Key 和 Value 缓存下来，这就是 **KV Cache**

- **MHA 的痛点**：对于每一个生成的 Token，模型需要从显存中读取庞大的 KV Cache。随着上下文变长、Batch Size 增大，读取 KV Cache 所需的显存带宽远大于计算（矩阵乘法）所需的算力。此时，GPU 的算力处于闲置状态，全在等内存传输数据
- **MQA 的解法**：由于所有注意力头共享同一个 Key 和 Value，**KV Cache 的大小直接缩小了 H 倍（H 为注意力头的数量）**。读取的数据量锐减，打破了内存带宽瓶颈，使得生成速度大幅提升

| 特性                  | Multi-Head Attention (MHA) | Multi-Query Attention (MQA)   |
| --------------------- | -------------------------- | ----------------------------- |
| **Query 数量**        | $H$（每个头一个）          | $H$（每个头一个）             |
| **Key/Value 数量**    | $H$（每个头一个）          | **$1$（所有头共享）**         |
| **KV Cache 显存占用** | 大（与头数成正比）         | **极小（仅为 MHA 的 $1/H$）** |
| **推理生成速度**      | 较慢（受限于显存带宽）     | **极快**                      |

## 05.2 为什么 Q 的多样性比 K 和 V 的多样性更重要

自注意力的核心公式：
$$
\alpha_{ij} = \text{softmax}\left(\frac{q_i^\top k_j}{\sqrt{d_k}}\right)
$$

$$
\text{output}i = \sum_j \alpha_{ij} \, v_j
$$

注意力权重$\alpha_{ij}$是由$q_i$和$k_j$的点积决定的。在多头注意力中，不同头的表达能力体现在：**不同头有不同的$\alpha$分布**，从而从$ V $中提取不同的信息



现在想一下，$\alpha$分布的多样性来自哪里？来自$q_i^\top k_j$这个点积。点积涉及$q$和 $k $两方：

如果$q$不同、$k$相同：$q_1^\top k$和$q_2^\top k$一般不同 → $\alpha$分布不同 ；如果$q$相同、$k$不同：$q^\top k_1$和 $q^\top k_2$ 一般不同 → $\alpha$分布不同 



两种方式都能产生多样性。MHA 是两边都不同，MQA 是只让 $q $这边不同。多样性确实会减少，但不是降为零。再看$ V $的角色。不同头的最终输出是 $\alpha_{ij} v_j$。如果$ V $是共享的，但 $\alphaα $不同，那加权求和的结果依然不同。就像同一批食材（$V$），不同的配比（$\alpha$）做出来的菜味道不同。**总结**：$Q$ 的多样性直接决定了注意力权重的多样性，而注意力权重的多样性是多头机制表达能力的主要来源。$K$ 和 $V $的多样性是"锦上添花"，去掉之后损失不大

## 05.3 代码

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math


class MultiQueryAttention(nn.Module):
    """
    多查询注意力（MQA）
    和 MHA 唯一的区别：K 和 V 只有 1 个头，Q 保留 h 个头
    """

    def __init__(self, d_model: int, n_heads: int):
        super().__init__()
        assert d_model % n_heads == 0, "d_model 必须被 n_heads 整除"

        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads

        # === Q 投影 ===
        # 和 MHA 完全一样：输出 d_model 维，包含 h 个头各 d_k 维
        # 概念上等价于 h 个独立的 (d_model, d_k) 矩阵合并
        self.W_Q = nn.Linear(d_model, d_model, bias=False)

        # === K 投影 ===
        # 关键区别：输出只有 d_k 维，不是 d_model 维
        # 因为只有 1 个头，不需要 h 份
        # 但输入仍然是完整的 d_model 维——投影从完整信息中提取
        self.W_K = nn.Linear(d_model, self.d_k, bias=False)

        # === V 投影 ===
        # 同 K，输出只有 d_k 维
        self.W_V = nn.Linear(d_model, self.d_k, bias=False)

        # === 输出投影 ===
        # 和 MHA 完全一样
        self.W_O = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x: torch.Tensor, mask: torch.Tensor | None = None):
        """
        x: (B, T, d_model) —— 一批序列，每个 token 是完整的 d_model 维
        """
        B, T, _ = x.shape

        # ========== Q 的处理：和 MHA 完全一样 ==========

        # 投影：(B, T, d_model) → (B, T, d_model)
        Q = self.W_Q(x)

        # reshape 拆成多头：(B, T, d_model) → (B, T, h, d_k) → (B, h, T, d_k)
        Q = Q.view(B, T, self.n_heads, self.d_k).transpose(1, 2)

        # 现在 Q 有 h 个头，每个头的 Query 是 d_k 维
        # Q 的形状: (B, h, T, d_k)

        # ========== K 和 V 的处理：这里是 MQA 的核心区别 ==========

        # 投影：注意输出维度是 d_k，不是 d_model！
        # (B, T, d_model) → (B, T, d_k)
        K = self.W_K(x)
        V = self.W_V(x)

        # 加一个 head 维度（大小为 1），为后面的广播做准备
        # (B, T, d_k) → (B, 1, T, d_k)
        K = K.unsqueeze(1)
        V = V.unsqueeze(1)

        # 为什么要 unsqueeze？
        # 因为 Q 是 (B, h, T, d_k)，有 4 个维度
        # K 必须也有 4 个维度才能做矩阵乘法
        # 加入大小为 1 的 head 维后，PyTorch 的广播机制
        # 会自动把 1 扩展为 h，等价于所有头看到同一份 K

        # ========== 计算注意力分数 ==========

        # Q @ K^T:
        # Q:   (B, h, T, d_k)
        # K^T: (B, 1, d_k, T)  ← 1 会被广播为 h
        # 结果: (B, h, T, T)   ← 每个头各有一个 T×T 的分数矩阵
        #
        # 虽然 K 只有 1 份，但因为 Q 有 h 份不同的，
        # 所以 Q_头i @ K^T 对于不同的 i 结果不同
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)

        # ========== 可选的 mask ==========
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float("-inf"))

        # ========== softmax：逐行归一化 ==========
        # (B, h, T, T) 的每一行变成概率分布
        attn_weights = F.softmax(scores, dim=-1)

        # ========== 加权求和 ==========
        # attn_weights: (B, h, T, T)
        # V:            (B, 1, T, d_k)  ← 1 又被广播为 h
        # 结果:          (B, h, T, d_k)
        #
        # 每个头用自己不同的权重对同一个 V 做加权求和
        # 权重不同 → 加权求和的结果不同 → 各头输出不同
        attn_output = torch.matmul(attn_weights, V)

        # ========== 拼接 + 输出投影：和 MHA 完全一样 ==========
        # (B, h, T, d_k) → (B, T, h, d_k) → (B, T, d_model)
        attn_output = attn_output.transpose(1, 2).contiguous().view(B, T, self.d_model)

        # 输出投影：让不同头的信息互相融合
        output = self.W_O(attn_output)  # (B, T, d_model)

        return output
```

# 06 分组注意力查询机制（Grouped-Query Attention，GQA）

## 06.1 MHA与MQA之间的平衡点

GQA 是 MHA 与 MQA 之间的折中方案：将 \(h\) 个 Query 头分为 \(g\) 组，每组共享一组 Key 和 Value。

- \(g = h\)：等价于标准 MHA（每个 Q 头有独立的 KV）
- \(g = 1\)：等价于 MQA（所有 Q 头共享同一组 KV）
- \(1 < g < h\)：在质量与效率之间取得平衡

| 特性                   | Multi-Head (MHA) | Grouped-Query (GQA)               | Multi-Query (MQA) |
| ---------------------- | ---------------- | --------------------------------- | ----------------- |
| **Query 头数 ($H$)**   | $H$              | $H$                               | $H$               |
| **KV 头数 ($G$)**      | $H$              | **$G$ (通常是 4 或 8)**           | $1$               |
| **KV Cache 大小**      | 巨大             | **较小（约为 MHA 的 $1/(H/G)$）** | 极小              |
| **生成质量 (Quality)** | 最佳             | **接近最佳**                      | 略有下降          |
| **推理速度 (Speed)**   | 慢               | **快**                            | 最快              |

## 06.2 为什么组内共享是合理的？

**注意力权重的多样性主要由 Q 决定，K 共享不会消除这种多样性**

每个头的注意力分数是$S_i = Q_i K^\top / \sqrt{d_k}$把$Q_i = XW_i^Q$，$K = XW_k^K$代入，得到：
$$
S_i=X(W_i^QW_k^{K^\top})X^\top/\sqrt{d_k}
$$
记$M_i = W_i^Q {W_k^K}^\top$，这个交互矩阵$M_i$完全决定了头$ i $的注意力模式。虽然 $W_k^K$ 是组内共享的（固定的），但$ W_i^Q$ 每个头各不相同，所以 $M_i$每个头也不同。只要 $W_k^K$不退化为零矩阵，不同的$W_i^Q$就必然产生不同的$M_i$，进而产生不同的注意力权重$ A_i$。**Q 的自由度足以保证各头的注意力模式是多样的**



再看输出端。同一组内各头共享 $V$，但因为$ A_i$不同，$A_i V$就是对 $V $的不同凸组合——只要 $V$ 的各行向量不完全相同（实际中不可能相同），不同的凸组合就会产生不同的结果。此外，实证研究也支持这个结论：在训好的 MHA 模型中，很多头的 K/V 表示高度相似（余弦相似度很高），大量头可以被剪枝而不显著影响性能。这说明独立的$h$份 K/V 中本身就存在大量冗余，合并为$ g $份造成的信息损失是有限的



**再问一个为什么**：模型真的能学会"把关注点相近的头放在一起"吗？

实际上，GQA 并不需要显式地让模型"分组"。**分组是通过参数结构强制实现的**：同一组的 Q 头，在代码中就是对应同一个 KV 头，这是架构设计决定的。模型训练时，会自动适应这个约束——同一组的 Q 头会学会从共享的 K/V 中提取各自需要的信息，而 K/V 的投影矩阵也会学会编码这一组 Q 头们共同需要的信息。这就像把 4 位记者分到一间办公室，一开始他们可能关注点不完全一样，但时间久了，他们会自然地协调分工——资料库管理员（K/V 的投影矩阵）会学会准备这 4 位记者都能用的资料，而 4 位记者也会学会从这套资料中各取所需

## 06.3 组间独立很重要

如果只是组内共享，那和 MQA（全部共享）有什么本质区别？区别在于**组间的 K 和 V 是完全独立的**。组 1 的 K/V 和组 2 的 K/V 来自不同的投影矩阵，对同一个输入会产生不同的表示



**共享的 $W_k^K$对交互矩阵施加了子空间约束，单一的 K/V 组会形成表达能力的硬性瓶颈。**当$W_k^K$固定时，所有$M_i = W_i^Q {W_k^K}^\top$的行空间都被限制在$W_k^K$的列空间内——这是一个$d_k$维的子空间。不管$W_i^Q$怎么变化，注意力计算"感知输入"的角度始终被锁定在这 $d_k$个方向上。同样，共享的$V^{(k)}$使得所有头的输出都落在$V^{(k)}$行空间这同一个$d_k$维子空间中。拼接后的向量虽名义上是$D$维的，但各个$d_k$维的块都来自同一个子空间，有效信息量被严重压缩



$g = 1$（MQA）时，整个注意力层只有一个$d_k$维的"观察窗口"和一个$d_k$维的"输出素材池"。$g > 1$时，每多一个独立组，就多一个 $d_k$维的方向，覆盖更大的空间。$g$个组联合可覆盖 $g \cdot d_k$维，直到 $g = h$时覆盖完整的 $D$维空间。但由于实际数据的分布不会均匀地占满整个$D$维空间（它有主成分方向），当 $g$个组已经覆盖了数据的主要方向后，再增加组数的信息增益就很小了。**这就是为什么$g = 8$左右质量就趋于饱和，而$g$继续增大只是在增加存储开销**



回到编辑部比喻：政治组的资料库里详细记录了各国领导人的发言和政策立场，经济组的资料库里详细记录了市场数据和企业财报。这两个资料库的内容完全不同，是因为它们的"筛选标准"（$W^K$投影矩阵）不同。在 MQA 中，所有人共用一个资料库，这个资料库必须既包含政治信息又包含经济信息，导致每个方面都不够深入。在 GQA 中，每组有自己的资料库，可以在自己负责的领域做得更深入



**这里要问一个关键的为什么**：$g = 8$就够了吗？为什么不需要$g = 32$？

这涉及一个信息论的直觉：**注意力头之间存在大量的冗余。**研究者做过这样的实验：在训好的 MHA 模型中，观察 32 个头的注意力权重矩阵。他们发现很多头的权重模式非常相似——比如头 3 和头 7 的注意力分布可能高度相关，差异主要在细节上。这意味着给每个头配独立的 K/V 是浪费的，因为好几个头的 K/V 实际上学到了差不多的东西。定量来看，Google 的 GQA 论文做了系统的实验：从$g=1$（MQA）到 $g=h$（MHA），逐步增加$g$。结果发现：

- $g $从 1 增到 8 时，模型质量显著提升
- $g$ 从 8 增到 hhh 时，质量提升非常微小
- 但 KV Cache 是线性增长的

这意味着$g=8$是一个"拐点"——之后的质量提升不值得额外的存储开销

## 06.4 代码

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class GroupedQueryAttention(nn.Module):
    def __init__(self, d_model, num_heads, num_kv_heads):
        super().__init__()
        self.num_heads = num_heads
        self.num_kv_heads = num_kv_heads
        
        # 每组共享一个 KV 头的 Query 头数量 (P = H / G)
        assert num_heads % num_kv_heads == 0, "Query头数必须能被KV头数整除"
        self.num_queries_per_kv = num_heads // num_kv_heads 
        self.head_dim = d_model // num_heads
        
        # Query 拥有所有 H 个头的维度
        self.W_q = nn.Linear(d_model, d_model)
        
        # Key 和 Value 只有 G 个头的维度 (num_kv_heads * head_dim)
        self.W_k = nn.Linear(d_model, num_kv_heads * self.head_dim)
        self.W_v = nn.Linear(d_model, num_kv_heads * self.head_dim)
        
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, x):
        batch_size, seq_len, _ = x.shape
        
        # 1. 计算 Q, K, V
        Q = self.W_q(x) # (B, S, num_heads * head_dim)
        K = self.W_k(x) # (B, S, num_kv_heads * head_dim)
        V = self.W_v(x) # (B, S, num_kv_heads * head_dim)
        
        # 2. Reshape 为多头格式
        # Q: (B, num_heads, S, head_dim)
        Q = Q.view(batch_size, seq_len, self.num_heads, self.head_dim).transpose(1, 2)
        
        # K, V: (B, num_kv_heads, S, head_dim)
        K = K.view(batch_size, seq_len, self.num_kv_heads, self.head_dim).transpose(1, 2)
        V = V.view(batch_size, seq_len, self.num_kv_heads, self.head_dim).transpose(1, 2)
        
        # 3. 核心步骤：扩展 K 和 V 以匹配 Query 头的数量
        # 使用 repeat_interleave 在 num_kv_heads 这个维度上复制
        # 例如 K 从 (B, 2, S, head_dim) 变为 (B, 8, S, head_dim)，假设 num_queries_per_kv = 4
        K = K.repeat_interleave(self.num_queries_per_kv, dim=1)
        V = V.repeat_interleave(self.num_queries_per_kv, dim=1)
        
        # 4. 计算 Scaled Dot-Product Attention (现在 Q, K, V 的头数完全一致了)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.head_dim)
        attn_weights = F.softmax(scores, dim=-1)
        
        context = torch.matmul(attn_weights, V) # (B, num_heads, S, head_dim)
        
        # 5. 拼接所有的头并投影
        context = context.transpose(1, 2).contiguous().view(batch_size, seq_len, self.num_heads * self.head_dim)
        
        output = self.W_o(context)
        return output

# 测试维度
batch_size, seq_len, d_model = 2, 10, 512
num_heads = 8      # H = 8
num_kv_heads = 2   # G = 2 (每 4 个 Query 头共享 1 个 KV 头)

gqa = GroupedQueryAttention(d_model, num_heads, num_kv_heads)
dummy_input = torch.randn(batch_size, seq_len, d_model)
output = gqa(dummy_input)

print("Q heads:", num_heads)
print("KV heads:", num_kv_heads)
print("Output shape:", output.shape) # 期望输出: [2, 10, 512]
```
