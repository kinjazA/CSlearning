# 01 张量的形状操作

## 01.1 reshape函数

> * reshape函数可以在保证张量数据不变的前提下改变数据的维度，将其转换成指定形状。

```python
import torch

# 假设有一盒巧克力，共 12 块 (4行3列)
x = torch.arange(12).reshape(4, 3)

# 写法 A: 变成 2 行 6 列
a = x.reshape(2, 6)

# 写法 B: 偷懒写法 (-1)
# 只确定要是 2 行，列数自己算
b = x.reshape(2, -1) 
```

## 01.2 view函数

> * 要求张量存储的内存是连续的，reshape则不要求。
> * 在调用 `view` 之前，必须先调用 **`.contiguous()`**。这个函数会强制把数据在内存中重新复制排序，使其变得连续。

```python
x = torch.arange(12)
a = x.view(3, 4)

# 如果内存不连续，必须先整理内存，再view；或者无脑reshape
# x.permute(...) 之后内存不连续
x.contiguous().view(3, 4) # 必须要加 contiguous()
```

## 01.3 unsqueeze函数

> * 用来在指定的维度增加一个size为1的维度
> * 其实就是在指定的维度上增加一层中括号，相当于是打包

```python
y = x.unsqueeze(0)   # 在最前面加维
y = x.unsqueeze(1)
y = x.unsqueeze(-1)  # 在最后加维
```

## 01.4 squeeze函数

> * 用于去掉指定维度且size为1的维度
> * 就是在指定的维度（该维度size为1）拆包

```python
y = x.squeeze()     # 去掉所有 size=1 的维
y = x.squeeze(0)    # 只去掉第0维（若它为1）
```

## 01.5 transpose函数

> * 一次只能交换**两个**维度

```python
import torch

# 假设我们在做 NLP 任务
# 形状: (Batch=2, 单词数=3, 词向量维度=4)
x = torch.randn(2, 3, 4)

# --- 场景：我想交换“单词数”和“词向量维度” ---
# 原索引: (0, 1, 2)
# 交换 1 和 2
y = x.transpose(1, 2)

print(f"原形状: {x.shape}") # torch.Size([2, 3, 4])
print(f"交换后: {y.shape}") # torch.Size([2, 4, 3]) -> 第0维没变
```

## 01.6 permute函数

> * 一次可以重新排列**所有**维度，这是处理高维张量（3维及以上）时的**首选**。
> * 需要传入一个维度的**索引列表**，告诉 PyTorch：“新的第0维是原来的第几号？新的第1维是原来的第几号？...”

```python
# 假设是图像数据：Batch=10, Height=256, Width=256, Channel=3
# 现在的索引顺序是: 0, 1, 2, 3
img = torch.randn(10, 256, 256, 3)

# --- 需求：变成 PyTorch 格式 (Batch, Channel, Height, Width) ---
# 也就是要把原来的 3号(C) 放到 1号位置
# 原来的 0(B) -> 还是 0
# 原来的 3(C) -> 变成 1
# 原来的 1(H) -> 变成 2
# 原来的 2(W) -> 变成 3

# 代码直接写出目标索引顺序
img_new = img.permute(0, 3, 1, 2)

print(f"原形状: {img.shape}")     # (10, 256, 256, 3)
print(f"Permute后: {img_new.shape}") # (10, 3, 256, 256)
```

在 PyTorch 代码中，**只要使用了 `permute` 或 `transpose`，且后面紧接着要改变形状**，请务必记住：

> **先 `contiguous`，再 `view`。**

# 02 张量的拼接操作

> * 核心就是两个函数：**`torch.cat`（拼接）** 和 **`torch.stack`（堆叠）**
> * `cat` 是接长，`stack` 是打包

## 02.1 cat函数

> * 除了指定的那个拼接维度 (`dim`) 之外，其他所有维度的大小必须完全一样
> * dim=0，就是在行做拼接，也就是列不变，相当于样本增多
> * dim=1，就是在列做拼接，也就是行不变，相当于特征增多

```python
import torch

# 创建两个形状为 (2, 3) 的张量
A = torch.tensor([[1, 1, 1], 
                  [2, 2, 2]])  # (2, 3)

B = torch.tensor([[3, 3, 3], 
                  [4, 4, 4]])  # (2, 3)

# --- 场景 1: 按行拼接 (dim=0, 竖着拼) ---
# 理解：行数变多，列数不变
C = torch.cat([A, B], dim=0)
print(f"按行拼: {C.shape}") 
# 输出: (4, 3) -> 2行+2行=4行

# --- 场景 2: 按列拼接 (dim=1, 横着拼) ---
# 理解：行数不变，列数变多
D = torch.cat([A, B], dim=1)
print(f"按列拼: {D.shape}")
# 输出: (2, 6) -> 3列+3列=6列
```

## 02.2 stack函数

> * 新增一个维度再拼（“堆叠”）
> * 所有输入张量 **shape 必须完全一致**，输出会**多出一个维度**，该维大小=输入个数

```python
import torch

# 还是刚才那两个 (2, 3) 的张量
A = torch.zeros(2, 3)
B = torch.zeros(2, 3)

# --- 场景: 堆叠 (创建新维度) ---

# dim=0: 在最前面加一个维度 (理解为：第几个张量)
S1 = torch.stack([A, B], dim=0)
print(f"Stack dim=0: {S1.shape}")
# 输出: (2, 2, 3) 
# 解释: 新出来的第一个 '2' 代表原来有两个张量

# dim=1: 在中间插一个维度
S2 = torch.stack([A, B], dim=1)
print(f"Stack dim=1: {S2.shape}")
# 输出: (2, 2, 3) 
# 解释: 原来的 (2,3) 变成了 (2, 1, 3)，然后在 dim=1 堆叠变成 (2, 2, 3)
```
