# 第12章 · 信号去噪方法对比

---

## 0. 前11章的回顾：你已经会去噪了

其实你从第2章开始就在做去噪：
- 第2章：卷积平滑（移动平均）= 最简单的去噪
- 第3-6章：低通/带通滤波器 = 精确控制去噪的频率边界
- 第6章：`filtfilt` = 离线零相位去噪

但不同的噪声类型需要不同的去噪策略。**一个好的去噪方法，不是"去掉越多越好"，而是"去掉噪声的同时尽量保留信号的有用特征"。**

这一章不是引入全新的概念，而是**把你会的方法和其他常用方法放在一起，做一个系统的横向对比**——帮你在面对实际数据时快速选出最合适的那个。

---

## 0.1 你的数据里有什么类型的噪声？

选择去噪方法的第一步，是判断噪声的"长相"：

| 噪声类型 | 长什么样 | 举例 | 推荐方法 |
|---------|---------|------|---------|
| **白噪声**（高斯） | 每个时刻独立、均匀分布在所有频率上 | 传感器的本底噪声 | 低通滤波、Savitzky-Golay |
| **脉冲噪声**（椒盐） | 偶发的、幅度极大的尖峰 | 数据传输错误、开关动作产生的尖峰 | **中值滤波** |
| **工频干扰** | 固定在某个频率（如50Hz）的周期性干扰 | 电力线干扰串入信号线 | **陷波器**（带阻滤波） |
| **基线漂移** | 极缓慢的上下波动 | 温度变化导致的传感器漂移 | 去趋势（`detrend`）、高通滤波 |

## 场景一："我有一个比较平滑的信号，想去除上面的随机毛刺"

### 1.1 你会怎么做？

你最先想到的可能是低通滤波（第3-6章）。这确实有效，但有一个问题：**低通滤波会让信号中原本"尖锐"的特征——比如一个突然的转折或一个窄峰——也被平滑掉。** 有时你需要保留这些特征。

### 1.2 Savitzky-Golay 滤波器 — 保边缘的平滑

```python
from scipy.signal import savgol_filter

y = savgol_filter(x, window_length=15, polyorder=3)
```

**它怎么做的？** 在一个滑动窗口内，用多项式（比如二次或三次）对数据做局部最小二乘拟合。窗口中心的拟合值就是输出。

**为什么它能保边缘？** 因为多项式可以弯曲。如果一个尖峰落在窗口内，多项式可以"追踪"这个尖峰的形状，而不是像移动平均那样把它抹平。阶数越高的多项式越能追踪弯曲——代价是噪声衰减越少。

**两个关键参数**：

`window_length`：滑动窗口的宽度。必须是奇数。越大 → 越平滑 → 但也越可能抹掉窄的特征。

`polyorder`：拟合多项式的阶数。必须 < `window_length`。越大 → 越能追踪信号中的弯曲和尖峰 → 但保留的噪声也越多。

**选择指南**：
- 你的信号比较平坦、没有急转弯 → `polyorder=2` 或 `3`，`window_length` 根据噪声水平调
- 你的信号有很多尖峰窄谷、你想保留它们的形状 → `polyorder=3` 或 `4`，`window_length` 设小一些
- 典型起点：`window_length=11`, `polyorder=3`

**Savitzky-Golay 的一个额外用途**：它可以同时计算平滑后的**导数**。在 `savgol_filter` 中设 `deriv=1` 就得到一阶导数（变化率），`deriv=2` 得到二阶导数（曲率）。这在光谱分析中非常常用——先平滑再求导，找到峰的确切位置。

## 场景二："我的数据里有偶发的、极大的尖峰干扰"

### 2.1 问题

你有一路传感器数据，正常情况下是平滑的。但偶尔——比如继电器动作的瞬间——会产生一个宽度只有几个采样点的极大尖峰。这种"脉冲噪声"用低通滤波或 Savitzky-Golay 效果都不好——因为尖峰虽然窄但幅度极大，滤波后仍然会在周围留下一圈"涟漪"。

### 2.2 中值滤波 `medfilt` — 专治脉冲噪声

```python
from scipy.signal import medfilt

y = medfilt(x, kernel_size=5)
```

**它怎么做的？** 在一个滑动窗口内，取窗口内所有值的中位数（而不是均值）作为输出。

**为什么中位数能去掉尖峰？** 因为中位数对极端值几乎免疫。一个尖峰在窗口内是最大值或最小值，但中位数只关心"排在中间的那个值"。只要尖峰的宽度小于窗口的一半，它就不会影响中位数——尖峰被直接跳过，不像均值那样被"摊薄"到邻域。

**参数**：`kernel_size` ——窗口大小。必须是奇数。越大 → 能去掉更宽的脉冲 → 但也可能抹掉你希望保留的窄峰。典型值 = 3 到 7。

**什么时候用？**
- 你看到信号中有偶然的巨大尖峰 → 中值滤波
- 如果是连续的高斯噪声（没有明显尖峰）→ 不要用中值滤波，它不如均值滤波效率高
- 可以先中值滤波去尖峰，再用低通滤波去高斯噪声——两者串联使用

## 场景三："我知道噪声的统计特性，想要最优的去噪效果"

### 3.1 Wiener 滤波 — 自适应去噪

```python
from scipy.signal import wiener

y = wiener(x, mysize=11, noise=None)
```

**它怎么做的？** Wiener 滤波器根据信号的局部统计特性（局部均值和局部方差）自动调整滤波强度：
- 在信号变化平缓的区域（方差小）→ 强滤波，大力去噪
- 在信号变化剧烈的区域（方差大，可能是有用特征）→ 弱滤波，尽量保留信号

**如果你不传 `noise` 参数**，Wiener 滤波器会自动从数据中估计噪声功率。这是它的便利之处——不需要你手动估计噪声水平。

**如果你知道噪声功率**（比如你知道传感器的噪声等级），传入 `noise` 参数可以得到真正最优的结果。

**局限**：假设噪声是平稳的高斯白噪声。对于有结构的噪声（如工频干扰），Wiener 滤波不如专用的陷波器。

## 5. 去噪效果如何量化评估？

做完去噪之后，你怎么知道好不好？如果你有 ground truth（干净信号），用以下指标：

| 指标 | 公式 | 含义 |
|------|------|------|
| **MSE** | mean((y_clean - y_true)²) | 平均平方误差——越小越好 |
| **SNR**（信噪比） | 10×log₁₀(信号功率/噪声功率) | 去噪后 SNR 应该比去噪前高 |
| **相关系数** | corr(y_clean, y_true) | 越接近1越好——形状保持了 |
| **峰值偏移** | peak(y_clean)位置 - peak(y_true)位置 | 越小越好——时间精度保持 |

**没有 ground truth 时**：
- 看残差（原始 - 去噪后）：理想的残差应该是纯噪声——没有结构、没有趋势、ACF 应接近白噪声
- 看频谱：去噪后的频谱在"有用频段"内应和原始接近，在"噪声频段"内应大幅降低

## 6. 去噪方法选择速查

| 你的噪声是... | 用... | 因为... |
|-------------|-------|--------|
| 持续的高频随机噪声 | 低通 FIR/IIR | 精确控制截止频率 |
| 随机噪声 + 你想保留尖锐特征 | **Savitzky-Golay** | 多项式拟合追踪弯曲 |
| 偶发的大幅尖峰 | **中值滤波** (`medfilt`) | 中位数免疫极端值 |
| 平稳高斯噪声 + 已知噪声水平 | **Wiener** | 局部自适应、统计最优 |
| 固定在某个频率的干扰 | **陷波器/带阻**（第3-5章） | 精确去掉特定频率 |
| 极缓慢的基线漂移 | **去趋势** (`detrend`) 或高通 | 只去掉极低频 |

## 7. 延伸：去噪思维在预测后处理中的应用

本章讲的去噪方法作用在**原始数据进入模型之前**。但在实际时序预测和异常检测中，同样的思维经常用在**模型输出之后**——对预测结果做后处理。处理对象变了，底层思想完全一致。

### 7.1 预测平滑 = 对预测序列做低通滤波

模型输出的预测值——尤其是 tree-based 模型（Random Forest、XGBoost）——经常出现不合理的锯齿状波动。你可以对预测序列做一次轻度的低通滤波或 Savitzky-Golay 平滑：

```python
forecast_raw = model.predict(X_future)
forecast_smooth = savgol_filter(forecast_raw, window_length=7, polyorder=2)
```

这就是把 S-G 滤波器用在了预测输出上，而不是原始观测上。同一个函数，同一个原理，只改变了作用对象。

异常分数的处理同理——异常检测模型输出一个分数序列，偶发的孤立尖峰可能是误报。对异常分数做中值滤波（`medfilt`）可以去掉这些孤立的假阳性——原理和去除脉冲噪声完全一致。

### 7.2 鲁棒投影（RPCA）——二维矩阵的去噪

#### 场景

假设你管理一个大型零售预测系统，同时预测 1000 个门店 × 500 个 SKU 的未来 30 天销量。把所有预测值排成一个矩阵：行 = 时间，列 = 门店-SKU 组合。

正常预测应该有**低秩结构**——大多数序列的销售模式是相似的，几百条预测曲线可以被少数几个"基础模式"（总体趋势、季节性、促销效应等）近似概括。但某些序列可能因为数据质量问题出现大幅偏差——在矩阵中表现为**稀疏的大误差**。

#### Robust PCA 的思路

RPCA 把这个预测矩阵 $M$ 分解为两部分：
$$M = L + S$$

- **$L$（低秩部分）**：所有序列共享的结构——"干净的预测"。平滑、有规律、可被少数几个因子概括。
- **$S$（稀疏部分）**：个别序列的异常偏差。它是稀疏的——只有少数位置有显著值——可能是数据错误或模型在某个特定序列上失效。

算法同时优化两个目标：$L$ 的秩尽可能低（结构简单），$S$ 尽可能稀疏（只有少数位置有显著值）。最终你取 $L$ 作为修正后的预测。

#### 和一维去噪的对应关系

| 维度 | 一维信号去噪 | 二维鲁棒投影 (RPCA) |
|------|-----------|-------------------|
| 输入 | 一个含噪信号 $x[n]$ | 一个含异常预测的矩阵 $M$ |
| 分解 | 干净信号 + 噪声 | 低秩结构 $L$ + 稀疏误差 $S$ |
| "干净"的定义 | 低频 / 平滑 | 低秩 / 可被少数因子概括 |
| "噪声"的定义 | 高频 / 随机 | 稀疏 / 只影响少数序列 |
| 核心操作 | 滤波（卷积）| 凸优化（核范数 + L1 正则化）|
| scipy 工具 | `scipy.signal` | `scipy.linalg` / `sklearn.decomposition` |

**底层哲学完全一致**：你假设真实世界有简洁的结构（平滑、低频、低秩）。观测 = 结构 + 偏离结构的噪声/异常。去噪和鲁棒投影都在做同一件事——提取结构。

### 7.3 层次预测的协调（Reconciliation）

你分别预测了全国、各省、各市的销量。三个模型独立输出，加起来不相等——全国 ≠ 各省之和，省 ≠ 各市之和。

协调就是把独立预测"投影"到一个满足加和约束的空间中：
$$\min_{\tilde{y}} \|\tilde{y} - \hat{y}\|^2 \quad \text{s.t.} \quad S\tilde{y} = 0$$

其中 $\hat{y}$ = 原始预测，$\tilde{y}$ = 协调后预测，$S$ = 加和约束矩阵（例如"全国 - 省₁ - 省₂ - … = 0"）。

这和去噪的数学形式完全一致：**去噪 = 把信号投影到"平滑子空间"；协调 = 把预测投影到"层次一致性子空间"。** 同样是从不符合约束的原始值到符合约束的修正值的投影。

### 7.4 频谱后处理——去掉预测中的虚假周期

模型预测结果中可能意外出现了不应存在的周期——比如日度预测里出现了 2 天的虚假振荡。对预测序列做频谱分析（第7章），如果发现不应该存在的显著峰 → 用陷波器（带阻滤波器，第3-5章）去掉该频率成分 → 得到修正后的预测。

```

                        │   信号处理工具箱   │
                        │  滤波/分解/频谱   │

                               │
              ┌────────────────┼────────────────┐
              ↓                ↓                ↓
        ┌──────────┐    ┌──────────┐    ┌──────────────┐
```

## 8. 本章要点速查

| 概念 | 一句话 |
|------|--------|
| `savgol_filter` | 局部多项式拟合——平滑的同时保留信号中的弯曲和尖峰 |
| `medfilt` | 滑动窗口中位数——专治偶发的大幅尖峰（脉冲噪声） |

| 去噪评估 | 有真值用 MSE/SNR；没真值看残差是否"像纯噪声" |
| 前处理 vs 后处理 | 同一个工具箱——滤波/分解/投影用在哪都行 |
| RPCA | 矩阵版的去噪——低秩=结构，稀疏=异常 |
