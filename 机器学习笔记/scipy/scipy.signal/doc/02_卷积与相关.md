## 0. 为什么学这个？——从你已经会的东西引入

### 0.1 你已经用过卷积了，只是不知道叫这个名字

做金融数据分析时，你一定写过这样的代码：

```python
# 计算 5 日简单移动平均线 (SMA-5)
sma_5 = prices.rolling(window=5).mean()
```

这就是卷积。**滚动窗口的加权平均 = 卷积的一种特例。**

用 `scipy.signal.convolve` 等价实现：

```python
from scipy.signal import convolve
kernel = np.ones(5) / 5          # 等权重：窗口内每个点贡献 1/5
sma_5 = convolve(prices, kernel, mode='same')
```

实际上，pandas 的 `.rolling()` 底层思想就是离散卷积，只是 scipy 的卷积更灵活——**你可以自定义任何权重模式**，而不仅仅是"等权重平均"。

### 0.2 本章的金融和电力场景

| 概念 | 金融场景 | 电力场景 |
|------|---------|---------|
| **卷积（平滑）** | 均线系统（SMA/EMA/WMA）、去噪 | 日负荷曲线平滑、提取典型用电模式 |
| **卷积（差分）** | 价格变化的边缘检测 | 负荷突变检测（如大型设备启停） |
| **相关（匹配）** | 在历史数据中搜索特定 K 线形态 | 在负荷曲线中搜索特定用电模式 |
| **相关（时延）** | 判断两支股票谁领先谁（lead-lag） | 判断区域负荷波动从哪个节点最先开始传播 |

### 0.3 与你的统计学知识的对接

| 统计学概念 | 信号处理对应 |
|-----------|-------------|
| 加权移动平均 WMA | 卷积（kernel 就是权重向量） |
| 两个随机变量的协方差 | 两个信号的互相关（去掉归一化就是协方差的"滑动版"） |
| 自相关函数 ACF | 信号与自己的互相关 |
| OLS 中的系数 β | 匹配滤波器——找最佳匹配的模板 |

## 1. 卷积的直觉：滑动加权平均

### 1.1 一个例子彻底搞懂

假设你有一支股票的 10 天收盘价：

```
价格:  [10, 12, 11, 13, 14, 13, 15, 14, 16, 15]
```

你想做 3 日简单移动平均。手动计算：

```
SMA_3[2] = (10 + 12 + 11) / 3 = 11.0   ← 第3天
SMA_3[3] = (12 + 11 + 13) / 3 = 12.0   ← 第4天
SMA_3[4] = (11 + 13 + 14) / 3 = 12.67  ← 第5天
...
```

写成"权重 × 值"的形式：

```
SMA_3[i] = 1/3 × price[i-2] + 1/3 × price[i-1] + 1/3 × price[i]
```

这三个权重 `[1/3, 1/3, 1/3]` 就是**卷积核 (kernel)**。卷积操作就是把这个核从信号开头滑到结尾，每一步计算加权和：

```
核: [1/3, 1/3, 1/3]
       ↓    ↓    ↓
信号: ... price[i-2], price[i-1], price[i], ...
       ─────────────────────────────
              卷积输出 y[i]
```

**一句话**：卷积 = 拿一个小数组（核）在大数组（信号）上滑动，每一步计算加权和。

### 1.2 为什么卷积比 rolling 更强大？

`pandas.rolling().mean()` 只能用等权重。卷积允许你设计**任意形状的权重**：

| 核的类型 | 权重模式 | 效果 | 金融/电力用途 |
|---------|---------|------|-------------|
| **均匀核** | `[1/5, 1/5, 1/5, 1/5, 1/5]` | 简单平滑 | SMA、日负荷平滑 |
| **三角形核** | `[1/9, 2/9, 3/9, 2/9, 1/9]` | 中间权重大，两端小 | 更平滑的均线（减少锯齿） |
| **指数衰减核** | `[0.5, 0.25, 0.125, ...]` | 近期权重大，远期快速衰减 | 近似 EMA |
| **差分核** | `[-1, 1]` | 计算相邻变化 | 价格的一阶差分（类似收益率） |
| **Sobel 核** | `[-1, 0, 1]` | 中心差分 | 检测趋势反转点 |

### 1.3 数学定义

两个离散序列 $x[n]$ 和 $h[n]$ 的卷积：$$y[n] = (x * h)[n] = \sum_{k=-\infty}^{\infty} x[k] \cdot h[n-k]$$

别被公式吓到。**翻译成人话**：输出 $y[n]$ 是把核 $h$ 翻转后，在信号 $x$ 上滑到位置 $n$ 处，对应元素相乘再求和。

> 📝 **注意"翻转"**：数学定义中 $h[n-k]$ 的负号意味着核被翻转了。但在实践中，如果你用的核是对称的（比如均线核 `[1,1,1]/3`），翻转不翻转结果一样。只有在非对称核（如 `[-1, 1]` 差分核）时，翻转才会影响结果。

## 2. scipy.signal 中的卷积函数

### 2.1 `convolve` — 基础卷积

```python
from scipy.signal import convolve

result = convolve(signal, kernel, mode='full')
```

#### mode 参数：三种边界处理方式

这是最容易搞混的地方。用 SMA-3 的例子（核 = `[1/3, 1/3, 1/3]`，信号长度 = 10）：

| mode | 输出长度 | 如何处理边界 | 金融数据中的含义 |
|------|---------|-------------|----------------|
| `full` | N + M - 1 = 12 | 核部分在信号外时也计算（补零） | 包含"半边"均线——边界上的不完整窗口也输出 |
| `same` | N = 10 | 输出与输入同长，自动处理边界 | **最常用**——对齐原始时间轴 |
| `valid` | N - M + 1 = 8 | 只输出核完全在信号内的部分 | 无边界效应，但输出更短——前 (M-1) 个和后 (M-1) 个点被丢弃 |

**`mode='full'`（输出长度 = N+M-1 = 12）**：
核滑过信号的所有可能位置，包括核只有一部分覆盖到信号的边缘位置。开头 2 个输出只用了 1-2 个信号点的部分信息（"半边"均线），中间 8 个输出用了完整的 3 个信号点，末尾 2 个输出又是"半边"。这种模式下你可以看到边界效应到底影响了哪些点。

**`mode='same'`（输出长度 = N = 10）**：
从 `full` 的 12 个输出中居中截取 10 个，保证输出和输入对齐。代价是开头约 (M-1)/2 = 1 个点和末尾约 M/2 = 1 个点是用"不完整窗口"算的，数值被稀释（趋于 0）。**金融数据推荐用 `mode='same'`**，这样 SMA 序列和价格序列可以直接对齐。

**`mode='valid'`（输出长度 = N-M+1 = 8）**：
只保留核完全覆盖信号的那些位置。开头 2 个和末尾 0 个位置被丢弃。这 8 个输出没有任何边界问题，但序列变短了，时间轴也偏移了——输出[0]对应原始信号[2]的位置。如果需要和原始序列严格对齐，这种方式不方便。

> 运行本章代码 `ch02_fig2_convolution_modes.png` 可以看到三种 mode 的 visual 对比。

**金融数据推荐用 `mode='same'`**，这样 SMA 序列和价格序列可以直接对齐。

### 2.2 `fftconvolve` — 频域卷积

```python
from scipy.signal import fftconvolve

result = fftconvolve(signal, kernel, mode='full')
```

**什么时候用？** 卷积在时域做是 $O(N \cdot M)$ 的复杂度（N=信号长度，M=核长度）。当核很长（比如 M > 500）时，转换到频域做更快（$O(N \log N)$）。

**选择指南**：

| 场景 | 用哪个 | 原因 |
|------|-------|------|
| 计算 SMA-5 / SMA-20 | `convolve` | 核很短（5-20），直接算更快 |
| 核长度 > 500 | `fftconvolve` | 频域方法超越时域方法 |
| 计算 200 日均线 | 建议 `fftconvolve` | 200 个点已经足够长 |
| 你不确定 | `fftconvolve` | 多数情况下更安全（自动选择最优算法） |

### 2.3 `convolve2d` — 二维卷积

```python
from scipy.signal import convolve2d

result = convolve2d(matrix, kernel, mode='same')
```

虽然本笔记聚焦一维信号，但二维卷积非常有用：
- 金融：对多个股票的相关系数矩阵做平滑
- 电力：对"时间 × 区域"的负荷热力图做空间平滑

## 3. 金融场景实战：用卷积设计各种均线

### 3.1 从 SMA 到自定义加权均线

```python
# SMA-5：等权重
sma5_kernel = np.ones(5) / 5
sma5 = convolve(prices, sma5_kernel, mode='same')

# WMA-5：线性递减权重（越近的权重越大）
wma5_kernel = np.array([1, 2, 3, 4, 5]) / 15  # 权重和为1
wma5 = convolve(prices, wma5_kernel, mode='same')

# 三角形均线：中间权重最大（更平滑）
tri5_kernel = np.array([1, 2, 3, 2, 1]) / 9
tri5 = convolve(prices, tri5_kernel, mode='same')
```

**SMA vs EMA 的本质区别**：
- SMA 是一个固定长度的矩形窗口——所有窗口内的点权重相同
- EMA 是一个无限长的指数衰减窗口——理论上所有历史点都有贡献，但越远贡献越小
- 用卷积实现 EMA 需要很长的核（理论上无限长），实战中通常截断到核长度 ≈ 4× 半衰期

> **💡 什么是"半衰期"？**
>
> EMA 的权重公式是 $w_k = \alpha(1-\alpha)^k$，其中 $k$ 是"距离现在多少个时间步"。历史数据的贡献以指数速度衰减。
>
> **半衰期** = 权重衰减到初始值一半所需的步数：$$\frac{w_h}{w_0} = (1-\alpha)^h = \frac{1}{2} \quad\Rightarrow\quad h = \frac{\ln 0.5}{\ln(1-\alpha)} \approx \frac{0.693}{\alpha}$$
> 
>举个例子：$\alpha = 0.1$ → 半衰期 ≈ 7 个时间步。意思是 **7 天前的数据，它对今天 EMA 值的贡献已经降到了当天数据的一半**。$\alpha = 0.05$ → 半衰期 ≈ 14 步——衰减更慢，"记忆"更长。
> 
>**为什么截断到 4× 半衰期？** 因为 $(1/2)^4 = 1/16 \approx 6\%$。4 个半衰期后权重只剩最初的 6%，几乎忽略不计了，继续往后算对结果影响微乎其微，但计算量却无限增长。截断是"精度 vs 计算量"的工程权衡。
> 
>**类比**：和放射性元素的半衰期是同一个数学模式——都是指数衰减。经过 1 个半衰期剩 50%，2 个剩 25%，4 个剩约 6%。EMA "遗忘"历史数据的速度，和放射性物质衰减的速度，本质上是同一种曲线。

### 3.2 用差分核检测趋势变化

```python
# 一阶差分：今天减昨天 → 类似于日收益率
diff1_kernel = np.array([-1, 1])
price_change = convolve(prices, diff1_kernel, mode='same')

# 检测"趋势加速"（二阶差分）：变化的变化
diff2_kernel = np.array([1, -2, 1])  # 对应二阶导数的离散近似
trend_accel = convolve(prices, diff2_kernel, mode='same')
```

这有什么意义？一阶差分告诉你是涨是跌，二阶差分告诉你**涨跌本身是在加速还是减速**——用来判断趋势是否在衰竭。

## 4. 相关 (Correlation)：在信号中找"相似模式"

### 4.1 相关 vs 卷积 —— 就差一个翻转

卷积公式：$y[n] = \sum x[k] \cdot h[n-k]$（核翻转）

相关公式：$y[n] = \sum x[k] \cdot h[k-n]$（核不翻转，或等价地，信号翻转）

**直觉区别**：
- 卷积：核在信号上"滑过"，像用一把刷子刷过数据
- 相关：核在信号上"搜寻"，**在每个位置输出"此处有多像目标模式"的得分**

### 4.2 `correlate` — 模板匹配

```python
from scipy.signal import correlate

# 在价格序列中搜索"头肩顶"形态
pattern = np.array([...])  # 头肩顶的模板
match_score = correlate(prices, pattern, mode='same')
# match_score 在模板匹配的位置达到峰值
```

**金融场景**：在历史价格中搜索某个特定的 K 线组合形态（如三只乌鸦、启明星），相关函数天然输出"匹配得分"。

**电力场景**：在日负荷曲线中搜索某个高耗能设备（如电弧炉）的典型用电模式，从而做非侵入式负荷分解。

### 4.3 互相关与时延估计

两个信号 $x$ 和 $y$ 的互相关 $R_{xy}[\tau]$ 衡量：**当把 $y$ 平移 $\tau$ 步后，和 $x$ 有多像。**

```python
from scipy.signal import correlate

# 两支行业相近的股票日收益率
# 判断 B 是否跟随 A（B 滞后于 A）
returns_A = ...  # 龙头股
returns_B = ...  # 跟随股

corr = correlate(returns_A, returns_B, mode='full')
lags = np.arange(-len(returns_A)+1, len(returns_B))

# 互相关最大的 lag 就是 B 相对于 A 的滞后天数
best_lag = lags[np.argmax(corr)]
print(f"B 滞后 A 约 {best_lag} 天")
```

**金融意义**：这是量化金融中的 **lead-lag 分析**——判断一个市场/板块/股票的变动是否领先于另一个。对于统计套利和配对交易非常重要。

**电力意义**：当电网出现扰动时，判断波动从哪个区域节点最先传播出来——帮助定位故障源。

### 4.4 自相关 = 信号和自己的互相关

$$R_{xx}[\tau] = \sum_{n} x[n] \cdot x[n+\tau]$$

这就是你在时间序列分析课上学过的**自相关函数 (ACF)**！信号处理中叫"自相关"，统计学中叫 ACF——同一个东西。

```python
# scipy 计算自相关
from scipy.signal import correlate
acf = correlate(x, x, mode='full')
acf = acf[len(acf)//2:]  # 只取非负延迟部分
acf = acf / acf[0]       # 归一化：ACF[0] = 1
```

## 5. 电力场景实战：负荷曲线分析

### 5.1 日负荷曲线平滑

电力负荷的原始数据充满随机波动（每个用户的开/关行为是随机的）。用卷积平滑可以提取"典型日负荷形状"：

```python
# 用三角形核平滑 96 点日负荷曲线（15分钟一个点）
kernel = signal.windows.triang(13)        # 约3小时的平滑窗口
kernel = kernel / kernel.sum()            # 归一化
smoothed_load = convolve(load, kernel, mode='same')
```

### 5.2 负荷突变检测

```python
# 检测负荷的突然变化（大型工业设备启动）
edge_kernel = np.array([-1, -1, -1, 0, 1, 1, 1]) / 3
# 这个核计算"未来3个点的平均"减去"过去3个点的平均"
load_edge = convolve(load, edge_kernel, mode='same')
# 正峰值 = 负荷突然上升，负峰值 = 负荷突然下降
```

### 5.3 在年度负荷数据中搜索季节性模式

```python
# 定义"夏季尖峰"模式的模板（连续高温导致空调负荷飙升）
summer_spike_template = ...  # 从历史典型夏季周提取
match = correlate(annual_load, summer_spike_template, mode='same')
# match 的高分值位置 = 检测到的夏季尖峰时段
```

## 6. 边界效应：你的均线为什么开头有 NaN？

这是每个做金融数据分析的人都遇到过的问题：

```python
# pandas
prices.rolling(5).mean()  # 前 4 天是 NaN

# scipy convolve with mode='same'
convolve(prices, np.ones(5)/5, mode='same')  # 前 2 天有边界伪影
```

### 6.1 三种处理方法对比

| 方法 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| 丢弃（`mode='valid'`） | 只保留完整窗口内的结果 | 无任何边界问题 | 序列变短，时序不对齐 |
| 零填充（`mode='same'` 默认） | 边界外当 0 处理 | 输出与输入对齐 | 边界附近的值被稀释（趋于 0） |
| 反射/镜像填充 | 边界外镜像反射信号 | 比零填充更自然 | 计算复杂 |
| `pandas` 方式 | 不做填充，边界输出 NaN | 明确标记不可靠区域 | 需要额外处理 NaN |

**实战建议**：如果你用卷积计算均线，且需要和原始序列对齐做后续分析——用 `mode='same'`，然后**手动裁剪掉前 (M-1)//2 和后 M//2 个点**（这些点的均线只用了"部分窗口"）。

## 7. 性能考量

| 方法 | 时间复杂度 | 核长度 M=5 | M=200 | M=5000 |
|------|-----------|-----------|-------|--------|
| `convolve` 直接算 | $O(N \cdot M)$ | ✅ 快 | 🟡 还行 | ❌ 慢 |
| `fftconvolve` | $O(N \log N)$ | 🟡 开销大 | ✅ 快 | ✅ 快 |
| `convolve` + `method='auto'` | 自动选 | ✅ 自动 | ✅ 自动 | ✅ 自动 |

新版本的 scipy 中 `convolve(method='auto')` 会自动选择最优算法，多数情况下直接用默认即可。

## 8. 本章要点速查

| 概念 | 一句话 |
|------|--------|
| 卷积 | 核在信号上滑动，每一步计算加权和 — 移动平均的数学本质 |
| 核 (kernel) | 一组权重，决定了"如何对邻域加权" |
| `convolve(sig, kernel, mode)` | scipy 的一维卷积函数 |
| `mode='same'` | 输出与输入等长，最常用但要注意边界 |
| `mode='valid'` | 去除了所有边界效应，输出变短 |
| `fftconvolve` | 频域卷积，长核（>500点）时更快 |
| `correlate` | 不翻转核的"卷积" → 衡量两个信号的相似度随平移的变化 |
| 互相关 | 两信号在不同时移下的相似度 → lead-lag 分析 |
| 自相关 | 信号和自己的互相关 = ACF |
