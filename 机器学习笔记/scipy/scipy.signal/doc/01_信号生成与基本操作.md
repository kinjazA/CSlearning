# 第1章 · 信号生成与基本操作

## 0. 前置认知：信号处理全景——学这个到底有什么用

### 0.0 统计学和信号处理：两套互补的眼睛

你学统计学的时候，核心任务大概是这几类：
- **描述**：均值、方差、分布——了解数据长什么样
- **推断**：假设检验、置信区间——从样本推断总体
- **建模**：回归、分类——用变量预测变量

信号处理有它自己的一套核心任务，**和统计学是平行且互补的，不是一个为另一个服务的附庸**。

**信号处理是干什么的？** 它是你分析有序数据（特别是时间序列）的另一套眼睛。统计学关注"分布和相关性"，信号处理关注"频率和波形"。两套眼睛一起用，你看到的东西比只用一套多得多。

整个 scipy.signal 的学习，围绕着六类独立的数据分析任务展开：

#### 任务一：看清楚你的数据（频谱分析 — 第7-8章）

统计给你均值、方差、分位数。但如果你的数据有**周期性结构**，这些统计量完全看不出来。

比如你有一年的日度气温数据。统计告诉你"均值 15°C、标准差 8°C"。但 FFT（第7章）立刻告诉你：**能量集中在年周期和日周期上**——数据结构一目了然。这不是 ML 特征，这就是分析本身。

#### 任务二：把信号拆开来看（滤波 — 第3-6章）

你学 ANOVA 的时候，把总方差拆成"组间方差 + 组内方差"。信号处理做同样的事，但按**频率**拆。

一支股票的价格 = 长期趋势 + 中期周期 + 短期波动 + 随机噪声。滤波就是把这四样东西分开。分完之后你可以单独分析每一个成分——这和 ANOVA 分解方差是同一类思维。

#### 任务三：把信号洗干净（去噪/去干扰 — 第6、12章）

你做过数据清洗——去掉异常值、处理缺失值。信号处理的"清洗"是按频率来的：电力负荷里混进了 50Hz 干扰 → 陷波器去掉；传感器数据有高频噪声 → 低通滤波平滑。

这不是"为 ML 服务"的预处理。你做描述性分析也需要干净数据。就像回归前处理离群值，不是因为影响特征质量，而是因为**数据不对**。

#### 任务四：发现事件和变化（峰值检测 — 第9章）

电网频率突然跳变 → 用瞬时频率追踪锁定跳变时刻。股票波动率突然放大 → 用频谱图看高频能量何时增强。脑电图在刺激后出现特定波形 → 用峰值检测自动找到它。

这不是 ML。这是**信号层面的诊断和发现**——"发生了什么、什么时候发生的"。

#### 任务五：比较两个信号的关系（相干性、互相关 — 第2、7章）

你学过相关系数——两个变量"总体上"是否相关。但两个信号可能**在某些频率上高度同步，在另一些频率上完全无关**。一个总相关系数会掩盖这种结构。

两只电力股在 20-60 天周期上高度同步（行业驱动），但在日间噪声上完全独立（个股特质）。相干性让你**按频率分解相关性**——比一个 Pearson r 精细得多。

#### 任务六：操作和变换数据（重采样、对齐 — 第11章）

降采样（15分钟→小时）必须先做抗混叠滤波，否则引入假信号。非均匀采样的数据想看周期性 → 用专门的方法而不是强行插值。两个信号有时延 → 用互相关找滞后天数。

这些是对数据本身的**操作**——属于数据工程，不是 ML 特征工程。

#### 一张图总结

```
原始数据
    │
    ├── 任务一：看清楚（频谱、时频图）
    │
    ├── 任务二：拆开看（滤波分离成分）
    │
    ├── 任务三：洗干净（去噪、去干扰）
    │
    ├── 任务四：找事件（峰值检测、变化发现）
    │
    ├── 任务五：比关系（相干性、互相关）
    │
    ├── 任务六：变换（重采样、对齐）
    │
    └── 然后你可以选择：
          ├── 继续做统计分析（数据更干净、结构更清楚）
          ├── 送入 ML 模型（特征质量更高）
          └── 直接得出结论（"检测到 N 次异常"）
```

#### 统计 vs 信号：两张互补的视角

| 统计学视角 | 信号处理视角 |
|-----------|------------|
| 数据 = 随机样本 | 数据 = 携带信息的有序序列 |
| 关注：分布、均值、方差 | 关注：频率、周期、波形 |
| 核心操作：估计、检验、回归 | 核心操作：滤波、变换、检测 |
| 问："这两个变量相关吗？" | 问："这两个信号在 10Hz 处相干吗？" |
| 问："均值是多少？" | 问："主频率是多少？" |

真正好的分析，是**两者结合**——用统计的严谨来描述不确定性，用信号的视角来揭示结构。Welch 功率谱估计（第7章）本身就是统计估计问题：用分段平均降低谱估计的方差，背后是估计理论。降采样前的抗混叠（第6章）就是在避免因采样不当产生的系统性偏差。

**ML 特征工程，只是在你有了一套完整的信号分析能力之后，顺带能做的一件事而已。**

#### 落地：这套方法在时序预测和异常检测中怎么用？

上面六类任务是"能力层"——你掌握了这些工具。但它们在你最关心的两个实际场景（预测和检测）中具体怎么落地？下面是按实战场景重新组织的指南。

---

**在时序预测中的应用**

**① 分解 = 滤波（对应第3-6章）**

时序预测的第一步往往是"看看数据长什么样"。STL 分解用 LOESS 平滑来提取趋势和季节成分，但你用滤波器可以做同样的事，且更精确：

- 低通滤波 → 提取趋势（替代 LOESS）
- 带通滤波 → 提取特定周期（"只保留 3-6 个月的波动作为季节成分"）
- 高通滤波 → 提取残差（去趋势去周期后剩下的）

和 STL 的关键区别：STL 用 LOESS 的窗口参数间接控制平滑程度。滤波器让你直接说"周期大于 30 天的归为趋势，小于 10 天的归为噪声"——频率边界是明确的。

分解之后的核心价值：你不需要一个模型硬啃原始序列。趋势用 ARIMA、季节成分用 Seasonal Naive、残差用 GARCH 捕捉波动率聚集——分别建模再叠加，通常效果好得多。

**② 频谱分析 → 发现周期、确定模型参数（对应第7章）**

SARIMA 需要你指定 seasonal period（如月度数据的 s=12）。Prophet 可以加多个傅里叶项來捕捉周期——但加多少项？每个周期的阶数设多少？

- 对训练数据做 Welch 功率谱估计
- 频谱图中显著的峰值 → 数据里真实存在的周期 → 你的模型参数
- 峰的高度 → 该周期的强度 → 傅里叶项阶数的依据

这不是靠经验瞎猜——频谱图直接告诉你数据的内在周期结构。

**③ 去噪 → 提高预测信噪比（对应第6、12章）**

高频金融数据（tick 级别），噪声成分远大于信号成分。直接把含噪数据喂给 LSTM/Transformer，模型要同时学"信号的规律"和"忽略噪声"——负担重，收敛慢，泛化差。

先用低通滤波器去掉确定是噪声的高频成分，再送入模型，预测精度通常会有明显提升。关键原则：**去噪不能过度**。把所有高频都去掉，突发事件（闪崩、政策冲击）的前兆也被抹掉了。滤波器设计就是在"保留有意义的高频"和"去掉确定性噪声"之间找平衡。

**④ 时频分析 → 处理非平稳数据（对应第8章）**

ARIMA 假设平稳（或差分后平稳）。但现实数据的频率结构往往随时间变化——金融市场的波动率聚集就是典型例子：平时小幅波动（高频能量低），危机期间剧烈波动（高频能量突然增强）。

频谱图让你看到频率结构的演化过程。可以据此**分段建模**：平稳段用 ARIMA，高波动段换 GARCH 或加 volatility 特征，而不是一个模型硬套全时段。

**⑤ 重采样和多尺度建模（对应第11章）**

- 分钟级数据预测日级走势 → 先降采样（抗混叠滤波 + 抽取），避免混叠引入假模式
- Hierarchical forecasting → 同一组数据在不同采样率下分别建模，融合多时间尺度的预测

**在异常检测中的应用**

**① 残差分析 = 高通滤波 + 阈值（对应第6、9章）**

这是最经典、最实用的做法：

```
原始信号 → 低通滤波（提取"正常模式"）
    ↓
残差 = 原始 - 正常模式
    ↓
残差中超过阈值的点 → 标记为异常
```

**为什么不在原始数据上直接设阈值？** 举一个例子：电网频率标称 50Hz，正常波动范围 49.90-50.10 Hz。如果频率从 50.00 缓慢漂移到 49.85 Hz——这已经是严重异常（可能是大型机组即将脱网），但它在"绝对阈值" [49.90, 50.10] 内并没有越界。用残差分析：先低通滤波得到"正常漂移趋势"，残差 = 原始 - 趋势，漂移在残差中累积显现，远早于绝对值越界就能检测到。

**② 频谱监控 = 检测频域结构变化（对应第7章）**

一个稳定运行的系统有固定的"频谱指纹"：
- 正常运行状态：能量集中在其固有频率及谐波上
- 异常发生初期：出现新的频率成分（故障特征频率）
- 异常恶化阶段：固有频率的能量分布改变

做法：维护一个"正常状态"的 Welch PSD 作为基线。每次新数据到达，计算当前时段 PSD，和基线对比——多了新峰或峰的位置漂移 → 告警。

这在电力系统中尤其关键：电网正常情况下频率功率集中在标称频率附近。如果突然出现 0.1-2 Hz 的低频振荡——意味着区域间的发电机群失去了同步，这是大停电的前兆，必须秒级响应。

**③ 相干性分析 = 检测"应该同步的信号不同步了"（对应第7章）**

很多系统中有相互校验的冗余信号：
- 两个区域电网频率高度相干（通过输电线耦合）
- 同一资产在两个交易所的价格高度同步
- 一个系统中两个传感器的读数应该相关

如果某个频段的相干性突然下降 → 该频段的耦合出了问题。一个总相关系数可能被其他频段的正常波动掩盖掉，但分频率的相干性可以精确指出"在 5-15 Hz 这个特定频段上失去同步"。

**④ 瞬时频率追踪 = 检测"变化模式"异常（对应第8、10章）**

Hilbert 变换（第10章）给出信号的瞬时频率。电网频率标称 50Hz，实际在 49.95-50.05 之间微小波动是正常的。但如果瞬时频率发生以下模式：
- 突然以 > 0.1 Hz/s 的速率下降 → 大型机组脱网
- 出现持续的周期性振荡 → 控制回路故障

关键洞察：**绝对值可能还在正常范围，但"变化的模式"暴露了问题。** 这就是为什么需要瞬时频率——它追踪的不是"在哪里"，而是"怎么变化"。

**⑤ 峰值检测 = 定位异常事件（对应第9章）**

对残差或特征序列做峰值检测：
- 高 prominence 的峰 → 不寻常的突变事件
- 一段时间内峰密度突然升高（频繁出现中等 prominence 的峰）→ 系统开始不稳定
- 峰的宽度变化：窄峰 → 冲击性故障，宽峰 → 渐进性劣化

**信号处理与纯 ML 的关系：前端 + 后端**

```
┌─────────────────────────────────────────────────────┐
│                    信号处理                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐          │
│  │ 前端      │  │          │  │ 后端      │          │
│  │ 去噪      │  │  ML 模型  │  │ 峰值筛选  │          │
│  │ 分解      │→│ (LSTM/   │→│ 频谱校验  │          │
│  │ 特征提取  │  │  XGBoost) │  │ 相干性分析 │          │
│  └──────────┘  └──────────┘  └──────────┘          │
└─────────────────────────────────────────────────────┘
```

- **前端**：滤波去噪、分解提取成分、频谱提取周期特征——数据在进模型之前就已经比原始数据"干净且结构化"了
- **后端**：模型输出异常分数后，对分数序列做峰值检测去掉孤立误报，对标记的异常事件做频谱分析判断异常类型

**这些方法不依赖 GPU、不需要海量标注数据。** 很多工业场景（电网、传感器监控）数据量不大但物理含义明确，信号处理比深度学习更直接、更可解释。

### 0.1 从统计学走进信号处理：概念翻译

> 以下各节写给有统计学/大数据背景、但无信号处理基础的学习者。我们用你熟悉的概念来做"翻译"，帮你建立直觉。

### 0.2 "信号"是什么？——和你已经知道的东西对接

**一句话**：信号 = 一组按顺序排列的数值，携带信息。

你做过统计分析对吧？那你每天都在和"信号"打交道：

| 你熟悉的概念 | 在信号处理中的称呼 | 例子 |
|-------------|-------------------|------|
| 时间序列 $\{y_t\}$ | 离散信号 $x[n]$ | 每日收盘价、每小时用电负荷 |
| 横截面数据 | （通常不叫信号） | 1000个人的身高体重 |
| 样本量 $N$ | 信号长度 $N$ / 信号持续时间 | 1000个交易日 |
| 采样频率 | 采样率 $f_s$ (Hz) | 日度数据 = 1/86400 Hz |
| 分布 $f(x)$ | 幅值分布 / 直方图 | 日收益率的经验分布 |

**关键区别只有一点**：信号处理特别关注数据的**频率成分**——也就是说，它不仅关心"数值是多少"，更关心"变化有多快"。

举个例子：你拿到一支股票的日收盘价，统计学思维会让你关注均值、方差、自相关。信号处理的思维会让你额外问：**这个价格序列里，哪些波动是"慢变化"（长期趋势），哪些是"快变化"（日内噪声）？**——这就是"频率"的直觉。

### 0.3 从你熟悉的场景建立"频率"直觉

你有过这种经验吗？

```
一支股票的价格走势：
  - 长期看，它从 50 涨到 80，花了 2 年 → 这是一个极低频率的"信号成分"
  - 中期看，它每几个月有一个涨跌循环 → 中等频率
  - 短期看，它每天上上下下 → 高频波动（部分可能是噪声）
```

这三种变化叠加在一起，就构成了你看到的完整价格走势。**信号处理的核心思想就是：把这三种不同"速度"的变化分开来分析。** 这就是滤波的本质——第3-6章会展开讲。

再看一个你更熟悉的：

```
电力负荷数据（每小时一个点）：
  - 每天有两个峰（上午和晚上） → 周期为 24 小时的频率成分
  - 每周工作日高、周末低 → 周期为 7 天的频率成分
  - 夏季高、冬季因供暖也高 → 周期为 1 年的频率成分
```

这三个周期性成分的叠加，加上一个随机波动，就构成了你看到的电力负荷曲线。用信号处理的工具（FFT、滤波）可以**自动**把它们分离出来，而不需要手动做季节性分解。

### 0.4 信号处理能帮你做什么？（金融 & 电力场景版）

| 任务 | 金融场景举例 | 电力场景举例 |
|------|------------|------------|
| **滤波/平滑** | 去掉股价的日内微观噪声，提取主要趋势 | 去掉电力负荷中的随机波动，得到典型日负荷曲线 |
| **频谱分析** | 发现收益率序列中隐藏的周期模式（如周度效应） | 确认负荷数据中的 24h / 7d / 365d 周期成分 |
| **特征提取** | 从分钟级价格序列提取波动率、趋势强度等特征 | 从负荷曲线提取峰值、谷值、爬坡率等特征 |
| **异常检测** | 检测闪崩或价格异常跳变 | 检测用电异常（如窃电、设备故障） |
| **预测预处理** | 在送入 LSTM 之前，对价格序列做去噪和平滑 | 对负荷序列去噪后再训练预测模型 |

### 0.5 你需要的前置概念（尽量用统计学语言表述）

**采样与离散化**

现实世界的时间是连续的，但计算机只能存储离散的点。就像你做统计调查只能在某些时间点上收集数据一样：

- 连续信号 $x(t)$：真实世界的某种量（股价的"真实"变动）
- 离散信号 $x[n]$：你实际观测到的数据（每秒/每天/每小时一个样本）
- 采样间隔：相邻两个观测之间的时间距离
- 采样率 $f_s$：每秒采集多少个点

$$x[n] = x(n \cdot T_s), \quad n = 0, 1, 2, \ldots$$

$T_s$ = 采样间隔（比如 T+1 交易日的 $T_s$ = 1天）。$f_s = 1/T_s$。

**频率的直觉**

频率 = "每单位时间内，某个模式重复出现了多少次"。

- 日度数据的"最高可检测频率"是每 2 天一个周期（Nyquist 定理）
- 如果某个波动每 1.5 天完成一次涨跌，日度数据无法正确捕捉它——你看到的会是一个**假的、更慢的波动**（混叠）

## 2. 标准测试波形生成

在实际开发中，你经常需要一个"已知属性"的信号来做测试——比如验证你的代码是否正确实现了某个算法。`scipy.signal` 提供了丰富的波形生成器。

### 2.1 正弦波——信号的"标准正态分布"

正弦波之于信号处理，就像正态分布之于统计学——它是理解一切的基准。

一个频率为 $f$ 的正弦波：
$$x(t) = A \cdot \sin(2\pi f t + \phi)$$

- $A$：幅值（amplitude）——波的高度
- $f$：频率（frequency）——每秒完成几个完整周期，单位 Hz
- $\phi$：相位（phase）——波从什么位置开始

**用统计学类比**：一个正弦波是"完全可预测"的——知道频率和幅值就知道全部。而真实世界的信号（股价、用电量）更像是一个正弦波 + 随机噪声（随机过程的一次实现）。

```python
import numpy as np

fs = 1000               # 采样率 1000 Hz（每秒1000个点）
t = np.arange(0, 2, 1/fs)  # 2秒的数据

f = 50                  # 频率 50 Hz
sine_wave = np.sin(2 * np.pi * f * t)
```

### 2.2 chirp — 频率随时间扫动的信号

**是什么**：一个频率从 $f_0$ 逐渐变到 $f_1$ 的正弦波。

**为什么重要**：
- 测试滤波器时，你一个 chirp 就能看到滤波器在所有频率上的表现，而不需要对每个频率单独测试
- 电力系统分析中，电网频率会随时间缓慢波动（标称 50Hz，实际在 49.8-50.2Hz 之间），chirp 可以模拟这种频率漂移
- 金融市场中，波动率的聚集效应可以看作某种"频率调制"——高波动期 = 高频成分增多

```python
from scipy.signal import chirp

chirp_signal = chirp(t, f0=10, f1=200, t1=2.0, method='linear')
# 生成一个信号，其频率从 t=0 时的 10Hz 线性变化到 t=2.0 时的 200Hz
```

四种扫频模式：

| 模式 | 瞬时频率变化规律 | 金融/电力类比 |
|------|----------------|-------------|
| `linear` | $f(t)$ 线性增长 | 每日用电负荷从凌晨低谷线性爬升到中午高峰 |
| `quadratic` | $f(t)$ 二次增长 | 波动率在危机期间加速上升 |
| `logarithmic` | $f(t)$ 对数增长 | 利率从高位的下降速度前快后慢 |
| `hyperbolic` | $f(t)$ 双曲变化 | 特定场景，用得较少 |

### 2.3 square / sawtooth — 方波和锯齿波

这两种波形在电子电路和音频合成中很常见。对统计学习者来说，它们最重要的性质是：

> **方波和锯齿波在频域包含丰富的谐波分量。**

这意味着什么？想象你拿到一个电力负荷数据，看起来像是有一个基础的正弦形状（24小时周期），但它的形状不是完美的正弦——早上爬升快、晚上下降慢。这种"非正弦"的形状，在频域就对应着**除了 24h 基本周期外，还有 12h、8h、6h 等谐波分量**。方波就是谐波含量最丰富的极端例子。

```python
from scipy.signal import square, sawtooth

# 方波：一个在 +1 和 -1 之间跳变的信号
sq = square(2 * np.pi * 10 * t, duty=0.5)   # 占空比 50%

# 锯齿波：线性上升、突降（或反之）
saw = sawtooth(2 * np.pi * 10 * t, width=1)  # width=1 → 上升沿
```

### 2.4 gausspulse — 高斯脉冲

一个被高斯型包络包裹的正弦波。在时域和频域都呈钟形。

**金融类比**：一次性的冲击事件——比如某天央行突然宣布降息，市场先剧烈反应（脉冲的峰值），然后逐渐消化信息、波动衰减（脉冲的拖尾）。

```python
from scipy.signal import gausspulse

# 返回 (信号, 包络)
pulse, envelope = gausspulse(t, fc=50)
```

### 2.5 unit_impulse — 单位脉冲

$$\delta[n] = \begin{cases} 1 & n = 0 \\ 0 & n \neq 0 \end{cases}$$

**为什么这个重要？** 在信号处理中，单位脉冲就好比统计学中的"单位冲击"——你把一个系统（模型）的输入换成 $\delta[n]$，看它的输出，这个输出（称为"脉冲响应"）完全描述了该系统的特性。这在第10章 LTI 系统分析中会深入展开。

金融中的类比：假设市场在 $t=0$ 时刻受到一个"瞬时冲击"（比如一笔大单），观察价格在后续时刻如何响应——这就是在做"脉冲响应分析"（你学 VAR 模型时应该见过这个词）。

## 3. 信号的时域基本特征

拿到一个信号后，第一件事是描述它。这些特征你在描述性统计里都学过——只不过换了个名字。

### 3.1 核心描述量

| 特征 | 计算方式 | 统计学对应 | 金融意义（以日收益率为例） | 电力意义（以日负荷为例） |
|------|---------|-----------|--------------------------|------------------------|
| **均值** | $\frac{1}{N}\sum x[n]$ | 样本均值 $\bar{x}$ | 平均日收益率 | 平均日用电量 |
| **标准差** | $\sqrt{\frac{1}{N}\sum(x[n]-\bar{x})^2}$ | 样本标准差 $s$ | 日收益率的波动率 | 负荷的波动幅度 |
| **RMS** | $\sqrt{\frac{1}{N}\sum x[n]^2}$ | 二阶原点矩的平方根 | 价格变动的有效幅度 | 负荷的有效值 |
| **最大值/最小值** | $\max(x), \min(x)$ | 极值 | 日内最高/最低价 | 日最大/最小负荷 |
| **峰峰值** | $\max - \min$ | 极差 | 日内振幅 | 峰谷差 |

### 3.2 形状特征（高阶矩）

这几个你在统计学中学过——它们描述分布的"形状"：

| 特征 | 含义 | 金融数据中的典型值 |
|------|------|-------------------|
| **偏度** Skewness | 分布不对称的方向和程度 | 日收益率通常左偏（负偏度），因为暴跌比暴涨更频繁 |
| **峭度** Kurtosis | 尾部的"厚度" | 日收益率峭度通常 > 3（尖峰厚尾——极端值比正态分布多） |
| **峰值因子** Crest Factor | $\max|x|$/RMS | 衡量是否有异常尖峰——金融危机期间峰值因子会飙升 |

### 3.3 过零率 (Zero-Crossing Rate)

信号穿过零值线的频率。这是一种非常粗略的"频率感"——过零率越高，说明信号振荡越快。

```python
zero_crossings = np.sum(np.abs(np.diff(np.sign(x)))) / 2
zero_crossing_rate = zero_crossings / len(x)
```

- 高频交易数据（tick 级）→ 过零率高
- 日度宏观经济指标 → 过零率低

### 3.4 从统计学视角看这些特征

你把 $\{x[n]\}_{n=0}^{N-1}$ 当作随机过程的一次实现：

```
信号样本  →  描述量  →  估计的是...

x[1], x[2], ..., x[N]  →  样本均值  →  均值函数 μ(t)（在平稳假设下为常数）
                        →  样本方差  →  方差函数 σ²(t)
                        →  直方图    →  一维边际分布 f(x)
                        →  样本偏度  →  分布的对称性
                        →  样本峭度  →  分布的尾部厚度
```

你对平稳性的理解在这里很关键：**只有信号是平稳的（或经过预处理使其平稳），这些全局描述量才有稳定的统计意义。** 如果信号的统计特性在变化（非平稳），你需要用时频分析工具（第8章）来看它们的局部特性。

## 4. 信号可视化最佳实践

### 4.1 时域波形

一条原则：**x 轴永远用物理时间（秒、分钟、天），不要用样本点序号。**

```python
# ❌ 不好的做法
plt.plot(signal)  # x 轴是样本序号 0, 1, 2, ...——别人不知道这是多久

# ✅ 好的做法
t = np.arange(len(signal)) / fs  # 时间轴
plt.plot(t, signal)               # x 轴是真实的秒/分钟/天
plt.xlabel('Time (s)')
plt.title(f'Signal (fs={fs} Hz, N={len(signal)})')  # 元信息写入标题
```

### 4.2 金融/电力数据的特殊考虑

- **金融数据**：x 轴用日期（matplotlib 的 `mdates`），标注休市日（防止因假期造成的"跳空"看起来像信号异常）
- **电力负荷**：数据量可能非常大（智能电表 15 分钟一个点 = 每天 96 个点 = 一年 35040 个点），长序列可视化时需要降低采样密度
- **双 y 轴**：股价和成交量在同一图上时用双 y 轴（但不要滥用——太多轴反而混乱）

### 4.3 长信号的处理

当一个信号有超过 5000 个点时，直接用默认线宽会导致图形糊成一片：

```python
if len(signal) > 5000:
    plt.plot(t, signal, linewidth=0.3)  # 细线
else:
    plt.plot(t, signal, linewidth=1.0)  # 正常线宽
```
