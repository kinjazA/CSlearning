## 0. 回顾与引入：第3章我们做了什么，为什么还不够？

### 0.1 先确认你理解了这几个核心概念

在进入 Remez 之前，我们先花一点篇幅把第3章的几个关键概念彻底讲透。如果你对下面任意一个概念还有疑问，先看这里再看后续内容。

#### 概念1：什么是"频率响应"？

想象你有一个黑盒子（滤波器），你输入一个纯正弦波，盒子输出另一个正弦波。如果输入正弦波的频率是 5Hz，输出可能和输入几乎一样（通过了）；如果输入是 80Hz，输出可能几乎为零（被阻挡了）。

**频率响应就是：对每一个可能的频率，回答"输出幅度是输入的多少倍"这个问题。** 画出图来，x 轴是频率，y 轴是"通过比例"（通常用分贝 dB 表示），这就是频率响应图。

#### 概念2：什么是"通带"和"阻带"？

- **通带 (Passband)**：你希望信号**完整通过**的频率范围。在这个范围内，频率响应接近 1（即 0 dB），信号几乎不被衰减。
- **阻带 (Stopband)**：你希望信号**被大幅削弱**的频率范围。在这个范围内，频率响应接近 0（即一个很大的负数 dB，如 -60 dB 意味着衰减到原来的 1/1000）。
- **过渡带 (Transition Band)**：通带和阻带之间的"缓冲区"。从"完整通过"到"几乎阻挡"不是瞬间发生的——中间总有一个过渡区域。

**用金融数据举例**：你想从日度股价中提取"20天以上的趋势"。

- 周期 = 20 天 → 频率 = 252/20 ≈ 12.6 次/年
- 通带：周期 > 20天（频率 < 12.6 次/年）→ 趋势成分，希望通过
- 阻带：周期 < 5天（频率 > 50 次/年）→ 日间噪声，希望阻挡
- 过渡带：周期 5-20天（频率 12.6-50 次/年）→ 灰色地带

你不可能只保留"恰好 20 天以上"的趋势而完美阻挡"19 天"的波动——总会有一个过渡区域。过渡带越窄，说明滤波器"刀切"得越锋利。

#### 概念3：什么是"纹波"（Ripple）？

理想情况下，通带内的频率响应应该完美地等于 1（0 dB），一条平线。但实际上它会有些小波动——在 0 dB 附近微微上下振荡。这个小波动就叫**通带纹波**。

同样，阻带内也不是完美的零——它会有一系列小"凸起"，叫**阻带纹波**。

**用统计学类比**：纹波就像回归模型中的残差。你希望残差为零（完美拟合），但实际上总有一些小的上下波动。纹波越小 = 滤波器越精确。纹波通常用分贝（dB）衡量——±0.01 dB 的纹波意味着实际增益在 0.9988 到 1.0012 之间，非常平；±1 dB 则意味着明显的波动。

#### 概念4：什么情况下"频率响应的误差"大？

窗口法（`firwin`）设计的滤波器，误差分布不均匀：

- 在通带中间（远离过渡带的位置）→ 误差很小，很平
- 在通带边缘（靠近过渡带）→ 误差变大，波动明显
- 在阻带中 → 越靠近过渡带，泄漏越多

这就像你做局部回归（LOESS）时，数据密集的地方拟合好，数据稀疏的地方拟合差。误差在整个频带上不是均等的。

### 0.2 窗口法的"不完美"——用一个比喻说清楚

想象你要在院子中间修一道篱笆，把前院（通带）和后院（阻带）隔开，但你不能修一堵完美的墙。你只能种一排灌木（有限长度的滤波器）。

- **窗口法**：你选了一种灌木品种（选窗函数），种了 31 棵（numtaps=31）。灌木之间的缝隙（过渡带）比较宽，而且靠近篱笆边缘的地方缝隙特别大（通带边缘误差大）。
- **Remez 法**：你请了一个园林设计师，他说："我可以在同样的 31 棵灌木里，把所有缝隙缩到最小，而且保证每个缝隙都一样大（误差均匀分布）。"这就叫**等波纹（equiripple）**——通带和阻带内的最大误差被压到同一个水平，而且过渡带比窗口法更窄。

**为什么叫"最优"？** 因为在"让最大误差最小化"这个标准下（minimax 准则），Parks-McClellan 算法（Remez 是它的实现）给出了数学上可证明的最优解——没有其他 31 阶 FIR 滤波器能在保持同样通带/阻带纹波的同时有更窄的过渡带。

### 0.3 什么时候用 Remez 而不是 firwin？

| 场景 | 用什么 | 原因 |
|------|-------|------|
| 你只是想做简单的平滑/去噪 | `firwin` + `hamming` | 代码少、够用、脑子不用转弯 |
| 你需要低延迟（阶数受限），但过渡带要尽可能窄 | `remez` | 同等阶数下过渡带最窄 |
| 你需要同时控制多个频带（>2个通带/阻带组合） | `remez` | `firwin` 只支持单通带或单阻带 |
| 你对通带平整度有要求但对阻带要求更高（或反过来） | `remez` | 用 `weight` 参数分配精度 |
| 你要做高频交易，延迟每多一天都是钱 | `firwin`/`remez` + `minimum_phase` | 牺牲相位换延迟 |

**一句话**：`firwin` 是日常代步的轿车，`remez` 是赛道上的跑车。轿车够用的时候开轿车，需要性能的时候上跑车。

## 1. Remez 算法：它在做什么？（直觉版，不涉及数学证明）

### 1.1 核心思想：让最大误差最小化

窗口法的思路是："我设计一个理想滤波器（无限长），然后用一个窗把它截短。"

Remez 的思路完全不同："我不关心理想滤波器长什么样。我直接提要求——在通带内误差不能超过 δ₁，在阻带内误差不能超过 δ₂，过渡带在 f₁ 到 f₂ 之间——然后让算法在这些约束下迭代搜索，直到找到误差最小的解。"

这就像：
- 窗口法 = 先画一条完美的直线，然后把两端剪掉
- Remez = 反复调整，直到所有位置偏离理想值的程度都一样小

**为什么叫"等波纹"？** 因为最优解有一个美丽的性质：通带内的误差峰值全部相等，阻带内的误差峰值也全部相等。就像你把一条橡皮筋拉紧——每个波峰波谷都在同一水平线上。运行 `code/ch04_FIR滤波器设计_Remez最优法.py` 会生成实际的频率响应图，你可以亲眼看到这种"均匀振荡"的图案。

### 1.2 与窗口法的五个关键区别

| 比较维度 | 窗口法 (`firwin`) | Remez 法 (`remez`) |
|---------|------------------|-------------------|
| **你怎么描述滤波器** | 说"我要低通，截止频率是 10Hz" | 说"0-8Hz 通过，12-20Hz 阻挡，8-12Hz是过渡带" |
| **过渡带怎么定** | 由窗函数的类型间接决定（你无法精确指定它有多宽） | 你直接指定过渡带的起点和终点 |
| **通带和阻带的精度** | 间接控制：换窗函数 + 加阶数 | 直接控制：用 `weight` 参数分配"精度预算" |
| **误差分布** | 通带中间误差小、边缘误差大（不均匀） | 通带和阻带内的误差均匀分布（等波纹） |
| **能设计什么** | 四种标准滤波器（低通/高通/带通/带阻） | 任意多带、任意增益组合 |

### 1.3 等波纹长什么样？（用文字描述）

运行本章代码后你会看到类似这样的图案：

- 在 0-10 Hz（通带）：幅度在 +0.02 dB 到 -0.02 dB 之间均匀振荡（像一条被压得很平的波浪线）
- 在 10-15 Hz（过渡带）：幅度从 0 dB 快速跌落到 -60 dB（像滑雪的陡坡）
- 在 15-50 Hz（阻带）：幅度在 -60 dB 到 -65 dB 之间均匀振荡（像一条被压得很低的波浪线）

关键特征是：通带里的"波浪"高度全一样，阻带里的"波浪"高度也全一样。这和窗口法形成鲜明对比——窗口法的波浪在通带边缘会变大、在阻带中越靠近过渡带越高。

## 2. `remez` 函数详解

### 2.1 函数签名和每个参数的含义

```python
from scipy.signal import remez

b = remez(numtaps, bands, desired, weight=None, fs=None)
```

返回的 `b` 是一个一维数组，就是滤波器的系数（核）。长度 = `numtaps`。

现在逐一解释每个参数：

#### `numtaps` — 滤波器的长度（= 阶数 + 1）

和 `firwin` 一样，必须是奇数（保证对称性）。**长度越大 → 过渡带可以越窄、纹波越小 → 但延迟越大。**

延迟天数 = `(numtaps - 1) / 2 / fs`。对于日度数据（fs=252），numtaps=31 的延迟约为 15 天。

#### `bands` — 频带边界列表

这是 `remez` 最核心也最容易出错的参数。它是一组频率值，每相邻两个值定义一个频带。**列表长度必须是偶数**（因为每两个值定义一个频带）。

```python
# 低通滤波器的 bands
bands = [0, 10,    # 第1频带：0 到 10 Hz（这将是通带）
         15, 50]   # 第2频带：15 到 50 Hz（这将是阻带）
```

注意 10 到 15 Hz 之间**没有定义任何频带**——这个间隔自动成为过渡带。过渡带的宽度 = 15 - 10 = 5 Hz。

**过渡带越窄 → 相同纹波要求下需要的 numtaps 越大。** 这是设计中最重要的权衡。如果你发现设计的滤波器实际纹波很大，多半是 numtaps 不够 + 过渡带太窄。

#### `desired` — 每个频带的理想增益

`desired` 的长度 = `len(bands) / 2`，每个值对应对应频带的理想增益。

```python
desired = [1, 0]   # 第1频带增益=1（信号不变），第2频带增益=0（信号被阻挡）
```

增益可以是 0 和 1 之外的任何值。比如 `desired=[1, 0.5, 0]` 表示三个频带的增益分别是 100%、50%、0%。这在需要"衰减但不完全阻挡"某些频段时非常有用。

**金融应用举例**：你想对高频交易信号做一个"半衰"处理——不完全去掉，但把它的影响力减半。设 `desired=[1, 0.5]`（低频全通过，高频衰减到 50%）。

#### `weight` — 误差的相对代价

`weight` 的长度 = `desired` 的长度，每个值决定对应频带的误差"有多难以接受"。

```python
weight = [1, 5]   # 阻带误差的代价是通带的 5 倍
```

这怎么理解？Remez 算法在优化时，目标是最小化加权后的最大误差。weight 越大，该频带的误差就被"放大"越多，算法就会花更多力气去压它。

- `weight=[1, 1]`：通带和阻带同等重要
- `weight=[1, 10]`：阻带的精度优先——算法会牺牲通带的一些平整度来换取阻带更深的衰减
- `weight=[10, 1]`：通带的平整度优先——适合你对信号保真度要求极高的场景

**金融应用指南**：
- 提取趋势（低通，去掉噪声）→ `weight=[1, 5]` 或 `[1, 10]`。噪声必须彻底去掉。
- 提取特定周期（带通，保留信号）→ `weight=[1, 1]`。通带和阻带同等重要。
- 去除工频干扰（带阻/陷波）→ `weight=[1, 20]` 或更高。干扰必须近乎完全消除。

#### `fs` — 采样率

和 `firwin` 一样。传了 `fs` 之后，`bands` 中的频率值直接用 Hz 表示，不需要手动做 Nyquist 归一化。

**强烈建议总是传 `fs` 参数**，避免手动归一化出错。

### 2.2 一个完整的最小例子

```python
from scipy.signal import remez

# 日度金融数据：fs=252 次/年
# 目标：保留周期 > 25 天的趋势，去掉周期 < 17 天的噪声

# 周期 25天 → 频率 252/25 ≈ 10 次/年
# 周期 17天 → 频率 252/17 ≈ 15 次/年

b = remez(
    numtaps=31,
    bands=[0, 10,      # 通带：0-10 次/年（周期 > 25天）
           15, 126],   # 阻带：15-126 次/年（周期 < 17天）
                       # 过渡带自动 = 10-15 次/年
    desired=[1, 0],    # 通带增益=1，阻带增益=0
    weight=[1, 5],     # 阻带精度优先 5 倍
    fs=252
)
# b 就是设计好的滤波器核，长度=31
```

### 2.3 多带滤波器：`firwin` 完全做不到的事情

这是 `remez` 的杀手锏。假设你要设计这样一个滤波器：

- 0-2 次/年（周期 > 126 天）：完全保留（gain=1）—— 这是超长趋势
- 5-12 次/年（周期 21-63 天）：完全阻挡（gain=0）—— 去掉中期波动
- 20-60 次/年（周期 4-13 天）：衰减一半（gain=0.5）—— 短期信号半保留

在 `firwin` 中这根本不可能——它只支持"一段通 + 一段阻"的结构。

在 `remez` 中，就是多写几组 band：

```python
bands   = [0, 2,    # 第1频带：超低频 → gain=1
           5, 12,   # 第2频带：中频 → gain=0
           20, 60]  # 第3频带：高频 → gain=0.5
desired = [1, 0, 0.5]
weight  = [1, 5, 3]

b = remez(numtaps=63, bands=bands, desired=desired, weight=weight, fs=252)
```

这种"三段式"滤波器在什么场景下有用？比如你分析电力负荷，想要同时：
- 保留年度趋势（超低频）
- 去掉日周期和半日周期（中频）
- 保留高频扰动（可能包含设备异常的征兆）

一步到位。

## 3. 最小相位 FIR —— 当延迟比相位更重要

### 3.1 问题：线性相位的代价是延迟

第3章讲过，FIR 滤波器（对称核）有一个很好的性质叫**线性相位**——所有频率成分经过滤波器后被延迟的时间完全相同，所以输出波形不会扭曲。

但这个好性质有代价：**群延迟 = (numtaps - 1) / 2 个样本**。

什么意思？numtaps=51 的日度滤波器，延迟 = 25 天。你今天看到的滤波后价格，反映的是 25 天前的真实趋势。对于月线级别的长期分析这还好，但如果你在做高频交易或实时监控，25 天的延迟是完全不可接受的。

### 3.2 解决办法：最小相位 FIR

有一个数学技巧可以大幅降低延迟：把滤波器的零点（多项式根的另一种形式）全部"搬到"单位圆内。结果：

- **幅度响应几乎不变**（频率选择性保留）
- **延迟大幅降低**（特别是高频成分几乎零延迟）
- **代价：相位不再是线性的**（不同频率延迟不同，波形会略微变形）

**取舍判断**：
- 你在做离线分析（如年报、学术研究）→ 保持线性相位，延迟无所谓
- 你在做实时监控（如高频交易、电网实时状态）→ 牺牲相位线性换低延迟
- 你关心的是信号的整体形状 → 保持线性相位
- 你关心的是"什么时候发生了什么" → 用最小相位

### 3.3 使用方式

```python
from scipy.signal import minimum_phase

# 先正常设计
b_linear = firwin(numtaps=51, cutoff=10, fs=252)    # 延迟 25 天

# 转换为最小相位
b_minphase = minimum_phase(b_linear, method='homomorphic')
# b_minphase 的长度会变短，延迟大幅降低
```

运行本章代码（`ch04_fig4_minimum_phase.png`）可以看到线性相位和最小相位的群延迟对比——最小相位的延迟通常降低 70-90%。

## 4. 完整的设计→分析→验证工作流

设计完滤波器后，一个标准的验证流程包括五个检查项：

### Step 1：检查频率响应

```python
from scipy.signal import freqz
w, h = freqz(b, worN=2048, fs=252)
magnitude_db = 20 * np.log10(np.abs(h))
```

看什么：
- 通带内是不是足够平？（纹波 < 0.1 dB 通常足够）
- 阻带衰减是不是足够？（< -40 dB 是及格线，< -60 dB 是优秀）
- 过渡带是不是在预期范围内？

### Step 2：检查群延迟

```python
from scipy.signal import group_delay
freqs, gd = group_delay((b, 1), fs=252)
```

看什么：
- 线性相位 FIR 的群延迟应该接近常数 = `(numtaps-1)/2`
- 如果用了 `minimum_phase`，群延迟应该远小于这个值

### Step 3：检查相位响应

```python
phase = np.unwrap(np.angle(h))
```

看什么：
- 线性相位 → 相位应该是频率的直线函数
- 最小相位 → 相位是曲线（没关系，正常的）

### Step 4：在实际信号上测试

```python
from scipy.signal import filtfilt
filtered = filtfilt(b, [1], test_signal)
```

看什么：
- 滤波后的信号形状是否符合预期？
- 开头和结尾有没有奇怪的边界效应？
- 趋势/周期被提取出来了吗？

### Step 5：与已知 ground truth 比较

如果你有真实的趋势/周期信号（比如合成数据中你知道真实成分），计算 MSE 来量化滤波器性能。本章代码中的 `ch04_fig5_trend_extraction.png` 就做了 SMA vs firwin vs remez 的量化对比。

## 5. firwin vs remez 决策指南

不需要背规则，问自己四个问题：

**问题1**：我需要非标准频带结构吗？（比如三段式滤波、中间增益 0.5）

- 是 → 必须用 `remez`
- 否 → 继续问题2

**问题2**：我的阶数预算紧张吗？（延迟要求严格，比如必须在 10 天内响应）

- 是 → 用 `remez`（阶数效率最高，省 30-40% 的延迟）
- 否 → 继续问题3

**问题3**：我需要精确控制通带 vs 阻带的相对精度吗？

- 是 → 用 `remez` + `weight`
- 否 → 继续问题4

**问题4**：我只是做常规平滑/去噪，没有特殊要求？

- 是 → 用 `firwin` + `hamming`（最简单、最不容易出错）

**一句话总结**：当你对滤波器性能没有精确的硬性指标时，`firwin` 足够了。当你开始说"我需要在 XX 天延迟内达到 YY 的衰减"时，切换到 `remez`。

## 6. 常见错误和调试方法

| 你做了什么 | 出现的症状 | 为什么 | 怎么修 |
|-----------|-----------|--------|--------|
| `bands` 值不是严格递增 | 报错或结果奇怪 | 算法要求频带边界从小到大排列 | 检查 bands 列表是否单调递增 |
| numtaps 太小但过渡带设得很窄 | 通带纹波很大、阻带衰减不够 | 阶数不足以支撑窄过渡带 | 增大 numtaps，或放宽过渡带 |
| weight 全设为 1 但需要很深阻带 | 阻带衰减不够（如只有 -30dB） | 算法把精度平分了 | 增大阻带的 weight，如 `[1, 10]` |
| 忘记传 `fs`，bands 按 Hz 写 | 滤波器完全不对 | bands 被当作 Nyquist 归一化值 | 加上 `fs=你的采样率` |
| 用 remez 做一个 firwin 就能干的事 | 代码变复杂但效果差不多 | remez 的优势没发挥出来 | 参考上面的决策指南 |
| numtaps 是偶数 | remez 报错 | FIR 线性相位要求奇数长度 | 确保 numtaps 是奇数 |

## 7. 本章要点速查

| 概念 | 一句话解释 | 金融/电力类比 |
|------|-----------|-------------|
| 等波纹 | 通带和阻带内的最大误差相等——Remez 最优的标志 | 所有交易日的最大回撤被控制在同一水平 |
| `bands` | 频带边界列表，相邻两个定义一个频带的起止 | 定义"哪些信息保留、哪些丢弃"的规则书 |
| `desired` | 每个频带的目标增益（1=通过，0=阻挡，中间值=衰减） | 基金经理的"信息处理策略" |
| `weight` | 每个频带的精度权重（越大=算法越努力压该带误差） | 对不同类型的错误分配不同的惩罚权重 |
| 过渡带 | 通带和阻带之间未定义的频率区域 | "灰色地带"——这部分频率既不是完全通过也不是完全阻挡 |
| 最小相位 FIR | 把延迟降到最低，代价是相位不再线性 | 用信息的轻微失真换更快的响应速度 |
| `remez` vs `firwin` | `firwin`=日常轿车，`remez`=赛道跑车 | OLS vs Ridge/Lasso 的关系 |
