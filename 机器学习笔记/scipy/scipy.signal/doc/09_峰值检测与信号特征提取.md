# 第9章 · 峰值检测与信号特征提取

---

## 0. 你为什么要学这个？

### 0.1 前八章的回顾

你已经掌握了信号的生成、滤波、频谱分析、时频分析。这些都是**全局视角**——看整个信号是什么样的。但很多实际问题的答案是**局部**的：

- "股票在这个月里有几次超过 5% 的涨幅？" → 你在找峰值
- "这个心电图里有多少次心跳？" → 你在找峰值之间的间隔
- "电力负荷一天有几个高峰？" → 你在找峰的位置和强度
- "这个传感器数据哪里出现了异常冲击？" → 你在找不寻常的峰

峰值检测做的事情是：**从连续的信号中，找出有意义的"事件"——峰、谷、突变点。** 这是从"信号"到"可以计数的、可以统计的离散事件"的桥梁。

### 0.2 这章要回答的问题

- 怎样自动找到信号中的峰值？哪些峰是"真正的峰"、哪些只是噪声的随机凸起？
- 怎样描述一个峰——它有多高、多突出、多宽？
- 当信号噪声很大时，怎样避免把噪声误检为峰？
- 除了峰，还有哪些值得提取的局部特征？

### 0.3 和你的统计学知识对接

| 统计学概念 | 信号处理对应 |
|-----------|-------------|
| 找分布的众数 (mode) | 找信号的峰值位置 |
| 分位数 (quantile) | 峰的高度阈值——只关心超过某个水平的峰 |
| 异常值检测 (outlier detection) | 找异常突出的峰（用 prominence） |
| 核密度估计的带宽 | 峰检测的平滑窗口——控制"多宽的凸起算一个峰" |
| 描述性统计 | 峰的高度、宽度、间距的分布 |

## 场景一："我想从数据里自动找出所有的峰"

### 1.1 最简单的情况

你有一组数据——比如一天的电力负荷，每15分钟一个点。你肉眼能看到两个峰（上午和晚上各一个用电高峰）。你想让程序自动找到它们。

`find_peaks` 是 scipy 中做这件事的核心函数：

```python
from scipy.signal import find_peaks

peaks, properties = find_peaks(load_data)
# peaks: 峰值位置的索引数组
# properties: 字典，包含每个峰的属性（高度、突出度、宽度等）
```

不传任何额外参数时，`find_peaks` 会用非常宽松的标准——基本上每个"局部凸起"都会被当作一个峰，包括噪声的微小波动。**这是它的默认行为，不是你想要的。**

真正的能力在于**通过参数定义"什么样的峰才算峰"**。

### 1.2 `find_peaks` 的五个核心参数

#### `height` — 峰必须达到多少高度

```python
peaks, _ = find_peaks(data, height=0.5)      # 必须 ≥ 0.5
peaks, _ = find_peaks(data, height=(0.3, 0.8))  # 必须在 0.3~0.8 之间
```

最直观的参数。低于门限的峰直接忽略。如果你知道"有用的信息至少出现在 0.5 以上"，就设这个。

#### `distance` — 峰之间至少隔多远

```python
peaks, _ = find_peaks(data, distance=10)  # 相邻峰至少隔10个采样点
```

如果你是做心率检测（ECG），知道两跳心跳至少隔 0.3 秒，采样率 100Hz → distance=30。这可以防止一次心跳的上升沿和下降沿被分别检测为两个峰。

#### `prominence` — 峰有多"突出"

**这是最重要也最容易误解的参数。** 我多花一些篇幅来解释它。

prominence（突出度）衡量的是：**这个峰比它周围的地形高出多少。**

具体来说：从峰顶出发，向两边找"下坡路"，一直下降到不再需要爬坡就能到达一个更高的峰为止。prominence = 峰顶高度 - 下降过程中遇到的最低点。

**直觉理解**：想象你在往这个山谷里灌水。水位不断上升。当水位上升到这样一个高度——"这个峰"和"旁边更高的峰"之间的谷地被水淹没，两个峰变成了同一个岛——这个高度差就是 prominence。

- prominence 大 = 这个峰是一个孤立的高峰，和周围格格不入
- prominence 小 = 这个峰只是主峰旁边的一个小凸起，稍微降一点就和主峰连成一片了

```python
peaks, properties = find_peaks(data, prominence=1.0)  # 突出度 ≥ 1.0
```

**prominence 是最实用的参数**，因为它天然区分了"真正的峰"和"噪声的小凸起"。噪声凸起的 prominence 通常很小——它稍微波动一下就回到原来的水平。真正的峰 prominence 大——要下降很多才能脱离它。

#### `width` — 峰有多宽

```python
peaks, properties = find_peaks(data, width=5)      # 宽度 ≥ 5 个采样点
peaks, properties = find_peaks(data, width=(3, 20))  # 宽度在 3~20 之间
```

宽度是在峰的"半高"处（或其他相对高度）测量的水平距离。宽峰 = 缓慢的波动（如趋势变化），窄峰 = 快速的事件（如冲击脉冲）。`width` 可以帮你区分这两类。

默认在峰高的 50%（半高宽）处测量。可以通过 `rel_height` 参数调整。

#### `threshold` — 峰和邻域的"对比度"

```python
peaks, _ = find_peaks(data, threshold=0.2)
```

峰顶和它紧邻的谷底之间的垂直距离必须 ≥ 0.2。**这和 prominence 容易混淆**——区别在于：
- threshold：只看紧邻的谷底（局部对比度）
- prominence：看"整个周围地形中最低的点"（全局对比度）

通常用 prominence 就够了。threshold 在你想排除"虽然突出但紧挨着一个深谷的峰"时有额外作用。

### 1.3 返回的 properties

```python
peaks, props = find_peaks(data, prominence=1.0, width=3)

print(props.keys())
# dict_keys(['prominences', 'left_bases', 'right_bases',
#            'widths', 'width_heights', 'left_ips', 'right_ips'])

# 每个属性的长度 = len(peaks)
print(props['prominences'])   # 每个峰的突出度
print(props['widths'])        # 每个峰的宽度
print(props['left_bases'])    # 每个峰左基底的索引
```

### 1.4 完整的调参思路

拿到数据后，按这个顺序调：

1. **先设 `prominence`**——这是最重要的参数。从宽松开始（比如设一个很小的值），逐步收紧直到你认可的结果。先不管别的参数。

2. **再加 `distance`**——如果同一个事件被检测到了多次（比如一次心跳检测到了两个峰），用 distance 限制。

3. **需要时加 `height`**——如果你有先验知识，"有用的信号至少/至多在这个范围内"。

4. **最后用 `width` 区分峰的类型**——宽峰和窄峰可能是不同物理原因造成的。

## 场景二："我用 prominence 还是搞不定——噪声太大了"

### 2.1 问题

当信号噪声很大时，即使设了合理的 prominence，也可能有噪声凸起侥幸超过门限，或者真正的峰被噪声掩盖而达不到门限。你需要比"逐个点比较"更鲁棒的方法。

### 2.2 `find_peaks_cwt` — 基于小波的峰检测

```python
from scipy.signal import find_peaks_cwt

peaks = find_peaks_cwt(data, widths=np.arange(1, 20))
```

这个方法不是逐点比较，而是：**用不同宽度的小波（Ricker/墨西哥帽）去卷积信号，在多个尺度上找一致的峰值。**

直觉上：一个"真正的峰"应该在不同尺度（不同分辨率）下都能被检测到。一个噪声凸起可能在小尺度（精细分辨率）下看起来像峰，但在大尺度（粗糙分辨率）下就消失了。`find_peaks_cwt` 利用了这一点——只在多个尺度上都被检测到的位置才报告为峰。

**优缺点**：
- 优点：对噪声更鲁棒，不需要手动调 prominence
- 缺点：计算更慢，返回结果只有位置没有属性（不像 `find_peaks` 返回丰富的 properties）
- `widths` 参数决定了"什么宽度的凸起算峰"——类似 bandpass 滤波器，只关注某个宽度范围的峰

### 2.3 什么时候用哪个？

| 场景 | 用 |
|------|-----|
| 信号噪声不大、你需要峰的属性（突出度、宽度等） | `find_peaks` |
| 信号噪声大、你只关心"峰在哪里" | `find_peaks_cwt` |
| 你需要细粒度控制（如排除太近的峰、限制高度范围） | `find_peaks` |
| 峰的形状不标准（不是对称的高斯峰） | `find_peaks_cwt`（小波卷积对形状更宽容） |

## 场景三："我不想找峰，我想找谷" 或 "我想找所有的局部极值"

### 3.1 `argrelmin` / `argrelmax` — 找所有局部极值

```python
from scipy.signal import argrelmax, argrelmin, argrelextrema

# 所有局部最大值的位置
max_positions = argrelmax(data, order=3)[0]

# 所有局部最小值的位置
min_positions = argrelmin(data, order=3)[0]

# 通用接口：局部极值
extrema = argrelextrema(data, np.greater, order=3)[0]   # 局部最大
extrema = argrelextrema(data, np.less, order=3)[0]      # 局部最小
```

`order` 参数：在一个长度为 order 的窗口内比较——当前点必须比它左右各 order 个点都大（或都小）才算极值。order 越大 = 检测到的极值越少 = 只保留"宏观"的极值。

**和 `find_peaks` 的区别**：
- `argrelmax` 找**所有**的局部最大值，没有 prominence/height 等筛选机制
- `find_peaks` 在此基础上加了丰富的筛选条件
- 通常 `find_peaks` 更适合实际应用，`argrelmax` 在需要所有极值位置（不做筛选）时有快速优势

### 3.2 信号包络提取

有时候你不需要每个单独的峰，而是需要信号的"包络线"——连接所有峰顶的平滑曲线，或是连接所有谷底的曲线。

包络的作用：
- 提取信号的"幅度调制"——信号的整体强度在怎么变化
- 去趋势——用包络线作为上下边界，观察信号在边界内的填充情况
- 故障检测——包络的突然变化往往指示系统状态变化

**简单做法**：找到所有峰的位置，用插值连接它们。

```python
peaks, _ = find_peaks(signal, distance=20)
peak_envelope = np.interp(np.arange(len(signal)), peaks, signal[peaks])
```

## 4. 本章要点速查

| 概念 | 一句话 |
|------|--------|
| `find_peaks` | scipy 的峰检测核心——通过参数定义"什么样的峰才算峰" |
| `prominence` | **最重要的参数**——峰比周围地形高出多少。天然区分真峰和噪声 |
| `height` | 峰的绝对高度门槛——最有先验信息时使用 |
| `distance` | 防止同一事件被检测为多个峰 |
| `width` | 峰的宽度——区分宽波段（趋势变化）和窄峰（冲击事件） |
| `threshold` | 峰和紧邻谷底的垂直距离——局部对比度 |
| `find_peaks_cwt` | 小波峰检测——噪声大时的鲁棒选择 |
| `argrelmax` / `argrelmin` | 找所有局部极值（不做筛选） |
| 包络 | 连接峰顶（或谷底）的曲线——描述信号的整体"轮廓" |
