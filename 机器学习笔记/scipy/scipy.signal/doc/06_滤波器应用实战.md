## 场景一："我想对一批历史数据做趋势提取"

### 1.1 你遇到的情况

你有一年的日度股票收盘价（约 252 个数据点），已经全部拿到了。你想从中提取"中长期趋势"——去掉日度噪声，保留真正的方向性变化。这是离线分析，所有数据都在你手里，没有实时性要求。

### 1.2 你会怎么做？

你首先想到的可能是第 3 章学的——设计一个低通 FIR 滤波器，然后用 `lfilter` 去施加：

```python
b = firwin(51, cutoff=12.6, fs=252)   # 保留 >20 天的趋势
trend = lfilter(b, [1], price)         # 因果滤波
```

`lfilter` 是因果滤波器——它计算每个输出时，只依赖"当前和过去"的输入。这意味着滤波后的趋势线会**落后于原始价格**。这个延迟有多大？对于 51 阶的 FIR 滤波器，延迟 = (51-1)/2 = 25 天。你今天看到的"趋势值"实际上反映的是 25 天前的市场状态。

**这对很多分析场景是个问题**。比如你想比较"趋势线在什么时候发生了拐点"，延迟会让所有拐点都晚 25 天才被你看到。

### 1.3 解决办法：`filtfilt` — 零相位滤波

`filtfilt` 解决了延迟问题。它的原理不复杂：

1. 正常做一次滤波（有延迟）
2. 把结果翻转（最后一个点变成第一个）
3. 再滤波一次
4. 再翻转回来

两次滤波的延迟恰好相互抵消——输出和输入完美对齐。

**代价是什么？**

第一，**边界效应**。`filtfilt` 在处理信号两端时需要假设"信号外面是什么"。默认用奇数对称延拓（把边界当镜子反射），这个假设不一定准确。所以滤波后，**开头和结尾各约 3×滤波器阶数个点不可靠**——在进一步分析前应该丢弃或标记为 NaN。

第二，**等效幅度响应是原始滤波器的平方**。你设计了 -3dB 截止频率 = 10 Hz 的滤波器，但 `filtfilt` 施加后，实际 -3dB 截止频率会偏移到更低的位置（大约 8 Hz）。因为两次滤波的级联让衰减加倍。

第三，**不能用于实时场景**。需要整个信号在手里才能做翻转。

```python
# 离线趋势提取的标准姿势
b = firwin(51, cutoff=12.6, fs=252)
trend = filtfilt(b, [1], price)

# 丢弃不可靠的边界（3 × 滤波器长度 ≈ 150 点）
trend[:150] = np.nan
trend[-150:] = np.nan
```

### 1.4 什么时候该用 `lfilter`？

`lfilter` 虽然会引入延迟，但它是**因果的**——不依赖未来数据。这种"纯因果"的特性在某些应用中反而是你需要的：
- 你正在做策略回测，模拟的正是"在只知道过去数据的情况下做决策"——`lfilter` 恰好符合这个场景
- 你需要评估"如果我只用过去信息，能多好地预测趋势"——`lfilter` 的输出就是真实可用的预测

## 场景二："我有一路实时数据流，需要在线处理"

### 2.1 你遇到的情况

你有一个实时数据源——比如股票逐笔成交、电力负荷每秒更新、或者传感器每分钟回传一次。数据是**一个接一个（或一批接一批）到达**的，你不能等全部数据收齐了再处理。你需要边收边处理，且处理结果要立即可用。

### 2.2 核心问题：滤波器有"记忆"

滤波器不是无状态的操作。一个 4 阶 IIR 滤波器需要记住"过去 4 个输入和过去 4 个输出"才能正确计算下一个输出。如果你每收到一批新数据就重新初始化，滤波器会丢掉了之前积累的信息——结果就是每批数据的开头都会出现一个**启动瞬态**：滤波器从零状态重新"热身"。

**直观类比**：这就像开车时每到一个红绿灯就重新点火发动——发动机每次都要从怠速状态重新拉升，效率极低且不稳定。你真正想要的是在绿灯亮起时发动机已经在巡航转速上。

### 2.3 正确做法：`sosfilt` + `zi` 状态传递

```python
from scipy.signal import butter, sosfilt, sosfilt_zi

# 1. 设计滤波器（SOS 格式）
sos = butter(4, 10, btype='low', fs=100, output='sos')

# 2. 计算稳态初始条件（消除启动瞬态）
zi = sosfilt_zi(sos) * first_data_value

# 3. 逐批处理，保持状态
for chunk in data_stream:
    output_chunk, zi = sosfilt(sos, chunk, zi=zi)
    # zi 自动更新为这批数据结束时的状态
    # 下一批数据会从当前状态接续下去
```

**`sosfilt_zi` 做了什么？** 它计算了这样一个初始状态：假设滤波器一直收到常数输入（等于初始值），它的输出早已稳定，不会有任何瞬态波动。用这个状态作为起点，滤波器从一开始就在"巡航模式"，不会出现从零开始的热身过程。

**为什么用 SOS 而不是 (b,a)？** 第 5 章已经讲过——高阶 IIR 的 (b,a) 格式有数值精度隐患。SOS 把高阶滤波器拆成多个二阶滤波器的串联，每个子滤波器的系数范围都很小，不存在精度问题。**任何 IIR 滤波都应该用 SOS 格式。** 这不仅是"推荐"，在阶数 > 6 时几乎是"必须"。

## 场景三："我对分析精度要求很高，尤其关心信号的精确时间点"

### 3.1 你遇到的情况

你正在分析一批电网频率数据，需要精确知道"频率扰动发生在哪个时刻"。或者你在分析 EEG 脑电数据，需要判断"事件相关电位出现在刺激后的第几毫秒"。**时间精度至关重要。**

### 3.2 问题：因果滤波会"移动"事件的时间位置

因果滤波器（`lfilter`、`sosfilt`）会给每个频率成分施加不同的时间延迟——低频延迟小、高频延迟大（取决于滤波器类型）。一个"尖峰"事件经过滤波后，不仅形状变圆滑了，**出现的位置也偏移了**。而且不同频率的偏移量不同——这就扭曲了波形的整体形状。

### 3.3 解法：`sosfiltfilt` — 离线零相位

```python
sos = butter(4, [8, 12], btype='band', fs=250, output='sos')
clean = sosfiltfilt(sos, eeg_signal)

# 丢弃边界
margin = 3 * 4 * 2  # 3 × 滤波阶数 × 2
clean[:margin] = np.nan
clean[-margin:] = np.nan
```

### 3.4 额外代价：幅度响应翻倍

这个代价在场景一中已经提到，但对精度敏感的场景尤其要注意：`filtfilt` 的等效幅度响应是原始滤波器的**平方**。

如果你设计了一个通带纹波 0.5 dB 的滤波器，`filtfilt` 后变成 1 dB。如果你设计了一个 -40 dB 阻带衰减的滤波器，`filtfilt` 后变成 -80 dB。如果你设计截止频率为 10 Hz，`filtfilt` 后实际截止更接近 8 Hz。

**实战建议**：如果截止频率的精确位置很重要，用 `filtfilt` 时把截止频率设高 15-20% 来补偿。

## 场景四："我想把高频采样的数据降采样到低频"

### 4.1 你遇到的情况

你有一份 15 分钟间隔的电力负荷数据（每天 96 个点）。你想把它聚合到小时级别（每天 24 个点），方便和别的小时级数据对齐做分析。或者你想把 1000 Hz 采样的传感器数据降到 100 Hz 来节省存储。

你的第一反应可能是直接用 `pandas.resample()` 或 `np.mean()` 对每个小时取平均。但这里面藏着一个陷阱。

### 4.2 问题：降采样 = 降低 Nyquist 频率 = 混叠风险

15 分钟采样的 Nyquist 频率 = 48 次/天（即 2 次/小时）。如果你降到小时级，新 Nyquist = 12 次/天（0.5 次/小时）。如果原始数据中有频率超过新 Nyquist 的成分——比如 3 次/小时（对应 20 分钟的周期波动）——它们在新 Nyquist 下会被**混叠**成假的低频信号。第 1 章演示过这个现象。

**降采样前，必须先做低通滤波**——把超过新 Nyquist 频率的成分全部去掉。这个操作叫**抗混叠滤波**。

### 4.3 正确做法

```python
from scipy.signal import decimate

# decimate 自动做了抗混叠滤波 + 降采样
load_hourly = decimate(load_15min, q=4, n=30, ftype='fir')
# q=4: 每4个点取1个（96→24）
# n=30:  抗混叠 FIR 滤波器的阶数
# ftype: 用 FIR（默认推荐，避免 IIR 的相位问题）

# 或者手动做
from scipy.signal import upfirdn
h = firwin(64, cutoff=0.22)   # 抗混叠低通，截止<新Nyquist
load_hourly = upfirdn(h, load_15min, up=1, down=4)
```

**`decimate` vs 手动 `upfirdn`**：
- `decimate` 是专门做降采样的，自动设计抗混叠滤波器 → 最简单
- `upfirdn` 更通用——可以升采样、降采样、或有理数倍率重采样 → 灵活

## 场景五："我对信号做了一系列处理后，结果发散了"

### 5.1 你遇到的情况

你做了一个 8 阶 Butterworth 带通滤波器，用 `lfilter(b, a, data)` 去施加，结果信号出现不合理的放大或振荡。检查了滤波器设计，频率响应看起来完全正常。

### 5.2 问题：高阶 IIR 的数值精度灾难

这是陷阱中的陷阱——**你设计的滤波器在理论上是完全正确的，但在计算机浮点运算中出了问题。**

`lfilter` 接收的 `a` 系数（反馈系数）可能跨越 10¹⁰ 甚至更大的范围。双精度浮点数只有约 15-16 位十进制有效数字。当系数范围极大时，有效位数不够用，微小的舍入误差在反馈回路中被反复放大，最终导致输出偏离理论值甚至发散。

**一个经验法则**：对于 `butter`（Butterworth IIR），N ≤ 4 通常安全；N 在 6-8 时开始有风险；N > 10 时几乎一定会出问题。

### 5.3 解决办法：永远用 SOS 格式

```python
# ❌ 可能有风险的做法
b, a = butter(8, 10, btype='low', fs=100)
y = lfilter(b, a, data)

# ✅ 安全的做法
sos = butter(8, 10, btype='low', fs=100, output='sos')
y = sosfilt(sos, data)
```

SOS 把 8 阶滤波器分解成 4 个二阶滤波器串联。每个子滤波器的系数范围很小（1-2 个数量级），不存在精度问题。而且 `sosfilt` 的计算速度也更快（因为它利用了二阶节的特殊结构）。

如果手头已经有 `(b, a)` 格式的滤波器（比如从别的地方拿到的），可以用 `tf2sos` 转换：

```python
from scipy.signal import tf2sos
sos = tf2sos(b, a)
y = sosfilt(sos, data)
```

## 6. 选择指南：一个场景一个问题

| 你的场景 | 用什么函数 | 一句话原因 |
|---------|----------|----------|
| 离线分析历史数据，需要无延迟对齐 | `sosfiltfilt` | 零相位无延迟，注意丢弃边界和幅度翻倍 |
| 实时数据流，逐批到达 | `sosfilt` + `zi` | 状态在批次间传递，消除启动瞬态 |
| 模拟"仅用过去信息"的回测 | `sosfilt` | 因果性恰好是你需要的 |
| 降采样（如 15分钟→小时） | `decimate` 或 `upfirdn` | 必须先做抗混叠滤波再降采样 |
| IIR 阶数 ≥ 6，或结果不对劲 | 检查是否用了 `sos` 格式 | 大概率是 (b,a) 格式的数值问题 |
| FIR 滤波器，简单平滑 | `lfilter(b, [1], x)` | FIR 没有反馈，不存在数值问题 |

## 7. 本章要点速查

| 概念 | 一句话 |
|------|--------|
| `lfilter` | 因果滤波——只依赖过去数据，有延迟，可实时 |
| `filtfilt` | 零相位滤波——前向+反向消除延迟，不可实时，幅度翻倍 |
| `sosfilt` | SOS 格式的因果滤波——IIR 的标准姿势 |
| `sosfiltfilt` | SOS 格式的零相位滤波——离线 IIR 的首选 |
| `zi` | 滤波器内部状态——实时处理的"记忆" |
| `sosfilt_zi` | 计算稳态初始条件——消除启动瞬态 |
| `tf2sos` | (b,a)→SOS 转换——拯救数值精度 |
| `decimate` | 带抗混叠保护的降采样 |
| `upfirdn` | 通用重采样——升采样+滤波+降采样一体 |
