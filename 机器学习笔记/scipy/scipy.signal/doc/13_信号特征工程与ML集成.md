### 0.2 信号特征 vs 传统 ML 特征

传统 ML 特征 = 从表格数据中选列（价格、成交量、市值...）。  
信号特征 = 从信号的**结构**中提取信息——它的趋势怎么样、周期成分有多强、峰的密度、频谱的形状...

**类比**：传统特征像是问"这个人多高多重"，信号特征像是问"这个人的走路姿势是什么样的"——后者捕捉的是**动态模式**。

## 1. 信号特征分类体系

### 1.1 时域特征（第2、9章）

直接从信号的时间序列中提取。你已经全部学过。

| 特征 | 公式 / 来源 | 描述什么 |
|------|-----------|---------|
| 均值 | `np.mean(x)` | 信号的中心位置 |
| 标准差 | `np.std(x)` | 信号的波动幅度 |
| RMS | `sqrt(mean(x²))` | 信号的有效能量 |
| 偏度 | 三阶标准化矩 | 分布的不对称性 |
| 峭度 | 四阶标准化矩 | 分布的尾部厚度——尖峰/厚尾 |
| 峰值因子 | `max(abs(x))/RMS` | 是否存在异常尖峰 |
| 过零率 | 符号变化次数/N | 信号的振荡频率——粗略的"频率感" |
| 峰的数量 | `find_peaks` | 信号中事件的密度 |
| 峰的平均突出度 | `prominence` 均值 | 事件的平均显著程度 |
| 峰间距的变异系数 | 峰间距 std/mean | 事件节奏的规律性 |

### 1.2 频域特征（第7章）

从信号的频谱中提取。

| 特征 | 来源 | 描述什么 |
|------|------|---------|
| 谱质心 | `sum(f × PSD) / sum(PSD)` | 频谱的"重心"——主频率在哪 |
| 谱带宽 | `sqrt(sum((f - centroid)² × PSD) / sum(PSD))` | 频率分布的宽度 |
| 频带能量比 | 低频段能量 / 总能量 | 低频成分占多大比重 |
| 主频率 | 最高峰对应的频率 | 最强的周期成分 |
| 主频率的幅度 | 最高峰的 PSD 值 | 主周期的强度 |
| 频谱熵 | `-sum(p × log(p))`，p 归一化 PSD | 频谱的"复杂度"——越平越复杂 |

### 1.3 时频特征（第8章）

当频率成分随时间变化时，从频谱图/小波变换中提取。

| 特征 | 来源 | 描述什么 |
|------|------|---------|
| 频谱图各频带的时序方差 | STFT | 频率成分的稳定性 |
| 瞬时频率的变异系数 | Hilbert（第10章）| 信号频率的稳定程度 |
| 瞬时频率的趋势 | Hilbert 瞬时频率的线性拟合 | 频率是否在漂移（如 chirp） |
| 包络的峰度和偏度 | Hilbert 包络 | 幅度调制的分布形状 |

### 1.4 非线性特征（扩展）

需要额外库（如 `nolds`、`antropy`），但概念本身值得了解：

| 特征 | 描述什么 |
|------|---------|
| 样本熵 (Sample Entropy) | 信号的复杂度和不可预测性——类似"信号的 AIC" |
| 近似熵 | 同上，但更适短序列 |
| 分形维度 | 信号的"粗糙程度" |
| Hurst 指数 | 长期记忆性——H>0.5=趋势持续，H<0.5=均值回归 |

这些非线性特征对于金融和生物医学信号特别有区分力。

## 2. 滑动窗口特征提取

### 2.1 核心问题

你不是对整条信号提一次特征，而是用**滑动窗口**——每个窗口提一组特征，窗口在信号上滑动，产生一个"特征序列"。这就是把一维信号转化为 ML 可以用的特征矩阵。

```
信号:  ─────────────────────────────────
        窗口1                    →  特征向量1
           窗口2                 →  特征向量2
              窗口3              →  特征向量3
                  ...

特征矩阵 X:  N_windows × N_features
标签 y:     N_windows × 1
```

### 2.2 三个关键参数

**窗口长度 (window_size)**：用多少个历史数据点。太长 → 特征迟钝（变化被平均掉）。太短 → 特征抖动（统计不可靠）。

**步长 (stride)**：窗口每次滑动多远。stride=1 → 密集采样（相邻窗口高度重叠），stride=window_size → 不重叠。

**边界处理**：开头 window_size-1 个点没有足够的历史。通常丢弃或填 NaN。

### 2.3 特征提取管线的正确实现

```python
def extract_window_features(signal, window_size, stride):
    features = []
    for start in range(0, len(signal) - window_size, stride):
        window = signal[start:start+window_size]

        feat = {}
        # 时域
        feat['mean'] = np.mean(window)
        feat['std'] = np.std(window)
        feat['skew'] = np.mean((window - feat['mean'])**3) / feat['std']**3
        feat['kurt'] = np.mean((window - feat['mean'])**4) / feat['std']**4

        # 频域（对该窗口做 Welch）
        freqs, psd = welch(window, fs=fs, nperseg=min(64, len(window)//4))
        feat['spectral_centroid'] = np.sum(freqs * psd) / np.sum(psd)
        feat['dominant_freq'] = freqs[np.argmax(psd)]

        # 峰特征
        peaks, props = find_peaks(window, prominence=np.std(window)*0.5)
        feat['n_peaks'] = len(peaks)
        feat['mean_prominence'] = np.mean(props['prominences']) if len(peaks) > 0 else 0

        features.append(feat)

    return pd.DataFrame(features)
```

## 3. 和 scikit-learn 的集成

### 3.1 构建 `Transformer`，嵌入 Pipeline

信号特征提取可以用 `sklearn` 的 `Transformer` 封装，然后和分类器/回归器一起放入 `Pipeline`：

```python
from sklearn.base import BaseEstimator, TransformerMixin
from sklearn.pipeline import Pipeline
from sklearn.ensemble import RandomForestClassifier

class SignalFeatureExtractor(BaseEstimator, TransformerMixin):
    def __init__(self, window_size=128, stride=64, fs=1.0):
        self.window_size = window_size
        self.stride = stride
        self.fs = fs

    def fit(self, X, y=None):
        return self

    def transform(self, X):
        # X 是 (n_samples, signal_length) 的数组
        features_list = []
        for i in range(X.shape[0]):
            signal = X[i]
            feats = extract_window_features(signal,
                                            self.window_size,
                                            self.stride)
            features_list.append(feats)
        return np.array(features_list)

# 管线
pipeline = Pipeline([
    ('features', SignalFeatureExtractor(window_size=128, fs=100)),
    ('classifier', RandomForestClassifier(n_estimators=100))
])

pipeline.fit(X_train_signals, y_train)
predictions = pipeline.predict(X_test_signals)
```

### 3.2 为什么封装成 Transformer？

1. **避免数据泄漏**：特征提取在 `fit`/`transform` 框架内，交叉验证时自动处理
2. **参数搜索**：`GridSearchCV` 可以同时搜索信号处理参数和模型参数
3. **可复现**：Pipeline 一次性保存和加载，部署方便

## 4. 端到端案例：时序分类

### 4.1 场景

你有 200 条 10 秒长的传感器记录（每条 1000 点 @ 100Hz）。50 条是"正常状态"，50 条是"状态A异常"，50 条是"状态B异常"，50 条是"状态C异常"。你想训练一个分类器来判断新信号属于哪种状态。

### 4.2 处理流程

```
原始信号 (200条 × 1000点)
    │
    ├── 预处理（第11章）
    │    去趋势 → 去噪（S-G） → 归一化
    │
    ├── 滑动窗口特征提取（第2/7/9章）
    │    每条信号 → 多个窗口 → 每个窗口提时域+频域+峰特征
    │
    ├── 训练/测试划分
    │
    ├── 分类器训练（Random Forest / XGBoost / SVM）
    │
    └── 评估（混淆矩阵、特征重要性）
```
