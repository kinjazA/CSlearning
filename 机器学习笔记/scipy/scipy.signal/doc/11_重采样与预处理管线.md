# 第11章 · 重采样与预处理管线

---

## 0. 为什么需要这一章？

前10章你学会了设计滤波器、分析频谱、检测峰值。但有一个问题一直悬着：**真实世界的数据很少"开箱即用"。**

你拿到的数据可能是：
- 15分钟间隔的电力负荷，但你想做小时级的分析 → 需要降采样
- 两个传感器，一个 100Hz、一个 256Hz → 需要统一采样率
- 股票价格有长期上涨趋势 → 在做周期分析之前需要去趋势
- 不同量纲的信号（温度和湿度）→ 需要标准化

这一章解决的就是这些问题——**把原始信号变成"可以直接分析"的状态。** 这是一个完整的预处理管线。

## 场景一："我想把高频数据降到低频"

### 1.1 你已经知道的问题

第6章（场景四）已经演示过降采样前必须做抗混叠滤波。这一章展开来讲具体的函数和参数。

### 1.2 `decimate` — 最简单的降采样

```python
from scipy.signal import decimate

y = decimate(x, q=4, n=30, ftype='fir')
# q:    降采样倍数（每q个点取1个）
# n:    抗混叠 FIR 滤波器的阶数（越大 → 过渡带越窄）
# ftype: 'fir'（默认推荐）或 'iir'（阶数低但相位非线性）
```

`decimate` 内部做了两件事：
1. 设计一个截止频率为 `fs/(2*q)` 的低通滤波器（即新 Nyquist 频率）
2. 先滤波再降采样

所以你不需要手动设计抗混叠滤波器——这是最安全、最省心的降采样方式。

**参数选择**：
- `q`：必须是整数。从 96 点/天 到 24 点/天 → q=4
- `n`：FIR 阶数。越大滤波越干净但计算越慢。默认 30 够用，如果信号在截止频率附近有强能量 → 增大到 50-60
- `ftype='fir'`：推荐。相位线性，波形不扭曲。`'iir'` 阶数低但相位非线性（参考第5章 IIR vs FIR 的讨论）

### 1.3 `resample_poly` — 非整数倍的降采样

如果降采样倍数不是整数（如从 256Hz 降到 100Hz = 2.56 倍），用 `resample_poly`：

```python
from scipy.signal import resample_poly

y = resample_poly(x, up=25, down=64, window=('kaiser', 5.0))
# 256 Hz × 25/64 = 100 Hz
```

它的思路：先升采样 up 倍（插入零值），再降采样 down 倍，中间用 FIR 滤波器来消除升采样引入的镜像频率和降采样前的抗混叠。

### 1.4 `resample` — 基于 FFT 的重采样

```python
from scipy.signal import resample

y = resample(x, num=500)
# 用 FFT 方法将信号重采样到 num 个点
```

原理：对信号做 FFT，在频域截断或补零，再逆 FFT 回来。**优势是简单快速，缺点是假设信号是周期性的**（想想第7章的频谱泄漏——如果首尾不匹配会引入伪影）。

**选择指南**：

| 场景 | 用什么 | 原因 |
|------|-------|------|
| 整数倍降采样 | `decimate` | 最省心，自动抗混叠 |
| 非整数倍 | `resample_poly` | 支持有理数倍率 |
| 只是改点数（不在乎采样率） | `resample` | FFT 方法最快 |
| 需要同时升采样+降采样 | `upfirdn` | 最灵活 |

### 1.5 `upfirdn` — 终极灵活性

```python
from scipy.signal import upfirdn

h = firwin(64, cutoff=0.4)  # 自定义 FIR 滤波器
y = upfirdn(h, x, up=2, down=3)
```

`upfirdn(h, x, up, down)` 在一个调用中完成：升采样 up 倍 → FIR 滤波(h) → 降采样 down 倍。当 `decimate` 或 `resample_poly` 的默认行为不满足需求时（比如你需要自定义滤波器的形状），用这个。

## 场景二："我的信号有基线漂移，怎样去掉趋势？"

### 2.1 问题

你有一段时间序列——比如股票价格长期上涨，或者传感器读数因温度变化而缓慢漂移。在做频谱分析或周期检测之前，你需要去掉这个缓慢的趋势（也叫"去基线漂移"）。因为趋势在频域表现为极低频的大能量，会掩盖你真正关心的频率成分。

### 2.2 `detrend` — 去线性/常数趋势

```python
from scipy.signal import detrend

# 去均值（去掉 DC 分量）
y = detrend(x, type='constant')

# 去线性趋势（拟合一条直线然后减去）
y = detrend(x, type='linear')
```

`type='constant'` → 减去信号的均值。这和 `x - np.mean(x)` 等价——去掉零频率成分（DC 分量）。

`type='linear'` → 用最小二乘法拟合一条直线，然后减去。这和"去线性趋势"等价——去掉一阶趋势。

**什么时候用哪个？**
- 你在做 FFT 之前 → 至少做 `type='constant'`（否则 DC 分量的巨大能量会淹没频谱图）
- 你的数据有明显的上升或下降趋势（如股价、气温长期变化）→ 做 `type='linear'`
- 趋势不是线性的（如抛物线型漂移）→ 用高通滤波器（截止频率设得非常低）

## 场景三："多个信号量纲不同，怎样放在一起比较？"

### 3.1 问题

你有温度（-10~40°C）和湿度（0~100%）两个信号。如果你想看它们的频谱图放在同一个坐标系里，或者把它们同时送入一个模型，量纲差异是个问题。

### 3.2 标准化策略

```python
# Z-score 标准化：均值为0，标准差为1
y = (x - np.mean(x)) / np.std(x)

# Min-max 归一化：缩放到 [0, 1]
y = (x - np.min(x)) / (np.max(x) - np.min(x))

# RMS 归一化：除以 RMS 值，使能量归一
y = x / np.sqrt(np.mean(x**2))
```

**选择指南**：
- 频谱分析前 → Z-score 或 RMS 归一化（保留相对结构）
- 送入 ML 模型前 → Z-score（最通用）
- 需要保留原始范围的比例关系 → Min-max
- 关心信号的"形状"而非绝对大小 → RMS 归一化

## 场景四："怎样构建一个可复用的预处理管线？"

### 4.1 设计原则

一个完整的信号预处理管线应该：
1. **顺序可配置**：去趋势 → 滤波 → 重采样 → 归一化，但顺序在某些场景下可能需要调整
2. **参数可追溯**：每次处理记录用了什么参数，方便复现
3. **边界处理一致**：滤波和重采样都会引入边界效应，统一处理策略

### 4.2 典型管线

```python
class SignalPreprocessor:
    def __init__(self, fs):
        self.fs = fs
        self.steps = []

    def detrend(self, type='linear'):
        self.steps.append(('detrend', type))
        return self

    def lowpass(self, cutoff_period, numtaps=51):
        self.steps.append(('lowpass', cutoff_period, numtaps))
        return self

    def decimate(self, q):
        self.steps.append(('decimate', q))
        return self

    def normalize(self, method='zscore'):
        self.steps.append(('normalize', method))
        return self

    def process(self, x):
        for step in self.steps:
            if step[0] == 'detrend':
                x = detrend(x, type=step[1])
            elif step[0] == 'lowpass':
                cutoff_freq = self.fs / step[1]
                b = firwin(step[2], cutoff_freq, fs=self.fs)
                x = filtfilt(b, [1], x)
            elif step[0] == 'decimate':
                x = decimate(x, q=step[1])
                self.fs /= step[1]  # 更新采样率！
            elif step[0] == 'normalize':
                if step[1] == 'zscore':
                    x = (x - np.mean(x)) / np.std(x)
                elif step[1] == 'minmax':
                    x = (x - np.min(x)) / (np.max(x) - np.min(x))
        return x
```

**管线顺序的注意事项**：
- 去趋势通常放最前面——先去掉不关心的低频成分
- 如果先滤波再降采样，降采样后记得更新 `fs`（后续操作的频率计算基于新采样率）
- 归一化通常放最后——在所有物理量操作完成之后再缩放

## 5. 本章要点速查

| 概念 | 一句话 |
|------|--------|
| `decimate` | 整数倍降采样——自动抗混叠，最省心 |
| `resample_poly` | 非整数倍重采样——升采样 up 倍再降采样 down 倍 |
| `resample` | FFT 重采样——快但假设信号周期性 |
| `upfirdn` | 最灵活——自定义滤波器 + 任意升降倍数 |
| `detrend` | 去除均值（constant）或线性趋势（linear） |
| Z-score | 均值为0标准差为1——最通用的标准化 |
| 管线顺序 | 去趋势 → 滤波 → 降采样 → 归一化 |
