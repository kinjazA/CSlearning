# 第5章 · IIR 滤波器设计与应用

---

## 0. 从 FIR 到 IIR：为什么需要另一种滤波器？

### 0.1 第3-4章的回顾

前两章你学会了 FIR 滤波器。FIR 有两个核心优势：
- **天然稳定**（没有反馈回路，不会自激振荡）
- **可以做到线性相位**（所有频率延迟相同，波形不扭曲）

但 FIR 也有一个不可回避的代价：**要达到陡峭的过渡带，需要很长的滤波器。**

举个例子：用日度数据（fs=252 次/年），你想设计一个低通滤波器，过渡带从 10 次/年（周期 25 天）到 12 次/年（周期 21 天）。这个过渡带只有 2 次/年，非常窄。用 FIR：
- `firwin` 大约需要 numtaps ≈ 200 → 延迟 100 天
- `remez` 大约需要 numtaps ≈ 120 → 延迟 60 天

100 天的延迟意味着你今天看到的"趋势"反映的是 3 个多月前的情况。对于很多应用来说，这太慢了。

### 0.2 IIR 的核心思想：用反馈换效率

**IIR = Infinite Impulse Response = 无限脉冲响应。**

和 FIR 只依赖"有限个过去输入"不同，IIR 滤波器**把过去的输出也作为输入**——这形成了一个反馈回路。

```
FIR:  输出[n] = 只取决于 输入[n], 输入[n-1], ..., 输入[n-M]
IIR:  输出[n] = 取决于 输入[n], 输入[n-1], ... + 输出[n-1], 输出[n-2], ...
                   ───────────────   ──────────────────────────
                     前馈部分                反馈部分
```

**反馈的效果**：信号的一部分被"循环利用"。就像复利——利息计入本金再产生利息，一个小变化经过反馈可以产生持久的影响。这也是为什么 IIR 的脉冲响应是"无限长"的——一个输入脉冲会通过反馈回路一直循环下去（逐渐衰减但不完全归零）。

**用统计学类比**：
- FIR = MA 模型（只依赖有限个过去的白噪声项）
- IIR = ARMA 模型（依赖过去的观测值 + 过去的白噪声项）
- AR 部分的"自回归"就是反馈——今天的值取决于昨天的值，昨天的值又取决于前天的值……

### 0.3 IIR vs FIR 的对比

| 维度 | FIR | IIR |
|------|-----|-----|
| 要达到陡峭过渡需要的阶数 | 高（50-200+） | **低（4-10）** |
| 延迟 | 大（阶数/2） | **小** |
| 稳定性 | **天然稳定**（无反馈） | 可能不稳定（反馈回路可能自激） |
| 相位 | **线性相位**（波形不扭曲） | 非线性相位（不同频率延迟不同） |
| 设计复杂度 | 简单（窗口法） | 较复杂（模拟原型 → 数字化） |
| 数值精度 | 稳定 | 高阶时可能出问题 → **必须用 SOS 形式** |
| 典型应用 | 需要保真度的场景 | 实时系统、嵌入式、陡峭截止 |

**一句话决策**：
- 你关心波形形状保真 → FIR
- 你关心低延迟 + 计算效率 → IIR
- 你两者都需要 → IIR + `filtfilt`（零相位滤波，第6章会讲）

## 1. IIR 滤波器是怎么设计出来的？

### 1.1 总体思路：从模拟到数字

FIR 滤波器的设计是纯数字的——直接在离散域操作。IIR 滤波器的设计走了一条不同的路：

1. **先设计一个模拟滤波器**（连续时间域）——利用 20 世纪已经非常成熟的模拟滤波器设计理论
2. **再把它转换成数字滤波器**（离散时间域）——用双线性变换（Bilinear Transform）

这就好比你先在"理想世界"（连续域）里画好蓝图，再把它"搬运"到"数字世界"（离散域）。

### 1.2 为什么模拟滤波器理论这么成熟？

20 世纪前半叶，工程师们在模拟电路（电阻、电容、电感）中大量使用滤波器。他们发现了几种具有优美数学性质的滤波器原型，每种都有明确的公式和表格——你只需要指定阶数和截止频率，所有参数就都确定了。

这些经典原型至今仍然是数字 IIR 设计的基石。scipy 支持的四种经典原型是 Butterworth、Chebyshev I、Chebyshev II（也叫 Inverse Chebyshev）和 Elliptic（也叫 Cauer）。第五种 Bessel 在某些场景下也有用。

### 1.3 双线性变换和频率预畸变

把模拟滤波器转换成数字滤波器时，有一个关键的步骤叫**双线性变换**。它把模拟域的频率轴（从 0 到无穷大）压缩映射到数字域的频率轴（从 0 到 Nyquist）。

这个压缩不是线性的——**频率轴被弯曲了**。模拟域的高频部分被压缩到数字域的一个很窄范围内。

这意味着：如果你直接在数字域指定截止频率，转换回模拟域设计后再转回来，实际截止频率会和你的预期有偏差。**这就是频率预畸变（pre-warping）要做的事情**——在设计之前先"预扭曲"频率，使转换后的数字截止频率落在你真正想要的位置。

**好在 scipy 的 IIR 设计函数自动做了预畸变**，你不需要手动处理。但知道这个背景有助于理解为什么设计参数里总是出现 `fs`。

## 2. 四种经典 IIR 原型

每一种原型代表一种"允许通过 vs 大量衰减"之间的不同权衡。用同一阶数设计低通滤波器时：

| 原型 | 通带内的表现 | 阻带衰减 | 过渡带宽度 | 相位特性 |
|------|------------|---------|-----------|---------|
| **Butterworth** | **最平坦**——像镜面一样 | 中等 | 中等 | 中等（比 Chebyshev 好，比 Bessel 差） |
| **Chebyshev I** | 有等波纹（设定范围内振荡） | 好 | 较窄 | 较差 |
| **Chebyshev II** | 平坦 | **很好**（阻带有等波纹） | 较窄 | 较差 |
| **Elliptic** | 有等波纹 | **最好** | **最窄** | **最差** |
| **Bessel** | 平坦 | 较差 | 宽 | **最好**（最平坦的群延迟） |

### 2.1 Butterworth —— 最平坦，默认首选

**一句话**：在通带内做到极致平坦——完全没有纹波。

Butterworth 滤波器的设计哲学是"让通带尽可能平坦，不追求最陡的过渡带"。它的频率响应在通带内是一条平滑下降的曲线，没有任何振荡。

- **优点**：通带内最平（maximally flat）。对所有频率的衰减都很平滑，没有突变。相位特性在 IIR 中也算不错。
- **缺点**：要达到和 Chebyshev/Elliptic 同等的过渡带，需要更高的阶数。
- **什么时候选它**：当你想要一个"干净"的滤波结果，不想引入任何人为的频域波动时。这是**日常使用中最常用的 IIR 原型**。

**类比**：Butterworth 是 IIR 世界里的 `hamming` 窗——均衡、可靠、不冒进。

### 2.2 Chebyshev I —— 通带"可以容忍一些不平"

**一句话**：在通带内允许可控的纹波（振荡），换取更窄的过渡带。

Chebyshev I 的设计哲学是"只要通带纹波不超过我设定的上限（比如 1 dB），过渡带能多窄就多窄"。这个"通带纹波上限"在 scipy 中由 `rp` 参数（ripple in passband）控制。

- **优点**：过渡带比 Butterworth 明显更窄。同等阶数下阻带衰减更强。
- **缺点**：通带内不是平的——有可控但可见的振荡。相位非线性比 Butterworth 更明显。
- **`rp` 参数**：通带最大允许纹波（dB）。典型值 0.1~3 dB。rp 越小 → 越接近 Butterworth → 过渡带越宽。

**类比**：Chebyshev I 就像你在回归中允许了更大的 bias 来换更小的 variance——你接受通带内有一定失真，换取更强的频率选择性。

### 2.3 Chebyshev II —— 反过来，让阻带去振荡

**一句话**：通带平坦，阻带内允许等波纹——波纹出现在阻带而不是通带。

这和 Chebyshev I 反过来。通带内和 Butterworth 一样平坦，但阻带里有固定的等波纹振荡。阻带衰减有一个最小值（由 `rs` 参数指定），阻带内的波纹幅度不会超过它。

- **优点**：通带平坦 + 过渡带比 Butterworth 窄。
- **缺点**：阻带内的振荡意味某些频率的衰减比其他频率更浅（虽然都在 `rs` 以上）。
- **`rs` 参数**：阻带最小衰减（dB）。典型值 20~60 dB。rs 越大 → 过渡带越宽。

### 2.4 Elliptic —— 极致频率选择性

**一句话**：通带内也有纹波、阻带内也有纹波——两边都牺牲平整度，换取最窄的过渡带。

Elliptic（也叫 Cauer）滤波器在所有 IIR 原型中，用给定阶数实现最陡峭的过渡带。代价是通带和阻带都有等波纹。

- **优点**：同阶数下过渡带最窄。可以达到几乎"垂直"的截止。
- **缺点**：通带不平、阻带不完全干净。相位特性是所有原型中最差的（群延迟变化最剧烈）。
- **`rp` + `rs`**：同时指定通带最大纹波和阻带最小衰减。

**类比**：Elliptic 之于 IIR ≈ Remez 之于 FIR。都是"用阶数效率最高"的设计方法。

### 2.5 Bessel —— 相位至上

**一句话**：不在乎过渡带有多宽，只在乎输出波形不被扭曲。

Bessel 滤波器的设计目标不是频率选择性，而是**群延迟尽量平坦**——所有频率成分被延迟的时间几乎相同。这意味着输出波形几乎不被扭曲（类似 FIR 线性相位的效果，但不完全一样）。

- **优点**：群延迟最平坦——脉冲保真度最高。
- **缺点**：过渡带非常宽。阻带衰减增长缓慢。
- **什么时候用**：你关心波形形状（比如要精确知道一个脉冲何时出现），而不仅仅是频率成分。

## 3. scipy 的 IIR 设计函数

### 3.1 五种原型的设计函数

每种原型都有一个对应的设计函数，接口几乎相同：

```python
from scipy.signal import butter, cheby1, cheby2, ellip, bessel

# Butterworth
b, a = butter(N=4, Wn=10, btype='low', fs=100)

# Chebyshev I（rp = 通带最大纹波 dB）
b, a = cheby1(N=4, rp=0.5, Wn=10, btype='low', fs=100)

# Chebyshev II（rs = 阻带最小衰减 dB）
b, a = cheby2(N=4, rs=40, Wn=10, btype='low', fs=100)

# Elliptic（两边都指定）
b, a = ellip(N=4, rp=0.5, rs=40, Wn=10, btype='low', fs=100)

# Bessel
b, a = bessel(N=4, Wn=10, btype='low', fs=100)
```

**返回的是什么？** 两个一维数组 `b` 和 `a`：
- `b`：前馈系数（feedforward）——作用在输入信号上
- `a`：反馈系数（feedback）——作用在过去的输出上

`a[0]` 总是 1（归一化）。IIR 滤波器的数学表达式：

```
a[0]*y[n] = b[0]*x[n] + b[1]*x[n-1] + ... + b[M]*x[n-M]
                      - a[1]*y[n-1] - a[2]*y[n-2] - ... - a[N]*y[n-N]
```

`y[n]` 同时取决于输入 `x` 和过去的输出 `y`——这就是反馈。

### 3.2 参数详解

#### `N` — 滤波器阶数

IIR 滤波器的阶数。**通常 4-8 阶就足够了**（对比 FIR 需要 50-200+）。这是 IIR 最大的优势——低阶数 = 低延迟 = 低计算量。

阶数越大 → 过渡带越窄 → 但可能带来数值稳定性问题（高阶 IIR 的系数范围极大，浮点误差会被放大）。

#### `Wn` — 临界频率

和 FIR 的 `cutoff` 类似，但有一个重要区别：**对于 IIR，截止频率通常指 -3 dB 点的位置**（即幅度衰减到 1/√2 ≈ 0.707 的频率）。

- 低通/高通：`Wn` 是一个标量
- 带通/带阻：`Wn` 是一个长度为 2 的列表 `[Wn_low, Wn_high]`

#### `btype` — 滤波器类型

`'lowpass'`（默认）、`'highpass'`、`'bandpass'`、`'bandstop'`。

#### `rp` — 通带最大纹波 (dB)

仅 Chebyshev I 和 Elliptic 需要。控制"通带内的振荡幅度上限"。典型值是 0.1 到 3 dB。越小越接近 Butterworth，越大过渡带越窄。

#### `rs` — 阻带最小衰减 (dB)

仅 Chebyshev II 和 Elliptic 需要。控制"阻带内的最小衰减"。典型值是 20 到 60 dB。越大阻带越干净。

#### `fs` — 采样率

和 FIR 一样——传了之后 `Wn` 用实际频率单位，不用手动归一化。**强烈建议总是传 `fs`。**

### 3.3 `iirdesign` 和 `iirfilter` — 自动化设计

如果你不想手动选原型和阶数：

**`iirdesign`**：你指定通带/阻带的频率和衰减要求，它自动选择最适合的原型和阶数。

```python
from scipy.signal import iirdesign

b, a = iirdesign(
    wp=10,    # 通带截止频率 (Hz)
    ws=15,    # 阻带起始频率 (Hz)
    gpass=1,  # 通带最大衰减 (dB)
    gstop=40, # 阻带最小衰减 (dB)
    fs=100
)
```

**`iirfilter`**：指定阶数和原型类型，设计任意类型的 IIR 滤波器（是上面五种函数的通用接口）。

```python
from scipy.signal import iirfilter

b, a = iirfilter(N=4, Wn=10, rp=0.5, rs=40,
                 btype='low', ftype='ellip', fs=100)
```

## 4. 为什么 IIR 必须用 SOS（二阶节）形式？

### 4.1 问题：高阶 IIR 的数值灾难

直接用 `b, a` 系数来滤波（即 `lfilter(b, a, signal)`）有一个严重的隐患：**当阶数较高（N > 4~6）时，浮点精度误差可能让滤波器变得不稳定，即使理论设计是稳定的。**

原因在于 `a` 中的系数范围可能跨越很多数量级（如 10⁻¹⁰ 到 10¹⁰），双精度浮点数的有效位数（约 15-16 位十进制有效数字）不够用。微小的舍入误差被反馈回路反复放大后，可能导致输出发散。

### 4.2 解法：二阶节（SOS）

把高阶滤波器分解为多个**二阶滤波器**的串联。每个二阶滤波器处理信号的一小部分，输出传给下一个二阶滤波器。

**为什么是二阶？** 二阶是可以用实数运算表示的最基本"原子"单元。任何高阶传递函数都可以分解为多个二阶（加上可能的一个一阶）传递函数的乘积。

每个 SOS 的系数范围远小于原始 `b, a`，因此数值稳定得多。

### 4.3 使用方式

```python
from scipy.signal import butter, sosfilt

# 设计 IIR 滤波器并自动转为 SOS 形式
sos = butter(N=4, Wn=10, btype='low', fs=100, output='sos')
# sos 是一个 (K, 6) 的二维数组，K = 二阶节的个数

# 用 sosfilt 滤波（而非 lfilter）
filtered = sosfilt(sos, signal)

# 如果你已经有了 (b, a)，也可以手动转换
from scipy.signal import tf2sos
sos = tf2sos(b, a)
```

**核心结论**：
- **永远用 `output='sos'` + `sosfilt`，不要直接用 `(b, a)` + `lfilter`。**
- 对于 N ≤ 4 的低阶滤波器，`(b, a)` 通常也安全，但养成 SOS 的习惯不会有坏处。
- SOS 格式还可以和 `sosfreqz`、`sosfiltfilt` 等函数配合使用。

## 5. IIR 稳定性分析

### 5.1 什么是稳定？

一个滤波器是**稳定的**，当且仅当对于任何有限输入，输出也是有限的。

对于 IIR，稳定性等价于**所有极点在 z 平面的单位圆内**。极点是什么？是传递函数分母多项式的根——也就是 `a` 系数的根。

这可以这样理解：极点代表系统的"固有频率/模式"。如果极点在单位圆内 → 系统对冲击的响应会逐渐衰减到零 → 稳定。如果极点在单位圆外 → 响应会发散 → 不稳定。

### 5.2 如何检查

```python
from scipy.signal import tf2zpk

# 获取零点、极点、增益
z, p, k = tf2zpk(b, a)

# 检查所有极点的模是否 < 1
is_stable = np.all(np.abs(p) < 1)
```

绘制零极点图是最直观的稳定性检查方式——本章代码中会演示。

## 6. IIR vs FIR 选择决策流程

问自己三个问题：

**问题1**：你对相位有严格要求吗？（波形必须不被扭曲？）

- 是 → FIR（线性相位），或者 IIR + `filtfilt`（零相位滤波，第6章）
- 否 → 继续问题2

**问题2**：你需要极低的延迟和计算开销吗？

- 是 → IIR（4-8 阶 vs FIR 50-200+ 阶）
- 否 → 继续问题3

**问题3**：你需要非常陡峭的过渡带吗？

- 是 → IIR（Elliptic 或 Chebyshev I）或 FIR（Remez 用高阶数）
- 否 → FIR（firwin + hamming）最简单可靠

**快速默记**：FIR = 稳定 + 线性相位 + 高延迟。IIR = 低阶 + 低延迟 + 非线性相位（可用 filtfilt 弥补）。

## 7. 常见错误

| 你做的 | 发生的现象 | 原因 | 修法 |
|--------|-----------|------|------|
| 高阶 IIR 直接用 `(b, a)` + `lfilter` | 滤波结果异常或发散 | 数值精度不足 | 用 `output='sos'` + `sosfilt` |
| `Wn` 不归一化也不传 `fs` | 滤波器完全不对 | 同 FIR | 传 `fs` 参数 |
| `btype` 写成了滤波器类型名 | 报错 | 只接受 `'lowpass'`/`'highpass'`/`'bandpass'`/`'bandstop'` | 检查拼写 |
| 带通时 `Wn` 顺序反了 | 滤波器逻辑反了 | `Wn=[low, high]` 必须 low < high | 检查值的大小关系 |
| 对实时数据用了 `filtfilt` | 输出超前（依赖未来数据） | `filtfilt` 需要整个信号才能做前向+后向过滤 | 实时应用用 `sosfilt` |

## 8. 本章要点速查

| 概念 | 一句话 |
|------|--------|
| IIR | 带反馈的滤波器——用过去的输出影响当前输出 → 脉冲响应无限长 |
| 反馈 | 类似 AR 模型中的自回归项——让信号"循环利用"，以低阶数实现强效果 |
| Butterworth | 通带最平坦，各方面均衡——IIR 的默认首选 |
| Chebyshev I / II | 通带（I型）或阻带（II型）允许纹波，换更窄过渡带 |
| Elliptic | 通带阻带都有纹波——同阶数下过渡带最窄 |
