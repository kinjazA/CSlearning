## 第2章 · 卷积与相关

**场景定位**：卷积是滤波器的数学基础，相关是模式匹配/模板检测的核心。这是信号处理中最重要的两个操作。

### 笔记内容
- 卷积的直观理解（滑动加权平均 → 平滑/去噪/边缘检测）
- `convolve` vs `fftconvolve` — 时域卷积 vs 频域卷积，效率边界
  - 何时用哪个？（kernel 长度阈值：通常 > 500 点用 fftconvolve）
- 相关操作 `correlate` — 模板匹配、信号对齐、时延估计
- 边界处理模式：`full` / `same` / `valid` 的含义与选择
- 二维卷积 `convolve2d` — 图像平滑/边缘检测的快速入门

### 代码要点
- 用卷积实现移动平均平滑（对比手动实现）
- 用相关实现信号中的模板检测（如检测 ECG 中的 QRS 波）
- 时延估计：两个传感器信号的到达时间差（TDOA）

> **实际场景**：传感器数据平滑、音频回声检测、图像边缘提取、GNSS 信号捕获中的互相关

## 第3章 · FIR 滤波器设计与应用（上）— 窗口法

**场景定位**：FIR 滤波器是工业界最常用的数字滤波器——天然稳定、线性相位。本章聚焦窗口法设计，是滤波器设计的入门首选。

### 笔记内容
- 滤波器基础概念速览
  - 低通/高通/带通/带阻的频域含义
  - 通带、阻带、过渡带、纹波（ripple）的工程定义
  - 阶数 vs 性能 vs 延迟的权衡
- 理想滤波器与窗口截断的 Gibbs 现象
- 窗函数族谱：`boxcar` / `hann` / `hamming` / `blackman` / `kaiser`
  - 主瓣宽度 vs 旁瓣衰减的取舍表
  - Kaiser 窗的参数 β 调节——工业界的通用选择
- `firwin` — 最常用的 FIR 设计函数
  - 标准滤波器类型映射（`lowpass` / `highpass` / `bandpass` / `bandstop`）
  - 截止频率归一化（Nyquist 频率的陷阱）
- `firwin2` — 任意频率响应设计

### 代码要点
- 设计一个 ECG 信号的低通滤波器（去除肌电噪声）
- 窗函数对比实验（同阶数、不同窗 → 频率响应对比）
- Kaiser 窗的 β 参数扫参分析

> **实际场景**：生物医学信号去噪（ECG/EEG 的基线漂移 + 工频干扰）、音频分频器

## 第4章 · FIR 滤波器设计与应用（下）— 最优法与实战

**场景定位**：窗口法简单但非最优。Remez/Parks-McClellan 算法在同等阶数下性能更优，是工业级 FIR 设计的标配。

- 滤波器分析与验证工具链

  - `group_delay` — 群延迟分析

- 滤波器系数量化效应（定点 DSP 的考量）

### 代码要点
- 用 `remez` 设计多带滤波器（如双通带滤波器）
- `firwin` vs `remez` 同阶数性能对比
- 搭建一个 `FilterDesigner` 工具类（统一接口：`design → analyze → visualize`）

> **实际场景**：通信系统中的多带滤波器、音频均衡器设计、抗混叠滤波器

## 第5章 · IIR 滤波器设计与应用

**场景定位**：IIR 滤波器以低阶数实现陡峭过渡——计算量小、延迟低，是嵌入式/实时系统的首选。

### 笔记内容
- IIR vs FIR 的工程选型决策树
  - 何时必须 FIR（线性相位需求）？何时 IIR 更优（低延迟/低算力）？
- 经典模拟滤波器原型
  - Butterworth — 最平坦通带，通用首选
  - Chebyshev I/II — 通带/阻带等波纹，更陡过渡
  - Elliptic/Cauer — 最陡过渡，通带阻带均有纹波
  - Bessel/Thomson — 最平坦群延迟（脉冲保真度最高）
- IIR 滤波器设计函数族
  - `butter` / `cheby1` / `cheby2` / `ellip` / `bessel`
  - `iirdesign` — 自动化 IIR 设计
  - `iirfilter` — 通用 IIR 设计接口
- 关键注意事项
  - 双线性变换的频率预畸变（pre-warping）
  - IIR 稳定性检查（极点是否在单位圆内）
  - 数值精度问题——**二阶节（SOS）形式是强制最佳实践**

### 代码要点
- 五种 IIR 原型同阶数横向对比（幅频、相频、群延迟）
- 设计实时心电监护的 IIR 带通滤波器
- 稳定性可视化（零极点图）

> **实际场景**：嵌入式传感器实时滤波、音频分频（cross-over）、工控系统中的 50/60Hz 陷波器

  - 数值稳定性分析（高阶 IIR 的系数敏感度）
  - `tf2sos` / `zpk2sos` — 转换到二阶节
- 设计→转换→应用的完整管线

### 代码要点

## 第7章 · 频谱分析（FFT 与功率谱密度估计）

**场景定位**：频谱分析是信号处理的核心——振动分析、音频处理、通信系统、脑电分析，几乎所有领域都需要。

### 笔记内容
- 傅里叶变换回顾
  - DFT 的工程直觉（不是数学推导）
  - `fft` / `rfft` / `ifft` / `irfft` — 实信号的高效变换
  - `fftfreq` / `rfftfreq` — 频率轴构建
- 频谱泄露、栅栏效应、频率分辨率的物理含义
- 功率谱密度（PSD）估计
  - 周期图法 `periodogram` — 基础但方差大
  - Welch 方法 `welch` — 分段平均，降低方差（**工业标配**）
  - `lombscargle` — 非均匀采样信号的周期图（天文、临床数据）
- Welch 方法的参数调优
  - 段长 `nperseg` vs 重叠 `noverlap` vs 频率分辨率
  - 窗函数选择对 PSD 估计的影响
- `csd` — 互功率谱密度（输入-输出关系分析）
- `coherence` — 相干性分析（两个信号的频域关联强度）

### 代码要点
- Welch PSD 参数扫参实验（`nperseg` / `noverlap` 对估计质量的影响）
- 振动信号的频谱特征提取（旋转机械故障诊断场景）
- 非均匀采样数据的 Lomb-Scargle 周期图

> **实际场景**：旋转机械振动监测、脑电频带功率提取（δ/θ/α/β/γ）、通信信号频谱感知

## 第8章 · 时频分析（STFT, 频谱图与小波）

**场景定位**：FFT 告诉你"有哪些频率"，但不知道"频率何时出现"。时频分析是非平稳信号（音乐、语音、地震、生物医学）的标配工具。

### 笔记内容
- 短时傅里叶变换（STFT）
  - `stft` / `istft` — 正向/逆向变换
  - 窗长、跳跃步长（hop size）与时间/频率分辨率的权衡（Heisenberg 不确定性）
  - 频谱图（spectrogram）的参数调优实战
- `spectrogram` — 一步到位的频谱图计算与可视化
- 连续小波变换（CWT）入门
  - `cwt` — 多尺度时频分析
  - 小波基的选择：`morlet2` / `ricker`（Mexican hat）
  - CWT vs STFT：什么时候用小波？
- 时频脊线（ridge）提取——瞬时频率追踪

### 代码要点
- 制作一个 chirp 信号的频谱图，观察扫频轨迹
- 音乐片段的频谱图分析（音高、节奏可视化）
- CWT 与 STFT 在同一信号上的时频分辨率对比
- 实现一个简单的瞬时频率追踪器

> **实际场景**：语音信号分析（语谱图）、地震信号时频特征、EEG 事件相关同步化、音乐信息检索

## 第9章 · 峰值检测与信号特征提取

**场景定位**：从信号中自动提取有意义的"事件"——峰值、谷值、拐点——是信号到特征的桥梁，直接影响 ML 模型效果。

  - `width` — 峰宽检测
  - `wlen` / `rel_height` — 宽度计算的窗口与相对高度
- `peak_prominences` / `peak_widths` — 峰属性的独立计算
- `find_peaks_cwt` — 基于小波的峰值检测（对噪声更鲁棒）
- `argrelmax` / `argrelmin` / `argrelextrema` — 局部极值检测
- 包络提取
  - Hilbert 包络（预告第10章）
  - 峰值包络 / RMS 包络

### 代码要点
- `find_peaks` 全部参数逐一实验，理解交互关系
- 不同噪声水平下的峰值检测鲁棒性对比
- ECG R 波自动检测器（`find_peaks` + prominence 调优）
- 振动信号的冲击特征提取（轴承故障检测场景）

> **实际场景**：ECG/PPG 心率检测、机械故障冲击特征提取、色谱峰积分、步态事件检测

## 第10章 · LTI 系统分析与 Hilbert 变换

**场景定位**：很多物理系统可以建模为 LTI 系统。Hilbert 变换提供信号的解析表示，是包络检测和相位分析的基础。

### 笔记内容

**Part A：LTI 系统**
- LTI 系统的三种表示形式
  - 传递函数 `TransferFunction`（`ba` 形式）
  - 零极点增益 `ZerosPolesGain`（`zpk` 形式）
  - 状态空间 `StateSpace`（`ABCD` 形式）
- 表示形式之间的转换：`tf2zpk` / `zpk2tf` / `tf2ss` / `ss2tf` / `ss2zpk`
- 系统分析
  - `freqresp` — 频率响应
  - `impulse` / `step` — 脉冲/阶跃响应
  - `bode` — Bode 图（幅频 + 相频）
- 系统串联与反馈：`series` / `parallel` / `feedback`
- 滤波器稳定性分析：极点在 z 平面的位置

**Part B：Hilbert 变换**
- 解析信号（analytic signal）的直观理解
- `hilbert` / `hilbert2` — Hilbert 变换
- 包络检测与瞬时相位
- 瞬时频率计算

### 代码要点
- 搭建一个二阶低通滤波器的 LTI 模型，完成完整分析
- 反馈系统稳定性分析（如 PLL 锁相环简化模型）
- ECG/振动信号的 Hilbert 包络提取
- 瞬时频率用于故障检测（如电机转速波动分析）

> **实际场景**：控制系统建模、传感器动力学补偿、通信信号解调、结构健康监测

## 第12章 · 信号去噪：方法对比与实战

**场景定位**：去噪是信号处理中最常见的需求。本章系统对比各类去噪方法，帮你建立"什么场景选什么方法"的决策框架。

### 笔记内容
- 去噪方法全景对比
  - 低通/带通滤波去噪（经典、可控）
  - Savitzky-Golay 滤波 `savgol_filter` — 保边缘的平滑（光谱分析标配）
  - 中值滤波 `medfilt` — 去除脉冲噪声（传感器尖峰）
  - Wiener 滤波 `wiener` — 自适应去噪（已知噪声统计时最优）
  - 小波阈值去噪 — 非平稳信号去噪
- Savitzky-Golay 的参数选择
  - 窗口长度 vs 多项式阶数的权衡
- 去噪效果的量化评估
  - SNR（信噪比）计算
  - MSE / MAE（有 ground truth 时）
  - 保真度 vs 平滑度的可视化对比

### 代码要点
- 同一含噪信号用 5 种方法去噪，量化+可视化对比
- 光谱数据的 Savitzky-Golay 平滑 + 导数谱计算
- 传感器数据的中值滤波去除偶发尖峰
- 编写 `DenoiseBenchmark` 对比评估工具

> **实际场景**：拉曼/红外光谱预处理、ECG 基线漂移 + 肌电干扰联合去除、工业传感器数据清洗

## 第13章 · 信号特征工程与 ML 集成

**场景定位**：从原始信号到机器学习模型的"最后一公里"。本章打通 `scipy.signal` → 特征矩阵 → `scikit-learn` / `PyTorch` 的完整链路。

### 笔记内容
- 信号特征分类体系
  - 时域特征：均值、方差、RMS、峰值因子、峭度、偏度、过零率
  - 频域特征：谱质心、谱带宽、谱滚降、频带能量比
  - 时频特征：特定频带随时间变化的能量
  - 非线性特征：样本熵、近似熵（预告 → `scipy.signal` 生态扩展）
- 滑动窗口特征提取框架
  - 窗口大小、步长的选择策略
  - 重叠率对特征质量的影响
- 特征标准化与选择
- 与 `scikit-learn` Pipeline 的无缝集成
- 端到端案例

## 第14章 · tsfresh 自动特征工程

**场景定位**：前 13 章教你手动从信号提取特征。本章引入 tsfresh，一行代码自动提取 1200+ 个特征，再用统计检验（Benjamini-Hochberg）自动筛选真正有区分力的特征。

### 笔记内容
- tsfresh 和前 13 章的分工：scipy.signal 做底层信号处理，tsfresh 做高层特征自动化
- 数据格式要求：(id, time, value) 长表格式
- `extract_features`：三级特征集（Minimal / Efficient / Comprehensive）
- `select_features`：统计检验 + FDR 矫正 → 自动保留显著特征
- `relevance_table`：查看每个特征和标签的关联显著性
- tsfresh 特征命名规则：`op__stat`（操作 + 统计量）
- 特征大类与 scipy.signal 章节的映射（FFT → 第7章 / 峰 → 第9章 / 小波 → 第8章……）
- 完整端到端模板代码（原始信号 → 特征 → 筛选 → 模型）
- 关键注意事项：数据泄漏、select_features 必须先划分再筛选
- tsfresh vs 手动特征的决策指南：探索用 tsfresh，生产用手动核心特征

> **实际场景**：多传感器故障分类的快速探索、基线模型搭建、特征空间初步筛选

### 学习目标定位

本大纲不是"为 ML 提取特征的教程"。信号处理是一套独立的数据分析体系：

- **分析视角**：统计学关注"分布和相关性"，信号处理关注"频率和波形"——两套眼睛互补
- **核心任务**：看清楚数据（频谱）→ 拆开看（滤波）→ 洗干净（去噪）→ 找事件（峰值检测）→ 比关系（相干性）→ 变换（重采样）
- **在时序预测中**：频谱分析确定周期参数、滤波分解成分分别建模、去噪提高信噪比、时频分析处理非平稳数据
- **在异常检测中**：残差分析检测偏离常态、频谱监控发现新频率成分、相干性分析检测同步性丧失、瞬时频率追踪发现变化模式异常
- **与 ML 的关系**：信号处理做前端（去噪/分解/特征提取）和后端（峰值筛选/频谱校验），ML 做中间的建模和分类
