## 0. 从你已经会的东西出发

### 0.1 第2章回顾：卷积平滑 = 最原始的低通滤波

第 2 章你学会了用卷积做平滑。比如一个 5 日均线：

```python
kernel = np.ones(5) / 5
smoothed = np.convolve(price, kernel, mode='same')
```

这个操作的本质是什么？你取了一个长度为 5 的窗口，把窗口内的值做了等权平均。在频率的视角下，这个操作等价于一个**低通滤波器**——慢变化（低频）保留了下来，快变化（高频）被抹平了。

**但这"低通"非常粗糙**。你没有办法精确地告诉它"请保留周期大于 20 天的波动，去掉周期小于 5 天的波动"。你只能说"取 5 天平均"或者"取 20 天平均"，然后猜测这大概对应了什么频率范围。

本章的目标就是回答这个问题：**如何精确地设计一个滤波器，明确指定哪些频率成分保留、哪些频率成分去掉？**

### 0.2 与你的统计学知识的对接

| 你已经知道的 | 本章对应的新概念 |
|-------------|---------------|
| 移动平均 (MA) | FIR 低通滤波器——保留低频、抑制高频 |
| 差分 (diff) | FIR 高通滤波器——保留高频、抑制低频 |
| 季节分解 (seasonal decomposition) | 带通滤波器——提取特定频率范围的成分 |
| OLS 残差 | 高通滤波后的结果——"去掉趋势后剩下的" |
| 光滑样条 (smoothing spline) | 低通滤波器——保留趋势 |
| Baxter-King 滤波器（宏观经济学） | 带通 FIR 滤波器——提取特定经济周期 |

## 1. 基础概念：用你最熟悉的方式来理解

在正式进入滤波器设计之前，我们需要建立几个基础概念。我会尽量用你已有的统计学直觉来解释。

### 1.1 频率是什么？

你学过时间序列，知道自相关函数（ACF）描述的是"现在和过去的关系"。**频率是另一种描述方式——它刻画的是数据变化的快慢。**

- **低频** = 变化慢 = 你时间序列中的长期趋势（比如一支股票从 2015 年到 2025 年的价格翻倍，这就是极低频的成分）
- **高频** = 变化快 = 短期波动或噪声（比如同一支股票在一分钟内的涨跌，这就是高频成分）

**频率和周期的关系**：频率 = 1 / 周期。

如果某个波动每 20 天完成一次完整的涨→跌→涨循环，那么：
- 它的周期 = 20 天
- 它的频率 = 1/20 = 0.05 次/天
- 在日度数据中（每年约 252 个交易日），也可以表示为 252/20 ≈ 12.6 次/年

### 1.2 什么是傅里叶变换？（直觉版，不是数学推导）

任何信号——不管它看起来多复杂——都可以拆解成一系列正弦波的叠加。这就是傅里叶变换的核心思想。

一个正弦波由三个属性完全确定：
- **频率**：每秒/每天振动多少次
- **幅度**：振动有多强（波的高度）
- **相位**：振动从哪个位置开始（波的"起步点"）

傅里叶变换做的事情就是：输入一个信号，输出"这个信号里有多少频率 f₁ 的成分、多少频率 f₂ 的成分……有多少频率 fₙ 的成分"。它把信号从"时间视角"（每个时刻的值是多少）转换到"频率视角"（每种频率的成分各有多少）。

### 1.3 什么是滤波？

**滤波 = 选择性保留或去除特定频率成分。**

一个滤波器就是一个"频率筛选器"。你告诉它哪些频率可以通过、哪些不能，它照做。

> **💡 什么是"增益"（gain）？**
>
> 增益就是滤波器对某个频率成分的**放大倍数**——一个乘法系数。
>
> | 增益值 | 含义 | 对应的 dB |
> |--------|------|----------|
> | **增益 = 1** | 原封不动通过——信号该频率的成分被乘以 1，完全保留 | 0 dB |
> | **增益 = 0.5** | 衰减一半——该频率成分的幅度被砍掉 50% | -6 dB |
> | **增益 = 0.1** | 大幅衰减——只剩原始幅度的 10% | -20 dB |
> | **增益 = 0** | 完全阻挡——该频率成分被乘以 0，彻底消失 | -∞ dB |
>
> 用日常类比：增益就像一个**音量旋钮**，但它不是对所有声音统一调节，而是对每个频率各有各的旋钮位置。低频成分的旋钮拧到"10"（增益≈1），高频成分的旋钮拧到"0"（增益≈0）——这就是低通滤波器。
>
> 你的信号进入滤波器后，被拆成不同频率的正弦波，各自乘以对应的增益，再重新组装成输出信号。**这就是"滤波"在数学上到底做了什么。**

四种基本类型（按频率从低到高描述）：

| 类型 | 行为 | 频率轴上的表现 | 直观理解 |
|------|------|-------------|---------|
| **低通** | 低频通过，高频阻挡 | 低频区增益≈1，高频区增益≈0 | 平滑——只看趋势，抹平噪声 |
| **高通** | 高频通过，低频阻挡 | 低频区增益≈0，高频区增益≈1 | 去趋势——只保留短期波动 |
| **带通** | 中间一段频率通过，两边阻挡 | 中间增益≈1，两边增益≈0 | 只看特定周期的波动（如"只关心 20-60 天的周期"） |
| **带阻** | 中间一段频率阻挡，两边通过 | 中间增益≈0，两边增益≈1 | 去掉特定干扰（如去掉 50Hz 电网噪声） |

**关键认知**：你无法做到"频率 10.00 Hz 以下全通过、10.01 Hz 以上全阻挡"这种完美截断。真实的滤波器在"通过"和"阻挡"之间总有一个过渡区域——这引出下一个概念。

### 1.4 通带、阻带、过渡带——滤波器的三个区域

**通带 (Passband)**：这个频率范围内的成分几乎不受影响地通过。理想的增益 = 1（即 0 dB）。实际中会有微小波动，叫**通带纹波**（passband ripple）。

**阻带 (Stopband)**：这个频率范围内的成分被大幅衰减。理想的增益 = 0（即 -∞ dB）。实际只能衰减到比如 -60 dB（衰减到原始值的 1/1000）。

**过渡带 (Transition Band)**：通带和阻带之间的"斜坡"。这部分频率既不完全通过也不完全阻挡。

---

#### 为什么一定有过渡带？——从你已经会的东西说起

你在第2章学过：**移动平均的窗口越长，平滑效果越强。** SMA-5 还能看到日间波动，SMA-200 几乎只剩趋势。

滤波器本质上也是移动平均——只是权重不是均等的，而是精心设计过的。所以**窗口长度的逻辑完全一样**：

> **滤波器长度 = 它能"看到"多长一段历史。**

现在关键来了——**区分两个接近的频率，需要足够长的观察时间。**

举个例子：你想在日度股票数据中分离"20 天周期"和"25 天周期"的波动。这两个周期只差 5 天。在 10 个交易日的窗口里，一个 20 天周期的正弦波和一个 25 天周期的正弦波看起来几乎一模一样——你无法可靠地区分它们。但如果你观察 200 个交易日，20 天周期走了 10 个完整循环，25 天周期走了 8 个——差异就非常明显了。

**一般规律：滤波器能分辨的最小频率差 ≈ fs / numtaps。**（fs = 采样率。）

也就是说：
- `numtaps = 21` → 只能分辨相差约 fs/21 的频率 → 过渡带必须这么宽
- `numtaps = 101` → 能分辨相差约 fs/101 的频率 → 过渡带可以窄 5 倍

**过渡带就是滤波器"分不清"的频率区间。** 在这个区间内，滤波器无法确定该放行还是阻挡，于是产生了一个从"通过"到"阻挡"的斜坡。滤波器越长 → 能分清的频率越精细 → 过渡带越窄。

这就是为什么有限长度的滤波器无法做到"一刀切"——不是因为你用的窗函数不够好，而是**数学上不可能**。长度本身就是频率分辨率的硬上限。你在第7章还会再次遇到这个原理——Welch 方法的频率分辨率也是 `fs / N`，完全一样的逻辑。

---

#### 那为什么不能无脑加长滤波器？

因为滤波器的输出有一个**固有的延迟**。

FIR 滤波器是对称的（这样才能保证相位不扭曲——后面会讲）。一个长度为 21 的对称滤波器，它的"重心"在第 11 个位置。这意味着：**今天的输出值，实际上代表的是 10 天前的信号状态。** 延迟 = `(numtaps - 1) / 2`。

拿 SMA 来类比：你想算第 100 天的 21 日均线，你需要第 90~110 天的数据。但第 110 天还没发生——所以你只能"向后看"，用第 80~100 天的数据。输出的均线值反映的是第 90 天附近的状态，落后当前 10 天。

**这就是核心权衡：**

| numtaps | 过渡带宽度 | 延迟 | 适合什么 |
|---------|----------|------|---------|
| 21 | 宽（频率分辨粗） | 10 天 | 实时交易信号——延迟小比精度重要 |
| 101 | 窄（频率分辨细） | 50 天 | 离线分析——精度比时效性重要 |
| 201 | 很窄 | 100 天 | 历史研究——你不在乎等多久 |

这和统计学里的困境完全一致——**更多数据 → 更精确的估计 → 但对最新变化反应更迟钝。** 用短窗口 = 方差大但偏差小（跟得上变化）；用长窗口 = 方差小但偏差大（趋势判断更可靠，但转弯时反应慢）。

你设计滤波器时，本质上就是在选一个点，落在"频率精度"和"时间响应速度"这条权衡曲线的某个位置上。

### 1.5 Nyquist 频率——你能"看到"的最高频率

#### 先搞清楚"采样率"在你世界里是什么

你每天都在和采样率打交道，只是没叫这个名字：

- 日度股价数据 → 每天记录一次 → 采样率 fs = 1 次/天
- 15 分钟电力负荷 → 每 15 分钟记录一次 → fs = 96 次/天
- 分钟级 tick 数据 → 每分钟记录一次 → fs = 60 次/小时

**采样率就是你记录数据的频率。** Nyquist 频率 = fs / 2，代表你能从这些数据中检测到的**最快波动**。

#### 核心问题：采样太慢会发生什么？

假设有一个真实的连续信号——比如某只股票的价格实际上在每秒都在变动。但你只记录了每日收盘价（fs = 1 次/天）。那么任何**周期小于 2 天的波动**，你都完全看不到——不仅如此，更糟糕的是它们会**伪装成另一种更慢的波动**出现在你的数据里。

假设某只股票的真实日内价格中存在一个短周期波动（比如由算法交易策略引起的微小振荡）：

$$r(t) = \epsilon(t) + 0.002 \cdot \cos(2\pi \cdot 0.7t)$$

其中 $\epsilon(t)$ 是随机噪声，0.002 是微小振幅，**0.7 cycles/day** 是真实频率——对应的真实周期约为 **1.43 天**。

但你只有日频数据（fs = 1 次/天），只能观测到 $r_n = r(n)$：

$$r_n = \epsilon_n + 0.002 \cdot \cos(2\pi \cdot 0.7n)$$

关键来了——利用余弦的周期性质：

$$\cos(2\pi \cdot 0.7n) = \cos(2\pi \cdot 0.7n - 2\pi n) = \cos(2\pi \cdot (-0.3)n) = \cos(2\pi \cdot 0.3n)$$

你实际在数据里看到的是：

$$r_n = \epsilon_n + 0.002 \cdot \cos(2\pi \cdot 0.3n)$$

你兴冲冲地做了频谱分析，发现 0.3 cycles/day 处有一个显著的峰，算出对应周期 T = 1/0.3 ≈ 3.33 天。你写了一篇研报：「该资产存在约 **3.3 天**的短周期波动。」

**但真实情况是：这个 3.3 天周期是假的，是混叠制造出来的幻影。** 真实的波动周期是 1.43 天——但它对应的频率 0.7 cycles/day > Nyquist（0.5 cycles/day），在日度采样下被折叠成了 0.3 cycles/day。

```
真实频率:  0.7 cycles/day → 周期 1.43 天
                    ↓  混叠 (|0.7 - 1| = 0.3)
观测频率:  0.3 cycles/day → 周期 3.33 天  ← 你以为的"发现"
```

这个例子揭示了一个令人不安的事实：**你用日度数据做频谱分析发现的任何一个"周期"，你都永远无法 100% 确定它是真实的低频周期，还是某个高于 0.5 cycles/day 的高频成分被混叠下来的幻影。** 在金融高频数据和日内交易分析中，这个问题尤其严重——因为你天然地对日内波动感兴趣，但日度采样根本"看"不到它们。

#### 日常类比

你肯定见过这个现象：电影里汽车轮子或直升机螺旋桨有时看起来在**缓慢倒转**。这是因为摄像机的帧率（采样率，通常 24 帧/秒）不够快，跟不上轮子的转速。轮子真实的转速被"混叠"成了一个慢得多的、甚至是反向的转速。这就是 aliasing 在视觉上的表现——和你采样信号时遇到的问题完全一样。

#### Nyquist = fs/2 的直觉推导

如果 fs = 10 Hz，你每秒记录 10 次。一个完整周期至少需要你记录到"上升"和"下降"两个状态，也就是**每个周期至少 2 个采样点**。所以：
- 最短可检测周期 = 2 个采样间隔 = 2/fs 秒
- 对应最高可检测频率 = 1 / (最短周期) = fs/2 = 5 Hz

任何频率 > 5 Hz 的信号，每个周期只能被采到不到 2 个点，就不够用来唯一确定这个频率——它会和某个低于 5 Hz 的频率产生完全相同的采样值。

#### 你的场景中的实际含义

| 你的数据 | fs | Nyquist | 意味着什么 |
|---------|-----|---------|-----------|
| 日度数据 | 1 次/天 | 0.5 次/天 | 你最多只能检测周期 ≥ 2 天的波动。**日内的高频涨跌对你来说是不可见的。** |
| 分钟级 tick | 60 次/小时 | 30 次/小时 | 周期 ≥ 2 分钟的波动可以检测。周期小于 2 分钟的波动会混叠。 |
| 15 分钟电力 | 96 次/天 | 48 次/天 | 周期 ≥ 30 分钟的负荷波动可以检测。更快的瞬时波动会混叠。 |

#### 和你后续章节的关系

这个知识点在第6章（降采样前必须先做抗混叠低通滤波）和第11章（`decimate` 内部自动做了这件事）会再次出现。现在你只需要记住：**Nyquist 是一堵硬墙——超过它的频率，你看不见，而且它们还会伪装成低频来骗你。**

在 scipy 中：截止频率必须 ≤ Nyquist。传 `fs` 参数后 scipy 自动处理单位转换，用你的实际频率单位（Hz、次/天）即可。**强烈推荐总是传 `fs` 参数。**

## 2. 为什么滤波器设计不能"一刀切"？—— Gibbs 现象

### 2.1 理想滤波器长什么样？

数学上，一个"完美的低通滤波器"在频域是一个矩形——截止频率以下增益 = 1，以上增益 = 0。

把这个矩形做逆傅里叶变换，得到它在时域的形态，是一个叫 **sinc 函数**（sin(x)/x 的形状）的东西。

sinc 函数有一个问题：它在时间轴上**向两边无限延伸**——它的尾部永远不等于零，只是逐渐衰减。也就是说，要实现完美的低通滤波器，你需要一个无限长的核。在实际中这当然是不可能的——你只能用有限长度的核。

### 2.2 直接截断 = 自找麻烦

你会很自然地想：那就把 sinc 函数截断——只取中间一段，两边扔掉。这等价于给 sinc 函数乘了一个矩形窗（在窗口内权重 = 1，窗口外权重 = 0）。

这带来一个严重的问题：**Gibbs 现象**。

在频域中，截止频率附近会产生明显的振荡和过冲（overshoot），幅度约为 9%。而且，本该被阻挡的频率也会"泄漏"——在频域上出现一系列不希望的小峰（称为旁瓣，sidelobe）。

**直觉理解**：想象你用剪刀把一张照片的边缘剪下来——剪口是锯齿状的，不是光滑的。频域中的"锯齿"和"毛刺"就是 Gibbs 振荡和旁瓣泄漏。

### 2.3 解决办法：窗函数

与其用矩形窗（一刀切），不如用一个**从中心向两端平滑衰减到零**的权重函数。这就是窗函数。

窗函数的效果：
- **时域**：核的两端被柔化——不再有突兀的截断
- **频域**：旁瓣大幅降低，Gibbs 振荡被显著抑制
- **代价**：过渡带变宽——从"允许通过"到"不允许通过"的斜坡变得没那么陡

这是一个在所有滤波器设计中反复出现的权衡：
- 旁瓣越低（泄漏越少）→ 过渡带越宽（选择性越差）
- 过渡带越窄（选择性越好）→ 旁瓣越高（泄漏越多）

**没有免费午餐。** 你总是在这两者之间选择。

## 3. 窗函数家族：每种窗代表一种权衡

### 3.1 为什么需要不同种类的窗？

不同的窗函数提供了不同的"旁瓣衰减 vs 过渡带宽度"权衡点。选择窗函数本质上就是选择你在这个权衡曲线上的位置。

### 3.2 常用窗函数一览

| 窗函数 | 旁瓣衰减 | 过渡带宽度 | 特点 | 推荐使用场景 |
|--------|---------|-----------|------|------------|
| 矩形窗 `boxcar` | 约 -13 dB | 最窄 | 就是直接截断。Gibbs 振荡严重，旁瓣高。 | 几乎不用 |
| `hann` | 约 -31 dB | 中等 | 两端平滑衰减到零。旁瓣衰减够用，过渡带不算太宽。各方面均衡。 | **默认首选**——大多数场景都够用 |
| `hamming` | 约 -41 dB | 中等 | 和 hann 很像，但两端不完全到零（有小台阶），这让近端旁瓣更低。 | 需要比 hann 更高的阻带衰减时 |
| `blackman` | 约 -57 dB | 较宽 | 旁瓣非常低，但过渡带明显更宽。 | 需要极高阻带衰减时 |
| `kaiser` | 可调 | 可调 | 有一个 β 参数，**一个窗覆盖所有场景**。 | **工业界标准选择** |

### 3.3 Kaiser 窗详解——一个参数统治一切

Kaiser 窗是实际工程中最常用的窗函数。它有一个 β 参数，让你能精确控制在"旁瓣衰减"和"过渡带宽度"之间的权衡位置。

**β 值的效果**：

| β 值 | 旁瓣水平 | 过渡带 | 大致相当于 |
|------|---------|--------|----------|
| 0 | -13 dB | 最窄 | 矩形窗 |
| 3 ~ 4 | -25 ~ -28 dB | 较窄 | 介于矩形和 hann 之间 |
| 5 | -30 dB | 中等 | 接近 hann |
| 6 | -36 dB | 中等偏宽 | 接近 hamming |
| 8.6 | -60 dB | 宽 | 接近 blackman |
| 12 | -90+ dB | 很宽 | 极端衰减需求 |

**怎么选 β？**

一条粗略的经验法则：
- 一般数据平滑 → β = 4 ~ 5
- 希望阻带可靠达到 -40 dB → β = 6
- 需要阻带达到 -60 dB → β = 8.6
- 需要极干净阻带 → β = 10 ~ 12

**核心直觉**：β 就像一个"精度预算"参数。你总共有 100 分的精度，β 决定你把分数分配给"过渡带陡峭度"还是"阻带干净度"。
- β 小 = 陡峭优先（过渡带窄，但阻带有些泄漏）
- β 大 = 干净优先（阻带几乎不漏，但过渡带宽）

## 4. `firwin` — 窗口法滤波器设计的核心函数

### 4.1 函数签名

```python
from scipy.signal import firwin

b = firwin(numtaps, cutoff, window='hamming', pass_zero=True, fs=None)
```

`b` 是返回的滤波器系数数组，长度 = `numtaps`。这就是滤波器的**核**——你可以用第 2 章学的卷积操作来应用它，也可以用 `filtfilt` 或 `lfilter`（第 6 章会详讲这些滤波方法的区别）。

### 4.2 参数逐一解读

#### `numtaps` — 滤波器长度

- 必须是**奇数**（这是 FIR 线性相位对称结构的数学要求）
- 长度越大 → 过渡带越窄 → 但延迟越大
- 延迟 = `(numtaps - 1) / 2` 个采样点

以日度数据（每年约 252 个交易日）为例：

| numtaps | 大致过渡带宽度 | 延迟（天） | 适合做什么 |
|---------|-------------|-----------|----------|
| 21 | 约 fs/7 ≈ 36 次/年 | 10 天 | 快速粗略平滑 |
| 41 | 约 fs/16 ≈ 16 次/年 | 20 天 | 一般用途 |
| 61 | 约 fs/25 ≈ 10 次/年 | 30 天 | **常用默认值** |
| 101 | 约 fs/43 ≈ 6 次/年 | 50 天 | 需要陡峭过渡 |
| 201 | 约 fs/90 ≈ 3 次/年 | 100 天 | 需要非常干净的分离 |

**注意**：过渡带宽度的近似公式只是粗略估计。实际取决于窗函数类型和具体频带位置。

#### `cutoff` — 截止频率

这是滤波器设计中**最容易出错的参数**。

如果不传 `fs`：cutoff 必须是归一化值，范围是 0 到 1。0 对应 0 Hz，1 对应 Nyquist 频率（= fs/2）。

如果传了 `fs`：cutoff 直接用实际频率值（和你传给 fs 的单位一致）。

```python
# ❌ 容易出错的方式（不传 fs，需要手动归一化）
b = firwin(51, cutoff=10/(1000/2))  # 10Hz 截止, 采样率1000Hz → Nyquist=500Hz → cutoff=0.02

# ✅ 推荐方式（传 fs，直接用实际频率）
b = firwin(51, cutoff=10, fs=1000)  # scipy 自动做归一化
```

**强烈建议总是传 `fs` 参数。** 少一次手动计算，少一个出错机会。

#### `pass_zero` — 决定滤波器类型

这个参数的含义是"零频率（DC 分量）是否通过"：

| `pass_zero` | 滤波器类型 | 含义 |
|------------|----------|------|
| `True`（默认值） | 低通 | 频率=0（即信号的均值/直流分量）保留 → 低频通过、高频阻挡 |
| `False` | 高通 | 频率=0 阻挡 → 去除了均值，高频通过、低频阻挡 |

当 `cutoff` 传入一个长度为 2 的列表时：

| `cutoff` 是列表 + `pass_zero` | 滤波器类型 | 含义 |
|------------------------------|----------|------|
| `cutoff=[f1, f2]`, `pass_zero=True` | **带阻** | 两个截止频率之间的频段被阻挡，两边通过 |
| `cutoff=[f1, f2]`, `pass_zero=False` | **带通** | 两个截止频率之间的频段通过，两边阻挡 |

```python
fs = 252  # 日度数据

# 低通：保留周期 > 20 天的趋势
b_low = firwin(51, cutoff=252/20, fs=252)  # pass_zero=True 是默认值

# 高通：保留周期 < 10 天的短期波动
b_high = firwin(51, cutoff=252/10, pass_zero=False, fs=252)

# 带通：保留周期 20-60 天的中期波动（频率 4.2 ~ 12.6 次/年）
b_band = firwin(51, cutoff=[252/60, 252/20], pass_zero=False, fs=252)

# 带阻：去掉周期 4-6 天的成分（频率 42 ~ 63 次/年）
b_stop = firwin(51, cutoff=[252/6, 252/4], pass_zero=True, fs=252)
```

#### `window` — 窗函数

可以传字符串名（`'hann'`、`'hamming'`、`'blackman'`），也可以传一个已创建好的窗函数数组。

用 Kaiser 窗的两种写法：

```python
from scipy.signal.windows import kaiser

# 写法1：传元组 (窗函数名, 参数)
b = firwin(51, cutoff=10, window=('kaiser', 6.0), fs=100)

# 写法2：先创建窗函数数组再传入
win = kaiser(51, beta=6.0)
b = firwin(51, cutoff=10, window=win, fs=100)
```

### 4.3 `firwin2` — 自定义频率响应（非标准形状）

`firwin` 的频带只有两档：通过（gain=1）或阻挡（gain=0）。如果你需要"某些频率衰减到 50% 但不是完全阻挡"，用 `firwin2`：

```python
from scipy.signal import firwin2

# 定义频率-增益曲线
freqs = [0, 0.1, 0.2, 0.3, 0.5]    # 归一化频率点（或传 fs 后用实际值）
gains = [1, 0.8, 0.3, 0.0, 0.0]    # 每个频率点对应的目标增益

b = firwin2(numtaps=51, freq=freqs, gain=gains, window='hamming')
```

这个功能在需要对不同频段施加不同程度衰减时有用。

## 5. 滤波器分析——验证你的设计

设计完之后，你需要检查滤波器是否真的按照你的意图在工作。

### 5.1 `freqz` — 查看频率响应

```python
from scipy.signal import freqz

w, h = freqz(b, worN=2048, fs=252)
# w: 频率轴（和 fs 相同单位）
# h: 复数频率响应

magnitude_db = 20 * np.log10(np.abs(h))  # 转为 dB
phase_rad = np.angle(h)                   # 相位（弧度）
```

`freqz` 输出的是**每个频率成分经过滤波器后的幅度变化和相位变化**。复数 `h` 的模 = 幅度增益（被放大/衰减了多少倍），角度 = 相位旋转（被延迟了多少）。

**为什么幅度用 dB？** 因为幅度变化范围极大——通带接近 1.0，阻带可能是 0.00001。在线性尺度下，阻带细节完全不可见。dB 尺度（20 × log₁₀）把大范围压缩了：
- 0 dB = 幅度没变
- -20 dB = 衰减到 1/10
- -40 dB = 衰减到 1/100
- -60 dB = 衰减到 1/1000
- -80 dB = 衰减到 1/10000

**怎么看 `freqz` 的图？**
- 在通带内，曲线应该接近 0 dB（平坦）
- 在阻带内，曲线应该在很低的 dB 值（如 -50 dB 以下）
- 通带和阻带之间的斜坡宽度 = 过渡带宽度

### 5.2 `group_delay` — 查看延迟

```python
from scipy.signal import group_delay

freqs, gd = group_delay((b, 1), fs=252)
# gd: 群延迟（单位=样本数）——每种频率成分被延迟了多少个采样点
```

**什么是群延迟？** 不同频率的正弦波经过滤波器后，出来的时间可能不一样。群延迟就是每种频率的"通过时间"。

**FIR 滤波器（对称核）的核心优势**：群延迟是**常数**——所有频率被延迟完全相同的样本数。这意味着**输出波形不会扭曲**（只是整体向后平移了一段时间），这叫做**线性相位**。

对于对称 FIR，常数群延迟 = `(numtaps - 1) / 2` 个样本。51 阶滤波器 = 延迟 25 个样本。（对日度数据来说就是 25 天。）

### 5.3 完整验证清单

设计完滤波器后，检查这四项：

1. **频率响应符合预期吗？** 通带平坦（纹波 < 0.1 dB），阻带衰减足够（< -40 dB 及格，< -60 dB 优秀）
2. **过渡带在可接受范围内吗？** 太宽说明 numtaps 不够
3. **延迟在可接受范围内吗？** 用 `group_delay` 确认
4. **在实际信号上测试了吗？** 用测试数据验证效果（本章代码会演示）

## 6. 常见错误速查

| 你做的 | 发生的现象 | 原因 | 修法 |
|--------|-----------|------|------|
| 没传 `fs`，cutoff 直接用 Hz 值 | 滤波器完全不对 | scipy 默认 Nyquist=1，10 Hz 被解读为 10×Nyquist（远超合法范围） | 加上 `fs=你的采样率` |
| `numtaps` 是偶数 | 报错或相位异常 | 线性相位 FIR 强制要求奇数长度 | 改为奇数 |
| cutoff > Nyquist | 报错 | 要求的频率超过采样率能表示的上限 | 降低 cutoff 或增大采样率 |
| `numtaps` 太小 | 过渡带很宽，阻带泄漏严重 | 有限阶数无法支撑窄过渡带 | 增大 numtaps |
| 用了矩形窗 | 频率响应剧烈振荡 | Gibbs 现象 | 至少用 hann 或 hamming |

| 概念 | 一句话 |
|------|--------|
| 频率 | 描述"信号变化有多快"——低频=慢变化=趋势，高频=快变化=噪声 |
| 傅里叶变换 | 把信号拆成不同频率正弦波的叠加——类似 PCA 把数据转到主成分空间 |
| FIR 滤波器 | 只依赖有限个过去输入的滤波器——天然稳定，可以做到线性相位 |
| 频率响应 | 滤波器对不同频率正弦波的"通过程度"——描述滤波器行为的最核心指标 |
| 通带/阻带/过渡带 | 保留的频段 / 阻挡的频段 / 两者之间的过渡斜坡 |
| Nyquist 频率 | 采样率的一半——你能检测的最高频率上限 |
| Gibbs 现象 | 直接截断理想滤波器 → 频域产生振荡和泄漏 |
| 窗函数 | 让核两端平滑衰减 → 消除 Gibbs 振荡 |
| Kaiser 窗 | 一个 β 参数在"旁瓣衰减"和"过渡带宽度"之间自由调节 |
| `firwin` | scipy 中窗口法 FIR 滤波器的主设计函数 |
| `freqz` | 绘制频率响应——验证设计是否符合预期 |
| 群延迟 | 每种频率通过滤波器的"用时"——FIR 对称核 = 常数延迟 = 无波形失真 |
