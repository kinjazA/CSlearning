# 第8章 · 时频分析（STFT, 频谱图, 小波）

## 0. FFT 的盲区：频率会随时间变化

### 0.1 第7章 FFT 的一个致命局限

第 7 章你学会了 FFT——把整个信号从时域变到频域，得到一个"全时段的频谱"。这隐含了一个假设：**信号的频率成分在整段时间内是不变的。**

但现实中的信号往往不是这样。想想这些例子：
- 一首歌：前奏是低频的鼓点，副歌加入高频的人声和吉他
- 股票市场：平时是小幅波动（高频为主），危机期间出现剧烈的趋势性下跌（低频突然变强）
- 电网频率：正常情况下稳定在 50 Hz 附近，发生扰动时频率突然偏移

FFT 只能告诉你"整首歌里有哪些频率"，不能告诉你"鼓声出现在前 30 秒，人声出现在 30 到 90 秒"。**FFT 丢掉了时间信息。**

### 0.2 你需要的是"时间+频率"的二维视图

时频分析在同一个图里同时展示：
- x 轴 = 时间（什么时候）
- y 轴 = 频率（什么频率）
- 颜色/亮度 = 该频率在该时刻有多强

这就像一张"频谱随时间变化的快照"。你可以直观地看到：某个频率成分是从什么时候开始的、什么时候结束的、它的强度是如何演化的。

### 0.3 两种主流方法

| 方法 | 思路 | 优点 | 缺点 |
|------|------|------|------|
| **STFT / 频谱图** | 把信号切成小段，每段做 FFT | 直观、计算快 | 时间分辨率和频率分辨率互斥 |
| **连续小波变换 (CWT)** | 用不同尺度的"小波"去匹配信号 | 低频分辨高、高频时间精度高 | 计算量大、参数选择更复杂 |

## 场景一："我想看一个信号的频率如何随时间变化"

### 1.1 问题

你有一个信号——比如一段音频、一次地震记录、或者一天的电力负荷。你怀疑它的频率成分在变化（比如早上低频为主，中午高频成分增多），但不确定。FFT 只告诉你总体的频率分布，你需要看**频率的"演变过程"**。

### 1.2 STFT 做了什么（直觉版）

STFT = Short-Time Fourier Transform = 短时傅里叶变换。

一句话描述：**拿一个窗，在信号上从左向右滑动。每滑动一步，对窗内的那一小段信号做一次 FFT。把每次 FFT 的结果按时间顺序排列起来，就是时频谱。**

```python
from scipy.signal import stft

f, t, Zxx = stft(x, fs=fs, window='hann', nperseg=256, noverlap=128)
# f: 频率轴 (Hz)
# t: 时间轴 (s) — 每个 FFT 段对应的时刻
# Zxx: 复数时频谱 (shape: [len(f), len(t)])
#       |Zxx[f_idx, t_idx]| = 频率f在时刻t的幅度
```

### 1.3 最关键的权衡：时间分辨率 vs 频率分辨率

这是 STFT 中你必须理解的第一个概念。

**窗越长（nperseg 越大）→ 频率分辨率高，但时间分辨率低。** 你能精确知道"有 10.2 Hz 的成分"，但不知道它具体在哪一刻出现——它在窗内的位置被模糊了。

**窗越短（nperseg 越小）→ 时间分辨率高，但频率分辨率低。** 你能精确知道"第 3.5 秒发生了什么"，但只能模糊地判断"大概是低频"——10 Hz 和 12 Hz 在短窗内无法区分。

**这是物理定律，不是算法缺陷。** 海森堡不确定性原理在信号处理中的体现：时间精度和频率精度不能同时达到极致。你的任务是为具体场景选择最合适的折中点。

**选择指南**：

| 你的场景 | 窗长选择 | 原因 |
|---------|---------|------|
| 关心"什么频率在什么时刻开始/结束" | 短窗 | 时间精度优先 |
| 关心"有几个精确的频率成分" | 长窗 | 频率精度优先 |
| 两者都关心（最常见） | 中等窗 + 50% 重叠 | 折中 |

### 1.4 另一种便利函数：`spectrogram`

`spectrogram` 是 STFT 的封装，直接返回功率谱密度（幅度的平方），省去手动处理复数的麻烦：

```python
from scipy.signal import spectrogram

f, t, Sxx = spectrogram(x, fs=fs, window='hann',
                        nperseg=256, noverlap=128)
# Sxx: 频谱图（实数，每个时频格的功率）
```

`spectrogram` 和 `stft` 的关系：`Sxx ≈ |Zxx|²`（更精确地说，Sxx 是功率谱密度估计）。如果你只想看"频谱长什么样"，用 `spectrogram` 更方便；如果你需要复数值做后续处理，用 `stft`。

### 1.5 逆变换 `istft` — 从时频谱恢复信号

```python
from scipy.signal import istft

t_recovered, x_recovered = istft(Zxx, fs=fs, window='hann',
                                  nperseg=256, noverlap=128)
```

这在某些场景下很有用：你可以在时频谱上编辑（比如删除某个时间段的某个频率成分），然后逆变换回时域信号——这就是很多音频编辑软件"频谱编辑"功能的底层原理。

## 场景二："我需要更好的低频分辨率"

### 2.1 STFT 的又一个局限

STFT 用固定长度的窗。这意味着：**所有频率上的时间分辨率和频率分辨率都是一样的。** 窗长 256 个点决定了低频和高频都是同一个时间/频率分辨率。

但很多实际信号在不同频段有不同的特性：
- 低频成分（如趋势）变化缓慢——你不需要毫秒级的时间精度
- 高频成分（如瞬态脉冲）变化极快——你不需要 0.01 Hz 的频率精度

你真正想要的是：**低频时频率分辨率高（时间精度可以牺牲），高频时时间分辨率高（频率精度可以牺牲）。**

### 2.2 小波变换 (CWT) 的思路

小波变换用"可变大小的窗"——低频时用宽窗（捕获完整的周期），高频时用窄窗（精确定位变化时刻）。

```python
from scipy.signal import cwt

# cwt(data, wavelet, widths)
# widths: 不同尺度（对应于不同的"频率"）
# wavelet: 小波基函数

widths = np.arange(1, 128)
cwt_result = cwt(x, signal.ricker, widths)
# cwt_result: (len(widths), len(x)) — 每个尺度在每个时刻的系数
```

### 2.3 小波基的选择

小波基是一个短促的"小波形"，在时域和频域都集中在一个小范围内。scipy 提供两种内置小波：

| 小波 | 形状 | 适合什么 |
|------|------|---------|
| `ricker`（墨西哥帽） | 中间一个正峰、两边负的裙边 | 通用——检测信号中的"峰"或"不连续点" |
| `morlet2` | 正弦波 × 高斯包络 | 更接近"传统"时频分析——适合检测振荡成分 |

`morlet2` 有一个额外参数 `w`（中心频率），控制"振荡"和"包络"的相对比例。w 越大 → 振荡越多 → 频率分辨越好但时间分辨越差。

### 2.4 CWT vs STFT：什么时候用小波？

| 场景 | 推荐 | 原因 |
|------|------|------|
| 快速浏览信号的时频概貌 | STFT / 频谱图 | 计算快、直观（频率轴清晰） |
| 检测瞬态、尖峰、不连续点 | CWT (ricker) | 对局部突变敏感 |
| 分析有多个时间尺度的信号（如金融数据的多尺度波动） | CWT | 可变尺度 = 不同尺度捕捉不同"速度"的变化 |
| 需要从时频谱恢复信号 | STFT + istft | 有现成的逆变换 |
| 不确定用哪个 | STFT / 频谱图 | 默认先用这个 |

## 场景三："我想追踪一个随时间变化的频率——比如 chirp 信号"

### 3.1 问题

你有一个 chirp 信号（频率从低扫到高），或者电网频率在缓慢漂移。你想精确追踪"瞬时频率"——每个时刻的主频率是多少。

### 3.2 做法

频谱图让你看到"频率随时间的变化轨迹"。要精确追踪这条轨迹（即时频脊线，ridge），常用的方法是：在每个时间点，找到频谱图中能量最大的频率——这就是该时刻的瞬时频率估计。

```python
f, t, Sxx = spectrogram(signal, fs=fs)
# 在每个时刻找能量最大的频率
instantaneous_freq_idx = np.argmax(Sxx, axis=0)  # 每列的最大值索引
instantaneous_freq = f[instantaneous_freq_idx]     # 对应的频率值
```

对于有噪声的信号，这个简单方法可能不稳定——噪声会在某些时刻能量超过真实信号。更 robust 的方法（如动态规划脊线追踪）超出了 scipy 的范围，但在实际中可以结合中值滤波等后处理来平滑瞬时频率轨迹。

## 4. 参数选择速查

### STFT / 频谱图

| 参数 | 作用 | 典型值 |
|------|------|--------|
| `nperseg` | 每段长度 → 控制频率分辨率 | 信号长度的 1/8 ~ 1/16 |
| `noverlap` | 段间重叠 → 让时间轴更平滑 | nperseg 的 50%~75% |
| `window` | 窗函数 → 减少频谱泄漏 | `'hann'`（通用） |

### CWT

| 参数 | 作用 | 典型值 |
|------|------|--------|
| `widths` | 尺度数组 → 每个尺度对应一个"频率" | `np.arange(1, 128)` |
| `wavelet` | 小波基 | `ricker`（检测瞬态）、`morlet2`（检测振荡） |

## 5. 本章要点速查

| 概念 | 一句话 |
|------|--------|
| FFT 的局限 | 告诉你"有什么频率"，不告诉你"频率何时出现" |
| STFT | 滑动窗 FFT → 每个时刻一小段 FFT |
| 频谱图 (spectrogram) | STFT 的可视化——x=时间, y=频率, 颜色=强度 |
| 时间 vs 频率分辨率 | 窗长决定：长窗=精频率粗时间，短窗=精时间粗频率。物理定律，无法两全。 |
| CWT | 小波变换——用可变尺度的窗，低频精频率、高频精时间 |
| `ricker` 小波 | 墨西哥帽形状 → 检测信号的峰和不连续点 |
| `morlet2` 小波 | 正弦×高斯 → 检测振荡成分 |
| `istft` | STFT 的逆变换——从时频谱恢复信号 |
