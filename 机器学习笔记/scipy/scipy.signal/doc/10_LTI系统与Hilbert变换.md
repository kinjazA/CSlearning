# 第10章 · LTI 系统分析与 Hilbert 变换

---

## 0. 这一章的两个主题为什么放在一起？

本章包含两个独立但互补的主题：

**LTI 系统分析**：把信号处理的对象（滤波器、物理系统）当作一个"黑盒子"来研究——输入进去什么，输出出来什么。关注系统的频率响应、冲击响应、稳定性。

**Hilbert 变换**：给信号做一个数学变换，提取信号的"包络"和"瞬时频率"——不再看信号本身，而是看它的"轮廓"和"节奏变化"。两者放在一起，因为它们是"信号分析"的两面：LTI 分析系统本身，Hilbert 分析信号的瞬时特性。

## Part A：LTI 系统分析

### A.1 什么是 LTI 系统？

**LTI = Linear Time-Invariant = 线性时不变。**

- **线性**：输入放大两倍，输出也放大两倍。两个输入的和 → 两个输出的和。这就是你学过的"线性"——OLS 回归也是线性的。
- **时不变**：系统的行为不随时间改变。今天输入一个脉冲和明天输入同样的脉冲，输出完全相同。

为什么重要？因为很多真实系统可以近似为 LTI：
- 一个电路（电阻、电容、电感组成的滤波器）
- 一个机械系统（弹簧+质量块+阻尼器）
- 一个经济系统的简化模型（利率变动对 GDP 的影响路径）
- 你之前设计的 FIR 和 IIR 滤波器——它们就是 LTI 系统

**和你的统计知识对接**：你在时间序列课上学过的**脉冲响应函数（IRF）**——VAR 模型里对某一变量施加一个冲击，观察其他变量随时间如何响应——那就是 LTI 系统的"脉冲响应"概念。学信号处理的 LTI 就是把这个概念一般化。

### A.2 系统的三种数学表示

在 scipy 中，一个 LTI 系统有三种等价表示：

#### ① 传递函数形式 — `(b, a)`

```python
from scipy.signal import TransferFunction

# b = 前馈系数（分子），a = 反馈系数（分母）
tf = TransferFunction(b, a)
```

这就是你第5章学的 IIR 滤波器的 `(b, a)` 格式。传递函数是输出和输入在频域的比值：
$$H(z) = \frac{b_0 + b_1 z^{-1} + \cdots + b_M z^{-M}}{a_0 + a_1 z^{-1} + \cdots + a_N z^{-N}}$$

**直觉**：$H(z)$ 告诉你"系统对不同频率的输入各放大/衰减多少、各延迟多少"。

#### ② 零极点增益形式 — `(z, p, k)`

```python
from scipy.signal import ZerosPolesGain

zpk = ZerosPolesGain(z, p, k)
# z = 零点数组（使 H(z)=0 的 z 值）
# p = 极点数组（使 H(z)→∞ 的 z 值）
# k = 增益因子
```

**零点和极点的直觉**：
- **零点** = 被系统"完全扼杀"的频率。如果输入恰好是零点对应的频率，输出 = 0。
- **极点** = 系统的"共振频率"。输入接近极点频率时，输出被极大放大。极点在单位圆内 = 系统稳定（输入消失后输出会衰减到零）；极点在单位圆外 = 系统不稳定（会自激振荡）。

**统计学类比**：零点就像回归中系数恰好为零的变量（完全不贡献）。极点就像回归中系数极大的变量（主导输出行为）。

#### ③ 状态空间形式 — `(A, B, C, D)`

```python
from scipy.signal import StateSpace

ss = StateSpace(A, B, C, D)
```

状态空间是 LTI 最通用的表示。它把系统描述为一组一阶微分（或差分）方程。对于复杂的多输入多输出系统（如控制系统），状态空间是最自然的形式。

本书聚焦前两种表示——它们和滤波器的关系更直接。状态空间在第11章会简单用到。

#### 三种形式互相转换

```python
from scipy.signal import tf2zpk, zpk2tf, tf2ss, ss2tf, ss2zpk

z, p, k = tf2zpk(b, a)       # 传递函数 → 零极点增益
b, a     = zpk2tf(z, p, k)   # 零极点增益 → 传递函数
A,B,C,D  = tf2ss(b, a)       # 传递函数 → 状态空间
b, a     = ss2tf(A,B,C,D)    # 状态空间 → 传递函数
z, p, k  = ss2zpk(A,B,C,D)   # 状态空间 → 零极点增益
```

### 场景一："我想知道系统对不同频率的响应"

#### 频率响应 `freqresp`

```python
from scipy.signal import freqresp

w, H = freqresp(tf, w=np.logspace(-1, 2, 1000))
# w: 频率数组 (rad/s)
# H: 复数频率响应 H(jω)
```

`freqresp` 和第3-5章的 `freqz` 本质相同——都告诉你"系统对不同频率各放大/衰减多少"。区别是：`freqz` 是针对滤波器系数 `(b, a)` 的便捷函数，`freqresp` 是针对 `TransferFunction` 对象的通用接口。

#### Bode 图 `bode`

```python
from scipy.signal import bode

w, mag, phase = bode(tf)
# mag:  幅度 (dB)
# phase: 相位 (度)
```

Bode 图是控制系统工程的标准可视化——上图画幅度（dB），下图画相位（度）。它让你一眼看到"系统在什么频率范围内增益大约是多少，什么频率开始衰减，衰减速率是多少"。

直观解读：如果 Bode 图在 10 Hz 附近从 0 dB 开始快速下降到 -40 dB 以下 → 这个系统对 10 Hz 以上的频率有强抑制作用 → 它相当于一个低通滤波器。

### 场景二："我想知道系统对冲击和阶跃的响应"

#### 冲击响应 `impulse`

```python
from scipy.signal import impulse

t, y = impulse(tf)
# 输入 = 单位脉冲 δ[n]（第1章学过）
# y = 系统对脉冲的输出
```

冲击响应是 LTI 系统的"身份标识"——知道冲击响应，就知道系统对所有可能输入的响应（通过卷积）。

**统计类比**：VAR 模型里你给变量 $X_1$ 一个单位冲击，观察 $X_2$ 在随后的时间里如何响应——这就是冲击响应。如果你把 VAR 看作一个 LTI 系统，VAR 的冲击响应函数就是 `impulse` 的输出。

#### 阶跃响应 `step`

```python
from scipy.signal import step

t, y = step(tf)
# 输入 = 从0突然跳到1并保持
# y = 系统的输出如何从0演化到稳态
```

阶跃响应回答："如果输入从一个水平突然跳到另一个水平，输出会怎样？"

- 输出需要多久达到稳态（稳定时间的 90%）？→ 系统的"响应速度"
- 输出有没有超过稳态值（超调）？→ 系统的"稳定性余量"
- 输出最后稳定在什么值？→ 系统的"DC 增益"

### 场景三："我想把多个系统串联或并联"

```python
from scipy.signal import series, parallel, feedback

# 串联：tf3 = tf1 → tf2
tf_series = series(tf1, tf2)

# 并联：tf3 = tf1 + tf2
tf_parallel = parallel(tf1, tf2)

# 反馈：tf_out = tf_forward / (1 + tf_forward × tf_feedback)
tf_closed = feedback(tf_forward, tf_feedback)
```

**实际意义**：
- 串联：两个滤波器的级联——先用低通去噪，再用带通提取特定频段
- 并联：两条滤波路径的结果相加
- 反馈：**控制系统最重要的结构**——输出的一部分被"返回"到输入，用来调整系统行为

## Part B：Hilbert 变换

### B.1 它能做什么？

Hilbert 变换解决一个具体问题：**给定一个振荡信号，怎么提取它的"包络"（轮廓）和"瞬时频率"（每个时刻的主频率）？**

第9章你学会了用峰值检测 + 插值来粗略提取包络。Hilbert 变换给出数学上严格的包络和瞬时频率。

### B.2 解析信号是什么？

一个实信号 $x(t)$ 的解析信号是：
$$x_a(t) = x(t) + j \cdot \mathcal{H}\{x(t)\}$$

其中 $\mathcal{H}\{x(t)\}$ 是 $x(t)$ 的 Hilbert 变换。$j$ 是虚数单位。

**直觉理解**（不涉及数学细节）：
- 原始信号 $x(t)$ 是你看到的振荡
- Hilbert 变换 $\mathcal{H}\{x(t)\}$ 是同一个振荡"旋转了 90°"的版本
- 两者合在一起（实部 + 虚部），就是一个"旋转矢量"的坐标

这个旋转矢量的**模** = 信号的瞬时包络（幅度轮廓）。这个旋转矢量的**角速度** = 瞬时频率。

**统计学类比**：解析信号就像把一维的时间序列扩展成一个二维的"相位空间"。在原始的一维序列中你看不到的"旋转速度"，在二维相位空间中变得一目了然。

### B.3 使用方式

```python
from scipy.signal import hilbert

# 计算解析信号
analytic = hilbert(real_signal)

# 瞬时包络 = 解析信号的模
envelope = np.abs(analytic)

# 瞬时相位 = 解析信号的角度
instantaneous_phase = np.angle(analytic)

# 瞬时频率 = 相位的导数（变化率）
instantaneous_freq = np.diff(np.unwrap(instantaneous_phase)) / (2 * np.pi)
# 如果有采样间隔 dt，除以 dt 得到 Hz
instantaneous_freq_hz = instantaneous_freq / dt
```

**`np.unwrap` 的作用**：相位在被限制在 $[-\pi, \pi]$ 范围内，每转一圈就会从 $\pi$ 跳到 $-\pi$（这叫相位缠绕）。`unwrap` 把这跳变修复为连续的变化——就像把一卷缠绕的线解开。

### B.4 包络和峰值包络的区别

- **峰值包络**（第9章）：只在峰的位置有值，中间靠插值。峰检测的质量直接影响包络。噪声下易受单峰误检影响。
- **Hilbert 包络**：每个采样点都有值，基于整个信号的数学变换。光滑、连续。对单个噪声尖峰不敏感。

**什么时候用哪个？**
- 快速看一眼信号的轮廓 → 峰值包络（计算快）
- 精确的包络分析、需要计算瞬时频率 → Hilbert（数学严格）

### 场景四："我想追踪信号的包络"

```python
analytic = hilbert(signal)
envelope = np.abs(analytic)
```

**应用**：
- 电力系统：电压信号的包络变化 → 判断是否存在振荡或不稳定
- 金融：波动率序列的包络 → 识别高波动期和低波动期的交替
- 结构健康监测：振动响应的包络衰减速度 → 判断阻尼是否正常

### 场景五："我想知道信号的瞬时频率——每个时刻的频率是多少"

这是 Hilbert 变换最强大的应用之一。FFT 给你"整个时段的平均频率分布"，瞬时频率给你"每个时刻的主频率"。

```python
phase = np.unwrap(np.angle(analytic))
inst_freq = np.diff(phase) / (2 * np.pi * dt)
```

**应用**：
- 电网频率监控：标称 50Hz，实际微变。瞬时频率追踪比 FFT 更灵敏地检测频率偏移。
- 雷达/声纳：chirp 信号的瞬时频率线性变化 → 验证发射机是否正常工作。

## 6. 本章要点速查

| 概念 | 一句话 |
|------|--------|
| LTI 系统 | 线性 + 时不变——大多数滤波器和物理系统的基本模型 |
| 传递函数 `(b,a)` | 系统在频域的输出/输入比值 |
| 零极点 `(z,p,k)` | 零点=扼杀频率，极点=共振频率。单位圆内=稳定 |
| `impulse` / `step` | 冲击响应=系统的"指纹"，阶跃响应=系统的"反应速度" |
| `bode` | 控制系统标准可视化——幅度+相位两张图 |
| `series` / `feedback` | 串联和反馈——搭建复杂系统的基本积木 |
| Hilbert 变换 | 把实信号变成"解析信号"——数学上严格的包络和瞬时频率 |
| 瞬时包络 | `np.abs(analytic)` — 信号的轮廓线 |
| 瞬时频率 | `np.diff(phase)` — 每个时刻的主频率 |
| `unwrap` | 解开相位的 $[-\pi,\pi]$ 缠绕，让相位连续变化 |
