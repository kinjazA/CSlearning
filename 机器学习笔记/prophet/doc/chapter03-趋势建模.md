# 第 3 章：趋势建模

趋势 $g(t)$ 是 Prophet 最重要的组件，描述了序列在忽略周期性波动后的长期变化方向。

---

## 3.1 两种趋势模型

### 3.1.1 线性趋势 (Linear Growth) — 默认

$$g(t) = (k + a(t)^T \delta) \cdot t + (m + a(t)^T \gamma)$$

- $k$：基础增长率
- $\delta$：各个变点处的增长率调整量
- $m$：偏移量 (offset)
- $a(t)$：指示向量，标记 $t$ 属于哪些变点区间

**适用场景**：大多数业务指标（收入、用户数、订单量），因为不存在物理上限。

```python
model = Prophet(growth='linear')  # 默认值，可不写
```

### 3.1.2 逻辑增长 (Logistic Growth)

$$g(t) = \frac{C(t)}{1 + \exp(-(k + a(t)^T \delta)(t - (m + a(t)^T \gamma)))}$$

- $C(t)$：**时变承载能力 (carrying capacity)**
- 趋势被约束在 $[0, C]$ 或 $[floor, cap]$ 之间

**适用场景**：有明确上限的指标（市场占有率、订阅渗透率、硬件容量）。

```python
model = Prophet(growth='logistic')
df['cap'] = 100.0      # 必须提供饱和上限
future['cap'] = 100.0
```

---

### 3.1.3 方法论插曲：不是所有序列都有趋势——差分 vs 直接建模

你可能会想：**如果我对原始序列做差分（$y_t - y_{t-1}$），趋势不就消失了吗？那我直接在差分后的序列上预测，再还原回去，不就行了？**这恰恰触及了时间序列预测中**两种根本不同的哲学**。理解这个分歧，你才能判断 Prophet 适合哪种场景、不适合哪种场景。

---

#### 平稳性：一个必须理解的概念

**平稳性 (Stationarity)** 的直觉定义：一条序列的统计属性——均值、方差、自相关——不随时间变化。

```
平稳序列：                        非平稳序列（有趋势）：
  │    ╱╲  ╱╲   ╱╲                  │                  ╱
  │   ╱  ╲╱  ╲ ╱  ╲                │               ╱
  │  ╱        ╲    ╲               │            ╱
  │ ╱          ╲                  │         ╱
  │╱            ╲──               │─────╱╱
  └──────────────                 └────────────────
  围绕固定均值波动                  均值随时间漂移
```

**为什么平稳性重要**：大多数经典统计方法（ARIMA、线性回归）假设序列是平稳的。如果序列有趋势，这些方法会失效——它们会把趋势误读为"永久性的序列结构"，而不是"需要剔除的干扰"。

---

#### 两条路线：差分 vs 直接建模

面对有趋势的序列，统计学历史上分化出两条路线：

```
非平稳序列
    │
    ├── 路线 A: 差分 (Differencing)
    │        └── ARIMA, SARIMA, ...
    │            1. y'_t = y_t - y_{t-1}     （消除趋势）
    │            2. 对平稳的 y' 建模
    │            3. 反向累加还原预测
    │
    └── 路线 B: 直接建模趋势 (Explicit Trend Modeling)
             └── Prophet, ETS, 结构时间序列, ...
                 1. y_t = Trend(t) + Season(t) + Noise
                 2. 同时估计趋势和季节
                 3. 趋势本身就是你要外推的东西
```

---

#### 路线 A：差分的思路（ARIMA 家族）

**一阶差分**：$y'_t = y_t - y_{t-1}$

```
原始序列 y:
  │          ╱
  │       ╱                    ← 有明显上升趋势
  │    ╱
  │╱╱
  └────────────

一阶差分 y':
  │  ╱╲ ╱╲  ╱╲ ╱╲
  │ ╱  ╲  ╲╱  ╲  ╲            ← 趋势消失，变成围绕 0 的波动
  │╱            ╲
  └────────────────
```

差分的本质：**我不关心趋势是什么——我直接把它从数据中"减掉"。** 预测时再做反向操作——把差分预测累加回去（积分）。

**优势**：
- 不需要假设趋势的形状（线性？抛物线？S 曲线？都不需要管）
- 对"趋势本身不可预测"的序列效果好（如股票价格）
- ARIMA 的差分阶数 $d$ 提供了一套系统的"去趋势"框架

**劣势**：
- 丢掉了趋势的信息——你不能回答"明年的增长率是多少"
- 长期预测时，累加操作会把短期误差放大
- 不能灵活地结合节假日、变点等结构化知识
- **最重要的是：差分假设趋势是"随机游走"，而大多数业务指标的趋势是结构性的（增长、饱和、转折），不是随机漫步**

---

#### 路线 B：Prophet 的思路

Prophet 选择路线 B——**不差分，直接给趋势建模**。

$$y(t) = \underbrace{g(t)}_{\text{显式趋势}} + s(t) + h(t) + \varepsilon$$

它的逻辑是：

1. **业务序列的趋势通常是有意义的**——"为什么增长？因为产品更好了、市场更大了"——这不是噪声，这是信号
2. **把趋势分解出来，你才能理解和干预**——"增长在 2023 年 Q2 降速了，是因为竞品上线"——这个洞察差分给不了你
3. **趋势本身是你最想预测的东西**——"明年体量会到多少？"——差分方法避开了这个问题，Prophet 直面它

---

#### 直接回答你的问题：差分后再用 Prophet 行不行？

技术上可以做，但你等于在做一件矛盾的事：**先用差分消灭了趋势，再让 Prophet 去找趋势**。

```python
# 伪代码：差分 + Prophet（不推荐）
df['y_diff'] = df['y'].diff()                    # Step 1: 差分（趋势消失）
model = Prophet()                                 # Step 2: Prophet 试图找趋势
model.fit(df.rename(columns={'y_diff': 'y'}))     # Step 3: 但趋势已经没了
# Prophet 的 g(t) 会拟合一条近乎水平的线
# → 你花了 Prophet 的成本，得到了 ARIMA 的效果
```

**如果你已经决定用差分，直接用 ARIMA/SARIMA，不要绕道 Prophet。** Prophet 的整个设计——变点检测、拉普拉斯先验、傅里叶季节性——都是围绕"趋势是真实存在的信号"这个前提构建的。

---

#### 序列没有趋势的情况

你的问题反过来也成立：**如果序列本身就没有趋势呢？**

```python
# 例：一个已经平稳的序列
# → 围绕固定均值波动，没有长期上升或下降
```

这种情况下：

| 做法 | 效果 |
|------|------|
| 用 ARIMA（$d=0$）| ✅ 天然适合——平稳序列是 ARMA 的理想场景 |
| 用 Prophet | ⚠️ 可用但过度设计——Prophet 的变点机制会尝试在噪声中找到"伪趋势" |

**如果你明知序列没有趋势**（比如已经做过平稳性检验），你可以把 `changepoint_prior_scale` 设得非常小（0.001），让 Prophet 的趋势线几乎水平——这样它退化为一个"季节性 + 节假日"的 GAM。但说实话，这种情况用更轻量的方法可能更好。

一个简单的判断方法：

```python
from statsmodels.tsa.stattools import adfuller

result = adfuller(df['y'].dropna())
print(f"ADF p-value: {result[1]:.4f}")
# p < 0.05 → 序列很可能已经平稳 → 趋势很弱或不存在
# p > 0.05 → 序列非平稳 → 存在趋势 → Prophet 适用
```

---

#### 第三条路：把 Prophet 的趋势分量和差分结合起来（进阶）

这不是"差分后给 Prophet"，而是反过来：**用 Prophet 提取趋势分量，然后对残差做更精细的建模**。

```python
# 1. Prophet 提取趋势和季节
model = Prophet()
model.fit(df)
forecast = model.predict(df[['ds']])

# 2. 残差 = 原始值 - Prophet 的趋势 - Prophet 的季节
df['residual'] = df['y'] - forecast['trend'] - forecast['yearly'] - forecast['weekly']

# 3. 残差可能是平稳的 → 用 ARIMA 精细建模
# 这相当于 Prophet 处理"大结构"，ARIMA 处理"剩余波动"
```

这就是第 14 章提到的混合预测的另一个变体——不只和 GBDT 混合，也可以和 ARIMA 混合。

---

#### 总结：差分 vs 直接趋势建模的选择指南

| 问自己 | 如果答案是"是" | 如果答案是"否" |
|--------|--------------|--------------|
| 趋势本身是我关心的输出吗？ | ✅ Prophet（路线 B） | → 下一题 |
| 趋势有可解释的结构性原因吗？ | ✅ Prophet（路线 B） | → 下一题 |
| 我有 < 30 个数据点吗？ | → 用简单方法（移动平均） | → 下一题 |
| 序列看起来像随机游走吗？ | ✅ 考虑差分（路线 A） | ✅ Prophet（路线 B） |

一个实用的经验法则：**如果你能对业务方说出"为什么这个序列在涨/跌"，你就应该用 Prophet 把趋势显式建模出来。如果你只能说"它就是随机在波动"，那差分方法更合适。**

## 3.2 变点 (Changepoints)：Prophet 如何让趋势"学会拐弯"

### 3.2.1 直觉：趋势不是一条直线

现实中的业务指标几乎不会匀速增长。产品上线、政策变化、竞品入场……这些事件会**改变增长斜率**。Prophet 用变点 (changepoints) 来刻画这种"趋势在特定时间点发生弯曲"的行为。

数学上，Prophet 把整段时间切成若干段，每段有自己的斜率。相邻段之间的斜率差就是 $\delta_j$——第 $j$ 个变点处的**增长率变化量**。

```
            变点1        变点2
              │            │
    ──────┐  ／            │
           │／  斜率变大     │  斜率变小
           │               ＼
           │                 ＼────────
    第1段    第2段             第3段
    δ₁≈0    δ₂=+0.5          δ₃=-0.3
```

> 关键问题来了：**Prophet 怎么知道哪些地方应该有变点、变多大？** 这就引出了贝叶斯视角下的"先验"概念。

---

### 3.2.2 插入概念：贝叶斯推断中的"先验"是什么

在进入变点细节之前，先用一个类比建立直觉。

**医检类比**：你去医院做检查，医生面对两个信息来源：

| 信息来源 | 对应 | 含义 |
|----------|------|------|
| 检查报告（数据） | **似然 (Likelihood)** | "在真实病情下，检查结果呈现阳性的概率是多少" |
| 该病的普遍发病率 | **先验 (Prior)** | "在没做检查之前，你得这病的概率有多大" |
| 综合诊断 | **后验 (Posterior)** | "结合检查结果和发病率，你实际得病的概率" |

用贝叶斯公式表达：

$$P(\text{有病} \mid \text{阳性}) = \frac{P(\text{阳性} \mid \text{有病}) \cdot P(\text{有病})}{P(\text{阳性})}$$

$$\text{后验} \propto \text{似然} \times \text{先验}$$

**先验的作用**：当数据（似然）不够强时，先验会"拉回"估计值，防止被噪声误导。这就是 Prophet 变点检测的核心机制。

---

### 3.2.3 拉普拉斯先验：让大多数 δⱼ "几乎为零"

Prophet 给每个候选变点的 $\delta_j$（增长率变化量）赋予一个 **拉普拉斯先验 (Laplace Prior)**：$$\delta_j \sim \text{Laplace}(0, \tau)$$

翻译成人话：**在没看到任何数据之前，Prophet "不相信"任何地方有变点——它默认所有 δⱼ 都是 0。只有数据提供了足够强的"证据"，δⱼ 才会显著偏离 0。**

#### 为什么是拉普拉斯而不是高斯？

这是整个设计的精髓。对比两种分布在 0 附近的形状：

| 分布 | 概率密度 | 在 0 附近的形状 | 产生的效果 |
|------|---------|----------------|-----------|
| **高斯先验** $\delta \sim N(0, \sigma^2)$ | $\propto \exp(-\delta^2/2\sigma^2)$ | 圆滑抛物线 | δ 趋近于 0 但不精确为 0，很多"微小但非零"的参数 |
| **拉普拉斯先验** $\delta \sim \text{Laplace}(0, \tau)$ | $\propto \exp(-|\delta|/\tau)$ | **尖峰** + **厚尾** | 大量 δ 被精确压缩到 0 附近，少数 δ 被允许很大 |

```
概率密度
  │
  │      ┌─ 拉普拉斯：尖峰 → 绝大多数 δ≈0
  │     ╱│╲
  │    ╱ │ ╲
  │   ╱  │  ╲
  │  ╱   │   ╲___ 拉普拉斯：厚尾 → 少数 δ 可以很大
  │ ╱    │       ╲___
  │╱     │           ╲___
  ││─────│───────────────│───│──→ δ
  0
  │      └─ 高斯：圆顶 → 很多 δ 在 0 附近但非零
  │         (微弱的、分散的偏转)
```

**拉普拉斯先验 = 自动特征选择**。25 个候选变点中，数据可能只支持其中 3-5 个有实质性的斜率变化，剩下的 δⱼ 被先验压缩到 ∼0——模型自动完成了"稀疏化"。

> 这就是你在提问中提到的"参数约等于 0"的含义：Prophet 放了 25 个候补变点，但拉普拉斯先验确保**大部分 δⱼ ≈ 0**（该段趋势没拐弯），只有少数被数据"激活"的变点有非零的 δⱼ。

#### 用 L1 正则化的视角理解

如果你熟悉机器学习中的 L1 vs L2 正则化：

| 正则化 | 贝叶斯对应 | 效果 |
|--------|-----------|------|
| **L2 正则** $\lambda \sum w_i^2$ | 高斯先验 | 所有参数缩小，但不为零 |
| **L1 正则** $\lambda \sum \|w_i\|$ | 拉普拉斯先验 | 产生稀疏解，很多参数精确为零 |

Prophet 的 `changepoint_prior_scale`（τ）就是控制这个"正则化强度"的旋钮。

---

### 3.2.4 `changepoint_prior_scale` (τ)：控制"你需要多强的证据才肯拐弯"

回忆拉普拉斯分布：$\delta_j \sim \text{Laplace}(0, \tau)$，τ 是拉普拉斯分布的**尺度参数 (scale)**，它控制了先验的"松紧"：

```
τ=0.001（极紧）           τ=0.05（默认）            τ=5.0（极松）
    │                       │                       │
  ╱ │ ╲                  ╱  │  ╲                ╱   │   ╲
 ╱  │  ╲               ╱    │    ╲            ╱     │     ╲
╱───│───╲            ╱──────│──────╲        ╱───────│───────╲
-0.01 0 +0.01      -0.5   0   +0.5      -5      0      +5

"数据再强我也不信"    "数据说话，适度灵活"    "数据说啥我都信"
→ 趋势几乎是直线       → 捕捉真正结构变化       → 追着噪声跑，过拟合
```

| τ 值 | 先验宽度 | 含义 | 表现 |
|------|---------|------|------|
| **0.001** | 极窄 | "我不信有任何变点" | 趋势几乎线性，只有极大证据才能让 δ 偏离 0 |
| **0.01** | 偏窄 | "我只信非常确定的变点" | 趋势平缓，大转折被捕捉，小波动被忽略 |
| **0.05** (默认) | 适中 | "让数据适度说话" | 明显趋势变化被捕捉，噪声被平滑 |
| **0.5** | 偏宽 | "我倾向于相信有很多变点" | 趋势灵活，可能追踪到噪声 |
| **5.0** | 极宽 | "每个波动都可能是变点" | 严重过拟合，趋势曲线追着每个数据点跑 |

**一句话总结**：τ 小 = 趋势僵硬，τ 大 = 趋势痉挛。最优值在中间，通过交叉验证确定。

---

### 3.2.5 变点的完整工作流

```
Step 1: 在前 80% 历史数据中均匀放置 25 个候选变点位置
         │
         ▼
Step 2: 对每个候选位置，引入一个 δⱼ（斜率变化量）
         │
         ▼
Step 3: 给所有 δⱼ 一个 Laplace(0, τ) 先验 —— "默认都是零"
         │
         ▼
Step 4: 用数据拟合（MCMC 或 MAP），数据"证据"会推动需要变≠0
         │
         ▼
Step 5: 得到后验：大多数 δⱼ ≈ 0，少数几个显著不为零
         │
         ▼
Step 6: 趋势 = 基斜率 + 各变点处被激活的 δⱼ
```

### 3.2.6 代码：变点检测与可视化

```python
from prophet import Prophet
from prophet.plot import add_changepoints_to_plot
import matplotlib.pyplot as plt

# 自动检测
model = Prophet(n_changepoints=25, changepoint_range=0.8)
model.fit(df)
forecast = model.predict(future)

# 可视化变点位置
fig = model.plot(forecast)
add_changepoints_to_plot(fig.gca(), model, forecast)
plt.show()

# 查看检测到的变点日期和强度
changepoints = model.changepoints           # 变点日期
deltas = model.params['delta'].mean(0)       # 平均斜率变化量
for cp, d in zip(changepoints, deltas):
    if abs(d) > 1e-4:                        # 过滤掉 ≈0 的变点
        print(f"变点 {cp.date()}: δ = {d:.4f}")
```

你也可以从业务上手动指定变点：

```python
model = Prophet(changepoints=['2023-01-15', '2023-06-01'])
```

> ⚠️ 手动指定后，`n_changepoints` 和 `changepoint_range` 会被忽略，Prophet 不再做自动检测。

### 3.2.7 `changepoint_range`：为什么 Prophet 只在历史前 80% 检测变点

`changepoint_range` 的默认值是 `0.8`。它的作用是：**变点只能出现在历史数据的前 80% 范围内，最后 20% 的数据禁止出现变点。**

```
changepoint_range = 0.8

  历史数据                                    未来
  |······························|          |·····|
  |←── 80% ──→|←── 20% ──→|                |预测期|
  |           |            |
  变点可以     变点禁止     预测时不再产生新变点
  出现在这     出现在这
  段区域       段区域
```

#### 为什么需要这个限制

这个设计源于 Prophet 作者对**变点检测的根本约束**：**你不能在序列的最末尾可靠地判断那里是不是一个变点。**

原因很简单——判断一个点是不是"趋势的转折"，你需要看到**它之后的数据**来确认。如果最后一个数据点突然跳了一下，它可能真的是趋势拐弯，也可能只是一个随机波动。只有过一段时间回头看，你才能分辨。

```
情况 A: 真的是拐弯                    情况 B: 只是随机波动
  │              ╱                      │  ╱╲
  │           ╱╱                        │ ╱  ╲
  │        ╱╱                           │╱    ╲
  │─────╱╱                              │       ╲
  │  ╱╱                    ← 最后       │        ╲────
  │╱                       一个点       │
  └────────────                        └────────────
  应该检测到变点                         不应该检测到变点
                                          (但只看最后一个点无法区分)
```

如果在最后一个点就允许变点，模型会在每个序列末尾"发现"趋势转折——其中大部分是噪声。这导致**预测的初始方向不可靠**——预测从最后几个点开始，如果那几个点恰好是噪声波动，趋势就会被带偏。

#### 为什么是 80% 而不是 90% 或 70%

`0.8` 是一个经验上的保守值，平衡了三点：

| 因素 | 影响 | 倾向 |
|------|------|------|
| **确认窗口** | 变点后需要足够的数据来"确认"这个拐弯是真实的 | 最后多留一些 → 倾向小值 |
| **灵敏度** | 最近的结构变化对预测最重要——如果去年发生了一个重大转折，你希望模型捕捉到 | 范围尽可能靠后 → 倾向大值 |
| **末端稳定性** | 变点越靠后，趋势在预测起点的斜率越不确定 | 末端稳定 → 倾向小值 |

80% 是经过大量业务序列测试后的折中：**在绝大多数场景中，最后 20% 的数据足以让你分辨"真实转折"和"随机波动"的差别，同时又不会错过太远的变点。**

#### 什么时候调整

```
changepoint_range = 0.8  (默认)
  → 适用 95% 的场景

changepoint_range = 0.9 ~ 0.95
  → 序列非常长（> 5 年日数据），最后 20% 也有好几年的数据
  → 你确信最近发生了结构性变化，不希望模型忽略它
  → 风险：趋势末端可能不稳定

changepoint_range = 0.5
  → 数据噪声极大，你想强制模型只捕捉早期的大趋势
  → 很少需要
```

来看一个具体的例子。假设你有 2020-2024 五年的日数据（~1825 天），在 2024 年 6 月发生了一次重大业务转型：

```python
# 默认 0.8：变点只能到 2024 年 1 月为止
# 2024 年 6 月的转型 → 被忽略（它在最后 20% 里）
model = Prophet(changepoint_range=0.8)

# 调到 0.95：变点可以到 2024 年 9 月
# 2024 年 6 月的转型 → 被检测到
# 但 2024 年 9 月到 12 月的任何噪声也可能被当成变点
model = Prophet(changepoint_range=0.95)
```

**一个更可靠的做法**：如果你**明确知道**某个时间点发生了结构性变化，不要靠调 `changepoint_range` 来碰运气——直接**手动指定变点**：

```python
model = Prophet(changepoints=['2024-06-01'])  # 精确控制
```

#### 与趋势不确定性的关系

`changepoint_range` 还间接影响预测区间。变点位置越靠后，Prophet 对趋势末端的斜率越不确定，预测区间就会越宽。这是因为趋势模拟（第 7 章）基于"历史变点的频率和幅度"来外推——如果变点可以一直延伸到数据末端，模型会认为"趋势极不稳定"，从而给出更宽的区间。

#### 一句话总结

`changepoint_range = 0.8` 的意思是：**别在最后 20% 的数据里找变点——尾巴上的波动太容易误判。** 了解这一点后，如果你有充分理由认为最近的波动是真实的结构变化，要么手动指定变点，要么谨慎地调大这个值，并接受趋势末端不稳定的代价。

## 3.3 趋势建模的实操框架

### 3.3.1 上手顺序

```
1. 从 linear + 默认 changepoint_prior_scale=0.05 开始
   │
2. 可视化：add_changepoints_to_plot 看变点是否合理
   │
   ├── 趋势太僵硬（明显弯没拐） → 增大 changepoint_prior_scale
   ├── 趋势太痉挛（追噪声）     → 减小 changepoint_prior_scale
   └── 业务有明确的结构变化时间点 → 手动指定 changepoints
   │
3. 交叉验证确认最优值（第 9 章详述）
   │
4. 确定参数后，查看 forecast['trend'] 理解趋势走向
```

### 3.3.2 梯度测试代码

```python
# 直观感受 τ 的影响
for tau in [0.001, 0.01, 0.1, 0.5, 1.0, 5.0]:
    m = Prophet(changepoint_prior_scale=tau, yearly_seasonality=False)
    m.fit(df)
    fc = m.predict(m.make_future_dataframe(365))
    plt.plot(fc['ds'], fc['trend'], label=f'τ={tau}', alpha=0.7)

plt.legend()
plt.title('不同 τ 下的趋势对比')
plt.show()
```

### 3.3.3 常见问题速查

| 症状 | 可能原因 | 行动 |
|------|---------|------|
| 预测趋势与历史趋势脱节 | `changepoint_range=0.8` 导致最近 20% 数据无变点可检测 | 可调整为 0.9-0.95（但需谨慎） |
| 趋势末端剧烈摆动 | `n_changepoints` 过多 + `changepoint_prior_scale` 过大 | 减少候选变点数或调紧先验 |
| 逻辑增长报错 | 未提供 `cap` 列 | `df['cap'] = C; future['cap'] = C` |
| 趋势预测过于保守 | τ 太小 | 调大 `changepoint_prior_scale` |
| 长期预测完全不靠谱 | horizon 超过了历史长度的合理范围 | 预测 horizon ≤ 历史长度的 1/3 |

## 3.4 其他趋势相关参数

### 不确定性采样

```python
model = Prophet(
    interval_width=0.80,           # 预测区间宽度（默认 80%）
    uncertainty_samples=1000,      # 不确定性采样次数
)
# 增大 uncertainty_samples → 更平滑的区间 → 更慢
# 设为 0 → 关闭不确定性估计 → 最快
```

### 逻辑增长的时变容量

```python
# 场景：服务器容量有计划性扩容
df.loc[df['ds'] < '2023-06-01', 'cap'] = 1000
df.loc[df['ds'] >= '2023-06-01', 'cap'] = 2000

future['cap'] = 2000  # 未来保持最新容量
```

## 3.5 本章核心概念清单

| 概念 | 一句话理解 |
|------|-----------|
| **先验 (Prior)** | 看到数据之前，你对参数的初始信念 |
| **似然 (Likelihood)** | 在给定参数下，数据出现的概率 |
| **后验 (Posterior)** | 先验 + 似然 → 综合判断 |
| **拉普拉斯先验** | "我默认参数为零"——产生稀疏解的分布 |
| **稀疏性 (Sparsity)** | 25 个候选变点中只有少数真正非零——自动特征选择 |
| **changepoint_prior_scale (τ)** | 拉普拉斯先验的"松紧度"——越小 = 越不信任变点 = 趋势越平直 |
| **δⱼ** | 第 j 个变点处的斜率变化量——多数情况 ≈ 0 |

### 参数速查

| 参数 | 默认值 | 作用 | 调节原则 |
|------|--------|------|----------|
| `growth` | `'linear'` | 线性 vs 逻辑增长 | 有物理上限才用 logistic |
| `n_changepoints` | `25` | 候选变点数量 | 通常不需要改 |
| `changepoint_range` | `0.8` | 变点检测范围（占历史数据比例）。最后 20% 禁止变点——尾巴上的波动太容易误判 | 确认近期有结构性变化 → 调大到 0.9~0.95（末端可能不稳定） |
| `changepoint_prior_scale` | `0.05` | **拉普拉斯先验尺度——最关键的旋钮** | 交叉验证确定 |
| `uncertainty_samples` | `1000` | 不确定性模拟次数 | 精度 vs 速度权衡 |
