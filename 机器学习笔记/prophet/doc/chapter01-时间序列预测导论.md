# 第 1 章：时间序列预测导论

## 1.1 什么是时间序列预测

时间序列是按时间顺序排列的数据点序列。时间序列预测的核心任务是：**利用历史观测值，预测未来时间点的值**。

```
历史数据: y₁, y₂, ..., yₜ
目标:    ŷ_{t+1}, ŷ_{t+2}, ..., ŷ_{t+h}  (h = 预测 horizon)
```

### 预测问题的三种基本形态

**1. 点预测 (Point Forecast)**

给出每个未来时刻的**单一预测值**——"明天卖 1000 件"。简单直接，但不包含任何不确定性信息。

$$ \hat{y}_t = \mathbb{E}[y_t \mid \text{data}] $$

> 场景：日常补货量估算。经理知道"1000"只是个估计，心里会默认有 ± 浮动。

---

**2. 区间预测 (Interval Forecast)**

给出一个**上界和下界**——"明天卖 850 到 1150 件"。它告诉你了不确定性的大小，但不告诉你区间内部各数值的可能性分布。

$$ [\hat{y}_{t, \text{lower}}, \hat{y}_{t, \text{upper}}] \quad \text{如：80% 区间}$$。它的回答形式是：**"真实值有 80% 的概率落在这个闭区间里"**。但它不会告诉你区间内是均匀分布的（每个值概率一样），还是接近 yhat 的可能性更大。

> 场景：风控评估。资金备付需要知道"最坏情况要准备多少"，区间下界够用。

---

**3. 概率预测 (Probabilistic Forecast)**

给出**完整的概率分布**——不只是上下界，还有**中间每个值的可能性**。

```
假设明天预测值的中位数是 1000：

区间预测:     [850, 1150]  ← 只告诉你"80%在这里面"
              ───╲_____╱───  不知道内部密度

概率预测:      完整分布形状
              ╱╲
             ╱  ╲
            ╱    ╲
           ╱      ╲
     ──╱╱          ╲╲────
      850  1000  1150
       ↑      ↑     ↑
     5%分位 中位数  95%分位

它还能回答:
  P(y > 1200) = ?  → "超额概率 3%"
  P(900 < y < 950) = ?  → "这个区间有 12% 的概率"
```

> 场景：供应链安全库存计算。需要知道"超出预测 1200 件的概率是多少"来确定安全库存量——区间预测回答不了，概率预测可以。

---

### 三个层次的关系

```
点预测          yhat            "明天 1000"
    ↓ 加入不确定性宽度
区间预测        [800, 1200]     "80% 概率在 800 到 1200"
    ↓ 保留完整分布形状
概率预测        完整分布         "超过 1200 的概率是 3%"
```

**区间预测 = 概率预测的"降级版"**——它把完整分布压缩成了两个数（下界、上界），丢弃了分布内部的形状信息。

| 维度 | 点预测 | 区间预测 | 概率预测 |
|------|--------|----------|----------|
| 信息量 | 最小——一个数 | 中等——两个数 | 最大——整个分布 |
| 回答"超过上限的概率" | ❌ | ❌ | ✅ |
| 计算成本 | 最低 | 中等 | 最高（MCMC） |
| 业务方的理解难度 | 低 | 中 | 高 |

### Prophet 对三者的天然支持

```python
# 点预测（默认 MAP 模式天然输出）
forecast['yhat']

# 区间预测（默认就有，基于趋势模拟）
forecast['yhat_lower'], forecast['yhat_upper']
# 默认 80% 区间，用 interval_width 控制

# 概率预测（需要开启 MCMC）
model = Prophet(mcmc_samples=300)
# 从完整后验的采样中，能回答"超过 1200 的概率是 3%"这类问题
```

## 1.2 时间序列的组成成分

经典分解法将一个时间序列拆解为四个成分：

$$y_t = T_t + S_t + C_t + R_t$$

| 成分 | 含义 | 周期特征 | 示例 |
|------|------|----------|------|
| **趋势 (Trend)** $T_t$ | 长期变化方向 | 非周期 | 逐年营收增长 |
| **季节性 (Seasonality)** $S_t$ | 固定周期的规律波动 | 固定周期 | 周末销量高峰、暑期旅游旺季 |
| **周期性 (Cyclicity)** $C_t$ | 非固定周期的波动 | 周期不固定 | 经济周期（3-5 年） |
| **残差 (Residual)** $R_t$ | 随机噪声 | 无 | 偶然事件冲击 |

> **Prophet 的简化**：Prophet 将周期性归入趋势变化点（changepoint）或节假日效应，核心建模的是 $T_t + S_t + H_t$（趋势 + 季节 + 节假日）。

### 各方法对比

| 方法 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| **ARIMA/SARIMA** | 理论成熟、可解释性强 | 需要平稳性、不支持缺失值、多季节处理复杂 | 稳定序列、短期预测 |

## 1.4 Prophet 的设计哲学与定位

### 1.4.1 GAM：Prophet 的数学骨架

Prophet 是一个**广义加性模型 (Generalized Additive Model, GAM)**。这个名字拆开来，每个词都承载了具体的设计选择。我们从线性回归开始逐步构建理解。

---

#### 第一步：从线性回归到"加性模型"

你最熟悉的线性回归是这样的：$$y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \cdots + \varepsilon$$

每个变量 $x_j$ 的贡献是 $\beta_j \cdot x_j$——变量值乘以一个系数。**变量和预测值之间的关系被强制为一条直线。**

加性模型把这条直线"松开"：$$y = \beta_0 + f_1(x_1) + f_2(x_2) + \cdots + \varepsilon$$

$f_j(\cdot)$ 不再是一个简单的系数 $\beta_j$，而是一个**任意形状的光滑函数**。它可以是抛物线、S 曲线、正弦波……任何形状都可以——只要每个变量的贡献是独立加在一起的。

```
线性模型：                      加性模型：
  y                               y
  │     ╱                          │    ╭──╮
  │   ╱  ← 只能是直线               │   ╱    ╲    ← 任意光滑形状
  │ ╱                              │ ╱         ╲__
  └───────── x                     └─────────────── x
```

**"加性"意味着**：各成分的效果互不干扰，彼此独立。趋势变再快，不影响你对周末效应的估计。这是可解释性的基石——你总能回答"预测变化里多少来自趋势、多少来自季节"。

---

#### 第二步："广义"——它解决什么问题

前面第一步讲了从线性回归到加性模型的进化：把 $y = \beta_0 + \beta_1 x$ 的直线松开，变成 $y = f_1(x_1) + f_2(x_2) + \cdots$ 的任意光滑形状。但这背后还藏着一个隐含假设。我们先把它挖出来。

---

**线性回归的隐含假设：响应变量必须正态**

我们最熟悉的线性回归，完整的数学表述其实是：

$$y = \beta_0 + \beta_1 x_1 + \cdots + \varepsilon, \quad \varepsilon \sim N(0, \sigma^2)$$

翻译成人话：**y 在给定 x 之后，围绕均值 $\beta_0 + \beta_1 x_1 + \cdots$ 呈正态分布**。

```
正态分布假设的含义：

       y │    ╭─ ╱╲ ─╮
         │    │ ╱  ╲ │   ← 对于每个 x 值，
         │  ╱ │╱    ╲│╔══╗  y 围绕均值 ± σ
         │ ╱  │      │║  ║  呈对称的正态分布
         │╱   │      │║  ║
         └──────────────────
              x₁   x₂   x₃
```

这个假设对许多场景是合理的——比如预测气温、销量（大部分情况下 y 的波动确实对称）。但一旦遇到下面这些数据，正态假设就出问题了。

---

**真实世界的非正态数据类型**

| 数据类型 | 例子 | 正态假设为什么不对 | 实际分布特征 |
|---------|------|------------------|-------------|
| **计数数据** | 每天注册用户数、订单数 | 只能是**非负整数**，不可能为负 | 方差随均值增大——均值大时波动也大 |
| **二值数据** | 用户是否流失、是否点击 | 只有 **0 或 1**，不是连续值 | 预测值要压在 [0,1] 之间 |
| **比例数据** | 转化率、市场占有率 | 取值只能在 **0~1 之间** | 两端有天然边界 |
| **右偏数据** | 客单价、响应时间 | 有长尾——少数极值拖得很长 | 使用取对数后近似正态 |
| **时间间隔** | 两次购买间隔天数 | 只能是正数，且右偏严重 | 指数分布或伽马分布 |

用正态分布去拟合计数数据会怎样？预测值可能出现负数（"今天注册 -50 个用户"），方差假设恒定但在计数数据中方差随均值变化——结果就是：**模型在数据量大的区域预测偏激，数据量小的区域预测偏保守**。

---

**"Generalized" 的解决方案：分布族 + 连接函数**

"Generalized" 做的就是：**不再强制 y 服从正态分布，而是允许 y 服从任何"指数族分布"**。

它引入了两个新东西：

**第一：分布族 (Distribution Family)** —— y 的条件分布是什么？

$$
y \sim \text{Distribution}\left(\text{mean} = \mu, \text{其他参数} = \cdots\right)
$$

**第二：连接函数 (Link Function)** —— 模型输出的加性组合 $\sum f_j$ 和 y 的期望 $\mu$ 之间是什么关系？

$$
g(\mathbb{E}[y]) = \sum f_j(x_j) \quad \Longleftrightarrow \quad \mathbb{E}[y] = g^{-1}\!\left(\sum f_j(x_j)\right)
$$

画成流程图就是：

```
  模型内部:     f₁(x₁) + f₂(x₂) + ...                         （加性组合，取值无约束）
                   │
          g⁻¹(·)  │  连接函数的逆变换
                   ▼
  数据尺度:     𝔼[y] = g⁻¹(f₁ + f₂ + ...)                     （映射到实际数据范围）
                   │
  实际观测:        y ～ Distribution(𝔼[y], ...)              （按概率分布生成观测值）
```

---

**如何选择分布族和连接函数（实用指南）**

实践中这从来不是"翻开统计学教科书查表"——而是根据**数据本身的性质**来判断：

```python
# 问自己两个问题：
# 1. y 的取值范围是什么？有无上下界？
# 2. y 的方差和均值有关系吗？
```

| y 的特征 | 推荐的分布族 | 推荐的连接函数 | 实际含义 |
|----------|-------------|---------------|----------|
| $-\infty < y < \infty$，方差恒定 | **正态 (Normal)** | 恒等 (Identity) $g(\mu) = \mu$ | 加性模型的默认形态 |
| $y \geq 0$，方差 $\propto$ 均值 | **泊松 (Poisson)** | 对数 (Log) $g(\mu) = \log(\mu)$ | 效应相乘——$e^{a+b}=e^a e^b$ |
| $0 \leq y \leq 1$ | **伯努利/二项 (Binomial)** | Logit $g(\mu) = \log\frac{\mu}{1-\mu}$ | 预测值自动压在 0~1 之间 |
| $y > 0$，右偏、长尾 | **伽马 (Gamma)** | 对数 (Log) $g(\mu) = \log(\mu)$ | 客单价、响应时间 |

用三个具体数字例子来理解连接函数的作用：

```python
# 例 1：恒等连接（正态）
# 预测用户注册数（但注册数不可能是负值？隐患在后面）
原始预测 = 100 + (-50) + 20 = 70   # ← 没问题，能加出正数，但可能加出负数

# 例 2：对数连接（泊松）
# 预测订单数——响应永远非负
log(𝔼[y]) = 4.6 + (-0.2) + 0.1 = 4.5  # ← 内部加出来是负的也没关系
𝔼[y] = exp(4.5) ≈ 90                  # ← 取指数后永远非负

# 例 3：Logit 连接（伯努利）
# 预测转化率——压在 0~1 之间
log(odds) = 0.5 + 1.2 + (-0.3) = 1.4  # ← 内部可以无穷大/无穷小
𝔼[y] = 1/(1 + e^{-1.4}) ≈ 0.80       # ← 取 Logit 逆后永远在 [0,1]
```

---

**在经典 GLM/GAM 中，完整的建模流程是：**

```
1. 观察 y 的类型（连续？计数？二值？）
     │
2. 选择分布族（Normal？Poisson？Binomial？）
     │
3. 根据分布族确定默认连接函数（一般不需要手动选）
     │
4. 模型在 𝔼[y] = g⁻¹(Σ fⱼ) 下做参数估计
     │
5. 评估残差，检查分布假设是否合理
```

---

**Prophet 实际做了什么**

现在回到 Prophet。Prophet 的 "G"（广义）**并没有使用上面这套能力**——它始终假设 $y$ 服从正态分布，不换似然函数，不换连接函数。换句话说，Prophet 的完整 GAM 表述其实是：

$$\boxed{y(t) = g(t) + s(t) + h(t) + r(t) + \varepsilon_t}, \quad \varepsilon_t \sim N(0, \sigma^2)$$

它始终是恒等连接的加性模式——只不过多给了你一个选项：**季节性和节假日可以直接加上去，还是按比例乘上去**。

| 模式 | 数学形式 | 含义 |
|------|---------|------|
| **加性** (默认) | $y = g(t) + s(t) + h(t) + \varepsilon$ | 周末效应永远是 +200，不随体量变化 |
| **乘性** | $y = g(t) \cdot \big(1 + s(t) + h(t)\big) + \varepsilon$ | 周末波动按体量比例缩放 |

```python
# 加性：体量 1000 时周末 +200，体量 10000 时还是 +200
# 乘性：体量 1000 时周末 +200，体量 10000 时周末 +2000
```

**但注意：这只是在调整"季节性如何叠加到趋势上"的公式结构——它并没有改变 y 的分布假设。y 仍然是正态的。**

---

**区分清楚**

```
经典 GLM 的 "Generalized"          Prophet 的加性/乘性
─────────────────────────────     ─────────────────────────
管什么: y 的数据类型               管什么: 季节性的叠加方式
        （整数？0/1？正数？）               （固定增量？按比例？）

怎么管: 换分布族 + 换连接函数      怎么管: 调整公式中的乘法项
        （泊松+Log连接）                    （y = trend × (1 + season)）

Prophet 是否使用: ❌                 Prophet 是否使用: ✅
                 始终用正态分布                 通过 seasonality_mode 切换
                 始终用恒等连接                 默认 additive，可切 multiplicative
```

> 所以"G"在 Prophet 里确实有点名不副实——但历史原因已经叫开了。只要记住：**Prophet 的 GAM 不是一个真正意义上的"广义"GAM，它没有换分布族的能力；它的加性/乘性切换是另一回事，用来解决"季节性怎么叠加"的问题。**

---

#### 第三步："模型"——从公式到概率

公式给出的是点估计。但预测需要**不确定性**——你不仅要告诉业务方"明天预计卖 1000"，还要说"80% 的概率在 850 到 1150 之间"。GAM 的 **"M"** 就是把公式嵌入一个概率框架：

$$\begin{aligned}
y(t) &\sim N(\hat{y}(t), \sigma^2)  &&\text{似然：观测值围绕预测值正态分布} \\
\delta_j &\sim \text{Laplace}(0, \tau) &&\text{先验：变点的斜率变化 → 稀疏} \\
a_n, b_n &\sim N(0, \sigma_s^2)  &&\text{先验：季节性系数 → 收缩} \\
\kappa_i &\sim N(0, \nu^2) &&\text{先验：节假日效应 → 收缩}
\end{aligned}$$

这带来三个实际好处：

| 好处 | 来源 | 在第 3 章见到 |
|------|------|-------------------|
| 自动防止过拟合 | 先验将参数向 0 收缩 | `changepoint_prior_scale` 压平噪声趋势 |
| 完整的预测分布 | 从后验中采样 → `yhat_lower` / `yhat_upper` | 第 7 章详述 |
| 缺失值天然兼容 | 概率模型在缺失处用先验平滑过渡 | 不同于 ARIMA 需要插值 |

---

#### 第四步：Prophet 的完整 GAM

把以上三步组装起来，就是 Prophet 的完整形式：

$$\boxed{y(t) = g(t) + s(t) + h(t) + r(t) + \varepsilon_t} \quad \text{或乘性：}\quad \boxed{y(t) = g(t) \cdot \big(1 + s(t) + h(t)\big) + r(t) + \varepsilon_t}$$

| 组件 | 函数 | 建模方式 | 对应章节 | 你的控制旋钮 |
|------|------|----------|----------|-------------|
| $g(t)$ | 趋势 | 分段线性 / 逻辑增长 + 变点 | 第 3 章 | `changepoint_prior_scale` |
| $s(t)$ | 季节性 | 傅里叶级数（多频率 sin/cos 叠加） | 第 4 章 | `seasonality_prior_scale`, 傅里叶阶数 |
| $h(t)$ | 节假日 | 指示变量 + 效应窗口 | 第 5 章 | `holidays_prior_scale`, `lower/upper_window` |
| $r(t)$ | 额外回归量 | 线性项 $\beta \cdot x$ | 第 6 章 | 各回归量的 `prior_scale` |
| $\varepsilon_t$ | 误差 | $N(0, \sigma^2)$ | — | `uncertainty_samples` (第 7 章) |

> **关键心智模型**：Prophet 不是一个黑箱。它是一张"配方表"——你把趋势、季节、节假日、外部变量各放一勺，模型帮你混合后品尝。不满意某个成分的味道？你知道该调哪个旋钮。

---

#### 第五步：GAM vs 其他模型家族 —— 为什么选它

| 模型家族 | 预测方式 | 可解释性 | 人机协作 |
|----------|---------|----------|----------|
| **线性回归** | $\sum \beta_j x_j$ | 极高——每个 β 就是效应大小 | ✅✅✅ |
| **GAM (Prophet)** | $\sum f_j(x_j)$ | 极高——每个 $f_j$ 可独立画图 | ✅✅✅ |
| **树模型 (XGBoost)** | 数百棵树的投票 | 中——需借助 SHAP 等工具 | ✅ |
| **深度学习 (LSTM)** | 百万参数的神经网络 | 低——黑箱 | ❌ |

Prophet 选择 GAM，本质上是在**自动化和可干预之间**取了一个平衡点：
- 比传统统计方法**更自动**（不需要手写季节性结构、手动处理缺失值）
- 比深度学习方法**更透明**（每个成分都能画出来给业务方看）
- 保留了**分析师介入的空间**（你可以说："双11当天肯定是异常值，我手动加个变点"）

---

### 1.4.2 为什么叫 "Prophet"（先知）

- **自动化**：默认配置就能给出不错的预测（"让预测民主化"）
- **可解释**：每个成分可独立可视化与分析
- **可干预**：分析师可以通过变点、节假日、额外回归量注入领域知识
- **鲁棒**：对缺失值、异常值、趋势变化有内置处理

### Prophet 的最佳适用场景

✅ **适合：**
- 日均及以上频率（天、周、月、季、年）
- 有明显的周期性模式（周/月/季/年）
- 存在节假日或特殊事件影响
- 历史数据 ≥ 1 个完整季节周期
- 需要可解释性（向业务方展示"为什么预测是这个数"）

❌ **不适合：**
- 分钟/秒级高频数据（用 ARIMA 或 DL）
- 没有明显季节模式的纯随机游走
- 需要同时预测多个变量的联合分布
- 数据量极小（< 10 个观测点）

## 1.5 本章小结

| 概念 | 要点 |
|------|------|
| 时间序列成分 | 趋势 + 季节 + 周期 + 残差 |
| **GAM 三大要素** | **加性**（各成分独立求和）+ **广义**（Prophet 的 "G" 不准确——它始终用正态分布，未使用经典 GLM 的分布族切换能力）+ **模型**（先验→后验的概率框架） |
| Prophet 的 GAM | $y = g(t) + s(t) + h(t) + r(t) + \varepsilon$（加性）或 $y = g(t) \cdot (1 + s(t) + h(t)) + r(t) + \varepsilon$（乘性），每个成分对应一个可控旋钮 |
| Prophet 定位 | 在自动化和可干预之间取平衡——比 ARIMA 更省心，比深度学习更透明 |
| 适用粒度 | 日均及以上，有明显季节性 |
| 不适用场景 | 高频数据、纯随机游走、多变量联合建模 |
