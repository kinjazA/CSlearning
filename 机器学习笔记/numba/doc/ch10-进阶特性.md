# 第十章：进阶特性

> **核心问题**：`@njit` 只能加速函数，不能加速类；NumPy 的函数在 Numba 里都有，但你自己写的辅助函数怎么让 Numba 识别？本章讲三个高级工具：`@jitclass`、`@overload`、`@generated_jit`。

---

## 10.1 `@jitclass` —— 编译 Python 类

### 10.1.1 为什么需要 `@jitclass`

```python
# ❌ 普通 Python 类不能在 @njit 中高效使用
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def distance(self, other):
        return np.sqrt((self.x - other.x)**2 + (self.y - other.y)**2)

@njit
def compute(p1, p2):
    return p1.distance(p2)   # ❌ Numba 回退到 object 模式，慢
```

`@jitclass` 让 Numba 能够编译整个类，生成机器码，性能接近 C 结构体。

### 10.1.2 基本用法

```python
from numba import jitclass, float64
from numba.types import int64

# 第一步：定义类的字段规范（类型注解）
spec = [
    ('x', float64),
    ('y', float64),
]

@jitclass(spec)
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def distance(self, other):
        dx = self.x - other.x
        dy = self.y - other.y
        return np.sqrt(dx * dx + dy * dy)
    
    def move(self, dx, dy):
        self.x += dx
        self.y += dy

# 使用：和普通 Python 类一样
@njit
def demo(p1, p2):
    p1.move(1.0, 2.0)
    return p1.distance(p2)

p1 = Point(0.0, 0.0)
p2 = Point(3.0, 4.0)
print(demo(p1, p2))   # ✅ 全部编译为机器码
```

### 10.1.3 `@jitclass` 的规则

| 支持 | 不支持 |
|------|--------|
| ✅ 基本类型字段（int/float/array） | ❌ 继承（不能用 `class A(B)`） |
| ✅ 方法（包括 `__init__`） | ❌ 类方法 / 静态方法 |
| ✅ 嵌套 `@jitclass` | ❌ 属性装饰器 `@property` |
| ✅ NumPy 数组字段 | ❌ Python 对象字段（list/dict/str） |
| ✅ 运算符重载（`__add__` 等） | ❌ 魔法方法（`__str__`/`__repr__`） |

### 10.1.4 数组字段

```python
from numba import float64

spec = [
    ('data', float64[:]),       # 一维 float64 数组
    ('size', int64),
]

@jitclass(spec)
class Vector:
    def __init__(self, n):
        self.data = np.zeros(n, dtype=np.float64)
        self.size = n
    
    def set(self, i, value):
        if 0 <= i < self.size:
            self.data[i] = value
    
    def norm(self):
        s = 0.0
        for x in self.data:
            s += x * x
        return np.sqrt(s)
```

### 10.1.5 运算符重载

```python
spec = [('x', float64), ('y', float64)]

@jitclass(spec)
class Vec2D:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __add__(self, other):
        return Vec2D(self.x + other.x, self.y + other.y)
    
    def __mul__(self, scalar):
        return Vec2D(self.x * scalar, self.y * scalar)

@njit
def demo():
    v1 = Vec2D(1.0, 2.0)
    v2 = Vec2D(3.0, 4.0)
    v3 = v1 + v2           # ✅ 调用 __add__
    v4 = v3 * 2.0          # ✅ 调用 __mul__
    return v4.x, v4.y
```

## 10.2 `@overload` —— 扩展 Numba 的函数库

### 10.2.1 问题场景

你写了一个辅助函数，想在 `@njit` 函数里调用：

```python
def my_clamp(x, lo, hi):
    """你自己写的工具函数"""
    if x < lo: return lo
    if x > hi: return hi
    return x

@njit
def process(arr):
    result = np.empty_like(arr)
    for i in range(len(arr)):
        result[i] = my_clamp(arr[i], 0.0, 1.0)  # ❌ Numba 不认识 my_clamp
    return result
```

Numba 不知道怎么编译 `my_clamp`，会报错或回退到 object 模式。

### 10.2.2 用 `@overload` 注册实现

```python
from numba import types
from numba.extending import overload

def my_clamp(x, lo, hi):
    """Python 版本（在外面可以调用）"""
    if x < lo: return lo
    if x > hi: return hi
    return x

@overload(my_clamp)
def my_clamp_impl(x, lo, hi):
    """Numba 版本（在 @njit 里调用时用这个）"""
    if isinstance(x, types.Float):
        def my_clamp_float(x, lo, hi):
            if x < lo: return lo
            if x > hi: return hi
            return x
        return my_clamp_float
    
    if isinstance(x, types.Integer):
        def my_clamp_int(x, lo, hi):
            if x < lo: return lo
            if x > hi: return hi
            return x
        return my_clamp_int

@njit
def process(arr):
    result = np.empty_like(arr)
    for i in range(len(arr)):
        result[i] = my_clamp(arr[i], 0.0, 1.0)  # ✅ 现在能编译了
    return result
```

**工作原理**：
1. 你调用 `my_clamp(arr[i], 0.0, 1.0)`
2. Numba 看到 `my_clamp` 有 `@overload` 注册
3. Numba 调用 `my_clamp_impl(x, lo, hi)` 并传入**类型对象**（不是实际值）
4. `my_clamp_impl` 检查类型，返回对应的实现函数
5. Numba 编译这个实现函数

### 10.2.3 实战：扩展 NumPy 函数

假设你想在 Numba 里用 `np.clip`，但发现 Numba 不支持（假设）：

```python
import numpy as np
from numba.extending import overload

@overload(np.clip)
def clip_impl(a, a_min, a_max):
    """给 np.clip 提供 Numba 实现"""
    def clip_scalar(a, a_min, a_max):
        if a < a_min: return a_min
        if a > a_max: return a_max
        return a
    return clip_scalar

# 现在可以在 @njit 里用 np.clip 了
```

## 10.3 `@generated_jit` —— 根据类型生成不同代码

### 10.3.1 什么是 `@generated_jit`

`@njit` 编译时，所有输入类型共享同一份代码。`@generated_jit` 允许**根据输入类型生成不同的代码**。

```python
from numba import generated_jit, types

@generated_jit(nopython=True)
def flexible_add(x, y):
    """根据输入类型生成不同实现"""
    
    # 这个函数在编译时执行，x 和 y 是类型对象
    if isinstance(x, types.Integer) and isinstance(y, types.Integer):
        # 整数版本：直接加
        def add_int(x, y):
            return x + y
        return add_int
    
    elif isinstance(x, types.Float) or isinstance(y, types.Float):
        # 浮点版本：加一个小偏移避免精度问题（示例）
        def add_float(x, y):
            return x + y + 1e-10
        return add_float
    
    else:
        # 其他类型：报错
        def add_unsupported(x, y):
            raise TypeError("Unsupported types")
        return add_unsupported
```

### 10.3.2 `@generated_jit` vs `@overload`

| | `@generated_jit` | `@overload` |
|--|-----------------|-------------|
| **用途** | 你自己的函数，根据类型选择实现 | 给已有函数（NumPy 等）注册实现 |
| **返回** | 返回一个实现函数 | 返回一个实现函数 |
| **典型场景** | 一个函数，多种类型各自优化 | 扩展 Numba 支持的函数库 |

### 10.3.3 实战：类型分发的距离函数

```python
@generated_jit(nopython=True)
def distance(a, b):
    """根据输入维度选择不同的距离计算"""
    
    if isinstance(a, types.Array):
        # 数组：欧氏距离
        def distance_array(a, b):
            s = 0.0
            for i in range(len(a)):
                diff = a[i] - b[i]
                s += diff * diff
            return np.sqrt(s)
        return distance_array
    
    elif isinstance(a, (types.Float, types.Integer)):
        # 标量：绝对值
        def distance_scalar(a, b):
            return abs(a - b)
        return distance_scalar
```

## 10.4 字面值类型（Literal Types）

### 10.4.1 编译时常量

有时你希望某个参数在编译时就确定，用于生成特化代码：

```python
from numba.typed import literal

@njit
def power(x, n):
    """普通版本：n 可以是任意整数"""
    result = 1.0
    for _ in range(n):
        result *= x
    return result

# 如果 n 总是 2/3/4 这种小常数，可以特化
@generated_jit(nopython=True)
def power_optimized(x, n):
    """n 是编译时常量时，展开循环"""
    if isinstance(n, types.Literal):
        n_val = n.literal_value
        if n_val == 2:
            def power_2(x, n):
                return x * x
            return power_2
        elif n_val == 3:
            def power_3(x, n):
                return x * x * x
            return power_3
    
    # 否则回退到通用版本
    def power_generic(x, n):
        result = 1.0
        for _ in range(n):
            result *= x
        return result
    return power_generic
```

## 10.5 扩展 Numba：完整流程示例

假设你要实现一个 Numba 不支持的 NumPy 函数 `np.median`：

```python
import numpy as np
from numba import njit
from numba.extending import overload

# Python 版本（外面可以用）
def my_median(arr):
    """手写 median（Numba 原生不支持）"""
    sorted_arr = np.sort(arr)
    n = len(sorted_arr)
    mid = n // 2
    if n % 2 == 0:
        return (sorted_arr[mid - 1] + sorted_arr[mid]) / 2.0
    else:
        return sorted_arr[mid]

# Numba 版本（在 @njit 里用）
@overload(my_median)
def my_median_impl(arr):
    def median_1d(arr):
        sorted_arr = np.sort(arr)
        n = len(sorted_arr)
        mid = n // 2
        if n % 2 == 0:
            return (sorted_arr[mid - 1] + sorted_arr[mid]) / 2.0
        else:
            return sorted_arr[mid]
    return median_1d

@njit
def process_data(data):
    # 现在可以在 @njit 里用 my_median 了
    return my_median(data)
```

## 10.6 性能对比：`@jitclass` vs 普通类

```python
import time

# 普通 Python 类
class PointPython:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def distance(self, other):
        dx = self.x - other.x
        dy = self.y - other.y
        return np.sqrt(dx * dx + dy * dy)

# Numba jitclass
spec = [('x', float64), ('y', float64)]

@jitclass(spec)
class PointJit:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def distance(self, other):
        dx = self.x - other.x
        dy = self.y - other.y
        return np.sqrt(dx * dx + dy * dy)

# 测试
n = 1000000

# Python 类
points_py = [PointPython(i, i+1) for i in range(n)]
t0 = time.perf_counter()
total = sum(points_py[i].distance(points_py[i+1]) for i in range(n-1))
t_py = time.perf_counter() - t0

# Jitclass（在 @njit 内使用）
@njit
def compute_jit(points):
    total = 0.0
    for i in range(len(points) - 1):
        total += points[i].distance(points[i+1])
    return total

points_jit = [PointJit(i, i+1) for i in range(n)]
compute_jit(points_jit[:10])  # 热身
t0 = time.perf_counter()
total_jit = compute_jit(points_jit)
t_jit = time.perf_counter() - t0

print(f"Python 类: {t_py:.4f}s")
print(f"Jitclass:  {t_jit:.4f}s  ← 快 {t_py/t_jit:.0f}×")
```

## 10.7 本章关键概念

| 概念 | 一句话 |
|------|--------|
| **`@jitclass`** | 编译 Python 类，性能接近 C 结构体 |
| **字段规范 (spec)** | 类的字段类型声明，`[('x', float64), ...]` |
| **`@overload`** | 给已有函数（如 NumPy）注册 Numba 实现 |
| **`@generated_jit`** | 根据输入类型生成不同代码（类型分发） |
| **字面值类型** | 编译时常量，用于生成特化代码 |
| **扩展流程** | 定义 Python 版本 → 用 `@overload` 注册 Numba 版本 |
