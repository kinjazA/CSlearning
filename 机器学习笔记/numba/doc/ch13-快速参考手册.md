## 13.1 一分钟上手模板

```python
import numpy as np
from numba import njit

# 最常用模板
@njit
def my_function(arr):
    result = np.empty_like(arr)
    for i in range(len(arr)):
        result[i] = arr[i] * 2 + 1
    return result

# 调用
arr = np.random.randn(1000000)
output = my_function(arr)  # 第一次慢（热身），之后飞快
```

**关键点**：
- `@njit` 装饰器必不可少
- 第一次调用慢（编译），后续秒出结果
- 只能用 NumPy 数组和标量，不能用 Python list/dict

## 13.2 核心装饰器速查

| 装饰器 | 用途 | 典型场景 |
|--------|------|---------|
| `@njit` | 通用加速 | 90% 的场景 |
| `@njit(cache=True)` | 缓存编译结果 | 避免重复热身 |
| `@njit(parallel=True)` + `prange` | 多核并行 | 大数组（>10万） |
| `@njit(nogil=True)` | 释放 GIL | 配合 Python `ThreadPoolExecutor` |
| `@vectorize` | 标量 → ufunc | 逐元素操作（ReLU、clip） |
| `@guvectorize` | 子数组 → ufunc | 滑动窗口、行列操作 |
| `@jitclass` | 编译类 | 复杂数据结构（粒子、节点） |

## 13.3 支持与不支持速查

### ✅ 支持的特性

| 类别 | 支持 |
|------|------|
| **控制流** | `if/else`, `for`, `while`, `break`, `continue` |
| **NumPy** | 数组创建、索引、切片、`sum/mean/std/max/min`、线代、随机数 |
| **数学** | `+/-/*//`, `**`, `sqrt/sin/cos/exp/log`, `abs/min/max` |
| **类型** | `int/float/bool`, NumPy 数组, `tuple`, `@jitclass` |
| **并行** | `prange`, `parallel=True` |

### ❌ 不支持的特性

| 类别 | 不支持 | 替代方案 |
|------|--------|---------|
| **容器** | Python `list/dict/set` | `numba.typed.List/Dict`, NumPy 数组 |
| **字符串** | 字符串操作 | 在 `@njit` 外处理 |
| **高级语法** | `yield`, `lambda`, `with`, `try/except` | 拆分逻辑 |
| **I/O** | `print`, 文件读写 | 在 `@njit` 外处理 |
| **模块导入** | 函数内 `import` | 在函数外导入 |

## 13.4 性能优化清单

### 🚀 必做（10-100× 加速）

| 优化 | 检查方法 | 修复 |
|------|---------|------|
| ✅ 用 `@njit` | - | 加装饰器 |
| ✅ 避免循环内分配 | `np.zeros` 在 `for` 里？ | 移到循环外 |
| ✅ 预分配输出数组 | 函数内 `np.empty_like()`？ | 改为 `out` 参数 |
| ✅ 类型稳定 | 变量类型在循环中变化？ | 统一类型（如全用 `float64`） |

### ⚡ 可选（1.5-5× 加速）

| 优化 | 何时用 | 如何做 |
|------|--------|--------|
| 循环融合 | 多个循环遍历同一数组 | 合并为一个循环 |
| 并行 | 数组 > 10万 + CPU 密集 | `@njit(parallel=True)` + `prange` |
| 遍历方向 | 二维数组 | 行优先（C-order）：外层循环行 |
| 无分支 | 循环内有 `if` | 用 `np.where` 或数学技巧 |

## 13.5 随机数可复现速查

### 问题：每次运行结果不同

```python
# ❌ 错误：外部 seed 不影响 Numba
np.random.seed(42)
@njit
def bad():
    return np.random.randn(10)
```

### 方案 1：函数入口播种（确定性）

```python
@njit
def good(seed):
    np.random.seed(seed)  # ← 在函数第一行
    return np.random.randn(10)

# 相同 seed → 相同结果
r1 = good(42)
r2 = good(42)
assert np.array_equal(r1, r2)  # ✅
```

### 方案 2：多次运行平均（降方差）

```python
def robust_estimate(data, n_runs=10):
    results = []
    for i in range(n_runs):
        results.append(my_function(data, seed=42+i))
    return np.mean(results), np.std(results)
```

**详见**：ch05 §5.6

## 13.6 并行计算速查

### 何时用 `parallel=True`

```
✅ 数组大小 > 10万
✅ 计算密集（循环内有复杂运算）
✅ 每次迭代独立（无依赖）

❌ 小数组（< 1万）
❌ I/O 密集
❌ 迭代之间有依赖
```

### `parallel=True` vs `nogil=True`

| | `parallel=True` | `nogil=True` |
|--|----------------|--------------|
| **谁创建线程** | Numba 自动 | Python 代码（`ThreadPoolExecutor`） |
| **用法** | `prange` | 配合 Python 多线程 |
| **场景** | 单个大数组内部并行 | 多个独立任务并行 |

```python
# parallel=True：Numba 内部并行
@njit(parallel=True)
def example1(arr):
    result = np.empty_like(arr)
    for i in prange(len(arr)):  # ← 自动分配到多核
        result[i] = heavy_compute(arr[i])
    return result

# nogil=True：配合 Python 多线程
from concurrent.futures import ThreadPoolExecutor

@njit(nogil=True)
def example2(task_data):
    return np.sum(task_data ** 2)

tasks = [np.random.randn(10000) for _ in range(8)]
with ThreadPoolExecutor(max_workers=8) as executor:
    results = list(executor.map(example2, tasks))  # 真正并行
```

## 13.7 机器学习典型场景

### 场景 1：距离矩阵（KNN/聚类）

```python
@njit(parallel=True)
def pairwise_distances(X):
    """欧氏距离矩阵 (n×n)"""
    n, d = X.shape
    D = np.zeros((n, n))
    for i in prange(n):
        for j in range(i+1, n):
            dist = 0.0
            for k in range(d):
                diff = X[i, k] - X[j, k]
                dist += diff * diff
            D[i, j] = np.sqrt(dist)
            D[j, i] = D[i, j]
    return D
```

### 场景 2：滑动窗口特征

```python
@njit
def rolling_mean(data, window):
    """滑动窗口均值"""
    n = len(data)
    result = np.empty(n)
    for i in range(n):
        start = max(0, i - window + 1)
        result[i] = np.mean(data[start:i+1])
    return result
```

### 场景 3：自定义损失函数

```python
@njit
def huber_loss(y_true, y_pred, delta=1.0):
    """Huber 损失（对异常值鲁棒）"""
    n = len(y_true)
    loss = 0.0
    for i in range(n):
        residual = abs(y_true[i] - y_pred[i])
        if residual <= delta:
            loss += 0.5 * residual ** 2
        else:
            loss += delta * (residual - 0.5 * delta)
    return loss / n
```

## 13.8 数值计算典型场景

### 场景 1：梯度下降

```python
@njit
def gradient_descent(grad_func, x0, lr=0.01, max_iter=1000):
    x = x0.copy()
    for _ in range(max_iter):
        grad = grad_func(x)
        x = x - lr * grad
        if np.linalg.norm(grad) < 1e-6:
            break
    return x
```

### 场景 2：数值积分

```python
@njit
def trapezoid(f, a, b, n):
    """梯形法则"""
    h = (b - a) / n
    result = 0.5 * (f(a) + f(b))
    for i in range(1, n):
        result += f(a + i * h)
    return result * h
```

### 场景 3：ODE 求解

```python
@njit
def euler_ode(f, y0, t_span, n_steps):
    """Euler 法求解 dy/dt = f(t, y)"""
    t0, t1 = t_span
    dt = (t1 - t0) / n_steps
    y = y0
    for i in range(n_steps):
        t = t0 + i * dt
        y = y + dt * f(t, y)
    return y
```

## 13.11 决策树：何时用 Numba

```
你的瓶颈在哪？
│
├─→ 循环密集？
│   ├─ 能向量化吗？
│   │  ├─ 能 → 用 NumPy（如 a+b*c）
│   │  └─ 不能 → 用 @njit
│   └─ 数组 > 10万？
│      └─ 是 → @njit(parallel=True) + prange
│
├─→ 自定义函数需要在 @njit 里调用？
│   └─ @overload 注册 Numba 实现
│
├─→ 需要编译类？
│   └─ @jitclass
│
├─→ I/O 密集？
│   └─ ❌ Numba 帮不了，考虑多进程
│
└─→ 需要 Python 对象（list/dict/str）？
    └─ ❌ 拆分：纯计算部分用 Numba，其余用 Python
```

## 13.12 调试三步法

```
1️⃣ 能编译吗？
   ├─ 能 → 跳到 2
   └─ 不能 → 看错误类型
       ├─ TypingError → 替换不支持的类型
       └─ LoweringError → 移除不支持的语法

2️⃣ 性能如预期吗？
   ├─ 是 → 完成
   └─ 否 → func.inspect_types() 搜索 'pyobject'
           → 找到回退点 → 替换为 Numba 支持的类型

3️⃣ 结果正确吗？
   ├─ 是 → 完成
   └─ 否 → 设 NUMBA_DISABLE_JIT=1
           → 用 Python 调试器（pdb/IDE）
```
