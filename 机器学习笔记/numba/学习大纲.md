## 第一章：Numba 概述与安装

- **1.1 为什么需要 Numba**
  - Python 的性能瓶颈：解释器开销、GIL、动态类型
  - 对比方案：Cython、PyPy、NumPy 向量化、原生 C 扩展
  - Numba 的定位：JIT 编译，零侵入性加速
- **1.2 Numba 工作原理**
  - LLVM 编译器架构简介
  - JIT（Just-In-Time）编译流程：Python 字节码 → LLVM IR → 机器码
  - 编译缓存机制
- **1.3 安装与环境配置**
  - `pip install numba` 及依赖（llvmlite）
  - 验证安装：`numba -s` 查看系统信息
  - 推荐 Python / NumPy 版本搭配

## 第二章：基础用法——JIT 装饰器

- **2.1 `@jit` 装饰器入门**
  - 第一个 Numba 加速示例
  - 对比纯 Python / NumPy / Numba 的性能
  - 理解"预热"（warm-up）和首次调用开销
- **2.2 `nopython` 模式 vs `object` 模式**
  - `nopython=True`：完全编译，最快但有限制
  - `object` 模式：回退到 Python C API，兼容但慢
  - 何时会触发模式回退及后果
- **2.3 `@njit` = `@jit(nopython=True)`**
  - 推荐默认使用 `@njit`
- **2.4 函数签名（Signature）**
  - 显式指定输入输出类型：`@njit(float64(float64, float64))`
  - 类型推断 vs 显式声明
  - 多签名与 `@generated_jit`
- **2.5 编译选项**
  - `cache=True`：跨会话缓存编译结果
  - `inline='always'`：函数内联
  - `fastmath=True`：放宽浮点精度换速度
  - `parallel=True`：自动并行化
  - `error_model='numpy'` / `'python'`
  - `boundscheck=False`：关闭越界检查
