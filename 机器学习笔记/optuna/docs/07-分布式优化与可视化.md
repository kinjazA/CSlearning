## 7.1 为什么需要分布式优化？

- **单机多核**：一台机器有多个 CPU/GPU，可以同时跑多个 trial
- **多机集群**：多台机器共享同一个 Study，大幅加速搜索
- **容错**：某个进程崩溃不影响其他进程

---

## 7.2 分布式优化的核心：Storage

默认情况下，Study 存储在内存中，无法被多个进程共享。分布式优化需要**持久化存储**：

```python
# 使用 SQLite（单机多进程最简单）
study = optuna.create_study(
    study_name='distributed_demo',
    storage='sqlite:///optuna_study.db',
    direction='minimize'
)
```

### 支持的存储后端

| 后端 | 连接字符串 | 适用场景 |
|------|-----------|---------|
| SQLite | `sqlite:///optuna.db` | 单机多进程 |
| PostgreSQL | `postgresql://user:pass@host/db` | 多机集群 |
| MySQL | `mysql://user:pass@host/db` | 企业环境 |

## 7.3 单机多进程并行

```python
import optuna
from multiprocessing import Process


def worker(process_id):
    study = optuna.load_study(
        study_name='distributed_demo',
        storage='sqlite:///optuna_study.db'
    )
    study.optimize(objective, n_trials=25)


if __name__ == '__main__':
    # 先创建 study
    optuna.create_study(
        study_name='distributed_demo',
        storage='sqlite:///optuna_study.db',
        direction='minimize',
        load_if_exists=True
    )

    # 启动4个进程，每个跑25个trial，总共100个
    processes = [Process(target=worker, args=(i,)) for i in range(4)]
    for p in processes:
        p.start()
    for p in processes:
        p.join()
```

### 更简单的方案：Optuna 内置并行

```python
study = optuna.create_study(direction='minimize')
study.optimize(objective, n_trials=100, n_jobs=4)  # 4进程并行
```

> `n_jobs` 只适用于单机。真正的分布式需要手动管理进程或使用 Joblib。

## 7.4 Optuna Dashboard

Optuna Dashboard 是一个基于 Web 的可视化工具，比内置的 matplotlib 功能更丰富。

### 安装和启动

```bash
pip install optuna-dashboard

# 启动（指定 study 的 storage）
optuna-dashboard sqlite:///optuna_study.db

# 浏览器打开 http://localhost:8080
```

### Dashboard 功能

| 功能 | 说明 |
|------|------|
| 优化历史 | 所有 trial 的值随时间变化 |
| 超参数重要性 | 哪个参数对结果影响最大 |
| 平行坐标图 | 多维度可视化参数与结果的关系 |
| 轮廓图 | 两两参数的影响热图 |
| 中间值 | 剪枝 trial 的中间结果 |
| Pareto 前沿 | 多目标优化的结果展示 |

## 7.5 超参数重要性分析

了解哪个参数最重要，可以帮助你：
1. 缩小搜索范围
2. 优先调整重要参数
3. 固定不重要的参数以加速搜索

### 使用 fANOVA 分析

```python
from optuna.importance import get_param_importances

importances = get_param_importances(study)

for param, importance in importances.items():
    print(f"{param}: {importance:.4f}")
```

### 输出示例

```
learning_rate: 0.4521    # 最重要！
n_layers: 0.2314
batch_size: 0.1567
dropout: 0.0982
optimizer: 0.0616        # 最不重要
```

**解读**：
- 把主要精力放在调 `learning_rate` 上
- `optimizer` 的选择对这个模型影响不大，可以固定为 Adam
