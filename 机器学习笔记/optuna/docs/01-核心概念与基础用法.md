# 第1章：核心概念与基础用法

## 本章目标
理解 Optuna 的基本工作流，掌握 Study、Trial、Objective 等核心概念，能够编写第一个超参数优化程序。

---

## 1.1 什么是 Optuna？

Optuna 是一个**开源的超参数优化框架**，专为机器学习设计。它的核心优势：

- **定义-by-run**：动态构建搜索空间，支持条件参数
- **高效采样**：基于贝叶斯优化的 TPE 算法
- **智能剪枝**：自动提前终止没有希望的试验
- **可视化**：提供丰富的实验分析工具
- **分布式**：支持多机并行搜索

---

## 1.2 核心概念

### Study（研究）
一个 Study 是一次完整的超参数优化会话。它包含：
- 优化方向（最大化 / 最小化）
- 所有已完成的 Trial
- 当前找到的最佳参数

```python
study = optuna.create_study(direction='minimize')  # 或 'maximize'
```

### Trial（试验）
一次 Trial 是对**一组特定超参数**的完整训练和评估。每个 Trial 有唯一编号，从 0 开始。

```python
def objective(trial):
    # trial 对象提供了建议超参数的 API
    x = trial.suggest_float('x', -10, 10)
    return (x - 2) ** 2
```

#### Trial 是怎么传进目标函数的？

你可能会困惑：`def objective(trial):` 中的 `trial` 参数是哪里来的？答案是：**Optuna 在每次试验时自动创建 Trial 对象并传入**，你不需要（也不能）手动创建它。

```python
def objective(trial):  # ← trial 由 Optuna 自动传入
    x = trial.suggest_float('x', -10, 10)
    return (x - 2) ** 2

study = optuna.create_study(direction='minimize')
study.optimize(objective, n_trials=100)

# Optuna 内部大致做这样的事情：
# for trial_number in range(100):
#     trial = Trial(study, trial_number)     # 1. 创建 Trial 对象
#     try:
#         value = objective(trial)            # 2. 传入目标函数，由你使用
#         study._add_completed_trial(trial, value)  # 3. 记录结果
#     except TrialPruned:
#         pass  # 剪枝的 trial 不参与最佳值竞争
```

> **关键理解**：你写的 `objective(trial)` 是一个**回调函数**（callback），Optuna 在每次试验时调用它。`trial` 对象是 Optuna 和你代码之间的**桥梁**——Optuna 通过它给你参数建议，你通过它报告训练进度。

#### Trial 对象的职责

| 职责 | 对应方法 | 说明 |
|------|---------|------|
| **建议超参数** | `suggest_float` / `suggest_int` / `suggest_categorical` | 从搜索空间中采样一组参数 |
| **报告中间结果** | `report(value, step)` | 告诉 Optuna 当前训练到了哪一步、效果如何 |
| **检查是否剪枝** | `should_prune()` | 询问 Pruner "我是不是应该提前停了？" |
| **存储元信息** | `set_user_attr(key, value)` | 保存模型权重、训练耗时等额外数据 |
| **记录身份** | `number` / `params` / `datetime_start` | 该 trial 的编号、参数、时间戳 |

#### 一个 Trial 的完整生命周期

```
1. 创建 Trial(number=5)
2. 采样器给出参数建议 → 你调用 suggest_* 获取
3. 用这些参数训练模型
4. 每个 epoch/fold 结束时 → 你调用 trial.report(val_acc, epoch)
5. 调用 trial.should_prune() → Pruner 判断是否提前终止
6a. 不剪枝 → 训练完成 → return 最终评估指标
6b. 剪枝   → raise optuna.TrialPruned() → 该 trial 提前结束
```

#### 重要规则：同一 Study 中参数名必须唯一

Optuna 通过**参数名字符串**来追踪每个超参数的重要性。不同条件分支中如果用了相同的名字，Optuna 会错误地认为它们是同一个参数：

```python
# ❌ 错误示范：两个分支用了同一个名字 'n_estimators'
if model_type == 'rf':
    n = trial.suggest_int('n_estimators', 50, 300)   # 名字冲突！
elif model_type == 'gb':
    n = trial.suggest_int('n_estimators', 50, 300)   # 名字冲突！

# ✅ 正确做法：用前缀区分
if model_type == 'rf':
    n = trial.suggest_int('rf_n_estimators', 50, 300)
elif model_type == 'gb':
    n = trial.suggest_int('gb_n_estimators', 50, 300)
```

### Epoch（轮次）—— 与 Trial 的区别

**Epoch 不属于 Optuna 的概念**，它是**深度学习（神经网络）**训练的术语：

- **一个 Epoch** = 用**全部训练数据**过一遍（forward + backward + 参数更新）
- **一个 Trial** = 用一组固定超参，训练多个 Epoch，最后评估一次

> **为什么不同 Epoch 的结果不一样？**
> 因为每个 Epoch 结束后，模型的权重都被梯度下降更新了。即使数据一样，模型参数变了，下一轮的预测结果也就变了。此外，随机因素（Dropout、数据 shuffle、BatchNorm 统计量）也会导致差异。

### 哪些模型有 Epoch，哪些没有？

| 类型 | 代表模型 | 训练方式 | 能否在训练中剪枝 |
|------|---------|---------|----------------|
| **深度学习** | PyTorch/TensorFlow/Keras 神经网络 | 多轮迭代（Epoch） | ✅ 每轮结束可剪枝 |
| **梯度提升** | XGBoost/LightGBM/CatBoost | 逐棵树构建（Boosting Round） | ✅ 支持回调函数中途剪枝 |
| **传统 ML** | sklearn 随机森林/SVM/线性回归 | 一次性拟合（fit） | ⚠️ 需手动拆分 CV fold 实现剪枝 |

**关键区别**：
- **神经网络**：训练过程天然分 epoch，剪枝就是 `report()` + `should_prune()` 放在 epoch 循环里
- **XGBoost**：没有 epoch，但有 `num_boost_round`（树的数量），通过回调函数（callback）在每轮 boosting 后报告中间结果
- **sklearn 随机森林**：一次性 fit 完所有树，没有天然的分段点。要实现剪枝，必须**手动拆分交叉验证 fold**，每完成一个 fold 就报告一次

### 三者的层级关系（以深度学习为例）

```
Study (一次超参优化任务)
├── Trial #0: lr=0.01, batch=64, layers=2  ← 一组超参
│   ├── Epoch 0: 用全部数据训练1轮, val_acc=0.45
│   ├── Epoch 1: 权重已更新, val_acc=0.62
│   ├── Epoch 2: val_acc=0.71
│   └── ... 直到 Epoch N（或被剪枝提前终止）
│   └── 返回最终评估指标 ← 这个值交给 Optuna
├── Trial #1: lr=0.001, batch=128, layers=3  ← 另一组超参
│   └── ...
└── ... 共 n_trials 个 Trial
```

**记忆口诀**：
- `Study` 是整场考试
- `Trial` 是每一套答题方案
- `Epoch` 是每套方案里的每一遍验算

### Objective Function（目标函数）
这是你**需要编写**的核心函数。它接收一个 `trial` 对象，完成以下任务：
1. 从 trial 中获取超参数建议（构建一套"答题方案"）
2. 用这些参数构建并训练模型（跑多个 epoch）
3. 返回一个标量评估指标（把最终成绩交给 Optuna）

## 1.3 参数建议 API 完整参考

`suggest_*` 是 Trial 对象最核心的方法家族。Optuna 提供三种官方方法，覆盖了超参数搜索的所有场景。

### suggest_float(name, low, high, **kwargs) — 浮点数

用于连续的数值参数，如学习率、Dropout 比例、正则化系数。

```python
# 均匀分布 — 在 [low, high] 内等概率采样
lr = trial.suggest_float('learning_rate', 1e-5, 1e-1)

# 对数均匀分布 — 在 [log(low), log(high)] 内等概率采样
# 推荐用于跨越多个数量级的参数（学习率、正则化系数）
lr = trial.suggest_float('learning_rate', 1e-5, 1e-1, log=True)

# 离散化 — 以 step 为步长取值
# 例如 low=0, high=1, step=0.1 → 只能取 0, 0.1, 0.2, ..., 1.0
dropout = trial.suggest_float('dropout', 0.0, 1.0, step=0.1)
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `name` | `str` | 参数名，同一 Study 中必须唯一 |
| `low` | `float` | 下界（含） |
| `high` | `float` | 上界（含） |
| `log` | `bool` | 是否在对数空间均匀采样，默认 `False` |
| `step` | `float` | 离散化步长，不指定则为连续采样 |

**适用场景**：
| 参数 | 分布 | 写法 |
|------|------|------|
| 学习率 | log | `suggest_float('lr', 1e-5, 1e-1, log=True)` |
| 权重衰减 | log | `suggest_float('wd', 1e-6, 1e-2, log=True)` |
| Dropout 比例 | 均匀 | `suggest_float('dropout', 0.0, 0.5)` |
| Momentum | 均匀 | `suggest_float('momentum', 0.5, 0.99)` |

### suggest_int(name, low, high, **kwargs) — 整数

用于需要整数的参数，如神经元数量、网络层数、batch size。

```python
# 均匀分布 — 在 [low, high] 内等概率取整数
n_estimators = trial.suggest_int('n_estimators', 50, 500)

# 指定步长 — 值必须是 low + k*step
batch_size = trial.suggest_int('batch_size', 16, 256, step=16)
# 可能的取值：16, 32, 48, ..., 256

# 对数空间
n_units = trial.suggest_int('n_units', 32, 1024, log=True)
```

**适用场景**：
| 参数 | 步长 | 写法 |
|------|------|------|
| 网络层数 | 无 | `suggest_int('n_layers', 1, 10)` |
| 神经元数 | 32/64 | `suggest_int('n_units', 32, 512, step=32)` |
| Batch size | 16/32 | `suggest_int('batch_size', 16, 256, step=16)` |
| Epoch 数 | 无 | `suggest_int('epochs', 10, 200)` |

### suggest_categorical(name, choices) — 离散类别

从给定列表中选一个值。`choices` 可以是**任意类型**（字符串、布尔、甚至 Python 类）：

```python
# 字符串选择
optimizer = trial.suggest_categorical('optimizer', ['Adam', 'SGD', 'RMSprop'])

# 布尔选择
use_bn = trial.suggest_categorical('use_batch_norm', [True, False])

# 函数/类选择（高级用法）
activation = trial.suggest_categorical('activation', [nn.ReLU, nn.GELU, nn.Tanh])
# 之后可以直接实例化：activation()
```

> **注意**：`choices` 中的元素如果是对象（如 `nn.ReLU`），它们会被 pickle 序列化到 storage 中。

### 已废弃但可能遇到的旧版 API

旧版 Optuna（v2.x）使用不同的方法名，你可能在别人的代码中看到：

| 旧版 API | 新版替代 |
|---------|---------|
| `trial.suggest_uniform(name, low, high)` | `trial.suggest_float(name, low, high)` |
| `trial.suggest_loguniform(name, low, high)` | `trial.suggest_float(name, low, high, log=True)` |
| `trial.suggest_discrete_uniform(name, low, high, q)` | `trial.suggest_float(name, low, high, step=q)` |

这些旧方法仍然可用，但不推荐在新代码中使用。

### 参数名命名规范

| 建议 | 好例子 | 坏例子 |
|------|--------|--------|
| 用描述性名称 | `learning_rate`, `n_layers` | `x1`, `param1` |
| 条件分支加前缀 | `rf_n_estimators`, `gb_max_depth` | 两个分支都用 `n_estimators` |
| 循环中加索引 | `n_units_l0`, `dropout_l1` | 循环中都用 `n_units` |
| 小写 + 下划线 | `batch_size` | `BatchSize`, `batchSize` |

## 1.4 Trial 对象的属性和方法（非 suggest 类）

除了 `suggest_*` 系列以外，Trial 对象还提供了用于报告进度、检查剪枝、存储自定义数据的方法和属性。

### 只读属性

这些属性记录 trial 的状态和元信息，你**不应该**修改它们：

```python
trial.number          # int — 试验编号，从 0 开始递增
trial.params          # dict — 本次试验已建议的所有参数，如 {'lr': 0.001, 'n_layers': 3}
                      #        注意：只有被 suggest_* 调用过的参数才会出现在这里
trial.values          # list[float] — 试验返回值（多目标优化时有多个值）
trial.datetime_start  # datetime — 试验开始时间
trial.state           # TrialState — 试验状态，可能的值：
                      #   COMPLETE（正常完成）
                      #   PRUNED（被剪枝）
                      #   FAIL（抛出异常）
                      #   RUNNING（正在进行，仅多进程场景可见）
trial.user_attrs      # dict — 用户自定义属性（只读访问，用 set_user_attr 设置）
trial.intermediate_values  # dict[int, float] — 所有中间报告值，如 {0: 0.45, 1: 0.62, 2: 0.71}
```

### report(value, step) — 报告中间结果

告诉 Optuna 当前训练到了第几步、效果如何。这是**剪枝的前提**。

```python
trial.report(value, step)
# value: float — 当前步的评估指标（准确率、损失值等）
# step: int  — 当前步编号（epoch 编号、fold 编号等），必须单调递增
```

**调用规则**：
- `step` 必须**严格单调递增**（如 0, 1, 2, ...），不能倒退或重复
- 通常每个 epoch 或每个 CV fold 结束时调用一次
- 报告的值会在 Optuna Dashboard 的"中间值"图表中展示

**完整示例**：
```python
def objective(trial):
    model = create_model(trial)
    for epoch in range(50):
        train(model, train_loader)
        val_acc = evaluate(model, val_loader)

        trial.report(val_acc, epoch)          # ← 报告当前 epoch 的验证准确率

        if trial.should_prune():              # ← 基于报告的值判断是否剪枝
            raise optuna.TrialPruned()

    return val_acc  # 最终返回值
```

### should_prune() → bool — 检查是否应该剪枝

询问 Pruner："当前 trial 的表现是否明显差于同期其他 trial？"

```python
if trial.should_prune():
    raise optuna.TrialPruned()
```

**工作机制**：
1. `should_prune()` 调用 Pruner（如 MedianPruner）
2. Pruner 查看所有已完成 trial 在同一步（step）的表现
3. 如果当前 trial 显著落后，返回 `True`

**注意事项**：
- 调用前**必须**先调用 `report()`，否则无法判断
- 如果没有配置 Pruner，`should_prune()` **永远返回 `False`**
- 抛出 `TrialPruned` 后，该 trial **不参与** `study.best_trial` 的竞争
- 性能开销极小（通常 < 1ms）

### set_user_attr(key, value) — 存储自定义属性

在 trial 对象上附加任意 Python 对象（会随 Study 一起持久化）。

```python
# 存储任意数据
trial.set_user_attr('model_state', model.state_dict())
trial.set_user_attr('train_time', elapsed_seconds)
trial.set_user_attr('note', '使用了较大的 batch size')
trial.set_user_attr('gpu_memory_mb', torch.cuda.max_memory_allocated() / 1024**2)

# 之后通过 trial.user_attrs 字典读取
model_state = trial.user_attrs['model_state']
```

**常用场景**：
- 在 callback 中保存最佳模型的权重
- 记录训练耗时、GPU 内存等元信息
- 标记试验的特殊条件（方便后续分析）

### FixedTrial — 用于调试的"假" Trial

当你只想**测试目标函数是否写对**（不想真的跑 Optuna 优化），可以用 `FixedTrial` 手动注入参数：

```python
from optuna.trial import FixedTrial

# 创建一个"假"的 trial，手动指定所有参数值
params = {
    'lr': 0.001,
    'n_layers': 3,
    'optimizer': 'Adam',
}
trial = FixedTrial(params)

# 用这个假 trial 测试目标函数
result = objective(trial)
print(f"用固定参数跑出的结果: {result}")
```

`FixedTrial` 不会真的参与优化——它只是让你能独立调用 `objective(trial)` 来验证逻辑是否正确。

### TrialPruned 异常

```python
raise optuna.TrialPruned()
```

这不是 bug，而是**正常控制流**——告诉 Optuna "这个 trial 被剪枝了，请跳到下一个"。Optuna 会捕获它并正确处理。

> **不要**用 `return` 来提前终止——那会让 Optuna 误认为这个 trial 正常完成了，得到一个不完整的结果。

## 1.5 运行优化

```python
study.optimize(objective, n_trials=100)
```

### optimize() 完整参数列表

| 参数 | 类型 | 说明 |
|------|------|------|
| `func` | `callable` | 目标函数，签名为 `func(trial) -> float` 或 `func(trial) -> tuple`（多目标） |
| `n_trials` | `int` | 要运行的试验次数，默认 `None`（与 `timeout` 至少指定一个） |
| `timeout` | `float` | 超时时间（秒），与 `n_trials` 至少指定一个 |
| `n_jobs` | `int` | 并行进程数（-1 表示使用所有 CPU），仅单机有效 |
| `catch` | `tuple[Exception]` | 捕获这些异常并标记 trial 为 FAIL，然后继续下一个 trial |
| `callbacks` | `list[callable]` | 回调函数列表，在每次 trial 开始/结束时调用 |
| `gc_after_trial` | `bool` | 每次 trial 后运行 `gc.collect()`，对 GPU 训练有帮助（默认 `False`） |
| `show_progress_bar` | `bool` | 是否显示进度条（默认 `True`） |

### 常用配置

```python
# 按次数运行
study.optimize(objective, n_trials=100)

# 按时长运行
study.optimize(objective, timeout=3600)  # 运行最多 1 小时

# 次数 + 时长（满足其一即停）
study.optimize(objective, n_trials=1000, timeout=7200)

# 捕获异常，避免偶然错误中断整个优化
study.optimize(objective, n_trials=100, catch=(ValueError, RuntimeError))

# 多进程加速（同一台机器）
study.optimize(objective, n_trials=100, n_jobs=4)

# 每次 trial 后清理 GPU 内存
study.optimize(objective, n_trials=100, gc_after_trial=True)
```

### Callback 回调

Callback 让你在每次 trial 的关键时刻执行自定义逻辑（如保存最佳模型、记录到 W&B）：

```python
def logging_callback(study, trial):
    """每次 trial 结束后被调用"""
    if trial.state == optuna.trial.TrialState.COMPLETE:
        print(f"Trial {trial.number}: value={trial.values[0]:.4f}, params={trial.params}")
    elif trial.state == optuna.trial.TrialState.PRUNED:
        print(f"Trial {trial.number}: 被剪枝 (step={max(trial.intermediate_values.keys())})")

def save_best_callback(study, trial):
    """当找到新的最佳结果时保存模型"""
    if study.best_trial.number == trial.number:
        torch.save(trial.user_attrs['model_state'], 'best_model.pth')
        print(f"✅ 新最佳！Trial {trial.number}, value={trial.values[0]:.4f}")

study.optimize(
    objective,
    n_trials=100,
    callbacks=[logging_callback, save_best_callback]
)
```

## 1.7 Study 对象常用 API 参考

`Study` 是管理整个优化会话的对象。以下是你需要知道的常用属性和方法。

### 最佳结果相关

```python
study.best_value        # float — 最佳目标值
study.best_params       # dict — 最佳参数，如 {'lr': 0.001, 'n_layers': 3}
study.best_trial        # FrozenTrial — 最佳 Trial 对象，可访问 .number, .params, .values 等
```

> **多目标优化**：`study.best_params` 返回**第一个目标的**最佳解。多目标场景请用 `study.best_trials` 获取 Pareto 前沿。

### 遍历所有 Trial

```python
# 获取所有已完成的 trial
for trial in study.trials:
    print(f"Trial {trial.number}: value={trial.values}, params={trial.params}")

# 按状态筛选
completed_trials = study.get_trials(states=[optuna.trial.TrialState.COMPLETE])
pruned_trials    = study.get_trials(states=[optuna.trial.TrialState.PRUNED])
failed_trials    = study.get_trials(states=[optuna.trial.TrialState.FAIL])

# 获取并排序
trials = study.trials
best_10 = sorted(trials, key=lambda t: t.values[0])[:10]  # 前 10 名
```

### 手动添加 Trial（告诉 Optuna 已知的好参数）

当你已经有了一些先验知识（如之前跑过的实验参数），可以用 `enqueue_trial` 或 `tell` 方法告诉 Optuna，让它从"有经验"的状态开始搜索。

#### enqueue_trial — 插入预定义参数

```python
# 在 optimize 之前，告诉 Optuna "请先试试这些参数"
study.enqueue_trial({'lr': 0.001, 'n_layers': 3, 'optimizer': 'Adam'})
study.enqueue_trial({'lr': 0.01, 'n_layers': 2, 'optimizer': 'SGD'})
study.optimize(objective, n_trials=100)
# Optuna 会先运行这 2 组预设参数，再继续用 TPE 采样
```

> **谁来决定下一组参数？** 调用 `study.enqueue_trial(params)` 时，你直接指定参数值。Optuna 把这些 trial 加入到队列中，优先执行它们。队列清空后，Optuna 恢复使用 Sampler（如 TPE）自动采样。

#### tell — 手动报告已知结果

更底层的 API，让你完全绕过 `optimize()` 来添加结果：

```python
# 在 optimize() 之前或之后都可以使用
study = optuna.create_study(direction='minimize')

# 告诉 Optuna："参数 {'x': 1.0} 的结果是 5.0"
trial = study.ask()                                    # 获取一个新的 trial
study.tell(trial, values=5.0)                          # 手动报告结果
# 或指定参数
trial = study.ask(fixed_distributions={'x': optuna.distributions.FloatDistribution(-5, 5)})
study.tell(trial, states=optuna.trial.TrialState.COMPLETE, values=4.0)
```

**使用场景**：分布式优化时，worker 从 master 获取 trial（`ask`），完成后报告结果（`tell`）。

study = optuna.create_study(

### create_study 完整参数参考

```python
study = optuna.create_study(
    study_name='my_study',           # str — Study 名称（持久化时需要）
    storage=None,                    # str — 存储后端 URL（None = 内存）
    direction='minimize',            # str — 'minimize' 或 'maximize'
    directions=['maximize', 'minimize'],  # list[str] — 多目标优化方向
    sampler=None,                    # BaseSampler — 采样器（默认 TPESampler）

)
```

## 1.9 本章小结

- Optuna 的核心是 **Study + Trial + Objective**
- `trial` 对象由 Optuna 自动创建并传入目标函数，你通过它获取参数建议和报告训练进度
- 使用 `suggest_float` / `suggest_int` / `suggest_categorical` 定义搜索空间
- 使用 `trial.report()` + `trial.should_prune()` 实现剪枝，用 `raise TrialPruned()` 提前终止
- `trial.set_user_attr()` 可存储模型权重等自定义数据
- `study.optimize()` 启动优化，支持 `callbacks`、`n_jobs`、`catch` 等参数
- 通过 `study.best_params` / `study.trials` / `study.trials_dataframe()` 获取和分析结果
- `study.enqueue_trial()` 可注入先验知识，`FixedTrial` 可用于调试目标函数
