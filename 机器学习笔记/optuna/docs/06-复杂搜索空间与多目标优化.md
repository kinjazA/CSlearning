## 6.1 条件参数（Conditional Parameters）

真实场景中，超参数之间往往有依赖关系：
- 选 Adam → 需要 beta1, beta2
- 选 SGD → 需要 momentum
- 选 CNN → 需要 kernel_size
- 选 RNN → 需要 num_layers, hidden_size

### 在 Optuna 中实现条件参数

```python
def objective(trial):
    optimizer_name = trial.suggest_categorical('optimizer', ['Adam', 'SGD'])

    lr = trial.suggest_float('lr', 1e-5, 1e-1, log=True)

    if optimizer_name == 'Adam':
        beta1 = trial.suggest_float('beta1', 0.5, 0.99)
        beta2 = trial.suggest_float('beta2', 0.9, 0.999)
        optimizer = optim.Adam(model.parameters(), lr=lr, betas=(beta1, beta2))
    else:  # SGD
        momentum = trial.suggest_float('momentum', 0.0, 0.99)
        optimizer = optim.SGD(model.parameters(), lr=lr, momentum=momentum)

    # ... 训练 ...
```

### Optuna 如何处理条件参数？

Optuna 的 TPE 采样器天生支持条件参数。它会：
1. 为每个条件分支维护独立的 $l(x)$ 和 $g(x)$
2. 只在相关的分支中采样和建模

这意味着当 optimizer='Adam' 时，momentum 参数**不会被采样**，也不会影响模型。

## 6.3 多目标优化

当需要同时优化**多个互相冲突**的目标时：
- 准确率 vs 推理速度
- 准确率 vs 模型大小
- F1-score vs 训练时间

### 创建多目标 Study

```python
study = optuna.create_study(
    directions=['maximize', 'minimize'],  # 最大化准确率，最小化推理时间
    study_name='multi_objective'
)
```

### 目标函数返回多个值

```python
def objective(trial):
    model = create_model(trial)
    accuracy = train_and_evaluate(model)
    inference_time = measure_inference_time(model)
    return accuracy, inference_time
```

### Pareto 前沿

多目标优化没有单一的"最佳"解，而是有一组 **Pareto 最优解**：
- 无法在不牺牲一个目标的情况下改进另一个目标

```python
# 获取 Pareto 前沿
pareto_trials = study.best_trials

for trial in pareto_trials:
    print(f"准确率: {trial.values[0]:.4f}, 推理时间: {trial.values[1]:.4f}")
    print(f"参数: {trial.params}")
```

### 可视化 Pareto 前沿

```python
import matplotlib.pyplot as plt

accuracies = [t.values[0] for t in study.best_trials]
times = [t.values[1] for t in study.best_trials]

plt.scatter(times, accuracies)
plt.xlabel('Inference Time')
plt.ylabel('Accuracy')
plt.title('Pareto Front')
plt.show()
```

## 6.4 约束优化

有时你需要满足某些硬约束：
- 模型参数量 < 1M
- 推理时间 < 10ms
- 内存占用 < 100MB

### 方式一：在目标函数中返回惩罚值

```python
def objective(trial):
    model = create_model(trial)
    accuracy = evaluate(model)
    params = count_parameters(model)

    # 如果参数量超标，给一个很大的惩罚
    if params > 1_000_000:
        return 0.0  # 假设准确率范围是 0-1

    return accuracy
```

### 方式二：使用约束优化（Optuna 3.0+）

```python
def objective(trial):
    model = create_model(trial)
    accuracy = evaluate(model)
    params = count_parameters(model)
    time_ms = measure_time(model)

    # 设置约束：params <= 1M, time <= 10ms
    trial.set_user_attr('constraint', (params <= 1_000_000, time_ms <= 10))

    return accuracy

study = optuna.create_study(
    direction='maximize',
    sampler=optuna.samplers.TPESampler(constraints_func=lambda trial: [
        0 if trial.user_attrs['constraint'][0] else 1,  # 0=满足约束
        0 if trial.user_attrs['constraint'][1] else 1,
    ])
)
```
