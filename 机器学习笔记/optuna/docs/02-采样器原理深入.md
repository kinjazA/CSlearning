# 第2章：采样器原理深入

## 本章目标
理解 Optuna 如何选择下一组超参数，深入掌握 TPE 算法原理，学会在不同场景选择合适的采样器。

---

## 2.1 为什么智能采样很重要？

假设你要优化 5 个超参数，每个参数有 10 个候选值：
- **网格搜索**：需要 $10^5 = 100{,}000$ 次试验，不可行
- **随机搜索**：100 次试验可能错过好区域
- **贝叶斯优化**：用前 N 次的结果指导第 N+1 次的选择，**用更少试验找到更优解**

---

## 2.2 三种搜索策略对比

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **网格搜索** | 遍历所有组合 | 简单、可复现 | 维度灾难、效率极低 |
| **随机搜索** | 随机采样 | 实现简单、比网格搜索好 | 无记忆，重复探索 |
| **贝叶斯优化** | 用历史数据建模，选择最有希望的点 | 样本效率高 | 计算开销略大 |

---

## 2.3 TPE（Tree-structured Parzen Estimator）详解

TPE 是 Optuna 的**默认采样器**，也是它效果好的核心原因。

### 2.3.1 两种建模思路：p(y|x) vs p(x|y)

贝叶斯优化的目标是：用已有的 N 次试验结果，决定第 N+1 次该试什么参数。

**传统方法（高斯过程 GP）**：直接建模 **p(y|x)**——"给定参数 x，预测性能 y 是多少"。这很直观，但 GP 有个痛点：它的核函数是为连续空间设计的，在**混合类型空间**（连续 + 离散 + 条件参数）中表现不佳。而且计算复杂度是 O(n³)，trial 多了会越来越慢。

**TPE 的思路**：反过来建模 **p(x|y)**——"给定性能水平 y，参数 x 呈现出什么样的分布"。这带来两个关键优势：

| 维度 | GP（建模 p(y\|x)） | TPE（建模 p(x\|y)） |
|---|---|---|
| 核心问题 | "参数 x=0.01 时，准确率大概多少？" | "准确率 > 0.9 的那些 trial，它们的参数长什么样？" |
| 连续参数 | ✅ 高斯核，效果好 | ✅ 用一维 KDE 建模 |
| 离散参数 | ❌ 不好处理 | ✅ 用计数 + 平滑 |
| 条件参数 | ❌ 需要特殊处理 | ✅ 天然支持（树结构） |
| 计算复杂度 | O(n³) | O(n·d)（d 为参数维度） |

### 2.3.2 核心操作：用分位数把历史数据切成两组

TPE 的第一步：把所有已完成的 trial 按表现分成"好组"和"差组"。

设 $y^*$ 为当前观测值的某个**分位数阈值**。默认取 $γ = 0.25$，即结果排前 25% 的是"好组"，其余是"差组"：

```
已完成 trial（按结果从小到大排列，越小越好）：
[0.45, 0.52, 0.55, 0.61, 0.63, 0.68, 0.71, 0.73, 0.82, 0.91]
  └─── 前 25% ──┘
  y* = 0.55（第 25 百分位）

好组（y < y*）：trial 0.45, 0.52  → 建模为 l(x) — "好参数的分布"
差组（y ≥ y*）：trial 0.55~0.91    → 建模为 g(x) — "差参数的分布"
```

于是有了 TPE 的核心公式：

$$p(x|y) = \begin{cases} l(x) & \text{if } y < y^* \\ g(x) & \text{if } y \geq y^* \end{cases}$$

> **γ 的作用**：γ 越小 → 好组越严格 → 更激进地"利用"已知好区域。γ 越大 → 好组越宽松 → 更多"探索"。默认 0.25（前 25%）是经验上平衡探索与利用的好选择。

### 2.3.3 用 Parzen 窗估计 l(x) 和 g(x)

"Parzen Estimator" 的名字来源于 **Parzen 窗**（Parzen Window），即**核密度估计**（Kernel Density Estimation, KDE）。

#### 连续参数：一维 KDE

对于每个连续参数（如学习率），TPE 在两组数据上分别拟合一维 KDE：

$$l(x) = \frac{1}{n_l} \sum_{x_i \in \text{好组}} k(x, x_i)$$

$$g(x) = \frac{1}{n_g} \sum_{x_j \in \text{差组}} k(x, x_j)$$

其中 $k(x, x_i)$ 是核函数（通常是**截断高斯核**，以观测点 $x_i$ 为中心），它在一个观测点周围放一个"小山包"。把所有山包叠加起来，就得到了概率密度曲线：

```
概率密度
  ↑
  │                  l(x)：好参数集中在某个区域
  │              ▄▄█▄▄
  │            ███████
  │          ███████████
  │
  │   g(x)：差参数散落在各处
  │   ▂▁▂▁  ▁▂▁▂▁  ▂▁▂▁▂
  │
  └──────────────────────────→ 参数值 x
```

**关键细节**：TPE 使用的核函数**以观测点为中心，带宽由最近邻距离自适应调整**——这意味着在数据密集的区域核更窄（更精确），在数据稀疏的区域核更宽（更平滑）。

#### 离散参数：带平滑的计数

对于 `suggest_categorical` 定义的离散参数，TPE 用**计数 + 平滑**来估计概率：

$$p(x = v) = \frac{\text{count}(v) + \alpha}{\text{total} + \alpha \cdot |\text{choices}|}$$

其中 α 是一个很小的平滑系数（默认约 1e-12），作用只是避免出现零概率。平滑系数极小意味着：如果一个值从未在好组中出现过，它的概率接近 0，几乎不会被采样。

#### 为什么分别建模每个参数？

默认情况下，TPE 对每个参数**独立**建模（单变量 KDE）。这意味着它不考虑参数间的相关性（如"学习率大时 batch size 也应该大"）。如果需要建模相关性，可以开启 `multivariate=True`，TPE 会用多维 KDE，但计算量会增大。

### 2.3.4 选择下一组参数：最大化 l(x)/g(x)

有了 l(x) 和 g(x) 之后，问题变成：**下一步该试哪个参数 x？**

根据贝叶斯优化的 **Expected Improvement（EI）** 准则，我们希望最大化期望提升：

$$\text{EI}_{y^*}(x) = \int_{-\infty}^{\infty} \max(y^* - y, 0) \cdot p(y|x) \, dy$$

经过贝叶斯公式 $p(y|x) = \frac{p(x|y) \cdot p(y)}{p(x)}$ 代入和化简（这里跳过推导），EI 可以表示为：

$$\text{EI}_{y^*}(x) \propto \left( \gamma + \frac{g(x)}{l(x)} \cdot (1 - \gamma) \right)^{-1}$$

关键结论：**l(x) 越大、g(x) 越小 → EI 越大**。因此在实际操作中，TPE 直接通过最大化 l(x)/g(x) 来选择下一组参数。

> **一句话直觉**：在"好参数频繁出现"且"差参数很少出现"的参数空间区域采样。

### 2.3.5 一次完整的 TPE 迭代（Step by Step）

假设你已经跑了 20 个 trial，现在 Sampler 要为第 21 个 trial 生成参数：

```
Step 1 — 确定分位数阈值
  取 γ = 0.25，按目标值排序，y* = 第 ⌊20×0.25⌋ = 第 5 名 trial 的值

Step 2 — 分组建模
  好组（前 5 名）：对每个超参数分别拟合 l(x) KDE
  差组（后 15 名）：对每个超参数分别拟合 g(x) KDE

Step 3 — 生成候选点
  从 l(x) 分布中随机采样 N 个候选参数组合（n_ei_candidates=24）
  注意：是从 l(x) 采样，不是从整个搜索空间均匀采样
        —— 这保证了候选点本身就是"看起来不错"的参数

Step 4 — 评分
  对每个候选点 x，计算 l(x)/g(x)
  选比值最大的那个作为第 21 个 trial 的参数

Step 5 — 返回并等待反馈
  把参数返回给 Objective 函数，训练完成后把结果加入历史
  有了 21 个数据点，下一次迭代的模型就更准了
```

**为什么从 l(x) 采样而不是从 g(x)？** 因为 l(x) 已经标记了"好区域"，从 l(x) 采样 → 候选点大概率落在好区域 → 评分阶段再用 g(x) 过滤掉那些"好区域中差参数也常见"的点 → 最终拿到的是**好参数常见但差参数少见的区域**。

### 2.3.6 为什么叫 "Tree-structured"？

超参数之间经常存在**条件依赖**。例如：

```
optimizer = Adam               optimizer = SGD
├── lr (log-uniform)           ├── lr (log-uniform)
├── beta1 (0.5~0.99)           └── momentum (0~0.99)
└── beta2 (0.9~0.999)
```

当 `optimizer=SGD` 时，`beta1` 和 `beta2` **不存在**——它们不参与这个 trial 的参数空间。

TPE 用树结构来处理这一点：

```
根节点: optimizer
├── 分支 Adam
│   ├── lr    →  独立的 l_adam(x) / g_adam(x)
│   ├── beta1 →  独立的 l_adam(x) / g_adam(x)
│   └── beta2 →  独立的 l_adam(x) / g_adam(x)
│
└── 分支 SGD
    ├── lr       →  独立的 l_sgd(x) / g_sgd(x)
    └── momentum →  独立的 l_sgd(x) / g_sgd(x)
```

- 每个条件分支维护**自己独立的** l(x) 和 g(x)
- `optimizer=SGD` 的分支只建模 `lr` 和 `momentum`，完全不知道 beta1/beta2 的存在
- 当某个分支的 trial 数量很少时，TPE 会借用父节点或其他分支的信息（层次化先验）

这对应你的代码中的**条件参数**模式——用 `if` 语句自然表达依赖关系：

```python
def objective(trial):
    optimizer = trial.suggest_categorical('optimizer', ['Adam', 'SGD'])
    # ↑ Optuna 自动识别这是条件分支的根节点
    if optimizer == 'Adam':
        beta1 = trial.suggest_float('beta1', 0.5, 0.99)
        # ↑ 只在 Adam 分支中建模 l(beta1)/g(beta1)
    else:
        momentum = trial.suggest_float('momentum', 0.0, 0.99)
        # ↑ 只在 SGD 分支中建模 l(momentum)/g(momentum)
```

### 2.3.7 TPE 的局限与参数调优

| 局限 | 原因 | 应对策略 |
|------|------|---------|
| 初始阶段等同于随机搜索 | 前 n_startup_trials 次没有数据建 KDE | 如果搜索空间大，适当增大 `n_startup_trials` |
| 高维连续空间（>20 维）效率下降 | 维数 = 连续超参数的个数，如 lr, dropout, weight_decay... 共 >20 个。高维空间数据稀疏，KDE 很难准确估计密度。注意：离散参数（categorical）不受此影响 | 换成 CMA-ES（专门为连续高维设计） |
| 默认不考虑参数间的交互效应 | 每个参数独立建模 | 开启 `multivariate=True`（但会增加计算量） |
| 对噪声敏感 | 噪声大的结果会误导好坏分组 | 增大 γ（放宽"好组"门槛），或用 CMA-ES |
| 离散参数无法表达排序信息 | 计数方法把 `[64, 128, 256]` 当作无序类别 | 对有序离散值用 `suggest_int` 代替 `suggest_categorical` |

**TPESampler 关键参数速查**：

```python
from optuna.samplers import TPESampler

sampler = TPESampler(
    n_startup_trials=10,    # 前 N 次随机探索（收集初始数据）
    n_ei_candidates=24,     # 每轮从 l(x) 中采样的候选数，越大越准但越慢
    multivariate=False,     # True=用多维 KDE 建模参数相关性（计算量大）
    group=False,            # True=将有关联的参数分组建模
    gamma=0.25,             # 隐式参数，好组的分位数（通过内部计算）
    seed=None,              # 随机种子，保证可复现
)
```

## 2.4 Optuna 内置采样器

| 采样器 | 类名 | 适用场景 |
|--------|------|----------|
| **TPE** | `TPESampler` | 通用首选，大多数情况最佳 |
| **CMA-ES** | `CmaEsSampler` | 连续参数、高维空间（>10维） |
| **随机** | `RandomSampler` | 基线对比、搜索空间很大时 |
| **网格** | `GridSampler` | 需要穷举少量组合 |
| **QMC** | `QMCSampler` | 准蒙特卡洛，比随机更均匀 |
| **Brute Force** | `BruteForceSampler` | 小空间穷举 |

### 使用方式

```python
from optuna.samplers import TPESampler, CmaEsSampler, RandomSampler

# 默认就是 TPE
study = optuna.create_study(direction='minimize')

# 显式指定
study = optuna.create_study(
    direction='minimize',
    sampler=TPESampler(n_startup_trials=10)  # 前10次随机探索
)
```

### TPESampler 关键参数

- `n_startup_trials`：前 N 次用随机搜索（默认10），收集初始数据
- `n_ei_candidates`：每次采样的候选数（默认24），越大越准但越慢
- `multivariate=True`：考虑参数间的相关性（默认False）
- `seed`：随机种子，保证可复现

## 2.6 最佳实践

1. **默认用 TPE**，不要无脑换
2. **增加 `n_startup_trials`** 如果搜索空间非常大或评估很 noisy
3. **开启 `multivariate=True`** 当参数间有明显依赖（如学习率和 batch size）
4. **先用 RandomSampler 跑基线**，确认你的 objective 函数没问题
5. **设置 seed** 保证实验可复现

---

## 2.7 本章小结

- TPE 通过建模 $l(x)/g(x)$ 实现高效采样
- 前 `n_startup_trials` 次随机探索，之后用贝叶斯指导
- 默认 TPE 适合绝大多数场景，高维连续空间考虑 CMA-ES
