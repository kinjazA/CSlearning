# 第 03 章 · 压缩策略

> **难度**：⭐⭐ | **核心参数**：`compress` | **一句话**：选对压缩算法和级别，省磁盘、省网络带宽、几乎不牺牲速度

---

## 3.1 解决什么问题？

第 02 章你学会了用 `compress=3` 存模型。但实际工作里你会碰到不同的场景：

- "模型文件 5GB，上传到模型仓库要 10 分钟" → 能不能更小？
- "服务启动加载模型要等 3 秒，影响冷启动时间" → 能不能更快？
- "1000 个超参组合的结果要存下来分析，磁盘快满了" → 怎么平衡？

本章回答一个问题：**在不同场景下，`compress` 参数该怎么选。**

---

## 3.2 可用的压缩算法一览

joblib 支持 6 种压缩算法。不需要全记住，理解它们各自适合什么场景就行：

| 算法 | 一句话 | 速度 | 压缩率 | 需要额外安装？ |
|------|--------|------|--------|---------------|
| `zlib` | **默认，什么都不要装** | ⭐⭐⭐ | ⭐⭐⭐ | ❌ Python 内置 |
| `lz4` | 极致速度，几乎不花时间 | ⭐⭐⭐⭐⭐ | ⭐⭐ | `uv add lz4` |
| `zstd` | 最佳全能选手 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | `uv add zstandard` |
| `lzma` | 压到最小，不介意慢 | ⭐ | ⭐⭐⭐⭐⭐ | ❌ Python 内置 |
| `gzip` | 兼容性之王，任何系统都能解 | ⭐⭐ | ⭐⭐⭐ | ❌ Python 内置 |
| `bz2` | 已被 zstd 淘汰 | ⭐ | ⭐⭐⭐ | ❌ Python 内置 |

### 3.2.1 使用方式

```python
# 方式 1：数字 → 默认用 zlib
joblib.dump(obj, 'f.joblib', compress=0)   # 不压缩
joblib.dump(obj, 'f.joblib', compress=3)   # zlib level 3（推荐默认）
joblib.dump(obj, 'f.joblib', compress=9)   # zlib level 9

# 方式 2：元组 → (算法, 级别)
joblib.dump(obj, 'f.joblib', compress=('zstd', 3))
joblib.dump(obj, 'f.joblib', compress=('lz4', 0))
joblib.dump(obj, 'f.joblib', compress=('lzma', 9))
```

## 3.3 各算法怎么选——场景决策

### 场景 1：日常开发，频繁读写

```python
joblib.dump(model, 'model.joblib', compress=3)
```

**为什么**：zlib level 3 是默认推荐。写入快、读取快、体积省一半。你什么额外包都不需要装。

### 场景 2：生产服务，启动速度敏感

```python
joblib.dump(model, 'model.joblib', compress=('lz4', 0))
```

**为什么**：服务冷启动时要加载模型，lz4 解压几乎不花时间——比 zlib 快 3-5 倍。代价是文件大 20-30%。对在线服务来说，加载速度比磁盘空间重要得多。

### 场景 3：有 zstd，追求最优平衡

```python
joblib.dump(model, 'model.joblib', compress=('zstd', 3))
```

**为什么**：如果你装了 `zstandard`，zstd 是当前最好的选择——解压速度和 lz4 接近，压缩率逼近 lzma。同时满足"写得快"和"文件小"。

### 场景 4：长期归档，一年读不了一次

```python
joblib.dump(model, 'model.joblib', compress=('lzma', 9))
```

**为什么**：lzma 压缩率最高（文件可以缩小 70-90%），但很慢。对一年才加载一次的归档模型来说，体积比速度重要。

### 场景 5：跨平台分发，要保证任何环境都能解

```python
joblib.dump(model, 'model.joblib', compress=('gzip', 3))
```

**为什么**：gzip 是 Linux 标配，任何机器上的 `gunzip` 都能解压。文件需要发给不确定环境的同事时，gzip 最保险。

## 3.4 level 怎么选——花多少 CPU 换多少空间

所有压缩算法都有一个 `level` 参数（0-9），控制压缩时投入多少计算量去"找规律"：

```
level 0  → 不压缩或几乎不压缩（原始体积）
level 1  → 快速扫一遍，找到明显的重复模式
level 3  → 平衡点 ← joblib 默认
level 9  → 反复搜索最优压缩策略，CPU 开销大
```

**关键规律**：level 1→3 提升明显，3→9 事倍功半。

举个实际例子（随机 float64 数组，压缩效果最差的情况）：

| level | 写入耗时 | 文件大小 | 压缩率 | 相对 level 0 的慢倍数 |
|-------|---------|---------|--------|---------------------|
| 0 | 0.05s | 80 MB | 100% | 1× |
| 1 | 0.12s | 72 MB | 90% | 2.4× |
| **3** | **0.15s** | **45 MB** | **56%** | **3×** |
| 5 | 0.22s | 42 MB | 52% | 4.4× |
| 7 | 0.35s | 40 MB | 50% | 7× |
| 9 | 0.52s | 38 MB | 48% | 10× |

**结论**：level 3 之后，慢很多，省很少。除非你是归档场景，否则别设 9。

如果是**有规律的数据**（稀疏矩阵、one-hot 编码），压缩效果会好得多——level 3 就能压到 10-20% 的体积。

## 3.5 压缩对数据内容的影响

### 3.5.1 不同数据类型的压缩效果差异巨大

| 数据类型 | 压缩效果 | 原因 |
|:-------:|:-------:|:----:|
| 随机浮点数 | 差（50-70%） | 没有重复模式，接近"噪声" |
| 零值为主的稀疏矩阵 | 极好（1-10%） | 大量连续零 → 压缩算法轻松处理 |
| 整数特征 | 好（20-40%） | 取值范围小，重复多 |
| 模型参数（树模型） | 中等（30-50%） | 阈值和叶节点值有一定规律 |
| 深度学习权重 | 中等（40-60%） | 权重分布接近正态，规律有限 |

### 3.5.2 压缩不影响数值精度

压缩是**无损**的。加载回来和保存前完全一致，不会有精度损失。这和 JPEG 压缩图片不同。

---

## 3.6 压缩 vs 不压缩：什么时候别压缩？

| 情况 | 建议 |
|------|------|
| 文件 < 10 MB | 不用压缩，开销不值得 |
| 频繁读写（一分钟几十次） | 不压缩或用 lz4 |
| 已经压缩过的数据（图片、视频） | 不再压缩，浪费时间 |
| 需要 mmap 零拷贝加载 | 不压缩（压缩后的文件不支持 mmap） |

**特别注意**：`compress != 0` 的文件不能用 `mmap_mode` 加载。因为压缩后的数据必须先解压到内存，无法直接映射。如果你需要 mmap，必须 `compress=0`。

```python
# ❌ 压缩 + mmap 不兼容
joblib.dump(data, 'f.joblib', compress=3)
data = joblib.load('f.joblib', mmap_mode='r')  # mmap_mode 被忽略，仍然全部加载

# ✅ 要么不压缩用 mmap
joblib.dump(data, 'f.joblib', compress=0)
data = joblib.load('f.joblib', mmap_mode='r')

# ✅ 要么压缩不用 mmap
joblib.dump(data, 'f.joblib', compress=('zstd', 3))
data = joblib.load('f.joblib')  # 正常加载
```

## 3.7 快速决策卡

以后选 compress 参数，直接看这张表：

```
我的场景：                               → 用什么
─────────────────────────────────────────────────────
日常开发，什么都不想操心                    → compress=3
要最快速度（服务冷启动）                    → compress=('lz4', 0)
装了 zstd，追求最优                        → compress=('zstd', 3)
要最小体积，不介意慢（归档）                → compress=('lzma', 9)
需要 mmap 零拷贝加载                       → compress=0
大数组需要 mmap + 部分内容需压缩             → 分别保存：数组 compress=0，其余 compress=3
文件要发给不确定环境的同事                   → compress=('gzip', 3)
```

## 3.8 本章要点

- [ ] `compress=3` 日常用，什么都不需要装
- [ ] `compress=('zstd', 3)` 最佳全能（`uv add zstandard` 后）
- [ ] `compress=('lz4', 0)` 服务冷启动首选——解压最快
- [ ] `compress=('lzma', 9)` 归档首选——体积最小
- [ ] level 选 3 就够了，3→9 事倍功半
- [ ] **压缩的文件不能用 mmap**——这是最重要的一个互斥关系
- [ ] 压缩是无损的，不影响数值精度
