# 01 `Set`接口

> * 表示一个不包含重复元素的集合。`Set` 接口继承自 `Collection` 接口，并且严格遵循数学中集合的定义：集合中的每个元素都是唯一的，即集合不允许包含重复的元素
> * **无序性**: `Set` 并不保证元素的顺序。某些 `Set` 的实现（如 `LinkedHashSet`）会保持插入顺序，而其他实现（如 `HashSet`）则不保证顺序
> * **唯一性**: `Set` 中的元素是唯一的，即集合不允许有重复的元素。如果试图将重复的元素添加到 `Set` 中，添加操作将不会成功（`add` 方法会返回 `false`）
> * **常见实现类**:
>   - **`HashSet`**: 基于哈希表实现的 `Set`，不保证元素的顺序
>   - **`LinkedHashSet`**: 保持元素插入的顺序，同时保持 `HashSet` 的性能特性
>   - **`TreeSet`**: 基于红黑树实现的 `Set`，保证元素按自然顺序或自定义的比较器顺序排序

## 01.1 常用方法

> * `Set` 接口继承自 `Collection` 接口，因此它包含 `Collection` 中的所有方法

添加元素

> * **`boolean add(E e)`**: 向集合中添加一个元素。如果集合因为调用此方法而发生改变（即添加了新元素），则返回 `true`，否则返回 `false`。如果试图添加的元素已经存在于集合中，则 `Set` 不会改变，并返回 `false`

```java
Set<String> set = new HashSet<>();
set.add("Apple");  // 返回 true
set.add("Banana");  // 返回 true
set.add("Apple");  // 返回 false，因为 "Apple" 已存在
```

> * **`boolean addAll(Collection<? extends E> c)`**: 将指定集合中的所有元素添加到 `Set` 中。如果由于这些元素的加入而使得 `Set` 发生了改变（即 `Set` 增加了新元素），则返回 `true`，否则返回 `false`

```java
Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");
List<String> list = Arrays.asList("Cherry", "Banana");
set.addAll(list);  // 返回 true, 因为 "Cherry" 是新元素
```

删除元素

> * **`boolean remove(Object o)`**: 如果集合中存在指定的元素，则将其移除，并返回 `true`；否则返回 `false`

```java
Set<String> set = new HashSet<>();
set.add("Apple");
boolean removed = set.remove("Apple");  // 返回 true
removed = set.remove("Banana");  // 返回 false，因为 "Banana" 不在集合中
```

> * **`void clear()`**: 清空集合中的所有元素，使其变为空集合

```java
Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");
set.clear();  // 清空集合
```

查询元素

> * **`boolean contains(Object o)`**: 如果集合中包含指定的元素，则返回 `true`

```java
Set<String> set = new HashSet<>();
set.add("Apple");
boolean containsApple = set.contains("Apple");  // 返回 true
```

> * **`int size()`**: 返回集合中的元素数量

```java
Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");
int size = set.size();  // 返回 2
```

> * **`boolean isEmpty()`**: 如果集合不包含任何元素，则返回 `true`

```java
Set<String> set = new HashSet<>();
boolean isEmpty = set.isEmpty();  // 返回 true
```

集合操作

> * **`boolean retainAll(Collection<?> c)`**: 仅保留集合中那些包含在指定集合中的元素，即从此集合中移除不在指定集合中的所有元素。如果 `Set` 因为此操作发生改变，则返回 `true`

```java
Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");
List<String> list = Arrays.asList("Banana", "Cherry");
set.retainAll(list);  // 只保留 "Banana"
```

> * **`boolean removeAll(Collection<?> c)`**: 从集合中移除包含在指定集合中的所有元素。如果 `Set` 因为此操作发生改变，则返回 `true`

```java
Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");
List<String> list = Arrays.asList("Banana", "Cherry");
set.removeAll(list);  // 移除 "Banana"
```

> * **`boolean containsAll(Collection<?> c)`**: 如果此集合包含指定集合中的所有元素，则返回 `true`

```java
Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");
List<String> list = Arrays.asList("Apple", "Banana");
boolean containsAll = set.containsAll(list);  // 返回 true
```

集合遍历

> * **增强 `for` 循环**: 使用增强 `for` 循环遍历 `Set`

```java
Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");

for (String fruit : set) {
    System.out.println(fruit);
}
```

> * **`Iterator<E> iterator()`**: 返回用于遍历 `Set` 中元素的迭代器

```java
Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");

Iterator<String> iterator = set.iterator();
while (iterator.hasNext()) {
    System.out.println(iterator.next());
}
```

## 01.2 应用场景

> * **去重**: 当需要存储一组唯一元素时，`Set` 是理想的选择。例如，在某个集合中存储一组用户 ID 或商品编号时，可以使用 `Set` 来自动去重
> * **数学集合操作**: `Set` 支持集合操作（如交集、并集、差集等），这些操作可以通过方法 `retainAll()`、`addAll()`、`removeAll()` 等实现
> * **快速查找**: 在一些场景下，需要快速判断某个元素是否存在于集合中，`Set` 提供了高效的 `contains()` 方法，特别是对于 `HashSet`

## 01.3 `Set` 接口与 `List` 接口比较

|    特性    | `Set`接口                                                    | `List`接口                             |
| :--------: | ------------------------------------------------------------ | :------------------------------------- |
|    定义    | 表示一组不允许重复的元素的集合                               | 表示有序且允许重复元素的集合           |
|    顺序    | 通常不保证元素的顺序。如 `LinkedHashSet` 保证插入顺序，`TreeSet` 保证排序顺序 | 保持元素的插入顺序，支持基于索引的访问 |
|   重复性   | 不允许重复元素                                               | 允许重复元素                           |
|    索引    | 不支持索引                                                   | 支持索引，可以使用索引进行增删改查     |
| 元素唯一性 | `Set` 保证每个元素的唯一性，基于元素的 `equals()` 和 `hashCode()` 方法来判断元素的相等性 | `List` 中元素可以重复，不要求元素唯一  |
